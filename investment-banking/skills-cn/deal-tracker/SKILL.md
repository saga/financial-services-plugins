# 交易跟踪器

## 工作流程

### 第一步：交易设置

对每笔交易，记录：
- **交易名称/代号**：项目 [名称]
- **客户**：卖方或买方名称
- **交易类型**：卖方、买方、融资、重组
- **角色**：牵头顾问、联席顾问、公允意见
- **交易规模**：预期企业价值
- **阶段**：委托前 → 已委托 → 营销 → IOI → 尽调 → 最终报价 → 签约 → 交割
- **团队**：MD、VP、Associate、Analyst
- **关键日期**：委托日期、CIM分发、IOI截止、管理层会议、最终报价截止、目标交割

### 第三步：行动事项

维护跨所有交易的行动事项清单：

| 行动事项 | 交易 | 负责人 | 截止日期 | 优先级 | 状态 |
|----------|------|--------|----------|--------|------|
| | | | | P0/P1/P2 | 待办/完成/受阻 |

### 第四步：每周交易评审

为每周团队会议生成摘要：

**对每个活跃交易：**
1. 一行状态更新
2. 本周关键进展
3. 即将到来的里程碑（未来2周）
4. 阻碍或风险
5. 下周行动事项

**管道摘要：**
- 按阶段的活跃交易总数
- 有风险的交易（错过里程碑、停滞的流程）
- 管道中的新委托/推介
- 本季度预期交割

### 第五步：输出

- Excel 工作簿包含：
  - 管道概述（所有交易，每笔一行）
  - 每笔交易的里程碑跟踪标签页
  - 行动事项主清单
  - 每周评审摘要
- 可选：用于邮件/Slack 分发的 Markdown 摘要

## 重要提示

- 至少每周更新跟踪器 — 过时的跟踪器比没有跟踪器更糟
- 标记里程碑滑动的交易 — 早期预警可防止意外
- 没有负责人和截止日期的行动事项不会被完成 — 要具体
- 管道视图应显示交易阶段、规模和可能性 — 对收入预测有用
- 保留买家/投资者反馈的笔记 — 反馈模式指导策略调整
- 单独归档已交割/失败的交易 — 保持活跃视图整洁
