# 财务数据包构建器

为私募股权、投资银行和资产管理构建专业、标准化的财务数据包。将CIM、推介备忘录、SEC文件、网络搜索或MCP服务器访问的财务数据转化为适合投资委员会审阅的专业Excel工作簿。

## 关键成功因素

每个数据包必须达到这些标准。任何一点的失败都会使交付物无法使用。

### 1. 数据准确性（对错误零容忍）
- 每个数字追溯到源文件并标注页码引用
- 完全使用基于公式的计算（无硬编码值）
- 交叉检查小计和总计的内部一致性
- 验证资产负债表平衡：资产 = 负债 + 所有者权益
- 确认现金流与资产负债表变动一致

### 2. 基本规则

**规则1：财务数据（衡量金额）→ 货币格式带$**
触发词：收入、销售额、收益、EBITDA、利润、亏损、成本、费用、现金、债务、资产、负债、权益、资本支出
格式：百万用 $#,##0.0，千用 $#,##0
负数：$(123.0) 而非 -$123

**规则2：运营数据（计数）→ 数字格式，无$**
触发词：单位、门店、地点、员工、客户、平方英尺、物业、人数
格式：#,##0 带逗号
负数：(123) 与表格其余部分一致

**规则3：百分比（比率和比率）→ 百分比格式**
触发词：利润率、增长、比率、百分比、收益率、回报、利用率、入住率
格式：0.0% 保留一位小数
显示：15.0% 而非 0.15

**规则4：年份 → 文本格式防止插入逗号**
格式：文本或自定义格式防止 2,024
显示：2020, 2021, 2022, 2023A, 2024E

**规则5：当上下文混合时，每个指标使用各自适当的格式**
示例：
```
分部分析, 2022, 2023, 2024
零售收入, $50.0, $55.0, $60.0
  门店数, 100, 110, 120
  单店收入, $0.5, $0.5, $0.5
```
收入和单店指标使用$，门店数使用数字格式。

**规则6：所有计算使用公式 → 永远不要硬编码计算值**
所有小计、总计、比率和派生指标必须基于公式，而非硬编码值。这确保准确性并允许动态更新。

### 3. 专业呈现标准

**格式标准：**

**颜色方案 - 两层：**

**第一层：字体颜色（必须应用）**
- **蓝色文本 (RGB: 0,0,255)**：所有硬编码输入（历史数据、假设），非普通文本
- **黑色文本 (RGB: 0,0,0)**：所有公式和计算
- **绿色文本 (RGB: 0,128,0)**：链接到其他工作表

**第二层：填充颜色（可选增强呈现）**
- 填充颜色是可选的，仅在用户请求或增强呈现时应用
- 如果用户请求颜色或专业格式，使用此标准方案：
  - **章节标题**：深蓝 (RGB: 68,114,196) 背景，白色文本
  - **副标题/列标题**：浅蓝 (RGB: 217,225,242) 背景，黑色文本
  - **输入单元格**：浅绿/奶油色 (RGB: 226,239,218) 背景，蓝色文本
  - **计算单元格**：白色背景，黑色文本

**始终应用：**
- 粗体标题，左对齐
- 数字右对齐
- 子项目2空格缩进
- 小计上方单下划线
- 最终总计下方双下划线
- 冻结行/列标题窗格
- 最少边框（仅在结构需要处）
- 一致的字体（通常 Calibri 或 Arial 11pt）

**永不包含：**
- 每个单元格周围的边框
- 多种字体或字号
- 除非特别要求否则不加图表
- 过度格式化或装饰

## 结构一致性

除非明确指示，否则使用标准8标签结构：
1. 执行摘要
2. 历史财务数据（利润表）
3. 资产负债表
4. 现金流量表
5. 运营指标
6. 物业/分部业绩（如适用）
7. 市场分析
8. 投资亮点

### 标签1：执行摘要
目的：为忙碌高管提供的一页概述

内容：
- 公司概述（2-3句商业模式描述）
- 关键投资亮点（3-5个要点）
- 财务快照表（近3年收入、EBITDA、增长率 + 预测）
- 交易概述（如适用）
- 突出显示的关键指标

格式：整洁、粗体标题、最少装饰、强调关键数字

### 标签2：历史财务数据（利润表）
目的：完整的损益历史

内容：
- 按分部/产品线的收入明细
- 销售成本/收入成本
- 毛利和毛利率%
- 运营费用明细（销售费用、研发费用、管理费用）
- EBITDA 和调整后 EBITDA
- 线下项目（折旧摊销、利息、税费）
- 净利润

格式：
- 年份作为列（文本格式：2020, 2021, 2022）
- $百万或$千（在顶部清楚注明单位）
- 所有财务数据使用会计格式
- 小计上方单下划线，净利润下方双下划线
- 所有数字右对齐

### 标签3：资产负债表
目的：期末财务状况

内容：
- 流动资产（现金、应收账款、存货、预付款、其他）
- 长期资产（固定资产、无形资产、商誉、其他）
- 流动负债（应付账款、应计费用、短期债务、其他）
- 长期负债（长期债务、递延税款、其他）
- 股东权益（普通股、留存收益、其他）

格式：
- 验证公式：资产 = 负债 + 权益
- 一致的日期标注
- 包含营运资本计算
- 主要小计上方单下划线，最终总计下方双下划线

### 标签4：现金流量表
目的：现金产生和使用分析

内容：
- 经营现金流（优先间接法）
- 投资现金流（资本支出、收购、资产出售）
- 筹资现金流（债务发行/偿还、股权、股息）
- 现金净变动
- 期初和期末现金余额

格式：
- 尽可能链接到利润表和资产负债表
- 展示净利润到经营现金流的调节
- 清晰标注现金用途（流出）vs 来源（流入）

### 标签5：运营指标
目的：非财务KPI和运营数据

内容（因行业而异）：
- 单位产量、客户数、地点数
- 生产力指标（人均收入、单店收入、单位收入）
- 产能利用率
- 市场份额
- 客户留存/流失率
- 行业特定KPI

**关键格式说明：**
运营指标无美元符号。这些是数量，非货币。

格式：
- 清楚注明单位（客户、员工、门店、平方英尺等）
- 带逗号的整数：1,250 而非 $1,250
- 比率用百分比：95.0%
- 数字右对齐

### 标签6：物业/分部业绩（如适用）
目的：按业务单元、物业或分部的详细明细

内容：
- 按分部的收入和盈利能力
- 按地点/产品的关键指标
- 分部特定KPI
- 比较业绩分析

格式：收入/EBITDA与财务标签一致，运营指标用数字格式

### 标签7：市场分析
目的：行业背景和竞争定位

内容：
- 市场规模和增长趋势
- 竞争格局概述
- 市场份额分析
- 行业基准和同行比较
- 监管环境（如相关）

格式：叙述文本和表格混合，引用市场数据来源

### 标签8：投资亮点
目的：关键投资论点的叙述性摘要

内容：
- 竞争优势的详细描述
- 增长机会和战略举措
- 风险因素和缓解策略
- 管理层评估和过往业绩
- 投资论点摘要

格式：清晰的标题、要点、简洁的段落

## 分步工作流程

### 第一阶段：文档处理和数据提取

**步骤1.1：分析源数据**
- 访问源材料：上传的文档、公开文件的网络搜索、或MCP服务器数据
- 审查数据结构并识别关键部分
- 定位财务报表（通常3-5年历史）
- 识别管理层预测（如包含）
- 注明财年结束日期
- 立即标记任何数据质量问题

**步骤1.2：提取财务报表**
- 定位历史利润表数据
- 提取资产负债表快照（年末或季末）
- 查找现金流量表
- 提取管理层预测（如有）
- 注明所有页码引用以便追溯

**步骤1.3：提取运营指标**
- 识别与行业相关的非财务KPI
- 捕获单位经济效益数据
- 提取客户/地点/产能数据
- 记录增长指标和趋势

**步骤1.4：提取市场和行业数据**
- 竞争定位信息
- 市场规模和增长率
- 行业基准数据
- 同行比较信息

**步骤1.5：记录关键背景**
- 交易结构和理由
- 管理团队背景
- 源材料中的投资亮点
- 风险因素和考虑事项
- 任何数据缺口或不一致

### 第二阶段：数据标准化

**步骤2.1：标准化会计呈现**
- 确保所有年份行项目名称一致
- 标准化收入确认处理
- 识别并记录一次性费用
- 如需要创建"调整后EBITDA"调节表
- 注明任何会计政策变更

**步骤2.2：应用格式检测逻辑**
对每个数据点，根据完整上下文确定格式：
- 读取标签名称、表格标题、列标题和行标签
- 应用基本规则（见上文）
- 不确定时，检查原始源文件
- 默认使用更简洁的格式（少即是多）

**步骤2.3：识别标准化调整**
常见调整需记录：
- 重组费用（如真正非经常性则加回）
- 股权激励费用（按行业标准加回）
- 收购相关成本（加回，注明金额）
- 法律和解或诉讼成本（评估复发风险）
- 资产出售或减值（从经营业绩中排除）
- 关联方调整（按市场费率标准化）
注意：来源引用格式因数据源而异（文档页码、网页URL、MCP数据服务器引用）

**步骤2.4：创建调整明细表**
对每个标准化：
- 记录调整内容和原因
- 引用来源（文档页码、URL或数据源引用）
- 按年量化金额影响
- 评估复发风险
- 展示从报告数据到调整后数据的计算

**步骤2.5：验证数据完整性**
- 使用公式确认小计正确汇总
- 验证资产负债表平衡
- 检查现金流与资产负债表变动一致
- 跨标签交叉检查数字一致性
- 标记任何差异进行调查

### 第三阶段：构建Excel工作簿

**步骤3.1：创建标准化标签结构**
创建包含以下标签的工作簿：
- 执行摘要
- 历史财务数据
- 资产负债表
- 现金流量
- 运营指标
- 物业业绩（如适用）
- 市场分析
- 投资亮点

**步骤3.2：用正确格式构建每个标签**
系统性地应用格式规则：
- 标题：粗体、左对齐、11pt字体
- 财务数据：货币格式 $#,##0.0（百万）
- 运营数据：数字格式 #,##0（无$）
- 百分比：0.0% 格式
- 年份：文本格式防止插入逗号
- 负数：使用括号的会计格式
- 下划线：小计上方单线，总计下方双线

**步骤3.3：插入计算公式**
- 所有小计和总计必须基于公式
- 适当情况下将资产负债表链接到利润表
- 将现金流链接到利润表和资产负债表
- 创建跨标签引用进行验证
- 避免硬编码任何计算值

**步骤3.4：应用专业呈现**
- 在每个数据标签冻结首行和首列
- 设置适当的列宽（通常12-15个字符）
- 所有数字数据右对齐
- 所有文本和标题左对齐
- 按会计标准添加单/双下划线
- 确保整洁、简洁的外观

### 第四阶段：情景构建（如包含预测）

**管理层情景：**
按源材料呈现公司预测：
- 提取所有管理层假设
- 记录增长率、利润率扩张、资本需求
- 注明关键驱动因素和敏感性
- 标记任何需要质疑的"曲棍球棒"拐点
- 清晰标注为"管理层情景"

**基准情景（风险调整后）：**
根据公司特定风险因素对管理层预测应用保守调整：
- 应用反映执行风险和历史预测准确性的收入增长折减
- 根据行业基准和经营杠杆适度调整利润率扩张假设
- 如增长依赖则增加资本支出假设
- 如低估则添加营运资本需求
- 如适用，根据整合复杂性延迟协同效应实现
- 记录所有调整及理由和支持分析

**下行情景（可选但建议用于LBO分析）：**
基于行业周期性和公司脆弱性的压力测试情景：
- 建模反映衰退风险或竞争压力的收入下降
- 假设压力下的利润率压缩（销量去杠杆、定价压力）
- 测试契约合规性和流动性
- 评估下行保护
- 记录正在压力测试的关键风险

**情景文档要求：**
创建假设明细表显示：
- 按情景的关键假设（收入增长、利润率、资本支出%）
- 每项调整的理由
- 关键变量的敏感性分析
- 历史预测准确性（如有）
- 与行业基准的比较

### 第五阶段：质量控制和验证

**步骤5.1：数据准确性检查**
验证：
- 每个数字追溯到来源（抽查样本，引用文档/URL/服务器）
- 所有计算基于公式（无硬编码值）
- 小计和总计数学正确
- 年份显示无逗号（2024 而非 2,024）
- 无公式错误：#REF!, #VALUE!, #DIV/0!, #N/A

**步骤5.2：格式一致性检查**
验证：
- 财务数据格式中有$符号
- 运营数据无$符号
- 百分比显示为%（15.0% 而非 0.15）
- 财务数据负数使用括号
- 标题粗体且左对齐
- 数字右对齐
- 年份为文本格式

**步骤5.3：结构和完整性检查**
确认：
- 所有必需标签存在且顺序正确
- 执行摘要简洁（适合一页）
- 所有关键指标全面捕获
- 从摘要到细节的逻辑流程
- 每个标签适当的详细程度
- 无缺失数据或不完整部分

**步骤5.4：专业呈现检查**
审查：
- 最少边框（仅用于结构）
- 一致的缩进（子项目2空格）
- 适当的会计下划线（单线和双线）
- 全文整洁、专业的外观
- 适当的列宽（不太窄或太宽）

**步骤5.5：文档和假设检查**
确保：
- 所有标准化调整有文档记录和理由
- 包含来源引用（文档页码、URL或数据源引用）
- 假设清晰陈述且合理
- 执行摘要准确且有影响力
- 文件名包含公司名称和日期

### 第六阶段：最终交付

**步骤6.1：创建执行摘要**
撰写简洁、有影响力的摘要，包括：
- 公司概述：商业模式、产品/服务、地理（2-3句）
- 关键财务指标：收入、EBITDA、增长率（表格格式）
- 投资亮点：3-5个关键优势或机会
- 值得注意的风险或考虑事项（简要）
- 交易背景（如适用）

**步骤6.2：最终文件准备**
- 以正确命名保存工作簿：公司名称_数据包_YYYY-MM-DD.xlsx

## 标准化模式

### EBITDA常见调整

**1. 重组费用**
- 如真正非经常性则加回（设施关闭、一次性裁员）
- 如公司每年都重组则不加回
- 记录具体性质和非经常性理由
- 示例："2023年重组：$3.0M设施关闭，源材料有记录，一次性事件"

**2. 股权激励费用**
- 行业标准：私募股权分析时加回
- 视为非现金运营费用
- 所有期间保持一致
- 注明是否异常高或包含一次性授予

**3. 收购相关成本**
- 加回交易费用、整合成本
- 按类型记录具体金额
- 不加回持续的整合投资
- 为每项调整引用来源

**4. 法律和解和诉讼**
- 如真正孤立事件则加回
- 评估复发风险（一次性和解 vs 诉讼模式）
- 记录和解性质
- 考虑是否为正常业务

**5. 资产出售或减值**
- 从经营EBITDA中排除资产出售损益
- 如真正非经常性则移除减值费用
- 记录出售/减值的资产及原因
- 如资产产生经营收入则调整收入

**6. 关联方调整**
- 标准化高于市场的关联方费用（租金、管理费）
