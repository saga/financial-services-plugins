## 视觉层次和布局

### 框和区块布局

幻灯片布局根据内容要求和模板设计而变化。常见元素包括：
- 带标题和副标题的标题区域
- 带标签侧边栏的内容框
- 结构化数据的表格
- 视觉数据展示的图表
- 幻灯片底部的脚注栏

具体布局应遵循所提供的模板。常见内容类型及其典型结构：
- **市场定义幻灯片**：带项目符号内容的标签框 + 评论区域
- **TAM/规模幻灯片**：指标突出显示 + 数据表格 + 关键要点
- **竞争分析**：比较表格或矩阵
- **财务摘要**：带支持数据表格的图表

### 对齐原则

**平行区块的垂直对齐：**

垂直堆叠的框应具有一致的：
- 左边距位置
- 项目符号缩进
- 文本起始位置
- 框宽度

水平相邻的框应具有一致的：
- 顶部位置
- 高度（在内容允许的情况下）
- 内部边距

## 文本格式

### 项目符号结构

避免无结构的文本堆砌。将内容分解为可扫描的项目符号。

**正确结构示例：**
```
✓  消费者移动和网页语言学习应用
   (Duolingo, Babbel, Memrise, Busuu)
✓  B2B企业语言培训平台
   (goFLUENT, Speexx, Learnship)
✓  在线辅导市场平台
   (italki, Preply, Cambly)
```

**错误结构示例（文本堆砌）：**
```
消费者移动/网页应用 (Duolingo, Babbel, Memrise, Busuu)
B2B企业平台 (Speexx, Rosetta Stone Enterprise)
在线辅导市场平台 (Preply, italki, Cambly)
```

### 项目符号指南

| 上下文 | 符号 | 用途 |
|--------|------|------|
| 包含/正面 | ✓（勾选） | 范围内项目，存在的功能 |
| 排除/负面 | ×（叉号） | 范围外项目，缺失的功能 |
| 中性列表 | •（项目符号） | 一般列举，评论 |
| 编号序列 | 1. 2. 3. | 流程步骤，排名 |
| 子项目符号 | ‣ 或 – | 主要项目下的次要要点 |

调整符号使用以匹配模板的现有惯例。

### 项目符号一致性

同一框/区块内的所有项目符号应具有相同的格式：
- 整个框内使用相同的项目符号符号（除非有意区分）
- 所有主要项目符号使用相同的缩进级别
- 相同的项目符号大小
- 项目符号与文本之间相同的间距
- 同一级别的所有项目符号文本使用相同的字体大小

### 字体大小指南

这些是典型范围——根据模板规格调整：

| 元素 | 典型大小（磅） | 样式 |
|------|----------------|------|
| 幻灯片标题 | 40-48 | 粗体 |
| 副标题/定义 | 18-22 | 粗体 |
| 区块标题 | 14-16 | 常规 |
| 正文/项目符号 | 12-14 | 常规 |
| 表格标题 | 10-12 | 粗体 |
| 表格正文 | 9-11 | 常规 |
| 脚注 | 8-9 | 斜体 |

### 文本密度指南

- **每个内容框最多6-7个项目符号**（根据空间调整）
- **每个项目符号最多2行**
- **括号示例**在同一行或下方缩进
- **避免孤立词**——调整换行以避免单个词在新行

## 表格创建

### 关键：使用实际表格对象

**表格必须是实际的表格对象，而非带制表符间距的文本。**

带制表符的文本永远无法正确对齐，看起来不专业。始终创建正确的表格对象。

### 表格结构指南

1. **列对齐**：
   - 文本列：左对齐（标题和内容都是）
   - 数值列：居中对齐或右对齐
   - 标题应与其列内容的对齐方式匹配

2. **标题行**：
   - 粗体文本
   - 着色背景（使用模板的品牌色）
   - 对比色文本以提高可读性
   - 对齐方式与列内容对齐方式匹配

3. **交替行**（可选）：
   - 交替行浅色着色可提高可读性

4. **汇总/合计行**：
   - 粗体文本
   - 较粗的上边框（分隔线）
   - 明显的背景着色

5. **表格宽度**：
   - 填充指定的区块宽度
   - 避免表格漂浮在空白区域

### XML实现模式，参见 [`xml-reference.md`](xml-reference.md#table-implementation)

## 图表和图片处理

### 从Excel粘贴图表

从Excel粘贴图表时：

1. **仅粘贴图表**——不要包含源数据表
2. **调整大小以填充指定区域**——图表不应显示为小缩略图
3. **保持纵横比**——不要扭曲图表
4. **验证可读性**——轴标签、图例、数据标签必须清晰可读

### 从Excel粘贴表格

从Excel粘贴表格时：

1. **仅粘贴格式化的表格**——排除任何源数据或计算
2. **调整大小以填充指定区域**——表格应占据其完整区块
3. **验证列宽**——调整以免文本被截断
4. **检查格式保留**——颜色、边框、字体可能需要调整

### 大小指南

**最小尺寸原则：**
- 图表：应占据其指定区域的相当部分
- 表格：完全填充指定的区块宽度
- 图片：根据上下文适当调整大小，绝不使用缩略图大小

**尺寸过小的视觉元素指标（避免这些）：**
- 图表仅占可用空间的一小部分
- 文本标签不可读
- 视觉周围有大片空白区域
- 视觉看起来像"缩略图"

### 正确调整大小工作流程

1. 确定目标区域尺寸
2. 粘贴图表/表格
3. 立即调整大小以填充目标区域
4. 验证所有文本保持可读
5. 如需要调整内部元素（图例位置、轴标签）

## 数据可视化

### 关键指标展示

展示关键指标（如TAM、CAGR、预测）时，考虑显示数值之间的关系而非静态列出：

- **视觉流程指示器**：显示进展的形状（箭头、V形、连接器）
- **大小层次**：主要指标使用较大字体，标签使用较小字体
- **空间排列**：定位元素以显示逻辑流程

### 箭头和流程指示器

如果使用箭头或流程指示器：
- 使用PowerPoint形状对象，而非文本字符
- 不要在最终演示中使用文本箭头（→, ⟹）
- 使用PowerPoint的形状工具或通过XML形状元素创建箭头

**XML实现，参见 [`xml-reference.md`](xml-reference.md#arrow-shapes)**

## 字体一致性

### 跨框字体一致性

同一层次级别的所有文本框应使用相同的字体大小。

**应匹配的同级框：**

| 框类型 | 应匹配 |
|--------|--------|
| "包含细分市场"内容 | "排除细分市场"内容 |
| "定义"内容 | "范围理由"内容 |
| 左列项目符号 | 右列项目符号 |
| 所有标签框 | 彼此 |
| 所有区块标题 | 彼此 |

### 验证流程

1. 识别同一层次级别的所有文本框
2. 检查每个框的字体大小
3. 如有任何框不同，调整所有框以匹配
4. 如果内容适合，默认使用较大尺寸；否则一致使用较小尺寸

**例外**：子项目符号或次要文本可能使用比主要项目符号更小的字体，但这必须在所有框中保持一致。

## 模板适配

这些标准应根据所使用的具体模板进行调整：

1. **颜色**：使用模板的品牌色而非指定具体颜色
2. **字体**：使用模板的字体系列
3. **间距**：匹配模板的现有间距惯例
4. **布局**：遵循模板的区块结构

无论模板如何都保持不变的关键原则：
- 文本必须在其背景上可读
- 表格必须是实际的表格对象
- 内容应适当填充可用空间
- 格式应在平行元素间保持一致
- 图表/图片应正确调整大小
