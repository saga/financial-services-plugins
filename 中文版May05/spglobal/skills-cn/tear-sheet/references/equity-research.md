# 股票研究简报 (Equity Research Tear Sheet)

## 目的
为买方或卖方分析师评估投资机会而提供的紧凑快照。重点关注估值、远期预测、财务轨迹以及分析师一致预期。所有内容旨在支持或质疑投资逻辑（Investment Thesis）。

**默认页长：** 1页。股票研究简报常规为单页——紧凑性是核心。如果用户要求更多空间，可扩展至2页。

## 查询方案

从以下查询开始。如果结果不完整，则分解为更细致的后续调查。

**查询 1 — 概况 + 市场数据：**
“[公司名称] 公司概况 板块 行业 市值 企业价值 股价 52周最高/最低 发行在外股数 贝塔系数(beta)”
→ 页眉、业务描述
→ **立即写入** 至 `/tmp/tear-sheet/company-profile.txt`

**查询 2 — 历史财务数据：**
“[公司名称] 年度损益表 营收 毛利 EBITDA 净利润 EPS 自由现金流 总负债 现金及现金等价物 过去4个财政年度”
→ 财务摘要（提取4年；展示3年；最早一年仅用于计算同比(YoY)增长）
→ **立即写入** 原始值至 `/tmp/tear-sheet/financials.csv`

**查询 2b — 业务部门（如有）：**
“[公司名称] 按部门分类营收 业务单元 过去2个财政年度”
→ 营收与部门分解（需要2年数据以计算同比增长）。如果无法获得部门数据，则跳过此部分——不要留白表。
→ **立即写入** 至 `/tmp/tear-sheet/segments.csv`（如无数据则跳过）

**查询 3 — 估值 + 一致预期：**
“[公司名称] P/E EV/EBITDA EV/Revenue 估值倍数 一致预期营收 EPS预测 分析师建议 目标价”
→ 估值快照、一致预期
→ **立即写入** 倍数至 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 预测至 `/tmp/tear-sheet/consensus.csv`

**查询 4 — 业绩会议：**
“[公司名称] 最近一次业绩电话会关键要点 指引(guidance)”
→ 业绩亮点
→ **立即写入** 至 `/tmp/tear-sheet/earnings.txt`

**查询 5 — 股票表现：**
“[公司名称] 股价回报率 1个月 3个月 6个月 1年 年初至今(YTD) 表现”
→ 股票表现（无中间文件——数据直接存入文档）

**查询 6（如果用户提供了可比公司）：**
“[可比公司名称] 市值 EV/Revenue EV/EBITDA 营收增长” —— 对每家可比公司重复操作
→ 估值快照的同业背景
→ **立即写入** 至 `/tmp/tear-sheet/peer-comps.csv`

## 模块

按优先级排序。如果受限于单页，请从底部开始删减。

### 1. 公司页眉
紧凑的键值对块，根据全局样式配置渲染为无框双栏表格。

左栏：股票代码（交易所）、板块/行业、总部所在地
右栏：股价、52周波动范围、市值、企业价值 (EV)、发行在外股数、贝塔系数 (beta)

### 2. 业务描述
2-3句简练的描述。**以分析师视角用你自己的语言重写** —— 请勿逐字粘贴 CIQ 的公司摘要。CIQ 摘要是输入信息；输出应当简洁且以投资逻辑为导向。

对于知名度高、覆盖面广的公司，假设分析师已了解其基本业务。重点关注*变化* —— 战略转型、资产组合重塑、新的增长引擎 —— 而非维基百科式的综述。例如，对于一家财富500强公司，不要花一整句说“向机构提供财务数据”。而应写为：“正将其业务组合转向高利润的数据与分析业务，待处理的 Mobility 业务剥离计划以及通过其 Kensho 子公司进行的激进 AI 投资将成为重点。”

对于知名度较低的公司，在转向投资逻辑相关动态之前，可以先用一句简短的“公司业务”描述。

| 市盈率 (P/E) | | |

| 企业价值/营收 (EV/Revenue) | | |
| 市现率 (P/FCF) | | |
| 股息收益率 | | |

**如果工具能提供远期倍数，则必须显示。** 在远期数据存在的情况下仅显示滚动倍数是重大缺陷——分析师基于预期收益对公司进行估值。如果无法获得远期倍数，则仅显示滚动倍数并注明“远期预测不适用(N/A)”。

如果用户提供了可比公司，请为每个公司添加列（或在可比公司多于3家时添加“同业中位数”列）。如果用户未提供但工具返回了同业数据，请包含“同业中位数”列。如果均无，则仅显示该公司自身的倍数。
