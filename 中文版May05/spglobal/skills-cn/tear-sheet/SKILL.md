# 金融简报 (Financial Tear Sheet) 生成器

通过 S&P Global MCP 工具从标普智汇 (S&P Capital IQ) 提取实时数据，生成针对特定受众的公司简报，并将结果格式化为专业的 Word 文档。

## 样式配置

以下是合理的默认设置。如需根据贵司的品牌进行定制，请修改此部分——常见的修改包括更换色板、更改字体（许多投行标准使用 Calibri）以及更新免责声明文本。

**颜色：**
- 主色（页眉条幅背景、板块标题文本）：#1F3864
- 强调色（签名板块高亮）：#2E75B6
- 表格标题行填充：#D6E4F0
- 表格隔行变色填充：#F2F2F2
- 表格边框：#CCCCCC
- 页眉条幅文本：#FFFFFF

**排版（docx-js 的字号单位为半磅）：**
- 字体族：Arial
- 公司名称：18pt 加粗 (size: 36)
- 板块标题：11pt 加粗 (size: 22)，使用主色
- 正文文本：9pt (size: 18)
- 表格文本：8.5pt (size: 17)
- 页脚/免责声明：7pt 斜体 (size: 14)
- 每个参考文件的“格式化说明”中均指定了各模板的特定覆盖设置。

**公司页眉条幅：**
- 页眉是一个跨越全页面宽度的深蓝色 (#1F3864) 条幅，公司名称以白色显示。
- **在条幅下方，必须使用一个跨越全页面宽度的无边框两栏表格来呈现键值对。** 左栏：公司标识信息（股票代码、总部、成立时间、员工人数、行业板块）。右栏：财务指标（市值、企业价值、股价、发行在外股数）。每个单元格包含加粗的标签和常规粗细的数值，且位于同一行（例如：“**市值** $124.7B”）。不要将所有字段左对齐排列在单栏中——这会浪费横向空间且显得不专业。两栏分布是将专业简报与普通文档区分开来的最重要视觉信号。
  - **实现：** 创建一个 2 栏表格，所有单元格设置 `borders: none` 且 `shading: none`。将列宽均设置为 50%。将左栏字段（股票代码、总部、成立时间、员工人数）作为独立段落放入左侧单元格。将右栏字段（市值、企业价值、股价、发行在外股数）放入右侧单元格。每个字段为一个段落：加粗部分跑标签，常规部分跑数值。
  - 每个栏目中的具体字段因受众而异——请参见参考文件的页眉规范。原则始终是：横向平铺，而非堆叠在左侧。
- **不要在页眉键值对板块使用有边框的表格。** 有边框的表格仅保留给财务数据使用。
- 页眉中的关键指标（市值、企业价值、股价）应显示为行内键值对，而非放在独立的有边框表格中。

**板块标题：**
- 每个板块标题的正下方需有一条横向制表线（细线，#CCCCCC，0.5pt），以在板块之间创建清晰的视觉分隔。
- **将该制表线渲染为标题段落本身的下边框**——不要为该线插入单独的段落元素。单独的段落会增加其自身的段前/段后间距，导致标题下方出现过多的空白。
- **实现：** 在 docx-js 中，通过 `paragraph.borders.bottom = { style: BorderStyle.SINGLE, size: 1, color: "CCCCCC" }` 为板块标题段落应用下边框。不要使用带有独立水平线元素的 `doc.addParagraph()`。不要使用 `thematicBreak`。边框必须位于标题段落本身，且段后间距为 0pt，使线紧贴标题文本。
- 间距：标题段落前 12pt，标题段落后 0pt，下一内容元素前 4pt。

**列表格式：**
- 所有的简报类型均使用单一列表符 (•)。不要在同一份或不同份简报中混合使用 •、-、▸ 或编号列表。
- **综合分析类列表**（业绩亮点、战略匹配、整合考量、交流引子）：使用缩进块状格式，左缩进 360 DXA (0.25")，并为列表符设置悬挂缩进。这些内容应在视觉上与正文区分开——它们属于解读性内容，应看起来与数据表和普通段落有所不同。
- **信息类列表**（关系板块内）：标准正文缩进 (180 DXA)，无悬挂缩进。
- **不要在任何列表板块应用左边框装饰。** 左边框样式在 docx-js 中的渲染效果不一致，且会产生视觉瑕疵。请通过缩进和字号差异来区分签名板块。

**表格（仅限财务数据）：**
- 标题行：表格标题填充色 (#D6E4F0)，深色加粗文本
- 正文行：白色与表格隔行填充色 (#F2F2F2) 交替
- 边框：表格边框色 (#CCCCCC)，细线 (BorderStyle.SINGLE, size 1)
- 单元格边距：顶/底 40 DXA，左/右 80 DXA
- 所有数值列右对齐
- 始终使用 ShadingType.CLEAR（绝不使用 SOLID——SOLID 会导致背景变黑）

**布局：**
- US Letter 纵向，0.75 英寸页边距（四周均为 1080 DXA）

**数字格式：**
- 货币：USD。除非公司收入 > $50B（此时使用十亿美元，保留一位小数），否则使用百万美元。在列标题中标注单位（如 "Revenue ($M)"），不要在单个单元格中重复标注。
