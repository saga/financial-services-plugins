# 初始覆盖报告

通过结构化的5任务工作流程创建机构级股权研究报告初始覆盖。每个任务必须单独执行并验证输入。

## 概述

本技能遵循机构标准（JPMorgan、高盛、摩根士丹利格式）生成全面的首次覆盖报告。任务单独执行，每个任务在继续之前验证前置条件。

**默认字体**：所有文档使用Times New Roman（除非用户另有指定）。

---

## ⚠️ 关键：一次执行一个任务

**本技能仅在单任务模式下运行。**

### 当用户请求完整流程时

当用户请求时：
- "为[公司]创建覆盖初始报告"
- "为[公司]撰写初始报告"
- "完成[公司]的整个股权研究流程"
- "完成[公司]的所有5个任务"
- 任何暗示运行多个任务或整个工作流程的请求

**必需的响应：**

1. **询问具体执行哪个任务：**
   ```
   我可以帮助您为[公司]创建股权研究初始报告。
   这涉及5个独立的任务，需要单独完成：

   1. 公司研究 - 研究业务、管理层、行业
   2. 财务建模 - 构建预测模型
   3. 估值分析 - DCF和可比公司
   4. 图表生成 - 创建25-35张图表
   5. 报告汇编 - 编制最终报告

   您想从哪个任务开始？
   ```

2. **当用户明确请求所有任务一起执行时：**
   ```
   我理解您希望完成整个初始报告流程。
   目前，本技能支持一次执行一个任务，这样可以更好地
   在每个阶段进行质量控制和审查。

   我们正在开发无缝的端到端工作流程，使这个过程更加自动化，
   但目前，我们需要单独完成每个任务。

   您想从任务1（公司研究）开始吗？
   ```

3. **永远不要自动假设从哪个任务开始** - 始终要求用户确认。

4. **永远不要自动链接多个任务** - 完成一个任务，交付输出，然后等待下一个用户请求。

### 任务执行规则

- ✅ 严格按照每个用户请求执行一个任务
- ✅ 在开始任务前始终验证前置条件
- ✅ 交付任务输出并确认完成
- ✅ 等待用户明确请求下一个任务
- ❌ 永远不要自动链接多个任务
- ❌ 永远不要假设用户想要继续下一个任务
- ❌ 在验证所需输入之前，永远不要执行任务3-5

### ⚠️ 可交付成果政策：不走捷径

**只交付指定的输出。不要创建额外的文档。**

每个任务都指定了确切的可交付成果。不要创建：
- ❌ "完成总结"
- ❌ "执行摘要"
- ❌ "快速参考指南"
- ❌ "后续步骤文档"
- ❌ "任务完成报告"
- ❌ 任何其他未明确指定的"有用"文档

**原因**：这些额外内容会浪费上下文，且不属于专业工作流程。

**应该交付的内容**：
- ✅ 任务1：研究文档（.md）— **仅此而已**
- ✅ 任务2：财务模型（.xlsx）— **仅此而已**
- ✅ 任务3：估值分析（.md）+添加到任务2文件的Excel标签页 — **仅此而已**
- ✅ 任务4：图表压缩文件（.zip）— **仅此而已**
- ✅ 任务5：最终报告（.docx）— **仅此而已**

**如果可交付成果未在上表中列出，请不要创建。**

## 如何使用本技能

### 用户请求模式及响应

**模式1：用户指定具体任务**
```
用户："对特斯拉使用initiating-coverage，任务1"
响应：✅ 立即执行任务1
```

**模式2：用户请求"初始报告"或"完整流程"**
```
用户："为特斯拉创建覆盖初始报告"
响应：❌ 不要自动开始任何任务
     ✅ 询问从哪个任务开始（见上述模板）
```

**模式3：用户想要执行"所有任务"或"整个工作流程"**
```
用户："我想为特斯拉完成所有5个任务"
响应：❌ 不要链接任务
     ✅ 解释一次一个的限制（见上述模板）
     ✅ 询问是否从任务1开始
```

### 正确的使用示例

**执行单个任务：**
```
"对特斯拉使用initiating-coverage技能，任务1"
"对特斯拉执行initiating-coverage的任务2"
"使用initiating-coverage技能对特斯拉运行任务3"
```

**完成完整报告（需要5个独立请求）：**
```
请求1："对特斯拉执行任务1" → 完成 → 交付输出
请求2："对特斯拉执行任务2" → 完成 → 交付输出
请求3："对特斯拉执行任务3" → 完成 → 交付输出
请求4："对特斯拉执行任务4" → 完成 → 交付输出
请求5："对特斯拉执行任务5" → 完成 → 交付输出
```

### 任务执行顺序

要完成初始报告，任务必须按照以下顺序单独执行：

```
请求1：任务1 - 公司研究（独立）
           ↓ [用户审查输出并请求下一个任务]
请求2：任务2 - 财务建模（独立）
           ↓ [用户审查输出并请求下一个任务]
请求3：任务3 - 估值分析（需要任务2输出）
           ↓ [用户审查输出并请求下一个任务]
请求4：任务4 - 图表生成（需要任务2和3输出）
           ↓ [用户审查输出并请求下一个任务]
请求5：任务5 - 报告汇编（需要所有先前任务输出）
```

**注意**：任务1和任务2可以按任意顺序运行。任务3-5有严格的依赖关系，必须在继续之前验证输入。

## 任务1：公司研究

**目的**：研究公司的业务、管理层、竞争地位、行业和风险。

**前置条件**：✅ 无（完全独立）
- 公司名称或股票代码

**流程**：
1. 验证已提供公司名称/股票代码
2. 从references/task1-company-research.md加载详细说明
3. 执行定性研究工作流程
4. 交付研究文档

**输出**：公司研究文档（6,000-8,000字）
- 公司概况和历史
- 管理层简历（3-4名高管每人300-400字）
- 产品和服务分析
- 行业概况
- 竞争分析（5-10个竞争对手）
- TAM规模
- 风险评估（8-12个风险）

**文件名**：`[公司]_[日期]_研究文档.md`

**⚠️ 仅交付这1个文件。没有完成总结，没有额外文档。**
