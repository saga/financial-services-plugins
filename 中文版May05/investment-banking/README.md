# 投资银行业务插件

为股票研究、价值评估、演示文稿撰写及交易材料准备提供的投行生产力工具。

## 功能特性

- **交易材料** - 自动化编写机密信息备忘录 (CIM)、推介简报 (Teaser)、流程函件 (Process Letter) 以及买方清单
- **演示文稿** - 生成一页纸条目式简报 (Strip Profile)，支持品牌模板的业务推介材料 (Pitch Deck)
- **交易执行支持** - 并购模型构建、交易进度跟踪及底层数据包 (Data Pack) 构建

## 安装

```bash
claude --plugin-dir /path/to/investment-banking
```

或将其复制到项目的 `.claude-plugin/` 目录下。

## 命令

| 命令 | 描述 |
|---------|-------------|
| `/one-pager [公司]` | 为业务推介材料生成一页纸简明概况 (Strip Profile) |
| `/cim [公司]` | 起草机密信息备忘录 (CIM) |
| `/teaser [公司]` | 生成匿名一页纸公司推介简报 (Teaser) |
| `/buyer-list [公司]` | 构建战略与财务买方资源池 |
| `/merger-model [交易]` | 进行并购增厚/稀释 (Accretion/Dilution) 分析 |
| `/process-letter [交易]` | 起草投标说明及交易流程往来函件 |
| `/deal-tracker` | 跟踪执行中的交易、关键里程碑及行动项 |

## 示例工作流

### 一页纸简明概况 (Strip Profile)
```
/one-pager [目标公司]

# 生成内容：
# - 使用 PPT 模板生成的单页公司概况
# - 包含四个象限：概况、业务线、财务摘要、股权结构
# - 遵循模板页边距及品牌视觉规范
```

### CIM 编写
```
/cim [目标公司]

# 生成内容：
# - 完整的 CIM 文档，包含执行摘要、业务概览、
#   财务分析及市场定位
```

### 并购模型
```
/merger-model [收购方] 收购 [目标公司]

# 生成内容：
# - 增厚/稀释分析 (Accretion/Dilution Analysis)
# - 资金来源与用途 (Sources and Uses)、备考财务报表
# - 针对收购价格及协同效应的敏感度分析
```

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到投资银行的“工具箱”！对于初学者，这里有几个核心概念的通俗解释：
>
> 1. **CIM 与 Teaser：公司的“深度简历”和“盲盒预览”**
>    - **CIM (机密信息备忘录)**：当一家公司想把自己卖掉时，投行会写一份长达几十页甚至上百页的详细说明书。因为它包含很多核心商业秘密，所以只有签了保密协议的买家才能看。它就像一份诚意满满的“深度简历”。
>    - **Teaser (推介简报)**：在买卖双方“互选”的初期，我们会发一份不写名字的单页简介。它只告诉你公司的行业、规模和亮点。就像一个去掉名字信息的“盲盒预览”，主要用来勾起买家的兴趣。
>
> 2. **增厚/稀释 (Accretion/Dilution)：这场联姻划算吗？**
>    并购就像两家公司“领证结婚”。如果合并后新公司的每股收益 (EPS) 比原来的要多，这就叫“增厚”；反之，如果每股收益变少了，就叫“稀释”。分析师需要通过模型计算，确保大买卖在账面上是划算的。
>
> 3. **资金来源与用途 (Sources and Uses)：买房的账本**
>    这其实就是投行的“记账表”。**来源 (Sources)** 说明买地、买公司的钱从哪领的（比如银行贷款、自己掏腰包）；**用途 (Uses)** 说明这些钱花到哪去了（比如给卖家的钱、给律师和投行的中介费）。两边的总数必须分毫不差地相等。
>
> 4. **协同效应 (Synergies)：1 + 1 > 2**
>    并购最迷人的地方在于，合并后的两家公司如果能裁撤重合的部门（省钱）或者共享销售网络（多赚钱），那么新公司的价值就会大于原来两家公司之和。这就是投行精英们口中常说的“协同效应”。
