# 投资组合AI准备度

description: 扫描投资组合，找出最高杠杆的AI机会，并排名应优先部署运营合伙人时间的领域。摄取多个投资组合公司的季度更新和财务数据，识别每个公司的快速成果，并将其堆叠为单一排名行动清单。在季度投资组合审查、年度规划或决定哪些公司优先获得AI投资时使用。当触发"AI准备度"、"AI机会扫描"、"我们应该在哪里部署AI"、"投资组合中的AI"、"AI快速成果"或"哪些被投公司适合AI"时使用。

## 工作流程

### 第1步：连接到投资组合数据

首先，询问用户投资组合材料存放在哪里。不要假设——提供选项：

- **MCP服务器** — 数据室、SharePoint、Google Drive或已连接的投资组合运营数据库
- **本地文件** — 磁盘上的文件夹路径，包含季度演示文稿、财务数据、董事会材料
- **文件上传** — 直接将PDF、PowerPoint或Excel拖入对话

连接后，拉取投资组合（或子集）的季度更新、董事会演示和财务数据。对于每个公司，提取：行业、收入、各职能部门人数、提到的技术栈，以及任何已在进行的AI/自动化举措。

如果用户提供单一公司，仍运行扫描，但跳过跨投资组合排名。

如果不明显，请提前询问：
- 每个公司的剩余持有期（AI回报在退出前12个月不太重要）
- 是否有任何被投公司已部署了有效的方案

### 第2步：每个公司扫描

对于每个公司，回答三个门控问题。全部是 → **执行**。任何一个否 → **等待**，并注明解锁条件。

1. **数据是否存在？** 他们能否产生该用例的干净输入——客户列表、发票数据、合同库——而无需6个月的数据项目？
2. **是否有负责人？** 管理团队中是否有人会推动此事，而不是只是"支持"的赞助人。
3. **我们能在30天内试点吗？** 一个团队、一个工作流程、现成工具。如果答案以"首先我们需要..."开头，那不是快速成果。

然后识别2-3个最高杠杆点。在成本结构和运营中寻找这些模式：

**后台（通常最快试点）**
- 发票处理、应付/应收账款匹配、费用分类
- 合同抽象 — 供应商协议、租赁、客户MSA
- 月末结算：调节、财务报告注释、贷款人报告初稿

**收入/前台**
- RFP和建议书初稿 — 如果收入是项目型的，这是大杠杆
- 销售电话摘要和CRM数据清洗
- 客户支持工单分类和初稿回复
- 配置/复杂产品报价

**运营（因行业而异）**
- SOP和质量文档生成
- 排程和调度（现场服务、物流）
- 代码生成和审查（软件类被投公司）

对于每个杠杆点，用一行捕获：替换了什么、每周节省的FTE小时数（假设30-50%，不是100%），以及它是现成购买还是需要轻度开发。

### 第3步：跨投资组合排名

将每个公司的每个杠杆点堆叠成一个列表。按以下标准排名：

1. **美元影响** — 年化EBITDA贡献（成本节省 + 收入提升，扣除工具成本）
2. **价值实现速度** — 到首次可衡量结果的月数
3. **概率** — 因数据质量、变革管理风险、管理团队能力而打折

平局决胜：优先剩余持有期少于18个月的机会 — 那些需要现在行动，否则就不行动。

输出堆叠表：

| 排名 | 公司 | 机会 | 预估EBITDA（$） | 实现月数 | 门控 | 第一步 |
|---|---|---|---|---|---|---|
| 1 | | | | | 执行 | |
| 2 | | | | | 执行 | |
| 3 | | | | | 等待 — [障碍] | |

### 第4步：寻找可复制的机会

投资组合中最高杠杆的行动是在多个公司运行一个成功的方案。扫描：

- **相同行业、相同职能** — 两个医疗服务的被投公司都有手动预先授权？一个实施，两个部署。
- **相同工具、不同公司** — 如果一个被投公司已有有效的发票处理设置，将拥有超过$X百万应付账款的所有其他被投公司标记为快速跟进者。
- **共享供应商杠杆** — 三个被投公司购买相同工具是定价对话。

列出每个可复制的机会，注明牵头公司（谁证明它）和跟进公司（谁复制）。

### 第5步：输出

为运营合伙人提供一页，在投资组合审查中构建：

1. **整个投资组合的前5名** — 第3步的排名表，包含负责人和30天第一步
2. **可复制的机会** — 2-3个可同时影响多个公司的方案
3. **每个公司的执行/等待** — 各一行；对于等待，注明解锁条件
4. **我们不做的** — 看起来不错但未通过门控的机会；为运营合伙人节省每季度重新审视的时间
5. ** aggregate EBITDA贡献** — 整个投资组合的AI机会总额，分解为第1年快速成果与第2-3年规模化

## 重要说明

- **按美元排名，不是按兴奋度。** 在4000万美元收入公司节省40万美元的无聊应付账款自动化，胜过每次都推荐炫酷的客户支持聊天机器人。
- **约束几乎总是数据，不是模型。** 如果公司无法生成干净的客户列表，AI不是第一个项目 — 数据清理才是。直接说明。
- **首先使用现成工具。** 定制开发对没有工程深度的公司来说缓慢、昂贵且脆弱。优先使用他们可以购买和部署的工具。
- **负责人是真正的门控。** 没有内部所有者，快速成果会在90天内夭折。如果管理团队中没有人想要，无论美元金额多大，都要标记为等待。
- **持有期驱动紧迫性。** 距离退出还有3年的公司可以承担基础数据项目。还有12个月的公司需要一些能在LTM EBITDA中显示的东西给CIM — 否则跳过。
- **失败的试点是信号。** 如果管理层已经尝试过某些东西但没有坚持，在再次提出之前找出原因。
