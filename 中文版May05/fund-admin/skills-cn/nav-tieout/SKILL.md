# NAV 勾稽核对 (NAV Tie-out)

给定已生成的 LP 报表以及该期间的 NAV 报告包（通过 nav MCP 获取），独立重新计算 LP 的资本账户并进行逐行对比。

> **生成的报表是受测对象。** NAV 报告包是唯一真实数据源（Source of Truth）。

## 重新计算 LP 资本账户

```
期初资本 (上一期报表的期末资本)
  + 出资 (本期间已支付的资本催缴)
  − 分配 (现金 + 实物)
  + 分配的净收益 / (亏损)
      = LP 持股比例 × (已实现 + 未实现损益 − 管理费 − 基金费用)
  − 业绩分成分配 (若在本期间结转)
期末资本
```

从 NAV 报告包中提取各项输入数据：LP 认缴比例、基金级损益组成部分、费用和开支总额、以及分配瀑布模型 (Waterfall) 的输出结果。

## 对比

将报表上的每一行项与您重新计算出的值进行对比。容差范围：`0.01`。对于每一处不匹配，请注明驱动该差异的输入项（例如，“分配的损益存在差异 —— 报表使用了 12.40% 的持股比例，而 NAV 报告包显示的 Q1 转让后比例为 12.38%”）。

## 其他检查项

- 本期报表的期末资本 = 下一期草案的 capital (如果可用)。
- 所有 LP 的期末资本总和 = 基金总 NAV（在舍入误差范围内）。
- 认缴额、未实缴额和可回收额需与认缴登记册 (Commitment Register) 保持一致。

## 输出

每一行项的通过/失败状态、重新计算的值与报表值的并列对比、以及标记列表。请勿直接编辑报表 —— 发布者将在审核标记后采取后续操作。

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到私募股权投资的世界！如果你对文中的金融术语感到困惑，这里有几个关键概念的“大白话”解释：
>
> 1. **NAV (Net Asset Value, 资产净值)**：简单来说，这就是基金现在“值多少钱”。它是基金持有的所有资产价值减去所有负债后的净额。
> 2. **LP (Limited Partner, 有限合伙人)**：就是基金的投资人。他们负责出钱，但不参与基金的日常管理操作。
> 3. **资本账户 (Capital Account)**：这就像是投资人在基金里的“个人存折”，记录了你投了多少钱（出资）、拿回了多少钱（分配），以及你在基金盈亏中分到了多少。
> 4. **勾稽核对 (Tie-out)**：这是一种专业的财务核查手段。想象一下你有两份账单，一份是你的购物小票，一份是银行流水，勾稽就是检查这两边的数据是否能严丝合缝地对应上。
> 5. **瀑布式分配 (Waterfall)**：这是基金赚钱后的“分钱规则”。钱会像瀑布一样流经不同的池子：先归还投资人的本金，再支付约定的利息，最后剩下的利润再由投资人和管理团队按约定的比例分成。
