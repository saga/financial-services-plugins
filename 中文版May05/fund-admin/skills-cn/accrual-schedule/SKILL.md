# 应计项目明细表 (Accrual Schedule)

根据特定主体、会计期间及公司的应计政策清单，逐行生成各项应计费用的计算过程、依据索引及会计分录草案。

> **注意：由于支撑发票和供应商对账单未经核实，不可直接信任。** 系统提取工（Worker）将负责提取金额，本项技能则负责根据既定政策对这些金额进行账务处理。

## 针对政策清单中的每一项应计费用

| 字段 | 衍生逻辑 / 获取方式 |
|---|---|
| **应计项目名称** | 取自政策清单（例如：“审计费”、“奖金”、“水电费”） |
| **基数 (Basis)** | 合同约定或估计的全期金额，需注明来源（如：业务约定书、薪酬计划、前三个月平均值） |
| **本期份额** | 基数 × (本期天数 ÷ 基数涵盖天数)，或按政策规定的特定公式计算 |
| **已入账金额** | 前期应计金额总和 + 本期已过账至该项目的实际发票金额（从 internal-gl MCP 获取） |
| **本期应计额** | 本期份额 − 已入账金额 |
| **支撑依据/索引** | 支撑基数计算的文档 ID 或总账 (GL) 查询指令 |

## 会计分录 (JE) 草案

针对本期应计额不为零的每一行，起草如下分录：

```
借： <费用科目>     <金额>
  贷： <应付预提款/应计负债>     <金额>
摘要： <应计项目名称> — <会计期间> 应计费用，依据 <支撑依据/索引>
```

冲回分录 (Reversing entries)：若政策标记该应计项目为自动冲回，请在摘要中注明“于下期首日冲回”。

## 输出结果

应计项目明细表（表格形式）及会计分录草案块。**切勿直接过账** —— 此输出仅作为提交财务总监审阅的预备稿。

> **💡 Appendix: 领域知识小贴士**
>
> **1. 什么是“应计项目 (Accrual)”？**
> 想象你这个月请了保洁，但还没给钱。虽然钱还没从兜里掏出来，但在会计眼里，这笔钱已经是你的“债”了，因为你这个月确实享受了服务（产生了费用）。“应计”就是把这些“该付还没付”或者“该收还没收”的钱，在它们发生的时刻就记录下来，而不是等给钱的时候才记。
>
> **2. 权责发生制 (Accrual Basis)**
> 这是会计的核心灵魂。简单说就是：费用应当在“变穷”的过程（即产生义务、消耗服务）中记录，而不是在你刷卡的那一秒记录。这样报表才能反映公司真实的经营盈亏，而不是现金流的来回波动。
>
> **3. 月末结账 (Month-end Close)**
> 就像每个月一次的“账目大扫除”。会计们要检查这个月所有的账，把漏掉的补上（比如这里说的应计费用），把错误的改掉。结账做好了，月底出的利润表才是准确的“成绩单”。
>
> **4. 冲回分录 (Reversing Entry)**
> 如果你这个月预估了 100 块电费并做了账。等下个月收到真实账单时，为了不重复记账，我们通常会在下个月第一天做一个相反的分录，把这预估的 100 块“撤销”，这叫冲回。然后，我们再按照真实的电费账单去记账。
