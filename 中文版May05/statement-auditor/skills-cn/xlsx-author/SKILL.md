# xlsx-author

当在**无界面模式**（托管代理 / CMA 模式）下运行，且需要交付 Excel 工作簿作为**文件产物**，而非通过 `mcp__office__excel_*` 插件编辑实时工作簿时，请使用此技能。

## 输出约定

- 写入路径为 `./out/<name>.xlsx`。如果 `./out/` 目录不存在，请先创建。
- 在最终回复中返回相对路径，以便编排层（Orchestration Layer）进行文件收集。

## 如何构建工作簿

编写一段简短的 Python 脚本并使用 Bash 运行。推荐使用 `openpyxl` 库：

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active; ws.title = "Inputs"
ws["B2"] = "Revenue"; ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")           # 蓝色 = 硬编码输入项
calc = wb.create_sheet("DCF")
calc["C5"] = "=Inputs!C2*(1+Inputs!C3)"        # 黑色 = 公式计算项
wb.save("./out/model.xlsx")
```

## 建模规范（遵循 `audit-xls`）

- **蓝/黑/绿原则。** 蓝色 = 硬编码输入项（Hardcoded Input）；黑色 = 公式计算项（Formula）；绿色 = 链接至其他工作表或外部文件的引用。
- **计算单元格严禁硬编码。** 所有计算单元格必须由公式构成；所有输入参数必须统一存放于“Inputs”标签页中。
- **命名区域 (Named Ranges)。** 任何被演示文稿（PPT）或备忘录（Memo）引用的数值，均需在 Excel 中定义命名区域。
- **平衡检查 (Balance Checks)。** 必须包含一个“Checks”标签页，用于核查勾稽关系（如资产负债表是否平衡、现金流量表是否与现金余额一致等），并直观显示 TRUE/FALSE。
- **一案一档。** 除非有明确要求，否则不要在现有工作簿中追加内容，每个模型应保持独立。

## 何时禁止使用

如果 `mcp__office__excel_*` 工具可用（即 Cowork 插件模式），应优先使用该工具——它能直接驱动用户的实时工作簿并支持复核节点。本技能仅作为无界面运行环境下的文件生成备选方案。

> **💡 Appendix: 领域知识小贴士**
> 
> 如果你是财务建模的新手，这些“金融规范”能让你的工作看起来非常专业：
> 
> 1. **DCF (现金流折现分析)**：这是金融界的“估值神器”。它的逻辑是：如果我能预测一家公司未来五年、十年能赚多少钱，再考虑到未来的钱不如现在的钱值钱（折现），把它们全部加总在一起，就能算出这家公司现在到底值多少钱。
> 2. **颜色编码 (Color Coding)**：在投行或专业财务分析中，Excel 里的每一个颜色都有含义。**蓝色**数字代表这是“手动打进去的原始数据”，**黑色**则代表“算出来的结果”。这样别人在看你的模型时，一眼就能发现哪些地方是可以调整的参数，哪些是不能乱动的逻辑。
> 3. **勾稽关系 (Ties & Balances)**：财务有三张大表（资产负债表、利润表、现金流量表），它们之间是环环相扣的。就像拼图一样，如果其中一个数据变了，其他地方必须跟着变且最终能“对得上”。如果“Checks”标签显示 FALSE，通常意味着你的模型逻辑出现了漏洞，需要检查。
