# 权益研究：业绩点评报告

创建专业的**业绩点评报告**，分析已覆盖公司的季度业绩，遵循机构级标准 (如摩根大通、高盛、摩根士坦利格式)。

**核心特征：**
- **篇幅**：8-12 页
- **字数**：3,000-5,000 字
- **表格**：1-3 个摘要表 (非详尽报表)
- **图表**：8-12 个图表
- **交付周期**：1-2 天 (财报发布后 24-48 小时内)
- **受众**：已熟悉该公司的客户
- **侧重点**：**增量**信息——实际业绩与预期对比、预测值调整、对投资逻辑的影响
- **字体**：全文使用 Times New Roman (除非用户另有说明)

## 适用场景

当用户请求以下内容时使用：
- “为 [公司] 2024 年第三季度业绩创建一份点评报告”
- “分析 [公司] 季度财务业绩”
- “[公司] 财报发布后点评”
- “[公司] Q1/Q2/Q3/Q4 业绩更新”

**在以下情况下不要使用：**
- 用户请求“首次覆盖报告 (initiation report)” → 使用其他对应技能
- 用户请求“快报 (flash note)”或“简评 (quick take)” → 使用不同格式
- 公司并未处于研究覆盖范围内 → 需先行发布首次覆盖报告

## 关键要求

### 1. 时效性
- 在业绩发布后 24-48 小时内发布
- 仅关注**新增**信息
- 不要冗长地重复公司背景介绍

### 2. 超预期/低于预期分析 (Beat/Miss Analysis)
- 首要说明公司业绩是超预期还是低于预期
- 量化偏差 (例如：“营收超出预期 1.2 亿美元，偏离度 3%”)
- 解释业绩与市场预期产生差异的**原因**

### 3. 摘要格式
- 表格限制在 1-3 个 (仅限摘要，而非完整报表)
- 无需提供完整的损益表/现金流量表/资产负债表 (仅列出核心指标)
- 假设读者已经阅读过首次覆盖报告

### 4. 引用与来源归属 ⭐⭐⭐ 强制要求

**至关重要**：使用**具体来源**和**可点击超链接**妥善引用所有数据。

**在每个图表和表格中包含具体引用及可点击链接：**

```
来源：2024年11月8日归档的 2024 Q3 10-Q 报表；公司业绩新闻稿
      [将 "10-Q" 链接至：https://www.sec.gov/cgi-bin/viewer?accession=...]
      [将 "业绩新闻稿" 链接至：https://investor.company.com/news/q3-2024]
```

**超链接在 Word 中的呈现方式：**
- 文件名显示为蓝色、带下划线的可点击链接
- 读者可以 Ctrl+Click 直接打开来源
- 禁止使用纯文本 URL——必须使用带显示文本的格式化超链接

**必须包含的来源列表：**

在每份业绩点评中引用：
- ✅ 业绩新闻稿 (带日期和 URL)
- ✅ 10-Q 季度报表 (带归档日期和 EDGAR 链接)
- ✅ 业绩电话会议纪要 (带日期)
- ✅ 投资者演示稿/补充材料 (如有)
- ✅ 市场共识预期来源 (Bloomberg/FactSet 等，带日期)
- ✅ 往期指引 (摘自上一季度材料)

**带可点击超链接的参考资料部分：**

在报告末尾包含“来源与参考资料”部分：

```
来源与参考资料

业绩相关材料 (2024 Q3):
• 业绩新闻稿 (2024年11月7日)
  [将整行链接至：https://investor.company.com/news/q3-2024-earnings]

• Form 10-Q (2024年11月8日归档)
  [链接至：https://www.sec.gov/cgi-bin/viewer?accession=...]

• 业绩电话会议纪要 (2024年11月7日)
  [链接至：https://seekingalpha.com/article/...]

• 投资者演示稿 (2024年11月7日)
  [链接至：https://investor.company.com/presentations/q3-2024.pdf]
```

**核对清单：**
- [ ] 每个图表均标明具体文件来源及日期
- [ ] 每个表格均标明文件引用来源
- [ ] 超预期/低于预期分析引用了带日期的市场共识来源
- [ ] 指引变更引用了当前及先前的指引来源
- [ ] 关键统计数据带有脚注
- [ ] 来源部分列出了所有带 URL 的材料
- [ ] 所有 URL 均为**可点击的超链接** (非纯文本)
- [ ] 所有 SEC 备案文件均链接至 EDGAR 查看器

### 5. 预期调整
- 根据最新业绩更新前瞻性预测
- 清晰显示“原预测”与“新预测”对比
- 解释调整内容及其原因

## 高阶工作流

业绩点评过程分为 5 个阶段：

### 阶段 1：数据采集 (30-60 分钟)

**🚨🚨🚨 核心警示：训练数据可能已过时 🚨🚨🚨**

**开始之前 - 请按顺序完成以下 4 个步骤：**
1. **确认今日日期** - 记下当前日期
2. **搜索最新信息** - 使用网页搜索："[公司名称] latest earnings results"
3. **验证日期** - 确认业绩发布于近 3 个月内
4. **核对纪要日期** - 验证电话会议纪要日期与业绩发布日期匹配

**常见错误**：直接使用训练数据中陈旧的业绩电话会议内容，而不是搜索最新的数据。

**要求：**
- ✅ 搜索最新业绩——不要依赖训练数据
- ✅ 记下今日日期和找到的发布日期
- ✅ 验证发布日期距离今天在 3 个月以内
- ✅ 验证纪要日期与发布日期一致
- ✅ 如果日期不匹配或已过时 (>3 个月)，请重新搜索

**参见 [references/workflow.md](references/workflow.md)** 获取详细的搜索程序和验证步骤。

### 阶段 2：深度分析 (2-3 小时)
- 针对每个核心指标进行超预期/低于预期分析
- 部门/区域/产品线明细拆解
- 利润率及指引分析
- 更新财务模型与预测值

**参见 [references/workflow.md](references/workflow.md)** 获取详细分析框架。

### 阶段 3：图表生成 (1-2 小时)
创建 8-12 个侧重于季度趋势和核心变量的图表：
- 季度营收趋势
- 季度 EPS (每股收益) 趋势
- 季度利润率趋势
- 分业务部门/地理区域营收拆解
- 关键运营指标
- 超预期对比摘要
- 预期修正对比
- 估值分析图表

**参见 [references/workflow.md](references/workflow.md)** 获取图表规格说明。

### 阶段 4：报告撰写 (2-3 小时)
创建具有特定结构的 8-12 页 DOCX 报告。

**参见 [references/report-structure.md](references/report-structure.md)** 获取完整的逐页模板和格式要求。

**高阶结构：**
- 第 1 页：业绩摘要、评级及目标价
- 第 2-3 页：详细业绩分析
- 第 4-5 页：核心指标及业绩指引
- 第 6-7 页：投资逻辑修正
- 第 8-10 页：估值分析及预期调整
- 第 11-12 页：附录 (可选)

### 阶段 5：质量核查与交付 (30 分钟)
在交付前核实内容、格式、准确性及及时性。

**参见 [references/best-practices.md](references/best-practices.md)** 获取质量核对清单及常见误区。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **权益研究 (Equity Research)**：简单来说，就是投行的分析师们专门研究各家公司的“股票值不值得买”。他们会写出厚厚的报告，给投资者提供买入或卖出的建议。
> 2. **业绩点评 (Earnings Update)**：公司每三个月会发一次“成绩单”。分析师在看完这份成绩单后，会第一时间写个短评，告诉大家这几个月公司到底是赚了还是亏了，和之前想的一样不一样。
> 3. **超预期 (Beat) vs 低于预期 (Miss)**：在发成绩单前，市场上的分析师会通过猜测给出一个“平均分”（市场预期）。如果公司实际得分比这个高，就是“Beat”；如果考砸了，就是“Miss”。这通常是股价涨跌的指挥棒。
> 4. **10-Q 报表**：这是美国上市公司必须向监管部门提交的正式季度财务报告。它比新闻稿更严肃、更详细，是分析师挖掘细节的“藏宝图”。
> 5. **指引 (Guidance)**：这是公司高管对未来一段时间业绩的自述。就像考试后对自己说“下学期我能考第一”，如果公司调高指引，通常意味着他们看好未来。
