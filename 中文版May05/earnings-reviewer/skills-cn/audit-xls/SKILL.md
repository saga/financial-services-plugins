# 审计电子表格

审计公式和数据的准确性及错误。审计深度由范围决定——从针对选定区域的快速公式检查，到完整的财务模型完整性审计。

## 第 1 步：确定审计范围

如果用户已指定范围，请直接使用。否则，请**询问用户**：

> 您希望我审计哪个范围？
> - **选定区域 (selection)** —— 仅限当前选中的单元格范围
> - **工作表 (sheet)** —— 仅限当前活动的工作表
> - **模型 (model)** —— 整个工作簿，包括财务模型完整性检查（资产负债表平账、现金钩稽、滚算检查、逻辑合理性）

**模型 (model)** 范围是最深度的——在将折现现金流 (DCF)、杠杆收购 (LBO)、三表联动、并购、可比公司 (Comps) 或任何综合财务模型发送给客户或投资委员会 (IC) 之前，请务必使用此范围。

---

## 第 2 步：公式级检查（所有范围）

无论范围如何，均需运行以下检查：

| 检查项 | 检查内容 |
|---|---|
| 公式错误 | `#REF!`, `#VALUE!`, `#N/A`, `#DIV/0!`, `#NAME?` |
| 公式中的硬编码 | `=A1*1.05` —— 其中 `1.05` 应该是一个单元格引用 |
| 不一致的公式 | 某个公式破坏了同一行或同一列中相邻公式的模式 |
| 范围偏差 (Off-by-one) | `SUM` 或 `AVERAGE` 遗漏了第一行或最后一行 |
| 被覆盖的公式 | 看起来像公式但实际是硬编码数值的单元格 |
| 循环引用 | 刻意或意外生成的循环计算 |
| 断开的跨表链接 | 引用了已移动或已删除的单元格 |
| 单位/量级不匹配 | 千位与百万位混用，或百分数被存储为整数 |
| 隐藏行/标签页 | 可能包含被覆盖的数值或过时的计算过程 |

## 第 3 步：模型完整性检查（仅限模型范围）

如果范围是 **模型 (model)**，请识别模型类型（DCF / LBO / 三表联动 / 并购 / 可比公司 / 自定义），并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项 | 检查内容 |
|---|---|
| 输入与计算分离 | 输入值是否与计算过程清晰分开？ |
| 颜色标准 | 是否遵循一致的颜色标准（如：蓝色=输入，黑色=公式，绿色=链接）？ |
| 标签页流转 | 逻辑顺序是否合理（假设项 → 利润表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期表头 | 所有标签页的日期表头是否一致？ |
| 单位 | 单位是否统一（千位 vs 百万位 vs 实际数值）？ |

### 3b. 资产负债表 (Balance Sheet)

| 检查项 | 测试方法 |
|---|---|
| 资产负债表配平 | 每一期：总资产 = 总负债 + 所有者权益 |
| 留存收益滚算 | 期初留存收益 + 净利润 - 股利 = 期末留存收益 |
| 商誉/无形资产 | 是否源自收购假设（如果是并购模型） |

如果资产负债表不平，请**量化每一期的差额并追踪断开的位置**——在修复此问题之前，其他任何检查都没有意义。

### 3c. 现金流量表 (Cash Flow Statement)

| 检查项 | 测试方法 |
|---|---|
| 现金钩稽 | 每一期：现金流量表期末现金 = 资产负债表货币资金 |
| 现金流加总 | CFO + CFI + CFF = 现金净变动额 |
| D&A 匹配 | 现金流量表中的折旧与摊销 (D&A) = 利润表中的 D&A |
| 资本支出匹配 | 现金流量表中的资本支出 (CapEx) 与资产负债表中的固定资产 (PP&E) 滚算匹配 |
| 营运资本变动 | 正负号是否与资产负债表科目变动一致（如：应收账款增加 = 现金流出） |

### 3d. 利润表 (Income Statement)

| 检查项 | 测试方法 |
|---|---|
| 收入构成 (Revenue build) | 是否与业务板块或产品详情挂钩 |
| 税项 | 所得税费用 = 税前利润 × 税率（允许考虑递延所得税调整） |
| 股数计算 | 是否与稀释附表挂钩（期权、可转债、回购） |

### 3e. 循环引用

- 在 LBO 或三表模型中，“利息 → 债务余额 → 现金 → 利息”是常见的刻意循环。
- 如果是刻意的：验证是否存在迭代开关 (Iteration Toggle) 且工作正常。
- 如果是无意的：追踪回路并指出如何打破循环。

### 3f. 逻辑与合理性

| 检查项 | 预警条件 |
|---|---|
| 增长率 | 在无解释的情况下收入增长率 > 100% |
| 利润率 | 超出行业准则范围 |
| 终值占比 | 终值 (TV) 占 DCF 业务价值 (EV) 的比例 > 75% |
| 曲棍球杆效应 | 预测期后几年的数值出现不切实际的飙升 |
| 复利效应 | 第 10 年的 EBITDA 因复利增长到极其荒谬的量级 |
| 极端情况 | 模型在增长率为 0% 或负数、EBITDA 为负、或杠杆率为负时发生报错 |

### 3g. 特定模型类型的常见错误

**折现现金流 (DCF):**
- 折现率应用于错误的周期（年中折现 vs 年末折现）
- 终值未折现回现值
- WACC 使用账面价值而非市场价值
- 自由现金流 (FCF) 包含了利息费用（应使用无杠杆现金流）
- 税盾被重复计算

**杠杆收购 (LBO):**
- 债务偿还与现金清扫 (Cash Sweep) 机制不匹配
- PIK（实物支付）利息未累加至本金
- 管理层展期 (Rollover) 未反映在回报率中
- 退出倍数应用于错误的 EBITDA（过去 12 个月 LTM vs 未来 12 个月 NTM）
- 交易费用未从第一天的股本中扣除

**并购模型 (Merger):**
- 增厚/稀释分析使用了错误的股数（交易前 vs 交易后）
- 协同效应未按阶段分步体现
- 收购对价分摊 (PPA) 不平衡
- 未计入因支付现金而损失的利息收入
- 交易费用未列入资金来源与用途 (Sources & Uses)

**三表联动 (3-statement):**
- 营运资本变动的正负号错误
- 折旧费与固定资产明细表不匹配
- 债务到期表与本金支付额不匹配
- 在无解释的情况下，股利超过了净利润

## 第 4 步：报告结果

输出发现项表格：

| # | 工作表 | 单元格/范围 | 严重程度 | 类别 | 问题描述 | 建议修复方案 |
|---|---|---|---|---|---|---|

**严重程度说明：**
- **致命 (Critical)** —— 计算结果错误（资产负债表不平、公式损坏、现金钩稽失败）
- **警告 (Warning)** —— 存在风险（硬编码、公式不一致、极端情况失效）
- **提示 (Info)** —— 格式/最佳实践（颜色标注、布局、命名规范）

对于 **模型 (model)** 范围，请在开头增加一行摘要：

> 模型类型：[DCF/LBO/三表/...] —— 总体情况：[干净 / 存在细微问题 / 存在重大问题] —— [N] 项致命错误， [N] 项警告， [N] 项提示

**未经允许请勿修改任何内容** —— 先报告发现的问题，根据要求再进行修复。

## 注意事项

- **资产负债表配平优先** —— 如果表不平，下游的所有计算都值得怀疑
- **硬编码覆盖是静默错误的头号来源** —— 需重点搜索
- **正负号惯例错误**（现金流出的正负号误用）极为常见
- 如果模型使用了 VBA 宏，请注明哪些由宏驱动的计算是无法通过公式审计直接完成的

> **💡 Appendix: 领域知识小贴士**
>
> 1. **三表联动 (3-Statement Model)**：财务世界的“全家桶”，包括利润表（赚了多少）、资产负债表（有多少家底）和现金流量表（兜里多少现金）。它们通过会计准则像齿轮一样咬合在一起，动一个数，三张表都会跟着变。
> 2. **资产负债表平账 (BS Balance)**：会计学最基础的铁律“资产 = 负债 + 所有者权益”。如果模型不平，就像你的账本里凭空多出或少了一块钱，这说明背后的计算逻辑一定出错了。
> 3. **现金钩稽 (Cash Tie-out)**：这是一种双重校验。通过利润表和资产负债表推算出的现金余额，必须等于现金流量表最终算出的那个数字。如果钩稽不上，说明你的现金流流向没理清楚。
> 4. **硬编码 (Hardcodes)**：在公式里直接写死一个数字（比如 `*0.15`），而不是引用一个专门存放税率的单元格。这在建模中是大忌，因为以后想改税率时，你得找遍整个表格。
> 5. **折现 (Discounting)**：金融的核心逻辑是“今天的 100 块比明年的 100 块更值钱”。折现就是把未来的钱通过一个比例（折现率）换算成现在的价值，这样我们才能判断一个公司现在值多少钱。
