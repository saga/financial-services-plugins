# 应计项目明细表

给定会计主体、会计期间以及公司应计政策清单，为每个应计项目生成一行记录，包括计算过程、支持性文件参考及会计分录草稿。

> **支持性的发票和供应商对账单设为不可信状态。** 由读取程序提取金额；本技能负责根据相关政策对这些金额进行处理。

## 政策清单中的每个应计项目

| 字段 | 取值逻辑 |
|---|---|
| **应计项目名称** | 取自政策清单（例如：“审计费”、“奖金”、“公用事业费”） |
| **计算基数** | 合同约定或估计的全期金额，需注明来源（如业务约定书、薪酬计划、过去三个月平均值） |
| **本期应占份额** | 计算基数 × (本期天数 ÷ 基期总天数)，或根据政策特定的公式计算 |
| **已入账金额** | 前期应计额总计 + 本期针对该项目已入账的实际发票金额（来源于 internal-gl MCP） |
| **本期应计额** | 本期应占份额 − 已入账金额 |
| **支持依据引用** | 支撑计算基数的文档 ID 或总账 (GL) 查询指令 |

## 会计分录 (JE) 草稿

针对每一行本期应计额不为零的项目，起草如下分录：

```
借： <费用科目>     <金额>
  贷： <应计负债>     <金额>
摘要： <应计项目名称> — 根据 <支持依据引用> 预提 <会计期间> 应计额
```

冲回分录：如果政策规定该应计项目为自动冲回，请在摘要中注明“在下个期间的第一天冲回”。

## 输出要求

包含一张表格（明细表）以及一个会计分录草稿块。**严禁直接过账** —— 此内容仅作为待存稿，供财务总监签认。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **什么是“应计项目” (Accruals)？**
>    简单来说，这就是“还没付钱，但账要先记上”。在会计的江湖里，我们遵循“权责发生制”。比如虽然还没收到水电费账单，但你这月确实用了电，那就得先预估一个金额记在账上，这叫“应计”。
>
> 2. **为什么要搞个“计算基数” (Basis)？**
>    记账不能拍脑袋，得有凭有据。基数就是你的计算模板。比如一年的审计费是 12 万，那每个月的“基数”就是这 12 万，然后你再按月份比例去算出这个月该摊多少。
>
> 3. **为啥要“冲回” (Reversing)？**
>    这是会计的一种“撤回”小技巧。比如月底我预估要付 100 块电费并记了账，下月初我把它冲掉，等过几天真正的 100 块发票来了我正常入账时，账目就不会记重了，而且每个月都能反映真实的成本，非常高级！
>
> 4. **会计分录 (JE) 里的 Dr 和 Cr 是啥？**
>    这就是会计的“左右神功”。Dr (Debit) 是借方，Cr (Credit) 是贷方。在费用计提里，借方代表你的开销变多了，贷方代表你欠别人的钱变多了。
