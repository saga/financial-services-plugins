# 可比公司分析 (Comparable Company Analysis)

## ⚠️ 关键：数据源优先级 (优先阅读)

**务必遵循以下数据源层级：**

1. **第一优先：检查 MCP 数据源** - 如果可以使用 S&P Kensho MCP、FactSet MCP 或 Daloopa MCP，请专精于使用这些来源获取财务和交易信息。
2. **严禁在已有上述 MCP 数据源的情况下使用网页搜索**。
3. **仅在 MCP 不可用时：** 才可使用彭博终端（Bloomberg Terminal）、SEC EDGAR 申报文件或其他机构级来源。
4. **绝不将网页搜索作为主要数据源** - 其缺乏机构级分析所需的准确性、审计追踪和可靠性。

**原因：** MCP 来源提供经过验证、带有正确引用标识的机构级数据。网页搜索结果可能早已过时、不准确，或在财务分析中不可靠。

---

## 概述
本技能旨在指导 Claude 构建结合了运营指标、估值倍数和统计基准分析的机构级可比公司分析清单。最终产出为一个结构化的 Excel 或电子表格，通过同行对比辅助明智的投资决策。

**参考材料与情境化：**

`examples/comps_example.xlsx` 中提供了一个可比公司分析示例。在参考该目录下的示例文件时，请灵活运用：

**建议将示例用于：**
- 理解结构层级（各板块的流向）
- 掌握所需的严谨程度（统计深度、文档标准）
- 学习原则（清晰的表头、透明的公式、审计追踪）

**严禁将示例用于：**
- 机械套用格式或指标
- 不顾语境地复制布局
- 无论受众是谁都应用相同的视觉风格

**请务必先行思考以下问题：**
1. **“您是否有偏好的格式，还是我应该根据模板风格进行适配？”**
2. **“受众是谁？”**（投资委员会、董事会演示、快速查阅、详细备忘录）
3. **“核心问题是什么？”**（估值、增长分析、竞争地位、效率分析）
4. **“背景是什么？”**（并购评估、投资决策、行业基准、业绩回顾）

**基于具体情况进行调整：**
- **行业背景**：大型科技股与新兴 SaaS 初创公司所需的指标截然不同。
- **行业特定需求**：及早加入相关指标（例如科技行业的云 ARR、企业客户数、开发者生态等）。
- **公司熟悉度**：知名公司可能不需要太多背景介绍，而应侧重于差异分析。
- **决策类型**：并购分析的重点与日常投后管理不同。

**核心原则：** 沿用模板原则（清晰结构、统计严谨、公式透明），但在执行上根据情境灵活变化。目标是实现“机构级质量”的分析，而非仅仅是“机构样式的”模板。

用户提供的示例和明确偏好始终优先于默认设置。

## 核心理念
**“先建立正确的结构，再让数据说话。”**

从强制引导战略性思考的表头开始，输入干净的数据，建立透明的公式，让统计结果自然呈现。一份优秀的可比分析应当让未参与制作的人也能立刻读懂。

## ⚠️ 关键：公式优于硬编码 + 分步验证

**环境说明 — Office JS 对比 Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS (`Excel.run(async (context) => {...})`)。通过 `range.formulas = [["=E7/C7"]]` 写入公式，而非 `range.values`。无需单独的重算步骤 —— Excel 会原生处理。使用 `range.format.*` 设置颜色/字体。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。写入 `cell.value = "=E7/C7"` (公式字符串)。
- 无论哪种方式，原则一致 —— 只需转换 API 调用即可。
- **Office JS 合并单元格陷阱：** 不要先调用 `.merge()` 再对合并范围设置 `.values`（会抛出 `InvalidArgument` 错误 —— 范围仍记录合并前的维度）。应先仅向左上角单元格写入值，然后再合并并格式化完整范围：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式而非硬编码：**
- 每个派生值（利润率、倍数、统计量）必须是引用输入单元格的 Excel 公式 —— 绝不能粘贴预先计算好的数字。
- 使用 Python/openpyxl 构建表格时：写入 `cell.value = "=E7/C7"` (公式字符串)，而不是 `cell.value = 0.687` (计算结果)。
- 唯一允许硬编码的值是原始输入数据（营收、EBITDA、股价等） —— 且每一个此类数值都要添加带有来源说明的单元格批注。
- 原因：当输入值变化时，模型必须自动更新。硬编码的利润率是潜伏的 Bug。

**与用户进行分步验证：**
- 设置好结构后 → 在填充数据前向用户展示表头布局。
- 输入原始数据后 → 展示输入区块，并在建立公式前确认来源/期间。
- 建立运营指标公式后 → 展示计算出的利润率，并在进入估值板块前与用户进行合理性检查。
- 建立估值倍数后 → 展示倍数，并在添加统计量前确认其看起来合理。
- 不要直接从头到尾建完整个表再提交 —— 通过确认每个板块来及早发现错误。

## 第一板块：文档结构与设置

### 标题块 (1-3 行)
```
第 1 行：[分析标题] - 可比公司分析
第 2 行：[公司列表及股票代码] • [公司 1 (TICK1)] • [公司 2 (TICK2)] • [公司 3 (TICK3)]
第 3 行：截至 [期间] | 除每股金额和比率外，所有金额单位均为 [百万/十亿美元 USD]
```

**原因：** 立即建立上下文。任何打开此文件的人都能知道他们在看什么、何时创建的以及如何解读数字。

### 视觉规范标准 (可选 - 用户偏好和上传的模板始终优先)

**重要：以下仅为建议的默认设置。请始终优先考虑：**
1. 用户的明确格式偏好
2. 任何上传模板文件的格式
3. 公司/团队的风格指南
4. 这些默认值（仅在没有其他指导时使用）

**建议字体与排版：**
- **字体族**：Times New Roman（专业、清晰、行业标准）
- **字体大小**：数据单元格 11pt，表头 12pt
- **加粗文本**：板块标题、公司名称、统计量标签

**默认颜色与阴影 — 专业蓝/灰调（极简为佳）：**
- **保持克制** — 仅使用蓝色和灰色。严禁加入绿色、橙色、红色或多种强调色。一份干净的可比分析表总共只应使用 3-4 种颜色。
- **板块标题**（例如“OPERATING STATISTICS & FINANCIAL METRICS”）：
  - 深蓝色背景 (`#1F4E79` 或 `#17365D` 海军蓝)
  - 白色加粗文字
  - 跨所有列的全行阴影
- **列标题**（例如“Company”, “Revenue”, “Margin”）：
  - 浅蓝色背景 (`#D9E1F2` 或类似的淡蓝色)
  - 黑色加粗文字
  - 居中对齐
- **数据行**：
  - 公司数据采用白色背景
  - 公式采用黑色文字；硬编码输入采用蓝色文字
- **统计行**（最大值、75 分位数等）：
  - 浅灰色背景 (`#F2F2F2`)
  - 黑色文字，标签左对齐
- **这就是全部调色板**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户模板另有要求，否则不添加其他颜色。

**建议格式规范：**
- **小数精度**：
  - 百分比：1 位小数 (12.3%)
  - 倍数：1 位小数 (13.5x)
  - 金额：无小数，使用千分位分隔符 (69,632)
  - 利润率展示为百分比：1 位小数 (68.7%)
- **边框**：无边框（洁净、极简的外观）
- **对齐**：所有指标均居中对齐，以保证整齐统一
- **单元格尺寸**：所有列宽需统一/平均，所有行高保持一致（营造专业的网格感）

**注意：** 如果用户提供了模板文件或指定了不同的格式，请改用用户的格式。

## 第二板块：运营统计与财务指标

### 核心列 (以此开始)
1. **Company (公司)** - 格式统一的名称
2. **Revenue (营收)** - 规模指标（根据语境可选 LTM、季度或年度）
3. **Revenue Growth (营收增长)** - 同比百分比变化
4. **Gross Profit (毛利)** - 营收减去销售成本 (COGS)
5. **Gross Margin (毛利率)** - 毛利/营收（基础盈利能力）
6. **EBITDA** - 息税折旧摊销前利润
7. **EBITDA Margin (EBITDA 利润率)** - EBITDA/营收（运营效率）

### 可选补充项 (根据行业/目的选择)
- **季度对比 LTM** - 若存在季节性因素，两者均需包含
- **Free Cash Flow (自由现金流)** - 适用于资本密集型或 SaaS 业务
- **FCF Margin (FCF 利润率)** - FCF/营收（现金生成效率）
- **Net Income (净利润)** - 适用于成熟、盈利的公司
- **Operating Income (营业收入)** - 适用于折旧摊销 (D&A) 差异较大的业务
- **CapEx 指标** - 适用于重资产行业
- **Rule of 40 (40 法则)** - 专用于 SaaS（增长率% + 利润率%）
- **FCF Conversion (FCF 转化率)** - 用于收益质量分析（进阶）

### 公式示例 (以第 7 行为例)
```excel
// 核心比率 - 始终计算得出
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率 - 若相关则包含
FCF 利润率: =[FCF]/[营收]
净利率: =[净利润]/[营收]
40 法则: =[增长率 %]+[FCF 利润率 %]
```

**金科玉律：** 每个比率都应该是 [某指标] / [营收] 或 [某指标] / [本表中的另一指标]。保持简单。

### 统计区块 (在公司数据之后)

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留出一个空行以视觉分隔]
- 最大值 (Maximum): =MAX(B7:B9)
- 75 分位数 (75th Percentile): =QUARTILE(B7:B9,3)
- 中位数 (Median): =MEDIAN(B7:B9)
- 25 分位数 (25th Percentile): =QUARTILE(B7:B9,1)
- 最小值 (Minimum): =MIN(B7:B9)
```

**需要统计量的列（可比指标）：**
- 营收增长 %、毛利率 %、EBITDA 利润率 %、EPS
- EV/Revenue, EV/EBITDA, P/E, 股息率 %, Beta

**不需要统计量的列（规模指标）：**
- 营收、EBITDA、净利润（绝对规模随公司体量变化）
- 市值、企业价值（不同体量的公司不具备可比性）

**注意：** 在公司数据和统计行之间添加一个空行。不要添加“行业统计”或“估值统计”标题行。

**原因：** 分位数展示的是分布情况而非仅仅是平均值。75 分位数的倍数可以告诉你“溢价”公司的交易水平。

## 第三板块：估值倍数与投资指标

### 核心估值列 (以此开始)
1. **Company (公司)** - 顺序与运营板块一致
2. **Market Cap (市值)** - 当前市场估值
3. **Enterprise Value (企业价值, EV)** - 市值 ± 净负债/现金
4. **EV/Revenue** - 市场为每 1 美元营收支付的倍数
5. **EV/EBITDA** - 市场为每 1 美元收益支付的倍数
6. **P/E Ratio (市盈率)** - 价格相对于净收益的比例

### 可选估值指标 (根据语境选择)
- **FCF Yield (自由现金流收益率)** - FCF/市值（用于关注现金流的分析）
- **PEG Ratio (PEG 指标)** - P/E / 增长率（用于高增长公司）
- **Price/Book (市净率)** - 市值对比账面价值（用于重资产业务）
- **ROE/ROA (净资产收益率/资产收益率)** - 回报指标（用于盈利能力对比）
- **Revenue/EBITDA CAGR** - 历史复合年增长率（用于趋势分析）
- **Asset Turnover (资产周转率)** - 营收/资产（用于衡量运营效率）
- **Debt/Equity (产权比率)** - 杠杆率（用于资本结构分析）

**关键原则：** 包含 3-5 个对该行业至关重要的核心倍数。不要仅仅因为可以计算就堆砌所有指标。

### 公式示例
```excel
// 核心倍数 - 始终包含
EV/Revenue: =[企业价值]/[LTM 营收]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
市盈率: =[市值]/[净利润]

// 可选倍数 - 若数据可用则包含
FCF 收益率: =[LTM FCF]/[市值]
PEG 指标: =[市盈率]/[增长率 %]
```

### 交叉引用规则
**关键：** 估值倍数必须引用运营指标板块。严禁重复输入相同的原始数据。如果营收在 C7，则 EV/Revenue 公式必须引用 C7。

### 统计区块
结构与运营板块一致：为每个指标计算最大值、75 分位、中位、25 分位和最小值。在公司数据和统计量之间留一空行。不要添加“估值统计”标题行。

## 第四板块：备注与方法论说明

### 必填项

**数据源与质量：**
- 数据来源（S&P Kensho MCP, FactSet MCP, Daloopa MCP, 彭博, SEC 申报文件）。
- 涵盖的期间（例如 2024 年第四季度，经审计数据）。
- 如何验证？（与 10-K/10-Q 交叉核对）。
- 注：优先使用 MCP 数据源以获得更高准确性和可追溯性。

**关键定义：**
- EBITDA 计算方法（毛利 + D&A，或营业利润 + D&A）。
- 自由现金流公式（经营性现金流 - 资本支出）。
- 特殊指标说明（40 法则、FCF 转化率）。
- 期间定义（LTM、CAGR 计算周期）。

**估值方法：**
- 企业价值如何计算？（市值 + 净负债）。
- 使用了哪些增长率？（历史 CAGR、前瞻预测）。
- 是否有任何调整？（剔除一次性项目、归一化利润率）。

**分析框架：**
- 投资逻辑是什么？（例如：云/SaaS 效率）。
- 哪些指标最重要？（现金产生能力、资本效率）。
- 读者应如何解读这些统计数据？（分位数提供的语境）。

## 第五板块：选择正确指标 (决策框架)

### 从“我要回答什么问题？”开始

**“哪家公司被低估了？”**
→ 重点关注：EV/Revenue, EV/EBITDA, P/E, 市值
→ 可跳过：运营细节、增长指标

**“哪家公司最高效？”**
→ 重点关注：毛利率、EBITDA 利润率、FCF 利润率、资产周转率
→ 可跳过：规模指标、绝对金额

**“哪家公司增长最快？”**
→ 重点关注：营收增长 %、EBITDA CAGR、用户/客户增长
→ 可跳过：利润率指标、杠杆率

**“哪家公司的现金生成能力最强？”**
→ 重点关注：FCF、FCF 利润率、FCF 转化率、资本支出强度
→ 可跳过：EBITDA、市盈率

### 行业特定指标选择

**软件/SaaS:**
必选：营收增长率、毛利率、40 法则
可选：ARR、净金额留存率 (NDR)、CAC 回本周期
跳过：资产周转率、库存指标

**制造业/工业:**
必选：EBITDA 利润率、资产周转率、资本支出/营收
可选：ROA、库存周转率、在手订单
跳过：40 法则、SaaS 指标

**金融服务:**
必选：ROE、ROA、成本收入比 (Efficiency Ratio)、P/E
可选：净息差 (NIM)、贷款损失拨备
跳过：毛利率、EBITDA（对银行无意义）

**零售/电子商务:**
必选：营收增长率、毛利率、库存周转率
可选：同店销售增长、客户获取成本 (CAC)
跳过：高额研发或资本支出指标

### “5-10 原则”

**5 个运营指标** - 营收、增长、2-3 个利润率/效率指标
**5 个估值指标** - 市值、EV、3 个倍数
**= 合计 10 列** - 足以说明问题，且不至于丢失重点

如果你的指标超过 15 个，说明包含了干扰项。请无情地删减。

## 第六板块：最佳实践与质量检查

### 开始之前
1. **定义同行组** - 公司必须具备真正的可比性（相似的商业模式、规模、地域）。
2. **选择正确期间** - LTM 可平滑季节性；季度数据可展示趋势。
3. **预先统一单位** - 选择百万还是十亿将影响整张表。
4. **映射数据源** - 清晰每个数字的来源。

### 构建过程中
1. **先输入所有原始数据** - 在编写公式前完成蓝色文字填充。
2. **为所有硬编码输入添加单元格批注** - 右键单元格 → 插入批注 → 记录来源或假设。

   **对于获取的数据，需精确引用：**
   - 示例：“彭博终端 - MSFT Equity DES, 访问日期 2024-10-02”
   - 示例：“2024 年 Q4 10-K 申报文件, 第 42 页, ‘Total Revenue’ 项”
   - 示例：“FactSet 一致预期，截至 2024-10-02”
   - **尽可能包含超链接**：右键单元格 → 链接 → 粘贴 SEC 申报、数据源或报告的 URL。

   **对于假设，需说明理由：**
   - 示例：“基于同行中位数假设 EBITDA 利润率为 15%，公司未披露该项数据”
   - 示例：“估计企业价值 = 市值 + $50M 净负债（源自 Q3 资产负债表，Q4 尚未公布）”
   - 示例：“前瞻市盈率基于市场一致预期的每股收益 $3.45（12 位分析师预期的均值）”

   **原因**：便于审计追踪、数据验证、假设透明化及后续更新。
3. **分行建立公式** - 在进行下一步前测试每个计算。
4. **为表头使用绝对引用** - `$C$6` 用于锁定标题行。
5. **格式保持一致** - 百分比展示为百分比，而非小数。
6. **添加条件格式** - 自动高亮异常值。

### 合理性检查 (Sanity Checks)
- **利润率测试**：毛利率 > EBITDA 利润率 > 净利率（定义上始终成立）。
- **倍数合理性**：
  - EV/Revenue：通常为 0.5-20x（因行业而异）。
  - EV/EBITDA：通常为 8-25x（跨行业相对一致）。
  - P/E：通常为 10-50x（取决于增长率）。
- **增长与倍数的相关性**：高增长通常意味着高倍数。
- **体量与效率的取舍**：大公司通常拥有更好的利润率（规模效益）。

### 应避免的常见错误
❌ 在公式中混淆市值 (Market Cap) 和企业价值 (EV)
❌ 分子和分母采用不同的时间跨度（LTM 对比季度）
❌ 在公式中直接使用硬编码数字而非单元格引用
❌ **硬编码输入未附带引用来源或解释假设的批注**
❌ 在有条件时缺失 SEC 文件或数据源的超链接
❌ 包含过多没有明确用途的指标
❌ 包含不可比公司（商业模式不同）
❌ 使用过时数据且未予披露
❌ 错误地计算百分比的平均值（应当使用中位数）

### 数据质量问题
🚩 期间不一致（混合使用了季度和年度数据）

🚩 对 EBITDA 为负的公司使用 EBITDA 倍数（应改用营收倍数）

🚩 利润率完全不符合行业常识

🚩 财年截止日不同（导致时间错位）
🚩 将纯业务公司与综合性集团混在一起
🚩 将商业模式本质不同的公司强行标记为“可比对象”

## 核心原则总结

1. **结构驱动洞察** - 正确的表头强制正确的思考。
2. **少即是多** - 10 个关键指标胜过 20 个无关紧要的数字。
3. **针对问题选指标** - 估值分析不等同于效率分析。
4. **统计揭示模式** - 中位数和分位数比平均值更能说明问题。
5. **透明胜过复杂** - 使用所有人都能读懂的简单公式。
6. **可比性至上** - 宁可缺项也不要强行对比不可比的公司。
7. **记录你的选择** - 在备注板块解释选择了哪些指标及其原因。

> **💡 Appendix: 领域知识小贴士**
>
> 想要看懂这份专业表格？其实这些高大上的名词并不难理解：
>
> 1. **什么是可比公司分析 (Comps)？**
>    想象一下你要卖掉自己的房子。你怎么定价？你肯定会看周围邻居最近卖了多少钱，这就是“可比分析”。在金融界，我们要给一家公司估值时，也会找它的“邻居”（同行业、规模相似的公司），看看大家在市场上的身价是多少。
>
> 2. **EBITDA 到底是什么？**
>    它的全称长得吓人，但本质上它是**“刨去杂质后的利润”**。有些公司借钱多（利息高），有些公司设备贵（折旧高），这会干扰我们看它赚钱的真实能力。EBITDA 把利息、税、折旧这些干扰项先踢开，让我们能公平地对比两家公司在核心业务上谁更赚钱。
>
> 3. **估值倍数 (Multiples) 是什么意思？**
>    这就是**“回本倍数”**。比如“市盈率 (P/E)”是 20，意思就是：如果你买下这家公司，按现在的盈利速度，你得花 20 年才能把本钱赚回来。在对比时，如果同行都是 15 倍，而这家是 20 倍，说明大家觉得它更有前途，愿意多花点钱：或者是它被由于某种原因被“炒作”得太贵了。
>
> 4. **为什么要看分位数 (Quartile)？**
>    只看平均数会由于一个特大号的“土豪”公司而被带偏。分位数能告诉我们一个区间：前 25% 的优等生表现如何，中间的普通学生表现如何。这能帮你判断目标公司到底是在班里名列前茅，还是在拖后腿。
