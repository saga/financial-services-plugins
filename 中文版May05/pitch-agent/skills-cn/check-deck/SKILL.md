# 投行文稿检查员 (IB Deck Checker)

对演示文稿进行跨维度的全面质量控制（QC）。阅读每页幻灯片，并报告审核结果。

## 环境检查

本技能适用于 PowerPoint 插件及聊天窗口。开始前请确认所处环境：

- **插件端** — 直接读取当前打开的在线文稿。
- **聊天端** — 读取上传的 `.pptx` 文件。

本功能仅限“读取并报告”，不涉及直接编辑，因此两者的工作流完全一致。

## 工作流

### 读取文稿

提取每张幻灯片的文本，并记录每行内容所属的页码。所有发现都必须关联到具体页码（例如：“5亿美元同时出现在第3页和第8页，但第15页显示为4.85亿美元”）。对于超过30页的文稿，仅凭工作记忆难以确保准确性 — 请将提取的文本写入文件，以便数据核对脚本进行处理。

脚本需要带有幻灯片标记的类 Markdown 格式，示例：

```
## Slide 1
[幻灯片 1 的文本内容]

## Slide 2
[幻灯片 2 的文本内容]
```

### 1. 数据一致性

对收集到的内容运行提取脚本：

```bash
python scripts/extract_numbers.py /tmp/deck_content.md --check
```

该脚本会自动实现单位标准化（$500M vs $500MM vs $500,000,000 → 识别为同一数值），对数值进行分类（收入、EBITDA、倍数、利润率），并指出同一指标类别的数值在不同页面是否存在冲突。这是最能捕捉到人类在多次校对后仍可能疏漏的环节。

除脚本自动标识的内容外，还需人工验证：
- 计算逻辑是否正确（各项之和是否等于总计，百分比加总是否正确，增长率是否与端点值匹配）
- 单位风格是否统一 — 演示文稿应统一使用 $M 或 $MM，不得混用
- 时间区间是否匹配 — 明确区分财年（FY）、最近12个月（LTM）或季度数据，并清晰标注

### 2. 数据与叙事一致性

将文稿陈述与支撑数据进行映射。这是演示文稿最容易出现隐性错误的环节 — 比如有人修改了第7页的图表，却忘记了第4页的文字描述。

- 趋势表述（如“利润率下滑”） — 图表走势是否确实向下？
- 市场地位断言（如“行业排名第一”） — 收入和市场份额数据是否支持该结论？
- 合理性校验 — 宣称在“1000亿美元规模的市场中排名第一”但年收入仅2亿美元，意味着份额只有0.2%，这显然不符合“排名第一”的常识。

### 3. 语言润色

投行演示文稿有其独特的语域。扫描并识别任何破坏专业感的表达：俚语或过于随意的用词（如 "pretty good", "a lot of"）、缩写、感叹号、缺乏数据支持的模糊定性词、或同一概念的前后称呼不一。

参考 `references/ib-terminology.md` 获取标准替换方案。

### 4. 视觉与格式质检 (QC)

对每页幻灯片进行标准的视觉校验，重点检查：缺失的图表数据来源（Source）、缺失的坐标轴标签、字体字号不统一、数字格式漂移（如同一文档内混用 1,000 和 1K）、日期格式不一、脚注（Footnote）及免责声明（Disclaimer）遗漏。

视觉校验能发现文本提取无法捕捉的重叠、溢出及对比度问题。切勿跳过此项 — 在文本汇总中，一个没有来源标注的图表看起来与标注完整的图表完全一样。

## 输出结果

使用 `references/report-format.md` 定义的结构，并按严重程度分类：

- **严重 (Critical)** — 数据不匹配、事实性错误、数据与叙述相矛盾。这些问题将阻碍交付给客户。
- **重要 (Important)** — 语言表达问题、缺失来源、术语前后不一。建议修改。
- **轻微 (Minor)** — 字体大小、间距、日期格式。属于润色范畴。

优先列出严重问题。如果没有发现，请明确说明 — “未发现数据一致性问题”本身就是一个重要的审核结论。

> **💡 Appendix: 领域知识小贴士**
>
> *   **投行演示文稿 (Pitch Deck)**：这是投行的“敲门砖”，用来向客户展示交易方案、行业洞察或公司价值。每一页都价值连城，任何小瑕疵都会显得不够专业，所以容错率极低。
> *   **数据勾稽 (Reconciliation)**：金融里的“对账”。比如第5页说今年赚了1亿，第20页的图表就不能显示是9千万。这种低级错误在投行是绝对的红线，会被视为严重的质量事故。
> *   **EBITDA**：税息折旧及摊销前利润。简单理解就是抛开税收和资产磨损这些账面因素，公司业务本身通过运营到底能“抓”多少现金回来。它是衡量公司赚钱能力的核心指标。
> *   **LTM (Last Twelve Months)**：最近十二个月。金融界不只看自然的日历年（1-12月），更喜欢看从此时此刻往前倒数12个月的业绩，因为这样得到的数据最“新鲜”，能反映最新的状态。
> *   **QC (Quality Control)**：在投行，这通常意味着“全手工像素级检查”。初级员工会盯着每一个逗号、每一个单位缩写，确保整份文稿看起来精准得像刚从精密仪器里打印出来的一样。
