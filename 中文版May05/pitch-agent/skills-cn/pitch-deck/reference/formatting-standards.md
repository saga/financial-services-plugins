## 视觉层级与布局

### 模块与章节布局

幻灯片布局因内容需求和模板设计而异。常见元素包括：
- 包含标题和副标题的页眉区域
- 带有标签侧边栏的内容模块
- 用于结构化数据的表格
- 用于数据视觉呈现的图表
- 幻灯片底部的脚注栏

具体的布局应遵循所提供的模板。常见内容类型及其典型结构：
- **市场定义幻灯片**：带有要点内容的标签框 + 评论区域
- **TAM/市场规模幻灯片**：指标凸显 + 数据表格 + 核心要点
- **竞品分析**：对比表或矩阵
- **财务摘要**：图表及配套数据表格

### 对齐原则

**平行章节的垂直对齐：**

垂直堆叠的模块应保持以下要素的一致性：
- 左页边距位置
- 项目符号缩进
- 文本起始位置
- 模块宽度

**水平相邻模块**应保持以下要素的一致性：
- 顶部对齐位置
- 高度（在内容允许的情况下）
- 内部填充（Padding）

## 文本格式化

### 项目符号结构

避免非结构化的文字堆砌。将内容拆分为易于扫读的项目要点。

**正确结构示例：**
```
✓  面向消费者的移动端及网页版语言学习应用
   (Duolingo, Babbel, Memrise, Busuu)
✓  B2B 企业级语言培训平台
   (goFLUENT, Speexx, Learnship)
✓  在线家教平台
   (italki, Preply, Cambly)
```

**错误结构示例（文字堆砌）：**
```
消费级移动/网页应用 (Duolingo, Babbel, Memrise, Busuu)
B2B 企业平台 (Speexx, Rosetta Stone Enterprise)
在线家教平台 (Preply, italki, Cambly)
```

### 项目符号使用指南

| 上下文 | 符号 | 用法 |
|---------|--------|-------|
| 包含/积极 | ✓ (对勾) | 范围内项目、现有功能 |
| 排除/消极 | × (叉号) | 范围外项目、缺失功能 |
| 中性列表 | • (圆点) | 普通列举、评论 |
| 数字序列 | 1. 2. 3. | 流程步骤、排名 |
| 次级符号 | ‣ 或 – | 主项下的次级要点 |

调整符号使用以匹配模板的现有惯例。

### 项目符号的一致性

模块/章节内的所有项目符号必须具有相同的格式：
- 整个模块使用相同的符号（除非有刻意的差异化设计）
- 所有一级项目符号的缩进级别相同
- 符号大小相同
- 符号与文本之间的间距相同
- 同一级别的所有要点文本字号相同

### 字号指南

以下为典型范围——请根据模板规范进行调整：

| 元素 | 典型字号 (pt) | 样式 |
|---------|-------------------|-------|
| 幻灯片标题 | 40-48 | 加粗 |
| 副标题/定义 | 18-22 | 加粗 |
| 章节标题 | 14-16 | 常规 |
| 正文/要点 | 12-14 | 常规 |
| 表头 | 10-12 | 加粗 |
| 表体 | 9-11 | 常规 |
| 脚注 | 8-9 | 斜体 |

### 文本密度指南

- 每个内容模块**最多 6-7 个要点**（根据空间调整）
- 每个项目要点**最多 2 行**
- **括号内的示例**应留在同级或在下方缩进显示
- **避免孤行字**——调整换行符以避免新行中出现单个文字

## 表格制作

### 关键：使用真实的表格对象

**表格必须是真实的表格对象，严禁使用带空格/制表符的纯文本。**

使用制表符拼凑的文本永远无法完美对齐，且显得极不专业。务必创建正式的表格对象。

### 表格结构指南

1. **列对齐**：
   - 文本列：左对齐（标题和内容均同）
   - 数值列：居中或右对齐
   - 标题应与其列内容保持对齐方式一致

2. **标题行**：
   - 文本加粗
   - 背景底纹（使用模板的品牌色）
   - 使用对比色以提高可读性
   - 对齐方式与列内容一致

3. **隔行换色**（可选）：
   - 交替行的浅色底纹可提高可读性

4. **汇总/总计行**：
   - 文本加粗
   - 使用较粗的顶边框（分隔线）
   - 使用独特的背景底纹

5. **表格宽度**：
   - 填满指定的区域宽度
   - 避免表格“悬浮”在大量白空间中

### 有关 XML 实现模式，请参阅 [`xml-reference.md`](xml-reference.md#table-implementation)

## 图表与图片处理

### 从 Excel 粘贴图表

从 Excel 粘贴图表时：

1. **仅粘贴图表**——不要包含源数据表
2. **调整大小以填满指定区域**——图表不应看起来像微缩图标
3. **保持纵横比**——不要使图表变形
4. **验证可读性**——坐标轴标签、图例、数据标签必须清晰可见

### 从 Excel 粘贴表格

从 Excel 粘贴表格时：

1. **仅粘贴格式化后的表格**——排除任何源数据或计算过程
2. **调整大小以填满指定区域**——表格应占据其完整的章节空间
3. **核查列宽**——调整以确保文本不被截断
4. **检查格式保留情况**——颜色、边框、字体可能需要重新调整

### 尺寸指南

**最小尺寸原则：**
- 图表：应占据其指定区域的大部分空间
- 表格：完全填满指定的章节宽度
- 图片：根据上下文调整至合适大小，严禁使用“缩略图”级别的大小

**视觉表现过小的迹象（应避免）：**
- 图表仅占据可用空间的一小部分
- 文字标签无法辨认
- 视觉元素周围有大面积空白
- 元素显示效果如同“缩略图”

### 正确的调整流程

1. 确认目标区域的尺寸
2. 粘贴图表/表格
3. 立即调整大小以填满目标区域
4. 验证所有文本仍然清晰可读
5. 必要时调整内部元素（图例位置、坐标轴标签）

## 数据可视化

### 关键指标显示

在展示关键指标（如 TAM、CAGR、预测数据）时，考虑展示数值之间的逻辑关系，而非静态列举：

- **视觉流向指示**：使用形状（箭头、人字形、连接线）展示进展
- **层级大小**：核心指标字号较大，标签字号较小
- **空间排列**：通过元素位置展示逻辑流向

### 箭头与流向指示

如果使用箭头或流向指示：
- 使用 PowerPoint 的形状对象，而非文本字符
- 最终演示文稿中不得使用基于文本的箭头（→, ⟹）
- 通过 PPT 形状工具或 XML 形状元素创建箭头

**有关 XML 实现，请参阅 [`xml-reference.md`](xml-reference.md#arrow-shapes)**

## 字体一致性

### 跨模块字体一致性

处于同一层级的所有文本框必须使用相同的字号。

**应保持一致的同级模块：**

| 模块类型 | 应与之匹配的模块 |
|----------|-------------------|
| “包含细分”内容 | “排除细分”内容 |
| “定义”内容 | “范畴依据”内容 |
| 左列要点内容 | 右列要点内容 |
| 所有标签框 | 彼此之间 |
| 所有章节标题 | 彼此之间 |

### 验证流程

1. 识别并列出处于同一层级的所有文本框
2. 检查每个文本框的字号
3. 若有差异，统一调整至一致
4. 若空间允许，默认采用较大的字号；否则一致采用较小字号

**例外情况**：次级要点或说明文字可以比主项要点字号小，但这种差异必须在**所有**模块中保持一致。

## 模板适配

以上标准应根据具体使用的模板进行适配：

1. **颜色**：使用模板指定的品牌色，而非自行规定特定颜色
2. **字体**：使用模板指定的字体系列
3. **间距**：匹配模板现有的间距惯例
4. **布局**：遵循模板的章节结构

无论模板如何变化，核心原则始终如一：
- 文本在背景色下必须清晰可读
- 表格必须是真实的表格对象
- 内容应恰当地填满可用空间
- 平行元素之间的格式必须保持一致
- 图表/图片必须大小合规

> **💡 Appendix: 领域知识小贴士**
>
> 制作一份高大上的 **Pitch Deck（融资推介材料）** 是初创企业向投资人化缘的关键。里面经常会提到几个让投资人眼睛发光的词：
>
> 1. **TAM (Total Addressable Market) - 潜在市场总量**：
>    这就是在吹……哦不，是在科学地告诉投资人：“如果我们的产品卖给全世界所有需要它的人，这个市场到底有多大？”它代表了你业务的想象空间上限。
>
> 2. **CAGR (Compound Annual Growth Rate) - 复合年均增长率**：
>    这是一个衡量增长的“魔法数字”。它不是简单的每年加法，而是假设每年以恒定速度像滚雪球一样复利增长。比如一个行业连续几年 CAGR 是 20%，那可是非常诱人的大蛋糕。
>
> 3. **Visual Hierarchy (视觉层级)**：
>    简单说就是“让重点跳出来”。投资人看每页 PPT 的时间可能只有几秒钟，通过字号大小、颜色对比，你要引导他们的眼睛第一眼看到最重要的数字。
>
> 4. **Orphan words (孤行字)**：
>    排版界的小尴尬。指一个段落的最后一行只有一个字或单词，这在专业文档里显得很不协调。优秀的排版会通过微调间距把这个“孤儿”拉回到上一行，让视觉上整齐划一。
