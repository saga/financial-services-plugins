## 关键验证公式

### 复合年增长率（CAGR）预测

**公式：**
```
终值 (Future Value) = 现值 (Present Value) × (1 + CAGR)^n
```

**变量说明：**
- 现值 (Present Value)：当前年份或基准年份的市场规模
- 复合年增长率 (CAGR)：复合年增长率（以小数表示，例如 16.4% = 0.164）
- n：基准年份与目标年份之间的年数

**验证示例：**
```
源数据称：2024年为221亿美元，CAGR为16.4% = 2030年为550亿美元

核实：22.1 × (1.164)^6 = 22.1 × 2.488 = 55.0 ✓
```

**计算 n（年数）：** 计算基准年与目标年之间的差值。例如：2024→2030 = 6年；2025→2030 = 5年。

### 估值倍数

**EV/Revenue（企业价值/营收倍数）：**
```
EV/Revenue 倍数 = 企业价值 (EV) ÷ 营业收入
隐含企业价值 = 营业收入 × 倍数
```

**EV/EBITDA（企业价值/EBITDA倍数）：**
```
EV/EBITDA 倍数 = 企业价值 (EV) ÷ EBITDA
隐含企业价值 = EBITDA × 倍数
```

**验证示例：**
```
源数据称：在一项营收为4500万美元的交易中，成交价为4.36亿美元，对应营收倍数为9.7倍

核实：436 ÷ 45 = 9.69 ≈ 9.7x ✓
```

### 市场份额

**公式：**
```
市场份额 = (细分市场规模 ÷ 总市场规模) × 100
```

**验证示例：**
```
源数据称：线上细分市场（180亿美元）占总市场（650亿美元）的28%

核实：18 ÷ 65 = 0.277 = 27.7% ≈ 28% ✓
```

### 增长率

**同比增长（YoY）：**
```
YoY 增长率 = (本年数值 - 上年数值) ÷ 上年数值 × 100
```

**基于起止点的 CAGR 计算：**
```
CAGR = (终值 ÷ 起始值)^(1/n) - 1
```

## 共识方法论

当源数据包含多个估算值时，请验证共识计算结果：

### 规模共识（区间法）

**方法：** 采用所有来源中的完整最小值-最大值区间。

**示例：**
```
来源数据：149亿、183亿、211亿、212亿、221亿（美元）
共识结果：150-220亿美元（取整至最接近的10亿单位）
```

### CAGR 共识（核心集群法）

**方法：** 剔除离群值（最高值和最低值），使用核心集群区间。

**示例：**
```
来源数据：10.6%, 16.4%, 17.2%, 19.0%, 22.7%
剔除离群值：10.6%（低点）, 22.7%（高点）
核心集群：16.4%, 17.2%, 19.0%
共识结果：16-19% 或 16-17%（保守估算）
```

### 预测共识

**方法：** 将共识 CAGR 应用于规模区间的中点值。

**示例：**
```
规模区间：150-220亿美元 → 中点值：185亿美元
CAGR 共识：16-17%
按 16% 计算：18.5 × (1.16)^6 = 451亿美元
按 17% 计算：18.5 × (1.17)^6 = 475亿美元
共识预测：450-480亿美元
```

## 取整准则

以下为**通用惯例**——请根据数值量级和模板风格进行调整：

| 数值类型 | 典型取整方式 | 示例 |
|------------|------------------|---------|
| 大型市场规模（100亿+） | 最接近的10亿单位 | 18.47 → 180亿美元 |
| 较小市场规模（<100亿） | 最接近的5亿或1亿单位 | 2.3 → 25亿美元 |
| 规模区间 | 与来源精度保持一致 | 14.9-22.1 → 150-221亿美元 |
| CAGR | 整数 % 或 0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 最接近的 5% 或匹配来源 | 27.7% → 25% 或 30% |
| 营收（百万美元） | 保留 1 位小数 | 18.47 → 18.5百万美元 |
| 倍数 | 保留 1 位小数 | 9.688 → 9.7x |

**取整原则：**
- 取整不应实质性地改变数字——对于较小的数值，请使用更高的精度；
- 一致性比精度更重要——同一类数字应采用相同的取整标准；
- 在创建区间时，低端值向下取整，高端值向上取整；
- 对于汇总统计数据（平均值、中位数），应与输入数据的精度匹配。

## 验证检查清单

在使用源数据中的任何计算值之前：

### 公式验证
- [ ] 预测是否使用了正确的 CAGR 公式：`PV × (1 + r)^n`
- [ ] 倍数计算是否为 EV ÷ 指标（确保没有颠倒）
- [ ] 增长率的分母是否使用了正确的基准年
- [ ] 百分比份额之和是否约等于 100%（如适用）

### 输入验证
- [ ] 基准年数据是否与原始文件一致
- [ ] CAGR/增长率是否符合所述的来源方法论
- [ ] 时间周期 (n) 计算是否正确
- [ ] 货币单位是否统一（例如 10 亿 vs 百万）

### 输出验证
- [ ] 计算结果是否与源数据标注的数字匹配
- [ ] 若不匹配，请调查是否存在方法论差异
- [ ] 取整规则是否应用一致
- [ ] 结果是否合乎逻辑（无数量级错误）

### 共识验证
- [ ] 区间计算是否涵盖了所有来源
- [ ] 离群值剔除方法是否有记录
- [ ] 中点值计算是否采用了正确的平均法
- [ ] 区间边界是否代表了实际的极值或记录的子集

## 需深入调查的预警信号

**预测偏差：**
- 计算得出的预测值与源数据偏差超过 5%；
- 可能原因：基准年不同、CAGR 不同或取整误差。

**倍数偏差：**
- 计算得出的倍数与源数据不符；
- 可能原因：指标定义不同（如 LTM 过去十二个月 vs. NTM 未来十二个月，总营收 vs. 净营收）。

**共识偏差：**
- 您得出的共识结果与源数据的共识结果不同；
- 可能原因：数据源剔除了某些数据点，或对离群值的处理方式不同。

**如有疑问：** 请在脚注中注明差异，并展示您的计算方法论。

> **💡 Appendix: 领域知识小贴士**
>
> 哪怕你是第一次接触财务分析，掌握这些核心概念也能让你瞬间“看穿”数据背后的逻辑：
>
> 1. **CAGR (复合年增长率)**：这是业务增长的“复利魔杖”。它不是简单的每年增长加起来平均，而是考虑了“利滚利”的效果。掌握它，你就能知道一项资产在多年间平稳增长的真实速度。
> 2. **EV (企业价值)**：这衡量的是买下一家公司的“真实身价”。它不仅看股票值多少钱（市值），还要加上公司欠的债并减去手里存的现金。这比单纯看市值更能反映收购一家公司需要掏多少钱。
> 3. **EBITDA**：我们可以把它理解为公司的“纯赚钱能力”。它排除了税收、利息以及像折旧这样并不直接产生现金流出的会计项目，是分析师判断公司业务经营好坏的最常用指标。
> 4. **估值倍数 (Multiples)**：这就像买房时的“单价”。一家公司值多少钱，通常要看它的身价是它收入或利润的多少倍。通过和同行业其他公司比较倍数，就能判断一家公司是跌出了“性价比”还是处于“泡沫”中。
> 5. **共识 (Consensus)**：当不同的研究机构对同一个市场给出不同的预测时，我们会取一个“中间地带”，即共识。剔除掉那些太乐观或太悲观的极端数据，能帮我们得到一个更客观、更稳健的判断。
