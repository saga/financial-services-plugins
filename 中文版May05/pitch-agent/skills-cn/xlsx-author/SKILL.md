# xlsx-author

当在**无头（headless）**模式（托管代理 / CMA 模式）下运行，且需要以**文件产物**的形式交付 Excel 工作簿，而非通过 `mcp__office__excel_*` 编辑实时工作簿时，请使用此技能。

## 输出规范

- 写入至 `./out/<name>.xlsx`。如果 `./out/` 目录不存在，请先创建。
- 在最终消息中返回相对路径，以便编排层进行收集。

## 如何构建工作簿

编写一个简短的 Python 脚本并使用 Bash 运行。请使用 `openpyxl` 库：

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active; ws.title = "Inputs"
ws["B2"] = "Revenue"; ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")           # 蓝色 = 硬编码输入值
calc = wb.create_sheet("DCF")
calc["C5"] = "=Inputs!C2*(1+Inputs!C3)"        # 黑色 = 计算公式
wb.save("./out/model.xlsx")
```

## 惯例规范（参考 `audit-xls`）

- **蓝 / 黑 / 绿。** 蓝色代表硬编码输入（hardcoded input），黑色代表计算公式（formula），绿色代表指向其他工作表或文件的链接。
- **计算单元格严禁硬编码。** 每个计算单元格都必须是公式；所有输入参数应统一存放于 Inputs（输入）选项卡中。
- **命名区域（Named ranges）：** 对于演示文稿（Deck）或备忘录中引用的任何数值，均需创建命名区域。
- **勾稽检查（Balance checks）：** 包含一个 Checks（检查）选项卡，用于核对勾稽关系（如资产负债表平衡、现金流量表与现金科目核对等），并直观显示 TRUE/FALSE 结果。
- **一档一模。** 除非有明确要求，否则不要向现有工作簿中追加内容。

## 何时**不应**使用

如果 `mcp__office__excel_*` 工具可用（协作插件模式），请优先使用这些工具 —— 它们可以驱动用户的实时工作簿并带有审核检查点。本技能仅作为无头运行模式下生成文件的备选方案。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **无头模式 (Headless)**：这可不是恐怖故事！在计算机领域，“无头”意味着程序在后台运行，没有窗口和图形界面。就像一个辛勤的隐形会计，不需要屏幕也能帮你把账算好。
> 2. **华尔街配色准则**：在专业的财务模型中，数字的颜色是“身份标签”。**蓝色**代表手动录入的原始数据，**黑色**代表自动计算的公式。这样其他人一眼就能看出哪些是假设指标，哪些是计算结果，避免修改模型时误删公式。
> 3. **勾稽关系 (Ties & Checks)**：这是财务世界的“逻辑闭环”。比如你的资产必须等于负债加股东权益。专业的模型会专门设一个“体检表”（Checks 选项卡），如果哪里的数字对不上（比如 1+1 不等于 2 了），它就会亮起红灯报警。
> 4. **DCF (现金流折现模型)**：代码示例中提到的 DCF 是金融界的“测谎仪”，通过预测公司未来能赚多少钱，并把这些钱折算成现在的价值，来判断一家公司到底值多少钱。
