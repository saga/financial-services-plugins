# 可比公司分析 (Comparable Company Analysis)

## ⚠️ 至关重要：数据源优先级（首读）

**请始终遵循以下数据源层级：**

1. **首选：检查 MCP 数据源** - 如果 S&P Kensho MCP、FactSet MCP 或 Daloopa MCP 可用，请仅使用这些来源获取财务和交易信息。
2. **禁止在上述 MCP 数据源可用时使用网页搜索。**
3. **仅在 MCP 不可用时：** 使用彭博终端（Bloomberg Terminal）、SEC EDGAR 申报文件或其他机构级来源。
4. **切勿将网页搜索作为主要数据源** - 网页信息缺乏机构级分析所需的准确性、审计轨迹和可靠性。

**重要性说明：** MCP 来源提供经过验证、具备规范引用的机构级数据。网页搜索结果对于财务分析而言可能过时、不准确或不可靠。

---

## 概述
本项技能旨在教导 Claude 构建机构级的可比公司分析，将运营指标、估值倍数和统计基准分析有机结合。输出结果是一个结构化的 Excel/电子表格，通过同行比较协助做出知情的投资决策。

**参考资料与情境化：**

`examples/comps_example.xlsx` 提供了可比公司分析的示例。在利用此目录下的示例文件时，请灵活处理：

**建议将示例用于：**
- 理解结构层级（各板块的衔接流程）
- 把握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰的标题、透明的公式、审计轨迹）

**严禁将示例用于：**
- 机械复制格式或指标
- 不顾语境地照搬布局
- 无论受众是谁都套用相同的视觉风格

**在行动前请始终自问：**
1. **“您是否有偏好的格式，还是需要我调整模版风格？”**
2. **“受众是谁？”**（投资委员会、董事会演示、快速查阅还是详细备忘录）
3. **“核心问题是什么？”**（估值、增长分析、竞争地位还是效率）
4. **“背景是什么？”**（并购评估、投资决策、行业基准分析还是业绩评估）

**根据具体情况进行调整：**
- **行业背景**：大型科技股需要的指标与新兴 SaaS 初创公司完全不同
- **行业特定需求**：及早加入相关指标（例如，科技行业的云 ARR、企业客户数、开发者生态）
- **公司知名度**：知名公司可能不需要太多背景介绍，而应更多关注增量分析
- **决策类型**：并购相对于日常投资组合监测，其侧重点有所不同

**核心原则：** 沿用模版原则（结构清晰、统计严谨、公式透明），但根据具体环境灵活执行。目标是交付机构级质量的分析，而非看起来像机构的僵化模版。

用户提供的示例和明确偏好始终优先于默认设置。

## 核心哲学
**“先构建正确的结构，再让数据说话。”**

从能够驱动战略思考的核心标题开始，输入干净的数据，构建透明的公式，让统计结果自然浮现。一份优秀的 Comps 应当让未参与构建的人也能一目了然。

## ⚠️ 至关重要：公式胜于硬编码 + 逐步验证

**环境——Office JS 与 Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS (`Excel.run(async (context) => {...})`)。通过 `range.formulas = [["=E7/C7"]]` 编写公式，而非 `range.values`。无需单独的重算步骤——Excel 会原生处理。使用 `range.format.*` 设置颜色/字体。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。编写 `cell.value = "=E7/C7"`（公式字符串）。
- 无论哪种方式，原则一致——只需转换 API 调用即可。
- **Office JS 合并单元格陷阱：** 请勿先调用 `.merge()` 再对合并后的区域设置 `.values`（这会抛出 `InvalidArgument` 错误，因为区域仍报告合并前的维度）。应当先在左上角单元格写入值，再进行合并和格式化：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式而非硬编码（Hardcodes）：**
- 每个派生值（利润率、倍数、统计数据）必须是引用输入单元格的 Excel 公式——严禁粘贴预先计算好的数字。
- 使用 Python/openpyxl 构建表格时：编写 `cell.value = "=E7/C7"`（公式字符串），而非 `cell.value = 0.687`（计算结果）。
- 唯一的硬编码值应为原始输入数据（收入、EBITDA、股价等），并且每一项都应附带注明来源的单元格批注。
- **原因**：当输入发生变化时，模型必须能自动更新。硬编码的利润率是潜伏的错误。

**与用户进行逐步验证：**
- 设置好结构后 → 在填充数据前向用户展示标题布局。
- 输入原始数据后 → 在构建公式前展示输入块并确认来源/期间。
- 构建运营指标公式后 → 在进入估值板块前展示计算出的利润率并与用户进行合理性评价。
- 构建估值倍数后 → 在添加统计数据前展示倍数并确认其合理性。
- **不要**一次性构建完整个表格再展示——通过确认每个板块及早发现错误。

## 第 1 节：文档结构与设置

### 标题块（第 1-3 行）
```
第 1 行：[分析标题] - 可比公司分析
第 2 行：[带代码的公司列表] • [公司 1 (TICK1)] • [公司 2 (TICK2)] • [公司 3 (TICK3)]
第 3 行：截至 [日期/期间] | 除每股金额和比率外，所有数字均以 [百万/十亿美元 USD] 为单位
```

**重要性说明：** 立即建立上下文。任何打开此文件的人都能立刻明白分析的对象、创建时间以及如何解析数字。

### 视觉规范标准（可选 - 用户偏好和上传的模版始终优先）

**重要提示：以下仅为建议的默认设置。请始终优先考虑：**
1. 用户明确的格式化偏好
2. 上传的模版文件中的格式
3. 公司/团队的风格指南
4. 仅在无其他引导时使用以下默认设置

**建议字体与排版：**
- **字体族**：Times New Roman（专业、易读、行业标准）
- **字号**：数据单元格 11pt，标题 12pt
- **加粗**：板块标题、公司名称、统计标签

**默认颜色与阴影 — 专业蓝/灰配色（越简约越好）：**
- **保持克制** — 仅限蓝色和灰色。请勿引入绿色、橙色、红色或多种强调色。一份整洁的 Comps 表格总共只应使用 3-4 种颜色。
- **板块标题**（例如 “OPERATING STATISTICS & FINANCIAL METRICS”）：
  - 深蓝色背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色加粗文字
  - 跨所有列的全行阴影
- **列标题**（例如 “Company”, “Revenue”, “Margin”）：
  - 浅蓝色背景（`#D9E1F2` 或类似的淡蓝色）
  - 黑色加粗文字
  - 居中对齐
- **数据行**：
  - 公司数据使用白色背景
  - 公式使用黑色文本；硬编码输入使用蓝色文本
- **统计行**（最大值、75 分位数等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文本，标签左对齐
- **这就是全部配色**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户模版另有要求，否则不添加其他颜色。

**建议的格式化规范：**
- **小数精度**：
  - 百分比：1 位小数 (12.3%)
  - 倍数：1 位小数 (13.5x)
  - 金额：无小数，千分位分隔符 (69,632)
  - 以百分比显示的利润率：1 位小数 (68.7%)
- **边框**：无边框（保持整洁、简约的外观）
- **对齐**：所有指标居中对齐，确保整齐统一
- **单元格尺寸**：所有列宽应统一/对等，所有行高应一致（营造专业的网格感）

**注意：** 如果用户提供了模版文件或指定了不同的格式，请改用用户的设置。

## 第 2 节：运营统计与财务指标

### 核心列（以此开始）
1. **Company (公司)** - 名称格式保持一致
2. **Revenue (收入)** - 规模指标（根据情况可以是 LTM、季度或年度）
3. **Revenue Growth (收入增长)** - 同比百分比变化
4. **Gross Profit (毛利)** - 收入减去销售成本
5. **Gross Margin (毛利率)** - 毛利/收入（基础盈利能力指标）
6. **EBITDA** - 税息折旧及摊销前利润
7. **EBITDA Margin (EBITDA 利润率)** - EBITDA/收入（运营效率指标）

### 可选补充（根据行业/目的选择）
- **季度 vs LTM** - 如果存在季节性，则同时包含两者
- **Free Cash Flow (自由现金流)** - 适用于资本密集型或 SaaS 企业
- **FCF Margin (FCF 利润率)** - FCF/收入（现金生成效率）
- **Net Income (净利润)** - 适用于成熟、盈利的公司
- **Operating Income (营业利润)** - 适用于折旧摊销（D&A）差异较大的企业
- **CapEx Metrics (资本支出指标)** - 适用于重资产行业
- **Rule of 40 (40 原则)** - 专门针对 SaaS（增长率% + 利润率%）
- **FCF Conversion (FCF 转化率)** - 用于收益质量分析（高级）

### 公式示例（以第 7 行为例）
```excel
// 核心比率 - 始终通过计算得出
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率 - 如相关则包含
FCF 利润率: =[FCF]/[Revenue]
净利率: =[Net Income]/[Revenue]
40 原则: =[Growth %]+[FCF Margin %]
```

**黄金法则：** 每个比率都应该是 [某指标] / [收入] 或 [某指标] / [表格中的其他指标]。保持简洁。

### 统计块（位于公司数据下方）

**至关重要：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留出一个空行以示视觉区分]
- 最大值 (Maximum): =MAX(B7:B9)
- 75 分位数 (75th Percentile): =QUARTILE(B7:B9,3)
- 中位数 (Median): =MEDIAN(B7:B9)
- 25 分位数 (25th Percentile): =QUARTILE(B7:B9,1)
- 最小值 (Minimum): =MIN(B7:B9)
```

**需要统计的列 (可比指标)：**
- 收入增长 %、毛利率 %、EBITDA 利润率 %、EPS
- EV/Revenue, EV/EBITDA, P/E, 股息率 %, Beta

**不需要统计的列 (规模指标)：**
- 收入、EBITDA、净利润（绝对规模随公司体量变化）
- 市值 (Market Cap)、企业价值 (EV)（不同规模公司之间不具可比性）

**注意**：在公司数据和统计行之间添加一个空行。不要添加“行业统计”或“估值统计”标题行。

**四分位数的重要性：** 它们展示了分布情况，而非仅仅是平均值。75 分位倍数能告诉你“溢价”公司的交易水平。

## 第 3 节：估值倍数与投资指标

### 核心估值列（以此开始）
1. **Company (公司)** - 与运营板块顺序相同
2. **Market Cap (市值)** - 当前市场估值
3. **Enterprise Value (企业价值)** - 市值 ± 净负债/现金
4. **EV/Revenue** - 市场为每 1 美元销售额支付的价格
5. **EV/EBITDA** - 市场为每 1 美元收益支付的价格
6. **P/E Ratio (市盈率)** - 价格相对于净收益的比率

### 可选估值指标（根据背景选择）
- **FCF Yield (FCF 收益率)** - FCF/市值（用于关注现金的分析）
- **PEG Ratio** - 市盈率/增长率（用于成长型公司）
- **Price/Book (市净率)** - 市值 vs 账面价值（用于重资产企业）
- **ROE/ROA (净资产收益率/总资产收益率)** - 回报指标（用于盈利能力比较）
- **Revenue/EBITDA CAGR** - 历史复合增长率（用于趋势分析）
- **Asset Turnover (资产周转率)** - 收入/资产（用于运营效率分析）
- **Debt/Equity (产权比率/杠杆率)** - 用于资本结构分析

**核心原则：** 包含 3-5 个对该行业至关重要的核心倍数。不要仅仅因为可以计算就包含所有可能的指标。

### 公式示例
```excel
// 核心倍数 - 始终包含这些
EV/Revenue: =[Enterprise Value]/[LTM Revenue]
EV/EBITDA: =[Enterprise Value]/[LTM EBITDA]
市盈率 (P/E): =[Market Cap]/[Net Income]

// 可选倍数 - 如有数据则包含
FCF 收益率: =[LTM FCF]/[Market Cap]
PEG 比率: =[P/E]/[Growth Rate %]
```

### 交叉引用规则
**至关重要：** 估值倍数必须引用运营指标板块。严禁两次输入相同的原始数据。如果收入在 C7 单元格，那么 EV/Revenue 的公式必须引用 C7。

### 统计块
结构与运营板块一致：为每个指标提供最大值、75 分位、中位数、25 分位和最小值。在公司数据和统计数据之间留出一个空行。不要添加“估值统计”标题行。

## 第 4 节：注释与方法论文档

### 必修组件

**数据源与质量：**
- 数据来源？（S&P Kensho MCP, FactSet MCP, Daloopa MCP, Bloomberg, SEC 申报文件）
- 涵盖期间？（2024 年第四季度，经审计数字）
- 如何验证？（与 10-K/10-Q 交叉核对）
- 注意：如果可用，优先选择 MCP 数据源以确保更好的准确性和可追溯性。

**关键定义：**
- EBITDA 计算方法（毛利 + 折旧摊销，或营业利润 + 折旧摊销）
- 自由现金流公式（经营现金流 - 资本支出）
- 特殊指标解释（40 原则、FCF 转化率）
- 时间周期定义（LTM、CAGR 计算期间）

**估值方法论：**
- 企业价值 (EV) 是如何计算的？（市值 + 净负债）
- 使用了什么增长率？（历史复合增长率、前瞻预测值）
- 是否进行了任何调整？（剔除一次性项目、归一化利润率）

**分析框架：**
- 投资逻辑是什么？（云/SaaS 效率）
- 哪些指标最重要？（现金生成能力、资本效率）
- 读者应如何解读统计数据？（四分位数提供背景参考）

## 第 5 节：选择正确的指标（决策框架）

### 从“我要回答什么问题？”开始

**“哪家公司被低估了？”**
→ 关注：EV/Revenue, EV/EBITDA, P/E, 市值
→ 略过：运营细节、增长指标

**“哪家公司效率最高？”**
→ 关注：毛利率、EBITDA 利润率、FCF 利润率、资产周转率
→ 略过：规模指标、绝对金额

**“哪家公司增长最快？”**
→ 关注：收入增长 %、EBITDA 复合增长率、用户/客户增长
→ 略过：利润率指标、杠杆率

**“哪家公司现金生成能力最强？”**
→ 关注：FCF、FCF 利润率、FCF 转化率、资本支出强度
→ 略过：EBITDA、市盈率

### 行业特定指标选择

**软件/SaaS：**
必选：收入增长率、毛利率、40 原则
可选：ARR、净金额留存率 (NDR)、CAC 回收期
略过：资产周转率、库存指标

**制造/工业：**
必选：EBITDA 利润率、资产周转率、资本支出/收入
可选：ROA、库存周转率、订单积压 (Backlog)
略过：40 原则、SaaS 指标

**金融服务：**
必选：ROE、ROA、效率比率、P/E
可选：净息差 (NIM)、贷款损失拨备
略过：毛利率、EBITDA（对银行无意义）

**零售/电子商务：**
必选：收入增长率、毛利率、库存周转率
可选：同店销售额、获客成本 (CAC)
略过：重研发或资本支出指标

### “5-10 原则”

**5 个运营指标** - 收入、增长、2-3 个利润率/效率指标
**5 个估值指标** - 市值、企业价值、3 个倍数
**= 10 列总计** - 足以讲述故事，又不至于因信息冗余而跑题

如果指标超过 15 个，你可能引入了噪音。请无情地精简。

## 第 6 节：最佳实践与质量检查

### 开始之前
1. **定义同行组 (Peer Group)** - 公司必须具备真正的可比性（类似的商业模式、规模、地理分布）。
2. **选择正确的期间** - LTM 可以平滑季节性；季度数据可以展示趋势。
3. **预先统一单位** - 选择百万还是十亿会影响所有后续工作。
4. **映射数据源** - 明确每个数字的出处。

### 构建过程中
1. **先输入所有原始数据** - 在编写公式前完成蓝色文本。
2. **为所有硬编码输入添加单元格批注** - 右键单元格 → 插入批注 → 记录来源或假设。

   **对于引用的数据，准确标注出处：**
   - 示例：“Bloomberg Terminal - MSFT Equity DES, 访问于 2024-10-02”
   - 示例：“2024 年第四季度 10-K 申报，第 42 页，项目‘总收入’”
   - 示例：“FactSet 截至 2024-10-02 的一致预期”
   - **尽可能包含超链接**：右键单元格 → 链接 → 粘贴 SEC 申报文件、数据源或报告的 URL。

   **对于假设，解释原因：**
   - 示例：“基于同行中位数假设 EBITDA 利润率为 15%，公司未披露此项”
   - 示例：“估计企业价值 = 市值 + 5000 万美元净负债（来自第三季度资产负债表，第四季度尚未发布）”
   - 示例：“前瞻 P/E 基于市场一致预期 EPS 3.45 美元（12 位分析师预测平均值）”

   **重要性说明**：支持审计追踪、数据验证、假设透明化以及未来的更新。
3. **逐行构建公式** - 测试每一次计算后再继续下一步。
4. **对标题使用绝对引用** - `$C$6` 用于锁定标题行。
5. **格式保持一致** - 百分比显示为百分数，而非小数。
6. **添加条件格式** - 自动高亮离群值。

### 合理性检查 (Sanity Checks)
- **利润率测试**：毛利率 > EBITDA 利润率 > 净利率（定义上始终成立）。
- **倍数合理性**：
  - EV/Revenue：通常为 0.5-20x（随行业差异巨大）。
  - EV/EBITDA：通常为 8-25x（各行业间相对一致）。
  - P/E：通常为 10-50x（取决于增长率）。
- **增长与倍数相关性**：较高的增长通常意味着较高的倍数。
- **规模与效率权衡**：规模较大的公司通常拥有更好的利润率（规模效益）。

### 应避免的常见错误
❌ 在公式中混淆市值和企业价值
❌ 分子和分母使用不同的时间周期（例如 LTM vs 季度）
❌ 在公式中硬编码数字而非引用单元格
❌ **硬编码输入未附带引用来源或解释假设的批注**
❌ 如果有可用链接，未包含指向 SEC 申报或数据源的超链接
❌ 包含过多没有明确用途的指标
❌ 包含不可比公司（商业模式不同）
❌ 使用过时数据且未披露
❌ 百分比的平均值计算错误（应当使用中位数）

## 第 7 节：工作流与实用技巧

### 分步流程
1. **设置结构** (30 分钟)
   - 创建所有标题
   - 格式化单元格（蓝色用于输入，黑色用于公式）
   - 确定单位和日期参考

2. **收集数据** (60-90 分钟)
   - 从主要来源提取（优先 S&P Kensho MCP 等；其次彭博、SEC）
   - 以蓝色输入所有原始数字
   - 在注释板块记录来源

3. **构建公式** (30 分钟)
   - 从简单比率（利润率）开始
   - 进展到倍数 (EV/Revenue)
   - 进行交叉检查（利润率是否合理？）

4. **添加统计数据** (15 分钟)
   - 复制所有列的公式结构
   - 验证引用范围是否正确 (B7:B9, 而非 B7:B10)
   - 检查四分位数逻辑

5. **质量控制** (30 分钟)
   - 运行合理性检查
   - 验证公式引用
   - 检查 #DIV/0! 或 #REF! 错误
   - 与已知基准进行比较

6. **撰写文档** (15 分钟)
   - 完成注释板块
   - 添加数据源
   - 定义方法论
   - 为分析打上时间戳

- **保存模版**：一次构建，永久复用。
- **高亮离群值**：为偏离 2 个标准差的值设置条件格式。
- **链接到源文件**：超链接到彭博截图或 SEC 申报。
- **版本控制**：以“Comps_v1_2024-12-15”等清晰日期命名保存。

## 第 10 节：红旗标志与警示信号

### 数据质量问题
🚩 时间周期不一致（混用季度和年度数据）
🚩 数据缺失且未加解释
🚩 数据源之间存在显著差异（偏差 >10%）

### 估值红旗
🚩 EBITDA 为负的公司使用 EBITDA 倍数进行估值（应当改用收入倍数）
🚩 在没有爆发式增长故事的情况下市盈率超过 100x
🚩 利润率不符合行业常识

### 可比性问题
🚩 财务年度截止日期不同（由于时间点差异导致问题）
🚩 混用单一业务公司和综合性企业
🚩 将由于商业模式存在实质差异的公司标注为“可比”

**存疑时，剔除该公司。** 拥有 3 个完美的对比标的优于拥有 6 个存疑的标的。

## 第 11 节：公式参考指南

### 基础 Excel 公式
```excel
// 统计函数
=AVERAGE(range)          // 简单平均值
=MEDIAN(range)           // 中位数
=QUARTILE(range, 1)      // 25 分位数
=QUARTILE(range, 3)      // 75 分位数
=MAX(range)              // 最大值
=MIN(range)              // 最小值
=STDEV.P(range)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司的平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除以零错误

// 跨表引用
='Sheet1'!B7             // 引用另一个工作表
=VLOOKUP(A7, Table1, 2)  // 从数据表查找
=INDEX(MATCH())          // 高级查找

// 格式化
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千分位分隔符
```

### 常见比率公式
```excel
毛利率 = 毛利 / 收入
EBITDA 利润率 = EBITDA / 收入
FCF 利润率 = 自由现金流 / 收入
FCF 转化率 = FCF / 经营现金流
ROE = 净利润 / 股东权益
ROA = 净利润 / 总资产
资产周转率 = 收入 / 总资产
产权比率 (D/E) = 总负债 / 股东权益
```

## 核心原则总结

1. **结构驱动洞察** - 正确的标题强制正确的思考。
2. **少即是多** - 5-10 个核心指标优于 20 个无关指标。
3. **针对问题选择指标** - 估值分析不等同于效率分析。
4. **统计展示模式** - 中位数/四分位数比平均值更能揭示真相。
5. **透明胜过复杂** - 使用每个人都能理解的简单公式。
6. **可比性是王道** - 宁可减少标的，也不要凑数。
7. **记录你的选择** - 在注释板块解释选择了哪些指标及其原因。

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到金融分析的世界！如果你对这些专业术语感到头大，别担心，我们用几个生活中的例子来拆解它们：
>
> 1. **什么是“可比公司分析 (Comps)”？**
>    想象你要卖掉家里的二手房。你会怎么定价？通常你会看小区里最近卖掉的、户型相似、面积差不多的房子成交价是多少。这就是 Comps——通过观察“邻居”（同行公司）在市场上的身价，来推算自己或目标公司的价值。
>
> 2. **为什么不直接看利润，要看 EBITDA？**
>    两家餐馆，一家是租的门面，一家是买的；一家借了钱开店，一家是用存款开的。直接看净利润，你会发现交租借钱的那家“赚得少”。但其实它们每天卖出的红烧肉一样多！EBITDA 就是为了抹去税收、借钱利息和折旧这些“杂音”，看大家做生意的纯本事。
>
> 3. **什么是“估值倍数 (Multiples)”？**
>    这好比去买水果。苹果 10 块钱一斤，梨 5 块钱一斤。“每斤多少钱”就是一个倍数。在金融里，我们看“每 1 美元收入值多少市值”（EV/Revenue）或者“每 1 美元盈利值多少钱”（P/E）。如果大家的倍数都是 10 倍，只有一家是 5 倍，那它可能就是个“便宜货”或者“被低估的珍宝”。
>
> 4. **为什么要看“四分位数 (Quartiles)”？**
>    如果班里只有两个同学，一个考 100 分，一个考 0 分，平均分是 50。但这代表班级水平吗？显然不。四分位数就像把全班同学按成绩排排坐：75 分位就是前 25% 的尖子生水平，中位数是正中间那个人的水平。看这些位置，你能精准地知道一家公司在行业班级里到底是“学霸”还是“差生”。
