# DCF 模型构建工具

## 概览

本技能遵循投资银行标准，创建机构级素质的权益估值 DCF 模型。每项分析都会生成一份详细的 Excel 模型（DCF 工作表的底部包含敏感性分析表）。

## 工具

- 默认使用用户提供的所有信息以及可用于数据溯源的 MCP 服务器。

## 核心约束——请先阅读

以下约束适用于所有 DCF 模型构建过程。在开始前请仔细审阅：

**环境：Office JS 与 Python/openpyxl：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS 环境）：** 直接使用 Office JS，不要使用 Python/openpyxl。通过 `range.formulas = [["=D19*(1+$B$8)"]]` 写入公式。无需单独的重算步骤，Excel 会原生计算。使用 `range.format.*` 进行样式设置。同样适用“公式优先于硬编码”原则：设置 `.formulas`，严禁对派生单元格设置 `.values`。
- **如果生成独立的 .xlsx 文件（无实时 Excel 会话）：** 按下文所述使用 Python/openpyxl，并在交付前运行 `recalc.py`。
- 本技能的后续部分使用 openpyxl 示例——在 Office JS 环境中请转换为对应的 API 调用，但所有原则（公式字符串、单元格批注、各板块核查点、敏感性分析循环）完全一致。

**⚠️ Office JS 合并单元格陷阱：** 在构建带有合并单元格的板块标题时，切勿先调用 `.merge()` 再对合并范围设置 `.values` —— Office JS 仍会按范围原始尺寸报告，并抛出 `InvalidArgument：输入数组的行数或列数与该范围的大小或维度不匹配`。正确做法是仅向左上角单元格写入值，然后再合并并格式化整个范围：

```js
// 错误做法 —— 会抛出 InvalidArgument:
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.values = [["MARKET DATA & KEY INPUTS"]];  // 1×1 数组对比 1×8 范围 → 失败

// 正确做法 —— 先对单个单元格赋值，再合并并格式化范围：
ws.getRange("A7").values = [["MARKET DATA & KEY INPUTS"]];
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.format.fill.color = "#1F4E79";
hdr.format.font.bold = true;
hdr.format.font.color = "#FFFFFF";
```

这适用于 DCF 中的每一个合并板块标题（市场数据、情景板块、现金流预测、终端价值、估值摘要、敏感性分析表）。

**公式优于硬编码（不可协商）：**
- 每一个预测值、利润率、折现因子、现值（PV）以及敏感性单元格必须是活的 Excel 公式——严禁在 Python 中计算后以数字形式写入。
- 使用 openpyxl 时：`ws["D20"] = "=D19*(1+$B$8)"` 是正确的；`ws["D20"] = calculated_revenue` 是错误的。
- 允许硬编码的数字仅限：(1) 原始历史输入值，(2) 假设驱动因素（增长率、WACC 输入值、终端增长率 g），(3) 当前市场数据（股价、债务余额）。
- 如果你发现自己在 Python 中进行计算并写入结果——请立即停止。模型必须在用户更改假设时自动联动。

**与用户分步确认（严禁直接端到端构建）：**
- 数据提取后 → 向用户展示原始输入块（营收、利润率、股数、净债务）并在预测前确认。
- 营收预测后 → 展示预测的顶线数据和增长率，在构建利润率板块前确认。
- FCF 构建后 → 展示完整的现金流排期表，在计算 WACC 前确认逻辑。
- WACC 计算后 → 展示计算过程和输入参数，在折现前确认。
- 终端价值 + PV 计算后 → 展示权益桥梁（企业价值 EV → 权益价值 → 每股价值），在生成敏感性分析表前确认。
- 在每个阶段拦截错误——如果在构建敏感性分析表后才发现利润率假设有误，意味着所有下游工作都需要重做。

**敏感性分析表：**
- **使用奇数的行数和列数**（标准为 5×5，有时为 7×7）——这能确保存在一个真正的中心单元格。
- **中心单元格 = 基准情景。** 构建轴值时，使中间行标题和中间列标题精确等于模型的实际假设（例如，如果基准 WACC = 9.0%，则中间行为 9.0%；如果终端增长率 g = 3.0%，则中间列为 3.0%）。中心单元格的输出结果必须等于模型实际隐含的每股股价——这是检查表格构建是否正确的“完备性检查”。
- **突出显示中心单元格**，使用中蓝色填充（`#BDD7EE`）+ 加粗字体，以便一眼识别基准情景。
- 所有单元格（通常为 3 个表 × 25 个单元格 = 75 个）必须填充完整的 DCF 重新计算公式。
- 使用 openpyxl 循环（或 Office JS 循环）以编程方式写入公式。
- 禁止使用占位符文本，禁止使用线性近似值，禁止需要手动操作的步骤。
- 每个单元格必须根据该假设组合重新计算完整的 DCF。

**单元格批注：**
- 在创建每个硬编码数值的同时添加单元格批注。
- 格式：“来源：[系统/文档], [日期], [参考内容], [URL（如有）]”。
- 在进入下一个板块前，所有蓝色输入值必须带有批注。
- 不要推迟到最后或写“TODO：添加来源”。

**模型布局规划：**
- 在写入任何公式之前，先确定所有板块的行位置。
- 先写入所有标题和标签。
- 其次写入所有板块分隔符和空行。
- 然后使用锁定的行位置写入公式。
- 公式创建后立即进行测试。

**公式重算：**
- 交付前运行 `python recalc.py model.xlsx 30`。
- 修复所有错误，直到退出状态为“success”。
- 必须确保零公式错误（#REF!, #DIV/0!, #VALUE! 等）。

**情景板块：**
- 为悲观（Bear）、基准（Base）、乐观（Bull）情景创建独立板块。
- 在每个板块内的预测年度水平方向显示假设。
- 使用 IF 公式：`=IF($B$6=1,[Bear单元格],IF($B$6=2,[Base单元格],[Bull单元格]))`。
- 验证公式是否引用了正确的情景板块单元格。

## DCF 流程工作流

### 第 1 步：数据提取与验证

从 MCP 服务器、用户提供的数据以及网络上获取数据。

**数据源优先级：**
1. **MCP 服务器**（如已配置）- 来自 Daloopa 等提供商的结构化财务数据。
2. **用户提供的数据** - 来自其研究的历史财务数据。
3. **网络搜索/提取** - 当前价格、Beta 值、债务和现金（必要时）。

**验证清单：**
- 核实净债务与净现金（对估值至关重要）。
- 确认摊薄后发行在外股数（检查近期回购/发行情况）。
- 验证历史利润率是否与业务模式一致。
- 将营收增长率与行业基准进行交叉对比。
- 验证税率是否合理（通常为 21-28%）。

### 第 2 步：历史数据分析（3-5 年）

分析并记录：
- **营收增长趋势**：计算 CAGR（复合年增长率），识别增长驱动因素。
- **利润率演变**：跟踪毛利率、EBIT 利润率、FCF 利润率。
- **资本密集度**：折旧摊销（D&A）和资本支出（CapEx）占营收的百分比。
- **营运资本效率**：净营运资本（NWC）变动占营收增长的百分比。
- **回报指标**：ROIC（投入资本回报率）、ROE（净资产收益率）趋势。

创建汇总表显示：
```
历史指标 (LTM):
营收: $X million
营收增长: X% CAGR
毛利率: X%
EBIT 利润率: X%
D&A 占营收比: X%
CapEx 占营收比: X%
FCF 利润率: X%
```

### 第 3 步：构建营收预测

**方法论：**
1. 从最新的实际营收开始（LTM 或最近财年）。
2. 将增长率应用于每个预测年度。
3. 同时显示金额数值和计算出的增长率 %。

**增长率框架：**
- 第 1-2 年：反映近期可见度的较高增长。
- 第 3-4 年：逐渐放缓至行业平均水平。
- 第 5 年及以后：趋近于终端增长率。

**公式结构：**
- 营收(第 N 年) = 营收(第 N-1 年) × (1 + 增长率)
- 增长率 %(第 N 年) = 营收(第 N 年) / 营收(第 N-1 年) - 1

**三支情景方法：**
```
悲观情景 (Bear Case): 保守增长 (如 8-12%)
基准情景 (Base Case): 最可能的情景 (如 12-16%)
乐观情景 (Bull Case): 乐观增长 (如 16-20%)
```

### 第 4 步：经营费用建模

**固定/变动成本分析：**

经营费用应模拟真实的经营杠杆：
- **销售与市场营销 (S&M)**：通常占营收的 15-40%，具体取决于业务模式。
- **研发 (R&D)**：科技公司通常占 10-30%。
- **一般与行政 (G&A)**：通常占营收的 8-15%，随公司规模扩大显示出杠杆效应。

**核心原则：**
- 所有百分比均基于“营收”，而非毛利。
- 模拟经营杠杆：百分比应随营收规模扩大而下降。
- 保持 S&M、R&D、G&A 的独立行项目。
- 计算 EBIT = 毛利 - 总经营费用。

**利润率扩张框架：**
```
当前状态 → 目标状态 (第 5 年)
毛利率: X% → Y% (基于规模、效率给出理由)
EBIT 利润率: X% → Y% (营收增长 + 费用杠杆的结果)
```

### 第 5 步：自由现金流计算

**按照正确顺序构建 FCF：**

```
EBIT
(-) 税收 (EBIT × 税率)
= NOPAT (税后净营业利润)
(+) 折旧与摊销 (非现金费用，占营收比)
(-) 资本支出 (占营收比，通常为 4-8%)
(-) 净营运资本变动 (Δ NWC)
= 无杠杆自由现金流 (Unlevered Free Cash Flow)
```

**营运资本建模：**
- 按照营收变动（Δ Revenue）的百分比计算。
- 典型范围：营收变动额的 -2% 到 +2%。
- 负数 = 现金来源（营运资本释放）。
- 正数 = 现金使用（营运资本占用）。

**维持性与增长性资本支出：**
- 维持性 CapEx：维持当前运营（约占营收 2-3%）。
- 增长性 CapEx：支持扩张（额外占营收 2-5%）。
- 总资本支出应与公司的增长战略保持一致。

### 第 6 步：资本成本 (WACC) 研究

**权益资本成本的 CAPM 方法：**

```
权益资本成本 (Cost of Equity) = 无风险利率 + Beta × 权益风险溢价

其中：
- 无风险利率 = 当前 10 年期国债收益率
- Beta = 5 年期月度股票 Beta 值（对比市场指数）
- 权益风险溢价 = 5.0-6.0% (市场标准)
```

**债务资本成本计算：**

```
税后债务成本 = 税前债务成本 × (1 - 税率)

通过以下方式确定税前债务成本：
- 信用评级（如有）
- 公司债券的当前收益率
- 财务报表中的利息支出 / 总债务
```

**资本结构权重：**

```
权益市场价值 = 当前股价 × 发行在外股数
净债务 = 总债务 - 现金及现金等价物
企业价值 (EV) = 市值 + 净债务

权益权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值

WACC = (权益资本成本 × 权益权重) + (税后债务成本 × 债务权重)
```

**特殊情况：**
- **净现金头寸**：如果现金 > 债务，则净债务为负数。
  - 债务权重可能为负。
  - WACC 计算据此调整。
- **无债务**：WACC = 权益资本成本。

**典型 WACC 范围：**
- 大盘、稳定型：7-9%
- 成长型公司：9-12%
- 高增长/高风险：12-15%

### 第 7 步：折现率应用（5-10 年预测）

**期中折现惯例 (Mid-Year Convention)：**
- 假设现金流发生在年中。
- 折现期：0.5, 1.5, 2.5, 3.5, 4.5 等。
- 折现因子 = 1 / (1 + WACC)^折现期

**现值计算：**
```
对于每个预测年度：
FCF 的现值 = 无杠杆 FCF × 折现因子

示例 (第 1 年):
FCF = $1,000
WACC = 10%
折现期 = 0.5
折现因子 = 1 / (1.10)^0.5 = 0.9535
现值 = $1,000 × 0.9535 = $954
```

**预测期选择：**
- **5 年**：大多数分析的标准做法。
- **7-10 年**：拥有较长增长跑道的高增长公司。
- **3 年**：成熟、稳定的业务。

### 第 8 步：终端价值计算

**永续增长法 (Perpetuity Growth Method) [首选]：**

```
终端 FCF = 最后一年 FCF × (1 + 终端增长率)
终端价值 = 终端 FCF / (WACC - 终端增长率)

核心约束：终端增长率 < WACC (否则产生无限价值)
```

**终端增长率选择：**
- 保守：2.0-2.5% (GDP 增长率)
- 中等：2.5-3.5%
- 激进：3.5-5.0% (仅适用于市场领导者)

**严禁超过**：无风险利率或长期 GDP 增长率。

**退出倍数法 (Exit Multiple Method) [备选]：**
```
终端价值 = 最后一年 EBITDA × 退出倍数

退出倍数来源：
- 行业可比交易倍数
- 先例交易倍数
- 典型范围：8-15x EBITDA
```

**终端价值的现值：**
```
终端价值现值 = 终端价值 / (1 + WACC)^最后折现期

其中最后折现期考虑了时间点：
采用期中折现惯例的 5 年模型：折现期 = 4.5
```

**终端价值完备性检查：**
- 应占企业价值（EV）的 50-70%。
- 如果 >75%，模型可能过度依赖终端假设。
- 如果 <40%，检查终端假设是否过于保守。

### 第 9 步：企业价值至权益价值桥梁

**估值摘要结构：**

```
(+) 预测期 FCF 现值之和 = $X million
(+) 终端价值现值 = $Y million
= 企业价值 (EV) = $Z million

(-) 净债务 [若为负值则 + 净现金] = $A million
= 权益价值 = $B million

÷ 摊薄后发行在外股数 = C million shares
= 隐含每股股价 = $XX.XX

当前股价 = $YY.YY
隐含回报 = (隐含价格 / 当前价格) - 1 = XX%
```

**关键调整项目：**
- **净债务 = 总债务 - 现金及等价物**
  - 若为正：从 EV 中减去（降低权益价值）。
  - 若为负（净现金）：加回至 EV（增加权益价值）。
- **使用摊薄后股数**：包括期权、受限股（RSU）、可转债。
- **其他调整项目**（如适用）：
  - 少数股东权益
  - 养老金负债
  - 经营租赁负债

**估值输出格式：**
```csv
估值组件,金额 ($M)
显性 FCF 现值,X.X
终端价值现值,Y.Y
企业价值,Z.Z
(-) 净债务,A.A
权益价值,B.B
,,
发行在外股数 (M),C.C
隐含每股股价,$XX.XX
当前每股股价,$YY.YY
隐含上行/下行空间,+XX%
```

### 第 10 步：敏感性分析

在 DCF 工作表底部构建**三张敏感性分析表**，展示估值随不同假设的变化情况：

1. **WACC 对比终端增长率** - 展示企业价值对折现率和永续增长率的敏感性。
2. **营收增长率 对比 EBIT 利润率** - 展示顶线增长和经营杠杆的影响。
3. **Beta 对比无风险利率** - 展示对权益成本组成部分的敏感性。

**实现方式**：这些是简单的 2D 网格（**不是** Excel 的“模拟运算表”功能），每个单元格中都有公式。每个单元格必须包含针对该假设组合的完整 DCF 重新计算公式。详细要求请参阅“核心约束”章节关于使用 openpyxl 编程填充所有 75 个单元格的部分。

### 情景板块选择模式 —— 遵循此方法

**假设按每个情景的独立板块进行组织：**

**核心结构 —— 每个部分标题占三行：**

```csv
悲观情景假设 (BEAR CASE ASSUMPTIONS) (部分标题，跨列合并)
假设项目,FY1,FY2,FY3,FY4,FY5
营收增长 (%),12%,10%,9%,8%,7%
EBIT 利润率 (%),45%,44%,43%,42%,41%

基准情景假设 (BASE CASE ASSUMPTIONS) (部分标题，跨列合并)
假设项目,FY1,FY2,FY3,FY4,FY5
营收增长 (%),16%,14%,12%,10%,9%
EBIT 利润率 (%),48%,49%,50%,51%,52%

乐观情景假设 (BULL CASE ASSUMPTIONS) (部分标题，跨列合并)
假设项目,FY1,FY2,FY3,FY4,FY5
营收增长 (%),20%,18%,15%,13%,11%
EBIT 利润率 (%),50%,51%,52%,53%,54%
```

**每个情景板块必须有一个列标题行**，在部分标题下方紧接着显示预测年份（FY2025E, FY2026E 等）。如果没有这一行，用户无法区分哪个值对应哪个年份。

**如何引用假设 —— 创建一个汇总列：**
1. 情景选择单元格（如 B6）包含 1=Bear, 2=Base, 或 3=Bull。
2. 创建一个汇总列，使用 INDEX 或 OFFSET 公式从正确的情景板块中提取数据。
3. 预测公式引用汇总列（清晰的单元格引用）。
4. 每个情景板块包含各预测年度的完整 DCF 假设。

**推荐的汇总列模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不要使用散落在到处的情景判断语句：**
`=IF($B$6=1,[Bear板块单元格],IF($B$6=2,[Base板块单元格],[Bull板块单元格]))`

汇总列方法使得逻辑集中化，模型更易于审计。

### 正确的营收预测模式

**创建带有 INDEX 公式的汇总列，然后在预测中引用它：**

**第 1 步 - 为 FY1 增长构建汇总列：**
`=INDEX([Bear FY1 增长]:[Bull FY1 增长], 1, $B$6)`

**第 2 步 - 营收预测引用汇总列：**
`Revenue Year 1: =D29*(1+$E$10)`

其中：
- D29 = 上一年营收
- $E$10 = FY1 增长的汇总列单元格（包含 INDEX 公式）
- $B$6 = 情景选择器 (1=Bear, 2=Base, 3=Bull)

**这种方法比在每个预测公式中嵌套 IF 语句更整洁**，且更容易审计当前使用的是哪种情景假设。

### 正确的 FCF 公式模式

**使用带有 INDEX 公式的汇总列，然后在 FCF 计算中引用它们：**

**汇总列方法：**
```csv
项目,公式,参考说明
折旧摊销 (D&A),=E29*$E$21,$E$21 = 折旧占营收比的汇总列
资本支出 (CapEx),=E29*$E$22,$E$22 = 资本支出占营收比的汇总列
净营运资本变动 (Δ NWC),=(E29-D29)*$E$23,$E$23 = 营运资本变动占营收比的汇总列
无杠杆 FCF,=E57+E58-E60-E62,E57=NOPAT E58=D&A E60=CapEx E62=Δ NWC
```

**每个汇总列单元格都包含一个 INDEX 公式**，根据情景选择器从相应的情景板块中提取数据。这能保持预测公式的简洁和可审计性。

在写入公式之前，请先确认情景板块的行位置并设置好汇总列。

### 正确的单元格批注格式

**每一个硬编码数值都需要这种格式：**

“来源：[系统/文档], [日期], [参考内容], [URL（如有）]”

**示例：**
```csv
项目,来源批注
股价,来源：市场数据脚本 2025-10-12 收盘价
发行在外股数,来源：10-K FY2024 第 45 页 附註 12
历史营收,来源：10-K FY2024 第 32 页 合并报表
Beta,来源：市场数据脚本 2025-10-12 5 年期月度 Beta
一致预测预期,来源：管理层指引 2024 年第三季度业绩电话会
```

**关键：每个情景板块需要三个结构元素：**

1. **部分标题行**（合并单元格）：例如“悲观情景假设 (BEAR CASE ASSUMPTIONS)”
2. **列标题行**显示年份 —— 必须包含，严禁跳过。
3. **数据行**显示假设数值。

**结构：**
```csv

假设项目,FY1,FY2,FY3,FY4,FY5
营收增长 (%),X%,X%,X%,X%,X%

营收增长 (%),X%,X%,X%,X%,X%
EBIT 利润率 (%),X%,X%,X%,X%,X%
终端增长率,X%,,,,
WACC,X%,,,,

假设项目,FY1,FY2,FY3,FY4,FY5

EBIT 利润率 (%),X%,X%,X%,X%,X%

```

**如果缺少显示预测年份（FY2025E, FY2026E 等）的列标题行，用户将无法判断假设值对应哪一年。此行是强制性的。**

**然后创建一个汇总列**（通常在右侧下一列），使用 INDEX 公式根据情景选择器从选定的情景板块中提取数值。所有的预测公式应引用该汇总列。

### 正确的敏感性分析表实现

**重要提示**：这些**不是** Excel 的“模拟运算表”功能。这些是使用 openpyxl 写入常规公式的简单网格。是的，这意味着总共约 75 个公式（3 张表 × 每张 25 个单元格），但这非常直接且必不可少。

**使用公式进行编程化填充：**

每张敏感性分析表必须完整填充公式，这些公式针对每种假设组合重新计算隐含股价。**不要使用 Excel 的“模拟运算表”功能**（它需要人工干预且无法通过 openpyxl 自动化）。

**实现方法 —— 具体示例：**

**表格结构 —— 5×5 网格（奇数维度，基准情景居中）：**

如果模型的基准 WACC = 9.0% 且基准终端增长率为 3.0%，围该轴对称构建轴：

```csv
WACC 与 终端增长率,  2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%,       [公式], [公式], [公式], [公式], [公式]
              8.5%,       [公式], [公式], [公式], [公式], [公式]
              9.0%,       [公式], [公式], [★  ], [公式], [公式]   ← 中间行 = 基准 WACC
              9.5%,       [公式], [公式], [公式], [公式], [公式]
             10.0%,       [公式], [公式], [公式], [公式], [公式]
                                   ↑
                          中间列 = 基准终端增长率 g
```

**★ = 中心单元格。** 它的公式输出结果必须等于模型实际的隐含股价（来自估值摘要）。对该单元格应用中蓝色填充（`#BDD7EE`）和加粗字体，使基准情景在视觉上锚定。

**轴值规则：** `axis_values = [base - 2*step, base - step, base, base + step, base + 2*step]` —— 基准值对称，奇数数量确保存在中心。

**公式模式 - 单元格 B88 (WACC=8.0%, 终端增长率=2.0%):**

B88 中的公式应使用以下参数重算隐含价格：
- 来自行标题的 WACC：`$A88` (8.0%)
- 来自列标题的终端增长率：`B$87` (2.0%)

**推荐做法：** 引用主 DCF 计算逻辑但替换这些值。

**示例公式结构：**
`=([使用 $A88 作为折现率的各年 FCF 现值之和] + [使用 B$87 作为增长率且 $A88 作为 WACC 的终端价值] - [净债务]) / [股数]`

**关键点 —— 为 5x5 网格中的每个单元格写入公式（每张表 25 个，总计 75 个）。** 使用 openpyxl 在循环中程序化写入这些公式。严禁跳过此步骤或留下占位符文本。

**Python 实现模式：**
```python
# 填充敏感性分析表的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用 wacc_value 和 term_growth_value 的公式
        formula = f"=<使用 {wacc_value} 和 {term_growth_value} 重新计算的 DCF>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性分析表必须在打开模型时立即可用，无需用户进行任何手动操作。**

<common_mistakes>

本章节包含构建 DCF 模型时应避免的所有错误模式。

### 错误：简化的敏感性分析表近似值或占位符文本

**不要使用线性近似值：**

```
// 错误 - 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设呈线性关系

// 错误 - 除法捷径
B105: =B88/(1+(E48-0.07))      // 未重算完整的 DCF
```

**不要留下占位符文本：**
```
// 错误 - 占位符注释
"注：请使用 Excel 模拟运算表功能 (数据 → 模拟分析 → 模拟运算表) 来填充此表。"

// 错误 - 空单元格
[因为“这太复杂”而留空单元格]
```

**不要混淆术语：**
- ❌ “敏感性分析表需要 Excel 的模拟运算表功能”（错误 —— 那是我们无法自动化的特定 Excel 工具）
- ✅ “敏感性分析表是每个单元格都有公式的简单网格”（正确 —— 这是我们构建的内容）

**为什么这些捷径是错误的：**
- 线性近似公式并未真正重新计算 DCF —— 它们只是应用了简单的数学调整。
- 估值关系是非线性的，因此结果将不准确。
- 占位符文本需要用户手动干预。
- 交付模型时模型并非立即可用。
- 不够专业，无法交付给客户。
- 空单元格 = 未完成的交付成果。

**常见的“拒做”理由：**
“写 75+ 个公式感觉太复杂了，我还是留个便签让用户手动完成吧。”

**现实情况：** 使用 Python 的 openpyxl 循环，写 75 个公式非常简单。每个公式都遵循相同的模式 —— 只是替换行列引用。这是交付成果的必须部分。

**你应该：** 为每个敏感性单元格填充公式，针对特定的假设组合重新计算完整的 DCF。

### 错误：缺少单元格批注

**不要这样做：**
- 创建了所有硬编码输入却不加批注。
- 想着“我待会儿再加”。
- 写上“TODO：添加来源”。
- 留下没有文档说明的蓝色输入值。

**为什么这是错误的：**
- 无法核实数据来源。
- 违反了 xlsx 技能要求。
- 无法通过审计。
- 浪费以后修复的时间。

**你应该：** 在创建每个硬编码数值的同时添加单元格批注。

### 错误：公式行引用偏移

**症状：**
FCF 部分引用了错误的假设行：
`折旧摊销: =E29*$E$34    // 应为 $E$21，但引用了错误行`
`资本支出: =E29*$E$41    // 应为 $E$22，但行号发生了偏移`

**为什么会发生这种情况：**
1. 先写了公式。
2. 然后插入了标题行。
3. 所有行引用发生了偏移。
4. 现在公式指向错误的单元格 → 导致 #REF! 错误。

**你应该：** 先锁定行布局，再写入公式。

### 错误：跨情景假设仅各占一行

**不要这样组织假设：**
```csv
假设项目,悲观,基准,乐观
营收增长 FY1,10%,13%,16%
营收增长 FY2,9%,12%,15%
```
这种纵向布局使得很难一眼看出单一情景下跨年份的演变情况。

**为什么这是错误的：**
- 难以直观审视每个情景内假设随年份演进的情况。
- 难以跨越整个预测期对比不同情景的假设。
- 情景逻辑的复核不够直观。

**你应该：**
- 为每个情景（Bear, Base, Bull）创建独立板块。
- 在每个板块内，将假设按预测年度水平排列。
- 这使得每个情景的假设作为一套整体更易于审视。

### 错误：缺失框线

**不要交付没有框线的模型：**
- 板块界限不清晰。
- 所有单元格混在一起。
- 难以阅读且极不专业。

**为什么这是错误的：**
- 达不到交付客户的标准。
- 难以导航查阅。
- 看起来很不专业。

**你应该：** 在所有主要板块周围添加边框。

### 错误：字体颜色错误或无颜色区分

**不要这样做：**
- 所有文字都是黑色的。
- 仅使用填充颜色（不修改字体颜色）。
- 混淆了哪些单元格应为蓝色或黑色。

**为什么这是错误的：**
- 无法区分输入值和公式。
- 审计变得不可能。
- 违反了 xlsx 技能要求。

**你应该：** 所有硬编码输入使用蓝色字体，所有公式使用黑色字体，表间链接使用绿色字体。

### 错误：基于毛利计算经营费用

**不要这样做：**
`销售费用 (S&M): =E33*0.15    // E33 = 毛利 (错误)`

**为什么这是错误的：**
- 经营费用随营收规模变动，而非毛利。
- 会导致不真实的利润率演化。
- 与企业实际运营情况不符。

**你应该：**
`销售费用 (S&M): =E29*0.15    // E29 = 营收 (正确)`

### 前五大错误摘要

1. **公式行引用偏移** → 在写入公式前，定义好所有行位置。
2. **缺少单元格批注** → 在创建单元格时立即添加批注，不要推到最后。
3. **简化的敏感性分析表** → 为所有单元格填充完整的 DCF 重算公式，而非近似值。
4. **情景板块引用错误** → 确保 IF 公式引用了正确的悲观/基准/乐观板块。
5. **缺少框线** → 添加专业的板块边框，确保交付物具备专业外观。

此外，还需注意以下错误：

### WACC 计算错误
- 在资本结构中混用账面价值和市场价值。
- 错误地使用了权益 Beta 而非资产/无杠杆 Beta。
- 债务成本中使用了错误的税率。
- 错误的无风险利率（必须使用当前的 10 年期国债收益率）。
- 未能根据净债务/净现金头寸进行调整。

### 增长假设缺陷
- 终端增长率 > WACC（会产生无限价值）。
- 预测增长率与历史表现严重不符。
- 忽视了行业增长约束。
- 营收增长与单位经济效益不匹配。
- 在没有运营依据的情况下假设利润率扩张。

### 终端价值错误
- 使用了错误的增长方法（永续增长法 vs 退出倍数法）。
- 终端价值占企业价值的 >80%（暗示过度依赖终端阶段）。
- 终端利润率与永续状态假设不一致。
- 终端价值使用了错误的折现期。

### 现金流预测错误
- 经营费用基于毛利而非营收计算。
- D&A/CapEx 百分比与业务模式错位。
- 营运资本变动计算不正确。
- 年份间税率不一致。
- NOPAT 计算错误。

**这些是最常见的错误。在开始任何 DCF 构建前，请重新阅读本节。**

</common_mistakes>

## Excel 文件创建

**本技能使用 `xlsx` 技能执行所有电子表格操作。** `xlsx` 技能提供：
- 标准化的公式构建规则。
- 数字格式规范。
- 通过 `recalc.py` 脚本自动重新计算公式。
- 全面的错误检查和验证。

本技能创建的所有 Excel 文件必须符合 `xlsx` 技能的要求，包括零公式错误和正确的重算。

## 质量标准

每一个 DCF 模型都必须最大化实现：
1. **真实的营收和利润率假设**，基于历史表现。
2. **适当的资本成本计算**，采用正确的 CAPM 方法。
3. **全面的敏感性分析**，展示估值范围。
4. **清晰的终端价值计算**，并附带支撑论据。
5. **专业的模型结构**，支持情景分析。
6. **透明的文档说明**，记录所有关键假设。

## 输入要求

### 最低要求输入
1. **公司标识**：股票代码或公司名称。
2. **增长假设**：预测期的营收增长率（或“使用一致预期”）。
3. **可选参数**：
   - 预测年限（默认：5 年）。
   - 情景案例（悲观/基准/乐观情景的增长和利润率假设）。
   - 终端增长率（默认：2.5-3.0%）。
   - 特定 WACC 输入值（如果不使用 CAPM）。

## Excel 模型结构

### 工作表架构

创建 **两张工作表**：

1. **DCF** - 主估值模型，底部附带敏感性分析。
2. **WACC** - 资本成本计算。

**核心核心**：敏感性分析表放在 DCF 工作表的**底部**（而不是单独的工作表）。这可以保证所有估值输出集中在一起。

### 公式重算（强制性）

在创建或修改 Excel 模型后，必须使用 `xlsx` 技能的 `recalc.py` 脚本**重新计算所有公式**：

```bash
python recalc.py [Excel文件路径] [超时秒数]
```

示例：
```bash
python recalc.py AAPL_DCF_Model_2025-10-12.xlsx 30
```

该脚本将：
- 使用 LibreOffice 重新计算所有工作表中的所有公式。
- 扫描所有单元格中的 Excel 错误 (#REF!, #DIV/0!, #VALUE!, #NAME?, #NULL!, #NUM!, #N/A)。
- 返回包含错误位置和数量的详细 JSON。

**预期输出格式：**
```json
{
  "status": "success",           // 或 "errors_found"
  "total_errors": 0,              // 错误总数
  "total_formulas": 42,           // 文件中的公式数量
  "error_summary": {}             // 仅在发现错误时存在
}
```

**如果发现错误**，输出将包含细节：
```json
{
  "status": "errors_found",
  "total_errors": 2,
  "total_formulas": 42,
  "error_summary": {
    "#REF!": {
      "count": 2,
      "locations": ["DCF!B25", "DCF!C25"]
    }
  }
}
```

**修复所有错误**并重新运行 `recalc.py` 直到状态为 “success”，然后方可交付模型。

### 格式标准

**重要提示**：遵循 `xlsx` 技能关于公式构建和数字格式的规范。DCF 技能在此基础上增加了特定的视觉呈现标准。

**配色方案 —— 两个图层**：

**第 1 层：字体颜色（xlsx 技能强制要求）**
- **蓝色字体 (RGB: 0,0,255)**：所有硬编码输入（股价、股数、历史数据、假设）。
- **黑色字体 (RGB: 0,0,0)**：所有公式和计算过程。
- **绿色字体 (RGB: 0,128,0)**：表间跳转链接（引用 WACC 工作表的引用项）。

**第 2 层：填充颜色 —— 专业蓝/灰色调（除非用户另有规定，否则为默认值）**
- **保持简约** —— 仅使用蓝色和灰色作为填充。严禁引入绿色、黄色、橙色或多个强调色。颜色过多的模型看起来很不专业。
- **默认填充调色板：**
  - **板块标题**：深蓝色（RGB: 31,78,121 / `#1F4E79`）背景，白色加粗字体。
  - **子标题/列标题**：浅蓝色（RGB: 217,225,242 / `#D9E1F2`）背景，黑色加粗字体。
  - **输入单元格**：浅灰色（RGB: 242,242,242 / `#F2F2F2`）背景，蓝色字体 —— 或者如果你追求极致简约，可以使用白色背景+蓝色字体。
  - **计算单元格**：白色背景，黑色字体。
  - **输出/摘要行**（每股价值、EV 等）：中蓝色（RGB: 189,215,238 / `#BDD7EE`）背景，黑色加粗字体。
- **就这些 —— 3 种蓝色 + 1 种灰色 + 白色。** 顶住增加更多颜色的冲动。
- 用户提供的模板或明确的颜色偏好始终优先于这些默认值。

**两个图层如何协同工作：**
- 输入单元格：蓝色字体 + 浅灰填充 = “硬编码输入”。
- 公式单元格：黑色字体 + 白色背景 = “计算数值”。
- 表间链接：绿色字体 + 白色背景 = “引用自其他表”。
- 关键输出：黑色加粗字体 + 中蓝填充 = “这就是答案”。

**字体颜色告诉你它“是什么”（输入/公式/链接）。填充颜色告诉你你“在哪里”（标题/数据/输出）。**

### 边框标准（专业外观的要求）

**粗边框** (1.5pt) 包围主要板块：
- 关键输入 (KEY INPUTS) 板块。
- 预测假设 (PROJECTION ASSUMPTIONS) 板块。
- 5 年现金流预测 (5-YEAR CASH FLOW PROJECTION) 板块。
- 终端价值 (TERMINAL VALUE) 板块。
- 估值摘要 (VALUATION SUMMARY) 板块。
- 每一个敏感性分析 (SENSITIVITY ANALYSIS) 表。

**中等边框** (1pt) 分隔子板块：
- 公司详情 对比 历史表现。
- 增长假设 对比 EBIT 利润率 对比 FCF 参数。

**细边框** (0.5pt) 包围数据表：
- 情景假设表（悲观 | 基准 | 乐观 | 已选）。
- 历史 对比 预测财务矩阵。

**无边框：** 表格内的单个单元格（保持清晰、易于扫读）。

**边框是强制性的** —— 没有专业边框的模型不具备交付客户的资格。

**数字格式**（遵循 xlsx 技能标准）：
- **年份**：格式化为文本字符串（如 “2024” 而不是 “2,024”）。
- **百分比**：`0.0%`（保留一位小数）。
- **货币**：百万单位使用 `$#,##0`；每股数值使用 `$#,##0.00` —— 始终在标题中指明单位（“Revenue ($mm)”）。
- **零值**：使用数字格式使所有零显示为 “-” （如 `$#,##0;($#,##0);-`）。
- **大额数字**：使用千分位分隔符 `#,##0`。
- **负数**：使用括号 `(#,##0)`（不用负号）。

**单元格批注（所有硬编码输入均强制要求）**：

根据 `xlsx` 技能，所有硬编码值必须有记录来源的单元格批注项。格式：“来源：[系统/文档], [日期], [参考内容], [URL（如有）]”。

**核心要点**：在**创建单元格时同步添加批注**。不要留到最后。

**关键 WACC 公式：**
```
市值 = 股价 × 股数
净债务 = 总债务 - 现金
企业价值 = 市值 + 净债务
权益权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值
WACC = (权益成本 × 权益权重) + (税后债务成本 × 债务权重)
```

### 敏感性分析（DCF 工作表底部）

**术语提醒**：“敏感性表” = 带有行标题、列标题且每个数据单元格中都有公式的简单 2D 网格。**不是** Excel 的“模拟运算表”功能（数据 → 模拟分析 → 模拟运算表）。你将使用 openpyxl 将常规 Excel 公式写入每个单元格。

**位置**：DCF 工作表的第 87 行及以下（**不是**单独的工作表）。

**垂直堆叠三张敏感性分析表：**

1. **WACC 对比 终端增长率** (第 87-100 行) - 5x5 网格 = 25 个带有公式的单元格。
2. **营收增长率 对比 EBIT 利润率** (第 102-115 行) - 5x5 网格 = 25 个带有公式的单元格。
3. **Beta 对比 无风险利率** (第 117-130 行) - 5x5 网格 = 25 个带有公式的单元格。

**总共需写入的公式：75 个**（这是强制要求的，而非可选）。

**核心要点**：所有敏感性分析表单元格必须使用 openpyxl 编程化填充。严禁使用线性近似快捷方式。严禁留下关于手动步骤的占位符文本或提示。严禁因为“太复杂”而留空单元格 —— 使用 Python 循环来生成公式。

**表格设置：**
1. 创建带有行列标题的表格结构（需测试的假设值）。
2. 为**每一个**数据单元格填充公式，该公式需：
   - 使用行标题值（例如 WACC = 9.0%）。
   - 使用列标题值（例如 终端增长率 = 3.0%）。
   - 针对这些特定假设重新计算完整的 DCF。
   - 返回该情景下隐含的每股股价。
3. 交付时，所有单元格必须包含可运行的公式。
4. 使用条件格式对单元格进行处理：高值使用绿色系，低值使用红色系。

6. 表格之间保留 1-2 行空行。

**无需手动干预** —— 当用户打开文件时，敏感性分析表必须是全功能可用的。

## 情景选择器实现

**三支情景框架：**

### 悲观情景 (Bear Case)
- 保守的营收增长（处于历史范围的低端）。
- 利润率受压或无扩张。
- 较高的 WACC（风险溢价上升）。
- 较低的终端增长率。
- 较高的资本支出假设。

### 基准情景 (Base Case)
- 按照市场一致预期或管理层指引的营收增长。
- 基于经营杠杆的温和利润率扩张。
- 当前市场隐含的 WACC。
- 与 GDP 保持一致的终端增长率 (2.5-3.0%)。
- 标准的资本支出假设。

### 乐观情景 (Bull Case)
- 乐观的营收增长（处于预测范围的高端）。
- 显著的利润率扩张。
- 较低的 WACC（风险溢价下降）。
- 较高的终端增长率 (3.5-5.0%)。
- 较低的资本支出强度。

**公式实现：**

**严禁到处使用嵌套的 IF 公式。** 相反，应创建一个汇总列，使用 INDEX 或 OFFSET 公式从对应的情景板块中提取数据。

**推荐模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`，其中 `B10:D10` = 悲观/基准/乐观数值，`1` = 行偏置，`$B$6` = 情景选择器（1, 2, 或 3）。

**然后在所有预测中引用该汇总列**：
`Revenue Year 1: =D29*(1+$E$10)`，其中 $E$10 是第一年增长率的汇总列对应值。

这种方法将情景逻辑集中化，使模型更易于审计和维护。

## 常见变体

### 高增长科技公司
- 较长的预测期（7-10 年）。
- 较高的初始增长率 (20-30%)。
- 随时间推移显著的利润率扩张。
- 较高的 WACC (12-15%)。
- 对单位经济效益（用户数、ARPU 等）进行建模。

### 成熟/稳定型公司
- 较短的预测期（3-5 年）。
- 温和的增长率 (GDP +1-3%)。
- 稳定的利润率。
- 较低的 WACC (7-9%)。
- 侧重于现金回流和资本配置。

### 周期性公司
- 覆盖完整经济周期的建模。
- 在周期中点实现利润率正常化。
- 考虑谷底和巅峰情景。
- 针对周期性调整 Beta 值。

### 多业务部门公司
- 为每个业务单元分别建立 DCF。
- 各部门有不同的增长率和利润率。
- 分部估值法 (SOTP)。
- 考虑协同效应。

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到估值的世界！如果你是金融小白，这里的几个核心概念就像是你的“探险指南”：
>
> 1. **DCF (现金流量折现模型)**：想象你有一棵会结金币的树。你想知道这棵树现在值多少钱，你不能只看它今年结了多少。你要预测它未来每年结多少金币，但因为“明年的 1 块钱不如今天的 1 块钱值钱”，所以我们要把未来的钱都“打个折”算到现在，这就是 DCF 实质在做的事。
> 2. **WACC (加权平均资本成本)**：这可以理解为公司用钱的“成本”或是投资人的“底线收益率”。公司不管是借钱（债务）还是找人投资（权益），都要付代价。WACC 就是把这些代价加权算出一个综合的百分比。在 DCF 里，它就是那个把未来钱“打折”的折现率。
> 3. **自由现金流 (FCF)**：这是公司在维持经营、买完设备后，真正能装进兜里“随意支配”的钱。利润可能是账面数字，但 FCF 才是真金白银。
> 4. **终端价值 (Terminal Value)**：我们不可能无限预测下去。通常预测 5-10 年后，我们假设公司进入了一个稳定的“永续增长”状态。这段从第 6 年或第 11 年直到永远的价值，就叫终端价值。它通常占公司总价值的一大半。
> 5. **敏感性分析**：金融预测不是算命，没人能百分之百准。所以我们要测试：如果 WACC 变高一点，或者增长慢一点，估值会怎么变？这就像是给估值做个“压力测试”，看看在不同天气下，公司的价值稳不稳。
