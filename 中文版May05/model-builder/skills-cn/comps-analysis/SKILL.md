# 可比公司分析 (Comparable Company Analysis)

## ⚠️ 至关重要：数据源优先级（必读）

**请始终遵循以下数据源层级：**

1. **首选：检查 MCP 数据源** - 如果可以使用 S&P Kensho MCP、FactSet MCP 或 Daloopa MCP，请优先且仅使用这些工具获取财务和交易信息。
2. **不要使用网页搜索** - 如果上述 MCP 数据源可用，请严禁使用网页搜索。
3. **仅在 MCP 不可用时：** 才使用 Bloomberg 终端、SEC EDGAR 申报文件或其他机构级来源。
4. **切勿将网页搜索作为主要数据源** - 网页搜索缺乏机构级分析所需的准确性、审计追踪和可靠性。

**为什么这很重要：** MCP 来源提供经过验证的机构级数据，并附带规范的引用。网页搜索结果对于财务分析而言可能已经过时、不准确或不可靠。

---

## 概述
本技能旨在指导 Claude 构建机构级的可比公司分析，将运营指标、估值倍数和统计基准分析有机结合。输出结果为一个结构化的 Excel/电子表格，通过同行对比实现明智的投资决策。

**参考材料与情境化：**

在 `examples/comps_example.xlsx` 中提供了一个可比公司分析示例。在处理此目录下的示例文件时，请智能使用：

**建议将示例用于：**
- 理解结构层级（各板块的流转逻辑）
- 把握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰的表头、透明的公式、审计追踪）

**不要将示例用于：**
- 机械套用格式或指标
- 不考虑具体情境地复制布局
- 忽视受众群体而强行应用相同的视觉风格

**请始终先问自己：**
1. **“您是否有偏好的格式，还是我应该调整模板风格？”**
2. **“受众是谁？”**（投资委员会、董事会演示、快速参考、详细备忘录）
3. **“核心问题是什么？”**（估值、增长分析、竞争地位、效率）
4. **“具体情境是什么？”**（并购评估、投资决策、行业基准分析、业绩评估）

**根据具体情况进行调整：**
- **行业背景**：大型科技股需要的指标与新兴 SaaS 初创公司完全不同。
- **行业特定需求**：及早加入相关指标（例如，科技行业的云 ARR、企业客户数、开发者生态系统）。
- **公司熟悉度**：知名公司可能不需要太多背景介绍，而应更多关注差异分析（Delta Analysis）。
- **决策类型**：并购分析与持续投资组合监控的侧重点不同。

**核心原则：** 沿用模板原则（结构清晰、统计严谨、公式透明），但根据情境调整执行方式。目标是提供“机构级质量”的分析，而非“看起来像机构”的模板。

用户提供的示例和明确偏好始终优先于默认设置。

## 核心哲学
**“先构建正确的结构，再让数据诉说事实。”**

从强制触发战略思考的表头开始，输入干净的数据，构建透明的公式，让统计结论自动浮现。一份优秀的可比分析应当能让非制作者也能立即读懂。

## ⚠️ 至关重要：公式替代硬编码 + 逐步验证

**环境 —— Office JS vs Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS (`Excel.run(async (context) => {...})`)。通过 `range.formulas = [["=E7/C7"]]` 写入公式，而非 `range.values`。无需单独的重算步骤 —— Excel 会原生处理。使用 `range.format.*` 设置颜色/字体。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。写入 `cell.value = "=E7/C7"`（公式字符串）。
- 无论哪种方式，原则一致 —— 仅需转换 API 调用。
- **Office JS 合并单元格陷阱：** 切勿先调用 `.merge()` 再对合并范围设置 `.values`（会抛出 `InvalidArgument` 错误 —— 范围仍报告其合并前的维度）。应先将值写入左上角单元格，然后再进行合并及格式化：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**使用公式，而非硬编码：**
- 每个衍生值（利润率、倍数、统计数据）必须是引用输入单元格的 Excel 公式 —— 严禁粘贴预先计算好的数字。
- 使用 Python/openpyxl 构建表格时：写入 `cell.value = "=E7/C7"`（公式字符串），而不是 `cell.value = 0.687`（计算结果）。
- 唯一的硬编码值应为原始输入数据（营收、EBITDA、股价等） —— 且每个此类数值都应附带注明出处的单元格批注。
- 原因：模型必须在输入发生变化时自动更新。硬编码的利润率是一个随时可能爆发的“静默漏洞”。

**与用户进行分步验证：**
- 设置完结构后 → 在填充数据前向用户展示表头布局。
- 输入原始数据后 → 在构建公式前向用户展示输入块并确认来源/期间。
- 构建运营指标公式后 → 在进入估值板块前向用户展示计算出的利润率并进行合理性检验。
- 构建估值倍数后 → 在添加统计数据前确认倍数看起来是否合理。
- 不要从头到尾建完整个表格再展示 —— 通过确认每个板块来及早发现错误。

## 第 1 节：文档结构与设置

### 表头块 (第 1-3 行)
```
第 1 行: [分析标题] - 可比公司分析 (COMPARABLE COMPANY ANALYSIS)
第 2 行: [公司列表及代码] • [公司 1 (代码1)] • [公司 2 (代码2)] • [公司 3 (代码3)]
第 3 行: 截至 [期间] | 除每股金额和比率外，所有数字均以 [百万/十亿美元 USD] 为单位
```

**为什么这很重要：** 立即确立情境。任何打开此文件的人都能知道他们在看什么、是什么时候创建的，以及如何解读这些数字。

### 视觉规范标准（可选 —— 用户偏好和上传的模板始终优先）

**重要提示：这些仅为建议的默认设置。请始终优先考虑：**
1. 用户的明确格式偏好
2. 任何上传模板文件中的格式
3. 公司/团队的风格指南
4. 这些默认值（仅在未提供其他指导时使用）

**建议字体与排版：**
- **字体族**：Times New Roman（专业、清晰、行业标准）
- **字号**：数据单元格 11pt，表头 12pt
- **加粗文本**：板块标题、公司名称、统计标签

**默认颜色与阴影 —— 专业蓝/灰调（极简为佳）：**
- **保持克制** —— 仅使用蓝色和灰色。严禁加入绿色、橙色、红色或多种辅助色。一份干净的可比分析表总共只应使用 3-4 种颜色。
- **板块标题**（例如，“运营统计与财务指标”）：
  - 深蓝色背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色加粗文本
  - 跨所有列的整行阴影
- **列标题**（例如，“公司”、“营收”、“利润率”）：
  - 浅蓝色背景（`#D9E1F2` 或类似的淡蓝色）
  - 黑色加粗文本
  - 居中对齐
- **数据行**：
  - 公司数据使用白色背景
  - 公式使用黑色文本；硬编码输入使用蓝色文本
- **统计行**（最大值、75 分位数等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文本，标签左对齐
- **这就是全部配色**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户模板另有说明，否则不使用其他颜色。

**建议格式规范：**
- **小数精度**：
  - 百分比：1 位小数 (12.3%)
  - 倍数：1 位小数 (13.5x)
  - 金额：不保留小数，使用千位分隔符 (69,632)
  - 以百分比显示的利润率：1 位小数 (68.7%)
- **边框**：无边框（外观整洁、简约）
- **对齐**：所有指标居中对齐，以获得统一、整洁的外观
- **单元格尺寸**：所有列宽应整齐一致，所有行高应保持一致（构建专业的网格感）

**注意：** 如果用户提供了模板文件或指定了不同的格式，请改用其提供的格式。

## 第 2 节：运营统计与财务指标

### 核心列（以此开始）
1. **公司 (Company)** - 格式统一的名称
2. **营收 (Revenue)** - 规模指标（根据情境可为 LTM、季度或年度）
3. **营收增长率 (Revenue Growth)** - 同比百分比变化
4. **毛利 (Gross Profit)** - 营收减去营业成本
5. **毛利率 (Gross Margin)** - 毛利/营收（基础获利能力）
6. **EBITDA** - 税息折旧及摊销前利润
7. **EBITDA 利润率 (EBITDA Margin)** - EBITDA/营收（运营效率）

### 可选补充（根据行业/目的选择）
- **季度 vs LTM** - 如果存在季节性影响，则两者均包含
- **自由现金流 (Free Cash Flow)** - 适用于资本密集型或 SaaS 业务
- **FCF 利润率** - FCF/营收（现金产生效率）
- **净利润 (Net Income)** - 适用于成熟且盈利的公司
- **营业利润 (Operating Income)** - 适用于折旧摊销 (D&A) 差异较大的业务
- **资本支出 (CapEx) 指标** - 适用于重资产行业
- **40 原则 (Rule of 40)** - 专用于 SaaS（增长率% + 利润率%）
- **FCF 转换率** - 适用于收益质量分析（高级）

### 公式示例（以第 7 行为例）
```excel
// 核心比率 - 始终计算所得
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率 - 如相关则包含
FCF 利润率: =[FCF]/[营收]
净利率: =[净利润]/[营收]
40 原则: =[增长率%]+[FCF 利润率%]
```

**金律：** 每个比率都应该是 [某指标] / [营收] 或 [某指标] / [本表中的另一指标]。保持简单。

### 统计块 (位于公司数据之后)

**至关重要：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留出一个空行用于视觉分隔]
- 最大值 (Maximum): =MAX(B7:B9)
- 75 分位数 (75th Percentile): =QUARTILE(B7:B9,3)
- 中位数 (Median): =MEDIAN(B7:B9)
- 25 分位数 (25th Percentile): =QUARTILE(B7:B9,1)
- 最小值 (Minimum): =MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- 营收增长率 %、毛利率 %、EBITDA 利润率 %、EPS
- EV/营收、EV/EBITDA、P/E、股息率 %、Beta

**不需要统计的列（规模指标）：**
- 营收、EBITDA、净利润（绝对规模随公司规模而异）
- 市值、企业价值（在不同规模的公司之间不可比）

**注意：** 在公司数据和统计行之间添加一个空行作为视觉分隔。不要添加“行业统计”或“估值统计”等表头行。

**为什么分位数很重要：** 它们展示了分布情况，而不仅仅是平均值。75 分位数倍数能告诉你“优质”公司的交易水平。

## 第 3 节：估值倍数与投资指标

### 核心估值列（以此开始）
1. **公司** - 与运营部分顺序一致
2. **市值 (Market Cap)** - 当前市场估值
3. **企业价值 (Enterprise Value)** - 市值 ± 净负债/现金
4. **EV/营收** - 市场为每 1 美元营收支付的对价
5. **EV/EBITDA** - 市场为每 1 美元盈余支付的对价
6. **市盈率 (P/E Ratio)** - 股价相对于净利润的比率

### 可选估值指标（根据情境选择）
- **FCF 收益率 (FCF Yield)** - FCF/市值（用于关注现金的分析）
- **PEG 比率** - P/E/增长率（用于增长型公司）
- **市净率 (Price/Book)** - 市值与账面价值对比（用于重资产业务）
- **ROE/ROA** - 回报指标（用于获利能力对比）
- **营收/EBITDA 复合年增长率 (CAGR)** - 历史增长率（用于趋势分析）
- **资产周转率** - 营收/资产（用于运营效率）
- **权益乘数 (Debt/Equity)** - 杠杆率（用于资本结构分析）

**核心原则：** 包含 3-5 个对你所在行业至关重要的核心倍数。不要仅仅因为可以就包含所有可能的指标。

### 公式示例
```excel
// 核心倍数 - 始终包含这些
EV/营收: =[企业价值]/[LTM 营收]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
市盈率 (P/E): =[市值]/[净利润]

// 可选倍数 - 如果数据可用则包含
FCF 收益率: =[LTM FCF]/[市值]
PEG 比率: =[P/E]/[增长率 %]
```

### 交叉引用规则
**至关重要：** 估值倍数必须引用运营指标板块。切勿重复输入两次相同的原始数据。如果营收在 C7，那么 EV/营收的公式应直接引用 C7。

### 统计块
结构与运营部分一致：每个指标都有最大值、75 分位数、中位数、25 分位数、最小值。在公司数据和统计数据之间留出一个空行。不要添加“估值统计”表头行。

## 第 4 节：注释与方法论说明

### 必需组件

**数据来源与质量：**
- 数据来源？（S&P Kensho MCP, FactSet MCP, Daloopa MCP, Bloomberg, SEC 申报文件）
- 涵盖期间？（2024 年第四季度，经审计数据）
- 如何验证？（与 10-K/10-Q 进行了核对）
- 注意：如果可用，请优先使用 MCP 数据源，以获得更好的准确性和可追溯性。

**关键定义：**
- EBITDA 计算方法（毛利 + 折旧摊销，或营业利润 + 折旧摊销）
- 自由现金流公式（经营现金流 - 资本支出）
- 特殊指标解释（40 原则、FCF 转换率）
- 时间段定义（LTM、CAGR 计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净负债）
- 使用了什么增长率？（历史 CAGR、前瞻性预测）
- 进行了哪些调整？（剔除一次性项目、利润率标准化）

**分析框架：**
- 投资逻辑是什么？（云/SaaS 效率）
- 哪些指标最重要？（现金产生、资本效率）
- 读者应如何解读统计数据？（分位数提供背景参考）

## 第 5 节：选择正确的指标（决策框架）

### 以“我要回答什么问题？”开始

**“哪家公司被低估了？”**
→ 关注：EV/营收、EV/EBITDA、P/E、市值
→ 跳过：运营细节指标、增长指标

**“哪家公司效率最高？”**
→ 关注：毛利率、EBITDA 利润率、FCF 利润率、资产周转率
→ 跳过：规模指标、绝对金额数字

**“哪家公司增长最快？”**
→ 关注：营收增长率 %、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆率

**“谁是最好的现金产生者？”**
→ 关注：FCF、FCF 利润率、FCF 转换率、资本支出强度
→ 跳过：EBITDA、P/E 比率

### 行业特定指标选择

**软件/SaaS：**
必需：营收增长、毛利率、40 原则
可选：ARR、净金额留存率 (NDR)、CAC 回本周期
跳过：资产周转率、库存指标

**制造/工业：**
必需：EBITDA 利润率、资产周转率、资本支出/营收
可选：ROA、库存周转率、在手订单 (Backlog)
跳过：40 原则、SaaS 指标

**金融服务：**
必需：ROE、ROA、效率比率 (Efficiency Ratio)、P/E
可选：净息差 (NIM)、贷款损失准备
跳过：毛利率、EBITDA（对银行而言无意义）

**零售/电子商务：**
必需：营收增长、毛利率、库存周转率
可选：同店销售增长、客户获取成本 (CAC)
跳过：高研发或资本支出指标

### “5-10 原则”

**5 个运营指标** - 营收、增长、2-3 个利润率/效率指标
**5 个估值指标** - 市值、EV、3 个估值倍数
**= 10 个总列数** - 足以讲好故事，且不至于多到迷失重点。

如果你有超过 15 个指标，可能包含太多杂音。应果断删减。

## 第 6 节：最佳实践与质量检查

### 开始之前
1. **定义同行组 (Peer Group)** - 公司必须具备真正的可比性（相似的商业模式、规模、地域）。
2. **选择正确的期间** - LTM 可平滑季节性；季度数据可展示趋势。
3. **预先统一单位** - 选择“百万”还是“十亿”会影响全局。
4. **映射数据源** - 明确每个数字的出处。

### 制作过程中
1. **先输入所有原始数据** - 在编写公式前完成蓝色文本的填充。
2. **为所有硬编码输入添加单元格批注** - 右键单元格 → 插入批注 → 注明出处或假设。

   **对于获取的数据，请准确引用出处：**
   - 示例：“Bloomberg Terminal - MSFT Equity DES, accessed 2024-10-02”
   - 示例：“2024 Q4 10-K filing, page 42, line item 'Total Revenue'”
   - 示例：“FactSet consensus estimate as of 2024-10-02”
   - **尽可能加入超链接**：右键单元格 → 链接 → 粘贴 SEC 申报文件、数据源或报告的 URL。

   **对于假设，解释其逻辑：**
   - 示例：“基于同行中位数假设 15% 的 EBITDA 利润率，公司未披露该项数据”
   - 示例：“估算的 企业价值 = 市值 + $50M 净负债（源自 Q3 资产负债表，Q4 尚未发布）”
   - 示例：“远期 P/E 基于市场一致预期的 $3.45 EPS（12 位分析师预测值的平均值）”

   **为什么这很重要**：支持审计追踪、数据核查、假设透明度以及未来的模型更新。
3. **逐行构建公式** - 在继续下一步之前测试每个计算。
4. **对表头使用绝对引用** - `$C$6` 用于锁定表头行。
5. **格式保持一致** - 百分比显示为百分数，而不是小数。
6. **添加条件格式** - 自动突出显示离群值。

### 合理性检验
- **利润率测试**：毛利率 > EBITDA 利润率 > 净利率（按定义始终成立）。
- **倍数合理性**：
  - EV/营收：通常为 0.5-20x（因行业而异）。
  - EV/EBITDA：通常为 8-25x（各行业相对一致）。
  - P/E：通常为 10-50x（取决于增长率）。
- **增长与倍数的关联**：高增长通常意味着高倍数。
- **规模与效率的权衡**：较大的公司通常拥有更高的利润率（规模效益）。

### 应避免的常见错误
❌ 在公式中混合使用市值和企业价值。
❌ 分子和分母使用不同的时间段（LTM vs 季度）。
❌ 在公式中硬编码数字而非单元格引用。
❌ **硬编码输入未附带注明出处或解释假设的单元格批注。**
❌ 在有条件的情况下缺失指向 SEC 文件或数据源的超链接。
❌ 包含太多没有明确目的指标。
❌ 包含不可比的公司（商业模式不同）。
❌ 使用过时数据且未加说明。
❌ 错误地计算百分比的平均值（应使用中位数）。

### 步骤流程
1. **设置结构** (30 分钟)
   - 创建所有表头。
   - 格式化单元格（蓝色用于输入，黑色用于公式）。
   - 锁定单位和日期引用。

2. **收集数据** (60-90 分钟)
   - 从主要来源提取（首选 MCP；其次 Bloomberg, SEC）。
   - 以蓝色输入所有原始数字。
   - 在注释部分记录来源。

3. **构建公式** (30 分钟)
   - 从简单比率（利润率）开始。
   - 进阶到估值倍数 (EV/营收)。
   - 添加交叉检查（利润率是否合乎逻辑？）。

4. **添加统计数据** (15 分钟)
   - 为所有列复制公式结构。
   - 验证引用范围是否正确 (B7:B9，而非 B7:B10)。
   - 检查四分位数逻辑。

5. **质量控制** (30 分钟)
   - 进行合理性检验。
   - 验证公式引用。
   - 检查 #DIV/0! 或 #REF! 错误。
   - 与已知基准进行对比。

6. **撰写文档** (15 分钟)
   - 完成注释部分。
   - 添加数据源。
   - 定义方法论。
   - 为分析打上日期戳。

## 第 10 节：红旗与警示信号

### 数据质量问题
🚩 时间段不一致（混合使用了季度和年度数据）。  
🚩 缺少数据且未加解释。  
🚩 数据源之间存在显著差异（偏差 >10%）。

### 估值红旗
🚩 对 EBITDA 为负的公司使用 EBITDA 倍数（应使用营收倍数）。  
🚩 在没有爆发式增长逻辑的情况下，P/E 比率 >100x。  
🚩 利润率在行业逻辑内讲不通。

### 可比性问题
🚩 财政年度截止日期不同（会导致时间匹配问题）。  
🚩 混合了纯业务公司（Pure-play）和控股集团公司。  
🚩 商业模式存在本质差异却被贴上“可比公司”标签。

**如有疑问，剔除该公司。** 宁要 3 个完美的可比，也不要 6 个存疑的可比。

## 第 11 节：公式参考指南

### 基础 Excel 公式
```excel
// 统计函数
=AVERAGE(范围)          // 算术平均值
=MEDIAN(范围)           // 中位数
=QUARTILE(范围, 1)      // 25 分位数
=QUARTILE(范围, 3)      // 75 分位数
=MAX(范围)              // 最大值
=MIN(范围)              // 最小值
=STDEV.P(范围)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司的平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除零错误

// 跨表引用
='表1'!B7               // 引用另一个工作表
=VLOOKUP(A7, 表1, 2)     // 从数据表中查找
=INDEX(MATCH())          // 高级查找
```

### 常见比率公式
```excel
毛利率 = 毛利 / 营收
EBITDA 利润率 = EBITDA / 营收
FCF 利润率 = 自由现金流 / 营收
FCF 转换率 = FCF / 经营活动现金流
ROE = 净利润 / 股东权益
ROA = 净利润 / 总资产
资产周转率 = 营收 / 总资产
权益乘数 = 总负债 / 股东权益
```

## 核心原则总结

1. **结构决定洞察** - 正确的表头强制产生正确的思考。
2. **少即是多** - 10 个关键指标胜过 20 个无关紧要的数字。
3. **根据问题选择指标** - 估值分析 ≠ 效率分析。
4. **统计揭示规律** - 中位数/分位数比平均值更能反映现实。
5. **透明胜过复杂** - 使用每个人都能理解的简单公式。
6. **可比性是核心** - 宁缺毋滥，不要强行加入不匹配的公司。
7. **记录你的选择** - 在注释部分解释为什么选择这些指标。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **什么是 Comps？**
>    就像买房子要看附近邻居卖多少钱一样，“Comps”就是通过看市场上“长得像”的公司的表现，来给自己的公司定个价。
>
> 2. **企业价值 (EV) vs. 市值 (Market Cap)**
>    “市值”只是股票的价格总额，而“企业价值”是你买下整个公司真正要付的钱。就像买带房贷的房子：你要付房款（市值），还得把房贷（债务）接过来。
>
> 3. **为什么大家总在谈 EBITDA？**
>    它扣除了税收、利息和那些会计上的折旧抹账，能最直观地看到一个公司靠核心业务“搞钱”的能力，是跨公司对比的神器。
>
> 4. **倍数 (Multiples) 是什么意思？**
>    它是财务界的“价格标签”。比如 P/E 是 20 倍，意思就是你想赚到 1 块钱利润，得先花 20 块钱买股票。
>
> 5. **为什么要看分位数（75/25 分位）？**
>    平均数容易被一个超级好或者超级烂的公司带偏。分位数能让你知道：如果你是行业里的优等生（75 分位档次），该值多少钱？这样比更客观。
