# 数据清洗 (Clean Data)

清洗活跃工作表或指定范围内的凌乱数据。

## 环境

- **如果运行在 Excel 内部（Office 加载项 / Office JS）：** 直接使用 Office JS (`Excel.run(async (context) => {...})`)。通过 `range.values` 读取数据，通过 `range.formulas = [["=TRIM(A2)"]]` 写入辅助列公式。“原地更新”与“辅助列更新”的决策逻辑依然适用。
- **如果操作独立 .xlsx 文件：** 使用 Python/openpyxl。

## 工作流

### 步骤 1：确定范围

- 如果已指定范围（例如 `A1:F200`），则使用该范围。
- 否则，使用当前活跃工作表的所有已使用范围 (Used Range)。
- 对每一列进行画像分析：检测其主导数据类型（文本 / 数值 / 日期）并识别离群值 (outliers)。

### 步骤 2：检测问题

| 问题类型 | 识别特征 |
|---|---|
| 空格 (Whitespace) | 首尾空格、双空格 |
| 大小写 (Casing) | 分类列中大小写不统一（例如 `usa` / `USA` / `Usa`） |
| 文本格式数值 | 以文本形式存储的数值单元格；数值中混入多余的 `$`、`,`、`%` |
| 日期 (Dates) | 同一列中存在混合格式（例如 `3/8/26`、`2026-03-08`、`March 8 2026`） |
| 重复项 (Duplicates) | 完全重复的行，以及近似重复项（仅大小写或空格存在差异） |
| 空值 (Blanks) | 在数据密集列中出现的空单元格 |
| 混合类型 (Mixed types) | 某列 98% 为数字，但包含 3 条文本条目 |
| 编码 (Encoding) | 乱码 (Mojibake，如 `Ã©`、`â€™`)、不可打印字符 |
| 错误值 (Errors) | `#REF!`、`#N/A`、`#VALUE!`、`#DIV/0!` |

### 步骤 4：应用执行

- **优先使用公式而非硬编码的清洗值** —— 凡是清洗结果能用公式表达的情况（例如 `=TRIM(A2)`、`=VALUE(SUBSTITUTE(B2,"$",""))`、`=UPPER(C2)`、`=DATEVALUE(D2)`），应在相邻的辅助列中写入公式，而非在 Python 中计算结果后直接覆盖原值。这能确保数据转换过程的透明性与可审计性 (auditable)。
- 仅在用户明确要求或无法通过公式实现（例如修复编码/乱码）时，才在原地覆盖计算后的数值。
- 对于破坏性操作（删除重复项、填充空值、覆盖原值），必须先取得用户确认。
- 在完成每一类修复（空格 → 大小写 → 数值转换 → 日期 → 去重）后，向用户展示变更示例并获取确认，再进行下一类修复。
- 最终报告修复前后的变动摘要。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **数据清洗 (Data Cleaning)**：在金融分析中，这就像是在盖房子前清理地基。如果原始数据里有空格或乱码，求和公式或折现模型就会报错。清洗就是为了确保数据的“准确性”和“一致性”。
> 2. **文本格式数值 (Number-as-text)**：这在从银行系统导出数据时经常发生。虽然肉眼看起来是数字，但 Excel 把它当成文字，导致无法进行加减乘除。将其转换回数值格式是金融建模的第一步。
> 3. **辅助列 (Helper Column)**：这是 Excel 高手的“秘密武器”。直接修改原始数据可能会导致无法回溯。建立一个辅助列写上公式，不仅能清洗数据，还能让审计人员一眼看出数据经历了怎样的转换，这在金融合规中非常重要。
> 4. **冗余空格 (Whitespace)**：别小看单元格末尾的一个小空格，它会让 `VLOOKUP` 等查找公式失效。数据清洗能自动把这些“看不见的麻烦”通通扫除。
