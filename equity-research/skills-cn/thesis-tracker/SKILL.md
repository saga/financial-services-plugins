# 论点追踪器

为投资组合持仓和观察名单维护和更新投资论点。

## 工作流程

### 步骤1：定义或加载论点

如果创建新论点：
- **公司**：名称和股票代码
- **头寸**：多头或空头
- **论点陈述**：1-2句话核心论点（例如，"多头ACME——定价权+运营杠杆驱动的利润率扩张，软件组合转向"）
- **关键支柱**：3-5个支持论点
- **关键风险**：3-5个会使论点失效的风险
- **催化剂**：可能证明/反驳论点即将发生的事件（业绩、产品发布、监管决策）
- **目标价/估值**：论点实现时值多少钱
- **止损触发**：什么会让你退出

如果更新现有论点，向用户询问新数据点或发展。

### 步骤2：更新日志

对于每个新数据点或发展：

- **日期**：何时发生
- **数据点**：什么变化了（业绩超预期、管理层离职、竞争对手动向等）
- **论点影响**：这是否强化、弱化或中和了特定支柱？
- **行动**：无变化/增加头寸/削减/退出
- **更新后的信念度**：高/中/低

### 步骤3：论点记分卡

维护持续记分卡：

| 支柱 | 原预期 | 当前状态 | 趋势 |
|------|--------|---------|------|
| 收入增长>20% | 符合预期 | Q3为22% | 稳定 |
| 利润率扩张 | 落后 | 利润率同比持平 | 令人担忧 |
| 新产品发布 | 待定 | 推迟至Q2 | 观察 |

### 步骤4：催化剂日历

跟踪即将到来的催化剂：

| 日期 | 事件 | 预期影响 | 备注 |
|------|------|---------|------|
| | | | |

### 步骤5：输出

适用于以下场景的论点摘要：
- 晨会讨论
- 投资组合审查
- 风险委员会演示

格式：简洁的markdown或Word文档，包含记分卡、最新更新和当前信念度。

## 重要说明

- 论点应该是可证伪的——如果没有什么能反驳它，那就不是论点
- 与证实证据一样严格地跟踪反驳证据
- 至少每季度审查论点，即使没有发生戏剧性事件
- 如果用户管理多个头寸，提供全面的投资组合论点审查
- 以结构化格式存储论点数据，以便跨会话引用
