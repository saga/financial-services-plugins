# 业绩更新详细工作流程

本文档提供业绩更新流程每个阶段的详细分步说明。

## ⚠️⚠️⚠️ 关键警告：始终使用最新业绩数据 ⚠️⚠️⚠️

**首先阅读此内容：**

训练数据已过时。主动搜索并获取最新业绩材料。使用过时业绩数据是业绩分析中的头号错误。

**开始之前：**
1. **检查今天日期** - 写下当前日期
2. **搜索最新** - 使用网络搜索查找最新业绩
3. **验证日期** - 确认业绩发布在最近3个月内
4. **如超过3个月** - 获取错误季度，重新搜索

## 第一阶段：业绩数据收集（30-60分钟）

### 步骤1：识别最新业绩期

**关键**：始终搜索最新业绩 - 不要依赖知识截止日期。
**关键**：永远不要使用训练中的业绩数据 - 已过时。

**步骤1a：搜索最新业绩新闻稿**

**🚨 主动搜索 - 训练数据已过时。🚨**

**必需步骤1：检查今天日期**
- **明确写下今天日期**：[月] [日]，[年]
- **用于验证**找到的任何业绩在3个月内
- **示例**："今天是2024年10月29日"

**必需步骤2：搜索"最新业绩"**
- **使用网络搜索**，查询如：
  - `[公司名称]最新业绩结果`
  - `[公司名称]最近季度业绩`
  - `[股票代码]业绩最新季度`
- **或搜索公司投资者关系网站**：
  - 访问`investor.[公司].com`或`[公司].com/investors`
  - 导航至"新闻稿"、"新闻"或"业绩"部分
  - **按日期排序找到最新发布**
  - 查找关键词："业绩"、"结果"、"财务结果"、"季度结果"

**必需步骤3：验证发布日期**
- **查看找到的业绩新闻稿日期**
- **计算**：此日期是否在最近3个月内？
- **如是** → 继续下一步
- **如否（超过3个月）** → 🚨 错误季度 - 重新搜索更新内容

**❌ 常见错误避免：**
- ❌ 使用训练中的业绩数据而不搜索
- ❌ 基于预期假设"Q3 2024"是最新
- ❌ 抓取第一个找到的业绩发布而不检查日期
- ❌ 不将发布日期与今天日期比较
- ❌ 发布已4个月以上仍继续进行

**✅ 正确方法：**
- ✅ 首先检查今天日期
- ✅ 明确搜索"最新"或"最近"
- ✅ 阅读材料上的实际发布日期
- ✅ 确认发布日期在最近3个月内
- ✅ 如不确定，用不同关键词重新搜索

**必需步骤4：识别季度**
- **阅读标题/头条**识别季度（Q1、Q2、Q3、Q4或财季）
- **阅读文档本身的发布日期**
- **验证季度名称和日期都是最近的**

**步骤1b：了解公司财年日历**

从搜索识别最新季度后，了解公司财年以正确解读：

**常见财年模式：**
- **日历年（CY）**：Q1=1-3月，Q2=4-6月，Q3=7-9月，Q4=10-12月
- **耐克财年**：Q1=6-8月，Q2=9-11月，Q3=12-2月，Q4=3-5月（5月财年末）
- **苹果财年**：Q1=10-12月，Q2=1-3月，Q3=4-6月，Q4=7-9月（9月财年末）
- **沃尔玛财年**：Q1=2-4月，Q2=5-7月，Q3=8-10月，Q4=11-1月（1月财年末）

许多公司在业绩发布标题中说明财年。如需要可搜索`[公司]财年日历`。

**步骤1c：必需验证 - 验证获取最新数据**

🛑 **停止 - 在验证以下所有内容前不要继续：**

- [ ] ✅ **今天日期已写下**：[月] [日]，[年]
- [ ] ✅ **已主动搜索**使用"最新业绩"或"最近业绩"
- [ ] ✅ **找到业绩发布日期**：[月] [日]，[年]
- [ ] ✅ **验证发布在最近3个月内**（计算！）
- [ ] ✅ **未假设**季度仅基于今天日期
- [ ] ✅ **能看到实际新闻稿**确认季度/期间
- [ ] ✅ **已打开并阅读**实际业绩材料（非仅假设存在）

**🚨 危险信号 - 如以下任一为真，获取错误季度：**
- 🚨 发布日期超过90天
- 🚨 依赖预期而非搜索找到的内容
- 🚨 未实际看到确认此季度存在的新闻稿或文件
- 🚨 使用训练数据而不搜索
- 🚨 无法说明确切发布日期
- 🚨 找到的发布日期来自2023年或更早（当今天是2024年+）

**如有任何危险信号**：停止并重新搜索。不要使用过时数据继续。

### 步骤2：收集业绩材料

搜索并确认最新季度后，收集以下内容：

**⚠️ 重要：搜索并访问实际文档 - 不要依赖训练数据。**

**主要材料（必需）：**
- **业绩新闻稿** - 通常在公司投资者关系网站"新闻稿"或"新闻"下
  - 导航至IR网站并找到实际新闻稿
  - 搜索模式："[公司名称]最新业绩"、"[公司名称]Q[X][年]业绩结果"
  - 查找PDF或HTML版本
  - **验证日期与步骤1找到的匹配**（应在最近1-3个月内）
  - **阅读实际文档**确认季度并获取报告数字

- **10-Q或10-K文件** - 在SEC EDGAR（sec.gov/edgar/searchedgar/companysearch.html）
  - 按股票代码搜索
  - Q1-3季度：查找最新10-Q
  - Q4季度：查找10-K（年度报告）
  - 注：可能在业绩发布后1-5天提交
  - 直接链接格式：`https://www.sec.gov/cgi-bin/viewer?accession=[存档号]`

- **业绩电话会记录** - 🚨 **验证记录上的日期** 🚨
  - **搜索**："[公司]最新业绩电话会记录"或"[公司]Q[X][年]业绩电话会记录"
  - **来源**：
    - 公司IR网站（部分直接发布记录）
    - Seeking Alpha：搜索"[公司][最新季度]业绩电话会记录"
    - AlphaStreet、Motley Fool（替代来源）
  - **关键日期检查**：
    - ✅ **使用任何记录前，验证记录本身的日期**
    - ✅ **记录日期必须与步骤1的业绩发布日期匹配**
    - ✅ **如记录显示"Q2 2023"但发布是"Q3 2024"，获取错误记录**
    - 🚨 **常见错误**：抓取旧记录而不检查日期
  - 如记录尚不可用，收听网络广播回放或注明等待记录

**补充材料（如可用）：**
- **投资者演示/幻灯片** - 通常在IR网站与新闻稿一起发布
  - 通常标题为"Q[X][年]业绩演示"或"投资者演示"
  - PDF格式，含管理层在业绩电话会上展示的幻灯片

- **补充数据文件** - 部分公司提供详细指标的Excel文件
  - 查找"补充财务信息"或"投资者数据表"

**参考材料（用于比较）：**
- **上一季度结果** - 用于环比比较
  - 来自上一季度业绩新闻稿（90天前）

- **去年同季度** - 用于同比比较
  - 来自去年同季度（4个季度前）

- **之前预期** - 如此前覆盖此公司
  - 来自上次业绩更新或首次覆盖报告
  - 检查对此季度指标的预期

- **共识预期** - 来自Bloomberg、FactSet、Refinitiv或Yahoo Finance
  - 关键：使用业绩发布前的预期
  - 查找"截至[业绩前日期]"确保公告前共识
  - 超预期/不及预期分析需要

**🛑 继续步骤3前必需验证：**

**日期 - 验证所有日期匹配：**
- [ ] ✅ **今天日期已写下**：_______________
- [ ] ✅ **业绩发布日期**：_______________（必须在今天3个月内）
- [ ] ✅ **业绩电话会记录日期**：_______________（必须与发布日期匹配±1天）
- [ ] ✅ **10-Q/10-K提交日期**：_______________（必须与发布同季度）
- [ ] ✅ **所有材料显示相同季度**（如都显示"Q3 2024"，非混合季度）

**搜索与访问 - 验证主动搜索完成：**
- [ ] ✅ **已搜索**"最新业绩"（非基于当前日期假设）
- [ ] ✅ **已访问**实际业绩新闻稿并阅读
- [ ] ✅ **已打开**实际业绩电话会记录并验证日期
- [ ] ✅ **已确认**这是最新季度（通过检查日期）
- [ ] ✅ 有完整财务结果（收入、EPS、利润率等）来自实际发布
- [ ] ✅ 有业绩发布前共识预期及来源日期

**🚨 危险信号 - 如以下任一为真则停止：**
- 🚨 未实际搜索或访问业绩材料
- 🚨 使用记忆或训练数据而非当前文档
- 🚨 业绩发布日期超过90天
- 🚨 无法说明业绩发布的确切日期
- 🚨 记录日期与发布日期不匹配
- 🚨 材料显示不同季度（如发布显示Q3但记录显示Q2）
- 🚨 抓取第一个结果而不验证日期

### 步骤4：从电话会识别关键主题

收听或阅读业绩电话会记录并记录：
- 管理层语气（自信、谨慎、防御？）
- 强调的关键话题（产品发布、地区趋势、竞争）
- 分析师问题（投资者关注什么？）
- 提供的指引（上调、下调、维持、首次提供？）
- 任何意外或意外评论

## 第二阶段：分析（2-3小时）

### 步骤5：超预期/不及预期分析

对每个超预期或不及预期的关键指标，解释：

**如超预期：**
- 什么推动了超预期表现？
- 是一次性还是可持续的？
- 管理层是否指引未来更高？
- 这如何影响我们的论点？

**如不及预期：**
- 哪里出了问题？
- 是公司特有还是行业普遍？
- 管理层是否在采取纠正措施？
- 这如何影响我们的论点？

**示例格式：**
```
■ **收入超预期3%，由DTC强劲表现驱动**

收入$13.5B超出我们预期$13.1B $400M（3%），超出共识$13.2B $300M（2%）。
超预期主要由直接面向消费者渠道推动，同比增长18%（vs.我们12%预期），
抵消了弱于预期的批发（-5% vs.持平预期）。管理层指出强劲的数字需求
和成功的产品发布（Pegasus 40跑鞋、新Jordan配色）为关键驱动因素。
DTC现占总收入42%，vs.一年前38%，展示成功的渠道转型策略。
```

### 步骤6：分部/地区/产品分析

按以下分析表现：
- 业务分部（如多分部公司）
- 地区（北美、欧洲、中国等）
- 产品类别
- 渠道（零售、批发、电商）

识别：
- 什么超预期？
- 什么不及预期？
- vs.之前季度的趋势
- 管理层对各地区展望的评论

### 步骤7：利润率分析

分析盈利能力：
- 毛利率：上升或下降？为什么？
- 营业利润率：上升或下降？为什么？
- 关键驱动因素（定价、组合、成本、杠杆）
- 未来展望

### 步骤8：指引分析

如公司提供指引：
- 比较新指引与之前指引
- 比较内部预期和市场预期
- 评估可信度（公司有保守记录吗？超预期记录？）
- 识别指引背后的关键假设

如公司未提供指引：
- 明确注明
- 基于结果和评论提供独立展望

### 步骤9：更新财务模型

更新以下预期：
- 当年（剩余季度）
- 明年
- 可能后年

**清晰展示：**
```
更新预期：
─────────────────────────────────────────────────
                        旧预期    新预期    变化      原因
FY2024E收入            $XX.XB    $XX.XB    +X.X%    [简要原因]
FY2024E EBITDA         $X.XB     $X.XB     +X.X%    [简要原因]
FY2024E EPS            $X.XX     $X.XX     +X.X%    [简要原因]

FY2025E收入            $XX.XB    $XX.XB    +X.X%    [简要原因]
FY2025E EBITDA         $X.XB     $X.XB     +X.X%    [简要原因]
FY2025E EPS            $X.XX     $X.XX     +X.X%    [简要原因]
```

### 步骤10：更新估值与目标价

基于更新预期：
- 重新计算DCF（使用更新现金流）
- 更新可比公司倍数（如同行组已发布业绩）
- 确定新公允价值
- 决定目标价是否变化

**目标价决策：**
- 如预期显著变化（>5%）→ 通常改变目标价
- 如预期边际变化（<5%）→ 可能维持目标价
- 如论点强化/弱化 → 可能即使无预期变化也改变

### 步骤11：评估评级影响

决定是否改变评级：
- 如结果显著好于预期 + 指引上调 → 考虑上调
- 如结果显著更差 + 指引下调 → 考虑下调
- 如符合预期或混合 → 通常维持评级

**考虑：**
- 股价反应（上涨/下跌/持平？）
- 估值（相对于新预期昂贵/便宜？）
- 风险/回报（不对称性改变？）

## 第三阶段：图表生成（1-2小时）

### 步骤12：生成8-12个图表

创建专注于季度趋势和新内容的图表。

**必需图表（共8-12个）：**

1. **季度收入进展**（柱状图）
   - 最近8-12个季度
   - 显示每季度超预期/不及预期vs.预期
   - 突出当前季度

2. **季度EPS进展**（柱状图）
   - 最近8-12个季度
   - 显示超预期/不及预期vs.预期
   - 调整后和GAAP

3. **季度利润率趋势**（折线图）
   - 毛利率、EBIT利润率、净利润率
   - 最近8-12个季度
   - 显示轨迹

4. **按分部/地区收入**（堆叠柱状图或表格）
   - 当前季度vs.同比
   - 各分部增长率

5. **关键运营指标**（多折线图）
   - 客户数、ARPU、销售单位等（相关的任何指标）
   - 最近8-12个季度

6. **超预期/不及预期摘要**（瀑布图或表格）
   - 显示超预期/不及预期组成部分
   - 与预期差异的驱动因素

7. **预期修正图**（前后比较）
   - 旧FY预期vs.新FY预期
   - 显示变化的柱状图

8. **估值图**（P/E或EV/EBITDA倍数）
   - 历史倍数区间
   - 当前倍数
   - 公允价值倍数

**可选图表（如空间允许）：**
- 同行比较（如同行已发布业绩）
- 指引vs.市场比较
- 现金流指标
- 资产负债表亮点（如显著）

**图表样式指南：**
- 专注于趋势（季度进展）
- 突出变化（超预期/不及预期、预期修正）
- 保持简单清晰（这是快速周转报告）

## 第四阶段：报告创建（2-3小时）

### 步骤13：创建DOCX报告

使用DOCX技能创建8-12页报告。

参见[report-structure.md](report-structure.md)获取完整逐页模板和格式要求。

**关键步骤：**
1. 创建第1页含业绩摘要和快速要点
2. 添加详细结果分析（第2-3页）
3. 包含关键指标和指引（第4-5页）
4. 更新投资论点（第6-7页）
5. 提供估值和预期（第8-10页）
6. 如需要添加附录（第11-12页）
7. 全文嵌入所有8-12个图表
8. 添加1-3个摘要表
9. 包含完整来源部分及可点击超链接

### 步骤14：可选 - 更新XLS模型

如此公司存在完整财务模型（来自首次覆盖），用以下更新：
- 实际Q[X]结果
- 未来季度修正预期
- 更新估值

**注**：业绩更新时，完整XLS文件是可选的（非首次覆盖报告那样必需）。DOCX报告是主要交付物。

如创建XLS，包含：
- 季度模型标签页
- 更新年度预测
- 修正DCF
- 更新可比分析

## 第五阶段：质量检查与交付（30分钟）

### 步骤15：质量检查清单

发布前验证：

**内容：**
- [ ] 超预期/不及预期清晰陈述并量化
- [ ] 关键驱动因素已解释（非仅"表现强劲"）
- [ ] 提供更新预期（显示旧vs.新）
- [ ] 目标价已更新或明确维持
- [ ] 评级已确认或更改并附理由
- [ ] 指引已分析（如提供）
- [ ] 论点影响已评估

**格式：**
- [ ] 第1页有摘要框和关键要点
- [ ] 所有表格有来源行
- [ ] 所有图表已编号和说明
- [ ] 预期表显示旧vs.新
- [ ] 8-12个图表全文嵌入
- [ ] 报告为8-12页（不太长，不太短）

**准确性：**
