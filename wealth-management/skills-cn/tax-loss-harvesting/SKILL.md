# 税损收割

识别应税账户中的税损收割机会。

## 工作流

### 步骤1：识别候选

扫描应税账户中未实现亏损的持仓：

| 证券 | 资产类别 | 成本基础 | 当前价值 | 未实现亏损 | 持有期间 | 亏损% |
|------|----------|----------|----------|------------|---------|-------|
| | | | | | ST/LT | |

**优先排序：**
1. 最大绝对亏损（最大税务利益）
2. 优先短期亏损（抵消按普通收入税率征税的短期收益）
3. 亏损百分比最大的持仓（不太可能快速恢复）

### 步骤2：盈亏预算

计算客户的税务状况：

| 类别 | 金额 |
|------|------|
| 本年已实现短期收益 | |
| 本年已实现长期收益 | |
| 本年已实现亏损 | |
| 净盈亏位置 | |
| 以前年度结转亏损 | |
| **目标收割金额** | |

**节税估算：**
- 短期亏损 × 边际普通收入税率
- 长期亏损 × 资本利得税率
- 最多$3,000净亏损扣除普通收入
- 超出部分结转

### 步骤3：替换证券

对于每个收割候选，建议保持类似市场敞口的替换：
- 保持类似市场敞口（相同资产类别、行业、地区）
- 不是"实质相同"（洗售规则）
- 具有类似风险/回报特征

| 卖出 | 替换为 | 理由 | 跟踪误差风险 |
|------|--------|------|-------------|
| SPDR标普500 (SPY) | iShares核心标普500 (IVV) | 相同指数，不同基金家族 | 最小 |
| Vanguard总国际 (VXUS) | iShares MSCI ACWI ex-US (ACWX) | 类似敞口，不同指数 | 低 |
| 个股ABC | 行业ETF (XLK) | 更广泛敞口，无洗售风险 | 中等 |

### 步骤4：洗售检查

执行前验证无洗售：

- 检查家庭所有账户（应税、IRA、Roth、配偶账户）
- 30天回溯：我们过去30天购买了实质相同的证券吗？
- 30天前瞻：30天内阻止回购相同证券
- 检查可能导致洗售的股息再投资计划（DRIP）
- 记录每笔交易的洗售窗口

| 卖出证券 | 洗售窗口开始 | 窗口结束 | DRIP活跃？ | 风险 |

| 交易# | 账户 | 操作 | 证券 | 股数 | 预估收益 | 预估亏损 | 替换 | 备注 |

| | | 卖出 | | | | | | |
| | | 买入 | | | | | | |

**摘要：**
- 预估总收割亏损：$
- 预估节税：$（边际税率%）
- 投资组合净影响：最小（替换证券保持敞口）
- 洗售窗口管理：[日期]

### 步骤6：收割后跟踪

30天后，可选：
- 换回原证券（如偏好）
- 保留替换证券（如无理由换回）
- 更新成本基础记录
- 记录用于税务报告

### 步骤7：输出

- 收割机会清单（Excel）
- 交易执行表
- 洗售跟踪日历
- 节税估算摘要
- 替换证券理由

## 重要说明

- 洗售规则严格——违规将使亏损无效并调整成本基础
- 实质相同意味着相同证券，而不是相同资产类别——跟踪不同指数的ETF通常可以
- 始终协调包括退休账户在内的所有家庭账户
- 考虑长期成本基础逐步降低——收割重置成本基础，意味着以后更多收益
- 年底是收割旺季，但全年都有机会
- 12月共同基金资本利得分配可能产生额外收割紧迫性
- 为税务报告和合规记录所有内容
- 不是所有亏损都值得收割——交易成本和跟踪误差有真实成本
