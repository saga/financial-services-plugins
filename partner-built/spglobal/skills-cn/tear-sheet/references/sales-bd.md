# 销售/业务拓展简报

## 目的
为销售或BD团队准备客户会议的简报文档。强调理解潜在客户的业务、领导层公开优先事项，以及识别将您的产品/服务与其需求联系的角度。财务简化为规模和轨迹。

这是最叙事驱动的简报类型。读起来像简报备忘录，而非数据表。

**默认页数：** 1-2页。无严格惯例——优先考虑可读性而非压缩。

## 查询计划

从这些查询开始。如结果不完整则跟进。

**查询1 — 公司概况 + 财务：**
"[公司] 业务描述 概况 产品 服务 总部 员工 行业 产业 收入 毛利率 最近2个财年"
→ 标题、公司概述、财务快照
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`
→ **立即写入** 原始财务数据到 `/tmp/tear-sheet/financials.csv`

**查询2 — 战略 + 业绩：**
"[公司] 最近业绩会 战略优先事项 CEO评论 指引 关键举措"
→ 战略优先事项
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询3 — 关系 + 新闻：**
"[公司] 主要客户 供应商 合作伙伴 竞争对手 业务关系 近期收购 合作伙伴关系 公告"
→ 关键关系、近期新闻
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

三个查询通常足够。对于知名公司，这些返回丰富结果。对于较小或私有公司，查询2和3可能稀疏——这没关系。定性章节仍能提供价值，因为Claude将可用内容综合为有用的框架。

## 章节

按优先级顺序列出。如空间受限从底部删减。

### 1. 公司标题
精简——仅识别，无估值指标。

| 字段 | 备注 |
|---|---|
| 公司名称 | 大号、粗体 |
| 股票代码与交易所 | 如上市；私有则省略 |
| 总部 | 城市、州/国家 |
| 行业/产业 | |
| 员工 | |
| 年收入（最新） | 舍入到最近的$B或$M |
| 市值 | 如上市；私有则省略 |
| FY收入指引 | 如业绩数据可用；例如，"6–8%有机增长" |
| FY EPS或EPS指引 | 如上市公司；从`earnings.txt`提取 |
| 关键财务指标 | 调整EPS、EBITDA利润率或类似——一个标题数字 |

前瞻指引和EPS数据来自查询2（业绩）——填充标题时从`earnings.txt`获取，而非仅`company-profile.txt`。

顶部一个水平带。无企业价值，无Beta——此受众不需要。

### 2. 公司概述
最重要的章节。销售代表阅读后应能带着对潜在客户的扎实理解走进会议。

**不要粘贴CIQ公司摘要。** 它读起来像SEC文件，会失去非金融受众。用3-5句通俗语言重写：
- 公司做什么（无投资者术语——说"销售"而非"变现"，说"年收入"而非"顶线CAGR"）
- 谁是他们的客户
- 他们如何赚钱
- 他们在哪里运营
- 什么使他们独特

用散文段落撰写。非金融人士应理解每句话。

### 3. 战略优先事项与管理层评论
领导层现在关注什么？如果您知道CEO在谈论什么，您可以调整您的推介。

**这不是业绩回顾。** 不要重复股票研究简报的相同要点。提取3-5个*战略主题*，带主题标签和支持数据点：

- **AI/数据变现：** 管理层正激进投资AI能力，在FY2024分配$X用于研发并推出三个新的AI驱动产品。
- **利润率扩张：** CEO将运营效率列为首要优先事项，目标通过自动化实现200个基点的运营利润率改善。
- **国际增长：** 在EMEA开设新办事处；欧洲现占收入25%，从18%上升。

每个要点应回答："这家公司现在关心什么，有什么证据支持？"销售代表阅读后应立即看到连接其产品的角度。

如业绩会数据不可用（私有公司常见），替换为"近期发展"，从工具返回的任何新闻或公告中提取。

### 4. 财务快照
简化。单个紧凑表格——仅规模、方向和一个盈利能力信号。销售代表需要知道三件事：这家公司多大、是否在增长、是否健康。

| 指标 | FY[上年] | FY[最新] | 同比变化 |
|---|---|---|---|
| 收入 | | | +X% |
| 毛利率% | | | +X.Xpp |
| 员工 | | | |

最多三行。无EBITDA、无EV倍数、无资产负债表、无净利润。毛利率（非EBITDA、非净利润）是销售受众正确的单一盈利能力指标——它直观（"扣除直接成本后保留多少"）并表明业务健康，无需金融素养。如代表需要更深入的财务，他们可以请求股票研究版本。

对于私有公司，此表格可能仅有收入和员工——两个显示规模和增长的数据点对销售会议足够。

### 5. 关键关系与生态
帮助代表理解潜在客户的世界并识别角度。

四个标签列表，各3-5个名称，带一行描述符：

**客户：** 他们卖给谁。帮助理解他们的市场进入和共享目标客户。
示例："JPMorgan Chase — 客户（企业分析平台部署）"

**供应商/技术：** 他们买什么。识别替代或整合机会。
示例："AWS — 供应商（主要云基础设施）；Snowflake — 供应商（数据仓库）"

**合作伙伴：** 他们与谁合作。联合销售角度、共同联系。
示例："Deloitte — 合作伙伴（企业客户实施）"

**竞争对手：** 他们与谁竞争。定位背景。
示例："Bloomberg — 竞争对手（金融数据终端）"

每个条目需要括号描述符，以便代表理解关系意味着什么，而非仅是谁。

如关系数据不可用，注明"关系数据不可用"并继续。

### 6. 近期新闻与发展
3-5个要点，每个一句话，最近的在前。如可用包含日期。**以最新业绩报告开头**——这是销售会议最重要的近期事件，为其他一切设定背景。然后：M&A、产品发布、领导层变动、重大胜利。

**在章节末尾包含即将到来的催化剂**，带"**即将到来：**"标签。示例：即将到来的业绩日期、待定资产剥离、产品发布、投资者日、监管里程碑。这些为销售外联创造自然的参与窗口，通常是联系的最佳理由（"我注意到您Q2有投资者日即将到来..."）。从业绩会评论和M&A数据（待定交易）中提取。

### 7. 对话开场（最先删减）
2-3个从上述数据综合的建议谈论点。**每个开场必须引用第3节的具体战略优先事项**——通用问题无用。框架化为问题。

**为每个开场标记建议的角色**——例如，（CTO）、（CFO）、（CDO）、（数据负责人）、（工程VP）、（采购负责人）。与CTO会面的代表需要与CFO会面不同的开场。如用户指定与谁会面，仅为该角色生成开场。

示例：
- *（CTO/CDO）* "您在上次业绩会上提到[第3节的具体AI举措]——这如何影响您的[相关需求]？"
- *（CFO/COO）* "鉴于您的[第3节利润率扩张目标]，您是否在评估可能加速该时间线的工具？"
- *（战略VP/企业发展）* "我注意到[公司]最近收购了[标的]——这是否改变了您对[能力]的方法？"

如用户描述他们销售什么产品/服务，将这些定制为将潜在客户的优先事项与该产品连接。如未描述，保持基于潜在客户的战略主题但在解决方案框架上通用。

**验证测试：** 最终确定每个对话开场前，验证它包含至少一个从简报中提取的具体数字、日期、产品名称或举措名称。可适用于行业中任何公司的问题不是好的对话开场。"您如何考虑AI？"是通用的。"您的CEO提到自2018年以来AI投资$1B和iLEVEL自动摄取功能20%采用率——这如何改变分析师的工作流程？"是具体的并展示准备。

此章节是Claude的综合，而非工具的原始数据。清楚标注。

## 格式说明
- **这应该是最温暖、最可读的简报。** 内容严谨，但格式应感觉像简报备忘录，而非数据终端。
- 正文：使用完整9磅（size: 18）。无压缩——可读性优于密度。
- 行间距：正文段落1.15倍（比默认略宽松）。此模板是空白为特色而非浪费的模板。
- 公司概述和战略优先事项应占页面约40-50%。
- 财务快照：视觉紧凑、次要突出。保持小而干净。
- 关键关系：考虑两列布局（左侧客户+供应商，右侧合作伙伴+竞争对手）以节省垂直空间同时保持描述符可见。
- 对话开场：使用缩进块式项目符号。这些是文档的回报——它们应在视觉上突出。
