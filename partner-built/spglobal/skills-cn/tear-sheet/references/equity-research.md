# 股票研究简报

## 目的
为买方或卖方分析师评估投资准备的密集快照。强调估值、前瞻预期、财务轨迹和分析师共识。所有内容支持或质疑投资论点。

**默认页数：** 1页。股票研究简报传统上是单页——密度是关键。如用户要求更多空间，扩展到2页。

## 查询计划

从这些查询开始。如结果不完整，分解为更窄的跟进。

**查询1 — 概况 + 市场数据：**
"[公司] 公司概况 行业 产业 市值 企业价值 股价 52周最高最低 流通股 Beta"
→ 标题、业务描述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询2 — 历史财务：**
"[公司] 年度利润表 收入 毛利润 EBITDA 净利润 EPS 自由现金流 总债务 现金及等价物 最近4个财年"
→ 财务摘要（拉取4年；显示3年；最早年份仅用于计算同比增速）
→ **立即写入** 原始数值到 `/tmp/tear-sheet/financials.csv`

**查询2b — 分部（如可用）：**
"[公司] 按分部收入 业务单元 最近2个财年"
→ 收入与分部分解（需要2年计算同比增速）。如无分部数据，跳过此章节——不要留空白表格。
→ **立即写入** `/tmp/tear-sheet/segments.csv`（如无分部数据返回则跳过）

**查询3 — 估值 + 共识：**
"[公司] P/E EV/EBITDA EV/收入 估值倍数 共识 收入 EPS预期 分析师建议 目标价"
→ 估值快照、共识预期
→ **立即写入** 倍数到 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 预期到 `/tmp/tear-sheet/consensus.csv`

**查询4 — 业绩：**
"[公司] 最近业绩会 关键要点 指引"
→ 业绩亮点
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询5 — 股价表现：**
"[公司] 股价回报 1个月 3个月 6个月 1年 年初至今 表现"
→ 股价表现（无中间文件——数据直接进入文档）

**查询6（如用户提供可比公司）：**
"[可比公司] 市值 EV/收入 EV/EBITDA 收入增长" —— 每个可比公司重复
→ 估值快照的同行背景
→ **立即写入** `/tmp/tear-sheet/peer-comps.csv`

## 章节

按优先级顺序列出。如限制在一页，从底部删减。

### 1. 公司标题
紧凑的键值块，按全局样式配置渲染为两列无边框表格。

左列：股票代码（交易所）、行业/产业、总部
右列：股价、52周区间、市值、EV、流通股、Beta

### 2. 业务描述
2-3句紧凑文字。**用您自己的话为分析师受众重写**——不要逐字粘贴CIQ公司摘要。CIQ摘要是输入；输出应简洁且以论点为导向。

对于知名、广泛覆盖的公司，假设分析师已了解基本业务。以*变化*开头——战略转型、产品组合重塑、新增长向量——而非维基百科式概述。例如，对于财富500强公司，不要用一句话讲"向机构提供金融数据"。相反："正将其产品组合重塑向高利润数据和分析业务，待定的Mobility分拆和通过Kensho子公司的激进AI投资。"

对于较不知名公司，简要的"他们做什么"句子在转向论点相关动态前是合适的。

### 3. 估值快照
股票简报的核心。

| 指标 | 滚动 | 前瞻（NTM） |
|---|---|---|
| P/E | | |
| EV/EBITDA | | |
| EV/收入 | | |
| P/FCF | | |
| 股息收益率 | | |

**前瞻倍数在工具可用时是强制性的。** 当存在前瞻数据时仅显示滚动倍数是重大缺口——分析师基于前瞻收益估值公司。如前瞻倍数不可用，仅显示滚动并注明"前瞻预期不可用。"

如用户提供可比公司，为每个可比公司添加列（或如有3+可比公司则添加"同行中位数"列）。如无用户可比公司，且工具返回同行数据，包含同行中位数列。如都没有，仅显示公司倍数。

### 4. 共识预期
华尔街预期——此简报类型的关键差异化因素。

**数据可用性说明：** 共识数据深度因公司和覆盖而异。包含工具返回的任何内容——分析师数量、目标价、买入/持有/卖出分布——但不要编造或估计任何共识数值。对于覆盖较少的公司，此章节可能仅包含收入和EPS预期，这仍有价值。

| 指标 | FY[年]预期 | FY[年+1]预期 |
|---|---|---|
| 收入 | | |
| EPS（调整后） | | |
| EBITDA | | |

使用调整后/标准化EPS（如可用）——GAAP EPS可能被一次性项目扭曲，对前瞻预期用处较小。

在主要预期表格下方，包含紧凑的分析师共识块（如工具数据可用）：

| 分析师共识 | |
|---|---|
| 平均目标价 | $XXX |
| 预期数量 | XX |
| 买入/持有/卖出 | XX / XX / XX |

如目标价或建议数据不可用，省略分析师共识块而非显示空行。不要编造这些数值。

如共识数据完全不可用，跳过整个章节并注明"共识预期不可用。"不要估计。

使用实际财年标签的密集单表。

| 指标（$M） | FY20XX | FY20XX | FY20XX |
|---|---|---|---|
| 收入 | | | |
| 收入增长率% | | | |
| 毛利率% | | | |
| EBITDA | | | |
| EBITDA利润率% | | | |
| 净利润 | | | |
| EPS（稀释） | | | |
| 自由现金流 | | | |
| 净债务 | | | |

从原始数据计算衍生指标（增长率%、利润率）而非单独查询。

**资本结构（财务摘要下方的独立紧凑子表）：**
不要将这些行追加到3年财务摘要表——先前年份列的空白单元格看起来像缺失数据。相反，直接在下方渲染为单独的2列子表（指标 | 数值），仅显示最近财年：

| 指标 | FY[最新] |
|---|---|
| 总债务 | $XXM |
| 现金及等价物 | $XXM |
| 净债务 | $XXM |

这匹配企业发展和投行/M&A简报中使用的资本结构格式。保持紧凑——财务摘要与此子表之间无额外间距。

### 6. 收入与分部分解
如工具返回分部数据，包含紧凑的分部收入表格。这对股票研究是重要背景——分析师需要看到哪些业务线在驱动增长。

| 分部 | 收入（$M） | 占比 | 同比增长 |
|---|---|---|---|
| [分部A] | | | |
| [分部B] | | | |

为单页保持紧凑：仅表格，无定性段落。如无分部数据，完全跳过此章节——不要包含空白或占位表格。

### 7. 近期业绩亮点
包含季度和日期（例如，"Q3 FY2025 — 2025年10月"）。3-4个要点，带**投资视角**——此受众需要分部级细节，而非战略主题：

**如有指引则以此开头。** 如管理层发布前瞻指引，使其成为第一个要点，带粗体**指引：**前缀（例如，"**指引：** FY2026 EPS $19.40–$19.65，意味着9–10%增长；有机收入增长6–8%。"）。指引是分析师最可操作的前瞻数据点——不要将其埋在运营亮点中。

**超预期/不及预期背景（数据允许时）：** 如报告期有共识预期数据，将标题结果要点框架化为超预期/不及预期："收入$X超预期/不及预期共识$Y Z%。"如工具无法获取报告期的共识数据，仅用绝对值和增长率描述结果——不要编造共识数值。

然后：
- 分部级表现：哪些业务线加速或减速
- 利润率轨迹：任何关于成本结构或盈利趋势的评论

归因于管理层："管理层强调…"或"CFO指出…"而非声明性主张。此章节应感觉像业绩回顾，而非新闻稿摘要。

### 8. 关键运营指标
2-3个行业相关KPI（如工具数据可用）（订阅用户数、同店销售、NIM等）。如无行业特定指标，从已检索的财务数据计算ROE和ROIC。可选——如空间紧张则删减。

### 9. 股价表现（最先删减）

| 期间 | 回报 |
|---|---|
| 1个月 | |
| 3个月 | |
| 年初至今 | |
| 1年 | |

## 格式说明
- **这是最密集的简报类型。** 应感觉明显比其他模板更压缩。
- 财务摘要表：8磅文本（size: 16），行高240 DXA。所有模板中最紧凑的间距。
- 估值快照和共识预期：8.5磅文本，占据视觉中心——分析师首先扫描这些。
- 正文：8.5磅（size: 17）——小于全局9磅默认值。每半磅在单页上都很重要。
- 章节间距：最小化。章节标题前6磅（非全局12磅），分隔线后2磅。
- 粗体任何显示显著拐点的指标（增长转正、利润率扩张/压缩）。
- 在每个环节优先考虑信息而非空白。
