# 投资银行/M&A简报

## 目的
用于交易背景的公司简介——潜在标的、收购方或推介材料收录。强调战略定位、分部细节、业务关系和M&A活动。估值以交易倍数框架化，而非投资论点。

**默认页数：** 1-2页。投行公司简介经常溢出到第二页，特别是当分部和M&A数据丰富时。

## 查询计划

从这些查询开始。如结果不完整，分解并跟进。

**查询1 — 概况 + 识别：**
"[公司] 业务描述 总部 成立 员工 行业 产业 所有权结构 主要股东"
→ 标题、业务概述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询2 — 分部 + 财务：**
"[公司] 按分部收入 业务单元 最近2个财年 AND 年度利润表 收入 毛利润 EBITDA 营业利润 净利润 资本支出 自由现金流 总债务 现金及等价物 最近4个财年"
→ 分部分解（需要2年计算同比增速），财务摘要（拉取4年；显示3年；最早年份仅用于计算同比增速）
→ **立即写入** 原始财务数据到 `/tmp/tear-sheet/financials.csv`
→ **立即写入** 分部数据到 `/tmp/tear-sheet/segments.csv`（如无分部数据返回则跳过）

**查询3 — 估值 + 可比公司：**
如用户提供特定可比公司，查询每个：
"[可比公司] 企业价值 EV/收入 EV/EBITDA 收入增长 EBITDA利润率"
否则：
使用竞争对手工具识别3-5家上市同行，然后拉取每家的EV/收入（NTM）、EV/EBITDA（NTM）、收入增长率%和EBITDA利润率%。
同时："[公司] 企业价值 市值 EV/收入 EV/EBITDA 估值倍数"
→ 标题（EV/市值），交易可比
→ **立即写入** 公司倍数到 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 同行数据到 `/tmp/tear-sheet/peer-comps.csv`

**查询4 — M&A活动：**
"[公司] 收购 资产剥离 已完成交易 最近5年 交易价值"
→ 公司自身交易
→ **立即写入** `/tmp/tear-sheet/ma-activity.csv`

**查询5 — 可比交易（数据允许时）：**
使用查询1结果中的具体行业（例如，"云可观测软件M&A交易"而非"科技M&A"）。如用户指定可比公司，也尝试："[可比公司] 收购 交易倍数。"
→ 可比交易
→ **追加** 可比交易到 `/tmp/tear-sheet/ma-activity.csv`（添加`type=precedent`以区分公司自身交易）

**查询6 — 关系 + 所有权：**
"[公司] 主要客户 供应商 合作伙伴 业务关系 机构持股 内部人持股"
→ 业务关系、所有权（数据允许时）
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

## 章节

按优先级顺序列出。参见下方"页数预算与删减顺序"获取明确的删减指导。

### 1. 公司标题
带交易相关标识符的键值块。

| 字段 | 备注 |
|---|---|
| 公司名称 | 大号、粗体 |
| 股票代码与交易所 | 如上市；私有则省略 |
| 总部位置 | 城市、州/国家 |
| 成立 | 年份 |
| 员工 | 大致人数 |
| 市值/估值 | 上市用市值；私有用最新已知估值 |
| 企业价值 | |
| 所有权 | 上市/PE支持（命名赞助商）/家族/其他 |

对于私有公司：显著注明"私有公司"；跳过股票代码、交易所。

### 2. 业务概述
**这是推介材料段落，不是CIQ摘要。** 不要粘贴数据工具中的公司描述。用4-6句话以推介材料文体重写：
- 描述收入模式（经常性 vs 交易性、订阅 vs 许可）
- 大规模命名客户群（"服务80%的财富500强银行"）
- 按名称识别竞争护城河（"X的专有数据集"、"唯一拥有Y的提供商"）
- 框架化对买家或卖家重要的增长向量

语气应自信且具体。银行家阅读此段落应立即理解公司的战略定位。

### 3. 收入与分部分解
按业务单元或产品线的收入。

| 分部 | 收入（$M） | 占比 | 同比增长 |
|---|---|---|---|
| [分部A] | | | |
| [分部B] | | | |
| **合计** | | **100%** | |

**如无分部数据：** 用2-3句定性描述收入构成替代表格（例如，"收入主要来自订阅许可（~70%）和专业服务（~30%）。"）。空白表格看起来有问题——定性描述仍传达有用信息。

### 4. 财务摘要（3年 + LTM）
实际财年作为列标题。**如LTM（最近十二个月）数据可用，在最近财年右侧添加LTM列。** 标记为"LTM [季度结束日期]"（例如，"LTM Q3 2025"）。银行家基于LTM进行估值交叉核对——仅财年视图可能滞后6+个月。

| 指标（$M） | FY20XX | FY20XX | FY20XX | LTM [日期] |
|---|---|---|---|---|
| 收入 | | | | |
| 收入增长率% | | | | |
| 毛利润 | | | | |
| 毛利率% | | | | |
| EBITDA | | | | |
| EBITDA利润率% | | | | |
| 营业利润 | | | | |
| 净利润 | | | | |
| 资本支出 | | | | |
| 自由现金流 | | | | |
| FCF利润率% | | | | |

在M&A背景下，利润率和FCF转换率与绝对规模同等重要——买家在评估效率和现金生成潜力。

**资本结构**（财务摘要下方的紧凑子表）：

| 指标 | 数值 |
|---|---|
| 总债务 | $XXM |
| 现金及等价物 | $XXM |
| 净债务 | $XXM |
| 净债务/EBITDA | X.Xx |
| S&P信用评级 | XX（如可用） |

从已检索的资产负债表数据提取。这是任何银行家公司简介的标准——总债务、杠杆比率和信用评级告诉买家融资情况。如信用评级不可用，省略该行。

### 5. 交易可比
公司倍数**和运营指标**——银行家一起使用，而非单独使用倍数。

**前瞻倍数在可用时是强制性的。** 表格必须包含NTM倍数（如工具返回）。当存在前瞻数据时仅显示滚动倍数是重大缺口——银行家基于前瞻收益定价交易。

**同行列是预期的。** 如用户提供可比公司，每个有自己的列。如未提供，使用竞争对手工具识别3-5家上市同行，拉取其倍数和运营指标，并包含它们。无同行背景的仅公司交易可比表对任何银行业用例都是不完整的。

| 指标 | [公司] | [可比1] | [可比2] | [可比3] | 同行中位数 |
|---|---|---|---|---|---|
| EV/收入（NTM） | | | | | |
| EV/EBITDA（NTM） | | | | | |
| 收入增长率% | | | | | |
| EBITDA利润率% | | | | | |

如尝试同行查询后仅公司自身倍数可用，显示这些并在表格下方列出选定的同行名称。

对于私有公司，跳过此章节。

### 6. M&A活动
两部分。**交易价值在此章节是预期的**——银行家会注意到缺失。

**硬规则：如M&A工具返回交易价值，必须出现在输出中。** 永远不要将已知交易价值降级为"未披露"——这是数据完整性违规。仅当工具真正不返回交易价值时使用"未披露"。

**a) 公司自身交易：**

**表格必须包含专门的交易价值列。** 不要将交易价值合并到备注或理由列——银行家扫描数字，专门列使其成为可能。

| 日期 | 标的/剥离资产 | 交易价值（$M） | 类型 | 理由 |
|---|---|---|---|---|
| | | | 收购/剥离 | |

如未找到："最近5年无已披露交易。"

**b) 可比行业交易（数据允许时）：**
使用之前识别的具体行业（例如，"云基础设施软件M&A"而非"科技M&A"）。如用户指定可比公司，也查找涉及这些特定公司的交易。

| 日期 | 标的 | 收购方 | EV（$M） | EV/收入 | EV/EBITDA |
|---|---|---|---|---|---|

如工具无法获取："数据源无法获取可比交易数据。"不要编造先例倍数。

### 7. 关键业务关系
利用S&P Capital IQ关系数据。按类型分组：

**客户：** 前3-5个具名客户
**供应商：** 关键供应商和技术提供商
**合作伙伴：** 战略联盟、分销、合资
**竞争对手：** 关键竞争对手名称

对于每个条目，包含关系类型和括号描述符解释关系性质（例如，"AWS — 云基础设施（主要托管提供商）"，"JPMorgan — 客户（企业分析平台）"）。仅名称无上下文无用。给此章节呼吸空间——它有战略价值，大多数竞争简报缺乏它。

### 8. 所有权快照（数据允许时——如工具无返回则省略）
如工具返回机构持股、内部人持股比例或主要持有人，包含紧凑区块：

主要机构持有人（如可用）：按持股比例列出前3-5名。
内部人持股：合计百分比。

对于PE支持公司，命名赞助商、收购年份和持股比例。所有权结构直接影响交易复杂度——广泛持有的上市公司、创始人控制公司和PE支持公司各自需要根本不同的交易方式。

如工具未返回所有权数据，完全省略此章节而非显示占位符。

**不要包含管理团队表格。** 无S&P Global工具返回高管数据——参见数据完整性规则10。来自训练数据的管理层名称将过时。

## 页数预算与删减顺序

**目标：最多2页。** 投行简介可以充分使用两页，但不得溢出到第三页。如内容超过2页，按此顺序删减（完全删减每个后再移至下一个）：

1. 所有权快照（第8节）——完全省略
2. 可比行业交易（第6b节）——保留公司自身M&A，删减先例
3. 业务关系（第7节）——压缩到前3竞争对手 + 前2供应商/合作伙伴/客户，描述符仅保留括号
4. 资本结构（第4节）——将净债务/EBITDA合并到财务摘要作为单行
5. 交易可比（第5节）——减少到3个同行

永不删减：业务概述（2）、收入与分部分解（3）、财务摘要（4核心）、M&A活动公司交易（6a）。

## 格式说明
- 业务概述和分部分解获得最多视觉权重。
- **M&A活动是标志性章节：** 使用强调色（#2E75B6）填充配白色粗体文本作为M&A表格标题——这是唯一使用与标准#D6E4F0不同标题颜色的表格。扫描时应引人注目。
- **业务关系：空间允许时给呼吸空间。** 每个关系组之间6磅，每个条目完整描述符。但如果达到第3页，按上述删减顺序压缩——此章节高价值但也最可压缩。
- 对于私有公司，标题、业务概述、财务和关系构成核心；其他章节可能稀疏。
- **两页纪律：** 渲染前，心理估算页数。如业务关系 + M&A活动合计将超过一整页，按删减顺序开始删减。
