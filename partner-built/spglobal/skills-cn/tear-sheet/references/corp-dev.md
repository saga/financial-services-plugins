# 企业发展部简报

## 目的
为内部企业发展团队评估潜在收购目标的简介文档。与投行/M&A简介（为推介讲述战略故事）不同，这是专注于战略契合度的分析性文档：目标公司的产品、客户、技术和运营如何与收购方重叠或互补。企业发展受众已经了解自己的公司——他们需要通过"我们应该买这个吗，整合会是什么样子"的视角来理解目标公司。

**默认页数：** 1-2页。企业发展团队习惯于更密集的简介，如果内容充实，不会因第二页而扣分。

## 查询计划

从这些查询开始。如果结果不完整，分解并跟进。

**查询1 — 目标公司概况：**
"[公司] 业务描述 产品 服务 技术 平台 总部 成立 员工 行业 产业"
→ 标题、业务与产品概述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询2 — 财务数据：**
"[公司] 年度利润表 收入 毛利润 EBITDA 营业利润 净利润 资本支出 自由现金流 研发费用 总债务 现金及等价物 最近4个财年"
→ 财务摘要（拉取4年；显示3年；最早年份仅用于计算同比增速。研发对企业发展尤为重要——他们想了解目标公司在IP上的投资）
→ **立即写入** 原始数值到 `/tmp/tear-sheet/financials.csv`

**查询3 — 分部 + 客户：**
"[公司] 按分部收入 业务单元 最近2个财年 主要客户 终端市场 客户集中度"
→ 收入构成（需要2年计算同比增速），客户分析
→ **立即写入** 分部数据到 `/tmp/tear-sheet/segments.csv`（如无分部数据则跳过）

**查询4 — 关系 + 竞争格局：**
"[公司] 主要客户 供应商 合作伙伴 竞争对手 业务关系 技术供应商"
→ 战略契合分析、生态图谱
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

**查询5 — 估值 + 同行：**
如用户提供特定可比公司：
"[可比公司] 企业价值 EV/收入 EV/EBITDA 收入增长 EBITDA利润率"
否则：
使用竞争对手工具识别3-5家上市同行，然后拉取每家的EV/收入（NTM）、EV/EBITDA（NTM）、收入增长率%和EBITDA利润率%。
同时："[公司] 企业价值 市值 估值倍数"
→ 估值背景
→ **立即写入** 公司倍数到 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 同行数据到 `/tmp/tear-sheet/peer-comps.csv`

**查询6 — 所有权（数据允许时）：**
"[公司] 股权结构 投资者 机构持股 内部人持股"
→ 所有权快照（无中间文件——如可用则包含在company-profile.txt中）

## 章节

按优先级顺序列出。参见下方"页数预算与删减顺序"获取明确的删减指导。

### 1. 公司标题
类似于投行/M&A但增加"战略相关性"标语。

| 字段 | 备注 |
|---|---|
| 公司名称 | 大号、粗体 |
| 股票代码与交易所 | 如上市；私有则省略 |
| 总部位置 | |
| 成立/员工 | |
| 企业价值 | 上市用市值；私有用最新估值 |
| 所有权 | 上市/PE支持/创始人主导/等 |
| **战略相关性** | 一句话：为什么这个目标对收购方有吸引力（从数据综合） |

"战略相关性"行是企业发展部独有的。从中性分析视角撰写，而非目标公司的营销语言。框架化收购方*得到什么*——而非目标公司*是什么*。好的："在大宗商品价格基准领域占主导地位，具有监管护城河和70%+毛利率——轻资产、高壁垒的数据资产。"坏的："全球领先的金融数据和分析提供商。"前者告诉企业发展团队为什么有吸引力；后者是新闻稿。

### 2. 业务与产品概述
**不要粘贴CIQ公司摘要。** 用4-6句话重写（一段，不是两段），专注于产品和技术——此受众评估收购，而非投资：
- 核心产品/服务及其解决的问题
- 技术或平台架构（高层次）
- 关键差异化和竞争护城河
- 客户细分——谁购买以及为什么
- 当前战略方向

以收购方视角撰写。读者在想："这与我们已有的如何契合？"描述应便于发现重叠和互补。

### 3. 收入构成与客户概况
两个子部分：

**a) 按分部收入**（如可用）：

| 分部 | 收入（$M） | 占比 | 同比增长 |
|---|---|---|---|

**b) 客户分析：**
在分部表格下方（如无分部数据则替代），撰写定性段落涵盖：
- 客户集中度：前5大客户是否占收入>30%？（收购风险的关键）
- 客户类型：企业 vs 中小企业 vs 消费者
- 合同结构：订阅 vs 交易 vs 许可（如已知）
- 地域构成

客户集中度是这里最重要的信号——如果工具返回任何关于主要客户或收入集中度的数据，突出显示。如数据稀疏，从业务描述和关系推断可推断的内容撰写。保持2-3句——这是分部表格的背景，不是独立分析。

### 4. 战略契合分析
**这是企业发展简报的标志性章节。** 必须包含——不要跳过或压缩。使用S&P Capital IQ的业务关系数据映射重叠和互补。

组织为三个类别。**每个类别必须是2-3句分析推理，而非公司名称列表。** 列出"客户：Microsoft, JPMorgan, Google"而无上下文是无用的。相反，解释重叠或互补对收购意味着什么。

**客户重叠：** 目标公司的客户是否也从收购方（或收购方所在行业）购买？共享客户 = 更容易交叉销售，但也存在渠道冲突风险。命名共享关系并解释影响。

**技术/产品互补：** 目标公司是否填补收购方产品组合的空白？他们的技术栈是什么样的？谁是他们的技术供应商？如果他们已在使用收购方的技术，解释为什么这是强整合信号。

**竞争替代：** 目标公司是竞争对手吗，或者收购他们会从市场中移除一个竞争对手？命名目标公司的主要竞争对手并解释收购后竞争格局如何变化。

此章节需要综合——结合关系数据与业务概述建立联系。如果用户告诉您收购方是谁，针对该收购方定制。如未告知，仍然以分析性方式通用撰写。

**长度纪律：** 上述每个类别2-3句目标是硬约束，不是建议。每个要点应先展示洞察，然后用一个具体数据点支持。实际输出经常膨胀到每个要点5-7句——这是第3页溢出的主要驱动因素。如果发现自己写了超过3句，将多余细节移至整合考量。

### 5. 财务摘要（3年）
类似于投行/M&A但额外强调研发和资本效率。

| 指标（$M） | FY20XX | FY20XX | FY20XX |
|---|---|---|---|
| 收入 | | | |
| 收入增长率% | | | |
| 毛利率% | | | |
| EBITDA | | | |
| EBITDA利润率% | | | |
| 研发费用 | | | |
| 研发占收入比% | | | |
| 资本支出 | | | |
| 自由现金流 | | | |
| FCF转换率（FCF/EBITDA） | | | |

研发强度表明目标公司价值中有多少在IP与服务中。FCF转换率表明整合复杂度——高资本支出企业更难整合。如研发费用未单独报告，在行中注明"未单独披露"而非"N/A"——这本身就有信息（表明研发可能嵌入在COGS或SG&A中）。

**资本结构**（财务摘要下方的紧凑子表）：

| 指标 | 数值 |
|---|---|
| 总债务 | $XXM |
| 现金及等价物 | $XXM |
| 净债务 | $XXM |
| 净债务/EBITDA | X.Xx |

为收购背景框架化：表格下方，包含一句话解释杠杆（例如，"净债务$X代表Y.Yx EBITDA杠杆，对于投资级收购方可管理，但在LBO情景下会加剧杠杆。"）。从已检索的资产负债表数据中提取。

### 6. 估值背景
**此章节对上市公司是必需的。** 企业发展团队需要市场背景来框架化潜在报价。如果工具返回估值倍数，此章节必须出现。不要为空间删减——先删减所有权快照（第7节）。

非正式估值——企业发展团队自己做DCF。这提供市场背景。

**a) 交易倍数**（如上市）：

**同行背景是必需的，非可选。** 如用户提供可比公司，每个显示在各自列中。如未提供可比公司，使用竞争对手工具识别3-5家上市同行，然后拉取每家的EV/收入（NTM）、EV/EBITDA（NTM）、收入增长率%和EBITDA利润率%。

| 指标 | [公司] | [同行1] | [同行2] | [同行3] | 同行中位数 |
|---|---|---|---|---|---|
| EV/收入（NTM） | | | | | |
| EV/EBITDA（NTM） | | | | | |
| 收入增长率% | | | | | |
| EBITDA利润率% | | | | | |

公司倍数单独看对企业发展团队毫无意义——他们需要相对背景来评估目标是便宜还是昂贵相对于同行组。这是目标简介的最低可行估值背景。

如同行倍数无法从工具获取，显示公司自身倍数并在表格下方列出选定的同行名称供读者参考。

**b) 先例交易（数据允许时）：**
如M&A工具返回涉及公司同行或同一子行业的近期交易，显示它们：

| 日期 | 标的 | 收购方 | EV（$M） | EV/收入 | EV/EBITDA |
|---|---|---|---|---|---|

框架化为背景："[子行业]近期交易估值目标为X-Yx收入。"

如工具无法获取先例交易数据，注明"数据源无法获取先例交易数据"而非静默省略。不要编造先例倍数。

对于私有公司，跳过交易倍数。先例交易（数据允许时）仍有用。

### 7. 所有权快照（数据允许时——如工具无返回则省略）
如工具返回所有权数据，包含紧凑区块：创始人/家族控制%、PE赞助商详情、机构集中度。所有权结构直接影响交易复杂度，与整合考量相关。

**不要包含管理团队表格。** 无S&P Global工具返回高管数据——参见数据完整性规则10。来自训练数据的管理层名称将过时。

### 8. 整合考量
**必需章节——不要删减。** 这是企业发展简报的分析顶点。从上述所有内容综合得出的观察。3-4个实质性要点：
- **关键风险：** 客户集中度、关键人依赖、技术债务指标
- **整合复杂度：** 地理位置（多国=更难）、员工数量、产品数量
- **协同信号：** 共享客户、互补产品、重叠的技术供应商
- **待解决问题：** 企业发展团队在报价前需要尽职调查什么？专注于具体、可操作的尽职项目——而非开放式战略问题。好的例子：监管审批时间线和管辖区、关键合同中的控制权变更条款、关键人保留风险（特别是创始人/CTO）、客户合同可转让性、IP所有权结构、未决诉讼。坏的例子："X的防御性如何？"或"长期市场机会是什么？"——这些是分析师问题，不是尽职项目。

清楚标注为分析观察，而非数据来源的事实。

## 页数预算与删减顺序

**目标：2页。** 企业发展简介可以实质性内容填满两页，但除非用户明确要求更长格式，不得溢出到第三页。如内容超过2页，按此顺序删减（完全删减每个后再移至下一个）：

1. 所有权快照（第7节）——完全省略
2. 收入构成下的客户分析文字（第3节）——减少到最多2句
3. 估值背景文字（第6节）——保留同行可比表，删减下方解释段落
4. 先例交易（第6b节）——保留"不可用"注释，如空间关键则移除
5. 资本结构文字——保留表格，删减下方解释段落
6. 战略契合分析（第4节）——将每个要点压缩到最多2-3句（不要删除章节）

永不删减：业务与产品概述（2）、财务摘要（5核心表格）、整合考量（8）。

## 格式说明
- **战略契合分析是核心。** 给它独特的格式：
  - 将文本提升到9.5磅（size: 19）——略大于标准正文。此章节应看起来是页面上最重要的内容。
  - 使用全局样式配置中的标准缩进块式项目符号（360 DXA左缩进）。
  - 不要应用左边框强调——它们在docx-js中渲染不一致。
- 财务摘要应突出显示研发和FCF转换率。
- 上市公司的估值背景与同行可比是必需的——不要在所有权快照之前删减。
- 对于私有目标，简介严重依赖定性章节（业务概述、战略契合、关系）——这是预期的且仍有高价值。
- 语气：分析和评估，非推广。这不是推介——这是评估。
- **两页纪律：** 渲染前，心理估算页数。战略契合（3个长要点）+整合考量（4个要点）的组合本身可能消耗整整一页——如是，先压缩战略契合要点。
