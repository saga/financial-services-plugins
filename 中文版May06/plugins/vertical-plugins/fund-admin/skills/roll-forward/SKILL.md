# 余额变动调节 (Roll-forward)

给定账户（或账户组）、实体及期间，生成一份将期初余额与期末余额对齐的变动表。

## 结构

```
期初余额 (根据上期结账数据)             X
  + 增加项 / 新发生的活动              A
  + 本期计提 (Accruals)               B
  - 上期计提冲回 (Reversals)          (C)
  - 支付 / 结算 (Payments)            (D)
  ± 重分类 / 调整 (Reclasses)          E
  ± 汇兑转换 (FX)                      F
期末余额 (根据本期期末总账 GL)          Y
```

## 逐行核对 (Tie-out)

- **期初余额** — 引用上期结账包或上期期末的总账余额。
- **每一项活动行** — 通过内部总账 (internal-gl) MCP 进行 GL 查询（账户 + 日期范围 + 日记账来源过滤）。需引用查询结果。
- **期末余额** — 引用本期期末日期的总账余额。

调节表**必须算通 (Footing)**：`X + A + B - C - D + E + F = Y`。如果不相等，差额即为“未解释项”——请如实反映，严禁为了凑平而随意填补数值。

## 输出要求

包含“核对至 (Ties to)”列的变动表，该列需注明每一行所引用的 GL 查询或文档。此外还需包含计算准确性检查（通过/失败，以及如有未解释的垂直差额）。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **Roll-forward (余额变动调节)**：在财务中，这就像追踪一个水池里的水量。你不仅要知道月底池里有多少水（期末余额），还要清楚月初有多少（期初），这个月流进来多少（增加/计提），又排出去多少（支付/冲回）。由于金融机构对准确性要求极高，每一滴水的流向都必须有据可查。
> 2. **总账 (General Ledger, GL)**：它是公司所有财务数据的“大账本”。无论是买办公用品还是支付上亿的投资款，最后都会在 GL 里体现。
> 3. **算通 (Footing)**：这是一个会计行话，指垂直核对表格中的数字相加是否等于总数。在电子表格时代，这看起来很简单，但在涉及复杂的汇率调节和手工调整时，经常会出现差错。
> 4. **计提 (Accruals)**：一种“先记账后给钱”的原则。比如，你这个月用了电，但电费单还没来，你必须先“预估”一笔电费记在账上。
