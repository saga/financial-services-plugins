# 可比公司分析 (Comparable Company Analysis)

## ⚠️ 关键：数据源优先级（请先阅读）

**请始终遵循以下数据源层级：**

1.  **首选：检查 MCP 数据源** - 如果可以使用 S&P Kensho MCP、FactSet MCP 或 Daloopa MCP，请直接使用这些工具获取财务和交易信息。
2.  **若有上述 MCP 数据源，切勿使用网络搜索。**
3.  **仅在 MCP 不可用时：** 才使用彭博终端（Bloomberg Terminal）、SEC EDGAR 申报文件或其他机构级来源。
4.  **严禁将网络搜索作为主要数据源** - 网络数据缺乏机构级分析所需的准确性、审计追踪和可靠性。

**原因：** MCP 来源提供经过验证的机构级数据并附带正确的引用。网络搜索结果对于财务分析而言可能已经过时、不准确或不可靠。

---

## 概述
本技能旨在指导 Claude 构建机构级可比公司分析，将运营指标、估值倍数和统计基准相结合。输出结果是一个结构化的 Excel/电子表格，旨在通过同行对比辅助投资决策。

**参考材料与上下文应用：**

`examples/comps_example.xlsx` 中提供了一个可比公司分析示例。在应用本技能目录下的示例文件时，请灵活处理：

**示例的正确用途：**
- 了解结构层级（各部分的衔接流程）
- 掌握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰的表头、透明的公式、审计追踪）

**示例的错误用途：**
- 机械照搬格式或指标
- 不考虑具体情况直接复制布局
- 不分受众群体套用相同的视觉风格

**在开始前请始终自问：**
1. **“您是否有偏好的格式，还是由我来调整模板风格？”**
2. **“受众是谁？”**（投资委员会、董事会演示、快速参考、详细备忘录）
3. **“核心问题是什么？”**（估值、增长分析、竞争地位、效率）
4. **“上下文背景是什么？”**（并购评估、投资决策、行业基准分析、绩效回顾）

**根据具体情况进行调整：**
- **行业背景**：大型科技股需要的指标与新兴 SaaS 初创公司完全不同。
- **特定行业需求**：及早添加相关指标（例如科技行业的云 ARR、企业客户数、开发者生态系统）。
- **公司熟悉度**：知名公司可能不需要太多背景介绍，应更多关注增量分析（Delta Analysis）。
- **决策类型**：并购分析与日常投资组合监控的侧重点不同。

**核心原则：** 遵循模板原则（结构清晰、统计严谨、公式透明），但在执行时根据背景灵活变化。目标是产出机构级质量的分析，而非仅仅是外表像机构级的模板。

用户提供的示例和明确偏好始终优先于默认设置。

## 核心哲学
**“先构建正确的结构，再让数据说话。”**

从迫使你进行战略思考的表头开始，输入干净的数据，构建透明的公式，让统计结果自然呈现。一份优秀的 Comps 表格应当让没有参与构建的人也能一目了然。

## ⚠️ 关键：公式重于硬编码 + 逐步验证

**环境分析 —— Office JS vs Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS (`Excel.run(async (context) => {...})`)。通过 `range.formulas = [["=E7/C7"]]` 写入公式，而非 `range.values`。无需单独的重算步骤 —— Excel 会原生处理。使用 `range.format.*` 设置颜色/字体。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。写入 `cell.value = "=E7/C7"`（公式字符串）。
- 无论哪种方式，原则一致 —— 只是 API 调用不同。
- **Office JS 合并单元格陷阱：** 切勿先调用 `.merge()` 再对合并后的区域设置 `.values`（这会抛出 `InvalidArgument` 错误 —— 区域仍报告合并前的维度）。正确的做法是仅向左上角单元格写入值，然后再进行合并和格式化：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式重于硬编码：**
- 每一个派生数值（利润率、倍数、统计数据）必须引用输入单元格的 Excel 公式 —— 严禁粘贴预先计算好的数字。
- 使用 Python/openpyxl 构建表格时：写入 `cell.value = "=E7/C7"`（公式字符串），而非 `cell.value = 0.687`（计算结果）。
- 唯一的硬编码值应为原始输入数据（收入、EBITDA、股价等），且每一个输入值都应附带注明来源的单元格批注。
- **原因**：当输入值改变时，模型必须能自动更新。硬编码的利润率是一个随时可能爆发的隐藏错误。

**与用户逐步验证：**
- 设置好结构后 → 在填充数据前向用户展示表头布局。
- 输入原始数据后 → 在构建公式前展示输入块，并确认数据来源/期间。
- 构建运营指标公式后 → 在进入估值部分前展示计算出的利润率并进行合理性检查。
- 构建估值倍数后 → 在添加统计数据前展示倍数并确认其合理性。
- **不要**等构建完整个表格后再演示 —— 通过确认每个部分及早发现错误。

## 第一部分：文档结构与设置

### 表头块（第 1-3 行）
```
第 1 行：[分析标题] - 可比公司分析
第 2 行：[带有股票代码的公司列表] • [公司 1 (TICK1)] • [公司 2 (TICK2)] • [公司 3 (TICK3)]
第 3 行：截至 [日期] | 除每股金额和比率外，所有金额单位均为 [百万/十亿美元 USD]
```

**重要性：** 立即建立上下文。任何打开此文件的人都能立刻明白分析对象、创建时间以及如何解读数字。

### 视觉规范标准（可选 - 用户偏好和上传的模板始终优先）

**关键：以下仅为建议默认设置。请始终优先考虑：**
1. 用户的明确格式偏好
2. 任何上传模板文件中的格式
3. 公司/团队风格指南
4. 这些默认值（仅在无其他指导时使用）

**建议字体与排版：**
- **字体**：Times New Roman（专业、易读，行业标准）
- **字号**：数据单元格为 11pt，表头为 12pt
- **加粗**：部分标题（Section headers）、公司名称、统计标签

**默认颜色与阴影 —— 专业蓝/灰调（极简为佳）：**
- **保持克制** —— 仅使用蓝色和灰色。切勿引入绿色、橙色、红色或多种强调色。一份整洁的 Comps 表格总共使用 3-4 种颜色。
- **部分标题**（例如 “运营统计与财务指标”）：
  - 深蓝色背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色加粗文字
  - 整行跨列着色
- **列标题**（例如 “公司”、“收入”、“利润率”）：
  - 浅蓝色背景（`#D9E1F2` 或类似的淡蓝色）
  - 黑色加粗文字
  - 居中对齐
- **数据行**：
  - 公司数据背景为白色
  - 公式使用黑色文字；硬编码输入使用蓝色文字
- **统计行**（最大值、75 分位数等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文字，标签左对齐
- **这就是全部配色**：深蓝 + 浅蓝 + 浅灰 + 白。除非用户模板另有规定，否则不要使用其他颜色。

**建议格式惯例：**
- **小数精度**：
  - 百分比：1 位小数 (12.3%)
  - 倍数：1 位小数 (13.5x)
  - 金额：无小数，使用千位分隔符 (69,632)
  - 以百分比显示的利润率：1 位小数 (68.7%)
- **边框**：不设边框（界面整洁极简）
- **对齐**：所有指标居中对齐，确保整齐划一
- **单元格尺寸**：所有列宽应整齐一致，所有行高应保持一致（营造专业的网格感）

**注**：如果用户提供了模板文件或指定了不同的格式，请改用用户的格式。

## 第二部分：运营统计与财务指标

### 核心列（以此开始）
1.  **公司 (Company)** - 命名格式保持一致
2.  **收入 (Revenue)** - 规模指标（根据情况使用 LTM、季度或年度数据）
3.  **收入增长率 (Revenue Growth)** - 同比百分比变化
4.  **毛利润 (Gross Profit)** - 收入减去销售成本
5.  **毛利率 (Gross Margin)** - 毛利润/收入（基本盈利能力）
6.  **EBITDA** - 息税折旧摊销前利润
7.  **EBITDA 利润率 (EBITDA Margin)** - EBITDA/收入（运营效率）

### 可选补充（根据行业/目的选择）
- **季度 vs LTM** - 如果存在季节性影响，则两者均包含
- **自由现金流 (Free Cash Flow)** - 适用于资本密集型或 SaaS 业务
- **FCF 利润率** - FCF/收入（现金生成效率）
- **净利润 (Net Income)** - 适用于成熟且盈利的公司
- **营业利润 (Operating Income)** - 适用于折旧摊销（D&A）差异较大的业务
- **资本支出指标 (CapEx metrics)** - 适用于重资产行业
- **40原则 (Rule of 40)** - 专门用于 SaaS（增长率% + 利润率%）
- **FCF 转化率 (FCF Conversion)** - 用于收益质量分析（高级）

### 公式示例（以第 7 行为例）
```excel
// 核心比率 - 这些始终通过计算得出
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率 - 若相关则包含
FCF 利润率: =[FCF]/[Revenue]
净利率: =[Net Income]/[Revenue]
40原则: =[增长率%]+[FCF 利润率%]
```

**黄金法则：** 每个比率都应该是 [某指标] / [收入] 或 [表内指标] / [表内指标]。保持简单。

### 统计块（位于公司数据下方）

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留出一个空行以实现视觉隔离]
- 最大值 (Maximum): =MAX(B7:B9)
- 75 分位数 (75th Percentile): =QUARTILE(B7:B9,3)
- 中位数 (Median): =MEDIAN(B7:B9)
- 25 分位数 (25th Percentile): =QUARTILE(B7:B9,1)
- 最小值 (Minimum): =MIN(B7:B9)
```

**需要统计数据的列（可比指标）：**
- 收入增长率 %、毛利率 %、EBITDA 利润率 %、EPS
- EV/Revenue、EV/EBITDA、P/E、股息率 %、Beta

**不需要统计数据的列（规模指标）：**
- 收入、EBITDA、净利润（绝对规模随公司规模变化）
- 市值、企业价值（不同规模公司之间不可比）

**注**：在公司数据和统计行之间添加一个空行。不要添加 “行业统计” 或 “估值统计” 的标题行。

**分位数的意义：** 它们展示了分布情况，而不仅仅是平均值。75 分位倍数能告诉你 “优质” 公司的交易价格。

## 第三部分：估值倍数与投资指标

### 核心估值列（以此开始）
1.  **公司** - 顺序与运营指标部分一致
2.  **市值 (Market Cap)** - 当前市场估值
3.  **企业价值 (Enterprise Value)** - 市值 ± 净债务/现金
4.  **EV/Revenue** - 每一美元销售额的市场定价
5.  **EV/EBITDA** - 每一美元收益的市场定价
6.  **市盈率 (P/E Ratio)** - 价格相对于净收益的比率

### 可选估值指标（根据背景选择）
- **FCF 收益率 (FCF Yield)** - FCF/市值（用于关注现金的分析）
- **PEG 比率** - P/E / 增长率（用于成长型公司）
- **市净率 (Price/Book)** - 市值 vs 账面价值（用于重资产业务）
- **ROE/ROA** - 回报指标（用于盈利能力对比）
- **收入/EBITDA 复合增长率 (CAGR)** - 历史增长率（用于趋势分析）
- **资产周转率 (Asset Turnover)** - 收入/资产（用于运营效率分析）
- **资产负债率 (Debt/Equity)** - 杠杆率（用于资本结构分析）

**关键原则：** 包含 3-5 个对你所在行业至关核心的倍数。平衡信息的全面性与简洁性。

### 公式示例
```excel
// 核心倍数 - 务必包含这些
EV/Revenue: =[企业价值]/[LTM 收入]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
市盈率 (P/E): =[市值]/[净利润]

// 可选倍数 - 若数据可用则包含
FCF 收益率: =[LTM FCF]/[市值]
PEG 比率: =[P/E]/[增长率 %]
```

### 交叉引用规则
**关键：** 估值倍数必须引用运营指标部分。严禁重复输入相同的原始数据。如果收入在 C7，那么 EV/Revenue 公式必须引用 C7。

### 统计块
结构与运营指标部分相同：为每个指标计算最大值、75 分位、中位数、25 分位和最小值。在公司数据和统计数据之间留出一个空行。不要添加 “估值统计” 标题行。

## 第四部分：附注与方法论文档

### 必要组成部分

**数据来源与质量：**
- 数据来源？（S&P Kensho MCP, FactSet MCP, Daloopa MCP, Bloomberg, SEC 申报文件）
- 覆盖期间？（2024 年第四季度，经审计数据）
- 如何验证？（与 10-K/10-Q 进行核对）
- 注：为了获得更高的准确性和追溯性，请优先使用 MCP 数据源。

**关键定义：**
- EBITDA 计算方法（毛利润 + 折旧摊销，或 营业利润 + 折旧摊销）
- 自由现金流公式（经营性现金流 - 资本支出）
- 特殊指标说明（40原则、FCF 转化率）
- 时间周期定义（LTM、CAGR 计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净债务）
- 使用了什么增长率？（历史复合增长率、预测数据）
- 是否进行了任何调整？（剔除一次性项目、利润率规范化）

**分析框架：**
- 投资逻辑是什么？（云服务/SaaS 效率）
- 哪些指标最重要？（获现能力、资本效率）
- 读者应如何解读统计数据？（分位数提供背景参考）

## 第五部分：选择正确的指标（决策框架）

### 从“我要回答什么问题？”开始

**“哪家公司被低估了？”**
→ 重点关注：EV/Revenue、EV/EBITDA、P/E、市值
→ 跳过：运营细节、增长指标

**“哪家公司的效率最高？”**
→ 重点关注：毛利率、EBITDA 利润率、FCF 利润率、资产周转率
→ 跳过：规模指标、绝对金额

**“哪家公司增长最快？”**
→ 重点关注：收入增长率 %、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆率

**“哪家公司的获现能力最强？”**
→ 重点关注：FCF、FCF 利润率、FCF 转化率、资本支出强度
→ 跳过：EBITDA、P/E 倍数

### 行业特定指标选择

**软件/SaaS：**
必选：收入增长率、毛利率、40原则
可选：ARR、净金额留存率 (NDR)、CAC 回本周期
跳过：资产周转率、库存指标

**制造/工业：**
必选：EBITDA 利润率、资产周转率、资本支出/收入
可选：ROA、库存周转率、积压订单
跳过：40原则、SaaS 指标

**金融服务：**
必选：ROE、ROA、成本收入比 (Efficiency Ratio)、P/E
可选：净息差 (NIM)、拨备
跳过：毛利率、EBITDA（对于银行没有意义）

**零售/电子商务：**
必选：收入增长率、毛利率、库存周转率
可选：同店销售增长、获客成本 (CAC)
跳过：高额研发或资本支出指标

### “5-10 原则”

**5 个运营指标** - 收入、增长、2-3 个利润率/效率指标
**5 个估值指标** - 市值、企业价值、3 个倍数
**= 10 列总计** - 足以说明问题，又不至于多到让人抓不住重点。

如果你的指标超过 15 个，可能包含了不少噪音。请无情剔除。

## 第六部分：最佳实践与质量检查

### 开始之前
1.  **定义同行组** - 公司必须具备真正的可比性（商业模式、规模、地理位置相似）
2.  **选择正确的期间** - LTM 能平滑季节性波动；季度数据则展示趋势
3.  **预先统一单位** - 万元、亿元或百万、十亿的选择会影响全局
4.  **映射数据源** - 明确每个数字的出处

### 构建过程中
1.  **先输入所有原始数据** - 在编写公式前完成蓝色文字的填充
2.  **为所有硬编码输入添加单元格批注** - 右键单元格 → 插入批注 → 记录来源或假设

    **对于引用的数据，请准确注明出处：**
    - 示例：“Bloomberg Terminal - MSFT Equity DES, accessed 2024-10-02”
    - 示例：“Q4 2024 10-K 申报文件, 第 42 页, 项目 ‘总收入’”
    - 示例：“FactSet 共识预期，截至 2024-10-02”
    **尽可能包含超链接**：右键单元格 → 链接 → 粘贴 SEC 申报文件、数据源或报告的 URL。

    **对于假设，说明理由：**
    - 示例：“基于同行中位数假设 EBITDA 利润率为 15%，公司未披露该数据”
    - 示例：“估算企业价值 = 市值 + 5000 万美元净债务（源自 Q3 资产负债表，Q4 尚未发布）”
    - 示例：“前瞻 P/E 基于市场共识 EPS $3.45（12 位分析师预期的平均值）”

    **重要性**：实现审计追踪、数据验证、假设透明度并方便未来更新。
3.  **逐行构建公式** - 在继续下一步之前测试每个计算结果
4.  **在表头使用绝对引用** - $C$6 可锁定标题行
5.  **保持格式一致** - 百分比显示为百分比，而非小数
6.  **添加条件格式** - 自动突出显示异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA 利润率 > 净利率（根据定义这是必然的）
- **倍数合理性**：
    - EV/Revenue：通常为 0.5-20x（随行业变化极大）
    - EV/EBITDA：通常为 8-25x（在各行业间相对一致）
    - P/E：通常为 10-50x（取决于增长率）
- **增长与倍数的相关性**：高增长通常意味着高倍数
- **规模与效率的权衡**：大公司通常拥有更好的利润率（规模效益）

### 应避免的常见错误
❌ 在公式中混淆市值和企业价值
❌ 分子和分母使用不同的时间段（LTM vs 季度）
❌ 在公式中硬编码数字而非引用单元格
❌ **硬编码输入未附带注明来源或说明假设的批注**
❌ 在有条件时不提供指向 SEC 申报文件或数据源的超链接
❌ 包含太多没有明确目的的指标
❌ 包含不具可比性的公司（商业模式不同）
❌ 使用过时数据且未予披露
❌ 错误地计算百分比的平均值（应使用中位数）

### 数据质量问题
🚩 时间段不一致（混用季度和年度数据）

### 基本 Excel 公式
```excel
// 统计函数
=AVERAGE(range)          // 简单平均值
=MEDIAN(range)           // 中位数
=QUARTILE(range, 1)      // 25 分位数
=QUARTILE(range, 3)      // 75 分位数
=MAX(range)              // 最大值
=MIN(range)              // 最小值
=STDEV.P(range)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司的平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除零错误

// 跨表引用
='Sheet1'!B7             // 引用另一张工作表
=VLOOKUP(A7, Table1, 2)  // 从数据表查找
=INDEX(MATCH())          // 高级查找

// 格式化
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千位分隔符
```

### 常见比率公式
```
毛利率 = 毛利润 / 收入
EBITDA 利润率 = EBITDA / 收入
FCF 利润率 = 自由现金流 / 收入
FCF 转化率 = FCF / 经营性现金流
ROE = 净利润 / 股东权益
ROA = 净利润 / 总资产
资产周转率 = 收入 / 总资产
资产负债率 = 总债务 / 股东权益
```

## 核心原则提示

1.  **结构引导洞察** —— 正确的表头强制正确的思考
2.  **少即是多** —— 5-10 个重要指标优于 20 个无关指标
3.  **根据问题选择指标** —— 估值分析 ≠ 效率分析
4.  **统计揭示模式** —— 中位数/分位数比平均值更能说明问题
5.  **透明胜过复杂** —— 使用人人都能理解的简单公式
6.  **可比性至上** —— 宁可剔除也不要勉强对比
7.  **记录你的选择** —— 在附注中说明选择了哪些指标及其原因

> **💡 Appendix: 领域知识小贴士**
>
> 即使你从来没学过金融，用生活中的例子也能秒懂这些“高大上”的概念：
>
> 1. **什么是可比公司分析 (Comps)？**
>    想象你要买一套房子，但不知道房东开的价格合不合理。你会去看隔壁栋、后面那条街类似户型的房子最近卖了多少钱。这就是 Comps。我们将性质相似的公司放在一起比价，就能知道某家公司是“买贵了”还是“捡漏了”。
>
> 2. **营运指标 (Operating Metrics)——“身体素质”：**
>    如果公司是一个运动员，收入就是他的身高，增长率是他的成长速度，而利润率（Margin）就是他的体脂率。体脂率低（利润率高）说明公司很强壮，每一块肌肉（每一块钱收入）都能转化成实实在在的力气（利润）。
>
> 3. **估值倍数 (Valuation Multiples)——“价格放大镜”：**
>    市盈率 (P/E) 最常用。如果一家公司每年赚 1 块钱，你愿意花 20 块钱买它，那市盈率就是 20 倍。这代表你愿意为了它未来的盈利等待 20 年。通常，大家觉得更有潜力的公司，这个倍数就越高。
>
> 4. **企业价值 (EV) vs 市值 (Market Cap)——“买房的总价”：**
>    *市值* 就像是房子的“首付”；而 *企业价值 (EV)* 才是“成交总价”。因为买下一家公司时，你不仅买下了它的股票（首付），还要负责它的债务（房贷），当然它账上的现金（你兜里的钱）可以抵消一部分。EV = 股票总价 + 欠别人的钱 - 剩下的现金。
>
> 5. **EBITDA——“赚钱的真本事”：**
>    这个词听起来很酷，其实它剔除了很多干扰（比如交多少税、利息支出等）。它衡量的是一家公司核心业务到底能产生多少现金，是衡量公司经营效率的“金标准”。
