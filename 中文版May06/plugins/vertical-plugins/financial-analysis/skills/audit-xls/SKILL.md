# 审计电子表格

审计公式与数据的准确性及错误。审计深度由范围决定——从对选定区域的快速公式检查，到对完整财务模型的合规性审计。

## 第 1 步：确定范围

如果用户已指定范围，直接使用。否则**询问用户**：

> 您希望审计哪个范围？
> - **选定区域 (selection)** —— 仅限当前选中的单元格区域
> - **工作表 (sheet)** —— 仅限当前活动的工作表
> - **模型 (model)** —— 整个工作簿，包括财务模型完整性检查（资产负债表平稳性、现金勾稽、滚动计算及逻辑合理性）

**模型**范围最为深入——在将折现现金流 (DCF)、杠杆收购 (LBO)、三表联动、并购 (Merger)、可比公司分析 (Comps) 或任何集成财务模型发送给客户或投委会 (IC) 之前，请务必使用此范围。

---

## 第 2 步：公式级检查（适用于所有范围）

无论范围如何，均需运行以下检查：

| 检查项 | 检查重点 |
|---|---|
| 公式错误 | `#REF!`, `#VALUE!`, `#N/A`, `#DIV/0!`, `#NAME?` |
| 公式内硬编码 | `=A1*1.05` —— `1.05` 应当采用单元格引用 |
| 公式不一致 | 某行或某列中的公式打破了相邻单元格的计算模式 |
| 区域偏移错误 | `SUM` 或 `AVERAGE` 漏掉了首行或末行 |
| 公式被复写 | 单元格看起来像公式，但实际是硬编码的数值 |
| 循环引用 | 检查是属于刻意设计还是无意失误 |
| 跨表链接断开 | 引用了已移动或删除的单元格 |
| 单位/量级不匹配 | 千位与百万位混用，或百分比存储为整数 |
| 隐藏行/隐藏标签页 | 可能包含复写操作或过时的计算逻辑 |

## 第 3 步：模型完整性检查（仅限“模型”范围）

如果范围是**模型**，请识别模型类型（DCF / LBO / 三表联动 / 并购 / 可比公司 / 自定义），并运行下述相应的完整性检查。

### 3a. 结构评审

| 检查项 | 检查重点 |
|---|---|
| 输入与计算分离 | 输入值是否与计算逻辑有清晰的界限？ |
| 配色规范 | 蓝色为输入，黑色为公式，绿色为链接 —— 模型是否遵循一致的配色标准？ |
| 标签页流向 | 逻辑顺序是否合理（假设 → 利润表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期表头 | 所有标签页的日期表头是否一致？ |
| 单位 | 单位是否统一（千位 vs 百万位 vs 原始数值）？ |

### 3b. 资产负债表 (BS)

| 检查项 | 测试方法 |
|---|---|
| 资产负债表配平 | 每一期：总资产 = 总负债 + 股东权益 |
| 留存收益滚动 | 期初留存收益 + 净利润 − 股利 = 期末留存收益 |
| 商誉/无形资产 | 是否来源于收购假设（针对并购模型） |

如果资产负债表不平，需**量化每一期的差额并追踪断开的位置** —— 在解决此问题之前，其他任何审计项均无意义。

### 3c. 现金流量表 (CF)

| 检查项 | 测试方法 |
|---|---|
| 现金勾稽 | 每一期：现金流量表期末现金 = 资产负债表货币资金 |
| 现金总额校验 | 经营性现金流 + 投资性现金流 + 筹资性现金流 = 现金净变动额 |
| 折旧摊销 (D&A) 匹配 | 现金流量表中的 D&A = 利润表中的 D&A |
| 资本支出 (CapEx) 匹配 | 现金流量表中的 CapEx 与资产负债表中的固定资产 (PP&E) 滚动计算一致 |
| 营运资本 (WC) 变动 | 符号是否与资产负债表变动一致（如应收、应付、存货的变动） |

### 3d. 利润表 (IS)

| 检查项 | 测试方法 |
|---|---|
| 收入拆解 | 收入总额是否与细分部门或产品细节勾稽 |
| 税项 | 所得税费用 = 税前利润 × 税率（允许考虑递延所得税调整） |
| 股份数 | 是否与稀释表一致（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息，是 LBO 或三表模型中常见的有意设计的循环引用
- 若是有意设计：验证“迭代计算”开关是否已开启并工作正常
- 若是无意导致：追踪循环路径并标记如何打破死循环

### 3f. 逻辑合理性

| 检查项 | 警告标识 |
|---|---|
| 增长率 | 营收增长率 > 100% 且无合理解释 |
| 利润率 | 明显超出行业常规范围 |
| 终值占比过高 | 终值 (TV) 占 DCF 企业价值 (EV) > 75%（需留意） |
| 曲棍球棒效应 | 预测期末年的数据呈现不切实际的激增 |
| 复合增长 | EBITDA 经过 10 年复合增长后达到荒谬的量级 |
| 极端情况处理 | 模型在 0% 或负增长、负 EBITDA、负杠杆情况下是否崩溃 |

### 3g. 特定模型类型的常见漏洞

**现金流折现 (DCF):**
- 折现率应用于错误的期间（期中 vs 期末）
- 终值未折现回现值
- WACC 使用账面价值而非市场价值
- 自由现金流 (FCF) 包含了利息费用（应使用无杠杆现金流）
- 税盾价值被重复计算

**杠杆收购 (LBO):**
- 债务偿还逻辑不符合现金扫频 (Cash Sweep) 机制
- PIK 利息未计入本金滚动
- 管理层股权滚存 (Rollover) 未在回报中体现
- 退出倍数应用于错误的 EBITDA（过去 12 个月 LTM vs 未来 12 个月 NTM）
- 交易费用/开支未从初始权益投入中扣除

**并购 (Merger):**
- 增厚/稀释分析使用了错误的股份数（交易前 vs 交易后）
- 协同效应未分阶段体现
- 购买价格分配 (PPA) 不平衡
- 未计入因支付现金而损失的利息收入
- 资金来源与用途表 (Sources & Uses) 中漏掉交易费用

**三表联动:**
- 营运资本变动的符号错误
- 折旧与固定资产 (PP&E) 附表不一致
- 债务到期计划与本金偿付不匹配
- 股利在无解释的情况下超过了净利润
