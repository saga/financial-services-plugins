# 三表财务模型模板填充指南

完成并填充整合财务模型模板，确保利润表（IS）、资产负债表（BS）和现金流量表（CF）之间的勾稽关系正确链接。

## ⚠️ 核心原则 — 填充模板前必读

**运行环境 — Office JS 与 Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS。通过 `range.formulas = [["=D14*(1+Assumptions!$B$5)"]]` 编写公式，绝不要对派生单元格使用 `range.values`。无需单独重算，Excel 会原生计算。使用 `context.workbook.worksheets.getItem(...)` 在标签页间跳转。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。编写 `ws["D15"] = "=D14*(1+Assumptions!$B$5)"`，并在交付前运行 `recalc.py`。
- **Office JS 合并单元格陷阱：** 不要先调用 `.merge()` 再在合并区域上设置 `.values`，这会抛出 `InvalidArgument` 错误，因为该区域仍会报告其合并前的维度。应先在左上角单元格写入值，然后再合并并格式化整个区域：`ws.getRange("A1").values = [["INCOME STATEMENT"]]; const h = ws.getRange("A1:G1"); h.merge(); h.format.fill.color = "#1F4E79";`
- 以下所有原则在两种环境下均同样适用。

**公式优于硬编码（不可逾越的底线）：**
- 每个预测单元格、余额滚动（Roll-forward）、报表链接和小计必须使用 Excel 公式，绝不能是预先计算好的数值。
- 使用 Python/openpyxl 时：编写公式字符串（`ws["D15"] = "=D14*(1 ...)"`），而非计算结果（`ws["D15"] = 12500`）。
- **唯二**允许包含硬编码数字的单元格是：(1) 历史实绩值（Actuals），(2) “假设”标签页中的驱动因素。
- 如果你发现自己在 Python 中计算了一个值并将其写入单元格 —— 请立即停止，改为编写公式。
- 原因：模型必须能够在情景切换或假设更改时灵活变动。硬编码会使所有后续的完整性检查静默失效。

**与用户进行分布确认：**
1. **完成模板映射后** → 向用户展示你识别出的标签页/科目，并在改动单元格前进行确认。
2. **填充历史数据后** → 向用户展示历史数据块，确认数值和期间与数据源匹配。
3. **完成利润表预测后** → 运行小计核对，向用户展示利润表预测，确认后再进行资产负债表。
4. **完成资产负债表后** → 向用户展示每一期的平衡检查（资产 = 负债 + 权益），确认后再进行现金流量表。
5. **完成现金流量表后** → 向用户展示现金钩稽结果（现金流期末现金 = 资产负债表现金），确认后再完成终稿。
6. **不要一次性填充整个模型并提交** —— 在每张报表完成时暂停，展示工作成果，及早发现错误。

## 格式规范 — 专业蓝/灰配色（除非模板/用户另有要求）

**保持颜色极简。** 单元格填充仅使用蓝色和灰色。严禁加入绿色、黄色、橙色或多种点缀色 —— 一个干净的模型需要克制。

| 元素 | 填充颜色 | 字体颜色 |
|---|---|---|
| 报表标题（IS / BS / CF 标题） | 深蓝 `#1F4E79` | 白色加粗 |
| 列标题（FY2024A, FY2025E 等） | 浅蓝 `#D9E1F2` | 黑色加粗 |
| 输入单元格（历史值、假设驱动因素） | 浅灰 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨表链接 | 白色 | 绿色 `#008000` |
| 检查行 / 核心合计 | 中蓝 `#BDD7EE` | 黑色加粗 |

**即：3 种蓝色 + 1 种灰色 + 白色。** 如果模板已有自带色系，请遵循模板。

字体颜色代表单元格的**性质**（输入/公式/链接）。填充颜色代表你所处的**位置**（表头/数据/检查项）。

## 模型结构

### 识别模板标签页组织方式

不同模板的标签页命名和组织方式各异。在填充前，请查看所有标签页以理解结构。以下是常见的标签页名称及其典型内容：

| 常见标签页名称 | 寻找的内容 |
|------------------|----------------------|
| IS, P&L, Income Statement | 利润表（损益表） |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资本明细表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销明细表 |
| Debt, Debt Schedule | 债务明细表 |
| NOL, Tax, DTA | 净经营亏损与税务明细表 |
| Assumptions, Inputs, Drivers | 驱动因素假设与输入参数 |
| Checks, Audit, Validation | 错误检查仪表盘 |

**模板审查清单**
- 识别模板中存在的标签页（并非所有模板都包含每个明细表）。
- 记录未在上方列出的模板特定标签页。
- 理解标签页间的依赖关系（例如，哪些明细表的数据会流入主表）。
- 定位每个标签页上的输入单元格与公式单元格。

### 理解模板结构

在填充前，先熟悉现有布局，确保数据填入正确位置且公式保持完整。

**识别行结构**
- 找到每个标签页顶部的模型标题。
- 识别科目表头及其视觉区分。
- 找到显示单位的行（单位：百万、$、%、倍数等）。
- 注意区分历史值（Actuals）与预测值（Estimates）期间的列标题。
- 确认期间标签（如 FY2024A, FY2025E）。
- 识别输入单元格与公式单元格（通常通过字体颜色区分）。

**识别列结构**
- 确认最左列的会计科目标签。
- 核实历史年度位于预测年度之前。
- 注意历史期间与预测期间之间的视觉边界。
- 检查所有标签页的列顺序是否一致。

**使用命名区域**
模板通常为关键输入和输出使用“命名区域”。在输入数据前：
- 检查模板中现有的命名区域（Excel 中：公式 → 名称管理器）。
- 常见的命名区域包括：营收增长率、成本占比、关键输出指标（净利润、EBITDA、总债务、现金）、情景切换开关。
- 确保输入的数据单元格确实流向了这些命名区域。

### 预测期间
- 模板通常从最后一个历史年度向后预测 5 年。
- 验证历史 (A) 与预测 (E) 列是否清晰隔离。
- 确认列标题使用财年表示法（如 FY2024A, FY2025E）。

## 利润率分析

**注意：仅当用户要求或模板明确需要时才进行以下分析。若未提及，请跳过此部分。**

在利润表（IS）页签计算并展示盈利能力指标，以追踪运营效率并进行同业对比。

### 核心利润率指标

| 指标 | 公式 | 衡量维度 |
|--------|---------|------------------|
| 毛利率 | 毛利 / 营业收入 | 定价能力、生产效率 |
| EBITDA 利润率 | EBITDA / 营业收入 | 核心业务盈利能力 |
| EBIT 利润率 | EBIT / 营业收入 | 扣除折旧摊销后的盈利能力 |
| 净利率 | 净利润 / 营业收入 | 最终盈利水平 |

### 包含利润率的利润表布局

在每个利润项目下方直接显示利润率百分比：
- 毛利下方显示毛利率 (%)
- EBIT 下方显示 EBIT 利润率 (%)
- EBITDA 下方显示 EBITDA 利润率 (%)
- 净利润下方显示 净利率 (%)

## 信用指标

**注意：仅当用户要求或模板明确需要时才进行以下分析。若未提及，请跳过此部分。**

在资产负债表（BS）页签计算并展示信用/杠杆指标，以评估财务健康状况、债务承受能力及融资契约合规性。

### 核心信用指标

| 指标 | 公式 | 衡量维度 |
|--------|---------|------------------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | (总债务 - 现金) / LTM EBITDA | 扣除现金后的杠杆水平 |
| 利息保障倍数 | EBITDA / 利息支出 | 偿债能力 |
| 债务 / 总资本 | 总债务 / (总债务 + 权益) | 资本结构 |
| 产权比率 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | (流动资产 - 存货) / 流动负债 | 变现能力 |

### 信用指标层级检查

验证“乐观情景”显示出最强的信用状况：
- 杠杆率：乐观 < 基准 < 悲观（越低越好）
- 利息覆盖率：乐观 > 基准 > 悲观（越高越好）
- 流动性：乐观 > 基准 > 悲观（越高越好）

### 融资契约监控

如果已知债务契约条款，需添加明确的合规性检查，将实际指标与契约阈值进行对比。

## 情景分析 (基准 / 乐观 / 悲观)

在假设标签页使用情景切换开关（下拉菜单），配合 CHOOSE 或 INDEX/MATCH 函数。

| 情景 | 描述 |
|----------|-------------|
| 基准情景 (Base) | 管理层指引或市场共识预期值 |
| 乐观情景 (Upside) | 高于指引的增长、利润率扩张 |
| 悲观情景 (Downside) | 低于趋势的增长、利润率收缩 |

**关键敏感性驱动因素**：营收增长、毛利率、管理及销售费用 %、应收/存货/应付周转天数、资本开支 %、利率、税率。

**情景审计检查**：切换开关时所有报表均会随之变动、所有情景下资产负债表均平衡、现金钩稽正确、层级逻辑成立（在净利润、EBITDA、自由现金流、利润率方面，乐观 > 基准 > 悲观）。

## 完成模型模板的步骤

本部分提供了在保持现有公式并确保数据完整性的前提下，完成任何三表模型模板的通用指南。

### 第 1 步：分析模板结构

在输入任何数据之前，彻底审查布局以理解其架构：

**识别输入单元格与公式单元格**
- 寻找区分两者的视觉线索（字体颜色、单元格阴影）。
- 通用惯例：蓝色字体 = 输入值，黑色字体 = 公式值，绿色字体 = 跨表链接。
- 使用 Excel 的“追踪引用单元格/从属单元格”功能来理解逻辑。
- 检查可能控制关键输入的命名区域。

**映射模板流程**
- 识别哪些标签页流向其他标签页（例如：假设 → 利润表 → 资产负债表 → 现金流量表）。
- 记录所有辅助明细表及其与主表的链接逻辑。

### 第 2 步：在不破坏公式的情况下填充数据

**数据输入金律**

| 原则 | 描述 |
|------|-------------|
| 仅编辑预测输入格 | 绝不要覆盖包含公式的单元格，除非有意替换该公式 |
| 保持单元格格式 | 复制数据时，使用“选择性粘贴 - 数值”（Ctrl+Shift+V） |
| 匹配模板单位 | 输入前核实模板使用的是千、百万还是原数值 |
| 遵循正负号惯例 | 遵循模板现有的正负号逻辑（例如费用是正数记账还是负数记账） |
| 检查循环引用 | 如果模板使用迭代计算，请确保已开启“启用迭代计算” |

**安全填充流程**
1. 识别专门指定的输入区域（通常有高亮或标签）。
2. 先填充历史数据，验证其在这些期间的公式计算是否正确。
3. 填充驱动预测计算的假设参数。
4. 审查输出结果，确认公式运行符合预期。
5. 若必须修改公式单元格，请先备份原始公式。

**处理预置公式**
- 如果公式引用的单元格尚未填充，可能会出现临时错误（如 #REF!, #DIV/0!），待输入完整后会消失。
- 当公式产生意外结果时，追踪其引用项以定位缺失或错误的输入。
- 严禁在未检查跨表公式依赖的情况下删除行/列。

### 第 3 步：验证公式

**公式完整性检查**

在依赖模板输出前，验证公式是否正常运行：

| 检查类型 | 方法 |
|------------|--------|
| 追踪引用项 | 选中公式格 → 公式 → 追踪引用单元格，确认其引用了正确的输入 |
| 追踪从属项 | 验证关键输入是否流向了预期的输出项 |
| 分步计算 | 使用“公式求值”功能逐步拆解复杂计算 |
| 检查硬编码 | 预测公式应引用假设参数，而非包含硬编码数值 |
| 测试已知值 | 输入简单的测试值验证输出是否符合逻辑 |
| 跨期一致性 | 确保同一预测逻辑应用于所有预测期间 |

**需警惕的常见公式问题**
- 混合绝对/相对引用导致在跨期复制时产生错误结果。
- 指向已删除区域或外部文件的断路链接（#REF! 错误）。
- 营收尚未产生的前期出现除以零错误（#DIV/0! 错误）。
- 循环引用警告（在利息计算中可能是刻意为之）。
- 预测列之间的公式不一致（使用 Ctrl+\ 查找差异）。

**验证跨表链接**
- 确认出现在多个标签页上的值是动态链接的，而非手动重复输入的。
- 验证明细表合计与主表相应科目完全一致。
- 确保所有标签页的时间跨度完全对齐。

### 第 4 步：按工作表质量检查

填充模板后，对每张表进行如下验证：

**利润表 (IS) 质检**
- 历史期间的营收数据与源数据匹配。
- 所有费用科目之和等于报告的总额。
- 小计（毛利、EBIT、EBT、净利润）计算正确。
- 所得税计算逻辑合理（能正确处理亏损结转）。
- 预测驱动因素引用的是假设页（无硬编码）。

**资产负债表 (BS) 质检**
- **资产 = 负债 + 权益**（每一期必须平衡，这是核心检查）。
- 现金余额与现金流量表期末现金一致。
- 营运资本科目与辅助明细表勾稽（如有）。
- 留存收益滚动正确：期初 + 净利润 - 股利 +/- 调整 = 期末。
- 债务余额与债务明细表一致。

**现金流量表 (CF) 质检**
- 经营性现金流顶部的净利润与利润表一致。
- 非现金加回项（折旧摊销、股份支付等）与源报表一致。
- 营运资本变动的正负号正确（资产增加 = 现金流出 = 负数）。
- 资本开支（CapEx）与固定资产滚动明细一致。
- 筹资活动与资产负债表中的债务及权益变动一致。
- 期末现金等于资产负债表中的现金。

### 第 5 步：跨表完整性核对

验证单表后，确认三张报表已完美整合：

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平衡表 | 资产 - 负债 - 权益 | = 0 |
| 现金钩稽 | CF 期末现金 - BS 现金 | = 0 |
| 净利润链接 | IS 净利润 - CF 起始净利润 | = 0 |
| 留存收益钩稽 | 期初 RE + 净利润 - 股利 - BS 期末 RE | = 0 |

### 第 6 步：最终复核

在认为模型完成前：
- 切换所有情景，确保每种情况下均通过检查。
- 解决或说明所有 #REF!, #DIV/0!, #VALUE!, #NAME? 错误。
- 确认所有输入格已填充（搜索占位符）。
- 验证所有标签页的单位（Units）保持一致。

## 模型验证与审计

本部分汇总了针对已完成模板的所有验证检查和审计程序。

### 核心链接（必须始终成立）

公式详情请参见 [references/formulas.md](references/formulas.md)。

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平衡 | 资产 - 负债 - 股东权益 | = 0 |
| 现金流对账 | 现金流量表期末现金 - 资产负债表现金 | = 0 |
| 月度/年度现金 | 闭口现金（月度） - 闭口现金（年度） | = 0 |
| 利润表链接 | 利润表净利润 - 现金流量表起始净利润 | = 0 |
| 留存收益核对 | 上期 RE + 净利润 + 股份支付 - 股利 - 本期 RE | = 0 |
| 权益融资核对 | Δ普通股/资本公积 (BS) - 股份发行 (CFF) | = 0 |

### 正负号惯例参考

| 报表 | 科目 | 记账符号惯例 |
|-----------|------|-----------------|
| 经营现金流 (CFO) | 折旧摊销 (D&A), 股份支付 (SBC) | 正值 (加回) |
| 经营现金流 (CFO) | 增加的应收账款 (ΔAR) | 负值 (资金占用) |
| 经营现金流 (CFO) | 增加的应付账款 (ΔAP) | 正值 (资金来源) |
| 投资现金流 (CFI) | 资本开支 (CapEx) | 负值 |
| 筹资现金流 (CFF) | 债务发行 | 正值 |
| 筹资现金流 (CFF) | 债务偿还 | 负值 |
| 筹资现金流 (CFF) | 股利支付 | 负值 |

### 循环引用处理

利息支出会产生循环：利息 → 净利润 → 现金 → 债务余额 → 利息。

在 Excel 中开启迭代计算：文件 → 选项 → 公式 → 启用迭代计算。建议设置最大迭代次数为 100，最大误差 0.001。并在假设页添加一个“断路器（Circuit Breaker）”开关。

### 检查类别汇总

**第 1 类：货币一致性**
- 在假设中明确币种。
- 所有标签页使用统一的货币符号和缩进。

**第 2 类：资产负债表完整性**
- 每一期均满足 资产 = 负债 + 权益。

**第 3 类：现金流勾稽**
- 现金流期末数 = 资产负债表现金。
- 经营流起始净利润 = 利润表净利润。
- 营运资本变动逻辑正确。

**第 4 类：税务与亏损结转 (NOL)**
- 只有当税前利润 (EBT) < 0 时才产生 NOL。
- 递延所得税资产 (DTA) 与明细表、资产负债表一致。
- NOL 的利用遵循相关法律限制。
- 当应纳税所得额 ≤ 0 时，所得税费用应为 0。

**第 5 类：情景一致性**
- 绝对值指标：乐观 > 基准 > 悲观（净利润、EBITDA、自由现金流）。
- 利润率指标：乐观 > 基准 > 悲观。
- 信用杠杆指标：乐观 < 基准 < 悲观（反向）。

### 总控公式 (Master Check)

将所有类别的状态汇总为一个总控单元格：
- 如果全部通过 → "✓ 全部检查通过"
- 如果任何一项失败 → "✗ 检测到错误 - 查看下方详情"

> **💡 Appendix: 领域知识小贴士**
>
> 核心财务概念极简说明：
>
> 1. **三表模型 (3-Statement Model)**：就像企业的“体检报告”。**利润表**看这公司这一年赚了多少钱，**资产负债表**看它手里有多少家底和欠债，**现金流量表**看兜里的现金到底是进来了还是流走了。
> 2. **勾稽关系 (Linkages)**：这是财务模型最迷人的地方。假设公司卖了一批货但还没收到钱，利润表上营收增加了，但资产负债表上“应收账款”也会增加，而现金流量表里会因为这笔钱没到手而扣掉这块现金。它们像齿轮一样，动一个，全都会动。
> 3. **硬编码 (Hardcodes)**：在模型里，除了历史数据，其他地方手动打数字就是“犯罪”。专业人士把假设写在专门的地方，其他地方全用公式连过去。这样你只要把“明年增长 10%”改成“20%”，整个模型瞬间就会自动更新。
> 4. **EBITDA**：读作“伊比特达”。它是看一家公司“搞钱能力”最纯粹的指标，去掉了各种非现金支出（折旧摊销）和杂七杂八的账务处理，就像看一个运动员在没穿沉重装备时的原始体能。
> 5. **营运资本 (Working Capital)**：这反映了公司在日常经营里压了多少“活钱”。如果客户总欠钱（应收账款高）或者仓库压太多货（存货高），虽然报表好看，但公司可能会因为没现金而倒闭，这就是财务分析里常说的“钱被生意困住了”。
