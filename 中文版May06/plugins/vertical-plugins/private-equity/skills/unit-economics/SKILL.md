# 单位经济效益分析 (Unit Economics Analysis)

description: 分析私募股权目标公司的单位经济效益——ARR群组分析、LTV/CAC、净留存率、回收期、收入质量和利润率瀑布。对于软件/SaaS、经常性收入和订阅业务至关重要。用于评估收入质量、建立群组分析或评估客户经济效益。触发词："单位经济效益"、"群组分析"、"ARR分析"、"LTV CAC"、"净留存"、"收入质量"或"客户经济效益"。

## 工作流程

### 步骤1：确定商业模式

确定收入模型以定制分析：
- **SaaS/订阅**：ARR、净留存率、群组
- **经常性服务**：合同价值、续约率、追加销售
- **交易/使用量计费**：每笔交易收入、量趋势、抽成率
- **混合模式**：按收入流细分

### 步骤2：核心指标

#### ARR/收入质量
- **ARR桥**：期初ARR → 新增 → 扩展 → 收缩 → 流失 → 期末ARR
- **ARR按群组**：年份分析——每个年度群组如何留存和增长？
- **收入集中度**：前10/20/50客户占总收入的百分比
- **收入类型**：经常性vs非经常性vs专业服务
- **合同结构**：ACV分布、多年期百分比、自动续约百分比

#### 客户经济效益
- **CAC（客户获取成本）**：总销售与营销支出/获取的新客户数
- **LTV（客户生命周期价值）**：(ARPU × 毛利率)/流失率
- **LTV:CAC比率**：健康业务目标>3x
- **CAC回收期**：收回获取成本所需的月数
- **混合vs细分**：按客户细分（企业vs中小企业vs中端市场）细分

#### 留存与扩展
- **总留存率**：期初ARR的留存百分比（不包括扩展）
- **净留存率(NDR)**：期初ARR的留存百分比（包括扩展）
- **客户流失率**：流失客户百分比
- **收入流失率**：流失收入百分比（通常与客户流失率不同）
- **扩展率**：追加销售+交叉销售占期初ARR的百分比

#### 群组分析
构建群组矩阵：

| 群组 | 第0年 | 第1年 | 第2年 | 第3年 | 第4年 |
|------|-------|-------|-------|-------|-------|
| 2020 | $1.0M | $1.1M | $1.2M | $1.1M | |
| 2021 | $1.5M | $1.7M | $1.8M | | |
| 2022 | $2.0M | $2.3M | | | |
| 2023 | $3.0M | | | | |

显示绝对金额和指数化（第0年=100%）视图。

#### 利润率瀑布
- 收入 → 毛利润 → 贡献毛利 → EBITDA
- 完全加载的单位经济效益：获取、服务和保留客户的成本是多少？
- 按收入流划分的毛利率（订阅vs服务vs其他）

### 步骤3：基准比较

将单位经济效益与相关基准进行比较：
- **SaaS 40法则**：增长率+EBITDA利润率>40%
- **SaaS魔法数字**：新增ARR/前期销售与营销支出>0.75x
- **NDR基准**：最佳>120%，良好>110%，令人担忧<100%
- **LTV:CAC**：最佳>5x，良好>3x，令人担忧<2x
- **总留存率**：最佳>95%，良好>90%，令人担忧<85%
- **CAC回收期**：最佳<12个月，良好<18个月，令人担忧>24个月

### 步骤5：输出

- Excel工作簿，包含ARR桥、群组矩阵、单位经济效益仪表板

## 重要提示

- 如果有可用的原始客户级别数据，务必要求——汇总指标可能隐藏问题
- 如果扩展足够强劲，NDR高于100%可能掩盖高总流失率——始终显示两者
- 群组分析是收入质量最重要的视图——争取获取此数据
- 区分合同ARR和实际确认收入
- 对于基于使用量的模型，关注消费趋势和扩展模式而非传统ARR指标
- 专业服务收入应单独评估——它不是经常性的，利润率通常较低

## 金融术语和知识解释

### 1. 单位经济效益 (Unit Economics)

**定义**：衡量单个客户或单位的收入与成本关系的指标。

**重要性**：
- 决定业务的长期可持续性
- 评估盈利能力
- 指导定价和营销策略

**核心问题**：
- 获取一个客户的成本是多少？
- 客户的生命周期价值是多少？
- 多久能收回客户获取成本？

### 2. ARR (Annual Recurring Revenue)

**定义**：年度经常性收入，衡量订阅业务的收入规模。

**计算方法**：
```
ARR = 活跃订阅客户数 × 平均年度合同价值(ACV)
```

**ARR桥分析**：
```
期末ARR = 期初ARR + 新增ARR + 扩展ARR - 收缩ARR - 流失ARR
```

**类型**：
- **新ARR**：新客户带来的收入
- **扩展ARR**：现有客户增加的收入
- **收缩ARR**：客户减少消费导致的收入减少
- **流失ARR**：客户流失导致的收入损失

### 3. LTV (Lifetime Value)

**定义**：客户在整个关系生命周期内为公司创造的总价值。

**计算公式**：
```
LTV = (ARPU × 毛利率) / 客户流失率
```

**详细公式**：
```
LTV = 平均客户生命周期 × 平均年度利润
```

**重要性**：
- 确定可接受的客户获取成本上限
- 优化客户保留策略
- 评估客户细分的价值

### 4. CAC (Customer Acquisition Cost)

**定义**：获取一个新客户的总成本。

**计算公式**：
```
CAC = 销售与营销总支出 / 新客户数量
```

**包括**：
- 销售团队薪酬
- 营销费用（广告、活动）
- 渠道合作伙伴佣金
- 促销折扣

**细分CAC**：
- **渠道CAC**：各渠道的获取成本
- **客户细分CAC**：不同客户类型的获取成本

### 5. LTV:CAC比率

**定义**：客户生命周期价值与客户获取成本的比率。

**计算公式**：
```
LTV:CAC = LTV / CAC
```

**基准解读**：
| 比率 | 评估 |
|------|------|
| >5x | 优秀 |
| 3-5x | 良好 |
| <2x | 不健康 |

**重要性**：
- 衡量获取客户的投资回报率
- 指导营销资源配置

### 6. 净留存率 (Net Dollar Retention, NDR)

**定义**：衡量现有客户收入的增长或下降。

**计算公式**：
```
NDR = (期初ARR + 扩展ARR - 收缩ARR - 流失ARR) / 期初ARR × 100%
```

**解读**：
- **NDR > 100%**：现有客户收入增长（理想状态）
- **NDR = 100%**：客户收入保持不变
- **NDR < 100%**：客户收入减少

**基准**：
- 优秀：>120%
- 良好：110%-120%
- 可接受：100%-110%
- 危险：<100%

### 7. 群组分析 (Cohort Analysis)

**定义**：按时间分组分析客户行为的方法。

**目的**：
- 评估不同时期客户的质量
- 识别客户留存模式
- 预测未来收入

**常见群组类型**：
- **获取群组**：按首次购买时间分组
- **行为群组**：按特定行为分组

**分析维度**：
- 留存率随时间变化
- 收入贡献变化
- 升级/降级模式

### 8. SaaS 40法则 (SaaS Rule of 40)

**定义**：评估SaaS公司健康状况的经验法则。

**公式**：
```
增长率 + EBITDA利润率 > 40%
```

**解读**：
- **>40%**：健康增长与盈利平衡
- **<40%**：需要优化增长或成本

**应用场景**：
- 评估成熟度
- 设定战略目标
- 投资者评估指标

### 9. SaaS魔法数字 (SaaS Magic Number)

**定义**：衡量销售与营销投资效率的指标。

**计算公式**：
```
魔法数字 = 新增ARR / 前期销售与营销支出
```

**解读**：
| 数值 | 评估 |
|------|------|
| >1.0 | 优秀 |
| 0.75-1.0 | 良好 |
| 0.5-0.75 | 中等 |
| <0.5 | 需要改进 |

**意义**：
- 衡量每投入1美元销售与营销费用能产生多少新增ARR
- 指示销售效率

### 10. CAC回收期 (CAC Payback Period)

**定义**：收回客户获取成本所需的时间。

**计算公式**：
```
CAC回收期（月）= CAC / (月均客户利润)
```

**基准**：
- 优秀：<12个月
- 良好：12-18个月
- 危险：>24个月

**影响因素**：
- 定价策略
- 毛利率
- 客户生命周期长度
