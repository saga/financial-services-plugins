# 滚调表

给定账户（或账户组）、实体和期间，生成期初到期末的滚调表。

## 结构

```
期初余额（上期结账包）      X
  + 新增 / 新活动                    A
  + 本期记账的应计                 B
  − 前期应计的回转                (C)
  − 付款 / 结算                     (D)
  ± 重分类 / 调整                     E
  ± 外汇折算                              F
期末余额（期末总账余额）           Y
```

## 结平每行

- **期初** — 上期结账包，或上期期末的总账余额。
- **每个活动行** — 通过internal-gl MCP的GL查询（账户 + 日期范围 + 日记账来源筛选）。引用查询。
- **期末** — 期末日期的总账余额。

时间表**必须结平**：`X + A + B − C − D + E + F = Y`。如果不平衡，**量化每期的差距并追溯哪里断开** — 在修复之前其他都不重要。

## 输出

滚调表，带"结平至"列，为每一行引用GL查询或文档，加上结平检查（通过/失败及任何未解释的差额）。

## Appendix: 金融背景知识

### 什么是滚调表（Roll-forward Schedule）？

滚调表显示账户余额从期初到期末的变动明细，是会计和对账中的核心工具。它帮助：
- 验证期末余额的正确性
- 识别异常变动
- 提供审计轨迹

### 关键术语解释

**期初余额（Beginning Balance）**：上期期末的账户余额，作为本期起点。

**新增/新活动（Additions/New Activity）**：本期新增的交易或活动，增加账户余额。

**应计记账（Accruals Booked）**：本期计提的应计项目，增加费用或负债。

**回转（Reversals）**：冲销前期应计的分录，减少负债。

**付款/结算（Payments/Settlements）**：实际支付现金，减少负债或资产。

**重分类/调整（Reclasses/Adjustments）**：将金额从一个账户转移到另一个账户，或校正错误。

**外汇折算（FX Translation）**：因汇率变动导致的外币余额价值变化。

**结平（Tie-out）**：验证计算结果的准确性，确保期初 + 变动 = 期末。

**未解释差额（Unexplained Delta）**：滚调表中期初 + 变动 ≠ 期末的差额，需要调查原因。

**internal-gl MCP**：通过Model Context Protocol访问公司内部总账系统的接口，用于提取余额和交易明细。

**审计轨迹（Audit Trail）**：从交易发起到最终记账的完整记录链，包含时间戳、操作人、系统ID等。

**总账（General Ledger, GL）**：公司所有财务交易的主记录，按账户分类汇总。

**重分类（Reclass）**：将金额从一个账户转移到另一个账户，通常因为初始分类错误，或需要按不同维度报告。
