# 审计电子表格

审计公式和数据的准确性和错误。范围决定深度——从对选定区域的快速公式检查到完整的财务模型完整性审计。

## 步骤1：确定范围

如果用户已指定范围，使用该范围。否则**询问他们**：

> 你想让我审计什么范围？
> - **selection（选定区域）** — 仅当前选定的范围
> - **sheet（工作表）** — 仅当前活动工作表
> - **model（模型）** — 整个工作簿，包括财务模型完整性检查（资产负债表平衡、现金匹配、滚动计算、逻辑合理性）

**model（模型）** 范围是最深的——在发送给客户或内部客户之前，用于DCF、LBO、三表联动、合并、可比公司或任何集成财务模型。

---

## 步骤2：公式级检查（所有范围）

无论范围如何，都运行以下检查：

| 检查项 | 检查内容 |
|---|---|
| 公式错误 | `#REF!`、`#VALUE!`、`#N/A`、`#DIV/0!`、`#NAME?` |
| 公式内硬编码 | `=A1*1.05` — `1.05` 应该是单元格引用 |
| 不一致的公式 | 打破行/列中相邻单元格模式的公式 |
| 偏移范围 | `SUM`/`AVERAGE` 遗漏第一行或最后一行 |
| 被覆盖的公式 | 看起来像公式但实际上是硬编码值的单元格 |
| 循环引用 | 有意或无意的 |
| 损坏的跨表链接 | 引用已移动或删除的单元格 |
| 单位/规模不匹配 | 千与百万混合，百分比存储为整数 |
| 隐藏行/标签 | 可能包含覆盖或过期计算 |

---

## 步骤3：模型完整性检查（仅限MODEL范围）

如果范围是**model**，识别模型类型（DCF / LBO / 三表联动 / 合并 / 可比公司 / 自定义）并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项 | 检查内容 |
|---|---|
| 输入/公式分离 | 输入是否与计算明确分离？ |
| 颜色约定 | 蓝色=输入，黑色=公式，绿色=链接 — 或模型使用的任何约定，是否一致应用？ |
| 标签流程 | 逻辑顺序（假设 → 损益表 → 资产负债表 → 现金流 → 估值）？ |
| 日期标题 | 所有标签一致？ |
| 单位 | 一致（千 vs 百万 vs 实际值）？ |

### 3b. 资产负债表

| 检查项 | 测试方法 |
|---|---|
| 资产负债表平衡 | 总资产 = 总负债 + 权益（每个期间） |
| 留存收益滚动 | 期初留存收益 + 净利润 − 股息 = 期末留存收益 |
| 商誉/无形资产 | 从收购假设流入（如果是并购） |

如果资产负债表不平衡，**量化每个期间的差距并追踪断点**——在此修复之前，其他一切都不重要。

### 3c. 现金流量表

| 检查项 | 测试方法 |
|---|---|
| 现金匹配 | 现金流量表期末现金 = 资产负债表现金（每个期间） |
| 现金流总和 | 经营现金流 + 投资现金流 + 筹资现金流 = 现金变动 |
| 折旧摊销匹配 | 现金流量表上的折旧摊销 = 损益表上的折旧摊销 |
| 资本支出匹配 | 现金流量表上的资本支出与资产负债表上的PP&E滚动一致 |
| 营运资金变化 | 符号与资产负债表变动匹配（应收账款变动、应付账款变动、存货变动） |

### 3d. 损益表

| 检查项 | 测试方法 |
|---|---|
| 收入构建 | 与部门/产品明细一致 |
| 税费 | 税费支出 = 税前收入 × 税率（允许递延税调整） |
| 股份数量 | 与稀释时间表一致（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是LBO/三表联动模型中常见的有意循环
- 如果是有意的：验证迭代切换存在并正常工作
- 如果是无意的：追踪循环并标记如何打破它

### 3f. 逻辑与合理性

| 检查项 | 标记条件 |
|---|---|
| 增长率 | 收入增长>100%而无解释 |
| 利润率 | 超出行业标准 |
| 终值主导 | 终值 > DCF企业价值的约75%（黄色警告） |
| 曲棍球棒效应 | 预测在后期不切实际地上升 |
| 复合增长 | EBITDA到第10年复合到荒谬的金额 |
| 边缘情况 | 模型在0%或负增长、负EBITDA、杠杆变为负数时崩溃 |

### 3g. 特定模型类型的错误

**DCF模型：**
- 折现率应用到错误期间（年中 vs 年末）
- 终值未折现
- WACC使用账面价值而非市场价值
- 自由现金流包含利息支出（应为无杠杆）
- 税盾重复计算

**LBO模型：**
- 债务偿还与现金扫尾机制不匹配
- PIK利息未计入本金
- 管理层滚动投资未反映在回报中
- 退出倍数应用到错误的EBITDA（LTM vs NTM）
- 费用未从第1天权益中扣除

**合并模型：**
- 摊薄/增厚使用错误的股份数量（交易前 vs 交易后）
- 协同效应未分阶段实施
- 购买价格分配不平衡
- 未包含现金的机会成本利息
- 交易费用未列入资金来源与用途

**三表联动模型：**
- 营运资金变化符号错误
- 折旧与PP&E时间表不匹配
- 债务到期时间表与本金偿还不匹配
- 股息超过净利润而无解释

## 步骤4：报告

输出发现表格：

| # | 工作表 | 单元格/范围 | 严重性 | 类别 | 问题 | 建议修复 |
|---|---|---|---|---|---|---|

**严重性：**
- **Critical（严重）** — 输出错误（资产负债表不平衡、公式损坏、现金不匹配）
- **Warning（警告）** — 风险（硬编码、不一致公式、边缘情况失败）
- **Info（信息）** — 风格/最佳实践（颜色编码、布局、命名）

对于**model**范围，添加摘要行：

> 模型类型：[DCF/LBO/三表联动/...] — 总体：[清洁 / 次要问题 / 主要问题] — [N]个严重，[N]个警告，[N]个信息

**未经询问不要更改任何内容** — 先报告，按需修复。

## 注意事项

- **资产负债表平衡优先** — 如果不平衡，下游一切都值得怀疑
- **硬编码覆盖是无声错误的头号来源** — 积极搜索
- **符号约定错误**（现金流出的正负号）极其常见
- 如果模型使用VBA宏，注意任何无法仅从公式审计的宏驱动计算

## 金融术语和知识解释

### 1. DCF模型 (Discounted Cash Flow Model)

DCF是一种估值方法，通过将未来现金流折现到当前价值来计算公司的内在价值。关键要素包括：
- **自由现金流(FCF)**：公司可自由支配的现金
- **折现率(WACC)**：加权平均资本成本
- **终值(TV)**：预测期后的公司价值

### 2. LBO模型 (Leveraged Buyout Model)

LBO是一种收购方式，收购方使用大量债务来融资收购。关键特点：
- **高杠杆**：债务占融资的大部分
- **现金扫尾(Cash Sweep)**：超额现金用于偿还债务
- **PIK利息(Payment-in-Kind)**：利息以增加本金的形式支付而非现金

### 3. 三表联动模型 (3-Statement Model)

将损益表、资产负债表和现金流量表相互连接的财务模型：
- **损益表** → 净利润流向资产负债表的留存收益
- **资产负债表** → 营运资金变动流向现金流量表
- **现金流量表** → 期末现金流向资产负债表

### 4. 资产负债表平衡 (BS Balance)

会计基本恒等式：**资产 = 负债 + 所有者权益**。如果不平衡，说明模型存在根本性错误。

### 5. 循环引用 (Circular Reference)

公式引用其自身单元格或依赖链最终回到自身的情况。在LBO和三表联动模型中有时是有意设计的（如利息→债务→现金→利息循环）。

### 6. WACC (Weighted Average Cost of Capital)

加权平均资本成本，计算公式：
WACC = (权益权重 × 权益成本) + (债务权重 × 债务成本 × (1 - 税率))

### 7. 终值 (Terminal Value)

DCF模型中预测期结束后的公司价值，通常采用永续增长法或退出倍数法计算。

### 8. 营运资金 (Working Capital)

营运资金 = 流动资产 - 流动负债，衡量公司日常运营所需的资金。

### 9. 协同效应 (Synergies)

并购后由于整合带来的额外价值，如成本节约、收入增加等。

### 10. 摊薄/增厚分析 (Accretion/Dilution Analysis)

衡量并购交易对每股收益(EPS)的影响：
- **Accretion（增厚）**：交易后EPS增加
- **Dilution（摊薄）**：交易后EPS减少
