# 审计电子表格

审计公式和数据以检查准确性和错误。范围决定深度 — 从选定区域的快速公式检查到完整的财务模型完整性审计。

## 步骤1：确定范围

如果用户已给出范围，使用它。否则**询问他们**：

> 你希望我审计什么范围？
> - **选定区域** — 仅当前选定范围
> - **工作表** — 仅当前活动工作表
> - **模型** — 整个工作簿，包括财务报表完整性检查（资产负债表平衡、现金结平、滚调表、逻辑合理性）

**模型**范围是最深入的 — 在发送DCF、LBO、三表、并购、可比分析或任何集成财务模型给客户或投资决策委员会前使用。

---

## 步骤2：公式级检查（所有范围）

无论范围如何，运行以下检查：

| 检查 | 要查找的内容 |
|------|--------------|
| 公式错误 | `#REF!`、`#VALUE!`、`#N/A`、`#DIV/0!`、`#NAME?` |
| 公式内硬编码 | `=A1*1.05` — `1.05`应该是单元格引用 |
| 不一致的公式 | 在一行/列中打破其邻居模式的公式 |
| 差一错误范围 | `SUM`/`AVERAGE`漏掉第一行或最后一行 |
| 粘贴覆盖的公式 | 看起来像公式但实际上是硬编码值的单元格 |
| 循环引用 | 有意或无意 |
| 断开的跨表链接 | 引用已移动或删除的单元格 |
| 单位/规模不匹配 | 千位与百万混合，%存储为整数 |
| 隐藏行/选项卡 | 可能包含覆盖或陈旧计算 |

## 步骤3：模型完整性检查（仅模型范围）

如果范围是**模型**，识别模型类型（DCF / LBO / 三表 / 并购 / 可比分析 / 自定义）并运行以下适当的完整性检查。

### 3a. 结构审查

| 检查 | 要查找的内容 |
|------|--------------|
| 输入/公式分离 | 输入是否与计算清楚分离？ |
| 颜色约定 | 蓝色=输入，黑色=公式，绿色=链接 — 或模型使用的任何约定，一致应用？ |
| 选项卡流 | 逻辑顺序（假设 → 损益表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期标题 | 所有选项卡一致？ |
| 单位 | 一致（千 vs. 百万 vs. 实际值）？ |

### 3b. 资产负债表

| 检查 | 测试 |
|------|------|
| 资产负债表平衡 | 总资产 = 总负债 + 权益（每个期间） |
| 留存收益滚调 | 前期RE + 净收入 − 股息 = 当期RE |
| 商誉/无形资产 | 从收购假设流动（如并购） |

如果资产负债表不平衡，**量化每个期间的差距并追溯哪里断开** — 在修复之前其他都不重要。

### 3c. 现金流量表

| 检查 | 测试 |
|------|------|
| 现金结平 | 现金流量表期末现金 = 资产负债表现金（每个期间） |
| 现金流量求和 | 经营现金流 + 投资现金流 + 融资现金流 = Δ现金 |
| 折旧摊销匹配 | 现金流量表上的D&A = 损益表上的D&A |
| 资本支出匹配 | 现金流量表上的资本支出与资产负债表上的PP&E滚调匹配 |
| 营运资本变化 | 符号匹配资产负债表变动（Δ应收账款、Δ应付账款、Δ存货） |

### 3d. 损益表

| 检查 | 测试 |
|------|------|
| 收入构建 | 与分部/产品明细挂钩 |
| 税收 | 所得税费用 = 税前收入 × 税率（允许递延税调整） |
| 股份数 | 与稀释计划挂钩（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是LBO/三表模型中常见的有意循环
- 如果有意：验证迭代开关存在且工作
- 如果无意：追溯循环并标记如何打破它

### 3f. 逻辑与合理性

| 检查 | 标记如果 |
|------|--------------|
| 增长率 | >100% 收入增长没有解释 |
| 利润率 | 超出行业规范 |
| 终值主导 | 终值 > 约DCF企业价值的75%（黄旗） |
| 曲棍球棒 | 预测在外延年度不切实际地上升 |
| 复利 | EBITDA到第十年复合到荒谬的美元 |
| 边缘案例 | 模型在0%或负增长、负EBITDA、杠杆变为负时崩溃 |

### 3g. 模型类型特定错误

**DCF模型：**
- WACC计算：权重总和应为1.0，无循环
- 终值：检查Gordon Growth方法 vs. 退出倍数方法
- 敏感性：所有敏感性表格都响应假设变化

**LBO模型：**
- 债务时间表：本金偿还 + 利息支付 = 债务减少
- IRR水柱：现金流按正确的优先级流水（偿债后股权）
- 股本回报：股权注入 + 后续投资者分红 = 股权价值

**三表模型：**
- 债务循环：利息支出 → 新债务 = 资产负债表平衡
- 营运资本：应收账款、存货、应付账款都随收入增长
- PP&E滚调：资本支出 − 折旧 = 净PP&E变化

**并购模型：**
- 收购会计：商誉计算 = 收购价格 − 可辨认净资产
- 协同效应：成本节约在损益表中正确体现
- 融资结构：债务 + 股权 = 收购价格 + 交易费用

**可比分析：**
- 倍数一致性：所有公司的EBITDA定义相同
- 统计：中位数/四分位正确计算
- 数据源：每个数字都有引用

## 步骤4：输出

### 快速检查（选定区域/工作表）
- 发现的错误列表，带单元格引用
- 每个修复建议

### 完整模型审计（模型范围）
- **问题摘要**，按严重程度排序（严重、中等、轻微）
- **每个问题的详细信息**：单元格引用、当前值、问题、修复建议
- **模型健康检查**：资产负债表平衡？现金结平？循环合理？
- **建议**：改进模型结构、文档、错误处理

### 输出格式
- **Excel注释**：对于快速检查，直接在单元格添加注释
- **Word/PDF报告**：对于完整审计，提供详细报告
- **修正模型**：如用户要求，提供修正版本

---

## 重要说明

- **资产负债表不平衡是阻断器** — 在修复之前不要继续其他检查
- **公式错误传播** — 一个`#REF!`可以破坏整个模型
- **硬编码数字是定时炸弹** — 当用户更改假设时，它们不会更新
- **颜色约定很重要** — 帮助用户快速区分输入和公式
- **测试边缘情况** — 用0%增长、负收入、最大债务测试模型
- **文档是救星** — 好的模型有假设说明、单元格注释、计算追踪

## Appendix: 金融背景知识

### 什么是模型审计（Model Audit）？

财务模型审计是系统化检查Excel财务模型的过程，确保：
- 公式正确且无错误
- 财务报表平衡（资产=负债+权益）
- 现金流计算正确
- 假设和逻辑合理
- 模型在实际场景中稳健

### 关键术语解释

**#REF!**：单元格引用无效，通常因为删除被引用的单元格或工作表。

**#VALUE!**：公式中的数据类型错误，如试图将文本与数字相加。

**#DIV/0!**：除以零错误，如`=B5/0`。

**#N/A**：值不可用，通常发生在VLOOKUP未找到匹配时。

**#NAME?**：Excel不认识公式中的文本，通常是拼写错误或缺少引号。

**资产负债表平衡**：总**资产**必须等于总**负债 + 股东权益**。这是会计等式的基础。

**现金结平（Cash Tie-out）**：现金流量表期末现金必须等于资产负债表上的现金余额。

**留存收益滚调（RE Rollforward）**：
```
期初留存收益
+ 本期净收入
- 支付的股息
= 期末留存收益
```

**PP&E**：不动产、厂房和设备（Property, Plant & Equipment），公司的固定资产。

**营运资本（Working Capital）**：
- **Δ应收账款**：销售增长时通常增加
- **Δ存货**：销售增长时通常增加
- **Δ应付账款**：采购增长时通常增加

**循环引用（Circular Reference）**：公式直接或间接引用自身。在LBO/三表模型中，利息支出 → 债务 → 现金 → 利息形成有意循环，需要启用迭代计算。

**WACC（加权平均资本成本）**：债务成本和股权成本的加权平均值，用作DCF中的折现率。

**终值（Terminal Value）**：预测期后的公司价值，通常使用：
- **Gordon Growth方法**：终值 = 最后一年的FCF × (1+g) / (WACC - g)
- **退出倍数方法**：终值 = 最后一年的EBITDA × 退出倍数

**敏感性分析（Sensitivity Analysis）**：测试关键变量（如WACC、增长率）变化对估值结果的影响。

**IRR（内部收益率）**：使项目净现值为零的折现率，衡量投资回报率。

**水柱图（Waterfall Chart）**：显示现金流如何从源头流向最终受益人的图表，常用于LBO中的现金流分配。

**协同效应（Synergies）**：并购后成本节约或收入增长，如减少重复部门、交叉销售。

**商誉（Goodwill）**：收购价格超过可辨认净资产公允价值的部分。

**并购会计（Acquisition Accounting）**：记录收购的会计方法，需要重新评估所有资产和负债至公允价值。
