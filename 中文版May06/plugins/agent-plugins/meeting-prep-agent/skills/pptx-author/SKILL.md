# PPTX 文档生成器

当以**无头模式**（托管代理/CMA 模式）运行时，且您需要将 PowerPoint 演示文稿作为**文件产物**交付，而非通过 `mcp__office__powerpoint_*` 编辑实时文档时，请使用此技能。

## 输出规范

- 写入 `./out/<name>.pptx`。如果 `./out/` 不存在，则创建它。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建演示文稿

编写一个简短的 Python 脚本，并用 Bash 运行。使用 `python-pptx` 库：

```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation("./templates/firm-template.pptx")  # 如果提供了模板
# 或者：prs = Presentation()

slide = prs.slides.add_slide(prs.slide_layouts[5])    # 仅标题布局
slide.shapes.title.text = "估值摘要"
# ... 添加表格/图表/文本框 ...

prs.save("./out/pitch-<target>.pptx")
```

## 约定（镜像实时 Office 的 `pitch-deck` 技能）

- **每张幻灯片一个观点**。标题陈述核心要点；正文提供支持。
- **每个数字都可追溯到模型**。如果某个数据来自 `./out/model.xlsx`，请在表名和单元格旁添加脚注。
- **使用公司模板**。如果 `./templates/` 下挂载了模板，则使用该模板；否则使用默认布局。
- **图表**：当保真度很重要时，优先嵌入从模型渲染的 PNG 图片，而非原生 pptx 图表。
- **不对外发送**。此技能只写入文件；从不发送电子邮件或上传。

## 何时不使用

如果 `mcp__office__powerpoint_*` 工具可用（Cowork 插件模式），请改用这些工具——它们以审核检查点驱动用户的实时文档。此技能是无头运行时生成文件的备用方案。

## 金融术语解释

### 1. PowerPoint 演示文稿 (PowerPoint Presentation)

**定义**：PowerPoint 是微软公司开发的演示文稿软件，用于创建包含文字、图表、图像和动画的幻灯片。在金融领域，PowerPoint 演示文稿是沟通和展示的核心工具。

**应用场景**：
- **路演演示**：向投资者展示公司价值主张和财务预测
- **投资推介**：向潜在买家展示标的公司亮点
- **季度报告**：向管理层和股东汇报业绩
- **培训材料**：用于员工培训和知识传递

**关键要素**：
- **结构清晰**：通常遵循"问题-解决方案-价值"的叙事结构
- **数据可视化**：使用图表直观展示财务数据
- **简洁有力**：每张幻灯片聚焦一个核心要点

### 2. 无头模式 (Headless Mode)

**定义**：指在没有图形用户界面（GUI）的环境中运行软件程序。在企业环境中，无头模式常用于自动化任务和后台服务。

**技术特点**：
- **无用户交互**：程序运行时不需要人工干预
- **服务器端执行**：通常部署在服务器上自动运行
- **资源高效**：不需要图形渲染，节省系统资源

**金融应用**：
- **自动化报告生成**：定期生成财务报告而无需人工操作
- **批量文档处理**：处理大量财务文档和数据文件
- **定时任务执行**：在特定时间自动运行财务计算和分析

### 3. 文档编排层 (Orchestration Layer)

**定义**：在软件架构中，编排层负责协调多个服务和组件，确保它们按照正确的顺序和逻辑执行。

**功能特点**：
- **工作流管理**：定义和执行复杂的业务流程
- **任务协调**：确保各组件按顺序完成工作
- **错误处理**：处理执行过程中的异常和错误
- **状态跟踪**：监控任务执行状态和进度

**金融场景**：
- **报告生成流程**：协调数据采集、分析和文档生成
- **交易处理**：管理交易执行的多个步骤
- **合规检查**：确保所有流程符合监管要求

### 4. 模板系统 (Template System)

**定义**：预先设计好格式和布局的文档框架，用于快速生成标准化的文档。

**优势**：
- **一致性**：确保所有输出文档风格统一
- **效率提升**：减少重复设计工作
- **品牌规范**：维护公司品牌形象
- **合规保证**：确保文档符合格式要求

**金融应用**：
- **财务报表模板**：标准化的利润表、资产负债表格式
- **投资报告模板**：统一的报告结构和样式
- **合同模板**：标准化的法律文档格式

### 5. 数据可追溯性 (Data Traceability)

**定义**：确保数据来源和计算过程可以被追踪和验证的能力。在金融领域，数据可追溯性是确保分析准确性和合规性的关键。

**重要性**：
- **审计要求**：监管机构要求金融机构能够追溯数据来源
- **错误排查**：便于发现和纠正数据错误
- **决策支持**：增强分析结果的可信度
- **合规合规**：满足行业监管要求

**最佳实践**：
- 记录每个数据点的来源
- 保存计算过程的中间结果
- 使用版本控制管理数据变更
- 建立数据血缘追踪系统
