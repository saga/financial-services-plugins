# pptx-author

当以**无头模式**运行（托管代理/CMA模式）且需要将PowerPoint演示文稿作为**文件产物**交付而不是通过`mcp__office__powerpoint_*`编辑实时文档时，使用此技能。

## 输出合同

- 写入`./out/<name>.pptx`。如果`./out/`不存在，则创建它。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建演示文稿

编写一个简短的Python脚本并用Bash运行它。使用`python-pptx`：

```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation("./templates/firm-template.pptx")  # 如果提供了模板
# 或: prs = Presentation()

slide = prs.slides.add_slide(prs.slide_layouts[5])    # 仅标题
slide.shapes.title.text = "估值汇总"
# ... 添加表格/图表/文本框 ...

prs.save("./out/pitch-<目标>.pptx")
```

## 约定（镜像实时Office的`pitch-deck`技能）

- **每张幻灯片一个想法。** 标题说明要点；正文支持它。
- **每个数字追溯到模型。** 如果数字来自`./out/model.xlsx`，则在幻灯片和单元格上添加脚注。
- **使用公司模板**（当模板挂载在`./templates/`时）；否则使用默认布局。
- **图表**：当保真度重要时，优先嵌入从模型渲染的PNG而非原生pptx图表。
- **无外部发送。** 此技能写入文件；从不发送电子邮件或上传。

## 何时不使用

如果`mcp__office__powerpoint_*`工具可用（同事插件模式），请改用那些 — 它们驱动用户的实时文档并带有审查检查点。此技能是无头运行的文件生产回退。

## 金融术语和知识详细解释

### 1. PowerPoint演示文稿（Pitch Deck）

**定义**：Pitch Deck是一组用于向投资者或利益相关者展示公司或投资机会的幻灯片。

**标准内容结构**：
| 部分 | 内容 |
|------|------|
| 封面 | 公司名称、logo、联系信息 |
| 问题/痛点 | 市场机会 |
| 解决方案 | 产品/服务 |
| 商业模式 | 如何赚钱 |
| 市场分析 | TAM、SAM、SOM |
| 竞争分析 | 差异化优势 |
| 团队 | 核心成员 |
| 财务 | 关键指标 |
| 融资需求 | 本轮融资、用途 |
| 退出策略 | 潜在退出路径 |

**为什么重要**：
- 融资必备工具
- 第一印象关键
- 清晰传达投资论点
- 投资决策依据

---

### 2. python-pptx库

**定义**：python-pptx是一个用于创建和修改PowerPoint演示文稿的Python库。

**基本功能**：
- 创建新演示文稿
- 添加幻灯片
- 设置文本和格式
- 插入表格和图表
- 保存为.pptx文件

**为什么重要**：
- 自动化演示文稿创建
- 与数据模型集成
- 批量生成报告
- 减少手动工作

---

### 3. 无头模式（Headless Mode）

**定义**：无头模式是指在没有图形用户界面的情况下运行程序。

**特点**：
- 无需打开应用程序
- 后台运行
- 自动化任务
- 服务器部署

**为什么重要**：
- 支持自动化工作流
- 可批量处理
- 无人值守运行
- 云端部署

---

### 4. 文件产物（File Artifact）

**定义**：文件产物是程序生成的输出文件，可以保存和分发。

**常见类型**：
- .pptx演示文稿
- .xlsx Excel模型
- .pdf报告
- .docx文档

**为什么重要**：
- 可保存和共享
- 独立于原始程序
- 便于审查
- 记录输出

---

### 5. 幻灯片布局（Slide Layout）

**定义**：幻灯片布局是预定义的幻灯片模板。

**常见布局类型**：
- 标题幻灯片
- 标题和内容
- 两栏内容
- 空白幻灯片
- 标题和副标题

**为什么重要**：
- 保持一致外观
- 加快创建速度
- 专业呈现
- 品牌统一
