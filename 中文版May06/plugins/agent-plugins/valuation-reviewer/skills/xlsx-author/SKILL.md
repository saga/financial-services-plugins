# xlsx-author

当运行**无头模式**（托管代理/CMA模式）且需要将Excel工作簿作为**文件制品**交付时，使用此技能，而不是通过`mcp__office__excel_*`编辑实时工作簿。

## 输出约定

- 写入`./out/<name>.xlsx`。如果`./out/`不存在则创建。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建工作簿

编写简短的Python脚本并通过Bash运行。使用`openpyxl`：

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active; ws.title = "Inputs"
ws["B2"] = "Revenue"; ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")           # 蓝色 = 硬编码输入
calc = wb.create_sheet("DCF")
calc["C5"] = "=Inputs!C2*(1+Inputs!C3)"        # 黑色 = 公式
wb.save("./out/model.xlsx")
```

## 约定（镜像`audit-xls`）

- **蓝色/黑色/绿色**。蓝色 = 硬编码输入，黑色 = 公式，绿色 = 链接到另一个工作表/文件。
- **计算单元格中无硬编码值**。每个计算单元格都是公式；每个输入都位于Inputs标签页上。
- **命名范围**用于从演示文稿或备忘录引用的任何值。
- **平衡检查**。包含一个Checks标签页，用于验证（资产负债表平衡、现金流与现金匹配等）并显示TRUE/FALSE。
- **每个文件一个模型**。除非明确要求，否则不要附加到现有工作簿。

## 何时不使用

如果`mcp__office__excel_*`工具可用（Cowork插件模式），请改用这些工具 — 它们驱动用户的实时工作簿并带有审查检查点。此技能是无头运行时的文件生成备选方案。

## 📚 金融术语和知识解释

### 1. 无头模式（Headless Mode）

**定义**：在没有图形用户界面（GUI）的环境中运行程序。

**应用场景**：
- 服务器端自动化
- 批量处理任务
- 云环境执行
- 托管代理服务

**优势**：
- 资源占用低
- 可扩展性强
- 无需人工干预
- 适合自动化流程

### 2. 文件制品（File Artifact）

**定义**：流程或任务产生的输出文件。

**类型**：
- Excel工作簿
- PDF文档
- CSV文件
- 演示文稿

**管理方式**：
- 版本控制
- 存储归档
- 元数据标记
- 访问控制

### 3. 编排层（Orchestration Layer）

**定义**：协调多个组件和任务的软件层。

**功能**：
- 任务调度
- 工作流管理
- 错误处理
- 资源分配
- 状态跟踪

**重要性**：
- 确保流程一致性
- 提高效率
- 便于监控和审计

### 4. openpyxl

**定义**：用于读写Excel 2010 xlsx/xlsm/xltx/xltm文件的Python库。

**主要功能**：
- 创建和修改工作簿
- 单元格操作

- 图表创建

**使用场景**：
- 自动化报告生成

- 批量操作

### 5. 财务模型最佳实践

**颜色编码标准**：
| 颜色 | 含义 | 用途 |
|------|------|------|
| **蓝色** | 硬编码输入 | 假设、参数 |
| **黑色** | 公式 | 计算、推导 |
| **绿色** | 外部链接 | 引用其他文件 |

**结构原则**：
- **分离输入与计算**：输入集中在Inputs标签页
- **透明性**：公式清晰可追溯
- **健壮性**：包含检查和验证
- **文档化**：注释说明假设和逻辑

### 6. 命名范围（Named Ranges）

**定义**：为单元格或单元格区域分配的描述性名称。

**好处**：
- 提高公式可读性
- 便于引用和维护
- 支持动态范围
- 减少错误

**示例**：
```
=SUM(MonthlyRevenue)
=VLOOKUP(ProductID, ProductList, 2, FALSE)
```

### 7. 模型检查（Model Checks）

**定义**：验证财务模型准确性和一致性的测试。

**常见检查**：
- **资产负债表平衡**：资产 = 负债 + 权益
- **现金流验证**：期末现金 = 期初现金 + 净现金流
- **计算交叉检查**：不同方法得出相同结果
- **边界检查**：值在合理范围内

**输出**：TRUE/FALSE结果，便于快速识别问题

### 8. DCF模型（贴现现金流模型）

**定义**：Discounted Cash Flow，通过折现未来现金流来评估资产价值的方法。

**步骤**：
1. 预测未来现金流
2. 确定折现率（WACC）
3. 计算现值
4. 计算终端价值
5. 汇总得到净现值

**公式**：
```
DCF价值 = Σ [CFt / (1 + WACC)^t] + 终端价值 / (1 + WACC)^n
```

**用途**：企业估值、投资决策、项目评估

### 9. WACC（加权平均资本成本）

**定义**：Weighted Average Cost of Capital，公司融资的平均成本。

**计算公式**：
```
WACC = (E/V) × Re + (D/V) × Rd × (1 - Tc)
```

其中：
- E = 股权价值
- D = 债务价值
- V = E + D
- Re = 股权成本
- Rd = 债务成本
- Tc = 税率

**用途**：作为DCF模型的折现率
