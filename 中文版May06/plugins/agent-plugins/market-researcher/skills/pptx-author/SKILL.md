# pptx-author

当以**无头模式**运行（托管代理/CMA模式）且需要将PowerPoint演示文稿作为**文件产物**交付而不是通过`mcp__office__powerpoint_*`编辑实时文档时，使用此技能。

## 输出合同

- 写入`./out/<name>.pptx`。如果`./out/`不存在，则创建它。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建演示文稿

编写一个简短的Python脚本并用Bash运行它。使用`python-pptx`：

```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation("./templates/firm-template.pptx")  # 如果提供了模板
# 或：prs = Presentation()

slide = prs.slides.add_slide(prs.slide_layouts[5])    # 仅标题
slide.shapes.title.text = "估值摘要"
# ... 添加表格/图表/文本框 ...

prs.save("./out/pitch-<target>.pptx")
```

## 约定（镜像实时Office `pitch-deck`技能）

- **每张幻灯片一个想法。** 标题陈述要点；正文支持它。
- **每个数字追溯到模型。** 如果数字来自`./out/model.xlsx`，在脚注中注明工作表和单元格。
- **使用公司模板**当模板挂载在`./templates/`时；否则使用默认布局。
- **图表**：当保真度重要时，优先嵌入从模型渲染的PNG而非原生pptx图表。
- **无外部发送。** 此技能写入文件；从不发送邮件或上传。

## 何时不使用

如果`mcp__office__powerpoint_*`工具可用（同事插件模式），请改用那些 — 它们驱动用户的实时文档并带有审查检查点。此技能是无头运行的文件生产回退。
