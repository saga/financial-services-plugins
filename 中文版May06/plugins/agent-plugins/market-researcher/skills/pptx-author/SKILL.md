# pptx-author

当运行在**无界面模式**（托管代理/CMA模式）时，且需要将PowerPoint演示文稿作为**文件产物**交付，而不是通过 `mcp__office__powerpoint_*` 编辑实时文档时，使用此技能。

## 输出约定

- 写入到 `./out/<name>.pptx`。如果 `./out/` 不存在则创建。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建演示文稿

编写一个简短的Python脚本并通过Bash运行。使用 `python-pptx`：

```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation("./templates/firm-template.pptx")  # 如果提供了模板
# 或者: prs = Presentation()

slide = prs.slides.add_slide(prs.slide_layouts[5])    # 仅标题布局
slide.shapes.title.text = "Valuation Summary"
# ... 添加表格 / 图表 / 文本框 ...

prs.save("./out/pitch-<target>.pptx")
```

## 约定（镜像实时Office `pitch-deck`技能）

- **每张幻灯片一个主题。** 标题陈述要点；正文提供支持。
- **每个数字都可追溯到模型。** 如果数字来自 `./out/model.xlsx`，在脚注中标明工作表和单元格。
- **使用公司模板**（当模板挂载在 `./templates/` 时）；否则使用默认布局。
- **图表**：当保真度很重要时，优先嵌入从模型渲染的PNG，而不是原生pptx图表。
- **不对外发送。** 此技能只写入文件；从不发送邮件或上传。

## 何时不使用

如果 `mcp__office__powerpoint_*` 工具可用（Cowork插件模式），请改用这些工具——它们通过审查检查点操作用户的实时文档。此技能是无界面运行时生成文件的备选方案。

## 金融术语解释

### 1. PowerPoint演示文稿（Pitch Deck）

**定义**：Pitch Deck是一种用于向投资者、客户或合作伙伴展示商业理念或投资机会的演示文稿。

**典型结构**：
- **封面页**：公司名称、logo、日期
- **执行摘要**：核心价值主张
- **市场机会**：市场规模、增长趋势
- **产品/服务**：解决方案描述
- **商业模式**：盈利方式
- **竞争分析**：竞争格局
- **财务预测**：收入、利润预测
- **融资需求**：资金用途
- **团队介绍**：核心团队背景

**设计原则**：
- 简洁明了，避免信息过载
- 视觉吸引力强
- 数据可视化（图表、图形）
- 清晰的逻辑流程

### 2. 估值摘要（Valuation Summary）

**定义**：估值摘要是对公司或资产价值的总结，通常包括多种估值方法的结果。

**常见估值方法**：
- **DCF（折现现金流）**：基于未来现金流的现值
- **可比公司分析**：与同类上市公司比较
- **交易可比性分析**：与同类并购交易比较
- **资产基础法**：基于资产价值

**关键输出**：
- 企业价值（Enterprise Value）
- 股权价值（Equity Value）
- 每股价值（Per Share Value）
- 估值区间（Valuation Range）

### 3. 演示文稿模板（Presentation Template）

**定义**：演示文稿模板是预定义的设计格式，确保演示文稿的一致性和专业性。

**模板组成**：
- **母版幻灯片**：统一的字体、颜色、布局
- **布局模板**：标题页、内容页、图表页等
- **主题颜色**：品牌色彩方案
- **字体样式**：标题字体、正文字体

**优点**：
- 提高制作效率
- 保持品牌一致性
- 确保专业外观

### 4. 数据可视化（Data Visualization）

**定义**：数据可视化是将数据转换为图形或图表的过程，便于理解和分析。

**常见图表类型**：
- **柱状图**：比较不同类别数据
- **折线图**：显示趋势变化
- **饼图**：展示比例关系
- **散点图**：显示变量关系
- **瀑布图**：展示累计变化

**设计原则**：
- 清晰标注坐标轴
- 使用适当的刻度
- 添加数据标签
- 保持简洁

### 5. 图表嵌入（Chart Embedding）

**定义**：图表嵌入是将外部生成的图表插入到演示文稿中的过程。

**方法**：
- **PNG嵌入**：将图表保存为图片后插入
- **原生图表**：使用PowerPoint内置图表功能
- **链接图表**：保持与源文件的动态链接

**选择因素**：
- 保真度要求
- 更新频率
- 文件大小限制
- 编辑需求

### 6. 企业价值（Enterprise Value, EV）

**定义**：企业价值是衡量公司总价值的指标，考虑了股权和债务。

**计算公式**：
```
EV = 市值 + 债务 - 现金及等价物
```

**组成部分**：
- **市值**：普通股的市场价值
- **债务**：短期和长期债务
- **现金**：现金及现金等价物

**用途**：
- 公司估值
- 并购交易定价
- 财务分析

### 7. 财务预测（Financial Projections）

**定义**：财务预测是对未来财务表现的估计，通常包括收入、利润和现金流。

**组成部分**：
- 收入预测
- 成本预测
- 利润预测
- 现金流预测

**方法**：
- 自上而下方法
- 自下而上方法
- 基于市场的方法

**重要假设**：
- 增长率
- 利润率
- 资本支出

### 8. 融资需求（Funding Requirements）

**定义**：融资需求是指公司为实现其目标所需的资金数额。

**资金用途**：
- 产品开发
- 市场推广
- 团队扩张
- 运营资金
- 收购

**融资类型**：
- 股权融资
- 债务融资
- 混合融资

**演示要点**：
- 所需金额
- 资金用途明细
- 预期回报

### 9. 可比公司分析（Comparable Company Analysis）

**定义**：可比公司分析是通过比较类似上市公司来评估目标公司价值的方法。

**步骤**：
1. 选择可比公司
2. 收集财务数据
3. 计算估值倍数
4. 应用倍数到目标公司

**常用倍数**：
- EV/Revenue
- EV/EBITDA
- P/E

**注意事项**：
- 确保可比性
- 考虑行业差异
- 调整非经常性项目

### 10. 交易可比性分析（Transaction Comparables）

**定义**：交易可比性分析是通过比较类似并购交易来评估目标公司价值的方法。

**步骤**：
1. 识别类似交易
2. 收集交易数据
3. 计算交易倍数
4. 应用倍数到目标公司

**优势**：
- 反映市场实际支付价格
- 考虑协同效应
- 适合并购场景

**局限性**：
- 交易数据可能有限
- 交易条件差异大
- 时间敏感性
