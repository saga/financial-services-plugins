# xlsx-author

当以**无头模式**运行（托管代理/CMA模式）且需要将Excel工作簿作为**文件产物**交付而不是通过`mcp__office__excel_*`编辑实时工作簿时，使用此技能。

## 输出合同

- 写入`./out/<name>.xlsx`。如果`./out/`不存在，则创建它。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建工作簿

编写一个简短的Python脚本并用Bash运行它。使用`openpyxl`：

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active; ws.title = "输入"
ws["B2"] = "收入"; ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")           # 蓝色 = 硬编码输入
calc = wb.create_sheet("DCF")
calc["C5"] = "=输入!C2*(1+输入!C3)"        # 黑色 = 公式
wb.save("./out/model.xlsx")
```

## 约定（镜像`audit-xls`）

- **蓝色/黑色/绿色。** 蓝色 = 硬编码输入，黑色 = 公式，绿色 = 链接到另一个工作表/文件。
- **计算单元格中无硬编码。** 每个计算单元格都是一个公式；每个输入都放在"输入"标签上。
- **命名范围**用于从演示文稿或备忘录引用的任何值。
- **平衡检查。** 包括一个"检查"标签，链接（BS平衡、CF链接到现金等）并显示TRUE/FALSE。
- **每个文件一个模型。** 除非明确要求，否则不要追加到现有工作簿。

## 何时不使用

如果`mcp__office__excel_*`工具可用（同事插件模式），请改用那些 — 它们驱动用户的实时工作簿并带有审查检查点。此技能是无头运行的文件生产回退。

### 2. openpyxl库

**定义**：openpyxl是一个用于读取和写入Excel .xlsx文件的Python库。

**核心功能**：
- 创建新工作簿
- 读取/写入单元格
- 设置样式和格式
- 创建图表
- 处理多个工作表

**为什么重要**：
- 自动化Excel报告生成
- 与数据模型集成
- 批量处理电子表格
- 无需安装Excel

---

### 3. 工作簿结构（Workbook Structure）

**定义**：Excel工作簿是包含一个或多个工作表的文件。

**组件**：
| 组件 | 描述 |
|------|------|
| 工作表 | 单个数据页面 |
| 单元格 | 行和列交叉的点 |
| 命名范围 | 单元格的命名引用 |
| 样式 | 字体、颜色、边框等 |

**为什么重要**：
- 组织财务数据
- 支持综合模型
- 便于数据引用
- 标准文件格式

---

### 4. 命名范围（Named Range）

**定义**：命名范围是为单元格或区域分配的可读名称。

**示例**：
```
总收入 = Sheet1!$C$2
审计调整 = Sheet1!$B$5
重大错报阈值 = Sheet1!$B$6
```

**为什么重要**：
- 使公式更易读
- 便于跨表引用
- 支持动态模型
- 行业最佳实践

---

### 5. 工作表标签（Worksheet）

**定义**：工作表是工作簿中的单个页面。

**常见命名**：
| 标签 | 内容 |
|------|------|
| 输入 | 假设和驱动因素 |
| IS/P&L | 利润表 |
| BS | 资产负债表 |
| CF | 现金流量表 |
| 调整 | 审计调整 |
| 检查 | 验证和平衡检查 |

**为什么重要**：
- 组织模型逻辑
- 分离关注点
- 便于维护
- 专业呈现

---

### 6. 字体颜色编码

**定义**：在Excel中用颜色表示单元格类型。

**标准**：
| 颜色 | 单元格类型 |
|------|-----------|
| 蓝色 | 硬编码输入 |
| 黑色 | 公式 |
| 绿色 | 跨表链接 |
| 紫色 | 同表链接 |

**为什么重要**：
- 快速识别单元格用途
- 防止意外覆盖
- 加速模型审查
- 行业标准

---

**定义**：文件产物是程序生成的输出文件。

**优势**：

---

### 9. 平衡检查（Balance Check）

**定义**：验证财务模型正确性的测试。

**核心检查**：
| 检查 | 公式 | 预期 |
|------|------|------|
| BS平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金核对 | CF期末现金 - BS现金 | = 0 |
| RE结转 | 期初 + NI - 股息 - 期末 | = 0 |

**为什么重要**：
- 确保模型准确
- 防止错误决策
- 审计标准要求
- 专业标准

---
