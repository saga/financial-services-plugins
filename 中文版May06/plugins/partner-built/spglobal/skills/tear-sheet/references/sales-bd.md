# 销售 / 业务拓展概览页面

## 用途
为销售或业务拓展团队准备客户会议而制作的简报文档。侧重了解潜在客户的业务、领导层公开关注的事项，以及将您的产品或服务与他们的需求连接起来的切入点。财务数据简化为规模和趋势。

这是叙事性最强的概览页类型。读起来像简报备忘录，而非数据表。

**默认页数：** 1-2 页。无严格惯例——可读性优先于紧凑性。

## 查询计划

从以下查询开始。如结果不完整，跟进更细的查询。

**查询 1 — 公司简介 + 财务数据：**
"[公司] 业务描述 概况 产品 服务 总部 员工 行业 收入 毛利率 最近2个财年"
→ 标题、公司概述、财务快照
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`
→ **立即写入** `/tmp/tear-sheet/financials.csv`

**查询 2 — 战略 + 财报：**
"[公司] 最近一次财报电话会 战略重点 CEO 评论 指引 关键举措"
→ 战略重点
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询 3 — 关系 + 新闻：**
"[公司] 关键客户 供应商 合作伙伴 竞争对手 业务关系 近期收购 合作伙伴关系 公告"
→ 关键关系、近期新闻
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

三个查询通常足够。对于知名公司，这些返回丰富的结果。对于较小或私人公司，查询 2 和 3 可能较稀疏——这没问题。定性章节仍然有价值，因为 Claude 会将任何可用数据综合为有用的框架。

## 章节

按优先级排序。如空间受限，从底部开始裁剪。

### 1. 公司标题
简化版——仅识别信息，无需估值指标。

| 字段 | 说明 |
|---|---|
| 公司名称 | 大号加粗 |
| 股票代码和交易所 | 如上市；未上市则省略 |
| 总部 | 城市、州/国家 |
| 行业 | |
| 员工 | |
| 年收入（最新） | 按最近十亿或百万取整 |
| 市值 | 如上市；未上市则省略 |
| 财年收入指引 | 如财报数据可用；例如"6-8%有机增长" |
| 财年 EPS 或 EPS 指引 | 如上市公司；从 `earnings.txt` 提取 |
| 关键财务指标 | 调整后 EPS、EBITDA 利润率或类似——一条标题数字 |

前瞻指引和 EPS 数据来自查询 2（财报）——填充标题时从 `earnings.txt` 引用，而不是仅从 `company-profile.txt`。

顶部一条水平带。无企业价值、无Beta——这个受众不需要它们。

### 2. 公司概述
最重要的部分。销售代表阅读后应能在会议中对该潜在客户有准确把握。

**不要直接粘贴 CIQ 公司摘要。** 它读起来像 SEC 文件，会失去非金融读者的注意力。以 3-5 句通俗语言重写：
- 公司做什么（不用投资术语——用"卖"而不是"变现"，用"年收入"而不是"复合年增长率"）
- 他们的客户是谁
- 他们如何赚钱
- 在哪些地方运营
- 他们与众不同之处

使用散文段落。非金融专业人士应能理解每句话。

### 3. 战略重点与高管评论
领导层现在关注什么？如果您知道 CEO 在谈什么，就找到了切入口。

**这不是财报回顾。** 不要重用与股权研究概览页相同的要点。提取 3-5 个**战略主题**，附主题标签和支持数据点：

- **AI/数据变现：** 管理层正在大力投资 AI 能力，在 FY2024 投入 X 于研发，推出了三款新的 AI 驱动产品。
- **利润率扩张：** CEO 将运营效率列为首要任务，目标通过自动化实现 200bps 的营业利润率提升。
- **国际增长：** 在 EMEA 开设新办公室；欧洲现占收入的 25%，高于之前的 18%。

每条要点应回答："这家公司现在关心什么，有什么证据支撑？"销售代表应从中立即看到连接自己产品的切入点。

如财报电话会数据不可用（私人公司常见），以"近期发展"替代，从工具返回的任何新闻或公告中提取信息。

### 4. 财务快照
简化版。单个紧凑表格——仅需规模、方向和一个盈利信号。销售代表需要知道三件事：这家公司多大、在增长吗、健康吗。

| 指标 | FY[前一年] | FY[最新] | 同比变化 |
|---|---|---|---|
| 收入 | | | +X% |
| 毛利率% | | | +X.Xpp |
| 员工 | | | |

最多三行。无 EBITDA、无 EV 倍数、无资产负债表、无净利润。毛利（而非 EBITDA 或净利润）是销售受众的合适单一盈利指标——它直观（"扣掉直接成本后剩多少"），无需金融知识也能说明业务健康状况。如果代表需要更深入的财务信息，可以要求股权研究版本。

对于私人公司，此表可能只有收入和员工——两项显示规模和增长的数据点对销售会议已足够。

### 5. 关键关系与生态
帮助代表了解潜在客户的世界并找到切入点。

四个标签列表，每类 3-5 个名称，附带一行注释：

**客户：** 他们卖给谁。帮助了解其市场策略和共享目标客户。
示例："JPMorgan Chase — 客户（企业分析平台部署）"

**供应商 / 技术：** 他们买什么。发现替代或整合机会。
示例："AWS — 供应商（主要云基础设施）；Snowflake — 供应商（数据仓库）"

**合作伙伴：** 他们与谁合作。联合销售角度、共同联系人。
示例："Deloitte — 合作伙伴（企业客户实施）"

**竞争对手：** 他们与谁竞争。定位背景。
示例："Bloomberg — 竞争对手（金融数据终端）"

每条需要一个括号注释，让代表了解**关系意味着什么**，而不仅仅是**是谁**。

如关系数据不可用，注明"关系数据不可用"并继续。

### 6. 近期新闻与发展
3-5 条要点，每条一句话，最新在前。**以最近一期财报为首**——这是销售会议中最重要的近期事件，定下所有其他内容的基调。然后：并购、产品发布、领导层变更、重大赢单。

**在节末包含即将到来的催化剂**，用"**即将到来：**"标签。示例：即将到来的财报日期、待定剥离、产品发布、投资者日、监管里程碑。这些创造了自然的销售接触窗口，通常是联系的最佳理由（"我注意到贵公司Q2有投资者日……"）。从财报电话会评论和并购数据中提取（待定交易）。

### 7. 对话切入点（首先裁剪）
综合上述数据提出的 2-3 条建议谈话要点。**每条切入点必须引用第 3 节中的特定战略重点**——通用问题没用。以问题形式呈现。

**给每条切入点标记建议角色**——例如（CTO）、（CFO）、（CDO）、（数据主管）、（工程VP）、（采购主管）。与 CTO 会面的代表需要与 CFO 不同的开场白。如用户指定了会面对象，仅为该角色生成切入点。

示例：
- *(CTO/CDO)* "您在最近一期财报中提到了[第 3 节中的具体 AI 计划]——这对您的[相关需求]有什么影响？"
- *(CFO/COO)* "鉴于您的[第 3 节中的利润率扩张目标]，您是否在评估能加速实现该目标的工具？"
- *(战略VP/企发)* "我看到[公司]最近收购了[目标]——这是否改变了您对[能力]的策略？"

如用户描述了他们在销售什么产品/服务，将这些切入点与该产品连接。如未描述，保持切入点在潜在客户的战略主题中，但解决方案框架保持通用。

**验证测试：** 最终确定每条对话切入点前，确认它包含至少一个来自概览页的具体数字、日期、产品名或计划名称。一个适用于该行业任何公司的问题不是好的对话切入点。"您如何看待AI？"太泛。而"您的 CEO 提到自 2018 年以来投入了 10 亿美元在 AI 上，20% 的 iLEVEL 自动导入功能已被采纳——这对您的分析师工作流有何改变？"既具体又展现了准备充分。

本节是 Claude 的综合，而非来自工具的原始数据。清晰标记。

## 格式说明
- **这应是最温暖、最可读的概览页类型。** 内容严谨，但格式应像简报备忘录，而非数据终端。
- 正文：使用完整 9pt（size: 18）。不压缩——可读性优先于密度。
- 行间距：正文段落 1.15 倍（略高于默认）。此模板中空白是特色而非浪费。
- 公司概述和战略重点应占约 40-50% 的页面。
- 财务快照：视觉紧凑，次要地位。保持小巧干净。
- 关键关系：考虑两栏布局（客户 + 供应商在左，合作伙伴 + 竞争对手在右）以节省垂直空间同时保持注释可见。
- 对话切入点：使用缩进块级项目符号。这些是文档的回报——应在视觉上突出显示。

## 📖 本篇金融知识小贴士

**销售代表的财务知识**：销售不需要了解 EBITDA 或 DCF。他们需要知道公司规模（收入）、是否增长（增长率）和基本赚钱能力（毛利率）。这就是本节仅关注这三项的原因。

**年度经常性收入（ARR，Annual Recurring Revenue）**：SaaS 公司最重要的指标——每年从订阅客户那里获得的经常性收入。比总收入更能反映业务健康状况。

**毛利率（Gross Margin）**：（收入 - 直接成本）÷ 收入。衡量"每赚1块钱能留下多少"。高毛利（>70%）通常意味着产品有定价权和护城河。

**客户获取成本（CAC，Customer Acquisition Cost）**：获得一个新客户花了多少钱（销售 + 市场费用 ÷ 新增客户数）。与 LTV（客户生命周期价值）结合使用。

**近因效应（Recency Effect）**：人们更容易记住最近发生的事情。在销售中，最近的公司新闻（尤其财报）是最好的谈话切入点。

**话题切入点（Conversation Starter）**：基于对潜在客户的研究而设计的初始话题。好的切入点显示你做了功课，并直接切入对方关心的事情。

**理想客户画像（ICP，Ideal Customer Profile）**：最适合你产品的客户类型的详细描述。销售团队用 ICP 来优先选择目标客户。

**年度合同价值（ACV，Annual Contract Value）**：每个客户合同每年的平均价值。高 ACV 客户通常值得投入更多销售精力。
