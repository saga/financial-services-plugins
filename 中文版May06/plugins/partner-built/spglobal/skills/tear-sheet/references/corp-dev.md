# 企业发展概览页面

## 用途
供内部企业发展团队评估潜在收购目标的评估资料。与 IB/并购概览（讲一个叙事性"故事"做推介）不同，这是一份侧重战略契合度的分析文档：目标公司的产品、客户、技术和运营如何与收购方重叠或互补。企业发展团队已经了解自己公司——他们需要从"我们该不该买这个，整合会是什么样"的角度理解目标公司。

**默认页数：** 1-2 页。企业发展团队可以接受较密集的文档，如果内容充实，第二页也没问题。

## 查询计划

从以下查询开始。如结果不完整，拆分为更细的查询。

**查询 1 — 目标简介：**
"[公司] 业务描述 产品 服务 技术平台 总部 成立年份 员工 行业"
→ 标题、业务和产品概述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询 2 — 财务数据：**
"[公司] 年度利润表 收入 毛利润 EBITDA 营业利润 净利润 资本支出 自由现金流 研发费用 总债务 现金及等价物 最近4个财年"
→ 财务摘要（拉取4年，展示3年；最早一年仅用于同比增速计算。研发费用对企发团队尤其重要——他们想了解目标的IP投入）
→ **立即写入** `/tmp/tear-sheet/financials.csv`

**查询 3 — 细分 + 客户：**
"[公司] 收入按业务分部 最近2个财年 关键客户 终端市场 客户集中度"
→ 收入结构（需2年数据用于同比增速）、客户分析
→ **立即写入** `/tmp/tear-sheet/segments.csv`（无分部数据则跳过）

**查询 4 — 关系 + 竞争格局：**
"[公司] 关键客户 供应商 合作伙伴 竞争对手 业务关系 技术供应商"
→ 战略契合度分析、生态图谱
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

**查询 5 — 估值 + 同行：**
如用户提供了具体可比公司：
"[同一家公司] 企业价值 EV/收入 EV/EBITDA 收入增长 EBITDA利润率"
否则：
使用竞争对手工具识别 3-5 家上市同行，然后拉取每家的 EV/收入（NTM）、EV/EBITDA（NTM）、收入增长%和EBITDA利润率%。
同时："[公司] 企业价值 市值 估值倍数"
→ 估值背景
→ **立即写入** `/tmp/tear-sheet/valuation.csv`
→ **立即写入** `/tmp/tear-sheet/peer-comps.csv`

**查询 6 — 股权结构（数据许可）：**
"[公司] 股权结构 投资者 机构持仓 内部人持股"
→ 股权结构（无中间文件——如有则写入 company-profile.txt）

## 章节

按优先级排序。详见下方"页数预算与裁剪顺序"。

### 1. 公司标题
与 IB/并购类似，但增加"战略相关性"一句话简介。

| 字段 | 说明 |
|---|---|
| 公司名称 | 大号加粗 |
| 股票代码和交易所 | 如上市；未上市则省略 |
| 总部地点 | |
| 成立 / 员工 | |
| 企业价值 | 上市为市值；未上市为最近一轮估值 |
| 股权结构 | 上市 / PE背书 / 创始人主导 / 等 |
| **战略相关性** | 一句话：为什么这个目标对收购方有吸引力（从数据中综合得出） |

"战略相关性"一行是企发展独有。以中立分析视角撰写，而非来自目标的营销语言。说明收购方**获得**了什么——而非目标**是什么**。好的："在大宗商品价格基准中占据主导地位，拥有监管护城河和70%+的毛利率——轻资产、高壁垒的数据资产。"坏的："全球领先的金融数据和分析服务提供商。"前者告诉企发团队为什么有吸引力；后者是新闻稿。

### 2. 业务和产品概述
**不要直接粘贴 CIQ 公司摘要。** 以 4-6 句话（一段，不是两段）重写，侧重产品和技术的视角——这个受众评估的是收购，不是投资：
- 核心产品/服务及其解决的问题
- 技术或平台架构（高层）
- 关键差异点和竞争护城河
- 客户细分——谁在买，为什么
- 当前的战略方向

以收购方的视角撰写。读者在想："这怎么和我们已有的东西契合？"描述应便于发现重叠和互补之处。

### 3. 收入结构与客户分析
两个子部分：

**a) 按分部划分的收入**（如可用）：

| 分部 | 收入（百万美元） | 占总收入% | 同比增速 |
|---|---|---|---|

**b) 客户分析：**
在分部表下方（或若无分部数据则代替），写一段定性描述：
- 客户集中度：前5大客户是否 >30%收入？（对收购风险至关重要）
- 客户类型：企业 vs 中小企业 vs 消费者
- 合同结构：订阅 vs 交易型 vs 许可证（如已知）
- 地理分布

客户集中度是最重要的信号——如果工具返回任何关于大客户或收入集中的数据，突出显示。如果数据稀疏，从业务描述和关系中推断。2-3句话即可——这是分部表的背景，不是独立分析。

### 4. 战略契合度分析
**这是企业发展概览的核心章节。** 必需，不可跳过或压缩。使用 S&P Capital IQ 的业务关系数据来映射重叠和互补。

分为三个桶。**每个桶必须有 2-3 句分析推理，而不是公司名称列表。** 列出"客户：Microsoft、JPMorgan、Google"而没有背景是没用的。相反，要解释重叠或互补对收购的**意义**。

**客户重叠：** 目标的客户是否也购买收购方（或收购方行业）的产品？共享客户 = 更容易交叉销售，但也存在渠道冲突风险。指出共享关系并解释影响。

**技术/产品互补：** 目标是否填补了收购方产品组合中的空白？他们的技术栈是什么样的？技术供应商有哪些？如果他们已经在使用收购方的技术，解释为什么这是强整合信号。

**竞争替代：** 目标是否是竞争对手？收购他们是否会消除市场上的一个竞争对手？指出目标的主要竞争对手并解释收购后竞争格局的变化。

此部分需要综合——将关系数据与业务概述联系起来以得出结论。如果用户告诉了您收购方是谁，针对该收购方定制。如果没有，撰写通用但仍具分析性的内容。

**长度控制：** 以上每桶 2-3 句是不可妥协的约束，而非建议。每条要点的洞察应前置，然后用一个具体数据点支撑。实际输出中，此部分经常膨胀到每桶 5-7 句——这是导致第三页溢出的主要原因。如果发现自己在写超过 3 句，将多余细节移到"整合考量"部分。

| 指标（百万美元） | FY20XX | FY20XX | FY20XX |
|---|---|---|---|
| 收入 | | | |
| 收入增长率% | | | |
| 毛利率% | | | |
| EBITDA | | | |
| EBITDA利润率% | | | |
| 研发费用 | | | |
| 研发占总收入% | | | |
| 资本支出 | | | |
| 自由现金流 | | | |
| FCF转化率（FCF/EBITDA） | | | |

研发强度表明目标的市值有多少来自知识产权 vs 服务。FCF转化率表明整合的复杂度——高资本支出业务更难整合。如果研发费用未单独列报，在行中写"未单独披露"而不是"N/A"——这本身就有信息量（说明研发可能计入 COGS 或 SG&A）。

**资本结构**（财务摘要下的紧凑子表）：

| 指标 | 值 |
|---|---|
| 总债务 | XX百万美元 |
| 现金及等价物 | XX百万美元 |
| 净债务 | XX百万美元 |
| 净债务 / EBITDA | X.X倍 |

### 6. 估值背景
**上市公司此部分必需。** 企业发展团队需要市场背景来判断潜在报价。如果工具返回估值倍数，此部分必须出现。不要为了省空间裁剪它——先裁剪股权结构（第7节）。

**a) 交易倍数**（如上市）：

**同行背景不可或缺，而非可有可无。** 如用户提供了可比公司，每家单独一列。如未提供，使用竞争对手工具识别 3-5 家上市同行，然后拉取 EV/收入（NTM）、EV/EBITDA（NTM）、收入增长%和 EBITDA利润率%。

| 指标 | [公司] | [同行1] | [同行2] | [同行3] | 同行中位数 |
|---|---|---|---|---|---|
| EV/收入（NTM） | | | | | |
| EV/EBITDA（NTM） | | | | | |
| 收入增长率% | | | | | |
| EBITDA利润率% | | | | | |

孤立看一家公司的倍数对企发团队毫无信息量——他们需要相对背景来判断目标是贵还是便宜。这是目标评估的最小可行估值背景。

如同行倍数不可用，显示公司自身倍数并在表下列出同行名称供参考。

**b) 可比交易（数据许可）：**
如果并购工具返回涉及同行业公司或同子行业的近期交易，显示它们：

| 日期 | 目标 | 收购方 | EV（百万美元） | EV/收入 | EV/EBITDA |
|---|---|---|---|---|---|

作为背景说明："[子行业]近期交易估值在 X-Y 倍收入。"

如可比交易数据不可用，写"可比交易数据来自数据源不可用"而非静默省略。不要编造可比倍数。

私人公司跳过交易倍数。可比交易（数据许可）仍有价值。

### 7. 股权结构（数据许可——工具无返回则省略）
如工具返回股权数据，包含紧凑的块：创始人/家族控制%、PE 赞助商详情、机构集中度。股权结构直接影响交易复杂度，对整合考量相关。

**不要包含管理团队表。** 没有 S&P Global 工具返回高管数据——见数据完整性规则第10条。训练数据中的管理层姓名会过时。

### 8. 整合考量
**必需章节——不要裁剪。** 这是企业发展概览的分析顶峰。综合自以上所有内容的观察结论。3-4 条实质性要点：
- **关键风险：** 客户集中度、关键人依赖、技术债指标
- **整合复杂度：** 地理位置（多国 = 更难）、员工人数、产品数量
- **协同信号：** 共享客户、互补产品、重叠的技术供应商
- **待探索问题：** 企业发展团队在出价前需要尽调什么？聚焦于具体、可操作的尽调项——而非开放式的战略问题。好的示例：监管批准时间表和司法管辖区、关键合同的控制权变更条款、关键人留任风险（尤其创始人/CTO）、客户合同可转让性、IP 所有权结构、未决诉讼。坏的示例："X有多强的护城河？"或"长期市场机会有多大？"——这些是分析师的问题，不是尽调项。

明确标注为分析性观察，而非来自数据源的事实。

## 页数预算与裁剪顺序

**目标：2页。** 企业发展文档可以充实写满两页，但必须不溢到第三页，除非用户明确要求更长格式。如果内容超过 2 页，按以下顺序裁剪（每部分完全裁剪完再进入下一个）：

1. 股权结构（第7节）— 完全省略
2. 客户分析段落（第3节下）— 缩减至最多 2 句
3. 估值背景段落（第6节）— 保留同行对比表，裁剪下方的解释段落
4. 可比交易（第6b节）— 保留"不可用"注释，空间紧张时删除
5. 资本结构段落 — 保留表格，裁剪下方的解释段落
6. 战略契合度分析（第4节）— 将每条要点压缩至最多 2-3 句（不要删除整个章节）

永远不裁剪：业务和产品概述（2）、财务摘要（5 核心表）、整合考量（8）。

## 格式说明
- **战略契合度分析是核心。** 给它与众不同的格式：
  - 文字升为 9.5pt（size: 19）— 略大于标准正文。本节应看起来像页面上最重要的内容。
  - 使用全局样式配置中的标准缩进块级项目符号（360 DXA 左缩进）。
  - 不要使用左边框强调——它们在 docx-js 中渲染不一致。
- 财务摘要应突出显示研发和 FCF 转化率。
- 上市公司的估值背景（含同行对比）为必选项——裁剪顺序中应在股权结构之前。
- 私人目标的文档侧重定性章节（业务概述、战略契合、关系）——这符合预期且同样有价值。
- 语气：分析性和评估性，而非推广性。这不是推介——是评估。
- **两页纪律：** 渲染前，预估页数。战略契合（3条长要点）加整合考量（4条要点）本身就能占一页——如果是这样，先压缩战略契合的每条要点。

## 📖 本篇金融知识小贴士

**企业价值（EV, Enterprise Value）**：收购一家公司真需要花多少钱。市值 + 总负债 - 现金。比市值更完整，因为收购方要承担目标公司的债务。

**LBO（杠杆收购，Leveraged Buyout）**：借大量钱（杠杆）买公司，然后用被买公司的现金流还债。PE（私募股权）机构最爱的方式。

**FCF转化率**：自由现金流 ÷ EBITDA。衡量"赚到的钱有多少真正落袋"。越高越好——说明公司不需要大量再投资。

**研发强度（R&D Intensity）**：研发费用 ÷ 收入。越高说明公司是技术驱动型（如科技公司通常在15-20%），越低说明偏服务或制造型。

**协同（Synergy）**：1+1>2 的效果。收购后合并能节省成本或增加收入的部分。收购方愿意为协同效应支付溢价。

**反垄断（Antitrust）**：监管机构防止某家公司垄断市场的审查。大额收购可能需要反垄断批准，耗时 6-18 个月。

**客户集中度（Customer Concentration）**：前 N 大客户占总收入的比例。超过 30% 就是"集中"——失去一个大客户对公司是重大打击。

**管理层留任风险（Key Person Retention Risk）**：创始人或核心高管在收购后是否会离职。对人的收购中（如科技公司），这是最大的风险之一。
