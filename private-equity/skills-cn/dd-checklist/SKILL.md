# 尽职调查清单

生成并跟踪全面的尽职调查清单。

## 工作流

### 步骤1：确定尽调范围

向用户询问：
- **目标公司**：名称、行业、商业模式
- **交易类型**：平台收购、增补型、成长型、资本重组、剥离
- **交易规模/复杂度**：决定尽调深度
- **关键关注点**：任何已知的需优先处理的问题（客户集中度、监管、环境等）
- **时间表**：目标签署意向书/交割的时间？

### 步骤2：生成工作流清单

生成涵盖所有主要工作流的清单，针对行业定制：

**财务尽调**
- 盈利质量（QoE）——收入和EBITDA调整
- 营运资本分析——标准化vs实际
- 债务及债务类项目
- 资本支出（维护vs增长）
- 税务结构和敞口
- 审计历史和会计政策
- 备考调整（运行率、协同效应）

**商业尽调**
- 市场规模和增长（TAM/SAM/SOM）
- 竞争定位和市场份额
- 客户分析——集中度、留存、NPS
- 定价权和合同结构
- 销售管道和积压订单
- 商业化有效性

**法律尽调**
- 公司结构和组织图
- 重大合同（客户、供应商、合作伙伴）
- 诉讼历史和待决索赔
- 知识产权组合和保护
- 监管合规
- 雇佣协议和非竞争条款

**运营尽调**
- 管理团队评估
- 组织结构和关键人员风险
- IT系统和基础设施
- 供应链和供应商依赖
- 设施和房地产
- 保险覆盖

**人力资源/人员尽调**
- 组织图和人员趋势
- 薪酬对标
- 福利和养老金义务
- 关键员工留存风险
- 文化评估
- 工会/劳工协议

**IT/技术尽调**（适用于科技赋能业务）
- 技术栈和架构
- 技术债务评估
- 网络安全态势
- 数据隐私合规（GDPR、CCPA、SOC2）
- 产品路线图和研发支出
- 可扩展性评估

**环境/ESG**（适用时）
- 环境负债
- 监管合规历史
- ESG风险和机会

### 步骤3：状态跟踪

跟踪每个项目：

| 项目 | 工作流 | 优先级 | 状态 | 负责人 | 备注 |
|------|--------|--------|------|--------|------|
| QoE报告 | 财务 | P0 | 待处理 | | |
| 客户访谈 | 商业 | P0 | 进行中 | | 10个完成3个 |

状态选项：未开始 → 已请求 → 已收到 → 审查中 → 完成 → 红旗

### 步骤4：红旗摘要

维护在尽调过程中发现的红旗列表：
- 发现了什么
- 哪个工作流
- 严重程度（交易破坏者/重大/可控）
- 缓释方案或解决路径
- 对估值或交易条款的影响

### 步骤5：输出

- 每个工作流一个标签页的Excel工作簿（默认）
- 摘要仪表板：各工作流完成百分比、未完成项目、红旗
- 团队周报状态更新格式

## 行业特定补充

根据行业自动添加相关项目：
- **软件/SaaS**：ARR质量、队列分析、托管成本、SOC2
- **医疗健康**：监管批准、报销风险、支付方组合
- **工业**：设备状况、环境修复、安全记录
- **金融服务**：监管资本、合规历史、信用质量
- **消费**：品牌健康、渠道组合、季节性、库存管理

## 重要说明

- 优先处理影响意向书或交割的P0项目
- 标记卖家响应缓慢的项目——可能表明存在问题
- 将数据室内容与清单交叉参照以识别差距
- 随着尽调推进更新清单——这是动态文档
