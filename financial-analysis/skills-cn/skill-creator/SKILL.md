# 技能创建指南

## 关于技能 (Skills)

技能是模块化的、自包含的包，通过提供专门的知识、工作流和工具来扩展 Claude 的能力。它们类似于特定领域或任务的“入职培训指南”——将通用 Agent 转化为装备有专业程序化知识的专家。

### 技能包含的内容
1. **专门的工作流**：针对特定领域的跨步骤操作程序。
2. **工具集成**：操作特定文件格式或 API 的指令。
3. **领域专业知识**：公司特定的知识、架构或业务逻辑。
4. **捆绑资源**：用于复杂或重复任务的脚本、参考文档和资产。

## 核心原则

### 简洁至上 (Concise is Key)
上下文窗口是公共资源。技能与对话历史共享此空间。
**默认假设：Claude 已经非常聪明。** 仅添加 Claude 尚不具备的背景信息。宁可提供简洁的示例，也不要冗长的解释。

### 设置适当的自由度
*   **高自由度（纯文字指令）**：适用于有多种有效方法、决策依赖于上下文的任务。
*   **中自由度（伪代码或带参数的脚本）**：适用于存在首选模式但允许一定变化的任务。
*   **低自由度（具体脚本，极少参数）**：适用于操作极其脆弱、一致性至关重要的任务。

### 技能解剖 (Anatomy of a Skill)
```
skill-name/
├── SKILL.md (必填)
│   ├── YAML Frontmatter (name, description)
│   └── Markdown 指令正文
└── 捆绑资源 (可选)
    ├── scripts/    - 可执行代码
    ├── references/ - 仅在需要时加载到上下文的参考文档
    └── assets/     - 最终输出中使用的文件（模板、图标、字体等）
```

## 编写 SKILL.md 指南

### 1. Frontmatter (YAML)
*   **name**: 技能名称。
*   **description**: **关键触发机制**。描述技能的作用以及**何时**使用。Claude 通过阅读此描述决定是否激活该技能。

### 2. 正文 (Body)
*   使用**祈使句/动词原形**。
*   仅保留核心流程。将详细的架构说明、API 参考或长示例移至 `references/` 目录。

### 3. 资源引用
*   **scripts/**：用于需要确定性可靠性的任务。必须通过实际运行进行测试。
*   **references/**：用于存放 Claude 在工作时应查阅的文档（如：数据库架构、公司政策、详细工作流）。
*   **assets/**：Claude **不应读取**其实际内容的文件（如：PPT 模板、标志图片、样本文档）。Claude 仅需知道它们存在并能操作它们。

## 渐进式披露 (Progressive Disclosure)
为管理上下文效率，技能采用三层加载系统：
1. **元数据 (name + description)**：始终在上下文中（约 100 词）。
2. **SKILL.md 正文**：仅在技能触发时加载。
3. **捆绑资源**：根据需要由 Claude 主动读取。

**建议**：保持 SKILL.md 正文在 500 行以内。如果内容过多，请将其拆分到 `references/` 下的文件中。
