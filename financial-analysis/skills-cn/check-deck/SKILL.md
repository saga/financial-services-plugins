# 投行演示文稿检查

从四个维度对投资银行演示文稿进行全面质量检查。

## 前提条件

检查前先提取演示文稿内容：
```bash
python -m markitdown presentation.pptx > content.md
```

如需视觉检查，使用 `pptx` 技能工作流转换为图片。

## 检查流程

### 1. 数据一致性

提取数据并标注幻灯片来源：
```bash
python scripts/extract_numbers.py content.md --check
```

核实以下内容：
- 关键指标在所有幻灯片间一致（营收、EBITDA、倍数）
- 计算正确（合计、百分比、增长率）
- 单位一致（使用相同量级：百万 vs 十亿，% vs bps）
- 单位格式一致（如 $M vs $MM，$B vs $Bn——全文统一一种写法）
- 时间口径对齐（财年 vs 过去十二个月 vs 季度）

问题标记格式：
```
问题：营收数据不一致
- 第3、8张幻灯片：$5亿
- 第15张幻灯片（DCF输入）：$4.85亿
处理措施：核实并统一数据
```

### 2. 数据与叙事对齐

将文字描述映射至支撑数据：
- 趋势性表述 → 图表方向
- 市场地位描述 → 营收/份额数据
- 事实性断言 → 验证准确性

标记矛盾之处：
```
问题：叙事与数据矛盾
- 第4张幻灯片："利润率下滑"
- 第7张幻灯片图表：利润率从18%升至22%
处理措施：修改叙事或核查数据
```

检查合理性（如："$1000亿市场的第一名"但营收仅$2亿 = 0.2%市场份额）。

### 3. 语言规范

扫描以下问题：
- 随意表达（"很不错"、"一大堆"）
- 无具体数字的模糊量化词
- 缩略语、感叹号
- 术语不一致

替换规范参见 [references/ib-terminology.md](references/ib-terminology.md)。

问题标记格式：
```
问题：随意措辞（第12张幻灯片）
- "这笔交易显而易见"
→ "本次交易具有引人注目的价值主张"
```

### 4. 排版格式质控

逐张幻灯片审核：
- **图表**：数据来源注释、坐标轴标签、图例
- **字体**：字体一致性、字号层级
- **数字**：格式一致（1,000 vs 1K）
- **日期**：全文格式统一
- **脚注**：规范的来源引用和免责声明

## 输出结果

按照 [references/report-format.md](references/report-format.md) 中的模板呈现检查结论。

按严重程度分类：
- **严重**：数据不一致、事实错误
- **重要**：语言规范、数据与叙事对齐问题
- **次要**：排版格式不一致
