# 竞争格局分析

---

## 关键标准——每次分析均须遵守

### 数据来源文件优先

当提供了数据来源文件（Excel/CSV）时：
* **直接提取数值**——不自行计算；使用文件中显示的原始数字
* **保持一致性**——同一指标在多处出现时，确保数值完全相同
* **核实计算**——若要求计算某项内容，验证结果与来源数据中的相关数字一致
* **按原精度取整**——使用与来源文件相同的小数位数

### 严格遵从提示要求

当提示中指定了确切要求时，须逐字遵守：

**幻灯片标题和章节名称：**
* 若提示说"'概述与竞争范围'幻灯片"——使用该标题原文，不得意译
* 若提示说"在'细分市场构成'章节内"——使用原文"细分市场构成"作为章节标题
* 不得用创意性替代方案（如将"概述与竞争范围"改成"FY2024细分贡献分析"）

**图表 vs 表格：**
* 若提示说"嵌入图表"——创建真实的 PowerPoint 图表对象，不是表格
* 若提示说"数据标签必须显示"——标签应在图表元素上（柱、扇区、折线），不在表格单元格中
* 表格和图表不可互换——使用提示指定的格式

**完整数据系列：**
* 若提示列出7个竞争对手，包含全部7个——不能只放5或6个
* 若提示显示2015-2025年数据，包含所有年份——不能只取子集
* 若提示指定图表中有6个系列，包含所有6个——不能只放4个

**精确数值和措辞：**
* 若提示说"营收：$43.98B（同比+18%）"——使用完全相同的格式
* 若提示说"以4:1超过DoorDash，以8:1超过Lyft"——使用这些精确比值，不能写"7.6倍Lyft"
* 若提示给出具体百分比（如"优步30%，滴滴35%"），使用完全相同的数字

**若有疑问：** 重新阅读提示。明确指定的内容是要求，不是建议。

### 参考文件

本技能在 `references/` 文件夹中包含参考文件，使用方式如下：
* **`references/schemas.md`**——并购交易、情景分析和幻灯片结构的表格模板。构建财务表格或投资情景时参考。
* **`references/frameworks.md`**——按行业划分的2x2矩阵轴对配置。选择定位可视化维度时参考。

### 数据来源质量层级

当多方来源数据冲突时，按以下顺序优先使用：
1. **10-K / 年度报告**——经审计，可靠性最高
2. **业绩说明会 / 投资者演示文稿**——管理层评述、前瞻性指引
3. **卖方研究报告**——分析师预测，适用于私有公司规模估算
4. **行业报告**（麦肯锡、Gartner等）——市场规模、行业趋势
5. **新闻报道**——仅用于最新进展，须与一手来源核实

### 数据可比性
* **时间口径必须一致**——所有竞争对手指标均来自同一财年。标注例外情况："(FY24)" vs "(H1 2024)"
* **指标定义必须一致**——跨竞争对手使用相同的计算方法
* **货币统一**——国际比较中将所有数字换算为美元；注明汇率和使用日期
* **缺失数据用"—"表示**——单元格不留空；私有公司用"N/A"或带"[E]"标记的估算值
* **每个数字都需注明来源**——格式："[公司] [文件] ([日期])"
* **来源文件忠实性**——提供了Excel/CSV文件时，使用原始数值；不重新计算或以不同方式取整

### 设计与排版
* **幻灯片标题 = 洞见**——用"规模领先者正在拉开与利基玩家的距离"，不用"竞争分析"
* **幻灯片标题必须适配**——一行或两行均可，但不能溢出；必要时缩小字号（最小24pt）
* **路标数字化**——用"利润率低于40%"，不用"利润率下滑"
* **评级包含实际数据**——用"●●● $1600亿"，不仅是"●●●"
* **幻灯片编号必须有**——每张幻灯片须有页码

### 演示文稿特定要求
* **必须使用真实嵌入图表**——饼图、柱状图和折线图必须是真实的 PowerPoint 图表对象（通过 pptx 技能创建），不能是文字/ASCII 表示
* **匹配提示结构**——若提示指定了幻灯片结构，须遵从
* **竞争对手表格**——全面分析：每个竞争对手提供指标表 + 定性表；快速评估：单一综合表可接受
* **细分财务数据**——有条件时同时包含营收和EBITDA；私有竞争对手或信息披露有限时，仅营收表格可接受；注明"[EBITDA未披露]"

### 视觉规范

**间距与溢出防控：**
- 标题与内容间距——幻灯片标题底部与第一个内容元素之间最小0.4英寸
- 章节标题间距——章节标题与下方内容之间最小0.25英寸
- 元素缓冲——任意两个元素（表格、文本框、图表）之间最小0.2英寸
- 边距安全——所有内容距幻灯片边缘至少0.5英寸
- 文字溢出——文字放不下时，缩小字号或拆分到多张幻灯片；不允许文字被截断或重叠

**字体排版（须明确设置，不使用默认值）：**
- 幻灯片标题：28-32pt 粗体
- 章节标题：16-20pt 粗体
- 正文：11-14pt 常规
- 表格文字：10-12pt 常规
- 来源/脚注：8-10pt，灰色
- **一致性规则**：相同类型的元素在整个文档中使用相同字号

**图表排版：**
- **图例置于布局内**——始终设置 `include_in_layout=True`，使图例不与图表区域重叠
- **图例位置**——饼图（≤6项）置于右侧，折线图/柱状图（≤4系列）置于底部
- **系列过多**——若超过6个系列，考虑拆分为多张图表或使用表格替代
- **数据标签**——饼图在扇区上显示百分比，不能仅依赖图例

**颜色：**
- 有限配色——最多2-3种颜色（一种强调色用于突出重点）
- 低饱和色调——避免鲜艳/高饱和颜色；使用深蓝、灰色、柔和蓝
- 一致性应用——整个文档中颜色含义保持一致（如强调色用于关键指标）

**表格：**
- 浅灰色标题行，粗体文字
- 隔行底纹（细腻）或无底纹加细边框
- 数字右对齐，文字左对齐
- 充足单元格内边距——文字不应贴近边框

**评分可视化：**
- ●●● / ●●○ / ●○○ 系统，旁边附实际指标数据
- 在比较表格中位置统一

### 严格项 vs 灵活项

| 严格项（每次必须遵守） | 灵活项（视情况而定） |
|----------------------|-------------------|
| 提示指定时，使用精确的标题/章节名 | 提示未指定时，可用创意标题 |
| 提示说图表用图表；提示说表格用表格 | 提示未指定时，可选择可视化类型 |
| 包含提示中列出的所有数据点/竞争对手 | 提示未指定时，竞争对手数量可灵活 |
| 提示指定时，使用精确数值/比值 | 提示未指定精度时，可灵活取整 |
| 标题在幻灯片宽度内不溢出 | 竞争对手类别数量 |
| 元素间最小间距 | 选择比较维度 |
| 图表图例置于布局内 | 深度分析的竞争对手数量 |
| 文字/元素不重叠 | 可视化类型（2x2、雷达图、层级图） |

## 工作流程阶段

### 阶段一：明确需求
开始前确认：
* **范围**：单一公司深度分析，还是多公司比较？
* **输出格式**：演示文稿还是书面备忘录？
* **重点领域**：特定竞争对手、维度或战略问题？
* **投资背景**：是否需要情景/信号指标？
* **来源文件**：提供了哪些数据文件，应提取哪些数值？

### 阶段二：研究 → 提纲 → 评审 → 创作
**在提纲获得评审前，不要创作最终输出。**

以下十步分析工作流在阶段二执行。在创建最终幻灯片或文件前，先完成研究和提纲工作。

---

## 分析工作流

### 第0步：识别行业核心指标
深入分析前，识别对该行业最重要的3-5个指标：

| 行业 | 核心指标 |
|------|----------|
| SaaS | ARR、净留存率、CAC回收期、LTV/CAC、Rule of 40 |
| 支付 | GPV、手续费率、附加率、交易利润率 |
| 平台/市场 | GMV、佣金率、买卖家比率、复购率 |
| 零售 | 同店销售额、存货周转率、坪效 |
| 物流 | 量、单位成本、准时交付率%、产能利用率 |

对未列出的行业，识别投资者和运营者用于衡量业绩的3-5个指标。

在所有竞争对手比较中一致使用这些指标。

### 第1步：市场背景
- 市场规模（当前和预测）及来源
- 增长驱动因素和阻力
- 重塑行业的关键趋势

**正确示例：** "嵌入式支付市场2024年规模为$800-1000亿，CAGR 20-25%（麦肯锡2024年）"
**错误示例：** "市场规模大且增长迅速"

### 第2步：行业经济学

梳理价值流向，方法因行业类型而异：
* **垂直整合型**——各价值链层级及典型利润率
* **平台/网络型**——生态系统参与者及其间的价值流动
* **分散型**——整合动态及规模差异带来的利润率差异

### 第3步：目标公司画像

```
| 指标 | 数值 |
|------|------|
| 营收 | $49.6亿 |
| 增长 | 同比+26% |
| 毛利率 | 45% |
| 盈利能力 | 调整后EBITDA $3.73亿 |
| 客户数 | 13.4万 |
| 留存率 | 92% |
| 市场份额 | ~15% |
```

**多业务段公司，添加业务段拆分：**
```
| 业务段 | 营收 | 营收同比 | 营收占比 | EBITDA | EBITDA同比 | 利润率 |
|--------|------|----------|----------|--------|------------|--------|
| A段    | $251亿 | +26%  | 57%     | $65亿  | +31%       | 26%    |
| B段    | $138亿 | +31%  | 31%     | $25亿  | +64%       | 18%    |
| C段    | $51亿  | -2%   | 12%     | -$7.4亿 | -16%      | -1%    |
| 合计   | $440亿 | +18%  | 100%    | $65亿* | —          | 15%    |
```
*如适用，注明公司层面费用

### 第4步：竞争对手梳理

使用最适合的框架对竞争对手进行分组：
* **按商业模式**——平台型 vs 垂直型 vs 横向型
* **按细分市场**——大型企业 vs 中小企业 vs 消费者
* **按竞争姿态**——直接竞争 vs 相邻领域 vs 新兴威胁
* **按起源**——现有巨头 vs 颠覆者 vs 新进入者

### 第5步：定位可视化

| 可视化形式 | 最适用于 |
|-----------|----------|
| 2x2矩阵 | 两个主导竞争因素 |
| 雷达/蜘蛛图 | 多维度比较 |
| 层级图 | 自然分群/战略群组 |
| 价值链图 | 垂直整合行业 |
| 生态系统图 | 平台市场 |

### 第6步：竞争对手深度分析

**表1——财务指标：**
```
| 指标 | 数值 |
|------|------|
| 营收 | $X.XB |
| 增长 | 同比+XX% |
| 毛利率 | XX% |
| 市值 | $X.XB |
| 盈利能力 | $XXXM EBITDA |
| 客户数 | XXK |
| 留存率 | XX% |
| 市场份额 | ~XX% |
```

**表2——定性评估：**
```
| 类别 | 评估 |
|------|------|
| 业务 | 核心业务（一句话描述） |
| 优势 | 2-3个要点 |
| 劣势 | 2-3个要点 |
| 战略 | 当前重点 |
```

### 第7步：比较分析

```
| 维度 | 公司A | 公司B | 公司C |
|------|-------|-------|-------|
| 规模 | ●●● $1600亿 | ●●○ $450亿 | ●○○ $80亿 |
| 增长 | ●●○ +26% | ●●● +35% | ●●○ +22% |
| 利润率 | ●●○ 7.5% | ●○○ 3.2% | ●●● 15% |
```

### 第8步：战略背景
- 并购交易（估值倍数、战略逻辑）
- 合作与整合趋势
- 融资规律
- 监管动态

### 第9步：综合研判

**竞争护城河评估：**
使用以下类别评估每个竞争对手的持久优势：

| 护城河类型 | 评估内容 |
|-----------|----------|
| 网络效应 | 用户/供应商飞轮强度；双边 vs 同边网络效应 |
| 切换成本 | 技术整合深度、合同锁定、行为习惯 |
| 规模经济 | 规模带来的单位成本优势；最低有效规模 |
| 无形资产 | 品牌价值、专有数据、监管牌照、专利 |

将每项评为强/中/弱，并附上支撑证据。

**必须包含的综合要素：**
- 持久优势（难以复制）——对应上述护城河类别
- 结构性弱点（难以修复）
- 当前状态 vs 发展轨迹

**投资背景下：**
```
| 情景 | 概率 | 关键驱动因素 |
|------|------|------------|
| 乐观 | 30% | 市场份额增加、利润率扩张 |
| 基础 | 50% | 当前轨迹延续 |
| 悲观 | 20% | 竞争压力、利润率收缩 |
```

最终完成前，验证以下内容：

- ✅ 幻灯片标题与提示完全一致（未意译）
