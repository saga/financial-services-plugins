# 企业发展部并购项目评估简报

## 目的
本简报旨在为内部企业发展（Corp Dev）团队评估潜在收购目标提供画像。与投资银行/并购项目简报（旨在为路演讲述具有战略意义的“故事”）不同，这是一份侧重于战略契合度的分析文件：重点审视目标公司的产品、客户、技术和业务操作与收购方之间存在怎样的重叠或互补。企业发展部的受众早已深谙自身公司情况，他们需要透过“我们是否应该买下它，以及收购后的整合会是什么样子？”的视角来了解目标。

**默认页面长度：** 1-2页。企业发展团队习惯于阅读信息密度较高的简报，若内容充实，即使扩展到第二页也不会被视为减分。

## 查询路径与数据检索方案

从以下查询开始。如果结果不完整，请拆分并进行后续跟进。

**查询 1 — 目标公司画像：**
"[公司名称] 业务描述 产品 服务 技术平台 总部 成立时间 员工人数 行业板块 细分行业"
→ 用于：页眉、业务与产品概览
→ **立即写入** 至 `/tmp/tear-sheet/company-profile.txt`

**查询 2 — 财务数据：**
"[公司名称] 年度损益表 营收 毛利 EBITDA 营业利润 净利润 资本支出 自由现金流 研发费用 总债务 现金及等价物 最近4个财年"
→ 用于：财务摘要（提取4年数据；展示3年；仅使用最早一年计算同比增长。研发费用对企业发展部尤为重要，他们希望了解目标公司在知识产权上的投入）
→ **立即写入** 原始数值至 `/tmp/tear-sheet/financials.csv`

**查询 3 — 业务分部与客户：**
"[公司名称] 按分部/业务单元划分的营收 最近2个财年 关键客户 终端市场 客户集中度"
→ 用于：营收结构（需2年数据以计算同比）、客户分析
→ **立即写入** 分部数据至 `/tmp/tear-sheet/segments.csv`（若无分部数据则跳过）

**查询 4 — 业务关系与竞争格局：**
"[公司名称] 关键客户 供应商 合作伙伴 竞争对手 业务关系 技术供应商"
→ 用于：战略契合度分析、生态系统图谱
→ **立即写入** 至 `/tmp/tear-sheet/relationships.txt`

**查询 5 — 估值与对标：**
若用户提供了特定对标公司：
"[对标公司] 企业价值 EV/Revenue EV/EBITDA 营收增长率 EBITDA利润率"
否则：
使用竞争对手分析工具识别3-5家上市同行，并提取每家的 EV/Revenue (NTM)、EV/EBITDA (NTM)、营收增长率 % 以及 EBITDA 利润率 %。
此外："[公司名称] 企业价值 市值 估值倍数"
→ 用于：估值背景
→ **立即写入** 公司倍数至 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 同行数据至 `/tmp/tear-sheet/peer-comps.csv`

**查询 6 — 股权结构（视数据情况而定）：**
"[公司名称] 所有权结构 投资者 机构持股 内部人持股"
→ 用于：股权快照（无需中间文件 —— 如果可用，包含在 company-profile.txt 中）

## 章节内容

按优先级排序。关于具体的删减指导，请参阅下文的“页面预算与删减顺序”。

### 1. 公司页眉
与投行/并购简报类似，但增加了“战略相关性”标签。

| 字段 | 备注 |
|---|---|
| 公司名称 | 大号、加粗 |
| 股票代码与交易所 | 若为上市公司；私有公司则省略 |
| 总部所在地 | |
| 成立时间 / 员工人数 | |
| 企业价值 (EV) | 上市公司为市值；私有公司为最新估值 |
| 股权结构 | 公众上市公司 / PE支持 / 创始人控股 / 等 |
| **战略相关性** | 一句话总结：为什么该目标对收购方有吸引力（综合数据得出） |

“战略相关性”这一行是企业发展部简报所特有的。请从中立的分析视角撰写，而非采用目标公司的市场营销语言。重点界定收购方能“获得”什么，而非目标公司“是”什么。优秀示例：“在商品价格基准领域占据主导地位，拥有监管护城河及 70% 以上的毛利率 —— 这项轻资产、高壁垒的数据资产极具吸引力。” 欠佳示例：“全球领先的金融数据与分析提供商。” 前者告诉企业发展团队为何这项资产值得买；后者只是公司新闻稿。

### 2. 业务与产品概览
**切勿直接粘贴 Capital IQ 的公司摘要。** 用 4-6 句话（一个段落而非两个）重写，重点聚焦于产品和技术 —— 该受众群体评估的是“收购”，而非单纯的“投资”：
- 核心产品/服务及其解决的问题
- 技术或平台架构（高层级描述）
- 核心差异化优势与竞争护城河
- 客户群体 —— 谁在购买以及为何购买
- 当前战略方向

请以收购方的视角撰写。读者会思考：“这如何与我们现有的业务相契合？”描述编写应便于读者识别双方的重叠与互补之处。

### 3. 营收结构与客户画像
包含两个子章节：

**a) 按分部划分的营收**（若可用）：

| 业务分部 | 营收 ($M) | 占总比 % | 同比增长 |
|---|---|---|---|

**b) 客户分析：**
在分部表格下方（或分部数据缺失时替代展示），撰写一段定性描述，涵盖：
- 客户集中度：前五大客户营收占比是否 >30%？（这对于评估收购风险至关重要）
- 客户类型：大型企业 (Enterprise) vs. 中小企业 (SMB) vs. 消费者 (Consumer)
- 合同结构：订阅制 vs. 交易制 vs. 许可授权（若已知）
- 地域分布

客户集中度是此处最重要的信号 —— 如果工具返回了关于大客户或营收集中度的任何数据，请重点标注。如果数据稀少，则根据业务描述和业务关系进行推断。保持在 2-3 句话左右 —— 它是分部表格的背景补充，而非独立的详尽分析。

### 4. 战略契合度分析
**这是企业发展部简报的招牌章节。** 该部分为必填项 —— 请勿跳过或压缩。利用来自 S&P Capital IQ 的业务关系数据来描绘重叠与互补关系。

分为以下三个维度。**每个维度必须包含 2-3 句分析性论证，而非简单的公司清单。** 仅罗列“客户：微软、摩根大通、谷歌”而无背景信息是毫无意义的。相反，应解释这种重叠或互补对于收购意味着什么。

**客户重叠：** 目标的客户是否也购买收购方（或收购方所在行业）的产品？共享客户意味着更容易进行交叉销售，但也存在渠道冲突的风险。列举共享的业务关系并解释其背后的含义。

**技术/产品互补：** 目标公司是否填补了收购方产品组合中的空白？他们的技术栈表现如何？他们的技术供应商是谁？如果他们已在使用收购方的技术，请解释为何这是强烈的整合信号。

**竞争替代：** 目标公司是竞争对手吗？收购他们是否能消除市场中的一个竞争因素？列出目标的核心竞争对手，并解释收购后竞争格局将如何变化。

此章节需要高度的综合归纳 —— 将业务关系数据与业务概览结合起来以建立逻辑关联。如果用户告知了收购方是谁，请为该收购方量身定制。如果不知道，请以通用但依然具有分析性的方式撰写。

**字数约束：** 上述每个维度 2-3 句话的长度是硬性约束，而非建议。每个要点应先陈述洞察结论，然后用一个具体数据点支持。实际输出往往会膨胀到每点 5-7 句话 —— 这是导致内容溢出至第3页的主要原因。如果你发现自己写了超过 3 句话，请将多余的细节移至“整合考量”章节。
