# 首次覆盖报告质量控制检查清单

在交付首次覆盖报告（Initiation Report）之前，请务必确认以下所有项目均已完成。

## 核心最低标准 - 报告必须满足以下要求

**若存在以下情况，严禁交付：**
- ❌ DOCX 报告少于 30 页 → **不完整**
- ❌ 嵌入图表少于 25 个 → **不完整**
- ❌ 综合表格少于 12 个 → **不完整**
- ❌ 字数少于 10,000 字 → **不完整**
- ❌ 缺失 XLS 财务模型文件 → **缺失交付物**
- ❌ 图表仅为文字描述而非实际生成的 PNG/JPG 图片文件 → **严重质量漏洞**

## 交付物检查清单

- [ ] 已创建 DOCX 格式报告文件
- [ ] 已创建 XLS 财务模型文件
- [ ] 两个文件命名均符合规范：`[公司名]_首次覆盖报告_[日期].docx` 以及 `[公司名]_财务模型_[日期].xlsx`

## DOCX 报告 - 篇幅与内容

**篇幅校验：**
- [ ] 报告页数在 30-50 页之间（以最终文档页码为准）
- [ ] 字数在 10,000-15,000 字之间
- [ ] 若少于 30 页：**立即停止**并补充更多分析内容

**可视化元素：**
- [ ] 嵌入 25-35 个图表（计数确认：_____ 个图表）
- [ ] 所有图表均为真实的 PNG/JPG 图片文件（**严禁**使用文字描述替代）
- [ ] 包含 12-20 个综合表格（计数确认：_____ 个表格）
- [ ] 图表与表格贯穿全文，严禁在文末集中堆砌

**图表要求：**
- [ ] 按产品拆分的收入分析：采用堆积面积图格式 ✓
- [ ] 按地区拆分的收入分析：采用堆积柱形图格式 ✓
- [ ] DCF 敏感性分析：带颜色编码的二维热力图 ✓
- [ ] 估值“足球场图”：横向柱状图 ✓
- [ ] 所有其他图表均为真实的图片文件 ✓

**表格要求：**
- [ ] 完整利润表（40-50 行），包含 5 年历史数据 + 5 年预测数据
- [ ] 完整现金流量表（30-40 行）
- [ ] 完整资产负债表（35-45 行）
- [ ] 按产品拆分的收入表（20-30 行）
- [ ] 按地区拆分的收入表（15-20 行）
- [ ] 按渠道拆分的收入表（10-15 行）
- [ ] 可比公司分析表及统计摘要（最大值/75分位数/中位数/25分位数/最小值）
- [ ] DCF 计算表（30-40 行）
- [ ] WACC 计算表（8-10 行）
- [ ] 两个敏感性分析表
- [ ] 2-3 个额外的财务或竞态分析表

## DOCX 报告 - 结构布局

**第 1 页要求：**
- [ ] 包含“首次覆盖报告”（INITIATING COVERAGE）标题（**严禁**标注为“公司更新”）
- [ ] 标题须体现核心逻辑（而非追踪事件，如“强劲的四季度业绩”）
- [ ] 配备评级框，包含评级、现价、目标价、52周股价区间、市值、企业价值（EV）
- [ ] 3-4 个段落长度的要点逻辑，使用 ■ 符号并加粗标题
- [ ] 财务与估值指标表，包含 2-3 年历史数据、2 年预测数据
- [ ] 表格中使用“A”后缀代表实际值（Actuals），“E”后缀代表预测值（Estimates）
- [ ] 所有可视化素材下方均需标注数据来源

**内容板块：**
- [ ] 目录（第 2 页）
- [ ] 投资逻辑及风险提示（3-5 页）
- [ ] 公司概况（6-12 页），包括：
  - [ ] 公司业务描述
  - [ ] 发展历程及里程碑
  - [ ] 管理层简介（每位高管约 300-400 字，包含 3-4 名核心高管）
  - [ ] 产品/服务详情
  - [ ] 竞争格局分析
