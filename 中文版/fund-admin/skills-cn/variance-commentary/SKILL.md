# 差异分析解读

根据同一业务范围内的本期实际数、上期实际数及预算数，生成一份差异分析表。

## 差异阈值

若满足以下**任一**条件，则需对该行项目进行解读：

- 绝对差异 ≥ 公司设定的重要性阈值（利用提供的值；默认取该行金额的 5% 或固定底限中的较大者）。
- 该项目属于“必须备注”清单（如：收入、人员成本、现金）。

## 针对每个标记项

| 列名 | 内容 |
|---|---|
| **科目项目** | 账户名称或报表项目标题 |
| **本期 / 上期 / 预算** | 三个对应的数值 |
| **较上期差异** 及 **较预算差异** | 差异金额及百分比 |
| **驱动因素** | 用一句话解释业务层面波动的动因 —— 而非简单重复数值变动 |

驱动因素应解释“为什么”而不是“是什么”：例如“由于 5 月产品发布新增了 GPU 预留实例，导致云服务支出增长 120 万美元” —— 而非“云服务支出增长了 120 万美元 (18%)”。

## 获取驱动因素

通过 `internal-gl` MCP 查看科目背后的具体活动（总账分录来源分解、供应商结构、人数变动、量阶分析等）。如果从数据中无法明确驱动因素，请注明“驱动因素不明 —— 标记由财务主管确认”，严禁凭空捏造。

## 输出要求

一份差异分析表，外加一段简短的叙述性总结（3-5 句话），概述本期变动最显著的项目。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **差异分析 (Variance Analysis)**：这就像是给公司的财务状况做“找茬游戏”。我们会把现在的表现和过去（上期）或者原本的计划（预算）进行对比，看看哪里跑偏了。
> 2. **重要性阈值 (Materiality Threshold)**：财务人的滤网。公司每天产生的账目成千上万，我们不能所有的波动都去查。只有那些“块头”足够大、足以影响老板判断的差异，才达到这个“阈值”，需要专门写报告。
> 3. **驱动因素 (Driver)**：这是老板最关心的“幕后推手”。光说“钱花多了”没用，得说清楚是“因为新招了人”还是“供应商涨价了”。找到驱动因素，才能对症下药。
> 4. **损益表 (P&L) 与资产负债表 (Balance Sheet)**：损益表看的是这一阵子你是赚了还是亏了（流量）；资产负债表看的是你现在手里有多少家底和欠债（存量）。
