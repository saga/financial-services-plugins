# 余额变动表 (Roll-forward)

针对给定的账户（或账户组）、实体及期间，生成一份期初与期末勾稽一致的余额变动表。

## 结构

```
期初余额（按上期结账值）                 X
  + 增加额 / 新发生额                  A
  + 本期计提的应计项目                 B
  − 上期应计项目的冲回                (C)
  − 付款 / 结算                       (D)
  ± 重分类 / 调整项                    E
  ± 外币折算差异                       F
期末余额（按期末总账值）                 Y
```

## 各行勾稽要求

- **期初余额** —— 参见上期结账报告包，或上期期末日期的总账余额。
- **各发生额明细行** —— 通过 internal-gl MCP 进行总账查询（维度包括：账户 + 日期范围 + 凭证来源过滤）。请引用相关查询。
- **期末余额** —— 参见本期期末日期的总账余额。

报表 **必须通过纵向加总校验 (Footing)**：`X + A + B − C − D + E + F = Y`。若结果不相等，产生的缺口即为“未解释项” —— 请务必如实反映，**严禁为了强行配平而擅自填补差额 (No Plug)**。

## 输出要求

余额变动表需包含“勾稽至 (Ties to)”列，针对每一行引用总账查询或证明文档，并附带加总校验结果（通过/未通过，以及存在的未解释差异金额）。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **什么是 Roll-forward（余额变动表）？**
> 想象你在管一个存钱罐。月初里面有 100 块（期初），本月你存了 50 块（增加），又花了 20 块（减少），月底数数还剩 130 块（期末）。Roll-forward 就是这张清晰的清单，它不仅告诉你最后剩多少钱，更重要的是解释了这些钱是怎么变动的。在财务审计中，这是验证数据真实性的“头号神器”。
>
> 2. **这里的“勾稽”（Tie/Tie out）是什么意思？**
> 在财务世界里，数字不能孤立存在。你说你存了 10 块钱，就得从银行流水（总账 GL）里找出那笔记录来证明。这种“数字对得上、证据找得到”的严谨对比过程，就是“勾稽”。
>
> 3. **计提（Accruals）与冲回（Reversals）：**
> 计提是指“活儿已经干了，虽然还没付钱，但得先把费用记上”。到了下个月正式付钱时，我们需要把之前预记的这笔账“冲回（抵消）”，以免重复计算。这就像是你先在小本本上记下“欠隔壁老王 10 块”，等真给钱时，要把这行字划掉重写。
>
> 4. **加总校验（Footing）：**
> 听起来很高大上，其实就是“小学数学复核”。把那一列数字从头加到尾，看看最后的结果是不是真的等于期末数。如果 `1 + 1` 不等于 `2`，那是绝对不能“强行凑数”掩盖过去的，财务人必须把那个哪怕只有 1 分钱的差额给揪出来！
