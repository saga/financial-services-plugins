# Pitch 代理 — 托管代理模板

## 概述

可比公司分析、先例交易、LBO → 品牌化 Pitch 材料，端到端。与 [`pitch-agent`](../../plugins/agent-plugins/pitch-agent) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export CAPIQ_MCP_URL=... DALOOPA_MCP_URL=...
../../scripts/deploy-managed-agent.sh pitch-agent
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。

## 安全与交接

任务分解拆分 —— 较少涉及不可信输入（数据来自 CapIQ/Daloopa MCP），更多关注并行性和产物隔离。只有一个工作节点持有 `Write`：

| 叶子节点 | 工具 | 连接器 |
|---|---|---|
| `researcher` | `Read`、`Grep` | CapIQ、Daloopa（只读） |
| `modeler` | `Read`、`Bash`（沙箱化） | CapIQ、Daloopa（只读） |
| **`deck-writer`**（写入持有者） | `Read`、`Write`、`Edit` | 无 |

产物通过 `pptx-author` / `xlsx-author` 输出到 `./out/pitch-<目标>.pptx` 和 `./out/model.xlsx`。

**交接：** 如需在投资论点变化后重建模型，编排器发出指向 `model-builder` 的 `handoff_request`；`scripts/orchestrate.py`（或您的工作流引擎）将其作为新引导事件路由。参见该脚本中的白名单 + 载荷验证模式。

## 金融术语和知识解释

### 1. Pitch 材料（Pitch Book）

**定义**：Pitch 材料是投资银行向客户（通常是公司或私募股权基金）展示的演示文稿，用于争取并购顾问、融资安排或其他投行服务的机会。

**Pitch 材料类型**：

| 类型 | 目的 | 典型内容 |
|------|------|---------|
| 卖方 Pitch | 争取出售客户公司的顾问权 | 市场概览、估值分析、潜在买家 |
| 买方 Pitch | 争取代表收购方的顾问权 | 目标筛选、交易结构、融资方案 |
| 融资 Pitch | 争取债务或股权融资安排 | 市场条件、投资者需求、定价策略 |
| 防御 Pitch | 帮助客户抵御敌意收购 | 防御策略、估值论证、替代方案 |

**标准结构**：
1. **封面**：交易概述、银行品牌
2. **团队介绍**：负责团队的资历和经验
3. **市场概览**：行业趋势、交易环境
4. **公司概况**：目标公司的业务和财务亮点
5. **估值分析**：可比公司、先例交易、DCF
6. **交易逻辑**：战略 rationale、协同效应
7. **执行方案**：时间表、关键里程碑
8. **附录**：详细数据、假设、免责声明

---

### 2. 可比公司分析（Trading Comps）

**定义**：可比公司分析是通过与同行业上市公司的估值倍数比较，为目标公司估值的方法。

**选择标准**：
- 同一行业或细分市场
- 相似的规模（收入、市值）
- 相似的增长阶段
- 相似的盈利能力和资本结构

**常用倍数**：

| 倍数 | 适用场景 | 计算公式 |
|------|---------|---------|
| EV/EBITDA | 资本结构差异大的公司 | 企业价值 / EBITDA |
| EV/Revenue | 高成长、未盈利公司 | 企业价值 / 收入 |
| P/E | 盈利稳定的成熟公司 | 股价 / 每股收益 |
| P/B | 金融机构、重资产公司 | 股价 / 每股净资产 |

**分析输出**：
- 倍数范围（最低、最高、中位数、平均数）
- 隐含估值范围
- 相对于可比公司的溢价/折价分析

---

### 3. 先例交易分析（Precedent Transactions）

**定义**：先例交易分析是基于历史并购交易的估值倍数，评估目标公司在控制权变更情景下的价值。

**与可比公司分析的区别**：

| 特征 | 可比公司分析 | 先例交易分析 |
|------|------------|------------|
| 估值基础 | 少数股权市场价值 | 控制权溢价 |
| 数据来源 | 公开交易价格 | 已宣布交易价格 |
| 适用性 | 持续经营价值 | 并购/出售价值 |
| 溢价 | 通常较低 | 通常含 20-40% 控制权溢价 |

**交易筛选标准**：
- 同行业或相关行业
- 近期交易（通常 3-5 年内）
- 相似规模（交易价值）
- 相似的收购动机（战略 vs 财务）

**关键指标**：
- **交易价值/收入**：适用于所有公司
- **交易价值/EBITDA**：最常用指标
- **交易价值/EBIT**：适用于资本结构相似的公司
- **溢价百分比**：交易价格 vs 交易前股价

---

### 4. LBO 分析在 Pitch 中的应用

**定义**：在 Pitch 材料中展示 LBO（杠杆收购）分析，向客户（通常是私募股权基金）展示潜在的收购回报和可行性。

**LBO 模型关键输出**：

| 指标 | 说明 | 目标值 |
|------|------|--------|
| IRR（内部收益率） | 投资回报率 | 通常 20-30% |
| 现金回报倍数 | 退出价值/投资成本 | 通常 2-3x |
| 债务/EBITDA | 杠杆水平 | 通常 4-6x |
| 利息覆盖倍数 | EBITDA/利息支出 | 通常 > 2x |

**假设情景**：
- **基础情景**：保守假设，展示最低回报
- **上行情景**：乐观假设，展示最佳回报
- **下行情景**：压力测试，展示风险承受能力

**Pitch 中的展示**：
- 简化版 LBO 模型（通常 1-2 页）
- 关键假设和敏感性分析
- 与同类交易的回报比较

---

### 5. 品牌化和格式化

**定义**：品牌化是指将投行的视觉识别（Logo、颜色、字体）应用到 Pitch 材料中，确保专业性和一致性。

**品牌元素**：

| 元素 | 说明 | 规范 |
|------|------|------|
| Logo | 银行标志 | 标准尺寸和位置 |
| 配色 | 银行品牌色 | 主色、辅色、强调色 |
| 字体 | 标准字体族 | 标题、正文、注释字体 |
| 版式 | 页面布局 | 页边距、网格系统 |
| 图表风格 | 数据可视化 | 颜色、样式、标注 |

**格式化标准**：
- 每页标题清晰，内容简洁
- 数据来源标注完整
- 免责声明在每页底部
- 页码和机密性标注

### 6. 三级安全隔离详解

**研究层（researcher）**：
- **职责**：从 CapIQ 和 Daloopa 收集市场和公司数据
- **数据源**：可信的金融数据提供商
- **权限**：只读访问
- **安全**：数据源相对可信

**建模层（modeler）**：
- **职责**：在沙箱环境中构建财务模型
- **工具**：Read、Bash（沙箱化）
- **数据源**：CapIQ、Daloopa（只读）
- **安全**：沙箱环境限制系统访问

**材料生成层（deck-writer）**：
- **职责**：生成最终的 Pitch 材料 PowerPoint 和 Excel 模型
- **特点**：唯一拥有 Write 权限的节点
- **输出**：
  - `./out/pitch-<目标>.pptx`
  - `./out/model.xlsx`
- **安全**：
  - 不直接接触外部数据
  - 只处理经过验证的结构化数据
  - 通过 pptx-author 和 xlsx-author 技能生成文件

### 7. 产物隔离

**定义**：产物隔离是确保不同任务或客户的输出文件相互独立，防止信息泄露或混淆。

**隔离措施**：
- **目录隔离**：每个任务有独立的输出目录
- **命名规范**：文件名包含目标标识和时间戳
- **权限控制**：不同用户/任务只能访问自己的输出
- **清理机制**：任务完成后自动清理临时文件

**输出文件管理**：
```
./out/
├── pitch-TargetCo-20240315.pptx
├── model-TargetCo-20240315.xlsx
└── backup/
    ├── pitch-TargetCo-20240315-v1.pptx
    └── pitch-TargetCo-20240315-v2.pptx
```

### 8. 交接机制：从 Pitch 到建模

**触发场景**：
- Pitch 过程中投资论点发生重大变化
- 需要更新或重建财务模型
- 客户要求不同的估值情景

**交接流程**：
1. **Pitch Agent** 识别需要模型更新的需求
2. 生成指向 `model-builder` 的 `handoff_request`
3. 传递上下文信息：
   - 目标公司名称和股票代码
   - 最新投资论点
   - 关键假设变化
   - 需要的模型类型（DCF、LBO、Comps）
4. **Model Builder** 接收事件，构建或更新模型
5. 返回更新后的模型文件
6. **Pitch Agent** 将新模型纳入 Pitch 材料

**信息传递**：
- 通过 `handoff_request` 传递结构化数据
- 模型文件通过共享存储交换
- 编排脚本负责路由和验证
