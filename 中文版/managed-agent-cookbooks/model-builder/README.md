## 安全与交接

任务分解拆分 —— 输入来自可信的 MCP，因此拆分主要关注产物隔离和重新验证。只有一个工作节点持有 `Write`：

| 叶子节点 | 工具 | 连接器 |
|---|---|---|
| `data-puller` | `Read`、`Grep` | CapIQ、Daloopa（只读） |
| **`builder`**（写入持有者） | `Read`、`Write`、`Edit`、`Bash`（沙箱化） | 无 |
| `auditor` | `Read`、`Grep` | 无 |

`auditor` 在 `builder` 写入 `./out/model.xlsx` 后重新检查勾稽关系和平衡。

**交接：** 当从 `earnings-reviewer` 或 `pitch-agent` 调用时，调用代理的 `handoff_request` 由 `scripts/orchestrate.py` 路由至此。

## 金融术语和知识解释

### 1. 财务建模（Financial Modeling）

**定义**：财务建模是使用 Excel 等工具构建公司财务表现的数学表示，用于估值、预测和决策支持。

**主要模型类型**：

| 模型类型 | 用途 | 复杂度 |
|---------|------|--------|
| 三表模型（3-Statement Model） | 预测利润表、资产负债表、现金流量表 | 高 |
| DCF 模型 | 基于未来现金流折现估值 | 高 |
| LBO 模型 | 评估杠杆收购的可行性和回报 | 高 |
| 可比公司分析（Comps） | 基于市场倍数估值 | 中 |
| 并购模型（M&A Model） | 分析并购交易的财务影响 | 高 |
| 分部加总模型（Sum-of-the-Parts） | 对各业务分部分别估值后加总 | 中 |

**模型构建原则**：
- **一致性**：三张报表之间的勾稽关系必须正确
- **灵活性**：关键假设易于修改，自动更新
- **透明性**：公式清晰，便于审计和验证
- **稳健性**：极端情景下仍能合理运行

---

### 2. DCF 模型（折现现金流模型）

**定义**：DCF 模型通过预测公司未来自由现金流并将其折现到当前，计算公司的内在价值。

**核心公式**：
```
企业价值 = Σ [FCF_t / (1 + WACC)^t] + [终值 / (1 + WACC)^n]
```

**关键组成部分**：

**自由现金流（Free Cash Flow, FCF）**：
```
FCF = EBIT × (1 - 税率) + 折旧摊销 - 资本支出 - 营运资本增加
```

**折现率（WACC）**：
```
WACC = (E/V) × Re + (D/V) × Rd × (1 - 税率)
```
- E：股权价值
- D：债务价值
- V = E + D：企业总价值
- Re：股权成本（通常用 CAPM 计算）
- Rd：债务成本

**终值（Terminal Value）**：
```
终值 = FCF_n × (1 + g) / (WACC - g)
```
- g：永续增长率（通常 2-3%）

**建模步骤**：
1. 预测未来 5-10 年的收入和利润
2. 计算每年的自由现金流
3. 确定适当的 WACC
4. 计算预测期现金流现值
5. 计算终值并折现
6. 加总得到企业价值
7. 减去净债务得到股权价值
8. 除以流通股数得到每股价值

### 3. LBO 模型（杠杆收购模型）

**定义**：LBO 模型评估使用大量债务融资收购公司的可行性，分析投资者的回报率和偿债能力。

**LBO 结构**：

| 资本来源 | 典型比例 | 成本 |
|---------|---------|------|
| 银行贷款（Senior Debt） | 40-60% | 基准利率 + 利差 |
| 高收益债券（High Yield） | 20-30% | 8-12% |
| 夹层融资（Mezzanine） | 10-15% | 12-18% |
| 股权投资（Equity） | 20-30% | 预期 IRR 20-30% |

**关键指标**：

**投资回报率（IRR）**：
- 目标：通常 20-30%
- 计算：考虑退出时的股权价值
- 敏感性：对退出倍数和持有期敏感

**现金偿债倍数（Debt Service Coverage Ratio, DSCR）**：
```
DSCR = EBITDA / 债务本息支付
```
- 要求：通常 > 1.25x
- 意义：衡量偿债能力

**债务/EBITDA 比率**：
- 收购时：通常 4-6x
- 退出时：通常 2-3x
- 意义：衡量杠杆水平

**模型假设**：
- **持有期**：通常 3-7 年
- **退出倍数**：基于可比交易
- **收入增长**：保守假设 2-5%
- **成本节约**：协同效应量化

### 4. 三表模型（3-Statement Model）

**定义**：三表模型是同时预测利润表、资产负债表和现金流量表的集成财务模型。

**三张报表的勾稽关系**：

**利润表 → 资产负债表**：
- 净利润 → 留存收益增加
- 折旧费用 → 累计折旧增加
- 利息费用 → 债务余额变化

**资产负债表 → 现金流量表**：
- 营运资本变化 → 经营活动现金流
- 资本支出 → 投资活动现金流
- 债务发行/偿还 → 筹资活动现金流

**现金流量表 → 资产负债表**：
- 现金变化 → 现金及等价物余额
- 债务变化 → 长期/短期债务余额

**建模顺序**：
1. **利润表**：收入 → 成本 → 营业利润 → 净利润
2. **资产负债表**：资产 = 负债 + 权益
3. **现金流量表**：经营活动 + 投资活动 + 筹资活动
4. **循环引用**：现金余额影响利息收入/支出

**关键驱动因素**：

| 驱动因素 | 影响 | 典型假设来源 |
|---------|------|------------|
| 收入增长率 | 利润表、资产负债表 | 行业趋势、公司指引 |
| 毛利率 | 利润表 | 历史平均、竞争格局 |
| 营运资本天数 | 资产负债表、现金流 | 行业标准、公司政策 |
| 资本支出/收入 | 资产负债表、现金流 | 维护性 + 增长性 capex |
| 股息支付率 | 现金流量表、权益 | 公司政策、增长阶段 |

### 5. 可比公司分析（Trading Comps）

**定义**：可比公司分析是通过与同行业上市公司的估值倍数比较，评估目标公司价值的方法。

**企业价值（Enterprise Value, EV）计算**：
```
EV = 市值 + 总债务 - 现金及等价物 + 少数股东权益
```

**常用倍数**：

| 倍数 | 公式 | 适用场景 |
|------|------|---------|
| EV/EBITDA | 企业价值 / EBITDA | 资本结构差异大的公司 |
| EV/Revenue | 企业价值 / 收入 | 高成长、未盈利公司 |
| P/E | 市值 / 净利润 | 盈利稳定的成熟公司 |
| P/B | 市值 / 账面价值 | 金融机构、重资产公司 |

**可比公司筛选**：
1. **行业匹配**：同一 GICS/ICB 分类
2. **规模相近**：收入、市值差异 < 3x
3. **增长阶段**：相似的成长型/成熟型特征
4. **地理区域**：相似的市场环境
5. **财务特征**：相似的盈利能力和资本结构

**分析输出**：
- 倍数范围（最低、最高、中位数、平均数）
- 隐含估值范围
- 溢价/折价分析
- 敏感性分析

### 6. 三级安全隔离详解

**数据拉取层（data-puller）**：
- **职责**：从 CapIQ 和 Daloopa 获取标准化财务数据
- **数据源**：可信的金融数据提供商
- **权限**：只读访问，无法修改数据源
- **安全**：数据源相对可信，但仍需最小权限

**模型构建层（builder）**：
- **职责**：在 Excel 中构建财务模型
- **特点**：唯一拥有 Write 权限的节点
- **工具**：Read、Write、Edit、Bash（沙箱化）
- **安全**：
  - 沙箱环境限制系统访问
  - 只能写入指定的 `./out/` 目录
  - 无法访问外部网络或系统

**审计验证层（auditor）**：
- **职责**：验证模型的勾稽关系和计算正确性
- **权限**：只读访问，无法修改模型
- **检查项**：
  - 资产负债表是否平衡（资产 = 负债 + 权益）
  - 现金流量表与资产负债表勾稽是否正确
  - 公式中是否有硬编码数字
  - 关键假设是否合理

---

### 7. 模型审计要点

**勾稽关系检查**：

| 检查项 | 验证方法 | 常见问题 |
|--------|---------|---------|
| 资产负债表平衡 | 资产 = 负债 + 权益 | 循环引用错误 |
| 现金流勾稽 | 现金变动 = 三张表现金变动 | 分类错误 |
| 折旧一致性 | 利润表折旧 = 资产负债表累计折旧变动 | 遗漏 |
| 利息计算 | 利息费用 = 平均债务余额 × 利率 | 时间权重错误 |
| 股权变动 | 期末权益 = 期初权益 + 净利润 - 股息 | 遗漏其他综合收益 |

**质量检查清单**：
- [ ] 所有硬编码数字都有注释说明
- [ ] 关键假设集中管理，易于修改
- [ ] 公式引用正确，无断开链接
- [ ] 极端情景测试通过
- [ ] 输出格式符合公司标准

### 8. 沙箱环境（Sandbox）

**定义**：沙箱环境是隔离的执行环境，限制程序对系统资源的访问，防止恶意或错误操作影响主系统。

**沙箱限制**：
- **文件系统**：只能访问指定目录（`./out/`）
- **网络访问**：禁止外部网络连接
- **系统调用**：限制危险的系统调用
- **资源限制**：CPU、内存使用上限

**在模型构建中的应用**：
- 允许执行 Python/R 脚本进行复杂计算
- 防止脚本访问敏感系统文件
- 限制脚本的网络通信能力
- 确保所有输出都写入指定位置

### 9. 交接机制：从研究到建模

**触发场景**：
- 财报审核发现投资论点重大变化
- Pitch 材料需要更新估值模型
- 行业研究识别出值得建模的投资标的

**交接信息**：
```json
{
  "target_agent": "model-builder",
  "task_type": "dcf",
  "ticker": "AAPL",
  "context": {
    "investment_thesis": "服务业务增长加速",
    "key_assumptions": {
      "revenue_growth": "8% CAGR",
      "margin_expansion": "200bps"
    },
    "comparable_companies": ["MSFT", "GOOGL", "AMZN"]
  }
}
```

**建模流程**：
1. 接收交接事件和上下文
2. data-puller 获取最新财务数据
3. builder 构建财务模型
4. auditor 验证模型质量
5. 输出 `./out/model.xlsx`
6. 返回建模结果摘要

### 1. 什么是 Model Builder？

**一句话**：Model Builder 就是"模型建筑师"——它不是更新现有模型的数据，而是从零开始构建一个全新的、勾稽完美的财务模型。

**4 步骤**：

| 步骤 | 产出 |
|------|------|
| 数据拉取 | 从 CapIQ/Daloopa 拉取所有历史财务数据 |
| 模型构建 | 三表模型 → DCF/LBO/Comps 分析 |
| 模型审计 | 勾稽检查 → 合理性检查 → 错误修复 |
| 文件交付 | 格式化的 .xlsx 文件 |

---

### 给小白的一句话

> Model Builder 就是金融建模的"机器人建筑工"——从零构建一个财务模型在初级分析师手中需要 2 天时间（拉数据 4 小时、构建 8 小时、审计 4 小时）。Model Builder 把前面的机械化工作（拉数据、填公式、勾稽校验）自动化，让人集中精力做"建筑审查"：终值合理性？WACC 假设？隐含倍数落不落在同行区间内？好模型的精髓不是"公式多"——而是"每一个假设都有理由"。
