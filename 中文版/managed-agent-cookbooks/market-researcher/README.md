# 市场研究员 — 托管代理模板

## 概述

行业或主题 → 行业概览 → 竞争格局 → 同业对比 → 投资标的清单 → 研究报告。与 [`market-researcher`](../../plugins/agent-plugins/market-researcher) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export CAPIQ_MCP_URL=... FACTSET_MCP_URL=...
../../scripts/deploy-managed-agent.sh market-researcher
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。从研究队列事件启动，或在覆盖地图上分发。

## 安全与交接

第三方报告和发行人材料不可信。三级隔离：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| **`sector-reader`** | **是** | `Read`、`Grep` 仅限 | 无 |
| `comps-spreader` / 编排器 | 否 | `Read`、`Grep`、`Glob`、`Agent` | CapIQ、FactSet（只读） |
| **`note-writer`**（写入持有者） | 否 | `Read`、`Write`、`Edit` | 无 |

`sector-reader` 返回长度受限、模式验证的 JSON。`note-writer` 生成 `./out/primer-<行业>.docx`（如要求幻灯片则同时生成 `.pptx`）。

**交接：** 如需为投资标的清单中的单个公司建模，发出指向 `model-builder` 的 `handoff_request`；`scripts/orchestrate.py` 将其作为新引导事件路由。

## 金融术语和知识解释

### 1. 行业研究（Sector Research）

**定义**：行业研究是对特定经济部门的全面分析，包括市场规模、增长趋势、竞争格局、监管环境和关键成功因素。

**研究框架**：

| 维度 | 分析内容 | 数据来源 |
|------|---------|---------|
| 市场规模 | TAM、SAM、SOM | 行业报告、政府统计 |
| 增长趋势 | 历史增速、预测CAGR | 财务数据、专家访谈 |
| 竞争格局 | 市场份额、竞争对手 | 公司年报、行业报告 |
| 价值链 | 上下游关系、利润分配 | 供应链分析 |
| 监管环境 | 政策法规、准入壁垒 | 监管文件、法律文本 |
| 技术趋势 | 创新方向、颠覆风险 | 专利数据、技术报告 |

**TAM/SAM/SOM 模型**：
- **TAM（Total Addressable Market）**：总可及市场，理论上最大的市场机会
- **SAM（Serviceable Addressable Market）**：可服务可及市场，企业实际能触达的市场
- **SOM（Serviceable Obtainable Market）**：可获得服务市场，短期内实际能获取的市场份额

**示例**：
```
全球电动汽车市场
├── TAM: 3.5万亿美元（全球汽车市场）
├── SAM: 5000亿美元（电动汽车细分市场）
└── SOM: 50亿美元（本公司目标市场份额）
```

### 2. 竞争格局分析（Competitive Landscape）

**定义**：竞争格局分析是识别和评估行业内主要竞争对手的战略地位、优势和劣势的过程。

**分析工具**：

**波特五力模型**：
1. **现有竞争者**：行业竞争激烈程度
2. **潜在进入者**：新进入者的威胁
3. **替代品威胁**：替代产品/服务的风险
4. **供应商议价能力**：上游供应商的影响力
5. **买方议价能力**：下游客户的影响力

**战略群组分析**：
- 将竞争对手按战略特征分组
- 识别直接竞争者和间接竞争者
- 分析各群组的竞争策略

**竞争对手档案**：

| 要素 | 说明 |
|------|------|
| 公司概况 | 历史、规模、组织架构 |
| 财务表现 | 收入、利润、增长率 |
| 产品/服务 | 产品线、定价策略 |
| 市场定位 | 目标客户、价值主张 |
| 核心能力 | 技术优势、渠道优势 |
| 战略动向 | 并购、扩张、转型 |

---

### 3. 可比公司分析（Comparable Company Analysis, Comps）

**定义**：可比公司分析是通过与同行业类似公司的估值倍数比较，评估目标公司价值的方法。

**可比公司选择标准**：
- **行业相同**：同一细分行业或业务领域
- **规模相近**：收入、市值规模相似
- **增长阶段**：成长型、成熟型或衰退型
- **地理区域**：相似的市场环境和监管框架
- **财务特征**：相似的盈利能力和资本结构

**常用估值倍数**：

| 倍数 | 计算公式 | 适用场景 |
|------|---------|---------|
| P/E（市盈率） | 股价 / 每股收益 | 盈利稳定的成熟公司 |
| EV/EBITDA | 企业价值 / EBITDA | 资本结构差异大的公司 |
| EV/Revenue | 企业价值 / 收入 | 高成长、未盈利公司 |
| P/B（市净率） | 股价 / 每股净资产 | 重资产行业（银行、地产） |
| PEG | P/E / 盈利增长率 | 成长型公司 |

**分析步骤**：
1. **选择可比公司**：筛选 5-10 家最相似的公司
2. **收集数据**：获取财务数据和股价信息
3. **计算倍数**：为每家公司计算各项估值倍数
4. **确定范围**：找出倍数的最低值、最高值和中位数
5. **应用倍数**：用中位数或平均值乘以目标公司指标
6. **得出估值范围**：给出保守、基准和乐观三种情景

---

### 4. 投资标的清单（Ideas Shortlist）

**定义**：投资标的清单是经过初步筛选后，值得深入研究的投资机会列表。

**筛选流程**：

| 阶段 | 标准 | 输出数量 |
|------|------|---------|
| 初筛 | 行业趋势、基本面指标 | 50-100 家 |
| 定量筛选 | 估值、增长、质量指标 | 20-30 家 |
| 定性评估 | 竞争优势、管理层、ESG | 10-15 家 |
| 深度研究 | 财务模型、尽职调查 | 5-10 家 |
| 投资组合 | 风险收益、相关性 | 最终入选 |

**筛选指标**：

**估值指标**：
- P/E < 行业平均
- EV/EBITDA < 历史中位数
- P/B < 1.5
- 股息收益率 > 3%

**增长指标**：
- 收入 CAGR > 10%
- EPS CAGR > 15%
- 毛利率提升趋势

**质量指标**：
- ROE > 15%
- 负债率 < 50%
- 自由现金流为正
- 盈利稳定性

---

### 5. 研究报告（Research Note / Primer）

**定义**：研究报告是分析师对特定行业或公司的深入分析文档，为投资决策提供依据。

**报告类型**：

| 类型 | 目的 | 更新频率 |
|------|------|---------|
| 首次覆盖（Initiation） | 开始覆盖新公司 | 一次性 |
| 行业概览（Primer） | 行业全景分析 | 年度更新 |
| 财报点评（Earnings Review） | 季度业绩分析 | 季度 |
| 深度报告（Deep Dive） | 特定主题深入研究 | 不定期 |
| 快速报告（Flash Note） | 重大事件即时分析 | 事件驱动 |

**标准结构**：
1. **投资摘要**：核心观点、评级、目标价
2. **行业概览**：市场规模、增长驱动因素
3. **竞争分析**：主要玩家、市场份额
4. **公司分析**：商业模式、竞争优势
5. **财务分析**：历史表现、预测模型
6. **估值分析**：多种方法估值结果
7. **风险评估**：主要风险因素
8. **投资建议**：评级和目标价

### 6. 三级安全隔离详解

**行业资料读取层（sector-reader）**：
- **职责**：读取第三方研究报告、行业数据、新闻资讯
- **风险**：外部材料可能包含偏见、错误信息或恶意指令
- **防护**：
  - 只读权限，无法执行内容
  - 返回结构化 JSON，过滤原始格式
  - 长度限制和模式验证

**数据分析层（comps-spreader / Orchestrator）**：
- **职责**：执行可比分析、计算估值倍数、生成投资标的
- **权限**：调用 Agent 工具，访问 CapIQ 和 FactSet
- **连接器**：只读访问金融数据 MCP
- **安全**：基于结构化数据进行分析，不接触原始文档

**报告生成层（note-writer）**：
- **职责**：撰写研究报告，生成 Word 或 PowerPoint 文件
- **特点**：唯一拥有 Write 权限的节点
- **输出**：`./out/primer-<行业>.docx` 或 `.pptx`
- **安全**：不直接接触外部不可信材料

### 7. 数据来源与 MCP 连接器

**CapIQ（S&P Capital IQ）**：
- **定位**：综合金融数据平台
- **数据范围**：公司财务数据、估值倍数、交易数据
- **核心功能**：
  - 标准化财务数据（可比报表）
  - 估值模型和计算器
  - 可比公司筛选器
  - 并购交易数据库

**FactSet**：
- **定位**：机构投资者数据和分析平台
- **数据范围**：全球市场数据、研究报告、供应链数据
- **核心功能**：
  - 实时市场数据
  - 供应链关系图谱
  - 分析师预测数据
  - 投资组合分析工具

**数据使用原则**：
- 只读访问，防止意外修改
- 所有查询有审计日志
- 基于许可协议使用数据
- 遵守数据提供商的使用条款

---

### 8. 研究队列与覆盖地图

**研究队列（Research Queue）**：
- **定义**：待处理研究任务的有序列表
- **来源**：
  - 客户/投资组合经理请求
  - 市场事件触发（并购、业绩发布）
  - 定期研究计划
  - 风险监控告警
- **优先级**：紧急程度、客户重要性、市场影响

**覆盖地图（Coverage Map）**：
- **定义**：分析师负责覆盖的行业和公司分布图
- **维度**：
  - 行业分类（GICS、ICB）
  - 地理区域
  - 市值范围
  - 研究深度（核心覆盖 vs 边缘覆盖）
- **用途**：
  - 分配研究资源
  - 识别覆盖空白
  - 规划研究优先级

**分发机制**：
- 从队列或地图生成引导事件
- 自动分配给相应分析师或代理
- 跟踪任务进度和完成状态
- 汇总结果生成综合报告

---

### 9. 交接机制：从研究到建模

**触发场景**：
- 行业研究中发现具有投资价值的单个公司
- 需要为该公司建立详细的财务模型
- 进行深度估值分析

**交接流程**：
1. **市场研究员**完成行业分析
2. 识别值得深入研究的投资标的
3. 生成指向 `model-builder` 的 `handoff_request`
4. 传递关键信息：
   - 公司名称和股票代码
   - 初步投资论点
   - 关键假设和预测
   - 可比公司列表
5. **模型构建器**接收事件，开始建模
6. 生成 DCF、可比分析等估值模型
7. 返回估值结果和目标价格建议

**信息传递**：
- 上下文数据通过 `handoff_request` 传递
- 相关文件通过共享存储交换
- 编排脚本负责路由和验证

## Appendix: 金融背景知识

这份文件是"Market Researcher"托管代理模板的详细说明。Market Researcher 是行业/主题研究自动化引擎——当基金经理或分析师被问到"我想了解 XX 行业"时，自动生成一份行业概览报告初稿。

---

### 1. 什么是 Market Researcher？

**一句话**：Market Researcher 就是基金经理的"行业扫盲官"——当你需要快速了解一个陌生行业时，它帮你生成一份 10 页行业入门报告。

**4 步骤**：

| 步骤 | 产出 |
|------|------|
| 行业概览 | TAM（总市场规模）、增长率、关键驱动因素 |
| 竞争格局 | 主要玩家、市场份额、博弈态势 |
| 同行估值 | 可比公司的估值倍数分布 |
| 投资标的清单 | 值得关注的股票短名单 |

---

### 给小白的一句话

> Market Researcher 就是快速"行业调研外包"——当基金经理说"我们想看看中国的冷冻食品行业有没有机会"，初级分析师通常会花 3 天做行业扫盲：看 50 篇研报、提取关键数字、写一份入门报告。Market Researcher 30 分钟完成这个工作的初稿，你集中精力判断"这个行业的进入壁垒到底高不高？"——这是 AI 目前还做不好的深度判断。
