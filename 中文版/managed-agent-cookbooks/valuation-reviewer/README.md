# 估值审查员 — 托管代理模板

## 概述

接收 GP（普通合伙人）材料包、运行估值模板、准备 LP（有限合伙人）报告。与 [`valuation-reviewer`](../../plugins/agent-plugins/valuation-reviewer) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export PORTFOLIO_MCP_URL=...
../../scripts/deploy-managed-agent.sh valuation-reviewer
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。

## 安全与交接

GP 提供的估值材料包不可信。三级隔离：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| **`package-reader`** | **是** | `Read`、`Grep` 仅限 | 无 |
| `valuation-runner` / 编排器 | 否 | `Read`、`Grep`、`Glob`、`Agent` | portfolio（只读） |
| **`publisher`**（写入持有者） | 否 | `Read`、`Write`、`Edit` | 无 |

`package-reader` 返回长度受限、模式验证的 JSON。`publisher` 生成 `./out/lp-pack-<基金>.xlsx`。

**交接：** 如需将标记的投资组合公司输入总账对账流程，发出指向 `gl-reconciler` 的 `handoff_request`；`scripts/orchestrate.py` 路由。

**不保证：** LP 报告需要 IR（投资者关系）和 CCO（首席合规官）在代理外部签字确认。

## 金融术语和知识解释

### 1. 投资组合公司估值（Portfolio Company Valuation）

**定义**：投资组合公司估值是私募股权基金对其持有的非上市公司股权进行价值评估的过程，是基金 NAV 计算和 LP 报告的基础。

**估值频率**：
- **季度估值**：大多数基金每季度更新估值
- **年度审计**：年度财务报表需经外部审计
- **事件驱动**：重大事件（融资、并购）触发重新估值

**估值原则**：
- **一致性**：同一投资组合公司使用一致的估值方法
- **客观性**：基于可观察的市场数据
- **审慎性**：在不确定性下采取保守态度
- **文档化**：详细记录估值假设和计算过程

**估值委员会**：
- 负责审批投资组合公司估值
- 通常包括 GP 的投资合伙人、CFO 和合规官
- 每季度召开会议审查估值

---

### 2. GP 材料包（GP Package）

**定义**：GP 材料包是投资组合公司管理层或 GP 提供的用于估值的信息集合，包括财务数据、业务更新和市场信息。

**材料包内容**：

| 文档类型 | 说明 | 重要性 |
|---------|------|--------|
| 管理层报告 | 业务运营更新 | 高 |
| 财务报表 | 月度/季度财务数据 | 高 |
| 预算 vs 实际 | 业绩对比分析 | 中 |
| 市场数据 | 行业趋势、竞争动态 | 中 |
| 战略计划 | 未来增长计划 | 中 |
| 风险因素 | 业务风险更新 | 高 |

**材料包问题**：
- 管理层可能倾向于乐观估计
- 数据可能不完整或不及时
- 可能遗漏负面信息
- 格式和质量不统一

**验证措施**：
- 与历史数据对比检查一致性
- 与行业基准比较识别异常
- 要求管理层提供支持文档
- 独立进行市场调研

---

### 3. 估值模板（Valuation Template）

**定义**：估值模板是标准化的估值计算工具，确保所有投资组合公司使用一致的方法论和格式。

**模板组成**：

| 模块 | 内容 | 功能 |
|------|------|------|
| 输入页 | 财务数据、假设条件 | 数据录入 |
| 计算页 | 估值模型、公式 | 自动计算 |
| 输出页 | 估值结果、敏感性分析 | 结果展示 |
| 检查页 | 勾稽关系、合理性检查 | 质量控制 |

**常用估值方法**：

| 方法 | 适用场景 | 权重 |
|------|---------|------|
| 可比公司分析 | 有公开可比公司 | 30-50% |
| 先例交易分析 | 近期有并购交易 | 20-30% |
| DCF 模型 | 稳定现金流的公司 | 30-50% |
| 最近融资价格 | 近期有融资轮次 | 100%（如有） |

**估值加权**：
- 通常使用多种方法，给予不同权重
- 权重基于数据可靠性和适用性
- 最终估值为加权平均值

---

### 4. LP 报告准备

**定义**：LP 报告准备是将估值结果整理成标准格式，供投资者审阅的过程。

**报告内容**：

| 部分 | 说明 |
|------|------|
| 执行摘要 | 估值变更概览、关键驱动因素 |
| 投资组合摘要 | 各公司估值、占比、变动 |
| 详细估值 | 每家公司的估值方法和假设 |
| 市场分析 | 行业趋势、竞争格局 |
| 风险因素 | 主要风险和缓解措施 |
| 收益指标 | IRR、TVPI、DPI 等 |

**质量控制**：
- 估值方法符合 LPA 和监管要求
- 假设条件合理且有依据
- 计算准确，勾稽关系正确
- 格式统一，易于理解

### 5. 三级安全隔离详解

**材料读取层（package-reader）**：
- **职责**：读取 GP 提供的估值材料包
- **风险**：材料可能包含偏见、错误或不完整信息
- **防护**：
  - 只读权限，无法执行内容
  - 返回结构化 JSON
  - 长度限制和模式验证
  - 不直接传递给写入层

**估值运行层（valuation-runner / Orchestrator）**：
- **职责**：运行估值模板、计算估值结果
- **权限**：调用 Agent 工具，访问 portfolio MCP
- **连接器**：portfolio（只读访问）
- **安全**：基于结构化数据运行模型

**报告生成层（publisher）**：
- **职责**：生成 LP 报告 Excel 文件
- **特点**：唯一拥有 Write 权限的节点
- **输出**：`./out/lp-pack-<基金>.xlsx`
- **安全**：
  - 不直接修改 GP 提供的材料
  - 生成独立的报告文件
  - 需要人工审批后才能分发

---

### 6. Portfolio MCP

**定义**：Portfolio MCP 是连接估值审查代理与基金投资组合管理系统的只读接口。

**数据范围**：
- **投资组合信息**：持有的公司、股权比例、投资成本
- **历史估值**：过往季度的估值记录
- **现金流**：缴款、分配、费用记录
- **公司信息**：被投公司的基本资料和财务数据

**查询示例**：
```json
{
  "query_type": "portfolio_holdings",
  "fund": "Fund IV",
  "as_of_date": "2024-03-31",
  "include_valuation": true
}
```

**安全控制**：
- 只读访问，防止代理修改数据
- 基于角色的访问权限
- 审计日志记录所有查询
- 数据脱敏处理敏感信息

### 7. 交接机制：从估值到对账

**触发场景**：
- 估值审查发现投资组合公司财务数据异常
- 需要核实被投公司的财务记录
- 将标记的问题纳入总账对账流程

**交接流程**：
1. **Valuation Reviewer** 完成估值审查
2. 识别需要进一步核实的问题
3. 生成指向 `gl-reconciler` 的 `handoff_request`
4. 传递关键信息：
   - 投资组合公司名称
   - 异常事项描述
   - 相关财务数据
   - 建议的核实方向
5. **GL Reconciler** 接收事件，执行对账
6. 返回对账结果和发现

**信息传递**：
- 通过 `handoff_request` 传递结构化数据
- 相关文件通过共享存储交换
- 编排脚本负责路由和验证

---

### 8. 合规与审批

**IR（投资者关系）审批**：
- 审查报告内容的准确性和完整性
- 确保格式符合公司标准
- 验证披露信息的合规性
- 确认报告可以分发给 LP

**CCO（首席合规官）审批**：
- 审查估值方法的合规性
- 确保符合监管要求和 LPA 规定
- 检查利益冲突披露
- 确认报告符合反欺诈规定

**审批流程**：
1. Valuation Reviewer 生成初步报告
2. 投资团队审查估值假设
3. IR 团队审查报告格式和内容
4. CCO 审查合规性
5. 最终签字确认
6. 分发给 LP

**责任划分**：
- **GP**：对估值的准确性和公允性负最终责任
- **Valuation Reviewer**：提供初步分析和建议
- **IR**：确保报告质量和及时分发
- **CCO**：确保合规性和信息披露
