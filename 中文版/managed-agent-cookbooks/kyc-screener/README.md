# KYC 审查员 — 托管代理模板

## 概述

解析开户文件、运行规则引擎、筛查制裁名单/政治敏感人物（PEP）、标记缺失项。与 [`kyc-screener`](../../plugins/agent-plugins/kyc-screener) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export SCREENING_MCP_URL=...
../../scripts/deploy-managed-agent.sh kyc-screener
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。

## 安全与交接

开户文件不可信。三级隔离：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| **`doc-reader`** | **是** | `Read`、`Grep` 仅限 | 无 |
| `rules-engine` / 编排器 | 否 | `Read`、`Grep`、`Glob`、`Agent` | screening（只读） |
| **`escalator`**（写入持有者） | 否 | `Read`、`Write`、`Edit` | 无 |

`doc-reader` 返回长度受限、模式验证的 JSON。`escalator` 生成 `./out/escalation-<材料包>.xlsx`。

**不保证：** 本代理仅推荐风险评级；最终由合规官决定。

## 金融术语和知识解释

### 1. KYC（了解你的客户，Know Your Customer）

**定义**：KYC 是金融机构在建立业务关系前，对客户身份进行验证和尽职调查的一系列程序和法规要求。

**KYC 核心要素**：

| 要素 | 说明 | 验证方式 |
|------|------|---------|
| 身份识别（Identity） | 确认客户真实身份 | 身份证、护照、营业执照 |
| 地址验证（Address） | 确认客户居住或营业地址 | 水电账单、银行对账单 |
| 资金来源（Source of Funds） | 了解资金合法来源 | 工资单、税单、资产证明 |
| 业务目的（Business Purpose） | 了解开户目的和预期交易 | 客户声明、业务计划 |
| 受益所有人（UBO） | 识别最终控制人或受益人 | 股权结构图、声明书 |

**KYC 流程**：
1. **客户识别**：收集基本身份信息和文件
2. **尽职调查**：根据风险等级进行不同程度的调查
3. **风险评估**：评估客户洗钱和恐怖融资风险
4. **持续监控**：定期更新客户信息和交易监控
5. **记录保存**：按规定期限保存所有 KYC 记录

**法规依据**：
- **美国**：《爱国者法案》（USA PATRIOT Act）、FinCEN 规定
- **欧盟**：AMLD（反洗钱指令）
- **中国**：《反洗钱法》、《金融机构客户身份识别规定》
- **国际**：FATF（金融行动特别工作组）40 项建议

### 2. 制裁名单筛查（Sanctions Screening）

**定义**：制裁名单筛查是将客户信息与官方制裁名单进行比对，确保不与被制裁个人或实体建立业务关系。

**主要制裁名单**：

| 名单 | 发布机构 | 适用范围 |
|------|---------|---------|
| SDN 名单 | 美国财政部 OFAC | 全球 |
| 欧盟制裁名单 | 欧盟理事会 | 欧盟成员国 |
| 联合国制裁名单 | 联合国安理会 | 全球 |
| 英国制裁名单 | 英国财政部 OFSI | 英国 |
| 中国制裁名单 | 中国外交部 | 中国 |

**筛查类型**：
- **姓名筛查**：客户姓名 vs 制裁名单
- **实体筛查**：公司名、品牌名 vs 制裁实体
- **地址筛查**：营业地址 vs 制裁地区
- ** vessel 筛查**：船舶 IMO 编号 vs 制裁船舶

**匹配算法**：
- **精确匹配**：完全一致的姓名或标识
- **模糊匹配**：考虑拼写变体、音译差异
- **别名匹配**：识别化名、昵称、曾用名
- **关联匹配**：识别与被制裁人员的关联关系

**误报处理**：
- 常见姓名（如"王磊"、"Mohammed"）容易产生大量误报
- 需要人工复核确认真实匹配
- 记录误报处理理由和决策

### 3. PEP（政治敏感人物，Politically Exposed Person）

**定义**：PEP 是指担任或曾担任重要公共职务的个人及其直系亲属和密切关联人，由于其职位可能带来的腐败风险，需要加强尽职调查。

**PEP 分类**：

| 类别 | 说明 | 示例 |
|------|------|------|
| 外国 PEP | 非本国的重要公职人员 | 外国元首、部长、法官 |
| 国内 PEP | 本国的重要公职人员 | 总统、总理、央行行长 |
| 国际组织 PEP | 国际组织高级官员 | 联合国官员、世界银行高管 |
| 前 PEP | 已卸任但仍具影响力的前官员 | 前总统、前部长 |
| 家庭成员 | PEP 的直系亲属 | 配偶、子女、父母 |
| 密切关联人 | 与 PEP 关系密切的个人 | 商业伙伴、顾问 |

**PEP 风险因素**：
- **职位级别**：职位越高，风险越大
- **国家风险**：来自腐败高风险国家的 PEP 风险更高
- **业务关系**：与 PEP 的业务关系越密切，风险越大
- **交易特征**：大额、频繁、复杂的交易需要额外关注

**PEP 管理措施**：
1. **识别**：在开户时识别客户是否为 PEP
2. **审批**：PEP 客户开户需高级管理层审批
3. **加强监控**：对 PEP 账户进行持续交易监控
4. **定期审查**：每年重新评估 PEP 风险状况
5. **记录保存**：详细记录 PEP 相关决策和措施

### 4. 规则引擎（Rules Engine）

**定义**：规则引擎是根据预设的业务规则自动评估客户风险和处理逻辑的系统组件。

**规则类型**：

| 规则类别 | 说明 | 示例 |
|---------|------|------|
| 必填项规则 | 确保所有必要信息已收集 | 身份证、地址证明必须提供 |
| 格式规则 | 验证数据格式正确性 | 身份证号 18 位，护照号格式 |
| 一致性规则 | 检查数据之间的一致性 | 出生日期与身份证一致 |
| 风险评分规则 | 根据特征计算风险分数 | 高风险国家 +10 分 |
| 业务规则 | 特定业务场景的判断逻辑 | 企业客户必须提供营业执照 |

**规则执行流程**：
1. **加载规则**：从规则库加载适用的规则集
2. **数据输入**：接收客户信息和文档
3. **规则评估**：逐条评估规则条件
4. **结果汇总**：汇总所有规则执行结果
5. **决策输出**：生成通过/拒绝/需补充的决策

**规则维护**：
- 定期更新规则以反映法规变化
- 根据误报率调整规则阈值
- 记录规则变更历史和原因

### 5. 风险评级（Risk Rating）

**定义**：风险评级是根据客户特征、行为和背景信息，对其洗钱和恐怖融资风险进行分级评估的结果。

**典型风险等级**：

| 等级 | 颜色 | 说明 | 措施 |
|------|------|------|------|
| 低风险 | 绿色 | 普通个人客户，本地居民 | 标准尽职调查 |
| 中风险 | 黄色 | 小企业主、非居民 | 加强尽职调查 |
| 高风险 | 橙色 | PEP、高风险国家 | 强化尽职调查 |
| 禁止 | 红色 | 制裁名单匹配 | 拒绝开户/终止关系 |

**风险因素权重**：

| 因素 | 权重 | 说明 |
|------|------|------|
| 客户类型 | 高 | 个人 vs 企业 vs 金融机构 |
| 地理风险 | 高 | 所在国家/地区的洗钱风险 |
| 业务性质 | 中 | 行业类型和业务模式 |
| 交易特征 | 中 | 预期交易金额和频率 |
| 渠道风险 | 低 | 开户渠道（线上/线下） |

**评级调整**：
- **升级**：发现新的风险因素时提高风险等级
- **降级**：风险因素消除且观察期结束后降低等级
- **定期重评**：至少每年重新评估一次风险等级

---

### 6. 开户文件类型

**个人客户**：

| 文件类型 | 用途 | 验证要点 |
|---------|------|---------|
| 身份证/护照 | 身份验证 | 真实性、有效期、照片一致性 |
| 地址证明 | 地址验证 | 近期（3 个月内）、姓名一致 |
| 收入证明 | 资金来源 | 雇主信、工资单、税单 |
| 银行对账单 | 财务状况 | 交易历史、余额 |

**企业客户**：

| 文件类型 | 用途 | 验证要点 |
|---------|------|---------|
| 营业执照 | 合法经营 | 真实性、有效期、经营范围 |
| 公司章程 | 治理结构 | 股东信息、董事会成员 |
| 股权结构图 | 受益所有人 | 最终控制人、持股比例 |
| 财务报表 | 财务状况 | 审计报告、近期业绩 |
| 董事会决议 | 开户授权 | 签字人权限、决议有效性 |

### 7. 三级安全隔离详解

**文档读取层（doc-reader）**：
- **职责**：读取客户提交的开户文件
- **风险**：文件可能包含恶意指令或隐藏代码
- **防护**：
  - 只读权限，无法执行文件内容
  - 返回结构化 JSON，不保留原始格式
  - 长度限制防止内存溢出攻击
  - 模式验证确保输出安全

**规则处理层（rules-engine / Orchestrator）**：
- **职责**：运行规则引擎、查询筛查系统
- **权限**：可以调用外部筛查 API，但只读
- **连接器**：screening MCP（只读访问）
- **安全**：不直接接触原始文档，只处理结构化数据

**结果输出层（escalator）**：
- **职责**：生成审查结果和升级报告
- **特点**：唯一拥有 Write 权限的节点
- **输出**：`./out/escalation-<材料包>.xlsx`
- **安全**：从不接触原始开户文件

### 8. 合规官（Compliance Officer）

**定义**：合规官是负责确保金融机构遵守所有适用法律法规和内部政策的高级管理人员。

**职责范围**：
- **政策制定**：制定和更新反洗钱/反恐融资政策
- **风险评估**：定期评估机构的洗钱风险
- **监控报告**：监督交易监控系统的运行
- **调查配合**：配合监管机构的调查
- **培训教育**：组织员工合规培训

**独立性要求**：
- 合规官应独立于业务部门
- 直接向董事会或高级管理层报告
- 拥有足够的资源和权限履行职责

**KYC 中的角色**：
- 审批高风险客户开户
- 审查可疑交易报告（STR）
- 决定是否需要向监管机构报告
- 处理监管机构的查询和检查

---

### 9. 持续监控（Ongoing Monitoring）

**定义**：持续监控是在客户关系存续期间，定期审查客户信息和交易行为，及时发现风险变化的过程。

**监控内容**：
- **信息更新**：客户地址、职业、联系方式变更
- **交易监控**：异常交易模式识别
- **名单筛查**：定期重新筛查制裁名单
- **风险重评**：定期重新评估客户风险等级

**触发事件**：
- 客户信息变更
- 大额或异常交易
- 负面新闻或监管行动
- 定期审查到期

**监控频率**：

| 风险等级 | 审查频率 | 交易监控 |
|---------|---------|---------|
| 低风险 | 每 3-5 年 | 抽样监控 |
| 中风险 | 每 1-2 年 | 定期监控 |
| 高风险 | 每 6-12 个月 | 实时监控 |
