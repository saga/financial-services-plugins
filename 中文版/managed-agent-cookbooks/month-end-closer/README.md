# 月末结算员 — 托管代理模板

## 概述

应计项目、滚动预测、差异分析。与 [`month-end-closer`](../../plugins/agent-plugins/month-end-closer) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export GL_MCP_URL=...
../../scripts/deploy-managed-agent.sh month-end-closer
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。

## 安全与交接

支持性发票和供应商对账单不可信。三级隔离：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| **`ledger-reader`** | **是** | `Read`、`Grep` 仅限 | 无 |
| `rollforward` / 编排器 | 否 | `Read`、`Grep`、`Glob`、`Agent` | internal-gl（只读） |
| **`poster`**（写入持有者） | 否 | `Read`、`Write`、`Edit` | 无 |

`poster` 生成 `./out/close-package-<实体>-<期间>.xlsx`。日记账分录草稿为暂记状态，不直接过账到总账。

**交接：** 接收来自 `gl-reconciler` 的 `handoff_request` 事件，将已验证的差异纳入结算分析。

## 金融术语和知识解释

### 1. 月末结算（Month-End Close）

**定义**：月末结算是在每个会计期间结束时，完成所有财务记录、调整和报告的过程，确保财务报表准确反映公司财务状况。

**标准结算流程**：

| 步骤 | 任务 | 负责角色 | 时间 |
|------|------|---------|------|
| 1 | 数据收集 | 会计 | 期间结束后 1-2 天 |
| 2 | 对账 | 对账员 | 期间结束后 2-3 天 |
| 3 | 应计调整 | 会计 | 期间结束后 3-4 天 |
| 4 | 差异分析 | 财务分析师 | 期间结束后 4-5 天 |
| 5 | 管理层审查 | 财务总监 | 期间结束后 5-6 天 |
| 6 | 期间关闭 | 系统管理员 | 期间结束后 6-7 天 |
| 7 | 报告生成 | 财务报告 | 期间结束后 7-10 天 |

**结算周期**：
- **快速结算**：3-5 个工作日（最佳实践）
- **标准结算**：5-10 个工作日
- **延迟结算**：>10 个工作日（存在效率问题）

**结算挑战**：
- 数据分散在多个系统
- 手工对账耗时且易出错
- 跨时区业务增加复杂性
- 监管要求日益严格

---

### 2. 应计项目（Accruals）

**定义**：应计项目是在权责发生制会计下，记录已发生但尚未收到或支付现金的收入和费用。

**应计原则**：
- **收入确认**：当服务已提供或商品已交付时确认收入，不论是否收到现金
- **费用匹配**：费用应在相关收入确认的同一期间记录
- **重要性**：确保财务报表准确反映期间经营成果

**常见应计项目**：

| 类型 | 说明 | 分录示例 |
|------|------|---------|
| 应计费用 | 已发生未支付的费用 | 借：工资费用 / 贷：应付工资 |
| 应计收入 | 已赚取未收到的收入 | 借：应收账款 / 贷：收入 |
| 预付费用 | 已支付未发生的费用 | 借：预付费用 / 贷：现金 |
| 预收收入 | 已收到未赚取的收入 | 借：现金 / 贷：预收收入 |
| 折旧摊销 | 长期资产成本分摊 | 借：折旧费用 / 贷：累计折旧 |
| 坏账准备 | 应收账款减值 | 借：坏账费用 / 贷：坏账准备 |

**应计时间表（Accrual Schedule）**：
- 记录所有需要期末调整的项目
- 包括金额、计算依据和支持文档
- 作为审计轨迹的重要组成部分

---

### 3. 滚动预测（Roll-forward）

**定义**：滚动预测是在每个会计期间结束时，更新未来期间预测的过程，将实际结果纳入并调整未来预期。

**滚动预测 vs 年度预算**：

| 特征 | 滚动预测 | 年度预算 |
|------|---------|---------|
| 更新频率 | 每月/每季度 | 每年一次 |
| 时间跨度 | 始终向前滚动 12 个月 | 固定财年 |
| 灵活性 | 高，可随时调整 | 低，年中不易调整 |
| 准确性 | 更高，基于最新信息 | 可能过时 |
| 用途 | 运营决策 | 目标设定、绩效考核 |

**滚动预测流程**：
1. **实际数据录入**：将本月实际结果填入预测模型
2. **差异分析**：比较实际 vs 预测，分析原因
3. **假设更新**：根据最新信息调整关键假设
4. **预测修订**：更新未来月份的预测值
5. **情景分析**：考虑乐观、基准、悲观三种情景

**关键驱动因素**：
- 历史趋势和季节性
- 管理层业绩指引
- 宏观经济指标
- 行业动态和竞争格局
- 公司战略举措

---

### 4. 差异分析（Variance Analysis）

**定义**：差异分析是比较实际财务结果与预算或预测的差异，并分析差异产生原因的管理工具。

**差异类型**：

| 差异 | 定义 | 计算公式 |
|------|------|---------|
| 有利差异（Favorable） | 实际结果优于预期 | 实际收入 > 预算收入 |
| 不利差异（Unfavorable） | 实际结果差于预期 | 实际费用 > 预算费用 |
| 量差（Volume Variance） | 销量变化导致的差异 | (实际量 - 预算量) × 预算单价 |
| 价差（Price Variance） | 价格变化导致的差异 | (实际价 - 预算价) × 实际量 |
| 效率差异（Efficiency） | 投入产出效率变化 | (实际投入 - 标准投入) × 标准价格 |

**差异分析框架**：

**收入差异分解**：
```
总收入差异 = 量差 + 价差
          = (实际销量 - 预算销量) × 预算单价
          + (实际单价 - 预算单价) × 实际销量
```

**成本差异分解**：
```
总成本差异 = 量差 + 价差 + 效率差异
```

**分析步骤**：
1. 计算各项差异金额和百分比
2. 识别最大差异项（帕累托原则）
3. 调查差异根因（内部/外部因素）
4. 评估对未来期间的影响
5. 提出改进建议或调整措施

### 5. 日记账分录草稿（JE Drafts）

**定义**：日记账分录草稿是月末结算过程中编制的初步会计分录，需要经过审批后才能正式过账。

**草稿状态**：
- **暂记（Staged）**：已编制但未审批
- **待审批（Pending Approval）**：等待财务总监审批
- **已审批（Approved）**：可以正式过账
- **已过账（Posted）**：已录入总账系统
- **已拒绝（Rejected）**：需要修改或撤销

**审批权限**：

| 分录类型 | 审批人 | 金额阈值 |
|---------|--------|---------|
| 常规调整 | 会计经理 | < 10,000 |
| 重大调整 | 财务总监 | 10,000 - 100,000 |
| 非常规调整 | CFO | > 100,000 |

**审计要求**：
- 每笔分录需有完整的支持文档
- 分录原因必须清晰说明
- 审批签字不可缺失
- 所有修改需保留审计轨迹

---

### 6. 结算材料包（Close Package）

**定义**：结算材料包是月末结算完成后生成的完整文档集合，包含所有调整、分析和报告。

**材料包内容**：

| 文档 | 说明 | 用途 |
|------|------|------|
| 试算平衡表 | 所有科目的借贷余额 | 验证账务平衡 |
| 资产负债表 | 资产、负债、权益快照 | 财务状况 |
| 利润表 | 收入、费用、利润 | 经营成果 |
| 现金流量表 | 经营、投资、筹资现金流 | 现金管理 |
| 差异分析报告 | 实际 vs 预算/预测 | 业绩评估 |
| 应计调整表 | 所有期末调整分录 | 审计支持 |
| 对账报告 | 银行、往来账对账结果 | 准确性验证 |
| 管理报告 | 关键指标和管理层评论 | 决策支持 |

**材料包格式**：
- **Excel 工作簿**：数据表格、计算模型
- **PDF 报告**：正式报告、签字页
- **支持文档**：发票、合同、对账单扫描件

### 7. 三级安全隔离详解

**文档读取层（ledger-reader）**：
- **职责**：读取供应商发票、银行对账单等外部文档
- **风险**：外部文档可能包含错误信息或恶意内容
- **防护**：
  - 只读权限，无法执行内容
  - 返回结构化 JSON
  - 长度限制和模式验证

**数据处理层（rollforward / Orchestrator）**：
- **职责**：执行应计计算、差异分析、预测更新
- **权限**：调用 Agent 工具，访问内部总账 MCP
- **连接器**：internal-gl（只读访问）
- **安全**：基于结构化数据进行分析

**结果输出层（poster）**：
- **职责**：生成结算材料包 Excel 文件
- **特点**：唯一拥有 Write 权限的节点
- **输出**：`./out/close-package-<实体>-<期间>.xlsx`
- **安全**：
  - 不直接修改总账系统
  - 分录草稿为暂记状态
  - 需要人工审批后才能过账

### 8. 总账 MCP（GL MCP）

**定义**：总账 MCP 是连接月末结算代理与企业总账系统的只读接口，提供财务数据查询能力。

**数据范围**：
- **科目余额**：各会计科目的期初、本期发生、期末余额
- **交易明细**：按科目、期间、实体筛选的明细记录
- **多币种数据**：原币和本位币金额
- **维度数据**：按部门、项目、地区等维度的分解

**查询示例**：
```json
{
  "query_type": "account_balance",
  "entity": "US001",
  "period": "2024-03",
  "account": "1100-Cash",
  "currency": "USD"
}
```

**安全控制**：
- 只读访问，防止代理修改财务数据
- 基于角色的数据访问权限
- 所有查询记录审计日志
- 网络隔离，仅限内部访问

### 9. 交接机制：从对账到结算

**触发场景**：
- GL Reconciler 完成对账，发现已验证的差异
- 需要将差异纳入月末结算调整
- 生成完整的结算材料包

**交接信息**：
```json
{
  "source_agent": "gl-reconciler",
  "target_agent": "month-end-closer",
  "handoff_type": "verified_breaks",
  "data": {
    "entity": "US001",
    "period": "2024-03",
    "breaks": [
      {
        "break_id": "BRK001",
        "account": "1200-AR",
        "amount": 15000,
        "root_cause": "遗漏发票",
        "proposed_je": "借：应收账款 15000 / 贷：收入 15000"
      }
    ]
  }
}
```

**结算流程**：
1. 接收已验证的差异列表
2. 将差异纳入应计调整
3. 执行滚动预测更新
4. 进行差异分析
5. 生成结算材料包
6. 提交管理层审查
