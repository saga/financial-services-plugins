# 报表审计员 — 托管代理模板

## 概述

分发前审计预生成的 LP（有限合伙人）报表。与 [`statement-auditor`](../../plugins/agent-plugins/statement-auditor) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export NAV_MCP_URL=...
../../scripts/deploy-managed-agent.sh statement-auditor
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。

## 安全与交接

生成的报表被视为不可信（上游系统超出范围）。三级隔离：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| **`statement-reader`** | **是** | `Read`、`Grep` 仅限 | 无 |
| `reconciler` / 编排器 | 否 | `Read`、`Grep`、`Glob`、`Agent` | nav（只读） |
| **`flagger`**（写入持有者） | 否 | `Read`、`Write`、`Edit` | 无 |

`flagger` 生成 `./out/signoff-<批次>.xlsx`。

**不保证：** 本代理仅推荐通过/暂缓；IR（投资者关系）在人工签字确认后分发。

## 金融术语和知识解释

### 1. LP 报表（Limited Partner Statement）

**定义**：LP 报表是私募股权基金向有限合伙人（投资者）定期提供的报告，详细说明基金的投资表现、资产净值和分配情况。

**报表类型**：

| 报表类型 | 频率 | 主要内容 |
|---------|------|---------|
| 季度报告 | 每季度 | 投资组合更新、关键指标、市场评论 |
| 年度财务报告 | 每年 | 经审计的财务报表、详细附注 |
| 资本账户报表 | 每季度/每年 | 各 LP 的资本账户变动 |
| 分配通知 | 分配时 | 收益分配详情、税务信息 |
| K-1 表格 | 每年 | 美国税务申报用合伙人收入份额 |

**核心内容**：
- **投资组合摘要**：各投资项目的估值和表现
- **资产净值（NAV）**：基金总净值和每股/单位净值
- **收益指标**：IRR、倍数、 DPI、TVPI
- **现金流**：缴款（Capital Call）和分配（Distribution）
- **费用报告**：管理费、业绩报酬（Carried Interest）

---

### 2. 私募股权基金结构

**定义**：私募股权基金是一种投资工具，由普通合伙人（GP）管理，向有限合伙人（LP）募集资金，投资于非上市公司。

**参与方**：

| 角色 | 职责 | 风险/收益 |
|------|------|----------|
| 普通合伙人（GP） | 基金管理、投资决策 | 承担无限责任，收取管理费和业绩报酬 |
| 有限合伙人（LP） | 出资、监督 | 承担有限责任，获得收益分配 |
| 基金管理人 | 日常运营、合规 | 收取管理费 |
| 投资顾问 | 投资建议、尽职调查 | 咨询费 |

**基金生命周期**：

| 阶段 | 时长 | 活动 |
|------|------|------|
| 募集期 | 1-2 年 | 向投资者募集资金 |
| 投资期 | 3-5 年 | 寻找、评估、执行投资 |
| 持有期 | 3-7 年 | 管理投资组合公司 |
| 退出期 | 2-4 年 | 通过 IPO、并购等方式退出 |

**收益分配瀑布（Waterfall）**：
1. **返还本金**：先返还 LP 的投资本金
2. **优先回报（Preferred Return）**：通常 8% 的年化回报
3. **追赶条款（Catch-up）**：GP 获得部分业绩报酬
4. ** carried interest**：剩余收益按 80/20 分配（LP/GP）

---

### 3. NAV（资产净值，Net Asset Value）

**定义**：NAV 是基金资产减去负债后的净值，代表基金的真实价值。

**计算公式**：
```
NAV = 投资组合市值 + 现金及等价物 + 应收款项 - 应付款项 - 其他负债
```

**NAV 构成**：

| 项目 | 说明 | 估值方法 |
|------|------|---------|
| 投资资产 | 持有的股权/债权投资 | 市场法、收益法、成本法 |
| 现金 | 银行存款、货币基金 | 账面价值 |
| 应收管理费 | LP 欠缴的管理费 | 预期可收回金额 |
| 应付费用 | 待支付的费用 | 账面价值 |

**估值方法**：

| 方法 | 适用场景 | 特点 |
|------|---------|------|
| 市场法 | 有近期交易或可比公司 | 客观，但可能缺乏流动性折价 |
| 收益法（DCF） | 稳定现金流的成熟公司 | 理论严谨，但对假设敏感 |
| 成本法 | 早期投资或困境公司 | 保守，但可能低估价值 |
| 最近融资价格 | 近期有融资轮次 | 反映最新市场定价 |

**NAV 核对（NAV Tie-out）**：
- 验证 NAV 计算的正确性
- 核对各投资项目的估值
- 验证现金流记录的准确性
- 检查费用计算的合规性

---

### 4. 收益指标

**定义**：收益指标是衡量私募股权基金投资表现的量化标准。

**核心指标**：

| 指标 | 全称 | 计算公式 | 说明 |
|------|------|---------|------|
| IRR | 内部收益率 | 使 NPV=0 的折现率 | 考虑时间价值的年化回报 |
| DPI | 实付倍数 | 累计分配 / 累计缴款 | 已变现回报的倍数 |
| TVPI | 总倍数 | (累计分配 + NAV) / 累计缴款 | 总回报的倍数 |
| RVPI | 未实现倍数 | NAV / 累计缴款 | 未变现回报的倍数 |
| MOIC | 投资资本倍数 | 总价值 / 投资成本 | 简单的回报倍数 |

**指标解读**：

**IRR**：
- **优秀**：> 25%
- **良好**：15-25%
- **一般**：10-15%
- **较差**：< 10%

**DPI**：
- **> 1.0x**：已收回全部本金
- **< 1.0x**：尚未收回本金
- 早期基金 DPI 通常较低（投资期）

**TVPI**：
- **> 2.0x**：总回报超过 2 倍
- 随基金成熟逐渐从 RVPI 转向 DPI

### 5. 审计流程

**定义**：LP 报表审计是在向投资者分发前，对报表的准确性、完整性和合规性进行验证的过程。

**审计步骤**：

| 步骤 | 任务 | 验证内容 |
|------|------|---------|
| 1 | 数据完整性 | 所有投资项目都有记录 |
| 2 | 估值核对 | 估值方法正确，计算无误 |
| 3 | 现金流核对 | 缴款和分配记录准确 |
| 4 | 费用计算 | 管理费和业绩报酬计算正确 |
| 5 | NAV 计算 | 资产净值计算正确 |
| 6 | 收益指标 | IRR、DPI、TVPI 计算正确 |
| 7 | 合规检查 | 符合 LPA（有限合伙协议）规定 |
| 8 | 格式审查 | 报表格式符合标准 |

**常见错误**：
- 估值日期不一致
- 汇率转换错误
- 费用计算基数错误
- 收益指标公式错误
- 遗漏投资项目

### 6. 三级安全隔离详解

**报表读取层（statement-reader）**：
- **职责**：读取预生成的 LP 报表
- **风险**：报表可能包含错误或恶意内容
- **防护**：
  - 只读权限，无法执行内容
  - 返回结构化 JSON
  - 长度限制和模式验证

**核对层（reconciler / Orchestrator）**：
- **职责**：核对报表数据与 NAV 系统数据
- **权限**：调用 Agent 工具，访问 NAV MCP
- **连接器**：nav（只读访问）
- **安全**：基于可信系统数据进行核对

**结果标记层（flagger）**：
- **职责**：生成审计结果和签字文件
- **特点**：唯一拥有 Write 权限的节点
- **输出**：`./out/signoff-<批次>.xlsx`
- **安全**：
  - 不直接修改原始报表
  - 生成独立的审计文件
  - 推荐通过/暂缓，不直接分发

### 7. 投资者关系（IR）

**定义**：投资者关系是基金管理人与投资者之间的沟通管理职能，负责信息披露、投资者沟通和关系维护。

**IR 职责**：
- **报告分发**：按时向 LP 提供定期报告
- **查询响应**：回答投资者的问题和关切
- **会议组织**：安排年度大会、季度电话会
- **信息披露**：及时披露重大事件
- **关系维护**：建立和维护投资者关系

**报告分发流程**：
1. 基金会计生成初步报表
2. Statement Auditor 审计报表
3. 财务总监审查并批准
4. IR 团队准备分发材料
5. 通过安全渠道分发给 LP
6. 确认接收并记录

**合规要求**：
- 按 LPA 规定的时间和内容提供报告
- 确保所有投资者同时收到相同信息
- 保密处理敏感信息
- 保留分发记录

---

### 8. 有限合伙协议（LPA）

**定义**：LPA 是规范私募股权基金运作的法律文件，详细规定了 GP 和 LP 的权利、义务和收益分配机制。

**关键条款**：

| 条款 | 说明 | 典型内容 |
|------|------|---------|
| 基金期限 | 基金存续期 | 通常 10 年，可延长 2-3 年 |
| 投资限制 | 投资范围和限制 | 行业、地域、单一项目上限 |
| 管理费 | GP 收取的管理费 | 承诺资本的 1.5-2% / 年 |
| 业绩报酬 | Carried Interest | 超额收益的 20% |
| 分配机制 | 收益分配顺序 | 瀑布结构 |
| 报告义务 | 向 LP 提供报告 | 季度报告、年度审计报告 |
| 关键人条款 | 关键人员离职处理 | 暂停投资、LP 有权终止 |

**审计与 LPA 的关系**：
- 审计确保报表符合 LPA 规定
- 费用计算遵循 LPA 的费率结构
- 分配机制符合 LPA 的瀑布条款
- 报告时间和内容符合 LPA 要求
