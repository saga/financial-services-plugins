# 会议准备代理 — 托管代理模板

## 概述

每次客户会议前的简报材料准备。与 [`meeting-prep-agent`](../../plugins/agent-plugins/meeting-prep-agent) Cowork 插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export CRM_MCP_URL=... CAPIQ_MCP_URL=...
../../scripts/deploy-managed-agent.sh meeting-prep-agent
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。通常由您的工作流引擎根据日历事件自动触发。

## 安全与交接

客户提供的文件和 inbound 邮件不可信。三级拆分：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| `profiler` | 否 | `Read`、`Grep` | CRM、CapIQ（只读） |
| **`news-reader`** | **是** | `Read`、`Grep` 仅限 | 无 |
| **`pack-writer`**（写入持有者） | 否 | `Read`、`Write`、`Edit` | 无 |

`pack-writer` 生成 `./out/briefing-<客户>.pptx`；它从不直接打开客户提供的文件内容。

**不保证：** 本材料包供顾问使用，不面向客户发送。无客户-facing 发送。

## 金融术语和知识解释

### 1. 客户会议简报（Client Meeting Briefing）

**定义**：客户会议简报是财富顾问在会见客户前准备的信息材料，帮助顾问全面了解客户情况、市场动态和讨论要点。

**简报内容**：

| 模块 | 内容 | 目的 |
|------|------|------|
| 客户档案 | 基本信息、家庭状况、职业 | 建立个人联系 |
| 投资组合 | 持仓明细、业绩表现、风险敞口 | 评估投资状况 |
| 市场动态 | 相关新闻、市场走势、行业事件 | 提供谈话素材 |
| 待办事项 | 上次会议决议、待处理事项 | 跟进服务承诺 |
| 机会识别 | 新产品、市场机会、税务优化 | 提供增值服务 |
| 风险提醒 | 集中度风险、市场下行风险 | 风险提示和管理 |

**简报价值**：
- 提升顾问专业形象和准备度
- 确保会议高效、有针对性
- 避免遗漏重要事项
- 增强客户信任感和满意度

---

### 2. CRM（客户关系管理）系统

**定义**：CRM 系统是管理客户信息、交互记录和服务历史的软件平台，是财富管理的核心工具。

**核心功能**：

| 功能 | 说明 | 应用场景 |
|------|------|---------|
| 客户信息管理 | 存储客户基本资料和偏好 | 了解客户需求 |
| 交互记录 | 记录每次沟通内容和结果 | 服务连续性 |
| 任务管理 | 跟踪待办事项和跟进计划 | 确保服务承诺 |
| 文档管理 | 存储客户文件和协议 | 合规和审计 |
| 报告生成 | 自动生成客户报告 | 定期沟通 |
| 工作流自动化 | 触发提醒和任务分配 | 提高效率 |

**CRM 数据类型**：
- **静态数据**：姓名、地址、出生日期、风险承受能力
- **动态数据**：投资组合市值、交易记录、服务请求
- **行为数据**：网站访问、邮件打开、会议频率
- **偏好数据**：沟通方式、投资偏好、服务期望

**隐私和合规**：
- 客户数据受隐私法规保护（GDPR、CCPA）
- 需要客户同意才能收集和使用数据
- 数据访问需要权限控制和审计日志
- 保留期限结束后需要安全删除

### 3. 客户投资组合分析

**定义**：客户投资组合分析是评估客户持有的各类资产的表现、风险和配置合理性的过程。

**分析维度**：

**资产配置**：
| 资产类别 | 目标配置 | 实际配置 | 偏差 |
|---------|---------|---------|------|
| 股票 | 60% | 65% | +5% |
| 债券 | 30% | 25% | -5% |
| 现金 | 5% | 5% | 0% |
| 另类投资 | 5% | 5% | 0% |

**业绩归因**：
- **市场收益（Beta）**：市场整体涨跌带来的收益
- **超额收益（Alpha）**：投资经理选股/择时能力带来的超额收益
- **配置效应**：资产配置决策带来的收益
- **选择效应**：具体证券选择带来的收益

**风险指标**：
- **波动率（Volatility）**：收益率的标准差
- **最大回撤（Max Drawdown）**：峰值到谷底的最大跌幅
- **夏普比率（Sharpe Ratio）**：风险调整后的收益
- **贝塔（Beta）**：相对于市场的系统性风险

### 4. 市场新闻监控

**定义**：市场新闻监控是持续跟踪与客户投资组合相关的市场动态、公司新闻和宏观经济事件。

**监控范围**：

| 类别 | 内容 | 重要性 |
|------|------|--------|
| 持仓公司新闻 | 财报、并购、管理层变动 | 高 |
| 行业动态 | 监管变化、技术突破、竞争格局 | 高 |
| 宏观经济 | 利率、通胀、GDP、就业数据 | 中 |
| 地缘政治 | 贸易政策、冲突、选举 | 中 |
| 市场走势 | 大盘指数、板块轮动 | 低 |

**新闻来源**：
- **主流财经媒体**：WSJ、FT、Bloomberg、Reuters
- **行业专业媒体**：行业垂直媒体、研究博客
- **公司官方渠道**：投资者关系网站、新闻稿
- **社交媒体**：Twitter、LinkedIn（需谨慎验证）
- **监管公告**：SEC 文件、央行声明

**信息处理**：
- 自动筛选与客户相关的新闻
- 评估新闻对持仓的潜在影响
- 提取关键信息点
- 生成摘要和行动建议

### 5. 三级安全隔离详解

**客户档案层（profiler）**：
- **职责**：从 CRM 和 CapIQ 读取客户信息和持仓数据
- **数据源**：内部 CRM 系统、金融数据提供商
- **权限**：只读访问，无法修改客户数据
- **安全**：内部系统数据相对可信，但仍需权限控制

**新闻监控层（news-reader）**：
- **职责**：读取外部新闻和公开信息
- **风险**：新闻来源可能包含错误信息或偏见
- **防护**：
  - 只读权限，无法执行内容
  - 返回结构化 JSON
  - 长度限制和模式验证
  - 不直接传递给写入层

**材料生成层（pack-writer）**：
- **职责**：生成会议简报 PowerPoint 文件
- **特点**：唯一拥有 Write 权限的节点
- **输出**：`./out/briefing-<客户>.pptx`
- **安全**：
  - 不直接接触客户提供的原始文件
  - 只处理经过筛选的结构化数据
  - 生成内部使用的材料，不对外发送

### 6. 会议类型与准备重点

**定期回顾会议（Portfolio Review）**：
- **频率**：季度或半年一次
- **重点**：投资组合表现、资产配置调整、市场展望
- **材料**：业绩报告、归因分析、再平衡建议

**事件驱动会议（Event-driven Meeting）**：
- **触发**：重大市场事件、客户生活变化
- **重点**：事件影响评估、应对策略
- **材料**：事件分析、情景模拟、调整方案

**产品推荐会议（Product Presentation）**：
- **目的**：介绍新产品或投资机会
- **重点**：产品特点、风险收益、适合度
- **材料**：产品说明书、对比分析、案例研究

**税务规划会议（Tax Planning）**：
- **时机**：年末或税务年度结束前
- **重点**：税务优化策略、损失收割、捐赠规划
- **材料**：税务报告、优化方案、合规提醒

---

### 7. 合规注意事项

**材料使用限制**：
- 简报材料仅供内部顾问使用
- 不得直接转发给客户
- 需要经过合规审核后才能向客户展示
- 避免包含未经证实的市场传言

**投资建议规范**：
- 所有投资建议需基于充分研究
- 需考虑客户的风险承受能力和投资目标
- 需披露利益冲突和费用结构
- 保留投资建议的记录和依据

**隐私保护**：
- 客户信息严格保密
- 简报材料安全存储和销毁
- 访问权限最小化原则
- 定期审查数据使用合规性

### 8. 日历集成与工作流触发

**日历系统集成**：
- **读取日历**：自动识别即将举行的客户会议
- **提取信息**：客户姓名、会议时间、会议类型
- **提前准备**：在会议前自动生成简报材料
- **任务提醒**：提醒顾问查看和准备材料

**工作流引擎触发**：
```
日历事件创建
    ↓
识别客户和会议类型
    ↓
触发 Meeting Prep Agent
    ↓
收集客户数据和市场信息
    ↓
生成简报材料
    ↓
通知顾问材料已准备就绪
    ↓
顾问审阅和补充
    ↓
会议使用
```

**自动化程度**：
- **全自动**：常规定期会议，完全自动准备
- **半自动**：重要会议，自动生成后人工审阅
- **手动**：特殊会议，人工准备为主
