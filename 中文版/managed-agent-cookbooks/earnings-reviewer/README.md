# 财报审核员 — 托管代理模板

## 概述

财报电话会议 + 监管文件 → 模型更新 → 报告草稿。与 [`earnings-reviewer`](../../plugins/agent-plugins/earnings-reviewer) Cowork插件同源 —— 本目录是用于 `POST /v1/agents` 的托管代理操作手册。

## 部署

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export FACTSET_MCP_URL=... DALOOPA_MCP_URL=...
../../scripts/deploy-managed-agent.sh earnings-reviewer
```

## 引导事件

参见 [`steering-examples.json`](./steering-examples.json)。从您的编排层向覆盖列表分发 —— 每只股票一个会话。

## 安全与交接

会议记录和新闻稿均不可信。三级隔离：

| 层级 | 接触不可信文档？ | 工具 | 连接器 |
|---|---|---|---|
| **`transcript-reader`** | **是** | `Read`, `Grep` 仅限 | 无 |
| `model-updater` / 编排器 | 否 | `Read`, `Grep`, `Glob`, `Agent` | FactSet, Daloopa（只读） |
| **`note-writer`**（写入持有者） | 否 | `Read`, `Write`, `Edit` | 无 |

`transcript-reader` 返回长度受限、模式验证的JSON。`note-writer` 产出 `./out/note-<股票代码>.docx` 和更新后的模型 `./out/model-<股票代码>.xlsx`。

**交接：** 如需在财报驱动的投资论点变化后重建DCF，发出指向 `model-builder` 的 `handoff_request`；`scripts/orchestrate.py` 将其作为新引导事件路由。

## 金融术语和知识解释

### 1. 托管代理部署

**定义**：托管代理部署是将AI代理配置上传到Claude Managed Agent平台的过程，使其可以在生产环境中接收任务并执行。

**部署流程**：
1. **环境准备**：设置API密钥和MCP连接地址
2. **配置解析**：读取agent.yaml清单文件
3. **技能上传**：将所有技能模块注册到平台
4. **代理创建**：通过POST /v1/agents API创建代理实例
5. **验证测试**：发送测试事件确认代理正常工作

**关键环境变量**：
- `ANTHROPIC_API_KEY`：访问Claude API的密钥
- `FACTSET_MCP_URL`：FactSet数据源的MCP连接地址
- `DALOOPA_MCP_URL`：Daloopa数据源的MCP连接地址

**部署脚本功能**：
```bash
../../scripts/deploy-managed-agent.sh earnings-reviewer
```
- 自动解析agent.yaml配置
- 内联系统提示词文件
- 上传技能到CMA平台
- 创建叶子工作节点
- 输出代理ID和访问地址

### 2. 引导事件（Steering Events）

**定义**：引导事件是触发代理开始执行特定任务的指令，包含任务类型、目标对象和上下文信息。

**财报审核典型事件**：
```json
{
  "event_type": "process_earnings",
  "ticker": "AAPL",
  "period": "Q3-2024",
  "fiscal_year": "FY2024",
  "priority": "high",
  "analyst_notes": "关注iPhone销量和服务收入增长"
}
```

**事件字段说明**：
- `event_type`：事件类型，标识任务类别
- `ticker`：股票代码，目标公司
- `period`：报告期，如Q1、Q2、Q3、Q4或全年
- `priority`：优先级，影响处理顺序
- `analyst_notes`：分析师备注，提供额外上下文

**批量处理**：
- 从覆盖列表（Coverage List）批量生成事件
- 每只股票独立会话，互不干扰
- 编排层负责任务分发和结果收集

---

### 3. 三级安全隔离

**定义**：三级安全隔离是一种纵深防御架构，通过分层权限控制确保即使外层被攻破，核心系统仍然安全。

**层级详解**：

**第一层：读取层（ transcript-reader ）**
- **职责**：读取财报电话会议记录、新闻稿等外部文档
- **风险**：这些文档可能包含恶意指令或误导信息
- **防护**：只读权限，返回结构化JSON，长度限制

**第二层：处理层（ model-updater / Orchestrator ）**
- **职责**：分析数据、更新模型、做出推理
- **权限**：可以调用Agent工具，但只能读取外部数据
- **连接器**：FactSet、Daloopa等只读MCP

**第三层：写入层（ note-writer ）**
- **职责**：生成最终报告和更新后的模型文件
- **特点**：唯一拥有Write权限的节点
- **安全**：从不直接接触外部不可信文档

**数据流**：
```
外部文档 → 读取层（隔离）→ 处理层（分析）→ 写入层（输出）
```

### 4. 财报电话会议（Earnings Call）

**定义**：财报电话会议是上市公司在发布季度或年度财报后召开的投资者沟通会议，通常包括管理层业绩介绍和分析师问答环节。

**会议结构**：
1. **安全港声明（Safe Harbor）**：法律免责条款，说明前瞻性陈述的不确定性
2. **CEO/CFO开场**：概述业绩亮点和关键指标
3. **业务部门回顾**：各业务线表现和战略进展
4. **财务详情**：收入、利润、现金流等详细数据
5. **业绩指引（Guidance）**：对未来季度或年度的预测
6. **问答环节（Q&A）**：分析师提问，管理层回答

**关键关注点**：
- **业绩指引变化**：上调、下调或维持预期
- **语气分析**：管理层对业务的信心程度
- **回避问题**：管理层不愿回答的问题往往暗示风险
- **新战略方向**：业务转型、并购计划、市场扩张

**信息价值**：
- 电话会议记录比新闻稿更详细，包含管理层定性观点
- 问答环节揭示分析师关注焦点和市场预期
- 语气变化可能预示未来业绩趋势

### 5. 模型更新（Model Update）

**定义**：模型更新是将最新财报数据录入财务预测模型，调整假设和预测的过程。

**更新步骤**：
1. **数据录入**：将实际收入、利润等数据填入历史数据表
2. **差异分析**：比较实际值与先前预测，分析差异原因
3. **假设调整**：根据业绩指引和管理层评论更新关键假设
4. **预测修订**：重新计算未来季度和年度的预测值
5. **估值更新**：基于新预测更新DCF或可比公司估值

**关键输入**：
- **实际数据（Actuals）**：已公布的财务数据
- **一致预期（Consensus）**：华尔街分析师平均预测
- **业绩指引（Guidance）**：公司管理层官方预测
- **电话会议洞察**：管理层定性评论和语气

**输出成果**：
- 更新后的Excel模型文件
- 预测变更摘要
- 估值调整说明
- 投资评级建议

---

### 6. 报告草稿（Note Draft）

**定义**：报告草稿是分析师撰写的初步研究报告，总结财报分析结果，供高级分析师审阅修改。

**典型结构**：
1. **标题摘要**：一句话概括核心观点
2. **业绩概览**：关键指标实际值 vs 预期
3. **业务亮点**：各业务线表现和关键驱动因素
4. **差异分析**：实际 vs 预测的详细对比
5. **指引更新**：管理层最新业绩指引
6. **模型变更**：关键假设和预测调整
7. **估值更新**：目标价格和评级调整
8. **投资论点**：核心投资逻辑和风险因素

**质量标准**：
- 每个数字都有来源标注
- 逻辑清晰，论据充分
- 语言简洁专业
- 格式规范统一

**审阅流程**：
1. 初级分析师起草报告
2. 高级分析师审阅修改
3. 合规部门审核
4. 最终发布

---

### 7. 交接机制（Handoff）

**定义**：交接机制是代理之间传递任务的标准化方式，确保工作流无缝衔接。

**触发场景**：
- 财报分析发现投资论点重大变化，需要重建DCF模型
- 模型构建完成后，需要生成Pitch材料
- 估值审查发现异常，需要总账对账

**交接流程**：
1. **当前代理**完成分析，识别需要其他代理能力的任务
2. **生成handoff_request**：包含目标代理、任务描述、上下文数据
3. **编排脚本接收**：验证目标代理白名单
4. **创建新会话**：为目标代理创建独立工作会话
5. **传递上下文**：将相关数据和文件传递给新代理
6. **执行并返回**：新代理完成任务，返回结果

**安全控制**：
- 目标代理白名单：只允许预定义的代理间调用
- 载荷验证：对传递的数据进行模式校验
- 上下文隔离：每个代理会话独立运行

---

### 8. 文件输出规范

**报告文件**：
- **格式**：Word文档（.docx）
- **命名**：`note-<股票代码>.docx`
- **路径**：`./out/note-<股票代码>.docx`
- **内容**：完整的分析报告，包含图表和数据表

**模型文件**：
- **格式**：Excel工作簿（.xlsx）
- **命名**：`model-<股票代码>.xlsx`
- **路径**：`./out/model-<股票代码>.xlsx`
- **内容**：更新后的财务模型，包含假设、预测和估值

**输出要求**：
- 文件格式符合公司标准模板
- 数据准确，公式正确
- 图表清晰，标注完整
- 便于高级分析师直接审阅修改
