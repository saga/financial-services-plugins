# xlsx-author

当处于 **无界面 (Headless)** 运行模式（托管代理 / CMA 模式），且需要交付 Excel 工作簿作为 **文件产物**，而非通过 `mcp__office__excel_*` 编辑实时工作簿时，请使用此技能。

## 输出约定

- 写入路径为 `./out/<name>.xlsx`。如果 `./out/` 目录不存在，请创建该目录。
- 在最终回复中返回相对路径，以便编排层进行文件收集。

## 如何构建工作簿

编写一段简短的 Python 脚本并使用 Bash 运行。推荐使用 `openpyxl` 库：

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active; ws.title = "Inputs"
ws["B2"] = "Revenue"; ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")           # 蓝色 = 硬编码输入项
calc = wb.create_sheet("DCF")
calc["C5"] = "=Inputs!C2*(1+Inputs!C3)"        # 黑色 = 公式计算项
wb.save("./out/model.xlsx")
```

## 规范指引（参照 `audit-xls`）

- **蓝/黑/绿 颜色标准：** 蓝色代表硬编码输入（Manual Input），黑色代表公式（Formula），绿色代表指向其他工作表或文件的链接（Link）。
- **计算单元格禁止硬编码：** 所有计算单元格必须通过公式实现；所有基础输入项应统一存放在 “Inputs” 标签页中。
- **命名区域（Named Ranges）：** 对于在演示文稿（Deck）或备忘录（Memo）中引用的任何关键数值，请务必定义命名区域。
- **平衡校验（Balance Checks）：** 必须包含一个 “Checks” 标签页，用于进行自动化钩稽检查（如资产负债表平衡、现金流量表与现金金额核对等），并直观显示 TRUE/FALSE。
- **一表一用：** 除非明确要求追加，否则默认不修改现有工作簿，而是创建新文件。

## 何时禁用

如果 `mcp__office__excel_*` 系列工具可用（协作插件模式），请优先使用这些工具 —— 它们可以直接操作用户的实时工作簿并支持审阅检查点。此技能仅作为无界面运行模式下生成文件文件的备选方案。

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到财务建模的世界！即使你是零基础小白，只要掌握以下几个核心概念，也能像专业投行分析师一样理解这份文档：
>
> 1.  **财务建模的“潜规则”：** 在专业的 Excel 财务模型中，颜色是有特殊含义的。**蓝色**代表这个数字是手动输入的原始数据（不能通过公式更改）；**黑色**代表它是通过公式算出来的；**绿色**则表示它是从别的表格“引用”过来的。遵循这个标准，别人看你的模型就像在看清晰的说明书。
> 2.  **DCF (现金流折现模型)：** 文档代码里提到的 DCF 是金融界的“价值秤”。简单来说，它通过预测一家公司未来几年能赚多少钱，然后按照一定的比例（折现率）算算在“今天”这些钱值多少。它是评估公司值不值得投资的核心工具。
> 3.  **钩稽关系与平衡校验 (Balance Checks)：** 会计学最迷人的地方就是“借贷必相等”。在 Excel 模型里，我们会专门设一个 Checks 页面。如果资产负债表不平，它会显示 **FALSE** 来提醒你账算错啦！它是财务人员心中最后一道防线。
> 4.  **无界面模式 (Headless)：** 想象一下一个隐形的财务助手。它不需要像你一样点开 Excel 软件、移动鼠标，而是在计算机后台直接“变”出一个完整的财务报表文件。这种方式速度极快，适合大规模批量处理任务。
