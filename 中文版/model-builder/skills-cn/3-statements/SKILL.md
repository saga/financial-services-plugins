# 三表财务模型模板填充指南

完成并填充集成的财务模型模板，确保利润表（Income Statement）、资产负债表（Balance Sheet）与现金流量表（Cash Flow Statement）之间建立正确的勾稽关系。

## ⚠️ 核心原则 —— 在填充任何模板前必读

**环境执行 —— Office JS vs Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS。通过 `range.formulas = [["=D14*(1+Assumptions!$B$5)"]]` 编写公式 —— 对于派生单元格，严禁使用 `range.values`。无需手动重算，Excel 会自动原生计算。使用 `context.workbook.worksheets.getItem(...)` 进行工作表导航。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。编写 `ws["D15"] = "=D14*(1+Assumptions!$B$5)"`，随后在交付前运行 `recalc.py`。
- **Office JS 合并单元格陷阱：** 严禁先调用 `.merge()` 再对合并后的区域设置 `.values` —— 这会因区域维度与预合并状态不符而抛出 `InvalidArgument` 错误。应先向左上角单元格写入数值，再进行合并及格式化：`ws.getRange("A1").values = [["INCOME STATEMENT"]]; const h = ws.getRange("A1:G1"); h.merge(); h.format.fill.color = "#1F4E79";`
- 以下所有原则在两种环境下均同等适用。

**公式优于硬编码（不容协商）：**
- 每一个预测单元格、滚动结转（Roll-forward）、勾稽链接和分类小计必须是 Excel 公式 —— 严禁使用预计算的数值。
- 使用 Python/openpyxl 时：编写公式字符串（`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`），而非计算结果（`ws["D15"] = 12500`）。
- 仅允许在以下位置使用硬编码数字：(1) 历史实际值，(2) “假设（Assumptions）”工作表中的驱动因素。
- 如果你发现自己在 Python 中计算了一个值并将其写入单元格 —— 立即停止，改为编写公式。
- 原理：模型必须能够在场景切换或假设更改时灵活变动。硬编码会静默破坏后续所有的完整性检查。

**与用户进行逐步验证：**
1. **完成模板映射后** → 向用户展示你识别出的工作表和科目板块，确认无误后再修改任何单元格。
2. **填充历史数据后** → 向用户展示历史数据区块，确认数值和期间与源数据匹配。
3. **构建利润表（IS）预测后** → 运行小计检查，向用户展示预测的利润表，确认后转入资产负债表（BS）。
4. **构建资产负债表（BS）后** → 向用户展示每一期的平衡检查（资产 = 负债 + 权益），确认后转入现金流量表（CF）。
5. **构建现金流量表（CF）后** → 向用户展示现金对账结果（CF 期末现金 = BS 现金），确认后最终定稿。
6. **严禁一次性填充整个模型并直接交付** —— 必须在每张报表完成后停顿，展示工作成果，及早发现错误。

## 格式规范 —— 专业蓝/灰配色（除非模板或用户另有指定）

**保持色彩简约。** 仅使用蓝色和灰色填充单元格。严禁引入绿色、黄色、橙色或多种强调色 —— 洁净的模型源于克制。

| 元素 | 填充颜色 | 字体颜色 |
|---|---|---|
| 板块标题 (IS / BS / CF 标题) | 深蓝色 `#1F4E79` | 白色加粗 |
| 列标题 (FY2024A, FY2025E 等) | 浅蓝色 `#D9E1F2` | 黑色加粗 |
| 输入单元格 (历史值、假设驱动因素) | 浅灰色 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨表链接 | 白色 | 绿色 `#008000` |
| 检查行 / 核心总计 | 中蓝色 `#BDD7EE` | 黑色加粗 |

**即：3 种蓝色 + 1 种灰色 + 白色。** 如果模板已有自己的色系，请遵循模板。

字体颜色代表单元格的**属性**（输入/公式/链接）。填充颜色代表你所处的**位置**（标题/数据/检查）。

## 模型结构

### 识别模板工作表组织

不同模板的工作表命名习惯和组织方式各异。在填充前，请审阅所有工作表以理解结构。以下是常见的工作表名称及其典型内容：

| 常见工作表名称 | 寻找的内容 |
|------------------|----------------------|
| IS, P&L, Income Statement | 利润表（损益表） |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资本明细表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销（D&A）表 |
| Debt, Debt Schedule | 债务明细表 |
| NOL, Tax, DTA | 净经营亏损（NOL）/ 税项表 |
| Assumptions, Inputs, Drivers | 驱动因素假设与输入 |
| Checks, Audit, Validation | 错误检查仪表盘 |

**模板审阅清单**
- 识别模板中存在哪些工作表（并非所有模板都包含完整明细表）。
- 记录未列在上述列表中的模板特定工作表。
- 理解工作表间的依赖关系（例如，哪些明细表的数据流入主报表）。
- 定位每张表上的输入单元格与公式单元格。

### 理解模板布局

在填充模板前，熟悉其现有布局，以确保数据输入在正确位置且公式保持完整。

**识别行结构**
- 定位每张工作表顶部的模型标题。
- 识别板块标题及其视觉分隔。
- 找到单位行，标明单位为百万、%、倍数（x）等。
- 留意区分实际值（Actuals）与预测值（Estimates）期间的列标题。
- 确认期间标签（如 FY2024A, FY2025E）。
- 识别输入单元格与公式单元格（通常通过字体颜色区分）。

**识别列结构**
- 确认最左列的行项目标签。
- 验证历史年度位于预测年度之前。
- 注意分隔历史与预测期间的视觉边框。
- 检查各工作表的列顺序是否一致。

**使用命名区域（Named Ranges）**
模板通常对核心输入和输出使用命名区域。在填充前：
- 审阅模板中现有的命名区域（Excel 中：公式 → 名称管理器）。
- 常见的命名区域包括：收入增长率、成本百分比、核心输出（净利润、EBITDA、总债务、现金）、场景选择单元格。
- 确保输入的数据进入了流入这些命名区域的单元格。

### 预测期间
- 模板通常从最后一个历史年度起预测 5 年。
- 验证历史 (A) 与预测 (E) 列是否清晰分隔。
- 确认列使用财年表示法（如 FY2024A, FY2025E）。

## 利润率分析

**注意：仅在用户要求或模板明确要求时执行以下分析。若无要求，请跳过此部分。**

在利润表（IS）标签页计算并展示盈利能力指标，以跟踪运营效率并进行同业对比。

### 核心指标

| 指标 | 公式 | 衡量意义 |
|--------|---------|------------------|
| 毛利率 (Gross Margin) | 毛利润 / 营业收入 | 定价权、生产效率 |
| EBITDA 利润率 | EBITDA / 营业收入 | 核心经营盈利能力 |
| EBIT 利润率 | EBIT / 营业收入 | 扣除折旧摊销后的经营盈利能力 |
| 净利率 (Net Income Margin) | 净利润 / 营业收入 | 最终盈利能力 |

### 利润率布局规范

在每个利润行项目正下方直接显示百分比：
- 毛利率 % 位于毛利润下方
- EBIT 利润率 % 位于 EBIT 下方
- EBITDA 利润率 % 位于 EBITDA 下方
- 净利率 % 位于净利润下方

## 信贷指标

**注意：仅在用户要求或模板明确要求时执行以下分析。若无要求，请跳过此部分。**

在资产负债表（BS）标签页计算并展示信贷/杠杆指标，以评估财务健康状况、举债能力及债务契约合规性。

### 核心信贷指标

| 指标 | 公式 | 衡量意义 |
|--------|---------|------------------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | (总债务 - 现金) / LTM EBITDA | 剔除现金后的杠杆水平 |
| 利息保障倍数 | EBITDA / 利息支出 | 偿债能力 |
| 债务 / 总资本 | 总债务 / (总债务 + 权益) | 资本结构 |
| 债务 / 权益比 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | (流动资产 - 存货) / 流动负债 | 即时流动性 |

### 指标层级逻辑检查

验证在“乐观情形（Upside）”中显示最强的信贷状况：
- 杠杆率：乐观 < 基准 < 悲观（越低越好）
- 保障倍数：乐观 > 基准 > 悲观（越高越好）
- 流动性：乐观 > 基准 > 悲观（越高越好）

### 债务契约跟踪

如果已知债务契约（Covenants），添加明确的合规性检查，将实际指标与契约阈值进行对比。

## 场景分析 (基准 / 乐观 / 悲观)

在“假设（Assumptions）”工作表中使用场景切换开关（下拉菜单），配合 `CHOOSE` 或 `INDEX/MATCH` 公式。

| 场景 | 描述 |
|----------|-------------|
| 基准情形 (Base) | 管理层指引或市场共识预期 |
| 乐观情形 (Upside) | 高于指引的增长、利润率扩张 |
| 悲观情形 (Downside) | 低于趋势的增长、利润率压缩 |

**需敏感化处理的核心驱动因素**：收入增长、毛利率、销售及管理费用（SG&A）占比、应收/存货/应付周转天数（DSO/DIO/DPO）、资本开支（CapEx）占比、利率、税率。

**场景审计检查**：切换开关后所有报表均随之变动、所有场景下资产负债表均配平、现金流对账一致、层级关系成立（净利润、EBITDA、自由现金流、利润率在乐观 > 基准 > 悲观）。

## 完成模型模板的实施步骤

本部分提供了完成任何三表财务模型模板的通用指南，旨在保留现有公式并确保数据完整性。

### 第 1 步：分析模板结构

在输入任何数据前，彻底审阅模板解析其架构：

**区分输入与公式单元格**
- 寻找视觉线索（字体颜色、单元格阴影）：蓝字 = 输入，黑字 = 公式，绿字 = 跨表链接。
- 使用 Excel 的“追踪引用单元格/从属单元格”功能理解逻辑关系。
- 检查可能控制核心输入的命名区域。

**绘制模板数据流**
- 识别各表间的先后顺序（如：假设 → 利润表 → 资产负债表 → 现金流量表）。
- 记录支持性明细表及其与主报表的链接。

### 第 2 步：填充数据且不破坏公式

**数据录入的黄金准则**

| 准则 | 描述 |
|------|-------------|
| 仅编辑输入单元格 | 严禁覆盖含有公式的单元格（除非有意替换该公式） |
| 保留单元格引用 | 复制数据时，使用“选择性粘贴数值”（Ctrl+Shift+V）以防覆盖格式 |
| 匹配模板单位 | 输入前验证模板单位是千、百万还是数值原值 |
| 尊重符号规范 | 遵循模板现有的正负号规范（如费用记为正数还是负数） |
| 检查循环引用 | 如果模板使用迭代计算，确保开启了“启用迭代计算” |

**安全填充流程**
1. 识别指定的输入单元格（通常有高亮或标签）。
2. 先填充历史数据，验证这些期间的公式计算是否正确。
3. 填充驱动预测计算的假设参数。
4. 审阅计算产出，确认公式按预期运行。
5. 若必须修改公式单元格，请在改动前记录原始公式。

**处理预置公式**
- 如果公式引用的单元格尚未填充，出现临时错误（#REF!, #DIV/0!）是正常的，待输入完成后会消失。
- 当公式产生异常结果时，追踪其引用项以定位缺失或错误的输入。
- 严禁在未检查跨表依赖项的情况下删除行或列。

### 第 3 步：验证公式

**公式完整性检查**

| 检查类型 | 方法 |
|------------|--------|
| 追踪引用单元格 | 选中公式单元格 → 公式 → 追踪引用单元格，验证引用位置正确 |
| 追踪从属单元格 | 验证核心输入已流向预期的输出单元格 |
| 分步求值 | 使用“公式求值”功能解析复杂的逻辑计算 |
| 检查硬编码 | 预测公式应引用假设，而非包含手写数字 |
| 测试已知值 | 输入简单的测试数值，验证产出是否符合预期 |
| 跨期一致性 | 确保同一预测逻辑应用到了所有预测年度 |

**易出现的公式问题**
- 混合绝对/相对引用导致向后填充时结果错误。
- 指向外部文件或已删除区域的毁损链接（#REF! 错误）。
- 在收入尚未起步的早期期间出现除以零（#DIV/0! 错误）。
- 循环引用警告（在利息计算中可能是有意的）。
- 预测列之间的公式不一致（使用 Ctrl+\ 查找差异）。

### 第 4 步：各表专项质量检查

填充完毕后，对每张报表进行如下验证：

**利润表 (IS) 检查**
- 历史期间的收入数字与源数据匹配。
- 所有费用明细项之和等于报告的总额。
- 小计（毛利润、EBIT、EBT、净利润）计算正确。
- 所得税逻辑完备（能正确处理亏损）。
- 预测驱动因素引用了“假设”标签页（无硬编码）。

**资产负债表 (BS) 检查**
- 资产 = 负债 + 权益（每一期均需满足，这是首要检查）。
- 现金余额与现金流量表期末现金一致。
- 营运资本科目与明细表（若有）核对一致。
- 未分配利润（RE）滚动正确：上期 RE + 净利润 - 股利 +/- 调整 = 本期期末 RE。
- 债务余额与债务明细表（若有）一致。

**现金流量表 (CF) 检查**
- 顶部的净利润与利润表一致。
- 非现金加回项（折旧摊销、股份支付等）与明细表或原报表一致。
- 营运资本变动符号正确（资产增加 = 现金流出 = 负数）。
- 资本开支（CapEx）与固定资产（PP&E）表一致。
- 筹资活动与资产负债表中的债务和权益科目变动一致。
- 期末现金与资产负债表资产项下的现金一致。

### 第 5 步：跨表勾稽完整性检查

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平稳 | 资产 - 负债 - 权益 | = 0 |
| 现金对账 | CF 期末现金 - BS 现金 | = 0 |
| 净利润链接 | IS 净利润 - CF 起始净利润 | = 0 |
| 未分配利润结转 | 上期 RE + 净利润 - 股利 - 本期期末 RE | = 0 |

### 第 6 步：最终复核

- 切换所有场景，确保每种情形下检查项均通过。
- 处理或记录所有的 #REF!, #DIV/0!, #VALUE!, #NAME? 错误。
- 确认所有单位在各表中保持一致。
- 在进行任何额外修饰前保存一个“洁净版”。

## 模型验证与审计

### 核心勾稽关系（必须恒等）

公式细节请参阅 [references/formulas.md](references/formulas.md)。

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平稳 | 资产 - 负债 - 权益 | = 0 |
| 现金对账 | CF 期末现金 - BS 现金 | = 0 |
| 净利润链接 | IS 净利润 - CF 起始净利润 | = 0 |
| 未分配利润结转 | 上期 RE + 净利润 + 股份支付 - 股利 - 本期 BS 期末 RE | = 0 |
| 股权筹资对账 | Δ普通股/资本公积 (BS) - 股票发行 (CFF) | = 0 |

### 正负号规范参考

| 报表 | 项目 | 符号惯例 |
|-----------|------|-----------------|
| 经营现金流 (CFO) | 折旧摊销 (D&A), 股份支付 (SBC) | 正数 (加回项) |
| 经营现金流 (CFO) | Δ应收账款 (增加) | 负数 (现金占用) |
| 经营现金流 (CFO) | Δ应付账款 (增加) | 正数 (现金来源) |
| 投资现金流 (CFI) | 资本开支 (CapEx) | 负数 |
| 筹资现金流 (CFF) | 债务发行 | 正数 |
| 筹资现金流 (CFF) | 偿还债务 | 负数 |
| 筹资现金流 (CFF) | 派发股利 | 负数 |

### 循环引用处理

利息支出会产生循环：利息支出 → 净利润 → 现金 → 债务余额 → 利息支出。

在 Excel 中启用迭代计算：文件 → 选项 → 公式 → 启用迭代计算。最多迭代次数设为 100，最大误差设为 0.001。务必在“假设”工作表中添加一个“断路器（circuit breaker）”开关。

### 审计分类清单

**1. 币种一致性**：确认所有标签页使用的货币符号和量级（如：百万）完全统一。
**2. 资产负债表平衡**：公式 `资产 - 负债 - 权益` 必须在每一期都等于 0。
**3. 现金流完整性**：期末现金必须精准对接资产负债表。
**4. 未分配利润**：清晰展示从上期到本期的演变过程，便于排查错误。
**5. 营运资本**：对 DSO, DIO, DPO 进行合理性检查，超出正常范围时需标注。
**6. 债务与股权**：确保筹资活动与资产负债表科目的变动额完美匹配。
**7. 净经营亏损 (NOL)**：
   - 初始 NOL 通常为 0。
   - 只有当税前利润 (EBT) < 0 时才产生 NOL。
   - 递延所得税资产 (DTA) 需与资产负债表对应。
   - 抵扣额通常受限于 EBT 的 80%（遵循 2017 年后的联邦税法）。
**8. 公式诚信度**：成本和费用应由收入百分比驱动，严禁在预测期出现硬编码数值。

### 总检查公式

将所有子项状态汇总为一个总开关：
- 若全部通过 → "✓ 所有检查通过"
- 若任一失败 → "✗ 检测到错误 - 请复核下方明细"

> **💡 Appendix: 领域知识小贴士**
>
> **1. 什么是“三表模型”？**
> 想象你在为一家公司做“深度体检”。利润表是公司的“胃口”，看它赚了多少钱；资产负债表是公司的“骨架”，看它有多少家底和欠债；现金流量表则是公司的“血液”，看钱实际上是怎么流进流出的。三表模型就是把这三张表用逻辑连在一起，牵一发而动全身。
>
> **2. 灵魂所在：勾稽关系 (Linkages)**
> 财务模型不是三张独立的表格，它们更像是一台精密仪器的齿轮。比如：利润表的最后一项“净利润”，会流向资产负债表的“未分配利润”，同时也是现金流量表的起点。如果这三个齿轮没对齐，模型就会“爆掉”（资产负债表不平）。
>
> **3. 为什么执着于“蓝字”和“黑字”？**
> 在华尔街的职业标准里，颜色是有职业语言的。**蓝色**代表这是“手写”输入的假设，是可以改动的变量；**黑色**代表这是公式计算出来的，绝对不能乱动。这种规范是为了让任何一个接手你模型的人，一眼就能看出哪里是“因”，哪里是“果”。
>
> **4. 场景分析：给未来做“备选方案”**
> 生意总有起伏。场景分析就是给模型装上“档位”：基准是现在的路，乐观是顺风车，悲观是泥泞路。通过一个下拉菜单切换，你就能立刻看到如果收入下滑 20%，公司的现金流还能撑几个月。这就是财务决策的魅力所在。
