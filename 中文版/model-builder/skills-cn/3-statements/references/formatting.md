# 财务建模格式标准指南

| 报表元素 | 格式要求 |
|---------|--------|
| 硬编码输入（手动输入项） | 蓝色字体 |
| 公式 | 黑色字体 |
| 跨表链接 | 绿色字体 |
| 勾稽检查单元格 | 报错为红色，平稳（配平）为绿色 |
| 负数 | 使用括号，而非负号 |
| 货币符号 | 大额数字不留小数，每股数值保留2位小数 |
| 百分比 | 保留1位小数 |
| 标题 | 加粗，带下框线 |
| 单位行 | 在标题下方标明单位（百万美元、% 等） |

## 视觉区分准则

- 历史年度列与预测年度列之间使用细细的纵向框线分隔
- 报表大类总计（如：资产总计）下方使用粗框线
- 小计项下方使用单线框
- 最终项/合计项下方使用双线框

## 合计及小计行格式

所有合计及小计行的数值必须使用 **加粗格式**，以便将汇总数据与明细行项目清晰区分开来。

### 利润表（损益表）页签
| 行项目 | 格式要求 |
|-----|------------|
| 营业总收入 | 加粗 |
| 营业总成本（收入成本） | 加粗 |
| 毛利润 | 加粗 |
| 销售及管理费用总额 (SG&A) | 加粗 |
| EBITDA（息税折旧摊销前利润） | 加粗 |
| EBIT（息税前利润） | 加粗 |
| EBT（税前利润） | 加粗 |
| 税后净利润 | 加粗 |

### 资产负债表页签
| 行项目 | 格式要求 |
|-----|------------|
| 流动资产总计 | 加粗 |
| 非流动资产总计 | 加粗 |
| 其他资产总计 | 加粗 |
| 资产总计 | 加粗 |
| 流动负债总计 | 加粗 |
| 非流动负债总计 | 加粗 |
| 权益总额 | 加粗 |
| 负债及权益总计 | 加粗 |

### 现金流量表页签
| 行项目 | 格式要求 |
|-----|------------|
| 营运资本变动前的经营活动产生的现金流 | 加粗 |
| 营运资本变动总额 | 加粗 |
| 经营活动产生的现金流量净额 | 加粗 |
| 投资活动产生的现金流量净额 | 加粗 |
| 筹资活动产生的现金流量净额 | 加粗 |
| 期末现金余额 | 加粗 |

**注：** 本列表未穷尽所有项目。模型中任何代表合计、小计或汇总计算的行均应应用加粗格式。

## 资产负债表勾稽检查行格式

资产负债表勾稽项（位于“负债及权益总计”下方）应采用条件格式，将非零值显示为红色。当资产负债表准确配平（检查值 = 0）时，数值以黑色或标准格式显示。

| 检查值状态 | 字体颜色 |
|-------------|------------|
| = 0（已配平） | 黑色（标准） |
| ≠ 0（存在误差） | 红色 |

**实现方式：** 应用自定义数字格式 `[Red][<>0]0.00;[Red][<>0](0.00);0.00` 或使用 Excel 条件格式规则：“单元格数值 ≠ 0” → 红色字体。

## 利润率行格式

| 报表元素 | 格式要求 |
|---------|--------|
| 利润率 % 行 | 缩进、斜体、保留1位小数 |
| 趋势向好 | 无特殊格式（或浅绿色） |
| 趋势恶化 | 标记待查（淡黄色） |
| 低于同业平均水平 | 考虑高亮显示以供讨论 |

## 信用指标格式

| 报表元素 | 格式要求 |
|---------|--------|
| 杠杆倍数 | 保留1位小数并带“x”后缀（如：2.5x） |
| 百分比 | 保留1位小数并带“%”后缀 |
| 净负债为负 | 使用括号，表示净现金头寸 |
| 模块标题 | 加粗，“CREDIT METRICS”（信用指标） |
| 分隔线 | 信用指标模块上方使用细线框 |

## 信用指标颜色阈值

| 指标 | 绿色（安全） | 黄色（预警） | 红色（风险） |
|--------|-------|--------|-----|
| 总债务 / EBITDA | < 2.5x | 2.5x-4.0x | > 4.0x |
| 净负债 / EBITDA | < 2.0x | 2.0x-3.5x | > 3.5x |
| 利息保障倍数 | > 4.0x | 2.5x-4.0x | < 2.5x |
| 负债 / 总资本 | < 40% | 40%-60% | > 60% |
| 流动比率 | > 1.5x | 1.0x-1.5x | < 1.0x |
| 速动比率 | > 1.0x | 0.75x-1.0x | < 0.75x |

## 检查页签（Checks Tab）条件格式

- 单元格包含“通过”标识 → 绿色填充
- 单元格包含“失败”标识 → 红色填充
- 单元格包含“警告”标识 → 黄色填充
- 差异单元格 = 0 → 浅绿色填充
- 差异单元格 ≠ 0 → 浅红色填充

## 利润率合理性校核标识

- 毛利率 < 0% → 错误：检查营业成本 (COGS)
- 毛利率 > 80% → 警告：核实收入/营业成本
- EBITDA 利润率 < 0% → 标识：经营性亏损
- EBITDA 利润率 > 50% → 警告：数值异常高
- 净利率 < 0% → 标识：净亏损（增长阶段可能允许）
- 净利率 > 毛利率 → 错误：公式逻辑问题

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到财务建模的世界！这些标准看起来规矩很多，但其实都是为了让复杂的账目一目了然。对于新手，这里有几个好玩的概念解释：
>
> 1.  **颜色魔法（蓝与黑）**：在建模中，蓝色代表“这是我手动填的数据”，黑色代表“这是电脑算的公式”。如果你把公式写成了蓝色，下次改动数据时，你的整个模型可能就“断掉”了。
> 2.  **勾稽检查（Check Rows）**：会计界有一个终极奥义，叫做“资产 = 负债 + 所有者权益”。如果这两个数字对不上，就叫“不平”。检查行就像一个警报器，只要不是0，就说明你的模型里有地方出错啦！
> 3.  **EBITDA（息税折旧摊销前利润）**：这是个大红人。它把那些复杂的利息、税收和设备折旧都踢开，只看公司主业到底能不能挣钱。就像看一个人的格斗实力，不看他有没有穿盔甲（税）或者带神兵利器（利息）。
> 4.  **杠杆倍数（The 'x' factor）**：金融里常用“几倍（x）”来形容。比如“2.5x 杠杆”，意思就是你借的钱是你挣的利润的 2.5 倍。倍数越高，风险越高，就像你在钢丝上叠罗汉，叠得越高越容易晃。
> 5.  **利润率（Margin）**：这代表了你做生意的“含金量”。如果毛利率是 80%，意味着你卖 100 块钱的东西，成本只要 20 块，剩下 80 块都是赚的差价（当然还没扣掉工资和房租）。
