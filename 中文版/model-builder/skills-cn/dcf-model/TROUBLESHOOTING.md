# DCF 模型故障排除指南

**阅读建议：** 当 recalc.py 报错、估值结果不合理或情景选择器（Case Selector）无法正常工作时，请参考此指南。

## 模型返回错误值

### #REF! 引用错误
- **常见原因：** 通常由于在插入标题行或调整布局后，公式引用了错误的行号。
- **解决方案：** 根据版面规划重新检查并构建正确的行引用，或推倒重来。
- **预防措施：** 在编写公式**之前**，先确定好所有行数的位置。

### #DIV/0! 除以零错误
- **常见原因：** 公式中出现了除数为零或引用了空单元格的情况。
- **解决方案：** 添加 IF 语句来处理零值：`=IF([除数]=0,0,[分子]/[除数])`。

### #VALUE! 数值错误
- **常见原因：** 计算中使用了错误的数据类型（例如将文本误当作数字）。
- **解决方案：** 核实所有输入项的格式均已设定为“数字”。

## 估值结果表现不合理

### 隐含价格过高
- 检查终值（Terminal Value）占比是否超过了企业价值（EV）的 80%。
- 核实永续增长率（Terminal Growth）是否小于加权平均资本成本（WACC）。
- 审查增长假设是否切合实际。
- 考虑利润率（Margins）假设是否过于乐观。

### 隐含价格过低
- 核实净负债（Net Debt）与净现金（Net Cash）的计算是否正确。
- 检查 WACC 是否设定过高。
- 审查业绩预测是否过于保守。
- 考虑永续增长率是否设定过低。

## 情景选择器失效

### 切换情景时汇总列未更新
- 核实情景选择单元格的值是否为 1、2 或 3。
- 检查 INDEX/OFFSET 公式是否引用了正确的行范围和选择器单元格。
- 确保在引用选择器单元格时使用了绝对引用（如 $B$6）。
- 进行手动测试：更改选择器单元格的值，验证预测数据是否随之联动更新。

> **💡 Appendix: 领域知识小贴士**
>
> *   **DCF（现金流折现模型）**：想象你有一棵能结金币的树。DCF 就是在计算：考虑到通货膨胀和风险，这棵树未来一辈子结的所有金币，折算成今天的钱一共值多少。
> *   **WACC（加权平均资本成本）**：这是公司筹钱的“平均成本”。你可以把它理解为投资这场生意的“门槛收益率”——如果生意的回报率达不到 WACC，那这笔买卖就不划算。
> *   **终值（Terminal Value）**：我们通常只能预测公司未来 5-10 年的表现。终值代表的是在那之后，假设公司能永远经营下去所产生的价值。如果这个数值占总价值比例太高（比如 80% 以上），说明你的估值太依赖遥远未来的“画饼”，风险比较大。
> *   **企业价值（EV） vs. 股权价值**：企业价值是整家公司的“总价”（包括欠银行的钱）。要算出每一股值多少钱，得先从 EV 里扣掉欠银行的钱（净负债），剩下的才是属于股东的钱。
