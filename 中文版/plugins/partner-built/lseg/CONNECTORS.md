# 连接器

本插件连接到 **LFA MCP Server**，该服务器提供 LSEG（伦敦证券交易所集团）的金融分析工具。所有工具均由单一的 MCP 服务器提供，无需额外的连接器。

## 命令如何引用工具

本插件中的命令按其确切的工具名称引用 MCP 工具（例如 `bond_price`、`interest_rate_curve`）。为清晰起见，工具按以下类别组织：

## 工具分类

| 类别 | 占位符 | 工具 | 说明 |
|------|--------|------|------|
| 债券定价 | `~~bond-pricing` | `bond_price`、`bond_future_price` | 含完整分析功能的债券和期货定价 |
| 外汇定价 | `~~fx-pricing` | `fx_spot_price`、`fx_forward_price` | 外汇即期和远期汇率定价 |
| 利率曲线 | `~~ir-curves` | `interest_rate_curve`、`inflation_curve` | 国债收益率曲线和通胀盈亏平衡率 |
| 信用曲线 | `~~credit-curves` | `credit_curve` | 按发行人类型的信用利差曲线 |
| 外汇曲线 | `~~fx-curves` | `fx_forward_curve` | 外汇远期点数曲线 |
| 期权 | `~~options` | `option_value`、`option_template_list` | 含希腊字母的期权估值 |
| 互换 | `~~swaps` | `ir_swap` | 利率互换定价 |
| 波动率曲面 | `~~volatility` | `fx_vol_surface`、`equity_vol_surface` | 外汇和股票隐含波动率曲面 |
| 量化分析 | `~~qa` | `qa_ibes_consensus`、`qa_company_fundamentals`、`qa_historical_equity_price`、`qa_macroeconomic` | 分析师预测、基本面、股价、宏观数据 |
| 时间序列 | `~~time-series` | `tscc_historical_pricing_summaries` | 历史价格汇总（日间/日内） |
| 固收分析 | `~~yieldbook` | `yieldbook_bond_reference`、`yieldbook_cashflow`、`yieldbook_scenario`、`fixed_income_risk_analytics` | 债券参考数据、现金流、情景分析、OAS/久期 |

## 完整工具参考

### 债券域
- **`bond_price`** — 计算债券定价、估值和分析。接受 ISIN、RIC、CUSIP 或 AssetId。返回收益率、久期、凸性、DV01、应计利息。支持通过价格/收益率覆盖进行假设情景分析。
- **`bond_future_price`** — 计算债券期货定价和分析。返回公允价值、最便宜可交割债券识别、交割篮子、转换因子和合约 DV01。

### 外汇域
- **`fx_spot_price`** — ISO 货币对的外汇即期汇率定价。返回中间价/买价/卖价。
- **`fx_forward_price`** — 特定期限或日期的外汇远期汇率定价。返回远期点数、远期全价和利差。

### 曲线域
- **`interest_rate_curve`** — 国债收益率曲线。两阶段：列出可用曲线，然后计算曲线点。返回平价/零息利率、贴现因子、远期利率。
- **`credit_curve`** — 信用利差曲线。按国家和发行人类型（企业、主权、机构等）搜索，然后计算利差期限结构。
- **`inflation_curve`** — 通胀盈亏平衡曲线。按国家/货币搜索，然后计算盈亏平衡利率和实际收益率。
- **`fx_forward_curve`** — 外汇远期点数曲线。列出曲线，然后计算所有标准期限的远期点数。

### 互换域
- **`ir_swap`** — 利率互换定价。两阶段：按货币/指数列出模板，然后在指定期限定价互换。返回平价利率、DV01、NPV。

### 期权域
- **`option_value`** — 期权估值，支持普通、障碍、二元和亚式期权。返回权利金、全部希腊字母（delta、gamma、vega、theta、rho）和风险指标。
- **`option_template_list`** — 列出可用于定价的期权模板。

### 波动率域
- **`fx_vol_surface`** — 使用 SABR 模型生成外汇波动率曲面。返回各期限和 delta 行权价的波动率曲面。
- **`equity_vol_surface`** — 股票隐含波动率曲面。支持通过 RIC 查询股票/指数，通过 RICROOT 查询期货。

### 量化分析域
- **`qa_ibes_consensus`** — IBES 分析师一致预期（EPS、收入、EBITDA、DPS）。前瞻性预测，含分析师数量、离散度和最高/最低范围。
- **`qa_company_fundamentals`** — 已报告的公司财务数据（利润表、资产负债表指标）。历史财年数据。
- **`qa_historical_equity_price`** — 历史股价，含 OHLCV、总回报和 Beta。
- **`qa_macroeconomic`** — 宏观经济指标数据库。通过助记符或描述搜索，获取最新值或时间序列。

### 时间序列域
- **`tscc_historical_pricing_summaries`** — 任意 RIC 的历史价格汇总。支持日间（日、周、月）和日内（1分钟到1小时）间隔。

### 固收分析（YieldBook）域
- **`yieldbook_bond_reference`** — 债券参考数据：证券类型、行业、评级、票息、到期日、发行人。
- **`yieldbook_cashflow`** — 债券现金流预测：未来票息和本金支付时间表。
- **`yieldbook_scenario`** — 债券情景分析：在平行利率变动下的价格/收益率。
- **`fixed_income_risk_analytics`** — 债券风险分析：OAS、有效久期、关键利率久期、凸性。

### 1. MCP 协议——"AI 和金融数据的翻译官"

**MCP（Model Context Protocol，模型上下文协议）** 是一种让 AI 助手能够调用外部工具的标准协议。你可以把它理解为"AI 和金融数据之间的翻译官"。

**想象一下**：你是一个只会说中文的人，想查纽约证券交易所的股票价格。你需要一个翻译——你用中文说"帮我查苹果的股价"，翻译把它转成交易所能理解的英文指令，交易所返回英文结果，翻译再转成中文给你。MCP 就是这个翻译。

**MCP 的工作流程**：
1. AI 助手发出请求："调用 `bond_price`，参数 ISIN=US1234567890"
2. MCP 服务器接收请求，调用 LSEG 的定价引擎
3. 定价引擎返回结果：收益率 4.5%、久期 6.2 年、DV01 $620/bp
4. MCP 服务器把结果返回给 AI 助手
5. AI 助手把结果呈现给你

**为什么用 MCP 而不是直接调用 API**：
- **标准化**：所有工具用统一的协议，AI 不需要学习每个 API 的独特格式
- **安全性**：MCP 服务器控制哪些工具可以被调用，防止 AI 做出危险操作
- **可扩展**：新增工具只需要在 MCP 服务器端注册，AI 自动就能用

### 2. 证券标识符——"每只债券、每只股票的身份证号"

金融世界里有数百万只证券（股票、债券、期权、基金……），每只都需要一个唯一的"身份证号"。但不同体系用的编号不同——就像同一个人可以有身份证号、护照号和社保号。

**三大标识符体系**：

| 标识符 | 全称 | 覆盖范围 | 格式 | 例子 |
|--------|------|---------|------|------|
| **ISIN** | International Securities Identification Number | 全球 | 12 位字母+数字 | US0378331005（苹果股票） |
| **CUSIP** | Committee on Uniform Security Identification Procedures | 北美（美国+加拿大） | 9 位字母+数字 | 037833100（苹果股票） |
| **RIC** | Reuters Instrument Code | LSEG/Refinitiv 体系 | 字母+点号后缀 | AAPL.OQ（苹果在纳斯达克） |

**RIC 的特殊之处**：
- RIC 是路透社（现 LSEG）自己设计的代码体系
- 后缀表示交易所：`.OQ` = 纳斯达克，`.N` = 纽约，`.L` = 伦敦
- 债券的 RIC 通常包含发行人缩写 + 票息 + 到期日，如 `US10YT=RR`（美国 10 年国债）
- RIC 在 LSEG 体系内最常用，因为格式统一、搜索方便

**什么时候用哪个**：
- **ISIN**：最通用，跨境交易时用。每只债券都有 ISIN
- **CUSIP**：美国债券最常用。美国国债和公司债通常用 CUSIP 查询
- **RIC**：在 LSEG 工具中最常用。如果你用 `bond_price`，RIC 通常是最方便的输入

**真实例子**：同一只苹果公司债券可能有三种标识：
- ISIN: US037833AR47
- CUSIP: 037833AR4
- RIC: AAP4376817=

### 3. 九大工具类别——"金融数据的九把钥匙"

本文件列出了 20+ 个工具，按九大类组织。每把"钥匙"打开金融世界的一扇门。

**① 债券定价（bond_price / bond_future_price）**：
- **干什么**：给债券和债券期货定价
- **输入**：ISIN/RIC/CUSIP → **输出**：净价、全价、收益率、久期、凸性、DV01
- **打比方**：你拿一只债券的"身份证号"去柜台，柜员告诉你它现在值多少钱、风险多大

**② 外汇定价（fx_spot_price / fx_forward_price）**：
- **干什么**：查即期汇率和远期汇率
- **输入**：货币对（如 EURUSD）→ **输出**：买价、卖价、中间价、远期点数
- **打比方**：你问"1 欧元现在值多少美元？"——即期告诉你现在，远期告诉你3个月后

**③ 曲线域（interest_rate_curve / credit_curve / inflation_curve / fx_forward_curve）**：
- **干什么**：获取各种收益率曲线和利差曲线
- **输入**：国家/货币/发行人类型 → **输出**：各期限利率/利差、贴现因子、远期利率
- **打比方**：曲线就像"利率地图"——告诉你不同期限的利率是多少

**④ 互换域（ir_swap）**：
- **干什么**：给利率互换定价
- **输入**：货币/指数/期限 → **输出**：平价利率、DV01、NPV
- **打比方**：你想要一个"固定换浮动"的互换——这个工具告诉你互换利率是多少

**⑤ 期权域（option_value / option_template_list）**：
- **干什么**：给期权定价，计算希腊字母
- **输入**：标的价格/行权价/期限/波动率 → **输出**：权利金、Delta、Gamma、Vega、Theta、Rho
- **打比方**：你买了一份保险——这个工具告诉你保险费多少、各种风险指标是多少

**⑥ 波动率域（fx_vol_surface / equity_vol_surface）**：
- **干什么**：生成隐含波动率曲面
- **输入**：货币对或股票 RIC → **输出**：各期限和行权价的隐含波动率
- **打比方**：波动率曲面就像"恐惧地图"——告诉你市场对不同情景的恐惧程度

**⑦ 量化分析（qa_ibes_consensus / qa_company_fundamentals / qa_historical_equity_price / qa_macroeconomic）**：
- **干什么**：获取分析师预测、公司财务、股价和宏观数据
- **打比方**：这是"研究部门"——帮你收集所有基本面和预期数据

**⑧ 时间序列（tscc_historical_pricing_summaries）**：
- **干什么**：获取历史价格数据
- **输入**：RIC + 时间范围 + 间隔 → **输出**：OHLCV 数据
- **打比方**：这是"历史档案馆"——帮你查看过去的价格走势

**⑨ YieldBook（yieldbook_bond_reference / yieldbook_cashflow / yieldbook_scenario / fixed_income_risk_analytics）**：
- **干什么**：债券参考数据、现金流预测、情景分析、风险分析
- **打比方**：这是"债券专科医院"——专门做债券的深度检查

### 4. YieldBook——"债券分析的瑞士军刀"

**YieldBook** 是 LSEG 旗下最专业的固定收益分析平台之一，最初由所罗门兄弟（Salomon Brothers）开发，后来被花旗收购，再后来被 LSEG 收购。它是债券交易员的"标配工具"。

**YieldBook 的四个工具**：

**① yieldbook_bond_reference**：
- 提供债券的"身份证信息"：证券类型、行业、评级、票息类型、赎回条款、发行人、国家
- **为什么重要**：你需要知道一只债券是"可赎回"还是"不可赎回"——这直接影响定价

**② yieldbook_cashflow**：
- 预测债券未来的每一笔现金流：什么时候付票息、什么时候还本金
- **为什么重要**：现金流是债券分析的基石——所有定价模型都基于现金流

**③ yieldbook_scenario**：
- 在不同利率情景下计算债券的价格变动
- **为什么重要**：回答"如果加息 100bp，这只债券亏多少？"

**④ fixed_income_risk_analytics**：
- 提供高级风险指标：OAS（期权调整利差）、有效久期、关键利率久期、凸性
- **为什么重要**：OAS 是可赎回债券的"真实利差"——它把赎回期权的影响剥离出来

### 5. SABR 模型——"波动率曲面的数学引擎"

**SABR（Stochastic Alpha Beta Rho）** 是金融界最流行的波动率模型之一，用于拟合期权隐含波动率曲面。`fx_vol_surface` 工具就使用 SABR 模型。

**为什么需要 SABR**：
- 简单的期权定价模型（如 Black-Scholes）假设波动率是常数——但现实中不同行权价的期权有不同的隐含波动率（这叫"波动率微笑"或"波动率偏斜"）
- SABR 模型能拟合这种"不是平的"波动率曲面

**SABR 的四个参数**：

| 参数 | 含义 | 类比 |
|------|------|------|
| **Alpha (α)** | 基准波动率水平 | "平均恐惧程度" |
| **Beta (β)** | 波动率与标的价格的关系 | "恐惧随价格变化的方式" |
| **Rho (ρ)** | 波动率与标的价格的相关性 | "越跌越怕还是越涨越怕" |
| **Nu (ν)** | 波动率的波动率 | "恐惧本身有多不稳定" |

**真实例子**：
- 股票期权通常是"越跌越怕"（Rho < 0）——崩盘时波动率飙升，这就是"波动率偏斜"
- 外汇期权通常是"两边都怕"（微笑形状）——暴涨暴跌时波动率都高
- SABR 能同时处理这两种情况

### 6. OHLCV——"技术分析的五要素"

**OHLCV** 是 Open（开盘价）、High（最高价）、Low（最低价）、Close（收盘价）、Volume（成交量）的缩写。它是技术分析的基础数据格式。

**每个要素的含义**：

| 要素 | 英文 | 含义 | 类比 |
|------|------|------|------|
| **O** | Open | 当天第一笔交易的价格 | 早上开门时的价格 |
| **H** | High | 当天最高成交价 | 当天最贵的一笔 |
| **L** | Low | 当天最低成交价 | 当天最便宜的一笔 |
| **C** | Close | 当天最后一笔交易的价格 | 关门时的价格 |
| **V** | Volume | 当天总成交股数/金额 | 多少人参与了交易 |

**为什么 OHLCV 重要**：
- **K 线图（Candlestick Chart）** 就是用 OHLC 画的——实体是开盘到收盘，影线是最高到最低
- **成交量（V）** 是"市场信心"的指标——放量上涨比缩量上涨更可信
- 本命令用 `tscc_historical_pricing_summaries` 获取 OHLCV 数据，支持日间（日/周/月）和日内（1分钟到1小时）间隔

> **给小白的一句话**：CONNECTORS.md 是 LSEG 插件的"工具箱目录"——20+ 个工具覆盖了债券、外汇、利率、期权、互换、波动率、量化分析、时间序列和固收分析九大领域。每个工具都是一把钥匙，打开金融数据的一扇门。你不需要记住所有工具——只需要知道"我要查什么，该用哪类工具"就够了。
