# 分析债券相对价值

> 本命令使用 LSEG 债券定价、收益率曲线、信用曲线和情景分析工具。可用工具参见 [CONNECTORS.md](../CONNECTORS.md)。

结合定价分析、收益率曲线背景、信用利差分解和利率冲击情景，对一只或多只债券进行相对价值分析。

领域知识参见 **bond-relative-value** 技能。

## 工作流程

### 1. 收集债券标识符

询问用户：
- 债券标识符 — ISIN、RIC 或 CUSIP（必填）
- 可选的基准债券用于比较
- 估值日期（可选，默认今日）

### 2. 对债券定价

使用标识符调用 `bond_price`。

提取：净价/全价、收益率、久期、凸性、DV01、货币。

如果提供了基准，也对其定价。

### 3. 获取无风险收益率曲线

调用 `interest_rate_curve`（先列出再计算）获取债券对应货币的曲线。

在债券到期日插值，计算 G-spread。

### 4. 获取信用利差曲线

调用 `credit_curve`（按国家/发行人类型搜索，然后计算）。

计算剩余利差 = 债券 G-spread 减去相同期限的信用曲线利差。剩余利差为正 = 便宜，为负 = 贵。

### 5. 运行情景分析

调用 `yieldbook_scenario`，设置平行利率变动：-100bp、-50bp、0bp、+50bp、+100bp。

提取每种情景下的价格变动和损益。

### 6. 整合报告

呈现：债券汇总表、利差分解（G-spread、信用利差、剩余利差）、情景损益表和贵贱评估。

如果提供了基准，包含并排比较。

## 输出格式

以贵/贱评估及支持证据开头，后跟利差分解和情景表格。

## 📖 本篇金融知识小贴士

**相对价值（Relative Value）**：判断一只债券相对于同类债券或基准是"贵"（rich）还是"便宜"（cheap）。贵 = 收益率偏低（价格偏高），便宜 = 收益率偏高（价格偏低）。

**G-spread**：债券收益率减去同期限国债收益率。衡量此债券比国债"多赚多少"。

**Z-spread**：在整条即期利率曲线上加的固定利差，使债券现金流的现值等于当前价格。比 G-spread 更精确。

**信用利差曲线（Credit Curve）**：同一发行人不同期限的信用利差连成的曲线，反映市场对信用风险随时间的看法。

**利差分解**：将债券总利差拆成"无风险利率部分 + 信用风险部分 + 剩余部分"。剩余部分反映流动性和技术面因素。

**平行利率变动**：假设所有期限的利率同时上涨或下跌相同幅度。这是最简单的利率情景分析方法。

**基点（bp，Basis Point）**：1个基点 = 0.01%。金融产品报价中极常用，说"加息25bp"就是0.25%。
