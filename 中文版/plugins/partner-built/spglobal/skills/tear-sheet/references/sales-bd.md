# 销售 / 业务拓展概览页面

## 用途
为销售或业务拓展团队准备客户会议而制作的简报文档。侧重了解潜在客户的业务、领导层公开关注的事项，以及将您的产品或服务与他们的需求连接起来的切入点。财务数据简化为规模和趋势。

这是叙事性最强的概览页类型。读起来像简报备忘录，而非数据表。

**默认页数：** 1-2 页。无严格惯例——可读性优先于紧凑性。

## 查询计划

从以下查询开始。如结果不完整，跟进更细的查询。

**查询 1 — 公司简介 + 财务数据：**
"[公司] 业务描述 概况 产品 服务 总部 员工 行业 收入 毛利率 最近2个财年"
→ 标题、公司概述、财务快照
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`
→ **立即写入** `/tmp/tear-sheet/financials.csv`

**查询 2 — 战略 + 财报：**
"[公司] 最近一次财报电话会 战略重点 CEO 评论 指引 关键举措"
→ 战略重点
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询 3 — 关系 + 新闻：**
"[公司] 关键客户 供应商 合作伙伴 竞争对手 业务关系 近期收购 合作伙伴关系 公告"
→ 关键关系、近期新闻
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

三个查询通常足够。对于知名公司，这些返回丰富的结果。对于较小或私人公司，查询 2 和 3 可能较稀疏——这没问题。定性章节仍然有价值，因为 Claude 会将任何可用数据综合为有用的框架。

## 章节

按优先级排序。如空间受限，从底部开始裁剪。

### 1. 公司标题
简化版——仅识别信息，无需估值指标。

| 字段 | 说明 |
|---|---|
| 公司名称 | 大号加粗 |
| 股票代码和交易所 | 如上市；未上市则省略 |
| 总部 | 城市、州/国家 |
| 行业 | |
| 员工 | |
| 年收入（最新） | 按最近十亿或百万取整 |
| 市值 | 如上市；未上市则省略 |
| 财年收入指引 | 如财报数据可用；例如"6-8%有机增长" |
| 财年 EPS 或 EPS 指引 | 如上市公司；从 `earnings.txt` 提取 |
| 关键财务指标 | 调整后 EPS、EBITDA 利润率或类似——一条标题数字 |

前瞻指引和 EPS 数据来自查询 2（财报）——填充标题时从 `earnings.txt` 引用，而不是仅从 `company-profile.txt`。

顶部一条水平带。无企业价值、无Beta——这个受众不需要它们。

### 2. 公司概述
最重要的部分。销售代表阅读后应能在会议中对该潜在客户有准确把握。

**不要直接粘贴 CIQ 公司摘要。** 它读起来像 SEC 文件，会失去非金融读者的注意力。以 3-5 句通俗语言重写：
- 公司做什么（不用投资术语——用"卖"而不是"变现"，用"年收入"而不是"复合年增长率"）
- 他们的客户是谁
- 他们如何赚钱
- 在哪些地方运营
- 他们与众不同之处

使用散文段落。非金融专业人士应能理解每句话。

### 3. 战略重点与高管评论
领导层现在关注什么？如果您知道 CEO 在谈什么，就找到了切入口。

**这不是财报回顾。** 不要重用与股权研究概览页相同的要点。提取 3-5 个**战略主题**，附主题标签和支持数据点：

- **AI/数据变现：** 管理层正在大力投资 AI 能力，在 FY2024 投入 X 于研发，推出了三款新的 AI 驱动产品。
- **利润率扩张：** CEO 将运营效率列为首要任务，目标通过自动化实现 200bps 的营业利润率提升。
- **国际增长：** 在 EMEA 开设新办公室；欧洲现占收入的 25%，高于之前的 18%。

每条要点应回答："这家公司现在关心什么，有什么证据支撑？"销售代表应从中立即看到连接自己产品的切入点。

如财报电话会数据不可用（私人公司常见），以"近期发展"替代，从工具返回的任何新闻或公告中提取信息。

### 4. 财务快照
简化版。单个紧凑表格——仅需规模、方向和一个盈利信号。销售代表需要知道三件事：这家公司多大、在增长吗、健康吗。

| 指标 | FY[前一年] | FY[最新] | 同比变化 |
|---|---|---|---|
| 收入 | | | +X% |
| 毛利率% | | | +X.Xpp |
| 员工 | | | |

最多三行。无 EBITDA、无 EV 倍数、无资产负债表、无净利润。毛利（而非 EBITDA 或净利润）是销售受众的合适单一盈利指标——它直观（"扣掉直接成本后剩多少"），无需金融知识也能说明业务健康状况。如果代表需要更深入的财务信息，可以要求股权研究版本。

对于私人公司，此表可能只有收入和员工——两项显示规模和增长的数据点对销售会议已足够。

### 5. 关键关系与生态
帮助代表了解潜在客户的世界并找到切入点。

四个标签列表，每类 3-5 个名称，附带一行注释：

**客户：** 他们卖给谁。帮助了解其市场策略和共享目标客户。
示例："JPMorgan Chase — 客户（企业分析平台部署）"

**供应商 / 技术：** 他们买什么。发现替代或整合机会。
示例："AWS — 供应商（主要云基础设施）；Snowflake — 供应商（数据仓库）"

**合作伙伴：** 他们与谁合作。联合销售角度、共同联系人。
示例："Deloitte — 合作伙伴（企业客户实施）"

**竞争对手：** 他们与谁竞争。定位背景。
示例："Bloomberg — 竞争对手（金融数据终端）"

每条需要一个括号注释，让代表了解**关系意味着什么**，而不仅仅是**是谁**。

如关系数据不可用，注明"关系数据不可用"并继续。

### 6. 近期新闻与发展
3-5 条要点，每条一句话，最新在前。**以最近一期财报为首**——这是销售会议中最重要的近期事件，定下所有其他内容的基调。然后：并购、产品发布、领导层变更、重大赢单。

**在节末包含即将到来的催化剂**，用"**即将到来：**"标签。示例：即将到来的财报日期、待定剥离、产品发布、投资者日、监管里程碑。这些创造了自然的销售接触窗口，通常是联系的最佳理由（"我注意到贵公司Q2有投资者日……"）。从财报电话会评论和并购数据中提取（待定交易）。

### 7. 对话切入点（首先裁剪）
综合上述数据提出的 2-3 条建议谈话要点。**每条切入点必须引用第 3 节中的特定战略重点**——通用问题没用。以问题形式呈现。

**给每条切入点标记建议角色**——例如（CTO）、（CFO）、（CDO）、（数据主管）、（工程VP）、（采购主管）。与 CTO 会面的代表需要与 CFO 不同的开场白。如用户指定了会面对象，仅为该角色生成切入点。

示例：
- *(CTO/CDO)* "您在最近一期财报中提到了[第 3 节中的具体 AI 计划]——这对您的[相关需求]有什么影响？"
- *(CFO/COO)* "鉴于您的[第 3 节中的利润率扩张目标]，您是否在评估能加速实现该目标的工具？"
- *(战略VP/企发)* "我看到[公司]最近收购了[目标]——这是否改变了您对[能力]的策略？"

如用户描述了他们在销售什么产品/服务，将这些切入点与该产品连接。如未描述，保持切入点在潜在客户的战略主题中，但解决方案框架保持通用。

**验证测试：** 最终确定每条对话切入点前，确认它包含至少一个来自概览页的具体数字、日期、产品名或计划名称。一个适用于该行业任何公司的问题不是好的对话切入点。"您如何看待AI？"太泛。而"您的 CEO 提到自 2018 年以来投入了 10 亿美元在 AI 上，20% 的 iLEVEL 自动导入功能已被采纳——这对您的分析师工作流有何改变？"既具体又展现了准备充分。

本节是 Claude 的综合，而非来自工具的原始数据。清晰标记。

## 格式说明
- **这应是最温暖、最可读的概览页类型。** 内容严谨，但格式应像简报备忘录，而非数据终端。
- 正文：使用完整 9pt（size: 18）。不压缩——可读性优先于密度。
- 行间距：正文段落 1.15 倍（略高于默认）。此模板中空白是特色而非浪费。
- 公司概述和战略重点应占约 40-50% 的页面。
- 财务快照：视觉紧凑，次要地位。保持小巧干净。
- 关键关系：考虑两栏布局（客户 + 供应商在左，合作伙伴 + 竞争对手在右）以节省垂直空间同时保持注释可见。
- 对话切入点：使用缩进块级项目符号。这些是文档的回报——应在视觉上突出显示。

## 📖 本篇金融知识小贴士

**销售代表的财务知识**：销售不需要了解 EBITDA 或 DCF。他们需要知道公司规模（收入）、是否增长（增长率）和基本赚钱能力（毛利率）。这就是本节仅关注这三项的原因。

**年度经常性收入（ARR，Annual Recurring Revenue）**：SaaS 公司最重要的指标——每年从订阅客户那里获得的经常性收入。比总收入更能反映业务健康状况。

**毛利率（Gross Margin）**：（收入 - 直接成本）÷ 收入。衡量"每赚1块钱能留下多少"。高毛利（>70%）通常意味着产品有定价权和护城河。

**客户获取成本（CAC，Customer Acquisition Cost）**：获得一个新客户花了多少钱（销售 + 市场费用 ÷ 新增客户数）。与 LTV（客户生命周期价值）结合使用。

**近因效应（Recency Effect）**：人们更容易记住最近发生的事情。在销售中，最近的公司新闻（尤其财报）是最好的谈话切入点。

**话题切入点（Conversation Starter）**：基于对潜在客户的研究而设计的初始话题。好的切入点显示你做了功课，并直接切入对方关心的事情。

**理想客户画像（ICP，Ideal Customer Profile）**：最适合你产品的客户类型的详细描述。销售团队用 ICP 来优先选择目标客户。

**年度合同价值（ACV，Annual Contract Value）**：每个客户合同每年的平均价值。高 ACV 客户通常值得投入更多销售精力。

## Appendix: 金融背景知识

你手上拿的这页纸，是给销售代表见客户前做功课用的"剧本"。它把客户公司的"故事"浓缩成一两页——公司干什么的、最近在忙什么、跟谁是竞争对手、最好的开场白是什么。讲穿了就是一件事：**让销售代表在 30 分钟的会面里像老朋友一样跟客户聊，而不是像个陌生的推销员**。

这份附录就来帮你搞懂这页"剧本"里出现的术语、逻辑和底层思维。

### 1. 销售这份工作到底在干嘛

#### 1.1 销售的本质

**想象一个场景**：你是 Snowflake 的销售，公司卖云数据仓库给企业。一天你要做的事是：

1. **找潜在客户**——研究哪些公司有大量数据要分析
2. **接触**——给 CIO 发邮件、打电话、加 LinkedIn
3. **摸需求**——第一次见面，了解对方在用什么工具，痛点是什么
4. **演示产品**——展示 Snowflake 怎么帮他解决问题
5. **谈判**——价格、合同期、付款方式
6. **成交**——签单
7. **交付**——把客户转给客户成功团队
8. **扩展**——第二年跟客户说"要不要再买 50% 的容量"

这就是 B2B 销售的日常。**销售不是"卖东西"，而是"帮人解决问题并赚到回报"**。

#### 1.2 销售 vs 业务拓展

很多公司把这两个词混着用，其实有区别：

- **销售（Sales）**：目标明确——**把已经识别好的潜在客户变成签单客户**。周期短（1-12 个月），看收入数字。
- **业务拓展（BD, Business Development）**：目标更虚——**建立关系、创造未来才有的机会**。周期长（6-24 个月），看管道（pipeline）、合作伙伴、战略合作。

举个具体例子：

- 销售：明天去见摩根大通的 CIO，推销我们的产品
- BD：跟 Snowflake 谈战略合作，6 个月后把我们的产品集成到 Snowflake 生态里

BD 的"交易"经常不是直接收入，而是**未来潜在收入的入口**。比如你跟 Snowflake 谈成合作，明年 Snowflake 的客户可能会自动成为你的线索——这种价值在 BD 的 KPI 里体现为"BD-influenced revenue"（BD 影响的收入）。

#### 1.3 销售漏斗——"100 个客户最后签几个"

销售流程像漏斗，**越往下越窄**：

```
100 个陌生人知道你公司（认知）
  ↓ 30 个成为"潜在客户"（线索 Lead）
  ↓ 10 个真的在评估（机会 Opportunity）
  ↓ 5 个认真谈判（Proposal）
  ↓ 2 个真要买（Negotiation）
  ↓ 1 个真签了（Closed Won）
```

**每一步的转化率都是销售管理的核心**。比如你的"线索 → 机会"只有 5%（而不是行业平均 20%），说明销售代表的"找对人"能力有问题。

**4× 法则**：要保证年底达成 $100 万销售目标，**销售管道（pipeline）里得有 $400 万的机会**。因为按经验，销售管道里只有 25% 真的能签下来。

#### 1.4 销售方法论

老销售有自己的"套路"，新销售要学这些套路。**最著名的几个框架**：

- **BANT**：Budget（客户有没有钱）、Authority（决策者是谁）、Need（客户是不是真有问题）、Timeline（什么时候买）——快速判断一个潜在客户值不值得花时间。
- **MEDDIC**：更复杂，针对大客户销售。要弄清**M**etrics（量化影响）、**E**conomic Buyer（真正掏钱的人）、**D**ecision Criteria（采购标准）、**D**ecision Process（决策流程）、**I**dentify Pain（找到客户的痛）、**C**hampion（找到内部支持者）。
- **SPIN**：顾问式销售——**S**ituation（了解现状）、**P**roblem（找问题）、**I**mplication（放大问题的后果）、**N**eed-Payoff（让客户自己说"我需要解决"）。
- **Challenger**：挑战式销售——主动教育客户、重新定义问题、逼客户思考。适合卖颠覆性产品。

**简单说**：BANT 帮你过滤"烂客户"，MEDDIC 帮你搞定"大客户"，SPIN 让你"成为客户的顾问"，Challenger 让你"教育市场"。

#### 1.5 销售组织里的角色

- **SDR（Sales Development Rep）/ BDR（Business Development Rep）**：专门开发新线索的年轻人。底薪低（$5 万）、靠提成、压力大。目标是每月产出 10-15 个合格线索。
- **AE（Account Executive）**：客户经理。SDR 给的线索到他手里变成签单。SMB AE 一年配额 $50-80 万，Enterprise AE $200-300 万+。
- **SE（Sales Engineer）/ SA（Solution Architect）**：售前技术。负责产品演示、技术答疑、POC（Proof of Concept，概念验证）。复杂产品必须有。
- **CSM（Customer Success Manager）**：客户成功。签单后接手，确保客户用得好、第二年续约、最好扩展买更多。SaaS 公司的命脉。
- **CRO（Chief Revenue Officer）**：销售总负责。
- **RevOps（Revenue Operations）**：把销售、营销、客户成功的数据打通、优化流程、选工具。

**典型销售代表的薪酬结构**（北美 SaaS 公司 2024 年水平）：
- SDR：底薪 $5 万 + 提成 → OTE（On-Target Earnings，目标总收入）$8-9 万
- SMB AE：底薪 $7 万 + 10% 提成 → OTE $14 万
- Enterprise AE：底薪 $15 万 + 10% 提成 → OTE $30-35 万

### 2. 销售只关心客户的 3 个数字——为什么

这页模板里反复说"销售不需要看 EBITDA、DCF、估值倍数"。为什么？

**因为销售代表不是来给客户估值的，是来卖东西的**。

客户买你东西的决策不是"我们公司值不值这个价"，而是"我花 100 万买你产品，能帮我省 500 万或者多赚 1000 万吗"。**这是 ROI 思维，不是估值思维**。

所以销售只需要看 3 个数字：

1. **收入（Revenue）**：客户有多大。决定它有没有钱买你的东西。苹果年收入 $3900 亿、Stripe 2024 年支付额 $1.4 万亿——这种客户花 $100 万买你产品跟玩一样。
2. **增长率（Growth Rate）**：客户在扩张还是在收缩。在扩张的公司才愿意花钱买新工具。增长率 > 20% 的是健康、激进的公司；负增长的公司在砍预算。
3. **毛利率（Gross Margin）**：客户赚不赚钱。高毛利率公司愿意花更多钱买工具。SaaS 80% 毛利 = 卖 $100 赚 $80 = 客户花得起；零售 20% 毛利 = 卖 $100 赚 $20 = 客户抠。

**真实案例**：2024 年 Snowflake 的 NRR（净收入留存率）133%，意思是老客户第二年自动多花了 33%。这种"客户自然扩展"才是 SaaS 销售的圣杯——不用费力找新客户，老客户自己就多买了。

### 3. 战略重点——"客户 CEO 心里到底在想什么"

#### 3.1 为什么要关心 CEO 想什么

销售代表见客户时，**最大的忌讳是"我不懂你的业务"**。如果你跟 Snowflake 的 CIO 聊，但完全不知道 Snowflake 最近在忙什么、CEO 反复强调什么，那这场对话就完了。

**怎么找到这些信息？**

1. **财报电话会**（季度）：CEO 和 CFO 在电话里讲未来 12 个月要做什么。反复出现的关键词 = 真正的战略重点。Snowflake 2024 年财报里"AI"、"Workload Optimization"出现 100 多次。
2. **致股东信**（年度）：巴菲特、贝佐斯这些 CEO 写的信就是"我的世界观"。
3. **10-K 风险因素**：上市公司必须披露公司面临的风险。**管理层的焦虑 = 你的销售机会**。如果客户 10-K 里 5 次提到"网络安全风险"，那 CISO 一定会买安全产品。
4. **招聘趋势**：LinkedIn 上客户的招聘速度能反映战略方向。客户突然猛招"AI Engineer" = 战略重点是 AI。
5. **行业大会演讲**：CEO 在大会讲什么、强调什么。

#### 3.2 常见战略主题

- **AI / 数据变现**——客户在投 AI。卖 AI 基础设施、AI 应用、数据中台。
- **利润率扩张**——客户在压缩成本。卖自动化、效率工具。
- **国际增长**——客户在出海。卖 i18n、本地化、合规。
- **平台化**——客户在变成平台。卖 API、开发者工具、生态系统。
- **整合并购**——客户在买买买。卖合并后的整合工具。
- **ESG**——客户在搞可持续。卖碳追踪、ESG 报告工具。

#### 3.3 怎么把战略和销售连起来

**具体例子**：

客户是 Snowflake，2024 年 CEO 在电话会反复说"我们要让客户的工作负载更便宜"（cost optimization）。

那销售可以这样说：

> "我注意到您在最近的财报中提到 20% 的客户在降本压力下重新评估云数据仓库的成本结构。我们的工具帮 Databricks 的客户平均节省 30% 的存储成本——您想了解一下我们怎么做到的？"

**这个开场白为什么有效**：
- ✅ 引用了客户的战略重点（成本优化）
- ✅ 引用了真实数字（30%）
- ✅ 用客户视角（不是"我们产品多牛"，而是"我们能帮您省钱"）
- ✅ 问题化引导下一步

**对比差的开场**：

> "我们是行业领先的 AI 数据平台，您有兴趣了解一下吗？"

❌ 没有引用任何客户的具体情况
❌ 没有针对客户的战略
❌ 听起来像群发邮件

#### 3.4 角色化切入点

不同的客户角色，关心不同的事。**你跟 CTO 谈技术，跟 CFO 谈 ROI，跟 CEO 谈战略**。

- **CTO / CDO（首席技术官/数据官）**：关心技术架构、平台化、AI 能力。开场聊"我们怎么帮您的数据架构升级"。
- **CFO（首席财务官）**：关心 ROI、成本、合规。开场聊"我们怎么帮您省 $X 百万"。
- **COO（首席运营官）**：关心运营效率、流程。开场聊"我们怎么让您的运营效率提升 30%"。
- **CISO（首席信息安全官）**：关心安全、合规。开场聊"我们怎么帮您通过 SOC 2 审计"。
- **CMO（首席营销官）**：关心获客、品牌。开场聊"我们怎么帮您降低 CAC"。
- **战略 VP / 企发**：关心并购、合作伙伴。开场聊"我们怎么帮您的下一个并购整合更快"。
- **采购 / IT 总监**：关心合规、安全、成本。开场聊"我们怎么帮您简化采购流程"。

**这页模板里"角色化切入点"的设计原则**就是：好的开场白必须**同时引用客户的具体战略 + 数字 + 角色关注**。

### 4. 财务快照（销售版）——3 行就够

#### 4.1 真实案例

假设你要去见一家叫 Initech 的客户（年收入 $5 亿、SaaS 公司、增长 30%）。你的财务快照表格应该是：

| 指标 | 去年 | 今年 | 同比 |
|------|------|------|------|
| 收入 | $4 亿 | $5 亿 | +25% |
| 毛利率 | 75% | 78% | +3pp |
| 员工 | 800 | 1000 | +25% |

**为什么这样就够了**：

- **收入 $5 亿**：说明客户规模大，IT 预算可能 $2000 万-$5000 万，你卖 $50 万的合同它不会皱眉头
- **增长 25%**：说明在扩张，会买新工具
- **毛利率 78%**：典型 SaaS，业务健康
- **员工 +25%**：跟收入增长匹配，没问题

**为什么不再加 EBITDA、净利润**：

- 销售不需要判断客户盈不盈利，只需要判断"它付不付得起钱"
- 客户亏钱也可能付得起钱（很多 SaaS 公司亏损但有现金）
- 复杂数字让销售代表疏远，对销售会议无价值

#### 4.2 私人公司怎么办

私人公司不一定公开财务数据。**替代方案**：

- **员工数**（领英有）：增长 = 在扩张
- **融资历史**（Crunchbase、PitchBook）：融了多少钱 = 还有多少现金
- **招聘速度**（LinkedIn Jobs）：招聘猛 = 在扩张
- **办公室扩张**（新闻、卫星图）：新办公室 = 在扩张
- **新闻 / 媒体曝光**（TechCrunch 等）：高曝光 = 在做事

### 5. 客户关系生态——4 个圈子

#### 5.1 4 类关系

任何公司都活在 4 类关系里：

1. **客户（他们卖给谁）**：帮你了解他们的市场、规模、品味
2. **供应商（他们买什么）**：发现替换/整合机会
3. **合作伙伴（他们跟谁合作）**：联合销售、互相介绍
4. **竞争对手（他们跟谁抢）**：定位参考

**真实案例**：假设你卖 CRM 工具，要去见 Snowflake。

- **客户**：Snowflake 卖云数据仓库给 Comcast、AT&T、Capital One。**意味着**：这些公司可能是你的潜在客户。
- **供应商**：Snowflake 用 AWS 作为云基础设施，用 Snowflake 自家的产品，用 Okta 做身份认证。**意味着**：可以跟 Okta 谈"我们互相推荐客户"。
- **合作伙伴**：Snowflake 跟 Databricks 既是合作伙伴（互相集成）又是竞争对手。**意味着**：复杂关系，需要小心。
- **竞争对手**：Snowflake 直接对标 Databricks、BigQuery、Redshift。**意味着**：定价参考。

#### 5.2 关系注释为什么重要

这页模板反复强调：**每条关系必须有"括号注释"**，比如 "AWS — 供应商（主要云基础设施）"。

原因很简单：

- ❌ "JPMorgan Chase" → 销售代表只知道"是个银行"，不知道怎么切入
- ✅ "JPMorgan Chase — 客户（已用 Snowflake 做企业分析）" → 销售代表立刻知道"哦，JPMorgan 已经在用云数仓了，可能也在评估其他数据工具"

注释 = 关系的"应用场景"。

### 6. 销售能用的催化剂

#### 6.1 什么是"催化剂"

催化剂（Catalyst）就是**即将发生的、能让你有理由联系客户的事件**。

**好的催化剂**：
- 客户即将发布财报（你可以提前一周发"准备问题清单"）
- 客户刚发布新产品（"我看到你们 X 发布了，我们 Y 可以配合"）
- 客户换了新 CFO（"新 CFO 上任通常会重新评估供应商"）
- 客户刚收购一家公司（"整合后通常需要 X"）
- 客户在开投资者日（"你们 X 战略提到了什么"）
- 监管变化（比如新的 AI 法规出炉）

#### 6.2 为什么催化剂是销售利器

因为**销售最怕被拒绝**。如果你说"我想跟您聊一下我们的产品"——80% 被拒绝。但如果你说"我看到您下个月有财报，想给您发个分析师关注点清单"——被接受的概率大得多。

**重要原则**：催化剂要在事件**前 2-4 周**联系，这样客户有时间安排。等事件结束了再联系，意义就不大了。

### 7. 怎么找客户信息——研究工具箱

#### 7.1 免费工具

- **LinkedIn**：看组织架构、关键员工、招聘信息
- **Crunchbase**：初创公司融资历史
- **SEC EDGAR**：上市公司财报（10-K、10-Q、8-K）
- **Google News**：新闻
- **G2 / TrustRadius**：客户评价

#### 7.2 付费工具

- **PitchBook**：PE / VC 数据库
- **Crunchbase Pro**：初创公司深度
- **ZoomInfo**：联系人和公司数据
- **Apollo**：联系人和意向
- **LinkedIn Sales Navigator**：领英销售版

#### 7.3 研究深度

### 8. 客户画像（ICP）——"我最该找谁"

#### 8.1 ICP 是什么

**ICP（Ideal Customer Profile，理想客户画像）**就是"我产品最适合卖给什么样的公司"。

**举例**：假设你卖企业级数据安全产品，你的 ICP 可能是：
- 行业：金融、医疗、政府
- 规模：1000-10000 员工
- 收入：$5 亿-$50 亿
- 地域：北美 + 西欧
- 痛点：被监管、被攻击过、需要 SOC 2 合规
- IT 预算：$1000 万+

**为什么要 ICP**：销售时间有限，不可能谁都是客户。**找对客户 = 销售效率提升 5 倍**。

#### 8.2 ICP vs 买方画像

- **ICP**：描述公司类型（金融行业、5000 员工、$20 亿收入）
- **Buyer Persona（买方画像）**：描述个人（45 岁、CIO、关心合规、被前公司攻击过）

**两者关系**：ICP 决定"找哪些公司"，Buyer Persona 决定"进公司后找谁"。

### 9. SaaS 销售专属——为什么这行特别

#### 9.1 SaaS 是什么

**SaaS（Software as a Service）** = 你不卖光盘、卖订阅。比如你不用买 Office 光盘，每月付 $10 用 Microsoft 365。

**SaaS 跟传统软件的区别**：
- 传统软件：一次付 $1000，以后用一辈子（90 年代）
- SaaS：每月付 $50（2020 年代）

**SaaS 对销售的意义**：
- **可预测收入**：知道客户签了 1 年合同，明年续约 = 已有收入
- **扩展销售**：客户增长就会多买（座位多了、用量多了）
- **客户成功至关重要**：客户用得好就续约，用得差就流失

#### 9.2 SaaS 关键指标

- **ARR（Annual Recurring Revenue）**：年化经常性收入。最重要。
- **MRR（Monthly Recurring Revenue）**：月度版本
- **ACV（Annual Contract Value）**：单客户年均贡献
- **NRR（Net Retention Rate）**：净收入留存率。> 100% 表示客户在加钱买
- **GRR（Gross Retention Rate）**：总留存率。> 90% 健康
- **LTV（Lifetime Value）**：一个客户一辈子贡献多少
- **CAC（Customer Acquisition Cost）**：获得一个客户花了多少
- **LTV/CAC**：投资回报。> 3 健康
- **CAC Payback**：CAC 回收期。最好 < 18 个月
- **Rule of 40**：增长率 + 利润率 > 40% 才算好公司

**真实数据（2024 年水平）**：
- Datadog：ARR 约 $26 亿、NRR 115%+
- Snowflake：ARR 约 $33 亿、NRR 127%
- Atlassian：NRR 120%+

#### 9.3 SMB / Mid-Market / Enterprise

SaaS 公司按客户规模分三层：

- **SMB**（中小企业）：客户 < $10K/年。销售周期 1-4 周。决策者部门主管。销售可以远程、内部。
- **Mid-Market**（中端市场）：$10K-$100K/年。销售周期 1-3 月。决策者 VP。远程为主。
- **Enterprise**（企业级）：> $100K/年。销售周期 6-18 月。决策者 C 级。需要现场拜访 + 复杂谈判。

**为什么分层**：不同规模的销售方式完全不一样。SMB 销售可以一次卖 100 个客户，Enterprise 销售可能要花 2 年搞定 1 个客户。

### 10. 销售漏斗的关键转化率

#### 10.1 4× 法则再深入

如果你年底目标 $100 万，按经验：
- 你得有 $400 万的"机会管道"（pipeline）
- 这意味着你今年要创造 $800 万 - $1500 万的"销售合格线索（SQL）"
- 这意味着你（或 SDR）要接触 5000-10000 个潜在客户
- 转化率大概 1-3%

**为什么是 4× 而不是 2×？** 因为销售管道里的"机会"质量参差不齐，预测的胜率（win rate）平均只有 25%。所以 4× 才能保底。

#### 10.2 销售周期越短越好

销售周期（从首次接触到签单的时间）越短，效率越高。

- SMB SaaS 销售周期 1-4 周
- Mid-Market SaaS 1-3 月
- Enterprise SaaS 6-18 月

**缩短销售周期的方法**：
- 客户研究做透（见客户前就知道他痛点）
- 决策者早期接触（不要 6 个月才发现"哦这事得找 CIO"）
- 提供 POC（概念验证）让客户"先试再买"
- 减少决策环节

### 11. CRM——销售的"记忆外挂"

#### 11.1 什么是 CRM

**CRM（Customer Relationship Management）**就是"销售专用的 Excel"。

记录：每个客户、每次联系、每个机会、每个活动。没有 CRM 的销售团队 = 没有记忆的团队。

#### 11.2 主要 CRM 系统

- **Salesforce**：行业标准，贵，但功能最全。大型企业首选。
- **HubSpot**：中小企业首选，界面友好。
- **Microsoft Dynamics 365**：已经用 Microsoft 的大公司首选。
- **Pipedrive**：销售导向、简单、适合小团队。
- **Gong / Chorus**：记录销售通话，分析"为什么丢单、为什么赢单"。

**为什么 CRM 重要**：销售代表一周见 20 个客户，没有 CRM 一周后就忘了谁说过什么。**CRM 是销售代表的"外脑"**。

### 12. 对话切入点设计——"好的开场白长什么样"

#### 12.1 黄金法则

> 好的切入点必须**同时**包含三件事：
> 1. 客户的具体战略（CEO 提过的、财报提过的、新闻提过的）
> 2. 一个具体数字（10 亿美元、20%、3 个月）
> 3. 一个具体问题

**差的开场**（"通用型"，对任何公司都适用）：

> "您对 AI 怎么看？"

✅ 没引用客户具体情况
✅ 没具体数字
✅ 问题太宽
✅ 听起来像群发

**好的开场**（"定制型"，只有这个公司适用）：

> "我注意到 Snowflake 在最近的财报中提到你们要在 2025 年把数据处理成本降低 20%。我们帮 Databricks 实现了 30% 的存储成本优化——您想了解一下我们怎么做的吗？"

✅ 引用了 Snowflake 自己的战略（降本）
✅ 用了具体数字（20%、30%）
✅ 提到了类似的客户（Databricks）
✅ 问题化引导

#### 12.2 切入点的"角色化"

不同的客户角色，要用不同的切入点：

- 对 **CTO**：聊技术架构、平台化
- 对 **CFO**：聊 ROI、利润率
- 对 **CMO**：聊获客成本、品牌
- 对 **CISO**：聊安全、合规
- 对 **战略 VP**：聊并购、合作伙伴

**为什么**：CFO 不想听你的产品有什么功能，他想听"花 100 万能省 500 万"。CTO 不想听省钱，他想听"我们怎么帮你解决 X 架构问题"。

### 13. 一些容易混淆的概念

#### 13.1 ARR vs Revenue

- **Revenue**：这季/这年实际确认的收入
- **ARR**：年化"经常性"收入。签了 3 年 $300 万合同 = ARR $100 万（虽然第一年现金只收到 $100 万）

**为什么 ARR 重要**：SaaS 公司的真实价值不在于"今年收了多少现金"，在于"未来 12 个月锁定了多少经常性收入"。

#### 13.2 线索、机会、签单——三个完全不同的事

- **线索（Lead）**：可能感兴趣的人
- **机会（Opportunity）**：在评估我们产品的潜在客户
- **签单（Closed Won）**：已经签合同

**线索 → 机会** 是 SDR 的工作（过滤）
**机会 → 签单** 是 AE 的工作（成交）

#### 13.3 Quota vs Forecast

- **Quota（配额）**：管理层给销售代表的目标
- **Forecast（预测）**：销售代表认为能完成多少

**为什么 Forecast 比 Quota 重要**：销售代表可能 quota 是 $100 万，但他心里清楚"这季大概能签 $80 万"。Forecast 决定公司财务怎么准备。

### 14. 给销售小白的一份速查清单

#### 14.1 见客户前的功课清单

- [ ] 客户做什么业务？（5 分钟看官网）
- [ ] 客户最近一次财报的关键数字？（收入、增长、毛利）
- [ ] 客户 CEO 反复强调什么战略？（财报电话会、致股东信）
- [ ] 客户最近有什么大新闻？（收购、新产品、新领导）
- [ ] 客户的客户是谁？供应商是谁？竞争对手是谁？
- [ ] 我要见的客户角色关心什么？（CTO 看技术、CFO 看 ROI）
- [ ] 3 个具体切入点（每个都引用客户的具体数字 + 战略）

#### 14.2 见客户时的黄金原则

- [ ] **不卖产品，卖问题**——先聊客户痛点，不要先说我们多牛
- [ ] **引用客户的具体事**——财报数字、CEO 的话、新闻
- [ ] **角色化**——跟 CTO 谈技术，跟 CFO 谈钱
- [ ] **问问题，不要讲 PPT**——好销售 70% 时间在问
- [ ] **记住催化点**——下次联系的"理由"
- [ ] **不贬低对手**——产品对比要客观
