# 销售 / 业务拓展概览页面

## 用途
为销售或业务拓展团队准备客户会议而制作的简报文档。侧重了解潜在客户的业务、领导层公开关注的事项，以及将您的产品或服务与他们的需求连接起来的切入点。财务数据简化为规模和趋势。

这是叙事性最强的概览页类型。读起来像简报备忘录，而非数据表。

**默认页数：** 1-2 页。无严格惯例——可读性优先于紧凑性。

## 查询计划

从以下查询开始。如结果不完整，跟进更细的查询。

**查询 1 — 公司简介 + 财务数据：**
"[公司] 业务描述 概况 产品 服务 总部 员工 行业 收入 毛利率 最近2个财年"
→ 标题、公司概述、财务快照
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`
→ **立即写入** `/tmp/tear-sheet/financials.csv`

**查询 2 — 战略 + 财报：**
"[公司] 最近一次财报电话会 战略重点 CEO 评论 指引 关键举措"
→ 战略重点
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询 3 — 关系 + 新闻：**
"[公司] 关键客户 供应商 合作伙伴 竞争对手 业务关系 近期收购 合作伙伴关系 公告"
→ 关键关系、近期新闻
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

三个查询通常足够。对于知名公司，这些返回丰富的结果。对于较小或私人公司，查询 2 和 3 可能较稀疏——这没问题。定性章节仍然有价值，因为 Claude 会将任何可用数据综合为有用的框架。

## 章节

按优先级排序。如空间受限，从底部开始裁剪。

### 1. 公司标题
简化版——仅识别信息，无需估值指标。

| 字段 | 说明 |
|---|---|
| 公司名称 | 大号加粗 |
| 股票代码和交易所 | 如上市；未上市则省略 |
| 总部 | 城市、州/国家 |
| 行业 | |
| 员工 | |
| 年收入（最新） | 按最近十亿或百万取整 |
| 市值 | 如上市；未上市则省略 |
| 财年收入指引 | 如财报数据可用；例如"6-8%有机增长" |
| 财年 EPS 或 EPS 指引 | 如上市公司；从 `earnings.txt` 提取 |
| 关键财务指标 | 调整后 EPS、EBITDA 利润率或类似——一条标题数字 |

前瞻指引和 EPS 数据来自查询 2（财报）——填充标题时从 `earnings.txt` 引用，而不是仅从 `company-profile.txt`。

顶部一条水平带。无企业价值、无Beta——这个受众不需要它们。

### 2. 公司概述
最重要的部分。销售代表阅读后应能在会议中对该潜在客户有准确把握。

**不要直接粘贴 CIQ 公司摘要。** 它读起来像 SEC 文件，会失去非金融读者的注意力。以 3-5 句通俗语言重写：
- 公司做什么（不用投资术语——用"卖"而不是"变现"，用"年收入"而不是"复合年增长率"）
- 他们的客户是谁
- 他们如何赚钱
- 在哪些地方运营
- 他们与众不同之处

使用散文段落。非金融专业人士应能理解每句话。

### 3. 战略重点与高管评论
领导层现在关注什么？如果您知道 CEO 在谈什么，就找到了切入口。

**这不是财报回顾。** 不要重用与股权研究概览页相同的要点。提取 3-5 个**战略主题**，附主题标签和支持数据点：

- **AI/数据变现：** 管理层正在大力投资 AI 能力，在 FY2024 投入 X 于研发，推出了三款新的 AI 驱动产品。
- **利润率扩张：** CEO 将运营效率列为首要任务，目标通过自动化实现 200bps 的营业利润率提升。
- **国际增长：** 在 EMEA 开设新办公室；欧洲现占收入的 25%，高于之前的 18%。

每条要点应回答："这家公司现在关心什么，有什么证据支撑？"销售代表应从中立即看到连接自己产品的切入点。

如财报电话会数据不可用（私人公司常见），以"近期发展"替代，从工具返回的任何新闻或公告中提取信息。

### 4. 财务快照
简化版。单个紧凑表格——仅需规模、方向和一个盈利信号。销售代表需要知道三件事：这家公司多大、在增长吗、健康吗。

| 指标 | FY[前一年] | FY[最新] | 同比变化 |
|---|---|---|---|
| 收入 | | | +X% |
| 毛利率% | | | +X.Xpp |
| 员工 | | | |

最多三行。无 EBITDA、无 EV 倍数、无资产负债表、无净利润。毛利（而非 EBITDA 或净利润）是销售受众的合适单一盈利指标——它直观（"扣掉直接成本后剩多少"），无需金融知识也能说明业务健康状况。如果代表需要更深入的财务信息，可以要求股权研究版本。

对于私人公司，此表可能只有收入和员工——两项显示规模和增长的数据点对销售会议已足够。

### 5. 关键关系与生态
帮助代表了解潜在客户的世界并找到切入点。

四个标签列表，每类 3-5 个名称，附带一行注释：

**客户：** 他们卖给谁。帮助了解其市场策略和共享目标客户。
示例："JPMorgan Chase — 客户（企业分析平台部署）"

**供应商 / 技术：** 他们买什么。发现替代或整合机会。
示例："AWS — 供应商（主要云基础设施）；Snowflake — 供应商（数据仓库）"

**合作伙伴：** 他们与谁合作。联合销售角度、共同联系人。
示例："Deloitte — 合作伙伴（企业客户实施）"

**竞争对手：** 他们与谁竞争。定位背景。
示例："Bloomberg — 竞争对手（金融数据终端）"

每条需要一个括号注释，让代表了解**关系意味着什么**，而不仅仅是**是谁**。

如关系数据不可用，注明"关系数据不可用"并继续。

### 6. 近期新闻与发展
3-5 条要点，每条一句话，最新在前。**以最近一期财报为首**——这是销售会议中最重要的近期事件，定下所有其他内容的基调。然后：并购、产品发布、领导层变更、重大赢单。

**在节末包含即将到来的催化剂**，用"**即将到来：**"标签。示例：即将到来的财报日期、待定剥离、产品发布、投资者日、监管里程碑。这些创造了自然的销售接触窗口，通常是联系的最佳理由（"我注意到贵公司Q2有投资者日……"）。从财报电话会评论和并购数据中提取（待定交易）。

### 7. 对话切入点（首先裁剪）
综合上述数据提出的 2-3 条建议谈话要点。**每条切入点必须引用第 3 节中的特定战略重点**——通用问题没用。以问题形式呈现。

**给每条切入点标记建议角色**——例如（CTO）、（CFO）、（CDO）、（数据主管）、（工程VP）、（采购主管）。与 CTO 会面的代表需要与 CFO 不同的开场白。如用户指定了会面对象，仅为该角色生成切入点。

示例：
- *(CTO/CDO)* "您在最近一期财报中提到了[第 3 节中的具体 AI 计划]——这对您的[相关需求]有什么影响？"
- *(CFO/COO)* "鉴于您的[第 3 节中的利润率扩张目标]，您是否在评估能加速实现该目标的工具？"
- *(战略VP/企发)* "我看到[公司]最近收购了[目标]——这是否改变了您对[能力]的策略？"

如用户描述了他们在销售什么产品/服务，将这些切入点与该产品连接。如未描述，保持切入点在潜在客户的战略主题中，但解决方案框架保持通用。

**验证测试：** 最终确定每条对话切入点前，确认它包含至少一个来自概览页的具体数字、日期、产品名或计划名称。一个适用于该行业任何公司的问题不是好的对话切入点。"您如何看待AI？"太泛。而"您的 CEO 提到自 2018 年以来投入了 10 亿美元在 AI 上，20% 的 iLEVEL 自动导入功能已被采纳——这对您的分析师工作流有何改变？"既具体又展现了准备充分。

本节是 Claude 的综合，而非来自工具的原始数据。清晰标记。

## 格式说明
- **这应是最温暖、最可读的概览页类型。** 内容严谨，但格式应像简报备忘录，而非数据终端。
- 正文：使用完整 9pt（size: 18）。不压缩——可读性优先于密度。
- 行间距：正文段落 1.15 倍（略高于默认）。此模板中空白是特色而非浪费。
- 公司概述和战略重点应占约 40-50% 的页面。
- 财务快照：视觉紧凑，次要地位。保持小巧干净。
- 关键关系：考虑两栏布局（客户 + 供应商在左，合作伙伴 + 竞争对手在右）以节省垂直空间同时保持注释可见。
- 对话切入点：使用缩进块级项目符号。这些是文档的回报——应在视觉上突出显示。

### 1. 销售代表只需要 3 个数字——为什么不需要 EBITDA 和 DCF

**想象一个场景**：你是 Snowflake 的销售代表，明天要去见摩根大通的 CIO。你花了 3 小时研究摩根大通的 EBITDA、DCF 估值、信用评级……到了会议上，CIO 说"我不关心我们的 DCF，我关心的是你们的产品能不能帮我的数据团队省时间"。

**这就是为什么销售版 Tearsheet 只保留 3 个数字**：

| 数字 | 为什么重要 | 怎么用 |
|------|-----------|--------|
| **收入** | 客户有多大？有没有钱买你的东西？ | 苹果年收入 $3900 亿——花 $100 万买产品跟玩一样 |
| **增长率** | 客户在扩张还是在收缩？ | 增长 > 20% = 愿意花钱买新工具；负增长 = 在砍预算 |
| **毛利率** | 客户赚不赚钱？ | SaaS 80% 毛利 = 花得起；零售 20% 毛利 = 抠 |

**为什么不需要 EBITDA？** 因为 EBITDA 是给投行家估值用的。销售代表不是来给客户估值的，是来卖东西的。客户买你产品的决策是"我花 100 万能不能省 500 万或多赚 1000 万"——这是 ROI 思维，不是估值思维。

**真实例子**：Snowflake 的 NRR（净收入留存率）133%，意思是老客户第二年自动多花了 33%。这种"客户自然扩展"才是 SaaS 销售的圣杯——不用费力找新客户，老客户自己就多买了。

---

### 2. 近因效应——"最近的事最好聊"

**近因效应（Recency Effect）** 是心理学概念：人们更容易记住最近发生的事。在销售中，这意味着**最近的公司新闻（尤其财报）是最好的谈话切入点**。

**想象一下**：你跟客户见面，有两种开场白：

- **差的开场**："您好，我是 Snowflake 的销售，我们产品很牛，您想了解一下吗？"——客户心想"又一个推销的"
- **好的开场**："我看到贵公司上季度财报中提到正在推进云数据迁移，我们的客户在做类似迁移时平均节省了 30% 的存储成本"——客户心想"这人做了功课，值得聊聊"

**为什么财报是最好的切入点**：
1. **CEO 亲自说的**——不是你编的，有权威性
2. **最近发生的**——客户正在执行中，跟你聊完可能马上行动
3. **公开可查的**——任何人都能看到，不涉及保密信息

**真实例子**：微软 2024 年财报反复强调"AI Copilot 正在改变企业工作流"。如果你卖 AI 相关产品，这就是最好的开场白："我看到萨提亚在最近一次财报中提到 Copilot 已经有 100 万付费用户——你们团队在用 AI 工具吗？"

---

### 3. 话题切入点——"做了功课的开场白"

**话题切入点（Conversation Starter）** 是基于对潜在客户研究设计的初始话题。好的切入点有三个特征：

1. **引用客户的具体信息**（不是泛泛而谈）
2. **连接到你的产品**（不是纯聊天）
3. **以问题形式呈现**（引导客户说话，不是你在推销）

**想象一下**：你要去见 Stripe 的 CTO。你看了他们的财报，发现 CEO 反复提到"全球支付基础设施的扩展"。你的产品是合规自动化工具。切入点可以是：

> "我注意到 Stripe 在最近一次投资者日提到正在进入 15 个新市场——每个新市场都有不同的合规要求，你们团队是怎么管理这些合规流程的？"

**这个切入点为什么有效**：
- 引用了 Stripe 的具体战略（进入 15 个新市场）
- 连接到你的产品（合规自动化）
- 问题化引导（"怎么管理的？"），让客户说话

**对比差的开场**："我们做合规自动化的，您有兴趣吗？"——没有引用任何客户信息，听起来像群发邮件。

**验证测试**：最终确定每条对话切入点前，确认它包含至少一个来自概览页的具体数字、日期、产品名或计划名称。一个适用于该行业任何公司的问题不是好的对话切入点。

---

### 4. 理想客户画像（ICP）——"不是所有公司都是你的客户"

**理想客户画像（Ideal Customer Profile）** 是最适合你产品的客户类型的详细描述。销售团队用 ICP 来优先选择目标客户——不是所有公司都值得花时间。

**想象一下**：你卖的是企业级数据安全产品，年合同价值 $50 万+。你的 ICP 可能是：
- 年收入 > $10 亿（花得起 $50 万）
- 有大量敏感数据（金融、医疗、政府）
- 正在经历数字化转型（需要新工具）
- 有 CISO 或安全团队（有决策者）

**为什么 ICP 重要**：
- **不浪费时间的公司**：年收入 $500 万的创业公司不会花 $50 万买你的产品
- **聚焦高转化率客户**：符合 ICP 的客户转化率可能是 20%，不符合的只有 2%
- **销售管道质量**：4× 法则——要达成 $100 万目标，管道里得有 $400 万机会

**真实例子**：Datadog 的 ICP 是"年收入 > $5 亿、使用云基础设施、有 DevOps 团队的中大型科技公司"。他们不会去追街边奶茶店——虽然奶茶店也需要监控，但合同价值太小。

### 5. 角色化切入点——"跟 CTO 聊技术，跟 CFO 聊 ROI"

**不同的客户角色，关心不同的事**。你跟 CTO 谈技术架构，跟 CFO 谈 ROI，跟 CEO 谈战略——用错话题等于浪费会议。

| 角色 | 关心什么 | 开场聊什么 |
|------|---------|-----------|
| **CTO / CDO** | 技术架构、平台化、AI 能力 | "我们怎么帮您的数据架构升级" |
| **CFO** | ROI、成本、合规 | "我们怎么帮您省 $X 百万" |
| **COO** | 运营效率、流程 | "我们怎么让运营效率提升 30%" |
| **CISO** | 安全、合规 | "我们怎么帮您通过 SOC 2 审计" |
| **战略 VP** | 并购、合作伙伴 | "我们怎么帮下一个并购整合更快" |

**真实例子**：你去见一家公司的 CTO 和 CFO。跟 CTO 说"我们的 AI 引擎用了 Transformer 架构，推理速度比竞品快 3 倍"——CTO 会感兴趣。跟 CFO 说同样的话——CFO 会走神。你应该说"我们的客户平均 6 个月回本，3 年 ROI 超过 300%"。

**所以这意味着什么**：这个模板的"对话切入点"章节会为每条切入点标记建议角色——这样销售代表知道哪句话该对谁说。

> **给小白的一句话**：Sales/BD 版的 Tearsheet 就像一份"客户会前准备清单"——不是让你变成金融专家，而是帮你在 30 分钟里**说对话、找对人、切入对的话题**。记住三个原则：**只看 3 个数字（收入、增长、毛利）、用最近的事做切入点、对不同角色说不同的话**。销售的本质不是"卖东西"，而是"帮人解决问题并赚到回报"——做好功课，让客户觉得你懂他，比任何产品功能介绍都有效。
