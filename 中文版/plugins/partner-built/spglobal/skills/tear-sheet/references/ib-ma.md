# 投资银行 / 并购概览页面

## 用途
供交易背景下的公司简介——潜在目标、收购方或推介书内容。侧重战略定位、分部详情、业务关系和并购活动。估值以交易倍数而非投资论点呈现。

**默认页数：** 1-2 页。投行公司简介通常会用到第二页，尤其在分部数据和并购信息丰富时。

## 查询计划

从以下查询开始。如结果不完整，拆分为更细的查询。

**查询 1 — 简介 + 识别：**
"[公司] 业务描述 总部 成立年份 员工 行业 股权结构 主要股东"
→ 标题、业务概述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询 2 — 分部 + 财务数据：**
"[公司] 收入按业务分部 最近2个财年 AND 年度利润表 收入 毛利润 EBITDA 营业利润 净利润 资本支出 自由现金流 总债务 现金及等价物 最近4个财年"
→ 分部结构（需要2年计算同比增速），财务摘要（拉取4年，展示3年；最早一年仅用于同比增速计算）
→ **立即写入** `/tmp/tear-sheet/financials.csv`
→ **立即写入** `/tmp/tear-sheet/segments.csv`（无分部数据则跳过）

**查询 3 — 估值 + 可比公司：**
如用户提供了具体可比公司，查询每家：
"[同一家公司] 企业价值 EV/收入 EV/EBITDA 收入增长 EBITDA利润率"
否则：
使用竞争对手工具识别 3-5 家上市同行，然后拉取 EV/收入（NTM）、EV/EBITDA（NTM）、收入增长%和EBITDA利润率%。
同时："[公司] 企业价值 市值 EV/收入 EV/EBITDA 估值倍数"
→ 标题（EV/市值）、交易可比公司
→ **立即写入** `/tmp/tear-sheet/valuation.csv`
→ **立即写入** `/tmp/tear-sheet/peer-comps.csv`

**查询 4 — 并购活动：**
"[公司] 收购 出售 已完成交易 最近5年 交易金额"
→ 公司自身交易
→ **立即写入** `/tmp/tear-sheet/ma-activity.csv`

**查询 5 — 可比交易（数据许可）：**
使用查询 1 结果中的特定行业（如"云可观测性软件并购交易"而非"科技并购"）。如用户指定了可比公司，也尝试："[同一家公司] 收购交易倍数。"
→ 可比交易
→ **追加** 可比交易到 `/tmp/tear-sheet/ma-activity.csv`（添加 `type=precedent` 以区分公司自身交易）

**查询 6 — 关系 + 股权结构：**
"[公司] 关键客户 供应商 合作伙伴 业务关系 机构持仓 内部人持股"
→ 业务关系、股权结构（数据许可）
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

## 章节

按优先级排序。详见下方"页数预算与裁剪顺序"。

### 1. 公司标题
含交易相关标识符的关键值块。

| 字段 | 说明 |
|---|---|
| 公司名称 | 大号加粗 |
| 股票代码和交易所 | 如上市；未上市则省略 |
| 总部地点 | 城市、州/国家 |
| 成立 | 年份 |
| 员工 | 约计人数 |
| 市值 / 估值 | 上市为市值；未上市为最近可知估值 |
| 企业价值 | |
| 股权结构 | 上市 / PE背书（注明赞助商）/ 家族 / 其他 |

私人公司：注明"私人公司"；跳过股票代码和交易所。

### 2. 业务概述
**这是推介书段落，而非 CIQ 摘要。** 不要粘贴数据工具的公司描述。以 4-6 句推介书风格重写：
- 描述收入模式（经常性 vs 交易型、订阅 vs 许可证）
- 以规模说明客户群（"服务 80% 的财富 500 强银行"）
- 点名竞争护城河（"专有的 X 数据集"、"唯一提供 Y 的供应商"）
- 指出对买方或卖方重要的增长方向

语气应自信且具体。阅读此段的银行家应立即理解公司的战略定位。

### 3. 收入与分部结构
按业务单元或产品线的收入。

| 分部 | 收入（百万美元） | 占总收入% | 同比增速 |
|---|---|---|---|
| [分部A] | | | |
| [分部B] | | | |
| **总计** | | **100%** | |

**如分部数据不可用：** 用 2-3 句话定性描述收入结构（例如"收入主要来自订阅许可证（约70%）和专业服务（约30%）。"）。空白表看起来像坏了——定性描述仍能传达有用信息。

### 4. 财务摘要（3年 + LTM）
实际财年作为列标题。**如 LTM（最近12个月）数据可用，在最近财年右侧加 LTM 列。** 标记为"LTM [季度结束日期]"（如"LTM Q3 2025"）。银行家依赖 LTM 做估值交叉验证——仅财年视角可能过时 6 个月以上。

| 指标（百万美元） | FY20XX | FY20XX | FY20XX | LTM [日期] |
|---|---|---|---|---|
| 收入 | | | | |
| 收入增长率% | | | | |
| 毛利润 | | | | |
| 毛利率% | | | | |
| EBITDA | | | | |
| EBITDA利润率% | | | | |
| 营业利润 | | | | |
| 净利润 | | | | |
| 资本支出 | | | | |
| 自由现金流 | | | | |
| FCF利润率% | | | | |

在并购背景下，利润率和 FCF 转化率与绝对规模同等重要——买方在评估效率和现金生成潜力。

**资本结构**（财务摘要下方的紧凑子表）：

| 指标 | 值 |
|---|---|
| 总债务 | XX百万美元 |
| 现金及等价物 | XX百万美元 |
| 净债务 | XX百万美元 |
| 净债务 / EBITDA | X.X倍 |
| S&P 信用评级 | XX（如可用） |

从已检索的资产负债表数据提取。这是任何银行家公司简介的标准内容——总债务、杠杆率和信用评级告诉买方融资情况。如信用评级不可用，省略该行。

### 5. 交易可比公司
公司倍数**和经营指标**——银行家一起使用这些，而非孤立倍数。

**有前瞻数据时，前瞻倍数必须显示。** 表中必须包含 NTM 倍数（如工具返回）。仅有回溯倍数而前瞻数据存在是重大缺失——银行家根据前瞻盈利为交易定价。

**同行列是预期的。** 如用户提供了可比公司，每家单独一列。如未提供，使用竞争对手工具识别 3-5 家上市同行，拉取它们的倍数和经营指标并包含在内。未含同行背景的仅公司交易可比表对任何投行用途都不完整。

| 指标 | [公司] | [可比1] | [可比2] | [可比3] | 同行中位数 |
|---|---|---|---|---|---|
| EV/收入（NTM） | | | | | |
| EV/EBITDA（NTM） | | | | | |
| 收入增长率% | | | | | |
| EBITDA利润率% | | | | | |

如尝试同行查询后仅公司自身倍数可用，显示这些并在表下列出所选同行名称。

私人公司跳过此节。

### 6. 并购活动
两部分。**此部分预期有交易金额——** 银行家注意到缺失时会介意。

**硬性规则：如果并购工具返回了交易价值，它必须出现在输出中。** 切勿将已知交易价值降级为"未披露"——这是数据完整性违规。仅在工具确实未返回该交易价值时才使用"未披露"。

**a) 公司自身交易：**

**表中必须包含专用的交易价值列。** 不要将交易价值合并到备注或理由列——银行家扫读数字，专用列才能实现这一点。

| 日期 | 目标 / 出售资产 | 交易价值（百万美元） | 类型 | 理由 |
|---|---|---|---|---|
| | | | 收购 / 剥离 | |

如未找到："过去5年无已披露交易。"

**b) 可比行业交易（数据许可）：**
使用先前识别的特定行业（如"云基础设施软件并购"而非"科技并购"）。如用户指定了可比公司，也寻找涉及这些特定公司的交易。

| 日期 | 目标 | 收购方 | EV（百万美元） | EV/收入 | EV/EBITDA |
|---|---|---|---|---|---|

如不可用："可比交易数据来自数据源不可用。"不要编造可比倍数。

### 7. 关键业务关系
利用 S&P Capital IQ 关系数据。按类型分组：

**客户：** 前 3-5 名知名客户
**供应商：** 关键供应商和技术提供商
**合作伙伴：** 战略联盟、分销商、合资企业
**竞争对手：** 主要竞争对手名称

每条包含关系类型和括号注释说明关系性质（例如"AWS — 云基础设施（主要托管提供商）"、"JPMorgan — 客户（企业分析平台）"）。仅有名称无背景的信息没有用。给此部分留出空间——它战略价值高，而大多数竞争概览缺乏此内容。

### 8. 股权结构（数据许可——工具无返回则省略）
如工具返回机构持仓、内部人持股比例或主要持有人，包含紧凑块：

主要机构持有人（如可用）：按持股%列前 3-5 名。
内部人持股：合计百分比。

PE 背书公司：注明赞助商、收购年份和持股比例。股权结构直接影响交易复杂度——广泛持有的上市公司、创始人控股公司和 PE 背书公司各自需要根本不同的交易方法。

如工具未返回股权数据，完全省略此节而非显示占位符。

**不要包含管理团队表。** 没有 S&P Global 工具返回高管数据——见数据完整性规则第10条。训练数据中的管理层姓名会过时。

## 页数预算与裁剪顺序

**目标：最多 2 页。** 投行简介可充分利用两页，但不得溢到第三页。如内容超过 2 页，按以下顺序裁剪（完全裁剪完每部分再进入下一个）：

1. 股权结构（第8节）— 完全省略
2. 可比行业交易（第6b节）— 保留公司自身并购，裁剪可比交易
3. 业务关系（第7节）— 压缩至前3大竞争对手 + 各前2家供应商/合作伙伴/客户，注释缩减为仅括号内容
4. 资本结构（第4节）— 将净债务/EBITDA 作为单独一行合并到财务摘要中
5. 交易可比公司（第5节）— 减少至3家同行

永远不裁剪：业务概述（2）、收入与分部结构（3）、财务摘要（4 核心表）、并购活动公司交易（6a）。

## 格式说明
- 业务概述和分部结构获得最多视觉权重。
- **并购活动是标志性章节：** 并购表头使用强调色（#2E75B6）填充白色加粗文字——这是唯一使用与标准 #D6E4F0 不同表头颜色的表。应在扫视中引人注目。
- **业务关系：** 空间允许时给足留白。每个关系组之间 6pt，每条附完整注释。但如果超出第三页，按上述裁剪顺序压缩——本章节价值高但也最可压缩。
- 私人公司：标题、业务概述、财务数据和关系构成核心；其他章节可能较稀疏。
- **两页纪律：** 渲染前，预估页数。如果业务关系 + 并购活动合起来超过一页，按裁剪顺序开始裁剪。

### 1. 这个模板的核心逻辑——"这家公司是猎人、猎物还是搅局者"

**想象一个场景**：你是高盛的并购分析师，客户问"Salesforce 在 AI 领域会做什么交易？"你需要快速判断：Salesforce 是潜在收购方（猎人）、被收购目标（猎物）、还是可能干扰别人交易的搅局者？

**这就是 IB/M&A 版 Tearsheet 要回答的核心问题**。它把一家公司放在并购交易的"棋盘"上，告诉你它的位置。

**模板的 8 个章节各有分工**：

| 章节 | 回答什么问题 | 为什么重要 |
|------|-------------|-----------|
| 公司标题 | 这是谁？ | 快速识别 |
| 业务概述 | 它干什么的？ | 判断战略方向 |
| 收入与分部 | 钱从哪来？ | 评估业务质量 |
| 财务摘要 | 赚多少？值多少？ | 估值基础 |
| 交易可比公司 | 同行卖多少？ | 定价参考 |
| 并购活动 | 它买过/卖过什么？ | 判断交易倾向 |
| 业务关系 | 它跟谁有关系？ | 找协同/风险 |
| 股权结构 | 谁控制它？ | 判断交易可行性 |

**所以这意味着什么**：这个模板不是"公司介绍"，而是"交易决策工具"。每个章节的数据都在帮银行家回答"该不该做这笔交易、怎么定价、风险在哪"。

---

### 2. CIM——并购交易的"产品说明书"

**CIM（Confidential Information Memorandum，保密信息备忘录）** 是卖方投行准备的详细文件，包含目标公司的全部信息——财务数据、业务模式、市场分析、客户列表、增长计划等。供潜在买家在签了保密协议（NDA）后阅读。

**想象一下**：你要买一套二手房，中介给你一本 50 页的"房屋详情册"——户型图、装修清单、物业费、周边配套、历史成交价。CIM 就是并购版的"房屋详情册"。

**CIM 里通常有什么**：
- **投资亮点（Investment Highlights）**：为什么这家公司值得买——市场地位、增长潜力、护城河
- **财务数据**：3-5 年的收入、EBITDA、净利润、自由现金流
- **业务描述**：产品、客户、竞争格局
- **增长机会**：未来 3-5 年的扩张计划
- **风险因素**：可能出问题的地方

**真实例子**：2024 年 Synopsys 收购 Ansys，高盛作为卖方顾问准备了 Ansys 的 CIM。潜在买家（如 Synopsys、Cadence）签 NDA 后才能看到这份文件，了解 Ansys 的全部财务和运营细节。

**所以这意味着什么**：CIM 是卖方投行的核心"产品"。好的 CIM 能让买家看完就想出价，差的 CIM 会让买家觉得"这公司没什么特别的"。

---

### 3. 两页纪律——投行文档的"黄金法则"

**投行文档有一条铁律：不超过 2 页**。为什么？

**想象一下**：你是 MD（董事总经理），一天要看 20 份公司简介。如果每份 5 页，你一天要看 100 页——根本看不完。但如果每份只有 2 页，你 1 小时就能扫完所有材料。

**两页纪律的裁剪顺序**（优先级从低到高）：

1. **股权结构** — 最先裁，因为交易可行性通常不取决于股权结构
2. **可比行业交易** — 保留公司自身并购，裁掉行业可比
3. **业务关系** — 压缩注释，但保留核心关系
4. **资本结构** — 合并到财务摘要
5. **交易可比公司** — 减少同行数量

**永远不裁剪**：业务概述、收入与分部、财务摘要核心表、并购活动（公司自身交易）

**真实例子**：高盛给客户做并购推介书时，第一页永远是公司标题+业务概述+财务摘要，第二页是估值+并购活动。如果空间不够，宁可砍掉股权结构，也不动财务数据。

---

### 4. 并购活动——这个模板的"标志性章节"

**为什么并购活动表要用不同颜色高亮？** 因为这是银行家扫读时最先看的内容。

**想象一下**：你看到 Salesforce 过去 5 年收购了 Slack（277 亿美元）、MuleSoft（65 亿美元）、Tableau（157 亿美元）——你立刻知道这是一家"通过收购增长"的公司。反过来，如果一家公司 5 年没有任何交易，它可能是"有机增长型"或者"还没开始收购"。

**并购活动表的两个部分**：

| 部分 | 内容 | 用途 |
|------|------|------|
| **公司自身交易** | 目标公司买过/卖过什么 | 判断交易倾向和整合能力 |
| **可比行业交易** | 同行业其他公司的并购 | 定价参考（Precedent Transactions） |

**硬性规则**：如果工具返回了交易价值，它**必须**出现在输出中。把已知交易价值写成"未披露"是数据完整性违规。

---

### 5. LTM vs FY——"新鲜数据"和"官方数据"的区别

**LTM（Last Twelve Months，最近 12 个月）** 和 **FY（Fiscal Year，财年）** 的区别对银行家至关重要。

**想象一下**：现在是 2026 年 6 月。苹果 FY2025（截至 2025 年 9 月）的收入是 3910 亿美元。但苹果 FY2026 Q1-Q3（截至 2026 年 6 月）的收入可能已经 3200 亿美元了。如果你只看 FY2025 数据，你用的是 9 个月前的旧数据。

**LTM = 最近 4 个季度的合计**，比最近一个完整财年更新鲜。银行家依赖 LTM 做估值交叉验证——仅财年视角可能过时 6 个月以上。

**真实例子**：你在 2026 年 6 月给一家公司做 Comps 估值。如果只看 FY2025 的 EBITDA，可能低估了公司价值（因为公司还在增长）。加上 LTM Q1 2026 的数据，估值会更准确。

**所以这意味着什么**：财务摘要表中，LTM 列必须出现在最近财年右侧——这是投行标准格式，不是可选项。

> **给小白的一句话**：IB/M&A 版的 Tearsheet 就像一份"战场情报简报"——不是给你讲这家公司的全部故事，而是帮你在 30 秒内判断"这家公司在交易棋盘上的位置"。CIM 是详细的"产品说明书"，两页纪律是"只说最重要的"，并购活动是"它过去的行为模式"。记住：**投行文档的价值不在于全面，而在于精准**——2 页纸让 MD 快速决策，比 20 页没人看的报告有用 100 倍。
