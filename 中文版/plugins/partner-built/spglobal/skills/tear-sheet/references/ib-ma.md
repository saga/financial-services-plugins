# 投资银行 / 并购概览页面

## 用途
供交易背景下的公司简介——潜在目标、收购方或推介书内容。侧重战略定位、分部详情、业务关系和并购活动。估值以交易倍数而非投资论点呈现。

**默认页数：** 1-2 页。投行公司简介通常会用到第二页，尤其在分部数据和并购信息丰富时。

## 查询计划

从以下查询开始。如结果不完整，拆分为更细的查询。

**查询 1 — 简介 + 识别：**
"[公司] 业务描述 总部 成立年份 员工 行业 股权结构 主要股东"
→ 标题、业务概述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询 2 — 分部 + 财务数据：**
"[公司] 收入按业务分部 最近2个财年 AND 年度利润表 收入 毛利润 EBITDA 营业利润 净利润 资本支出 自由现金流 总债务 现金及等价物 最近4个财年"
→ 分部结构（需要2年计算同比增速），财务摘要（拉取4年，展示3年；最早一年仅用于同比增速计算）
→ **立即写入** `/tmp/tear-sheet/financials.csv`
→ **立即写入** `/tmp/tear-sheet/segments.csv`（无分部数据则跳过）

**查询 3 — 估值 + 可比公司：**
如用户提供了具体可比公司，查询每家：
"[同一家公司] 企业价值 EV/收入 EV/EBITDA 收入增长 EBITDA利润率"
否则：
使用竞争对手工具识别 3-5 家上市同行，然后拉取 EV/收入（NTM）、EV/EBITDA（NTM）、收入增长%和EBITDA利润率%。
同时："[公司] 企业价值 市值 EV/收入 EV/EBITDA 估值倍数"
→ 标题（EV/市值）、交易可比公司
→ **立即写入** `/tmp/tear-sheet/valuation.csv`
→ **立即写入** `/tmp/tear-sheet/peer-comps.csv`

**查询 4 — 并购活动：**
"[公司] 收购 出售 已完成交易 最近5年 交易金额"
→ 公司自身交易
→ **立即写入** `/tmp/tear-sheet/ma-activity.csv`

**查询 5 — 可比交易（数据许可）：**
使用查询 1 结果中的特定行业（如"云可观测性软件并购交易"而非"科技并购"）。如用户指定了可比公司，也尝试："[同一家公司] 收购交易倍数。"
→ 可比交易
→ **追加** 可比交易到 `/tmp/tear-sheet/ma-activity.csv`（添加 `type=precedent` 以区分公司自身交易）

**查询 6 — 关系 + 股权结构：**
"[公司] 关键客户 供应商 合作伙伴 业务关系 机构持仓 内部人持股"
→ 业务关系、股权结构（数据许可）
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

## 章节

按优先级排序。详见下方"页数预算与裁剪顺序"。

### 1. 公司标题
含交易相关标识符的关键值块。

| 字段 | 说明 |
|---|---|
| 公司名称 | 大号加粗 |
| 股票代码和交易所 | 如上市；未上市则省略 |
| 总部地点 | 城市、州/国家 |
| 成立 | 年份 |
| 员工 | 约计人数 |
| 市值 / 估值 | 上市为市值；未上市为最近可知估值 |
| 企业价值 | |
| 股权结构 | 上市 / PE背书（注明赞助商）/ 家族 / 其他 |

私人公司：注明"私人公司"；跳过股票代码和交易所。

### 2. 业务概述
**这是推介书段落，而非 CIQ 摘要。** 不要粘贴数据工具的公司描述。以 4-6 句推介书风格重写：
- 描述收入模式（经常性 vs 交易型、订阅 vs 许可证）
- 以规模说明客户群（"服务 80% 的财富 500 强银行"）
- 点名竞争护城河（"专有的 X 数据集"、"唯一提供 Y 的供应商"）
- 指出对买方或卖方重要的增长方向

语气应自信且具体。阅读此段的银行家应立即理解公司的战略定位。

### 3. 收入与分部结构
按业务单元或产品线的收入。

| 分部 | 收入（百万美元） | 占总收入% | 同比增速 |
|---|---|---|---|
| [分部A] | | | |
| [分部B] | | | |
| **总计** | | **100%** | |

**如分部数据不可用：** 用 2-3 句话定性描述收入结构（例如"收入主要来自订阅许可证（约70%）和专业服务（约30%）。"）。空白表看起来像坏了——定性描述仍能传达有用信息。

### 4. 财务摘要（3年 + LTM）
实际财年作为列标题。**如 LTM（最近12个月）数据可用，在最近财年右侧加 LTM 列。** 标记为"LTM [季度结束日期]"（如"LTM Q3 2025"）。银行家依赖 LTM 做估值交叉验证——仅财年视角可能过时 6 个月以上。

| 指标（百万美元） | FY20XX | FY20XX | FY20XX | LTM [日期] |
|---|---|---|---|---|
| 收入 | | | | |
| 收入增长率% | | | | |
| 毛利润 | | | | |
| 毛利率% | | | | |
| EBITDA | | | | |
| EBITDA利润率% | | | | |
| 营业利润 | | | | |
| 净利润 | | | | |
| 资本支出 | | | | |
| 自由现金流 | | | | |
| FCF利润率% | | | | |

在并购背景下，利润率和 FCF 转化率与绝对规模同等重要——买方在评估效率和现金生成潜力。

**资本结构**（财务摘要下方的紧凑子表）：

| 指标 | 值 |
|---|---|
| 总债务 | XX百万美元 |
| 现金及等价物 | XX百万美元 |
| 净债务 | XX百万美元 |
| 净债务 / EBITDA | X.X倍 |
| S&P 信用评级 | XX（如可用） |

从已检索的资产负债表数据提取。这是任何银行家公司简介的标准内容——总债务、杠杆率和信用评级告诉买方融资情况。如信用评级不可用，省略该行。

### 5. 交易可比公司
公司倍数**和经营指标**——银行家一起使用这些，而非孤立倍数。

**有前瞻数据时，前瞻倍数必须显示。** 表中必须包含 NTM 倍数（如工具返回）。仅有回溯倍数而前瞻数据存在是重大缺失——银行家根据前瞻盈利为交易定价。

**同行列是预期的。** 如用户提供了可比公司，每家单独一列。如未提供，使用竞争对手工具识别 3-5 家上市同行，拉取它们的倍数和经营指标并包含在内。未含同行背景的仅公司交易可比表对任何投行用途都不完整。

| 指标 | [公司] | [可比1] | [可比2] | [可比3] | 同行中位数 |
|---|---|---|---|---|---|
| EV/收入（NTM） | | | | | |
| EV/EBITDA（NTM） | | | | | |
| 收入增长率% | | | | | |
| EBITDA利润率% | | | | | |

如尝试同行查询后仅公司自身倍数可用，显示这些并在表下列出所选同行名称。

私人公司跳过此节。

### 6. 并购活动
两部分。**此部分预期有交易金额——** 银行家注意到缺失时会介意。

**硬性规则：如果并购工具返回了交易价值，它必须出现在输出中。** 切勿将已知交易价值降级为"未披露"——这是数据完整性违规。仅在工具确实未返回该交易价值时才使用"未披露"。

**a) 公司自身交易：**

**表中必须包含专用的交易价值列。** 不要将交易价值合并到备注或理由列——银行家扫读数字，专用列才能实现这一点。

| 日期 | 目标 / 出售资产 | 交易价值（百万美元） | 类型 | 理由 |
|---|---|---|---|---|
| | | | 收购 / 剥离 | |

如未找到："过去5年无已披露交易。"

**b) 可比行业交易（数据许可）：**
使用先前识别的特定行业（如"云基础设施软件并购"而非"科技并购"）。如用户指定了可比公司，也寻找涉及这些特定公司的交易。

| 日期 | 目标 | 收购方 | EV（百万美元） | EV/收入 | EV/EBITDA |
|---|---|---|---|---|---|

如不可用："可比交易数据来自数据源不可用。"不要编造可比倍数。

### 7. 关键业务关系
利用 S&P Capital IQ 关系数据。按类型分组：

**客户：** 前 3-5 名知名客户
**供应商：** 关键供应商和技术提供商
**合作伙伴：** 战略联盟、分销商、合资企业
**竞争对手：** 主要竞争对手名称

每条包含关系类型和括号注释说明关系性质（例如"AWS — 云基础设施（主要托管提供商）"、"JPMorgan — 客户（企业分析平台）"）。仅有名称无背景的信息没有用。给此部分留出空间——它战略价值高，而大多数竞争概览缺乏此内容。

### 8. 股权结构（数据许可——工具无返回则省略）
如工具返回机构持仓、内部人持股比例或主要持有人，包含紧凑块：

主要机构持有人（如可用）：按持股%列前 3-5 名。
内部人持股：合计百分比。

PE 背书公司：注明赞助商、收购年份和持股比例。股权结构直接影响交易复杂度——广泛持有的上市公司、创始人控股公司和 PE 背书公司各自需要根本不同的交易方法。

如工具未返回股权数据，完全省略此节而非显示占位符。

**不要包含管理团队表。** 没有 S&P Global 工具返回高管数据——见数据完整性规则第10条。训练数据中的管理层姓名会过时。

## 页数预算与裁剪顺序

**目标：最多 2 页。** 投行简介可充分利用两页，但不得溢到第三页。如内容超过 2 页，按以下顺序裁剪（完全裁剪完每部分再进入下一个）：

1. 股权结构（第8节）— 完全省略
2. 可比行业交易（第6b节）— 保留公司自身并购，裁剪可比交易
3. 业务关系（第7节）— 压缩至前3大竞争对手 + 各前2家供应商/合作伙伴/客户，注释缩减为仅括号内容
4. 资本结构（第4节）— 将净债务/EBITDA 作为单独一行合并到财务摘要中
5. 交易可比公司（第5节）— 减少至3家同行

永远不裁剪：业务概述（2）、收入与分部结构（3）、财务摘要（4 核心表）、并购活动公司交易（6a）。

## 格式说明
- 业务概述和分部结构获得最多视觉权重。
- **并购活动是标志性章节：** 并购表头使用强调色（#2E75B6）填充白色加粗文字——这是唯一使用与标准 #D6E4F0 不同表头颜色的表。应在扫视中引人注目。
- **业务关系：** 空间允许时给足留白。每个关系组之间 6pt，每条附完整注释。但如果超出第三页，按上述裁剪顺序压缩——本章节价值高但也最可压缩。
- 私人公司：标题、业务概述、财务数据和关系构成核心；其他章节可能较稀疏。
- **两页纪律：** 渲染前，预估页数。如果业务关系 + 并购活动合起来超过一页，按裁剪顺序开始裁剪。

## 📖 本篇金融知识小贴士

**CIM（保密信息备忘录，Confidential Information Memorandum）**：卖方准备的详细文件，包含公司全部信息——财务数据、业务模式、市场分析等。供潜在买家尽调。类似"卖房时给买家看的全面检测报告"。

**推介书（Pitchbook）**：投行为争取业务或推销交易而制作的展示文档。通常有多页幻灯片，讲述一个叙事性的"故事"。

**可比交易（Precedent Transactions）**：历史上类似行业的并购交易及其估值倍数。投行用它们来估算目标公司当前应该卖多少钱。

**LTM（最近12个月，Last Twelve Months）**：最近4个季度的合计数据。比单个财年数据更新鲜——比如现在是2026年5月，FY2025数据可能已过时，但 LTM Q1 2026 是新鲜的。

**信用评级（Credit Rating）**：评级机构（标普S&P、穆迪、惠誉）对债务人偿债能力的评估。投资级（BBB-/Baa3及以上）vs 高收益/垃圾级。

**剥离（Divestiture）**：公司出售某个业务部门或资产。跟收购相反。

**交易来源与用途（Sources & Uses）**：收购钱从哪里来（债务+股权）和花到哪里去（支付股东+偿还债务+交易费用）。所有并购交易的财务蓝图。

## Appendix: 金融背景知识

你手上这页纸是一份"投资银行 / 并购概览页面"的模板，是给交易撮合者（IB 银行家、企业发展部、并购基金）准备的"客户画像"。它的核心是回答一个问题：**这家公司是潜在的收购方、潜在的目标、还是潜在的干扰者？** 这份附录就来帮你把这件事讲明白——讲清楚投行到底在干嘛、并购在讲什么、那些玄乎的术语背后是什么意思。

### 1. 先搞清楚"投资银行"是个什么东西

#### 1.1 投行的本质

**想象一个场景**：你的朋友开了家小公司，最近生意好、想上市，但完全不知道 IPO 是怎么回事。又有一家大公司想吃掉你的公司，但不知道该出多少钱。**这时候需要"中间人"——帮你融资、帮你卖/买公司**。

这就是投资银行（Investment Bank，简称 IB）。它不做普通人的存贷款业务（那是商业银行），而是**给大公司、上市公司、政府做"金融中间人"**。

#### 1.2 投行的两大核心业务

**用大白话讲就两件事**：

1. **帮人融资**（FICC、ECM、DCM）：让企业拿到钱
   - 公司要 IPO（首次公开上市），投行帮忙卖股票给公众。苹果 1980 年 IPO，是摩根士丹利帮的忙。
   - 公司要发债借钱，投行帮忙卖给机构投资者。微软 2024 年发 60 亿美元绿色债券，摩根大通和高盛承销。
   - 公司要给员工配股、要做可转债，投行设计产品+卖出去。

2. **帮人买卖公司**（M&A，企业并购）：让企业做大或退出
   - 卖方：投行帮公司找买家、谈价格、做尽调。微软想卖诺基亚手机业务时，高盛给建议。
   - 买方：投行帮公司找目标、估值、融资。微软 2016 年花 260 亿美元收购 LinkedIn，是摩根士丹利顾问。
   - 反向/防御：投行帮公司防御恶意收购，比如找"白衣骑士"。

#### 1.3 全球顶级投行

**Bulge Bracket（顶级巨头）**：
- **高盛（Goldman Sachs）**：并购之王。2010-2024 年并购顾问连续多年第一。
- **摩根士丹利（Morgan Stanley）**：并购顾问、财富管理见长。
- **JPMorgan（JPM）**：全方位最强。2023 年投行收入第一。
- **美银证券（BofA Securities）**：M&A 全美第二。
- **巴克莱、花旗、瑞银、德意志银行**：欧洲系强项。
- **精品投行（精品 Boutique）**：
  - **Lazard**：专注并购、重组。
  - **Evercore**：精品并购之王，擅长复杂交易。
  - **Centerview**：美国前财长 Rubin 创立，专注大型并购。
  - **Allen & Company**：科技/媒体并购专家，撮合过微软-领英、亚马逊-全食。
  - **Rothschild & Co**：欧洲历史最悠久的并购精品行。

**投行是金字塔结构**：毕业生挤破头进高盛/摩根斯坦利，干 2-3 年分析师（Analyst），天天凌晨 2 点回家；熬 3 年转助理（Associate），开始带团队；再熬到副总裁（VP）、董事总经理（MD/MD Director）。MD 是金字塔顶——一个 MD 一年能赚 500 万到 5000 万美元（看业绩），分析师只有 15-20 万。

#### 1.4 投行的"产品"

- **推介书（Pitchbook）**：投行给客户"打广告"用的演示文稿，目的是拿下项目。
- **CIM（Confidential Information Memorandum，保密信息备忘录）**：卖方项目里，目标公司给潜在买家看"完整介绍"。
- **估值模型（Valuation Model）**：Excel 模型，预测未来 5 年、算 DCF、算 Comps、画 football field。
- **尽职调查报告（Due Diligence Report）**：财务、法律、商业尽调后的发现。
- **定价分析（Pricing Analysis）**：上市定价或发债利率的论证。

### 2. 并购——"买公司到底是怎么回事"

#### 2.1 并购本质

**想象一个场景**：你开了家奶茶店，年赚 100 万。市场上一家大品牌说"我出 1000 万买你"。问题来了——凭什么 1000 万？为什么不是 500 万或 2000 万？这就是并购要解决的问题——**给一家公司算个"合理的"收购价，然后让买卖双方都点头**。

**真实的并购案例**：
- **2024 年 12 月**：**Synopsys 宣布以 350 亿美元收购 Ansys**——半导体设计软件巨头吃掉工程仿真软件。这是 2024 年科技业最大并购。
- **2024 年 7 月**：**Walmart 接近收购 Vizio**——零售巨头花 23 亿美元买智能电视品牌。
- **2023 年 12 月**：**Broadcom 完成 610 亿美元收购 VMware**——芯片巨头进军云计算基础设施。
- **2022 年 1 月**：**Microsoft 宣布 687 亿美元收购动视暴雪**（2023 年 10 月完成）——科技史上最大游戏并购，反垄断阻力巨大。
- **2016 年 9 月**：**Microsoft 260 亿美元收购 LinkedIn**——互联网历史上最经典的"互补型"并购。

#### 2.2 并购的 4 大类型

**用相亲来比喻**：

1. **横向并购（Horizontal M&A）**：**两个竞争对手结婚**。比如摩根大通收购第一银行（2004 年，580 亿美元）。优势：消灭竞争对手、合并重复部门、规模效应。劣势：反垄断审查最严。
2. **纵向并购（Vertical M&A）**：**上下游结婚**。比如苹果收购 Intel 的 modem 业务、亚马逊收购 Whole Foods。优势：控制供应链、减少依赖。劣势：可能被反垄断视为"垄断"。
3. **混合并购（Conglomerate M&A）**：**完全不同的两家人结婚**。比如迪士尼（电影）收购 ABC（电视）。优势：分散风险。劣势：管理不协同，1980 年代一度臭名昭著。
4. **平台型/生态型并购（Platform / Ecosystem M&A）**：**大公司买小公司"加料"**。比如 Salesforce 每年花几十亿买小 SaaS 公司整合到平台。Meta 收购 Instagram、WhatsApp 是经典案例。

#### 2.3 并购的流程（教科书版 vs 现实版）

**教科书说**：接触 → 签约 → 监管 → 交割（4-12 个月）

**现实里更长、更乱**：

```
1. 接触（2-3 月）：卖方找投行"打包装"，做 CIM
2. 一级市场（2-4 月）：卖方通过投行接触 5-15 个潜在买家
3. 签 NDA（1-2 周）：双方签保密协议，潜在买家才能看 CIM
4. 一级报价（2-3 周）：买家出"非约束性报价"
5. 缩小范围（2-4 周）：卖方挑 2-4 家进入第二轮
6. 尽调（4-8 周）：买家带会计师、律师进入数据室
7. 最终报价（1-2 周）：出"约束性报价"+SPA 草稿
8. SPA 谈判（2-4 周）：卖方股东同意价、分手费、赔偿
9. 监管审批（3-12 月）：反垄断、外国投资审查
10. 交割（Close）：钱交割、股票过户
```

**真实案例**：微软-动视暴雪从签约到完成花了 21 个月（2022 年 1 月宣布，2023 年 10 月完成），因为英国 CMA、欧盟 EC、美国 FTC 都审查了。

#### 2.4 并购的两种"打法"

**友好收购（Friendly Deal）**：
- 双方董事会都同意
- 共同公告、共同应对股东和监管
- 比如 Synopsys-Ansys，Synopsys CEO 说"我们一起做大蛋糕"

**恶意收购（Hostile Deal）**：
- 收购方绕过被收购方董事会，直接向股东报价
- 通常有"要约收购（Tender Offer）"+ 委托书之争（Proxy Fight）
- 经典案例：
  - **Peltz vs Disney 2024**：Peltz 反对迪士尼管理层，争夺董事会席位，最终失败。
  - **Ackman vs ADP 2017**：激进投资者 Ackman 想进董事会，被 ADP 拒绝。
  - **Microsoft vs Yahoo 2008**：微软想 446 亿美元恶意收购 Yahoo，被杨致远拒。
  - **InBev vs Anheuser-Busch 2008**：InBev 用 520 亿美元恶意收购百威，最后成功。

**白衣骑士（White Knight）**：被收购方找"更友好的买家"来对抗恶意收购方。比如 2007 年 Tribune 破产时，Chicago Tribune 想卖给 Sam Zell，后来变成复杂债务重组。

### 3. 估值——"公司到底值多少钱"

#### 3.1 估值三大方法

**想象你朋友开奶茶店**，现在他开价 100 万。但你说"等等，让我算算"。

**方法一：DCF（Discounted Cash Flow，现金流折现）**
你朋友说未来 5 年每年赚 20 万、25 万、30 万、35 万、40 万，第 6 年开始按 3% 永远增长。

DCF 思路：把这 5 年的钱 + 之后的"永续价值"打个折现到今天。假设折现率 10%，那今天值多少钱？

- 第 1 年的 20 万折现到现在 = 20 万 ÷ 1.10 = 18.2 万
- 第 2 年的 25 万折现到现在 = 25 万 ÷ 1.10² = 20.7 万
- ...
- 永续价值 = 第 6 年现金 ÷ (折现率 - 增长率) = 2.16 亿 × 折现

加起来减净债务 = 公司"内在价值"。

**真实公司举例**：高盛 2024 年给英伟达算的 DCF 目标价范围大概是 90-200 美元（不同假设不同结果）。

**方法二：相对估值（Comps）**
- 找同行可比公司
- 算他们的 P/E、EV/EBITDA 中位数
- 套到目标公司上

比如你要给 Anthropic 估值，参考 OpenAI、Cohere、行业中位数 EV/收入 25 倍，Anthropic 预计明年收入 10 亿，那 EV ≈ 250 亿。

**方法三：先例交易（Precedent Transactions）**
看过去类似公司被收购时卖了多少钱。反推目标公司值多少。

比如 2024 年 AI 行业并购：
- 微软"收编" Inflection：~65 亿美元
- 亚马逊收购 Adept：~9.7 亿美元
- Google 收购 Character.AI 部分团队：~25 亿美元
这些交易的 EV/收入倍数可以作为 AI 公司估值的参考。

#### 3.2 Football Field（估值带）

**投行最爱画的一张图**：把同一家公司用 5-6 种方法算出来的估值画成柱状图，组合在一起像橄榄球场。目的：**给客户一个"我估值很全面"的视觉印象**。

举例：某公司估值范围 $80-$150/股
- 52 周交易范围：$85-$130
- 分析师目标价：$95-$140
- DCF：$90-$130
- Comps：$80-$120
- 先例交易：$100-$150
- LBO：$85-$125

**客户的决策**：看这个 range，在跟卖方谈判时心里有数。

#### 3.3 估值倍数

| 倍数 | 怎么算 | 适合谁 |
|------|--------|--------|
| **P/E** | 股价 ÷ EPS | 盈利稳定的公司 |
| **EV/EBITDA** | 企业价值 ÷ EBITDA | 资本结构差异大的公司（重资产 vs 轻资产） |
| **EV/收入** | EV ÷ 收入 | 还没盈利的 SaaS、生物科技 |
| **P/FCF** | 股价 ÷ 每股自由现金流 | 现金流好的公司 |
| **Dividend Yield** | 每股股息 ÷ 股价 | 稳定派息公司 |

**EBITDA** 看着唬人，其实就是"扣掉利息、税金、机器设备折旧后，公司的进账"。比如你开奶茶店，扣掉牛奶、茶叶、糖、房租、人工但不算贷款利息和装修折旧，就是 EBITDA。

**EV（企业价值）** = 市值 + 净债务。比单纯市值更全面——"买下整家公司要花多少钱"。

**真实倍数（2024 年水平）**：
- 苹果：P/E 28-35 倍、EV/EBITDA 22-25 倍
- 微软：P/E 32-35 倍、EV/EBITDA 22-25 倍
- 英伟达：P/E 50-70 倍（按 LTM）、EV/EBITDA 40-50 倍
- Snowflake：P/E 不适用（亏损），EV/收入 15-20 倍
- 银行：P/E 8-12 倍

### 4. 杠杆收购（LBO）——"空手套白狼"的艺术

#### 4.1 LBO 是什么

**想象一个场景**：你想开奶茶店，需要 100 万。但你只有 30 万现金。你跟银行说"借我 70 万，我用奶茶店未来现金流还"。如果奶茶店每月赚 5 万，扣掉给银行的 2 万，你还能赚 3 万。**这就是 LBO 的核心**。

**专业定义**：LBO（Leveraged Buyout，杠杆收购）= 用大量借债买下一家稳定现金流的公司。

**LBO 三方角色**：
- **PE 基金（KKR、Bain Capital、Blackstone）**：出资 30-50%，找银行借 50-70%
- **银行（JPM、高盛）**：借钱给 PE，通常 6-7% 利率
- **目标公司**：用自己的现金流还债

**真实案例**：
- **1988 年 KKR 收购 RJR Nabisco**（250 亿美元）：史上最经典 LBO，纪录片《门口的野蛮人》讲的就是这个。
- **2007 年 Blackstone 收购 Hilton**（260 亿美元）：赚 140 亿美元退出。
- **2024 年 KKR 收购 Telecom Italia NetCo**（~220 亿欧元）：欧洲最大 LBO。
- **2024 年 博通收购 VMware**（610 亿美元）：高杠杆交易。

#### 4.2 LBO 数学

**核心问题**：PE 投 100 块 + 借 300 块买下公司 → 5 年后公司值 800 块 → PE 卖掉 → 还完贷款剩多少？

```
投入本金：100 块
借钱：300 块
买入：400 块

5 年后卖出：800 块
还贷款 + 利息：450 块（算上利息）
剩：350 块

PE 原始 100 块，剩 350 块 = 3.5 倍
年化 IRR（内部收益率）= 28%
```

**这就是 PE 圈"2-3-4-5 法则"**：
- 2 年内还 25% 债务
- 3 年内 EBITDA 翻倍
- 4 年内 IRR 30%
- 5 年后退出

#### 4.3 为什么用债务

**债务有税盾效应**。借钱利息可以抵税，等于政府补贴你借钱。比如公司赚 100 万，付 30 万利息，剩下 70 万才交税。

税务 + 杠杆 → 让 PE 投更少的钱 → 回报更高。这就是为什么 LBO 这么流行。

### 5. IPO——"公司第一次卖股票给公众"

#### 5.1 IPO 是什么

**想象一个场景**：你朋友开了家奶茶店 10 年，现在全国开了 500 家。他想拿一部分股份卖给公众，套现 50 亿。这样他可以拿这钱继续开分店、做营销、还债。

**这就是 IPO（Initial Public Offering，首次公开上市）**。

#### 5.2 真实案例

- **2024 年 10 月**：**ServiceTitan IPO**，市值 88 亿美元，做 SaaS 给承包商用。
- **2024 年 12 月**：**Anterix** 不太成功，重组再上市。
- **2023 年 9 月**：**ARM Holdings 在纳斯达克 IPO**，市值 545 亿美元——软银旗下芯片设计公司。
- **2021 年 12 月**：**Corebridge Financial** 上市，剥离自 AIG。
- **2019 年**：**Uber IPO**（市值 800 亿美元）、**Lyft IPO**（市值 240 亿美元）——共享经济上市潮。
- **2012 年**：**Facebook IPO**（市值 1000 亿美元，初期破发，后来翻了 6 倍）——最经典的"大公司上市案"。
- **1980 年**：**Apple IPO**——乔布斯时代标志性事件。

#### 5.3 IPO 流程

1. **选承销商（2-3 月）**：挑 1-2 家投行做主承销
2. **S-1 文件准备（3-6 月）**：法律文件，详细披露公司
3. **路演（2-4 周）**：CEO、CFO 全球飞，100+ 场投资人会议
4. **定价（1-2 天）**：根据需求定发行价
5. **上市（1 天）**：第一天交易

**首日表现**：
- **2019 年 Lyft**：发行 $72，第一天跌 8%
- **2019 年 Uber**：发行 $45，第一天跌 7%
- **2012 年 Facebook**：发行 $38，第一天破发
- **2019 年 CrowdStrike**：发行 $34，第一天涨 70%
- **2019 年 Zoom**：发行 $36，第一天涨 72%

#### 5.4 IPO 后会发生什么

- **锁定期（Lock-up）**：通常 180 天，原始股东不能卖
- **解禁日（Lock-up Expiry）**：6 个月后，原始股东可以卖——通常股价会跌
- **解禁后表现**：
  - 2018 年**Snap**：解禁后跌 15%
  - 2019 年**Lyft**：解禁后跌 5%
  - 2019 年**Beyond Meat**：解禁后涨 200%（市场过热）

**为什么**：6 个月里内部人憋着不卖，6 个月后集中卖，供给暴增。

### 6. 杠杆融资——"公司怎么借钱"

#### 6.1 债券 vs 股票

**想象你开奶茶店，需要扩张**：
- **股票融资**：让新股东进来，分未来利润。比如给新股东 30% 股份，换 100 万。
- **债务融资**：跟银行/投资人借 100 万，未来用利润还，**不用分股份**。

**债务的好处**：
- 不稀释股权（创始人保持控制权）
- 利息抵税（省税）
- 财务杠杆（赚得多分得多）

**债务的坏处**：
- 必须还（破产风险）
- 限制公司灵活性（负债率高的公司很难再借）

#### 6.2 债券类型

- **投资级债券（Investment Grade）**：BBB-（标普）/ Baa3（穆迪）及以上。微软、苹果、迪士尼的债都是投资级。利率低（3-5%）。
- **高收益债券 / 垃圾债（High Yield / Junk）**：BB+ 及以下。利息高（7-12%），但违约风险高。特斯拉早期发过垃圾债（2013 年，利率 7%）。
- **可转债（Convertible Bond）**：可以转成股票。利息低（1-3%），但股价涨到一定程度可以转股。特斯拉、Snowflake 都发过可转债。
- **永续债（Perpetual Bond）**：不还本金，只付利息。银行常用。

#### 6.3 信用评级

**三大评级机构**：
- **标普 S&P**：常用
- **穆迪 Moody's**：银行用得多
- **惠誉 Fitch**：欧洲用得多

**评级分级**（从高到低）：
- **AAA**：理论上最安全，但 2008 年金融危机后，AAA 评级不再那么神圣
- **AA / A**：高质量（苹果、微软、迪士尼都是 AA/A）
- **BBB-**：投资级最低
- **BB+ / BB / BB-**：垃圾级
- **CCC / CC / C**：高风险
- **D**：已违约

**真实案例**：
- **2008 年雷曼**：评级被下调到 D 后几天破产
- **2011 年标普下调美国主权评级**：从 AAA 到 AA+，美国历史上第一次

### 7. 信用分析——"借钱给人怎么判断他还不还得起"

#### 7.1 信用分析的本质

**想象场景**：你朋友想跟你借 100 万，3 年后还。你怎么判断他还得起？

- 看他的收入（够不够还）
- 看他的资产（还不起了能不能卖资产还）
- 看他的负债（是不是已经欠了一屁股债）
- 看他的还款历史（之前借的钱还了吗）

**信用分析就是这 4 件事的"专业化版本"**。

#### 7.2 三大信用分析机构

- **S&P Global Ratings（标普）**：全球最大评级机构
- **Moody's（穆迪）**：银行业最看重
- **Fitch（惠誉）**：欧洲用得多

#### 7.3 信用分析关键指标

- **杠杆率（Debt/EBITDA）**：债务 ÷ 利润。> 4 倍就要小心。LBO 收购后经常 6-7 倍。
- **利息覆盖倍数（Interest Coverage）**：EBITDA ÷ 利息。> 3 倍健康。
- **现金流量（FCF）**：有 FCF 才能还债。
- **资产质量**：有优质资产抵押的债更安全。

#### 7.4 真实案例

**AT&T 案例**：
- 2018 年 AT&T 850 亿美元收购时代华纳（Time Warner）
- 2015 年 AT&T 490 亿美元收购 DirecTV
- 2022 年剥离 WarnerMedia 给 Discovery
- 信用评级：A- → BBB（投资级最低）
- 杠杆率：峰值 3.5 倍
- 教训：过大的并购会让信用评级下降，融资成本上升

### 8. 监管与反垄断——"这交易政府让不让做"

#### 8.1 为什么需要反垄断

**想象场景**：镇上只有一家超市 + 一家药店 + 一家加油站，被一家公司全买了。现在全镇人都只能从他那里买东西。**这就是垄断**——他可以任意涨价。

反垄断就是防止这种事。

#### 8.2 反垄断四阶段

**美国（DOJ + FTC）**：
- **HSR Act（Hart-Scott-Rodino）申报**：超过一定规模的交易必须申报（2024 年是 1.194 亿美元）
- **Second Request 二次调查**：如果担心竞争，监管要进一步调查
- **谈判/和解**：双方可以谈条件（剥离业务、开放 API）
- **诉讼**：谈不拢就走法院

**欧盟（DG COMP）**：
- 比美国更严
- Phase 1 调查（25 个工作日）：简单案件
- Phase 2 调查（90 个工作日）：复杂案件
- 谷歌被罚过 4 次、共 80 亿欧元；苹果被罚 180 亿欧元（2024 年 3 月）

**中国（SAMR 国家市场监督管理总局）**：
- 经营者集中申报
- 2021 年起虎牙-斗鱼合并被否（首次否决）
- 2024 年英伟达 700 亿美元收购 ARM 被否

#### 8.3 真实反垄断案例

- **2024 年 Adobe 放弃 200 亿美元收购 Figma**：欧盟、英国反垄断压力太大。
- **2023 年 Microsoft 完成 690 亿美元收购动视暴雪**：被英国 CMA 阻挠，最后妥协（云游戏权剥离给 Ubisoft）。
- **2021 年 Nvidia 放弃收购 ARM**：全球监管反对。
- **2019 年 Bristol-Myers Squibb 740 亿美元收购 Celgene**：FTC 担忧肿瘤药垄断，要求剥离 Otezla。
- **2016 年 Pfizer-Allergan 合并取消**（1600 亿美元）：美国财政部针对"税收倒置"出新规，交易流产。

#### 8.4 政治风险

除了反垄断，还有：
- **CFIUS（美国外国投资委员会）**：审查外资收购美国公司
- **中国对外投资审查**：2021 年起滴滴 App 下架事件后，跨境数据并购变难
- **欧盟外资审查条例**：2019 年生效
- **行业特殊监管**：金融（Fed/OCC）、电信（FCC）、医疗（FDA）

### 9. 一些容易混淆的概念

#### 9.1 投资银行 vs 商业银行

- **投资银行（高盛、JPM）**：帮企业融资、并购、上市
- **商业银行（Chase、Wells Fargo）**：做存款、贷款、信用卡

**1933 年 Glass-Steagall 法案**曾强制分业经营，1999 年废除。2008 年金融危机后，Volcker Rule 又限制了商业银行自营业务。

#### 9.2 卖方 vs 买方投行

- **卖方投行（前台）**：高盛、摩根士丹利这些——撮合交易
- **买方投行（Buy-side Advisory）**：贝莱德、富达这种投资管理公司内部的并购团队

#### 9.3 Enterprise Value vs Market Cap

- **Market Cap 市值**：股价 × 流通股数
- **EV 企业价值**：市值 + 净债务 + 少数股东权益 - 投资

**买公司 = 买市值 + 承担债务 - 拿回现金**。EV 更全面。

#### 9.4 Enterprise Value vs Equity Value

- **EV（企业价值）**：买下整家公司要花的钱
- **Equity Value（股权价值）**：买下股东手里的股份要花的钱

**EV = Equity Value + Net Debt**。

#### 9.5 私有交易 vs 公开市场

- **公开市场**：股票在交易所买卖，价格透明
- **私有市场**：公司没上市，估值靠协商
- **私募基金（PE/VC）**：在私有市场投资
- **Pre-IPO 二级市场**：上市前的老股转让，比如 Forge、EquityZen

### 10. 投行 / 并购的 4 个关键概念

#### 10.1 Sources & Uses（S&U，来源与用途）

**每笔并购交易的钱从哪来、用到哪去**：

**Sources（来源）**：
- 新股发行：50 亿
- 银行贷款：100 亿
- 现有现金：20 亿
- 卖方票据：30 亿
- 合计：200 亿

**Uses（用途）**：
- 支付给目标公司股东：180 亿
- 偿还目标公司债务：10 亿
- 交易费用（投行、律师、会计师）：5 亿
- 整合准备金：5 亿
- 合计：200 亿

**为什么要这张表**：监管要看交易结构合理性、买方的财务压力。

#### 10.2 Accretion / Dilution（增厚 / 摊薄）

并购对买家每股收益（EPS）的影响：
- **Accretive（增厚）**：并购后 EPS 上升。比如买家 P/E 30 倍，目标 P/E 15 倍，并购后 P/E 下降，EPS 上升。
- **Dilutive（摊薄）**：并购后 EPS 下降。比如买家 P/E 15 倍，目标 P/E 50 倍，并购后 P/E 上升，EPS 下降。

**为什么重要**：增厚交易宣布后股价通常涨（市场欢迎），摊薄交易通常跌。

#### 10.3 Goodwill（商誉）

并购支付的钱 vs 目标公司账面价值的差额。

**举例**：你花 1000 万买了一家账面 600 万的公司。差额 400 万就是"商誉"——你为这家公司的"品牌、客户、技术"多付的钱。

商誉每年要"减值测试"（Impairment Test），如果发现不值 400 万了，要减值 → 当年亏损。**真实案例**：2023 年很多公司商誉减值——包括美国 AT&T 商誉减值 60 亿、康菲石油减 30 亿。

#### 10.4 Earn-out（对赌支付）

并购时不一次付清，分批付：**未来业绩达到才付剩下的钱**。

**举例**：你 1000 万买目标公司，约定目标公司 3 年内 EBITDA 达到 1000 万/年（合计 3000 万），你才付剩下的 200 万。

**好处**：
- 卖方能拿到更高的总价（如果做得好）
- 买方降低风险（做不好少付）

**真实案例**：
- 2017 年 Amazon 137 亿美元收购 Whole Foods：全部现金
- 2012 年 Google 125 亿美元收购 Motorola Mobility：后来 29 亿卖给 Lenovo，亏了 90 亿
- 2016 年 Microsoft 262 亿美元收购 LinkedIn：全额现金

#### 11.2 投行 vs 咨询 vs PE vs HF

### 12. 这页"并购概览"模板怎么用

#### 12.1 模板设计逻辑

**这页模板重点是"页面设计"，不是"信息内容"**。原因：
- 投行客户**视觉审美**很重要——华兴资本、高盛、摩根士丹利的报告都很"漂亮"
- 模板规定了颜色（#2E75B6 蓝色表头、#D6E4F0 浅蓝底色）、字体（Arial 8pt）
- 规定了表格的填充比例（每页 60-80% 满，留白 20-40%）
- 规定了信息密度（每张表 6-10 行）

#### 12.2 信息内容重点

**真正要传达的信息是**：
- **公司简介**（5 行能说清业务）
- **财务数据**（收入 4 年趋势、分部结构、利润）
- **业务关系**（客户/供应商/合作伙伴/竞争对手）
- **并购活动**（最近 2-3 年买/卖了什么）

**不要的内容**：
- 复杂的 DCF 表格（这是估值模型的事）
- 大量文字段落（客户没时间读）
- 不相关的数据（只保留最核心的 6 个数字）

#### 12.3 角色化使用

**这页模板会被谁用**：
- **IB 银行家**：做 pitchbook 时附这页，让客户"快速了解"
- **企业发展战略部**：评估潜在收购方/目标
- **PE 基金**：研究行业并购趋势
- **媒体 / 投资社区**：做报道参考

**为什么 2 页**：
- 1 页太单薄（信息量不够）
- 3 页+ 太长（没人看）
- 2 页是"黄金"——核心信息能传达，又能视觉上留印象

### 13. 给小白的一份速查清单

#### 13.1 读这页模板需要知道的 8 个关键概念

1. **投行（IB）**：帮企业融资、并购、上市的"中间人"
2. **M&A（并购）**：公司买公司，分友好和恶意
3. **LBO（杠杆收购）**：用大量借债买公司
4. **IPO（首次上市）**：公司第一次向公众卖股票
5. **DCF / Comps / Precedent**：三种主要估值方法
6. **EV/EBITDA、P/E**：最常用的估值倍数
7. **HSR / 反垄断**：政府审查
8. **Goodwill / Earn-out**：并购的会计概念

#### 13.2 看到陌生词的思考顺序

1. 这是个**公司**？**交易**？**比率**？**法规**？
2. 谁在做/关心这件事？
3. 为什么这件事重要？
4. 有什么真实案例？
5. 这件事最近有什么变化？

#### 13.3 投行 / 并购的"5 个真相"

1. **钱来自客户**：投行所有的钱都是客户付的——做 IPO 收 7%、做并购收"成功费"（0.5-2% 交易额）
2. **人是核心资产**：客户买的是"团队"，不是"公司"——明星银行家跳槽能带走客户
3. **周期性强**：经济好的时候 IPO/并购多，差的时候少——2023 年 IPO 募资额降 30%+
4. **监管越来越严**：反垄断、数据隐私、外资审查都越来越严
5. **科技在改变一切**：AI、加密、Web3 催生新型交易——但传统业务也还在

---

> **给小白的一句话**：投行 / 并购这行看起来金光闪闪，但讲穿了就是"**帮人赚钱、帮人花钱、帮人买东西、帮人卖东西**"。DCF、Comps、Precedent 那些高大上的模型，本质上都是在回答"这家公司值多少钱"——可以用计算器算，也可以用嘴皮子谈，但**最终都是人为决策**。把这页模板当成"一本字典的目录页"：先知道有哪些章节（投行、并购、估值、LBO、IPO、监管），需要的时候再深入读。**金融的门槛是知识，但金融的顶端是判断**——多看真实案例（比如微软-领英、英伟达-ARM），少记术语。
