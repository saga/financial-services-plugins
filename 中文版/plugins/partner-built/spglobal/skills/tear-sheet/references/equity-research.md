# 股权研究概览页面

## 用途
为买方或卖方分析师评估投资而设计的信息密集型快照。侧重于估值、前瞻预期、财务轨迹和分析师一致预期。一切内容都应支持或挑战投资论点。

**默认页数：** 1 页。股权研究概览页传统上是单页——信息密度是卖点。如用户要求更多篇幅，可扩展到 2 页。

## 查询计划

从以下查询开始。如结果不完整，拆分为更窄的后续查询。

**查询 1 — 简介 + 市场数据：**
"[公司] 公司概况 行业 市值 企业价值 股价 52周高低 流通股数 Beta"
→ 标题、业务描述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询 2 — 历史财务数据：**
"[公司] 年度利润表 收入 毛利润 EBITDA 净利润 EPS 自由现金流 总债务 现金及等价物 最近4个财年"
→ 财务摘要（拉取4年，展示3年；最早一年仅用于同比增速计算）
→ **立即写入** `/tmp/tear-sheet/financials.csv`

**查询 2b — 分部数据（如可用）：**
"[公司] 收入按业务分部 最近2个财年"
→ 收入与分部结构（需要2年数据计算同比增速）。如无分部数据，跳过此节——不要留空白表格。
→ **立即写入** `/tmp/tear-sheet/segments.csv`（无分部数据则跳过）

**查询 3 — 估值 + 一致预期：**
"[公司] P/E EV/EBITDA EV/收入 估值倍数 一致预期收入 EPS 分析师评级 目标价"
→ 估值快照、一致预期
→ **立即写入** `/tmp/tear-sheet/valuation.csv`
→ **立即写入** `/tmp/tear-sheet/consensus.csv`

**查询 4 — 财报：**
"[公司] 最近一次财报电话会 关键要点 指引"
→ 财报亮点
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询 5 — 股价表现：**
"[公司] 股价回报 1个月 3个月 6个月 1年 年初至今"
→ 股价表现（无中间文件——数据直接用于文档）

**查询 6（如用户提供了可比公司）：**
"[同一家公司] 市值 EV/收入 EV/EBITDA 收入增长" — 每家公司重复
→ 估值快照的同行背景
→ **立即写入** `/tmp/tear-sheet/peer-comps.csv`

## 章节

按优先级排序。如需限制在一页，从底部开始裁剪。

### 1. 公司标题
紧凑的关键值块，按全局样式配置渲染为两列无边框表格。

左列：股票代码（交易所）、行业/子行业、总部
右列：股价、52周范围、市值、EV、流通股数、Beta

### 2. 业务描述
2-3 句紧凑描述。**以分析师受众的角度用自己的话重写**——不要直接粘贴 CIQ 公司摘要。CIQ 摘要是输入，输出应简洁且以论点为导向。

对于知名度高、广泛覆盖的公司，假设分析师已经了解基本业务。以**正在变化**的内容开头——战略转型、投资组合重塑、新的增长方向——而不是维基百科式的概述。例如，对一家财富500强公司，不要花一句话写"为机构提供金融数据"。改为："正在将其投资组合转向更高利润的数据和分析业务，有正在进行的 Mobility 分拆计划，并通过其 Kensho 子公司加大 AI 投资。"

对于不太知名的公司，在转向论点相关的动态之前，可以写一句简短的"他们做什么"。

### 3. 估值快照
股权概览页的核心。

| 指标 | 回溯（Trailing） | 前瞻（NTM） |
|---|---|---|
| P/E | | |
| EV/EBITDA | | |
| EV/收入 | | |
| P/FCF | | |
| 股息率 | | |

**有前瞻数据时，前瞻倍数必须显示。** 只有回溯倍数而前瞻数据存在是重大缺失——分析师用前瞻盈利来估值。如前瞻倍数不可用，仅显示回溯倍数并注明"远期预测不可用"。

如用户提供了可比公司，为每家公司加一列（如有3家以上可比公司则加"同行中位数"列）。如用户未提供可比公司，而工具返回同行数据，包含同行中位数列。两者都不可用时，仅显示公司自身倍数。

### 4. 一致预期
市场在预期什么——这是此类概览页的关键差异点。

**数据可用性说明：** 一致预期数据的深度因公司和覆盖范围而异。包含工具返回的任何数据——分析师数量、目标价、买入/持有/卖出分布——但不要编造或估算任何一致预期数据。对于覆盖较少的公司，此节可能仅包含收入和 EPS 预测，仍然有价值。

| 指标 | FY[年]预测 | FY[年+1]预测 |
|---|---|---|
| 收入 | | |
| EPS（调整后） | | |
| EBITDA | | |

尽可能使用调整后 EPS——GAAP EPS 可能被一次性项目扭曲，前瞻预测使用价值较低。

在主要预测表下方，包含一个紧凑的分析师一致意见块（如工具提供数据）：

| 分析师一致意见 | |
|---|---|
| 平均目标价 | XXX美元 |
| 预测数量 | XX |
| 买入 / 持有 / 卖出 | XX / XX / XX |

如目标价或评级数据不可用，省略分析师一致意见块而非显示空行。不要编造这些数字。

如一致预期数据完全不可用，跳过整个小节并注明"一致预期不可用"。不要估算。

使用实际财年标签的密集单一表格。

| 指标（百万美元） | FY20XX | FY20XX | FY20XX |
|---|---|---|---|
| 收入 | | | |
| 收入增长率% | | | |
| 毛利率% | | | |
| EBITDA | | | |
| EBITDA利润率% | | | |
| 净利润 | | | |
| EPS（摊薄） | | | |
| 自由现金流 | | | |
| 净债务 | | | |

从原始数据计算衍生指标（增长率%、利润率），而非单独查询。

**资本结构（财务摘要下方的独立紧凑子表）：**
不要将这些行附加到 3 年财务摘要表中——前几年列中的空白单元格看起来像缺失数据。改为渲染为下方独立的 2 列子表（指标 | 值），仅显示最近一个财年：

| 指标 | FY[最新] |
|---|---|
| 总债务 | XX百万美元 |
| 现金及等价物 | XX百万美元 |
| 净债务 | XX百万美元 |

这与企发和 IB/并购概览页的资本结构格式一致。保持紧凑——财务摘要和此子表之间无额外间距。

### 6. 收入与分部结构
如分部数据可用，包含紧凑的分部收入表。这对股权研究分析师至关重要——他们需要看哪些业务线在推动增长。

| 分部 | 收入（百万美元） | 占总收入% | 同比增速 |
|---|---|---|---|
| [分部A] | | | |
| [分部B] | | | |

为一页概览保持紧凑：仅表格，无定性段落。如无分部数据，完全跳过此节——不要包含空白或占位表。

### 7. 近期财报亮点
包含季度和日期（如"Q3 FY2025 — 2025年10月"）。以**投资视角**的3-4条要点——这个受众想要分部层面的具体数据，而非战略主题：

**如果有指引，首先放在前面。** 如管理层发布了前瞻指引，将其作为第一条要点，用加粗的**指引：**前缀（例如"**指引：** FY2026 EPS $19.40-$19.65，隐含 9-10%增长；有机收入增长 6-8%。"）。指引是分析师可获得的最可操作的远期数据点——不要将其埋没在经营亮点中。

**超越/不及背景（数据许可）：** 如报告期的一致预期数据可用，将标题结果要点表述为超越/不及："收入 X 超越/不及一致预期 Y，差距 Z%。"如无报告期的一致预期数据，仅用绝对值和增长率描述结果——不要编造一致预期数字。

然后：
- 分部表现：哪些业务线加速或减速
- 利润率轨迹：关于成本结构或盈利趋势的评论

归属于管理层："管理层强调…"或"CFO 指出…"，而非陈述性表述。这部分应像财报回顾，而非新闻稿摘要。

### 8. 关键经营指标
2-3 个行业相关的 KPI（如数据可用）：订户数、同店销售、NIM 等。如无行业特定指标，从已检索的财务数据中计算 ROE 和 ROIC。可选——空间紧张时删除。

### 9. 股价表现（首先裁剪）
期间回报表。对此受众分析价值低——先于其他任何内容裁剪。

| 期间 | 回报 |
|---|---|
| 1个月 | |
| 3个月 | |
| 年初至今 | |
| 1年 | |

## 格式说明
- **这是最密集的概览页类型。** 应明显比其他模板更紧凑。
- 财务摘要表：8pt 字体（size: 16），行高 240 DXA。最紧凑的间距。
- 估值快照和一致预期：8.5pt 字体，占据视觉中心——分析师首先扫读这些。
- 正文：8.5pt（size: 17）— 小于全局默认的 9pt。每一半磅对一页纸都很重要。
- 章节间距：最小化。节标题前 6pt（非全局的 12pt），分割线后 2pt。
- 任何显示显著变化的指标（增长转正、利润率扩张/压缩）用加粗。
- 处处优先信息而非空白。

### 1. 这个模板跟 IB/M&A 版有什么不同——"看估值"vs"看交易"

**想象一个场景**：同一家公司（比如 Snowflake），投行家看的是"能不能卖掉、卖多少钱"（交易视角），而分析师看的是"该买还是该卖、现在贵不贵"（投资视角）。

**这就是 Equity Research 版和 IB/M&A 版的核心区别**：

| 维度 | IB/M&A 版 | Equity Research 版 |
|------|-----------|-------------------|
| 核心问题 | 这家公司值多少钱？ | 这只股票该买还是该卖？ |
| 估值侧重 | 交易倍数（EV/EBITDA、EV/收入） | 前瞻倍数（NTM P/E、NTM EV/EBITDA） |
| 数据重点 | LTM（已发生的） | NTM（预期的） |
| 独有章节 | 并购活动、Sources & Uses | 评级、目标价、一致预期偏差 |
| 受众 | 投行家、Corp Dev | 分析师、基金经理 |

**所以这意味着什么**：Equity Research 版最独特的地方是**前瞻数据**——分析师根据未来预期做判断，而不是根据过去数据。NTM（Next Twelve Months）倍数比 LTM 倍数更重要，因为"股票定价看未来"。

---

### 2. 回溯 vs 前瞻——"已经发生的"和"大家预期的"

**这是股权研究中最核心的概念之一**。

**想象一下**：你要判断苹果股票现在贵不贵。你有两个数据：
- **LTM P/E = 30 倍**（基于过去 12 个月的实际盈利）
- **NTM P/E = 25 倍**（基于未来 12 个月的分析师预期盈利）

哪个更靠谱？**答案是 NTM**。因为投资决策本质上是前瞻性的——你买的是公司未来的赚钱能力，不是过去的。

**但 LTM 也不是没用的**：

| 指标 | 用途 | 什么时候更重要 |
|------|------|--------------|
| **LTM（回溯）** | 验证公司实际表现 | 评估执行力、发现预期偏差 |
| **NTM（前瞻）** | 定价和投资决策 | 估值、买卖评级 |

**真实例子**：英伟达 2024 年 LTM P/E 约 60 倍（看起来很贵），但 NTM P/E 只有约 35 倍（因为分析师预期盈利会大幅增长）。如果你只看 LTM，你会觉得太贵了；但看 NTM，考虑到 AI 芯片的增长前景，35 倍其实不算离谱。

**所以这意味着什么**：这个模板中，**NTM 倍数必须显示**。仅有回溯倍数而前瞻数据存在是重大缺失——分析师根据前瞻盈利为股票定价。

---

### 3. 分红收益率——"股票给你多少利息"

**分红收益率 = 每股年度分红 ÷ 股价**。它告诉你：光靠分红，你每年能拿回投资本金的百分之几。

**想象一下**：你花 100 块买了一只股票，每年分红 3 块。分红收益率就是 3%。这就像你把钱存银行拿利息——3% 的分红收益率相当于 3% 的"存款利率"。

**不同公司的分红策略差异很大**：

| 公司类型 | 分红收益率 | 为什么 |
|---------|-----------|--------|
| **成熟蓝筹**（可口可乐、强生） | 2-4% | 赚钱稳定，不需要大量再投资，把利润分给股东 |
| **科技公司**（苹果、微软） | 0.5-1% | 更倾向于回购股票而非分红（税务原因） |
| **高增长公司**（英伟达、Snowflake） | 0% | 所有利润都再投资到增长中 |
| **公用事业**（电力、水务） | 4-6% | 监管要求稳定分红，增长慢 |

**真实例子**：可口可乐分红收益率约 3%，连续 60+ 年每年提高分红——这叫"分红贵族"（Dividend Aristocrat）。退休老人喜欢买这种股票，因为每年有稳定的现金收入。

**所以这意味着什么**：分红收益率不是越高越好。6% 的收益率可能意味着公司增长停滞、股价下跌（分母变小导致收益率变高）。要看收益率 + 增长率的组合。

---

### 4. 一致预期偏差——"市场预期和实际结果的差距"

**一致预期（Consensus Estimate）** 是所有覆盖该股票的分析师预测的平均值。当公司实际业绩跟一致预期不一样时，股价会剧烈波动。

**想象一下**：20 个分析师都预测苹果 Q2 EPS 为 1.50 美元。结果苹果报了 1.60 美元——超出预期 6.7%。股价可能涨 3-5%，因为"比大家想的好"。

**反过来**：如果苹果只报了 1.40 美元——低于预期 6.7%。股价可能跌 5-8%，因为"比大家想的差"。

**三种偏差方向**：

| 偏差 | 术语 | 股价反应 |
|------|------|---------|
| 实际 > 预期 | Beat（超预期） | 通常上涨 |
| 实际 = 预期 | In-line（符合预期） | 看指引 |
| 实际 < 预期 | Miss（不及预期） | 通常下跌 |

**但更重要的不是"Beat 还是 Miss"，而是"指引"**。公司 Beat 了但下调未来指引，股价照样跌。公司 Miss 了但上调指引，股价可能涨。

**真实例子**：Meta 2022 年 Q4 财报 Beat 了预期，但 CEO 扎克伯格说"2023 年会更艰难"——股价当天跌了 20%+。反过来，英伟达 2024 年 Q1 大幅 Beat 预期，同时上调了 Q2 指引——股价盘后涨了 25%。

### 5. GAAP vs 调整后——"官方数字"和"公司想让你看的数字"

**GAAP（Generally Accepted Accounting Principles）** 是美国法定会计准则。所有上市公司必须按 GAAP 报告财务数据。但很多公司还会报告"调整后（Adjusted / Non-GAAP）"数字——排除了一次性项目。

**想象一下**：你开奶茶店，一年赚了 100 万。但其中 30 万是卖了一台旧机器的一次性收入。GAAP 报告说赚了 100 万，调整后报告说赚了 70 万（排除了卖机器的钱）。哪个更能反映你的"真实经营能力"？答案是 70 万。

**常见的调整项目**：
- **股票期权费用（Stock-based Compensation）**：科技公司最爱调整掉，说"这不是现金支出"
- **重组费用（Restructuring Charges）**：裁员、关厂的一次性费用
- **并购相关费用**：收购时的整合成本
- **资产减值（Impairment）**：商誉减值等

**真实例子**：Snowflake 2024 财年 GAAP 净亏损 8 亿美元，但调整后只亏 2 亿美元——差了 6 亿美元的股票期权费用。分析师通常看调整后数字来评估经营表现，但投资者也要注意：如果一家公司每年都"调整"掉大量费用，那这些费用可能不是"一次性的"。

**所以这意味着什么**：这个模板中，财务数据通常优先显示调整后数字（因为分析师用这个做估值），但也会标注 GAAP 数字作为参考。

> **给小白的一句话**：Equity Research 版的 Tearsheet 就像一份"股票体检报告"——核心不是看公司过去多赚钱（那是 IB/M&A 版关心的事），而是看**未来能赚多少、现在贵不贵、市场预期跟实际差多远**。记住：**买股票买的是未来，不是过去**——NTM 比 LTM 重要，指引比 Beat/Miss 重要，调整后数字要看但也要警惕"过度调整"。
