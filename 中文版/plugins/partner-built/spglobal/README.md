# S&P Global 插件

本插件通过一系列预制技能，将 S&P Global 的金融数据和分析直接引入您的 AI 工作流。专为需要 AI 辅助研究、分析和文档生成的金融专业人士设计，所有数据均来源于权威的 S&P Global。

这些技能基于开放标准（MCP）构建，可在各种 AI 平台和代理框架中使用。虽然本插件遵循 Claude Cowork 标准，但所有技能及其底层数据层均与平台无关。如果您想在其它环境中使用这些技能，欢迎使用。

我们理解每个组织都有独特需求。这些技能是起点，帮助您快速构建最终产品。欢迎您根据公司的具体流程、模板和数据需求调整提示词、输出格式和工作流。

本插件中的技能按"原样"提供。生成的输出和数据不保证正确性。**务必验证 LLM 生成的输出是否准确。**

**S&P 与 Moody's、Fitch 并称"全球三大评级机构"**，其主权评级和公司评级直接影响全球数十万亿美元的债券定价。

#### 1.3 S&P 评级体系

**长期信用评级**（从高到低）：

| 评级 | 含义 | 典型发行人类别 |
|------|------|---------------|
| **AAA** | 最高信用质量，违约风险极低 | 美国、德国等顶级主权国家 |
| **AA+ / AA / AA-** | 很高信用质量 | 大型跨国公司、发达经济体 |
| **A+ / A / A-** | 高信用质量 | 优质企业、新兴市场大国 |
| **BBB+ / BBB / BBB-** | 投资级最低档 | 一般企业、新兴市场 |
| **BB+ / BB / BB-** | 投机级（垃圾级） | 高收益债券发行人 |
| **B+ / B / B-** | 高投机 | 财务状况较弱企业 |
| **CCC / CC / C** | 重大信用风险 | 困境企业 |
| **D** | 已违约 | 已发生违约事件 |

**评级影响**：
- 评级下调 1 档，融资成本上升 50-200 个基点
- 投资级（BBB-及以上）可纳入投资级债券指数
- 投机级（BB+及以下）即"垃圾债"，收益率显著更高

#### 2.1 Capital IQ 平台定位

**S&P Capital IQ Pro** 是金融行业最广泛使用的公司财务数据库之一，被誉为"金融从业者的 Google"。

**"四大金融数据库"**：Bloomberg、Capital IQ、FactSet、Refinitiv（被 LSEG 收购）是全球金融数据的四大巨头。

#### 2.4 Capital IQ 的局限

**数据问题**：
- 私募/小公司数据有限（尤其是新兴市场）
- 历史数据有时缺失或修订
- 财务调整（Adjustments）较多，需理解底层口径
- 标准化的财务数据可能与公司原始报表有差异

**替代/补充数据源**：
- **Bloomberg**：交易员偏好，数据更新更快
- **FactSet**：资管偏好，组合分析强
- **Refinitiv (LSEG)**：欧洲市场数据全
- **Wind（万得）**：中国市场数据最全
- **CB Insights**：私募股权和初创公司
- **Pitchbook**：PE/VC 数据
- **Preqin**：基金数据

### 3. Kensho 与 LLM-Ready API 详解

#### 3.1 Kensho 简介

**Kensho Technologies** 是 S&P Global 旗下的人工智能公司（2024 年商业部分被 Bain Capital 收购，但仍是 S&P Global 的 AI 合作伙伴）。

**核心使命**：用 AI 解决金融行业的复杂数据分析问题。

**技术亮点**：
- 自然语言处理（NLP）：让机器理解金融文档
- 知识图谱：连接公司、人物、交易、事件
- 机器学习：从非结构化数据中提取信息
- LLM 集成：把 AI 嵌入到金融工作流

#### 3.2 LLM-Ready API 是什么

**定义**：Kensho LLM-Ready API 是一个专为大型语言模型（LLM）优化的金融数据 API，将 S&P Global 的数据组织成 LLM 易于理解的格式。

**传统 API vs LLM-Ready API**：

| 维度 | 传统金融 API | LLM-Ready API |
|------|-------------|---------------|
| **数据格式** | JSON/XML，字段嵌套深 | 简化的"键-值"结构，LLM 友好 |
| **文档长度** | 需查文档理解字段 | 字段名直观，文档内置 |
| **错误处理** | 错误码、堆栈信息 | 自然语言错误说明 |
| **调用方式** | 需编程知识 | LLM 可自主调用 |
| **MCP 支持** | 不一定 | ✅ 原生支持 |
| **上下文长度** | 一次返回大量数据 | 分页、按需加载 |

#### 3.3 MCP（Model Context Protocol）

**定义**：MCP 是 Anthropic 提出的开放标准协议，让 LLM 能够安全地访问外部工具和数据源。

**MCP 的价值**：
- **标准化**：一个协议连接所有工具
- **安全性**：受控访问，避免 LLM 直接操作敏感系统
- **可组合**：不同 MCP server 可组合使用
- **生态化**：社区贡献的工具快速增长

**MCP 在金融行业的应用**：
- 接入 Bloomberg、Capital IQ、FactSet 等数据
- 连接企业内部 CRM、风控、交易系统
- 标准化 AI 与遗留系统集成

### 4. 公司概览页面（Tearsheet）深度解析

#### 4.1 什么是 Tearsheet

**定义**：Tearsheet（直译"流泪纸"，金融术语意译"概览页面"）是 1-2 页的精炼公司分析报告，是投行和投资研究的标准交付物之一。

**命名由来**：早期研究报告装订成大册，分析师翻阅时常撕下某些重要公司页面，torn out（撕下）的页面集合叫 tearsheet。

#### 4.2 Tearsheet 的四大受众类型

本插件明确支持四种受众，每种侧重点不同：

| 受众 | 英文 | 核心关注点 | 典型输出 |
|------|------|----------|---------|
| **股权研究** | Equity Research | 投资论点、增长催化剂、风险 | 投资建议（买入/持有/卖出） |
| **投资银行/M&A** | IB / M&A | 交易情境、协同效应、估值 | 交易结构建议 |
| **企业发展** | Corporate Development | 战略匹配度、收购协同 | 收购目标评估 |
| **销售/业务拓展** | Sales / BD | 客户需求匹配、决策链 | 客户谈话要点 |

**示例对比**：
- **同一公司（如 Salesforce）**：
  - 股权研究 → 增长前景、利润率提升、目标价
  - IB/M&A → 估值倍数、潜在买家、财务影响
  - 企业发展 → 与竞品对比、技术互补
  - 销售/BD → 客户痛点、决策者、采购流程

#### 4.3 Tearsheet 标准结构

**1. 公司概览**：
- 公司名称、股票代码、交易所
- 业务简介（1-2 句）
- 总部位置、成立时间、员工数
- 主要股东、董事会主席、CEO

**2. 关键财务数据**（3-5 年）：

| 指标 | 本期 | 上期 | YoY 变化 | 行业平均 |
|------|------|------|---------|---------|
| 收入 (Revenue) | $X bn | $Y bn | +Z% | +A% |
| 净利润 | $X bn | $Y bn | +Z% | +A% |
| 毛利率 | X% | Y% | +Z pp | A% |
| 营业利润率 | X% | Y% | +Z pp | A% |
| ROE | X% | Y% | +Z pp | A% |
| 自由现金流 | $X bn | $Y bn | +Z% | - |

**3. 业务拆分**（按部门/产品/地区）：
- 各业务收入占比
- 各业务增长率
- 关键产品/客户

**4. 估值指标**：
- 股价、市值、52周高低
- P/E、P/B、EV/EBITDA、EV/Revenue
- 分析师目标价、评级分布

**5. 增长驱动与风险**：
- 短期催化剂
- 长期增长动力
- 主要风险因素

**6. 同业对比**：
- 可比公司估值表
- 市占率分析
- 竞争优势

### 5. 财报预览（Earnings Preview）深度解析

#### 5.1 什么是财报预览

**定义**：Earnings Preview 是公司在正式发布季度财报（10-Q）或年度财报（10-K）前，分析师提前编写的"预告性研究报告"。

**目的**：
- 提前洞察市场预期
- 识别关键关注点和风险
- 准备发布会问答
- 调整持仓策略

#### 5.2 财报预览的核心内容

| 内容 | 说明 | 数据来源 |
|------|------|---------|
| **市场一致预期** | 分析师对收入/利润的一致预测中位数 | Capital IQ Estimates |
| **近期管理层指引** | 管理层在电话会、公告中给出的指引 | 8-K、新闻稿 |
| **运营指标** | 关键运营数据（KPI） | 公司公告、行业数据 |
| **分析师情绪** | 近期研报评级、目标价变动 | Bloomberg、CAP IQ |
| **历史对比** | 过去几个季度的"超预期"vs"不及预期" | Capital IQ |
| **关注要点** | 毛利率、用户增长、新业务等关键议题 | 综合分析 |
| **风险因素** | 可能导致不及预期的因素 | 综合分析 |

#### 5.3 一致预期（Consensus Estimates）详解

**定义**：一致预期是市场上所有覆盖某公司的卖方分析师，对公司未来业绩预测的中位数或平均值。

**计算方法**：
- **中位数法**（最常用）：所有预测的中位数
- **平均值法**：所有预测的算术平均
- **加权法**：按分析师历史准确度加权

**举例**：
- 20 位分析师预测 Salesforce 下一季度收入
- 预测值范围：$9.0B - $9.8B
- 中位数（Consensus）：$9.4B
- 实际公布：$9.45B → 略超预期（Beat by 0.5%）

**"Beat" 与 "Miss"**：
- **Beat**（超预期）：实际值 > 一致预期，股价通常上涨
- **In-line**（符合预期）：实际值 ≈ 一致预期，股价变化小
- **Miss**（不及预期）：实际值 < 一致预期，股价通常下跌

**重要细节**：
- Beat/Miss 不仅看数字本身，更看"方向"和"指引"
- 有时数字超预期但指引下调，股价反而下跌
- 分析师的"Whisper Number"（非官方预期）有时更准

#### 5.4 财报日历与发布周期

**美国上市公司财报披露规则**：
- **10-K**：年度财报，财年结束后 60-90 天内披露
- **10-Q**：季度财报，季度结束后 40-45 天内披露
- **8-K**：重大事件即时披露
- **Proxy**：股东大会前披露

**四季度分布**：
- Q4 财报（年报）：1-3 月集中发布（最密集）
- Q1 财报：4-5 月发布
- Q2 财报：7-8 月发布
- Q3 财报：10-11 月发布

**中国 A 股**：季报在季度结束后 30 天内披露，半年报 60 天，年报 120 天。

### 6. 并购交易摘要（Transaction Summary）深度解析

#### 6.1 什么是 M&A League Tables

**定义**：League Tables 是投行完成的并购、股权融资、债务融资交易的排行榜，按交易金额或数量排名。

**核心排行榜**：

| 排行榜 | 说明 | 用途 |
|--------|------|------|
| **M&A Advisory League Tables** | 并购顾问排名 | 投行品牌、客户开发 |
| **Equity Capital Markets (ECM) League Tables** | 股权融资排名 | 投行 IPO/增发业务 |
| **Debt Capital Markets (DCM) League Tables** | 债务融资排名 | 投行发债业务 |
| **Loan Syndications League Tables** | 银团贷款排名 | 商业银行贷款业务 |

**数据来源**：Bloomberg、Dealogic、Refinitiv、Capital IQ 都发布排行榜。

#### 6.2 M&A 交易的关键字段

| 字段 | 说明 | 示例 |
|------|------|------|
| **Announce Date** | 宣布日期 | 2024-06-15 |
| **Close Date** | 完成日期 | 2024-12-20 |
| **Target** | 目标公司 | Slack Technologies |
| **Acquiror** | 收购方 | Salesforce |
| **Deal Value** | 交易金额 | $27.7 bn |
| **Offer Price** | 报价 | $45.5 / 股 |
| **Premium** | 溢价 | +55% (vs 1 个月均价) |
| **Deal Type** | 交易类型 | 现金、股票、混合 |
| **EV/EBITDA** | 估值倍数 | 60x |
| **Advisors** | 财务顾问 | Goldman, JPM |
| **Legal Advisors** | 法律顾问 | Skadden |

#### 6.3 估值倍数深度解析

**EV/EBITDA 详细解释**：

**定义**：Enterprise Value（企业价值）除以 EBITDA（息税折旧摊销前利润），衡量"买下整个公司"的价格相对其盈利的倍数。

**计算**：
```
EV = 市值 + 净债务 + 少数股东权益 + 优先股 - 现金及等价物
EBITDA = 营业利润 + 折旧 + 摊销
```

**行业典型水平**：

| 行业 | EV/EBITDA 范围 | 解释 |
|------|---------------|------|
| 银行业 | 8-12x | 重资产，低成长 |
| 制造业 | 5-10x | 周期性强 |
| 消费品 | 10-15x | 品牌溢价 |
| 科技/SaaS | 20-40x | 高成长 |
| 生物医药 | 15-25x | 研发期 |
| 房地产 | 12-18x | 资产价值明确 |

**影响倍数的因素**：
- **增长率**：增长率高，倍数高
- **利润率**：利润率稳定且高，倍数高
- **资本密集度**：轻资产，倍数高
- **行业前景**：行业增长快，倍数高
- **规模**：大公司通常倍数较低
- **现金流可预测性**：可预测，倍数高

**M&A 溢价（Control Premium）**：
- 通常 20-40%（vs 未受影响的股价）
- 控制权价值：少数股东无法享受的协同效应
- 战略买家通常比财务买家出价更高

#### 6.4 并购流程详解

| 阶段 | 时长 | 主要工作 |
|------|------|---------|
| **1. 战略规划** | 1-2 月 | 制定并购战略、确定目标标准 |
| **2. 目标筛选** | 2-4 月 | 用 Capital IQ 筛选目标公司 |
| **3. 初步接触** | 1-2 月 | 签署 NDA，交换初步信息 |
| **4. 估值与报价** | 2-3 月 | DCF、Comps 估值，提出报价 |
| **5. 尽职调查** | 2-3 月 | 财务、法律、商业、运营 DD |
| **6. 谈判签约** | 1-2 月 | 谈判条款，签署 SPA |
| **7. 监管审批** | 3-12 月 | 反垄断（HSR）、外国投资审查 |
| **8. 交割完成** | 1-3 月 | 资金交割，整合开始 |

**常见失败原因**：
- 估值差异过大
- 尽职调查发现重大问题
- 监管反垄断否决
- 卖方股东否决
- 融资失败
- 整合协同效应不达预期

**统计**：并购交易中，约 30-50% 的交易最终未能完成（deal break rate）。

### 7. 卖方研究（Sell-side Research）详解

#### 7.1 什么是卖方研究

**定义**：卖方研究是投行/券商的研究部门发布的投资分析报告，主要服务买方机构（基金、资管）客户。

**商业模式**：
- 投行不直接向客户收费
- 通过研究报告吸引机构客户
- 机构客户通过投行进行交易，投行赚取佣金
- 间接补贴研究部门

**与买方研究区别**：

| 维度 | 卖方研究 | 买方研究 |
|------|---------|---------|
| **发布方** | 投行/券商 | 资管/基金内部 |
| **客户** | 外部机构投资者 | 内部投资团队 |
| **公开性** | 公开（订阅） | 不公开（内部） |
| **频率** | 季度/事件驱动 | 持续 |
| **覆盖广度** | 覆盖 30-100 家公司 | 深度覆盖 10-30 家 |
| **主要产出** | 研报、目标价、评级 | 投资决策、内部备忘录 |
| **合规要求** | 严格（MiFID II、研究独立性） | 较灵活 |

#### 7.2 卖方研究的标准格式

**Initiation Report（首次覆盖报告）**：
- 公司概况、行业地位
- 投资亮点与风险
- 详细财务模型
- 估值方法（DCF + Comps）
- 投资评级与目标价
- 30-50 页深度分析

**Update Note（更新报告）**：
- 季度业绩点评（2-5 页）
- 重大事件分析
- 评级/目标价调整

**Flash Note（快速点评）**：
- 重大事件即时反应
- 1-2 页短报告

**Earnings Preview（财报预览）**：
- 财报前预期分析
- 关键关注点
- 5-10 页

**Earnings Review（财报回顾）**：
- 财报后实际 vs 预期
- 管理层电话会要点
- 5-10 页

#### 7.3 投资评级体系

**标准评级**：

| 评级 | 含义 | 预期回报 |
|------|------|---------|
| **Buy / Outperform** | 买入 | > 15%（12 个月） |
| **Hold / Neutral** | 持有 | -10% ~ +15% |
| **Sell / Underperform** | 卖出 | < -10% |
| **Not Rated** | 未评级 | - |

**目标价（Target Price）**：
- 基于估值方法（DCF、Comps、Precedent Transactions）得出
- 通常给出 12 个月目标价
- 对应预期回报率

**评级分布规则**（MiFID II 后）：
- 投行需公布评级分布（Buy/Hold/Sell 各占比）
- 防止投行盲目给出"Buy"以讨好上市公司
- 确保研究独立性

### 8. 投行估值方法全解

#### 8.1 三大估值方法

| 方法 | 英文 | 适用场景 | 核心逻辑 |
|------|------|---------|---------|
| **现金流折现** | DCF | 持续经营的公司 | 未来现金流现值 |
| **可比公司分析** | Trading Comps | 有可比公司上市 | 行业平均倍数 |
| **先例交易分析** | Precedent Transactions | 并购情境 | 类似交易倍数 |

#### 8.2 DCF 模型详解

**核心公式**：
```
公司价值 = Σ(FCF_t / (1+WACC)^t) + TV / (1+WACC)^n
```

其中：
- FCF：自由现金流（Free Cash Flow）
- WACC：加权平均资本成本
- TV：终值（Terminal Value）
- t：年份

**WACC 计算**：
```
WACC = (E/V) × Re + (D/V) × Rd × (1 - Tc)
```

- E：股权市值
- D：债务市值
- V = E + D：总资本
- Re：股权成本（用 CAPM 计算）
- Rd：债务成本（利息率）
- Tc：所得税率

**CAPM（股权成本）**：
```
Re = Rf + β × ERP
```

- Rf：无风险利率（如 10 年期美债收益率）
- β：股票相对于市场的波动（行业平均 1.0）
- ERP：股权风险溢价（约 5-7%）

**终值计算**：
- **永续增长法**：TV = FCF × (1+g) / (WACC - g)
- **退出倍数法**：TV = EBITDA × 退出倍数

**DCF 优缺点**：
- ✅ 理论上最严谨
- ✅ 不受市场情绪影响
- ❌ 对假设极度敏感
- ❌ 终值占大头（60-80%）

#### 8.3 可比公司分析详解

**步骤**：
1. **筛选可比公司**：
   - 同行业
   - 同规模（市值 ±50%）
   - 同成长性
   - 同地理区域
   - 5-10 家为宜

2. **计算估值倍数**：
   - P/E（盈利公司）
   - EV/EBITDA（最常用）
   - EV/Revenue（亏损公司）
   - P/B（金融机构）
   - PEG（考虑成长性）

3. **取中位数 / 平均值**：
   - 中位数更稳健（不受极端值影响）

4. **应用到目标公司**：
   - 目标公司 EBITDA × 行业倍数 EV

5. **调整（折扣/溢价）**：
   - 流动性折扣
   - 控制权溢价
   - 战略协同

**典型 Comps 表**：

| 公司 | 代码 | 市值 ($M) | EV ($M) | EV/Revenue | EV/EBITDA | P/E | 利润率 |
|------|------|----------|---------|-----------|-----------|-----|--------|
| Company A | AAAA | 10,000 | 11,000 | 5.0x | 18.0x | 25.0x | 22% |
| Company B | BBBB | 8,000 | 9,200 | 4.5x | 16.0x | 22.0x | 20% |
| Company C | CCCC | 12,000 | 13,500 | 6.0x | 20.0x | 28.0x | 25% |
| **中位数** | - | 10,000 | 11,000 | **5.0x** | **18.0x** | **25.0x** | **22%** |
| 目标公司 | DDDD | 9,000 | 10,000 | - | 18.0x | - | 20% |

### 9. 财务建模核心概念

#### 9.1 三张报表模型（3-Statement Model）

**定义**：将利润表、资产负债表、现金流量表联动起来的综合财务模型，是估值建模的基础。

**联动关系**：

```
利润表：
  收入 → 成本 → 毛利 → 营业利润 → 净利润
                                ↓
资产负债表（变动项）                  ↓
  现金、应收、存货、应付、有息负债      ↓
                                ↓
现金流量表 ←────────────────────────┘
  经营 + 投资 + 融资 = 现金净变动
```

**关键勾稽**：
- 利润表的净利润 = 现金流量表"净利润"
- 现金流量表期末现金 = 资产负债表现金科目
- 资产负债表的留存收益变动 = 利润表净利润 - 分红

#### 9.2 LBO 模型（杠杆收购）

**定义**：LBO = Leveraged Buyout，用大量债务融资收购公司，用被收购公司的现金流偿还债务，剩余股权价值放大。

**典型资本结构**（以 6 倍 EBITDA 收购为例）：

| 层级 | 金额 (x EBITDA) | 利率 | 特征 |
|------|----------------|------|------|
| 银行债务 - 1st Lien | 4.0x | SOFR + 4-5% | 有资产抵押 |
| 银行债务 - 2nd Lien | 0.5x | SOFR + 6-7% | 次级 |
| 高收益债券 | 1.0x | 8-10% | 无抵押 |
| 夹层融资 | 0.5x | 12-15% | 股债混合 |
| **总债务** | **6.0x** | - | - |
| **股权出资** | **4.0x** | - | 风险/回报 |
| **企业总价值** | **10.0x** | - | - |

**LBO 收益来源**：
- **去杠杆（Deleveraging）**：用经营现金流偿还债务
- **EBITDA 增长**：被收购公司经营改善
- **倍数扩张**：退出时估值倍数提升
- **股利分配**：杠杆收购中的常见回报

**典型案例**：
- 1989：KKR 收购 RJR Nabisco，$250 亿，史上最大 LBO
- 2007： Blackstone 收购 Hilton，$260 亿
- 2013：Carl Icahn 收购 Dell（部分）

#### 9.3 Pitch Book（路演文档）详解

**定义**：Pitch Book 是投行向潜在客户展示的精美文档，用于推介交易服务、关系维护、赢得 mandate。

**典型结构**：
1. **封面**：公司 logo、项目名
2. **执行摘要**：1 页概览
3. **公司概览**：目标公司简介
4. **行业分析**：市场规模、增长、竞争
5. **交易概览**：交易结构、估值、协同
6. **估值分析**：DCF、Comps、Precedents
7. **团队介绍**：执行团队资历
8. **为什么选我们**：投行优势

**常见 Pitch Book 类型**：
- **Sell-side Pitch**：帮卖方卖公司
- **Buy-side Pitch**：帮买方找目标
- **Defense Pitch**：防御性 pitch（防止恶意收购）
- **General Pitch**：关系维护型

### 12. 关键财务指标速查

#### 12.1 盈利能力指标

| 指标 | 公式 | 含义 | 优秀标准 |
|------|------|------|---------|
| **毛利率** | (收入 - 直接成本) / 收入 | 产品本身的盈利能力 | > 40%（软件） |
| **营业利润率** | 营业利润 / 收入 | 主营业务的盈利能力 | > 15% |
| **净利润率** | 净利润 / 收入 | 最终盈利能力 | > 10% |
| **EBITDA Margin** | EBITDA / 收入 | 现金盈利能力 | > 20% |
| **ROE** | 净利润 / 股东权益 | 股东回报 | > 15% |
| **ROA** | 净利润 / 总资产 | 资产回报 | > 8% |
| **ROIC** | 税后营业利润 / 投入资本 | 核心业务回报 | > 12% |

#### 12.2 估值倍数

| 倍数 | 公式 | 适用 |
|------|------|------|
| **P/E** | 股价 / 每股收益 | 盈利公司 |
| **P/B** | 股价 / 每股净资产 | 银行、保险 |
| **EV/Revenue** | 企业价值 / 收入 | 亏损公司、SaaS |
| **EV/EBITDA** | 企业价值 / EBITDA | 通用 |
| **EV/EBIT** | 企业价值 / EBIT | 通用 |
| **P/S** | 股价 / 每股收入 | 亏损公司 |
| **PEG** | P/E / 增长率 | 高成长公司 |
| **Dividend Yield** | 股息 / 股价 | 价值股、成熟公司 |

#### 12.3 成长性指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **收入增长率** | (本期收入 - 上期) / 上期 | YoY 增长 |
| **净利润增长率** | (本期净利 - 上期) / 上期 | 利润增长 |
| **EBITDA 增长率** | EBITDA YoY 增长 | 经营增长 |
| **CAGR** | (终值/初值)^(1/年数) - 1 | 复合年增长率 |
| **NTM 增长率** | 未来 12 个月预期增长 | 远期视角 |

### 14. 监管与合规要点

#### 14.1 信息披露要求

| 地区 | 主要文件 | 监管机构 |
|------|---------|---------|
| **美国** | 10-K、10-Q、8-K、Proxy | SEC |
| **欧盟** | Annual Report、Half-year Report | ESMA |
| **英国** | Annual Report | FCA |
| **香港** | 招股书、年报、半年报 | SFC、HKEX |
| **中国** | 招股书、年报、季报 | CSRC、证监会 |

**Reg FD（公平披露规则）**：
- 美国 SEC 规定，公司不得选择性披露重大信息
- 所有投资者应同时获得重大信息
- 投行分析师受其约束

#### 14.2 研究独立性（MiFID II）

**MiFID II** 是欧盟金融工具市场指令，对投行研究有重要影响：

| 要求 | 说明 |
|------|------|
| **研究付费分离** | 投行不得将研究费用捆绑在交易佣金中 |
| **研究预算透明** | 客户需明确为研究付费 |
| **研究 vs 投行隔离** | 研究部门不得参与投行项目 |
| **记录保存** | 所有研究记录留存 5+ 年 |

**对中国/亚洲的影响**：部分全球投行在亚洲也采用类似规则。

#### 14.3 投行数据使用合规

| 风险 | 防护措施 |
|------|---------|
| **内幕信息** | 严格信息墙（Information Barrier） |
| **利益冲突** | 公开披露、回避机制 |
| **数据泄露** | 严格访问控制、加密 |
| **数据滥用** | 使用条款、审计日志 |

### 15. 实战案例：S&P Global 数据使用流程

#### 15.1 案例：评估 Salesforce 投资价值

**目标**：判断 Salesforce 股票是否值得买入

**步骤 1：基础信息（Capital IQ）**
- 公司：Salesforce, Inc. (CRM)
- 行业：企业软件 / SaaS
- 市值：约 $250B
- 主要业务：CRM 云、Sales Cloud、Service Cloud

**步骤 2：财务表现**

| 指标 | FY2022 | FY2023 | FY2024 | 行业 |
|------|--------|--------|--------|------|
| 收入 | $31.4B | $34.9B | $37.9B | - |
| YoY 增长 | - | 11.2% | 8.6% | 12% |
| 营业利润率 | 5.7% | 5.4% | 18.7% | 15% |
| 自由现金流 | $5.0B | $6.1B | $9.4B | - |

**步骤 3：一致预期**

| 指标 | NTM 共识 |
|------|---------|
| 收入 | $41.5B |
| 收入增长 | 9.5% |
| EPS | $6.50 |
| 目标价 | $320 (中位数) |
| 评级分布 | 25 Buy / 15 Hold / 2 Sell |

**步骤 4：估值倍数**

| 公司 | EV/Revenue | EV/EBITDA | P/E (NTM) | 增速 |
|------|-----------|-----------|----------|------|
| Salesforce | 6.5x | 22.0x | 28.0x | 9% |
| ServiceNow | 12.0x | 35.0x | 45.0x | 22% |
| Workday | 7.0x | 25.0x | 32.0x | 14% |
| 中位数 | 8.0x | 27.0x | 35.0x | - |

**步骤 5：投资判断**
- 增长放缓：从 25% 降至 9%
- 利润率显著提升（18.7% vs 5.4%）
- 估值合理：略低于行业平均
- **建议**：Hold（持有），等待增长加速
