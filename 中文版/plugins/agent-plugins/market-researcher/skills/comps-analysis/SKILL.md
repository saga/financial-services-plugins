# 可比公司分析

## ⚠️ 关键：数据源优先级（首先阅读）

**始终遵循此数据源层级：**

1. **优先：检查MCP数据源** — 如果S&P Kensho MCP、FactSet MCP或Daloopa MCP可用，专用于财务和交易信息
2. **如果上述MCP数据源可用，请勿使用网页搜索**
3. **仅当MCP不可用时：** 使用彭博终端、SEC EDGAR文件或其他机构来源
4. **绝不使用网页搜索作为主要数据源** — 它缺乏机构级分析所需的准确性、审计轨迹和可靠性

**为何重要：** MCP源提供经过验证的机构级数据，带有适当的引用。网页搜索结果可能过时、不准确或不可靠，不适用于金融分析。

---

## 概述

本技能教授如何构建结合运营指标、估值倍数和统计基准测试的机构级可比公司分析。输出是结构化的Excel/电子表格，通过同行比较实现明智的投资决策。

**参考材料与背景化：**

本技能目录中的`examples/comps_example.xlsx`提供了一个可比公司分析示例。使用这些示例文件时，请明智地使用：

**应该做的事：**
- 理解结构层次（各部分如何流转）
- 掌握预期的严格程度（统计深度、文档标准）
- 学习原则（清晰的标题、透明的公式、审计轨迹）

**不应该做的事：**
- 完全复制格式或指标
- 不考虑背景就复制布局
- 无论受众是谁都应用相同的视觉风格

**始终先问自己：**
1. **"你有偏好的格式吗，还是我应该调整模板风格？"**
2. **"受众是谁？"**（投资决策委员会、董事会演示、快速参考、详细备忘录）
3. **"关键问题是什么？"**（估值、增长分析、竞争定位、效率）
4. **"背景是什么？"**（并购评估、投资决策、行业基准测试、表现回顾）

**根据具体情况调整：**
- **行业背景**：大型科技巨头需要与新兴SaaS初创公司不同的指标
- **行业特定需求**：尽早添加相关指标（如科技行业的云ARR、企业客户、开发者生态系统）
- **公司熟悉度**：知名公司可能需要更少背景，更多聚焦于差异分析
- **决策类型**：并购需要与持续投资组合监控不同的侧重点

**核心原则：** 使用模板原则（清晰的结构、统计严谨性、透明的公式），但根据背景调整执行。目标是机构质量的分析，不是看起来像机构的模板。

用户提供的示例和明确偏好始终优先于默认值。

## 核心理念

**"先构建正确的结构，然后让数据讲述故事。"**

从强制思考重要事项的标题开始，输入干净的数据，构建透明的公式，让统计数据自动呈现。一个好的可比分析应该让没有构建它的人也能立即读懂。

## ⚠️ 关键：公式优于硬编码 + 逐步验证

**环境 — Office JS vs Python：**

- **如果在Excel内运行（Office加载项 / Office JS）：** 直接使用Office JS（`Excel.run(async (context) => {...})`）。通过`range.formulas = [["=E7/C7"]]`写入公式，而不是`range.values`。不需要单独的重算步骤 — Excel原生处理计算。使用`range.format.*`进行颜色/字体设置。

- **如果生成独立的.xlsx文件：** 使用Python/openpyxl。写入`cell.value = "=E7/C7"`（公式字符串）。

- 两种方式的原则相同 — 只需转换API调用。

- **Office JS合并单元格陷阱：** 不要在调用`.merge()`后对合并范围设置`.values` — Office JS仍会报告范围的原始维度，并抛出`InvalidArgument`异常。正确做法是仅将值写入左上角单元格，然后再合并并格式化整个范围：
  ```js
  ws.getRange("A1").values = [["科技 — 可比公司分析"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式，不是硬编码：**
- 每个衍生值（利润率、倍数、统计量）必须是引用输入单元格的Excel公式 — 绝不能是粘贴进去的预先计算的数字
- 使用Python/openpyxl构建工作表时：写入`cell.value = "=E7/C7"`（公式字符串），**不是**`cell.value = 0.687`（计算结果）
- 唯一应硬编码的值是原始输入数据（收入、EBITDA、股价等）— 每个值都要有带来源的单元格注释
- 原因：当输入更改时，模型必须自动更新。硬编码的利润率是等待发生的无声错误。

**与用户逐步验证：**
- 设置结构后 → 在填充数据前向用户展示标题布局
- 输入原始数据后 → 在构建公式前，向用户展示输入块并确认来源/期间
- 构建运营指标公式后 → 在转到估值前，向用户展示计算出的利润率和合理性检查
- 构建估值倍数后 → 在添加统计前，向用户展示倍数并确认它们看起来合理
- **不要**从头到尾构建完整个工作表然后才呈现 — 通过确认每个部分及早发现错误

## 第1部分：文档结构与设置

### 标题块（第1-3行）

```
第1行：[分析标题] - 可比公司分析
第2行：[公司列表及股票代码] • [公司1 (TICK1)] • [公司2 (TICK2)] • [公司3 (TICK3)]
第3行：截至[期间] | 所有数字单位为[百万/十亿美元]，每股金额和比率除外
```

**为何重要：** 立即建立背景。任何打开此文件的人都知道他们在看什么、何时创建以及如何解释数字。

### 视觉规范标准（可选 - 用户偏好和上传的模板始终覆盖）

**重要：这些仅是建议的默认值。始终优先考虑：**
1. 用户的明确格式偏好
2. 任何上传的模板文件的格式
3. 公司/团队风格指南
4. 这些默认值（仅当没有其他指导时）

**建议的字体和排版：**
- **字体家族**：Times New Roman（专业、可读、行业标准）
- **字体大小**：数据单元格11pt，标题12pt
- **粗体文本**：章节标题、公司名称、统计标签

**默认颜色和阴影 — 专业蓝/灰调色板（少即是多）：**
- **保持克制** — 仅使用蓝色和灰色。不要引入绿色、橙色、红色或多个强调色。干净的可比分析表总共使用3-4种颜色。
- **章节标题**（如"运营统计和财务指标"）：
  - 深蓝背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色粗体文本
  - 跨所有列的整行阴影
- **列标题**（如"公司"、"收入"、"利润率"）：
  - 浅蓝背景（`#D9E1F2` 或类似淡蓝色）
  - 黑色粗体文本
  - 居中对齐
- **数据行**：
  - 公司数据的白色背景
  - 公式的黑色文本；硬编码输入的蓝色文本
- **统计行**（最大值、第75百分位等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文本，左对齐标签

- **这就是整个调色板**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户模板另有说明，否则不要使用其他颜色。

**建议的格式约定：**
- **小数精度**：
  - 百分比：1位小数（12.3%）
  - 倍数：1位小数（13.5倍）
  - 美元金额：无小数，千位分隔符（69,632）
  - 利润率显示为百分比：1位小数（68.7%）
- **边框**：无边框（干净、极简外观）
- **对齐**：所有指标居中对齐，呈现干净、统一的外观
- **单元格尺寸**：所有列宽应均匀/均匀，所有行高应一致（创建干净、专业的网格）

**注意：** 如果用户提供了模板文件或指定了不同的格式，请改用那个。

## 第2部分：运营统计和财务指标

### 核心列（从这些开始）
1. **公司** — 格式一致的名称
2. **收入** — 规模指标（可以是LTM、季度或年度，取决于背景）
3. **收入增长** — 同比百分比变化
4. **毛利润** — 收入减去销售成本
5. **毛利率** — 毛利润/收入（基本盈利能力）
6. **EBITDA** — 息税折旧摊销前利润
7. **EBITDA利润率** — EBITDA/收入（运营效率）

### 可选添加项（根据行业/目的选择）
- **季度 vs LTM** — 如果季节性重要，则两者都包含
- **自由现金流** — 对于资本密集型或SaaS业务
- **FCF利润率** — 自由现金流/收入（现金生成效率）
- **净收入** — 对于成熟、盈利的公司
- **运营收入** — 对于折旧摊销变化的业务
- **资本支出指标** — 对于资产密集型行业
- **40法则** — 专门针对SaaS（增长率% + 利润率%）
- **FCF转换率** — 用于收益质量分析（高级）

### 公式示例（以第7行为例）
```excel
// 核心比率 - 这些总是计算得出的
毛利率 (F7): =E7/C7
EBITDA利润率 (H7): =G7/C7

// 可选比率 - 如果相关则包含
FCF利润率: =[FCF]/[收入]
净利润率: =[净收入]/[收入]
40法则: =[增长率%]+[FCF利润率%]
```

**黄金法则：** 每个比率都应该是[某物] / [收入] 或 [某物] / [此工作表中的某物]。保持简单。

### 统计块（公司数据之后）

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留一个空白行用于视觉分隔]
- 最大值: =MAX(B7:B9)
- 第75百分位: =QUARTILE(B7:B9,3)
- 中位数: =MEDIAN(B7:B9)
- 第25百分位: =QUARTILE(B7:B9,1)
- 最小值: =MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- 收入增长率%、毛利率%、EBITDA利润率%、EPS
- EV/收入、EV/EBITDA、P/E、股息收益率%、Beta

**不需要统计的列（规模指标）：**
- 收入、EBITDA、净收入（绝对规模因公司规模而异）
- 市值、企业价值（跨不同规模的公司不可比）

**注意：** 在公司数据和统计行之间添加一个空白行用于视觉分隔。**不要**添加"行业统计"或"估值统计"标题行。

**为何四分位重要：** 它们显示分布，不仅仅是平均值。第75百分位倍数告诉你"溢价"公司以多少倍交易。

## 第3部分：估值倍数和投资指标

### 核心估值列（从这些开始）
1. **公司** — 与运营部分相同的顺序
2. **市值** — 当前市场估值
3. **企业价值** — 市值 ± 净债务/现金
4. **EV/收入** — 每美元销售额市场支付多少
5. **EV/EBITDA** — 每美元收益市场支付多少
6. **P/E比率** — 价格相对于净收益

### 可选估值指标（根据背景选择）
- **FCF收益率** — 自由现金流/市值（用于现金焦点分析）
- **PEG比率** — P/E/增长率（用于成长型公司）
- **市净率** — 市场价值 vs. 账面价值（用于资产密集型业务）
- **ROE/ROA** — 回报指标（用于盈利能力比较）
- **收入/EBITDA CAGR** — 历史增长率（用于趋势分析）
- **资产周转率** — 收入/资产（用于运营效率）
- **债务/股权** — 杠杆（用于资本结构分析）

**关键原则：** 包含对你的行业重要的3-5个核心倍数。不要仅仅因为可以就包含每个可能的指标。

### 公式示例
```excel
// 核心倍数 - 始终包含这些
EV/收入: =[企业价值]/[LTM收入]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
P/E比率: =[市值]/[净收入]

// 可选倍数 - 如果数据可用则包含
FCF收益率: =[LTM FCF]/[市值]
PEG比率: =[P/E]/[增长率%]
```

### 交叉引用规则

**关键：** 估值倍数必须引用运营指标部分。永远不要两次输入相同的原始数据。如果收入在C7，那么EV/收入公式应该引用C7。

### 统计块

与运营部分相同的结构：每个指标的最大值、第75、中位数、第25、最小值。在公司数据和统计之间添加一个空白行用于视觉分隔。**不要**添加"估值统计"标题行。

## 第4部分：注释和方法论文档

### 必需组件

**数据来源和质量：**
- 数据来自哪里？（S&P Kensho MCP、FactSet MCP、Daloopa MCP、彭博、SEC文件）
- 涵盖什么期间？（2024年Q4、审计后数据）
- 如何验证？（与10-K/10-Q交叉核对）
- 注意：如果可用，优先使用MCP数据源（S&P Kensho、FactSet、Daloopa）以获得更好的准确性和可追溯性

**关键定义：**
- EBITDA计算方法（毛利润 + 折旧摊销，或运营收入 + 折旧摊销）
- 自由现金流公式（运营现金流 - 资本支出）
- 特殊指标解释（40法则、FCF转换率）
- 时间段定义（LTM、CAGR计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净债务）
- 使用了什么增长率？（历史CAGR、前瞻估计）
- 做了任何调整吗？（一次性项目排除、正常化利润率）

**分析框架：**
- 投资论点是什么？（云/SaaS效率）
- 哪些指标最重要？（现金生成、资本效率）
- 读者应如何解释统计数据？（四分位提供背景）

## 第5部分：选择合适的指标（决策框架）

### 从"我在回答什么问题？"开始

**"哪家公司被低估了？"**
→ 聚焦：EV/收入、EV/EBITDA、P/E、市值
→ 跳过：运营细节、增长指标

**"哪家公司最高效？"**
→ 聚焦：毛利率、EBITDA利润率、FCF利润率、资产周转率
→ 跳过：规模指标、绝对美元金额

**"哪家公司增长最快？"**
→ 聚焦：收入增长率%、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆比率

**"哪家是最好的现金生成器？"**
→ 聚焦：FCF、FCF利润率、FCF转换率、资本支出强度
→ 跳过：EBITDA、P/E比率

### 行业特定指标选择

**软件/SaaS：**
必须有：收入增长、毛利率、40法则
可选：ARR、净美元留存率、CAC回收期
跳过：资产周转率、库存指标

**制造/工业：**
必须有：EBITDA利润率、资产周转率、资本支出/收入
可选：ROA、库存周转、积压订单
跳过：40法则、SaaS指标

**金融服务：**
必须有：ROE、ROA、效率比率、P/E
可选：净息差、贷款损失准备金
跳过：毛利率、EBITDA（对银行无意义）

**零售/电商：**
必须有：收入增长、毛利率、库存周转率
可选：同店销售额、客户获取成本
跳过：重度研发或资本支出指标

### "5-10法则"

**5个运营指标** — 收入、增长、2-3个利润率/效率指标
**5个估值指标** — 市值、EV、3个倍数
**= 总共10列** — 足以讲述故事，不会多到让你失去线索

如果你有超过15个指标，你可能包含了噪音。无情地编辑。

## 第6部分：最佳实践和质量检查

### 开始之前
1. **定义同行组** — 公司必须真正可比（相似的商业模式、规模、地理位置）
2. **选择正确的期间** — LTM平滑季节性；季度显示趋势
3. **预先标准化单位** — 百万 vs. 十亿的决定影响一切
4. **映射数据源** — 知道每个数字来自哪里

### 构建时
1. **首先输入所有原始数据** — 在编写公式之前完成蓝色文本
2. **为所有硬编码输入添加单元格注释** — 右键单击单元格 → 插入注释 → 记录来源或假设

   **对于来源数据，准确引用其来源：**
   - 示例："彭博终端 - MSFT股票DES，2024-10-02访问"
   - 示例："2024年Q4 10-K文件，第42页，行项目'总收入'"
   - 示例："FactSet一致预期截至2024-10-02"

   **对于假设，解释推理：**
   - 示例："假设15% EBITDA利润率，基于同行中位数，公司未披露"
   - 示例："估计企业价值为市值 + 5000万美元净债务（来自Q3资产负债表，Q4尚未可用）"
   - 示例："前瞻P/E基于街一致预期EPS 3.45美元（12位分析师预期的平均值）"

   **为何重要**：启用审计轨迹、数据验证、假设透明度和未来更新

3. **逐行构建公式** — 在继续之前测试每个计算
4. **对标题使用绝对引用** — $C$6锁定标题行
5. **格式一致** — 百分比作为百分比，不是小数
6. **添加条件格式** — 自动高亮异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA利润率 > 净利润率（根据定义始终为真）
- **倍数合理性**：
  - EV/收入：通常0.5-20倍（因行业差异很大）
  - EV/EBITDA：通常8-25倍（跨行业相当一致）
  - P/E：通常10-50倍（取决于增长率）
- **增长-倍数相关性**：更高的增长通常意味着更高的倍数
- **规模-效率权衡**：更大的公司通常有更好的利润率（规模效益）

### 常见错误避免
❌ 在公式中混合市值和企业价值
❌ 对分子和分母使用不同的时间段（LTM vs 季度）
❌ 将数字硬编码到公式中而不是单元格引用
❌ **硬编码输入没有引用来源或解释假设的单元格注释**
❌ 当可用时，缺少到SEC文件或数据源的超链接
❌ 包含太多没有明确目的的指标
❌ 包含不可比的公司（不同的商业模式）
❌ 使用过时数据而不披露
❌ 不正确地计算百分比平均值（应该是中位数）

## 第10部分：红旗和警告信号

### 数据质量问题
🚩 不一致的时间段（混合季度和年度）
🚩 缺失数据没有解释
🚩 数据源之间的显著差异（>10%方差）

### 估值红旗
🚩 负EBITDA公司用EBITDA倍数估值（改用收入倍数）
🚩 P/E比率 >100倍没有超增长故事
🚩 行业利润率不合理

### 可比性问题
🚩 不同的财年结束（导致时间问题）
🚩 混合纯业务和集团公司
🚩 商业模式实质不同却标记为"可比"

### 基本Excel公式
```excel
// 统计函数
=AVERAGE(范围)          // 简单平均值
=MEDIAN(范围)           // 中间值
=QUARTILE(范围, 1)      // 第25百分位
=QUARTILE(范围, 3)      // 第75百分位
=MAX(范围)              // 最大值
=MIN(范围)              // 最小值
=STDEV.P(范围)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除零错误

// 跨表引用
='Sheet1'!B7             // 引用另一张表
=VLOOKUP(A7, Table1, 2)  // 从数据表查找
=INDEX(MATCH())          // 高级查找

// 格式化
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千位分隔符
```

### 常见比率公式
```excel
毛利率 = 毛利润 / 收入
EBITDA利润率 = EBITDA / 收入
FCF利润率 = 自由现金流 / 收入
FCF转换率 = FCF / 运营现金流
ROE = 净收入 / 股东权益
ROA = 净收入 / 总资产
资产周转率 = 收入 / 总资产
债务/股权 = 总债务 / 股东权益
```

## 关键原则总结

1. **结构驱动洞察** — 正确的标题强制正确的思考
2. **少即是多** — 重要的5-10个指标击败不重要的20个
3. **为问题选择指标** — 估值分析 ≠ 效率分析
4. **统计显示模式** — 中位数/四分位比平均值揭示更多
5. **透明度击败复杂性** — 每个人都理解的简单公式
6. **可比性是王道** — 排除比强制使用不好的可比更好
7. **记录你的选择** — 在注释部分解释哪些指标及原因

## Appendix: 金融背景知识

这份文件是"可比公司分析（Comps Analysis）"技能的详细说明。在 Market Researcher 的上下文中，Comps Analysis 是行业研究的基础——当你研究一个新行业时，必须搞清楚"这个行业里的公司大概值多少钱"。

---

### 1. Market Researcher 的 Comps 特殊之处

不同于投行做 Comps 是为了给一家公司定价，Market Researcher 做 Comps 是为了回答"这个行业整体是贵还是便宜"。

| 投行 Comps | 市场研究 Comps |
|-----------|-------------|
| 为 1 家公司定价 | 为整个行业"扫价" |
| 精确到目标价 | 粗略到"行业平均 PE 15x，最高 25x，最低 5x" |
| 可比公司需极精准 | 可比公司可以"大范围同类" |

---

### 2. 行业 Comps 的"三分法"

| 分位 | 含义 |
|------|------|
| 中位数 | 行业"正常"估值 |
| 75 分位 | "贵的"公司在什么水平 |
| 25 分位 | "便宜的"公司在什么水平 |

---

### 给小白的一句话

> Market Researcher 的 Comps 就是行业的"价格标签"——你不是要为某一家公司精确估值，而是告诉基金经理"这个行业平均 PE 15 倍，最贵的 25 倍，最便宜的 5 倍——所以现在整体是偏贵、正常还是历史上最便宜的时候"。这帮助基金经理在"要不要投这个行业"层面做判断，而不是"具体买哪一只"。
