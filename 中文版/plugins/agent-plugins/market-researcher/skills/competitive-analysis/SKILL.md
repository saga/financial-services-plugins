# 竞争格局地图

构建完整的竞争分析演示文稿。这是一个两阶段流程：先收集需求并获得大纲批准，然后构建。

## 环境检查

本技能在PowerPoint加载项和聊天中均可工作。开始前识别你所在的环境 — 机制不同，工作流不变：

- **加载项** — 演示文稿实时打开；直接在其中构建幻灯片。
- **聊天** — 生成 `.pptx` 文件（或构建到用户上传的文件中）。

以下内容在两者中都适用。

## 阶段1 — 确定分析范围

竞争分析对不同的人意味着不同的东西。在进行任何研究或幻灯片构建之前，使用 `ask_user_question` 确定他们真正想要什么。不要猜测 — 一个20张幻灯片的同行基准演示文稿和一个5张幻灯片的市场地图都是"竞争分析"，但形态完全不同。

如果可以，在一轮中收集（该工具最多接受4个问题）：

- **范围** — 单个目标公司及其竞争对手？还是多公司并排比较，没有主角？
- **竞争对手集合** — 哪些公司在范围内？如果用户命名了它们，请精确使用。如果他们说"常规嫌疑犯"，提出建议并确认。
- **受众和深度** — 给已经在该领域的人快速阅读，还是完整的入门指南？这决定了你是否需要市场规模、行业经济学和历史 — 或者可以跳到比较。
- **投资背景** — 他们需要牛市/基准/熊市情境和里程碑吗？这是下面的步骤9；如果这是战略审查而不是投资论点，请跳过。

如果他们上传了包含竞争对手数据的Excel/CSV，在开始提取数字之前，确认哪些列映射到哪些指标。源文件保真度很重要：完全按照给定的值使用，不要重新计算或重新舍入。

## 阶段2 — 大纲、批准，然后构建

**在大纲获得批准之前不要创建幻灯片。** 提议幻灯片标题和一行内容注释，呈现给用户，获得肯定。竞争演示文稿是10-20张内容 interlocking 的幻灯片 — 因为第4张幻灯片错误而重建代价高昂。大纲是廉价的迭代点。

提议大纲时，`ask_user_question` 很适合结构决策：哪种定位可视化（2×2矩阵 / 雷达图 / 层级图 — 下面的步骤5），如何分组竞争对手（按商业模式 / 细分 / 姿态 — 步骤4）。这些是用户可能有意见的品味选择。

## 标准 — 全程应用

### 提示保真度

当用户指定某些内容时，那是要求，不是建议：
- **幻灯片标题和章节名称** — 精确措辞。如果他们说"概览和竞争范围"，不要换成"FY2024竞争格局。"
- **图表 vs. 表格** — 不可互换。"嵌入式图表"意味着带有数据标签的真实图表对象，不是格式化的表格。
- **完整数据系列** — 如果他们列出7个竞争对手，包含所有7个。如果他们显示2015-2025，包含每一年。
- **精确值和比率** — "超过DoorDash 4:1，Lyft 8:1"意味着那些比率，不是"约7.6倍Lyft。"

### 来源质量，当来源冲突时

1. 10-K / 年报（已审计）
2. 财报电话会议 / 投资者演示（管理层评论）
3. 卖方研究（分析师估计，对私营公司规模估算有用）
4. 行业报告（麦肯锡、高德纳 — 市场规模、趋势）
5. 新闻（仅近期动态；对照主要来源验证）

### 数据可比性

- 所有竞争对手指标来自同一财年；明确标记例外（"FY24" vs "H1 2024"）
- 跨竞争对手使用相同的指标定义
- 国际公司转换为美元；注明汇率和日期
- 缺失数据显示为"-"或"N/A"，并带"[E]"标记表示估计 — 绝不要留空
- 每个数字都有引用："[公司] [文档] ([日期])"

### 设计

- **幻灯片标题是见解，不是标签。** "规模领导者正从利基玩家拉开距离" — 不是"竞争分析。"
- **里程碑是量化的。** "利润率低于40%" — 不是"利润率下降。"
- **评级显示实际值。** "●●● $160B" — 不是只有"●●●。"
- **图表是真实图表对象** — 不是伪装成图表外观的文本表格。

**排版** — 明确设置，不要依赖默认值：
- 幻灯片标题：28-32pt 粗体
- 章节标题：18-20pt 粗体
- 正文：14-16pt（绝不低于14pt）
- 表格文字：14pt
- 来源/脚注：14pt，灰色
- 相同元素类型 = 全程相同大小

**图表：**
- 图例在图表边界内，不要浮动在绘图区上方
- 饼图右侧图例（≤6片），折线图/条形图底部图例（≤4系列）
- 超过6个系列 → 拆分为多个图表或使用表格
- 饼图在切片上显示百分比，不仅仅是图例

**表格：**
- 浅灰色标题行，粗体
- 数字右对齐，文本左对齐
- 足够的单元格内边距，使文本不接触边框

**颜色：** 最多2-3种颜色。柔和 — 海军蓝、灰色、一个强调色。全程相同颜色含义。

### 什么是严格的 vs. 灵活的

| 总是 | 视情况而定 |
|------|------------|
| 用户指定时的精确标题/章节 | 他们未指定时的创意标题 |
| 用户说图表时的图表；说表格时的表格 | 未指定时的可视化类型 |
| 他们列出的每个竞争对手/数据点 | 未指定时的竞争对手数量 |
| 指定时的精确值 | 未指定时的舍入 |
| 标题适合而不溢出 | 竞争对手类别的数量 |
| 无重叠元素 | 比较哪些维度 |

## 分析工作流

### 步骤0 — 行业定义指标

在任何其他事情之前：这个行业实际运行的3-5个指标是什么？在每個竞争对手中一致使用这些。

| 行业 | 关键指标 |
|------|----------|
| SaaS | ARR、净留存率（NRR）、CAC回收期、LTV/CAC、40法则 |
| 支付 | GPV（总支付额）、抽成率（take rate）、附加率（attach rate）、交易利润率 |
| 市场平台 | GMV（总交易额）、抽成率、买方/卖方比率、复购率 |
| 零售 | 同店销售额、库存周转率、每平方英尺销售额 |
| 物流 | 运量、单位成本、准时交付率、产能利用率 |

行业未列出 — 选择投资者和运营商基准测试的指标。

### 步骤1 — 市场背景

规模、增长、驱动因素、逆风。带来源。

正确："2024年嵌入式支付市场规模800-1000亿美元，增长20-25% CAGR（麦肯锡2024）"
错误："市场庞大且快速增长"

### 步骤2 — 行业经济学

映射价值流动。方法取决于行业结构：
- **垂直结构** — 价值链层级，每层的典型利润率
- **平台/网络** — 生态系统参与者，它们之间的价值流动
- **分散** — 整合动态，按规模的利润率差异

### 步骤3 — 目标公司概况

```
| 指标 | 值 |
|------|-----|
| 收入 | $4.96B |
| 增长 | +26% 同比 |
| 毛利率 | 45% |
| 盈利能力 | $373M 调整后EBITDA |
| 客户 | 134K |
| 留存率 | 92% |
| 市场份额 | ~15% |
```

多细分公司添加细分分解：

```
| 细分 | 收入 | 收入同比 | 收入占比 | EBITDA | EBITDA同比 | 利润率 |
|------|------|----------|----------|---------|------------|--------|
| 细分A | $25.1B | +26% | 57% | $6.5B | +31% | 26% |
| 细分B | $13.8B | +31% | 31% | $2.5B | +64% | 18% |
| 细分C | $5.1B | -2% | 12% | -$74M | -16% | -1% |
| 总计 | $44.0B | +18% | 100% | $6.5B* | - | 15% |
```
*如适用，注明公司成本

### 步骤4 — 竞争对手映射

按适合的任何视角分组（如果用户未指定，这是个好的 `ask_user_question` 决策）：
- 按商业模式 — 平台 / 垂直 / 水平
- 按细分 — 企业 / 中小企业 / 消费者
- 按姿态 — 直接 / 相邻 / 新兴
- 按起源 — 在位者 / 颠覆者 / 新进入者

### 步骤5 — 定位可视化

| 类型 | 何时 |
|------|------|
| 2×2矩阵 | 两个主导竞争因素 |
| 雷达图/蜘蛛图 | 多因素比较 |
| 层级图 | 自然聚类为战略群体 |
| 价值链地图 | 垂直行业 |
| 生态系统地图 | 平台市场 |

参见 `references/frameworks.md` 了解按行业的2×2轴对。

### 步骤6 — 竞争对手深度研究

每个竞争对手两张表格。

**指标：**
```
| 指标 | 值 |
|------|-----|
| 收入 | $X.XB |
| 增长 | +XX% 同比 |
| 毛利率 | XX% |
| 市值 | $X.XB |
| 盈利能力 | $XXXM EBITDA |
| 客户 | XXK |
| 留存率 | XX% |
| 市场份额 | ~XX% |
```

**定性：**
```
| 类别 | 评估 |
|------|------|
| 业务 | 他们做什么（1句话） |
| 优势 | 2-3个要点 |
| 劣势 | 2-3个要点 |
| 战略 | 当前优先事项 |
```

### 步骤7 — 比较分析

```
| 维度 | 公司A | 公司B | 公司C |
|------|-------|-------|-------|
| 规模 | ●●● $160B | ●●○ $45B | ●○○ $8B |
| 增长 | ●●○ +26% | ●●● +35% | ●●○ +22% |
| 利润率 | ●●○ 7.5% | ●○○ 3.2% | ●●● 15% |
```

### 步骤8 — 战略背景

并购交易（倍数、理由）、合作伙伴关系趋势、融资模式、监管发展。参见 `references/schemas.md` 了解并购交易表格格式。

### 步骤9 — 综合

**护城河评估** — 对每个竞争对手评级 强 / 中等 / 弱：

| 护城河 | 评估什么 |
|--------|----------|
| 网络效应 | 用户/供应商飞轮强度；跨边 vs. 同边 |
| 转换成本 | 技术集成深度、合同锁定、行为习惯 |
| 规模经济 | 产量下的单位成本优势；最小有效规模 |
| 无形资产 | 品牌、专有数据、监管牌照、专利 |

**必需的合成元素：**
- 持久优势（难以复制）— 映射到护城河类别
- 结构性脆弱性（难以修复）
- 当前状态 vs. 轨迹

**用于投资背景**（如果阶段1范围确定说否，则跳过）：

```
| 情境 | 概率 | 关键驱动因素 |
|------|------|--------------|
| 牛市 | 30% | 市场份额增长，利润率扩张 |
| 基准 | 50% | 当前轨迹继续 |
| 熊市 | 20% | 竞争压力，利润率压缩 |
```

## 质量清单

完成前：

**提示保真度**
- 幻灯片标题逐字匹配用户指定的内容
- 他们说图表的地方用图表；说表格的地方用表格
- 他们列出的每个竞争对手/年份/数据点都存在
- 精确值和格式如指定

**数据一致性**
- 源文件值直接提取，不重新计算
- 相同指标在出现的每张幻灯片上显示相同值
- 与源相同的十进制精度

**布局**
- 标题适合而不溢出
- 无重叠元素
- 所有文本在容器内，无裁剪

**内容**
- 每个数字都有引用
- 所有指标来自同一财年（或标记）
- 幻灯片标题陈述见解，不是主题
- 图表是真实图表对象

在每张幻灯片上运行标准视觉验证检查 — 这能捕捉到你回顾XML时不会出现的重叠、溢出和低对比度文本。

本文件涉及的通用金融概念已整理至知识库，请按需查阅：

- **SaaS与科技公司**：ARR / NRR / Rule of 40 / CAC Payback / LTV/CAC → [SaaS与科技公司.md](../../../../金融知识学习/SaaS与科技公司.md)
- **公司战略**：护城河 / 波特五力 / 竞争优势 → [公司战略.md](../../../../金融知识学习/公司战略.md)

**本文件专属**：
- **竞争分析（Competitive Analysis）**：系统评估行业内竞争动态，了解市场定位、识别机会和风险。
- **2×2矩阵**：将竞争对手按两个关键维度分类的图表。
- **飞轮（Flywheel）**：自我强化的增长循环。
- **在位者/颠覆者（Incumbent/Disruptor）**：市场主导玩家 vs 通过创新挑战在位者的新公司。
- **抽成率（Take Rate）**：公司从交易额中收取的费用百分比。
- **GPV/GMV**：总支付额/总交易额。
