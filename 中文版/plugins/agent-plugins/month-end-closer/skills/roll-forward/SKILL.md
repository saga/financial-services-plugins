# 滚调表

给定账户（或账户组）、实体和期间，生成期初到期末的滚调表。

## 结构

```
期初余额（上期结账包）      X
  + 新增 / 新活动                    A
  + 本期记账的应计                 B
  − 前期应计的回转                (C)
  − 付款 / 结算                     (D)
  ± 重分类 / 调整                     E
  ± 外汇折算                              F
期末余额（期末总账余额）           Y
```

## 结平每行

- **期初** — 上期结账包，或上期期末的总账余额。
- **每个活动行** — 通过internal-gl MCP的GL查询（账户 + 日期范围 + 日记账来源筛选）。引用查询。
- **期末** — 期末日期的总账余额。

时间表**必须结平**：`X + A + B − C − D + E + F = Y`。如果不平衡，**量化每期的差距并追溯哪里断开** — 在修复之前其他都不重要。

## 输出

滚调表，带"结平至"列，为每一行引用GL查询或文档，加上结平检查（通过/失败及任何未解释的差额）。

## Appendix: 金融背景知识

这份文件是"滚调表（Roll Forward）"技能的详细说明。在企业财务中，Roll Forward 是最常用的对账工具——它把期初余额到期末余额的"路径"一步步还原出来。

---

### 1. 什么是 Roll Forward？

**类比**：你在追踪银行卡余额。
- 月初余额：10,000
- 这个月收到工资：+5,000
- 花掉房租：-2,000
- 花掉吃饭：-1,500
- 月末余额应该 = 10,000 + 5,000 - 2,000 - 1,500 = 11,500

如果银行 App 显示 11,200 而不是 11,500——那你就要"Roll Forward"逐笔核对，找出差了的 300 去哪了。

---

### 2. Roll Forward 的"标准公式"

```
期初余额（期初）
+ 本期增加（收入/折旧/应计调整）
- 本期减少（支付/冲回/摊销）
= 期末余额（期末）
```

核心原则：期末余额必须能追溯到总账的数字，不能"凭空出现"。

---

### 给小白的一句话

> Roll Forward 就像"追踪快递包裹"——从发货（期初余额）到收货（期末余额），中间的每一步（谁经手的、在哪里分拣的、什么时候派送的）都记录在案。在会计中，任何科目都可以做 Roll Forward：固定资产、应收账款、应计费用、递延收入……核心公式永远不变：期初 + 增加 - 减少 = 期末。期末余额必须跟总账系统里的数字完全一致。
