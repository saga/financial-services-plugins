# 波动说明

给定本期实际数、前期实际数和相同范围的预算，生成说明表。

## 阈值

如果**以下任一**为真，标记一行需要说明：
- 绝对波动 ≥ 公司的材料性阈值（使用提供的值；默认本期行项目的5%或固定下限，取较大者）
- 该行在"始终说明"列表中（收入、人力成本、现金）

## 对每个标记的行

| 列 | 内容 |
|------|------|
| **行项目** | 账户或标题 |
| **当期 / 前期 / 预算** | 三个值 |
| **vs. 前期差异** 和 **vs. 预算差异** | 金额和% |
| **驱动因素** | 一句话解释变动来自底层活动 — 不是数字的重复陈述 |

驱动因素解释*为什么*，不是*什么*："云支出增加120万美元，用于5月发布的GPU预订" — 不是"云支出增加120万美元（18%）"。

## 溯源驱动因素

通过internal-gl MCP查看行背后的活动（日记账来源分解、供应商组合、人力数变动、销量 × 费率）。如果数据不清楚驱动因素，写"驱动因素不明 — 标记供控制器审查"而不是编造。

## 输出

说明表加上简短叙述（3-5句话）总结本期最大变动。

## Appendix: 金融背景知识

### 什么是波动分析（Variance Analysis）？

波动分析是比较实际结果与前期或预算的过程，帮助：
- 识别异常变动
- 解释业绩差异
- 改进未来预算
- 检测错误或欺诈

### 关键术语解释

**波动（Variance）**：实际结果与比较基准（前期、预算）的差异。可以是：
- **有利（Favorable）**：收入高于预算或费用低于预算
- **不利（Unfavorable）**：收入低于预算或费用高于预算

**材料性阈值（Materiality Threshold）**：需要解释的波动下限。通常：
- 百分比法：本期行项目的5%
- 固定下限：如10,000美元
- 取两者中较大者

**绝对波动（Absolute Variance）**：差异的金额绝对值，如收入增加50万美元。

**相对波动（Relative Variance）**：差异的百分比，如收入增加10%。

**前期（Prior Period）**：上期实际结果，用于比较趋势。

**预算（Budget）**：管理层批准的预期业绩目标。

**底层活动（Underlying Activity）**：驱动数字变动的实际业务活动，如：
- 销量变化
- 价格调整
- 新员工入职
- 一次性项目（重组费用、诉讼和解）

**日记账来源分解（Journal-source Breakdown）**：按日记账分录的来源系统或类型分解账户余额，帮助识别变动驱动因素。

**供应商组合（Vendor Mix）**：不同供应商在总采购中的占比变化，可能解释成本波动。

**人力数变动（Headcount Delta）**：员工数量变化，驱动人力成本波动。

**销量 × 费率（Volume × Rate）**：分解收入或成本变动的方法：
- **销量影响**：销售更多/更少单位
- **费率影响**：单位价格/成本变化

**控制器审查（Controller Review）**：财务主管审查波动说明，确保解释合理、完整。

**始终说明列表（Always Comment List）**：无论波动大小，都需要解释的关键账户，如：
- 收入（关键业绩指标）
- 人力成本（通常占大比例）
- 现金（核心流动性）

**internal-gl MCP**：通过Model Context Protocol访问公司内部总账系统的接口，用于提取交易明细和日记账来源。
