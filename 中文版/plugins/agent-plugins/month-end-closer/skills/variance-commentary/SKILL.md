# 波动说明

给定本期实际数、前期实际数和相同范围的预算，生成说明表。

## 阈值

如果**以下任一**为真，标记一行需要说明：
- 绝对波动 ≥ 公司的材料性阈值（使用提供的值；默认本期行项目的5%或固定下限，取较大者）
- 该行在"始终说明"列表中（收入、人力成本、现金）

## 对每个标记的行

| 列 | 内容 |
|------|------|
| **行项目** | 账户或标题 |
| **当期 / 前期 / 预算** | 三个值 |
| **vs. 前期差异** 和 **vs. 预算差异** | 金额和% |
| **驱动因素** | 一句话解释变动来自底层活动 — 不是数字的重复陈述 |

驱动因素解释*为什么*，不是*什么*："云支出增加120万美元，用于5月发布的GPU预订" — 不是"云支出增加120万美元（18%）"。

## 溯源驱动因素

通过internal-gl MCP查看行背后的活动（日记账来源分解、供应商组合、人力数变动、销量 × 费率）。如果数据不清楚驱动因素，写"驱动因素不明 — 标记供控制器审查"而不是编造。

## 输出

说明表加上简短叙述（3-5句话）总结本期最大变动。

## Appendix: 金融背景知识

这份文件是"差异说明（Variance Commentary）"技能的详细说明。在企业财务中，管理层最怕看到报表上"某个数字大幅变化但没有解释"——这就是差异说明的价值：为每一个数字波动找到背后的故事。

---

### 1. 什么是差异说明？

**类比**：你是公司 CFO 的"翻译官"。CFO 看到报表问："为什么这个月营销费比上月多了 30%？"你说："因为 11 月是双十一，我们多花了 200 万在直播带货上。"——这就是差异说明：把数字变成故事。

---

### 2. 差异分析的"三层追问"

| 层 | 问题 | 例子 |
|----|------|------|
| 数字 | 差了多少？ | 营销费多了 30% |
| 原因 | 为什么？ | 双十一活动 |
| 影响 | 是好是坏？持续性的吗？ | 一次性的，12 月回归正常 |

---

### 3. 差异说明的"黄金模板"

```
[指标名称] 本月 $X vs 上月 $Y，变动 $Z (Z%)。
主要驱动：1) 驱动因素一（贡献 X%），2) 驱动因素二（贡献 Y%）。
判断：一次性的还是结构性的？
```

---

### 给小白的一句话

> Variance Commentary 就是给数字波动"写故事"——管理层看到报表上某个数字突然暴涨或暴跌，你要像侦探一样找出"为什么"和"还好吗"。不是简单说"涨了 30%"——而是要解释"因为双十一花了 200 万营销费，这是一次性的，下个月会回来"。三步走：差了多少 → 为什么 → 是好事还是坏事。
