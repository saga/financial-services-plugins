# 波动说明

给定本期实际数、前期实际数和相同范围的预算，生成说明表。

## 阈值

如果**以下任一**为真，标记一行需要说明：
- 绝对波动 ≥ 公司的材料性阈值（使用提供的值；默认本期行项目的5%或固定下限，取较大者）
- 该行在"始终说明"列表中（收入、人力成本、现金）

## 对每个标记的行

| 列 | 内容 |
|------|------|
| **行项目** | 账户或标题 |
| **当期 / 前期 / 预算** | 三个值 |
| **vs. 前期差异** 和 **vs. 预算差异** | 金额和% |
| **驱动因素** | 一句话解释变动来自底层活动 — 不是数字的重复陈述 |

驱动因素解释*为什么*，不是*什么*："云支出增加120万美元，用于5月发布的GPU预订" — 不是"云支出增加120万美元（18%）"。

## 溯源驱动因素

通过internal-gl MCP查看行背后的活动（日记账来源分解、供应商组合、人力数变动、销量 × 费率）。如果数据不清楚驱动因素，写"驱动因素不明 — 标记供控制器审查"而不是编造。

## 输出

说明表加上简短叙述（3-5句话）总结本期最大变动。

## Appendix: 金融背景知识

本文件涉及的通用金融概念已整理至知识库，请按需查阅：

- **财务会计**：三大报表 / GAAP vs 调整后 / 应计 / 滚调表 / 月末结账 / 波动分析 → [财务会计.md](../../../../金融知识学习/财务会计.md)

**本文件专属**：
- **材料性阈值**：需解释的波动下限。通常取百分比法（5%）和固定下限（如$10,000）中较大者。
- **日记账来源分解（Journal-source Breakdown）**：按日记账来源系统或类型分解账户余额，识别变动驱动因素。
- **始终说明列表（Always Comment List）**：无论波动大小都需解释的关键账户（如收入、人力成本、现金）。
- **internal-gl MCP**：通过 MCP 访问公司内部总账系统的接口，用于提取交易明细和日记账来源。
