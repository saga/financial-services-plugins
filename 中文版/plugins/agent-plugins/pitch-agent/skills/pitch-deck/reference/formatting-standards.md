## 视觉层次和布局

### 框和部分布局

幻灯片布局根据内容需求和模板设计而变化。常见元素包括：
- 带标题和副标题的页眉部分
- 带标签侧边栏的内容框
- 用于结构化数据的表格
- 用于可视化数据表示的图表
- 幻灯片底部的脚注栏

具体布局应遵循提供的模板。常见内容类型及其典型结构：
- **市场定义幻灯片**：带项目符号内容的标签框 + 评论部分
- **TAM/规模幻灯片**：指标标注 + 数据表 + 关键要点
- **竞争分析**：比较表或矩阵
- **财务摘要**：带支持数据表的图表

### 对齐原则

**平行部分的垂直对齐：**

垂直堆叠的框应具有一致的：
- 左边距位置
- 项目符号缩进
- 文本起始位置
- 框宽度

水平相邻的框应具有一致的：
- 顶部位置
- 高度（内容允许的情况下）
- 内部填充

## 文本格式

### 项目符号结构

避免无结构的文本堆积。将内容分解为易于扫描的项目符号。

**示例正确结构：**
```
✓ 消费者移动和网络语言学习应用
   (Duolingo, Babbel, Memrise, Busuu)
✓ B2B企业语言培训平台
   (goFLUENT, Speexx, Learnship)
✓ 在线辅导市场
   (italki, Preply, Cambly)
```

**示例错误结构（文本堆积）：**
```
Consumer mobile/web apps (Duolingo, Babbel, Memrise, Busuu)
B2B enterprise platforms (Speexx, Rosetta Stone Enterprise)
Online tutoring marketplaces (Preply, italki, Cambly)
```

### 项目符号符号指南

| 上下文 | 符号 | 用法 |
|--------|------|------|
| 包含/正面 | ✓ (勾选标记) | 范围内的项目，存在的功能 |
| 排除/负面 | × (叉号) | 范围外的项目，缺失的功能 |
| 中性列表 | • (项目符号) | 一般枚举，评论 |
| 编号序列 | 1. 2. 3. | 流程步骤，排名 |
| 子项目符号 | ‣ 或 – | 主项目符号下的次要要点 |

根据模板的现有约定调整符号使用。

### 项目符号一致性

框/部分内的所有项目符号应具有相同的格式：
- 整个框中使用相同的项目符号符号（除非有意区分）
- 所有主项目符号使用相同的缩进级别
- 相同的项目符号大小
- 项目符号和文本之间的间距相同
- 同一级别的所有项目符号文本使用相同的字体大小

### 字体大小指南

这些是典型范围 - 根据模板规格进行调整：

| 元素 | 典型大小（pt） | 样式 |
|------|----------------|------|
| 幻灯片标题 | 40-48 | 粗体 |
| 副标题/定义 | 18-22 | 粗体 |
| 部分标题 | 14-16 | 常规 |
| 正文/项目符号 | 12-14 | 常规 |
| 表格标题 | 10-12 | 粗体 |
| 表格正文 | 9-11 | 常规 |
| 脚注 | 8-9 | 斜体 |

### 文本密度指南

- **每个内容框最多 6-7 个项目符号**（根据空间调整）
- **每个项目符号最多 2 行**
- **括号示例**在同一行或缩进在下方
- **避免孤立单词** - 调整换行以避免新行上的单个单词

## 表格创建

### 重要：使用实际表格对象

**表格必须是实际的表格对象，而不是带制表符间距的文本。**

带制表符的文本永远无法正确对齐，看起来不专业。始终创建正确的表格对象。

### 表格结构指南

1. **列对齐**：
   - 文本列：左对齐（标题和内容）
   - 数字列：居中对齐或右对齐
   - 标题应与其列内容对齐

2. **标题行**：
   - 粗体文本
   - 阴影背景（使用模板的品牌颜色）
   - 对比度高的文本颜色以提高可读性
   - 对齐方式与列内容对齐方式匹配

3. **交替行**（可选）：
   - 交替行上的浅色阴影提高可读性

4. **汇总/总计行**：
   - 粗体文本
   - 较粗的顶部边框（分隔线）
   - 明显的背景阴影

5. **表格宽度**：
   - 填充指定部分的宽度
   - 避免表格悬浮在空白区域

### 有关 XML 实现模式，请参见 [`xml-reference.md`](xml-reference.md#table-implementation)

## 图表和图片处理

### 从 Excel 粘贴图表

从 Excel 粘贴图表时：

1. **仅粘贴图表** - 不包括源数据表
2. **调整大小以填充指定区域** - 图表不应显示为小缩略图
3. **保持纵横比** - 不扭曲图表
4. **验证可读性** - 轴标签、图例、数据标签必须清晰可读

### 从 Excel 粘贴表格

从 Excel 粘贴表格时：

1. **仅粘贴格式化表格** - 排除任何源数据或计算
2. **调整大小以填充指定区域** - 表格应占据其整个部分
3. **验证列宽** - 调整以使文本不被截断
4. **检查格式保留** - 颜色、边框、字体可能需要调整

### 大小指南

**最小尺寸原则：**
- 图表：应占据其指定区域的很大一部分
- 表格：完全填充指定部分的宽度
- 图片：根据上下文适当调整大小，绝不是缩略图大小

**尺寸过小的视觉指示器（避免这些）：**
- 图表只占可用空间的一小部分
- 文本标签不可读
- 视觉元素周围有大面积空白区域
- 视觉元素显示为"缩略图"

### 正确的尺寸调整工作流程

1. 确定目标区域尺寸
2. 粘贴图表/表格
3. 立即调整大小以填充目标区域
4. 验证所有文本保持可读
5. 根据需要调整内部元素（图例位置、轴标签）

## 数据可视化

### 关键指标显示

显示关键指标（例如 TAM、CAGR、预测）时，考虑显示值之间的关系，而不是静态列出它们：

- **视觉流程指示器**：显示进度的形状（箭头、V形、连接器）
- **大小层次**：主要指标使用较大字体，标签使用较小字体
- **空间排列**：定位元素以显示逻辑流程

### 箭头和流程指示器

如果使用箭头或流程指示器：
- 使用 PowerPoint 形状对象，而不是文本字符
- 在最终演示文稿中不使用基于文本的箭头（→, ⟹）
- 使用 PowerPoint 的形状工具或通过 XML 形状元素创建箭头

**有关 XML 实现，请参见 [`xml-reference.md`](xml-reference.md#arrow-shapes)**

## 字体一致性

### 跨框字体一致性

处于同一层次级别的所有文本框应使用相同的字体大小。

**应匹配的同级别框：**

| 框类型 | 应与以下匹配 |
|--------|--------------|
| "包含的细分市场"内容 | "排除的细分市场"内容 |
| "定义"内容 | "范围理由"内容 |
| 左列项目符号 | 右列项目符号 |
| 所有标签框 | 彼此 |
| 所有部分标题 | 彼此 |

### 验证流程

1. 识别处于同一层次级别的所有文本框
2. 检查每个框的字体大小
3. 如果任何框不同，调整所有框以匹配
4. 如果内容适合，默认使用较大尺寸；否则一致使用较小尺寸

**例外**：子项目符号或次要文本可以使用比主项目符号更小的字体，但这必须在所有框中保持一致。

## 模板适配

这些标准应根据所使用的特定模板进行调整：

1. **颜色**：使用模板的品牌颜色，而不是指定特定颜色
2. **字体**：使用模板的字体系列
3. **间距**：匹配模板的现有间距约定
4. **布局**：遵循模板的部分结构

无论模板如何，保持不变的关键原则：
- 文本必须在其背景上可读
- 表格必须是实际的表格对象
- 内容应适当填充可用空间
- 平行元素的格式应一致
- 图表/图片应适当调整大小

## 金融术语解释

### 1. 演示文稿设计（Presentation Design）

**定义**：演示文稿设计是指创建专业、清晰、有说服力的演示文稿的过程。

**关键要素**：
- **视觉层次**：引导观众注意力的元素排列
- **布局**：幻灯片上元素的组织方式
- **一致性**：整个演示文稿中格式的统一
- **可读性**：确保内容易于阅读和理解

**设计原则**：
- 简洁性：避免信息过载
- 视觉吸引力：使用适当的颜色和字体
- 逻辑性：清晰的信息流程

### 2. TAM（总体可寻址市场）

**定义**：TAM（Total Addressable Market）是指公司产品或服务可以服务的最大市场规模。

**计算方法**：
- 基于市场规模和目标客户数量
- 考虑产品定价和市场渗透率

**重要性**：
- 评估市场机会大小
- 支持商业计划和融资
- 帮助战略规划

### 3. 数据可视化（Data Visualization）

**定义**：数据可视化是将数据转换为视觉表示形式的过程，以便更容易理解和分析。

**类型**：
- **图表**：条形图、折线图、饼图等
- **表格**：结构化数据展示
- **信息图**：结合文字和图形的可视化

**原则**：
- 清晰性：避免不必要的复杂性
- 准确性：正确传达数据信息
- 美观性：视觉上吸引人

### 4. 视觉层次（Visual Hierarchy）

**定义**：视觉层次是指设计元素的排列方式，引导观众按照预期顺序查看内容。

**创建方法**：
- **大小**：较大的元素吸引更多注意力
- **颜色**：对比色突出重要元素
- **位置**：页面顶部和中心位置更显眼
- **字体粗细**：粗体文本更突出

**目的**：
- 引导观众注意力
- 提高信息吸收效率
- 增强演示文稿的逻辑性

### 5. 模板适配（Template Adaptation）

**定义**：模板适配是指根据特定模板的设计规范调整内容格式的过程。

**适配要点**：
- **颜色方案**：使用模板的品牌颜色
- **字体**：匹配模板的字体风格
- **布局**：遵循模板的结构
- **风格**：保持与模板一致的视觉风格

**重要性**：
- 保持专业性
- 确保品牌一致性
- 提高效率

### 6. 图表创建（Chart Creation）

**定义**：图表创建是指使用软件工具生成数据可视化图表的过程。

**步骤**：
1. 确定数据和目标
2. 选择合适的图表类型
3. 设计图表布局
4. 添加标签和注释
5. 优化视觉效果

**常见图表类型**：
- **条形图**：比较不同类别
- **折线图**：显示趋势和变化
- **饼图**：展示比例关系
- **散点图**：显示变量之间的关系

### 7. 表格设计（Table Design）

**定义**：表格设计是指创建清晰、易读的数据表格的过程。

**原则**：
- **对齐**：数字右对齐，文本左对齐
- **可读性**：使用适当的字体大小
- **视觉层次**：标题行和汇总行突出显示
- **简洁性**：避免不必要的边框和颜色

**元素**：
- **标题行**：粗体和背景色
- **数据行**：交替行颜色提高可读性
- **汇总行**：突出显示总计

### 8. 字体选择（Font Selection）

**定义**：字体选择是指为演示文稿选择合适的字体类型和大小的过程。

**考虑因素**：
- **可读性**：确保文本清晰可读
- **专业性**：选择适合商务场合的字体
- **一致性**：保持整个演示文稿的字体统一
- **品牌一致性**：如果有品牌指南，遵循品牌字体

**常用字体**：
- **标题**：Arial Black, Helvetica Bold
- **正文**：Arial, Helvetica, Calibri
- **特殊用途**：Times New Roman（正式文档）

### 9. 布局设计（Layout Design）

**定义**：布局设计是指组织幻灯片上元素位置和大小的过程。

**原则**：
- **平衡**：元素分布均匀
- **对齐**：使用网格系统对齐元素
- **留白**：适当的空白区域提高可读性
- **层次**：创建清晰的视觉层次

**常见布局类型**：
- **标题 + 内容**：简洁的单页布局
- **对比布局**：左右或上下对比
- **网格布局**：结构化的数据展示

### 10. 内容密度（Content Density）

**定义**：内容密度是指幻灯片上信息的紧凑程度。

**原则**：
- **适度**：避免信息过载
- **可读性**：确保内容易于阅读
- **重点突出**：只展示最重要的信息

**建议**：
- 每页最多 6-7 个项目符号
- 每个项目符号最多 2 行
- 使用简洁的语言

## Appendix: 金融背景知识

这份文件是"格式标准（Formatting Standards）"参考文档的详细说明。在投资银行中，PPT 的格式决定了第一印象——一个格式不统一的 Pitch Deck 会让客户觉得"这家投行不专业"。

---

### 1. 为什么格式这么重要？

**类比：**
你去两家律师事务所面试。一家律师穿着笔挺西装、皮鞋锃亮。另一家穿拖鞋、T恤、头发乱糟糟。你会选谁？

投行的 Pitch Deck 就是"着装"——格式就是投行穿给客户看的"西装"。如果格式不统一，客户第一时间就判断"这家不专业"。

---

### 2. 投行 PPT 的"六大格式铁律"

| 铁律 | 标准 | 为什么 |
|------|------|--------|
| 字体统一 | Arial/Calibri 全篇 | 避免"杂货铺"感 |
| 字号层级 | 标题 24pt、正文 12pt、脚注 9pt | 一眼看到重点 |
| 颜色一致 | 品牌色不超过 3 种 | 降低视觉噪音 |
| 对齐 | 文本框、图片严格对齐 | 散乱 = 不专业 |
| 间距一致 | 段落间距、页边距统一 | 呼吸感 |
| 数字格式 | 千分位逗号、百分比一位小数 | 可读性 |

---

### 3. 格式的"致命三错"

| 错误 | 客户解读 |
|------|---------|
| 两页字体不一样 | "他们不够仔细" |
| 数字格式不统一 | "他们连数字都搞不清楚" |
| 图表配色乱 | "他们审美有问题" |

---

### 给小白的一句话

> Formatting Standards 就是投行 PPT 的"着装规范"——就像律师必须穿西装出庭一样，投行的 PPT 也必须"穿上统一的制服"（统一的字体、颜色、对齐、数字格式）。一个 VP 曾说过："我看到字体不统一的 PPT，连内容都不会看——因为格式都搞不好的团队，不可能做好 $10 亿的交易。"
