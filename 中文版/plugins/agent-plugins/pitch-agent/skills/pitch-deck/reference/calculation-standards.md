## 关键验证公式

### CAGR 预测

**公式：**
```
未来价值 = 当前价值 × (1 + CAGR)^n
```

**变量：**
- 当前价值：当前/基准年市场规模
- CAGR：复合年增长率（以小数表示，例如 16.4% = 0.164）
- n：基准年与目标年之间的年数

**验证示例：**
```
来源声明：$221亿 (2024年) 以16.4% CAGR增长 = $550亿 (2030年)

验证：22.1 × (1.164)^6 = 22.1 × 2.488 = 55.0 ✓
```

**计算 n（年数）：** 计算基准年与目标年之间的年数。示例：2024→2030 = 6年，2025→2030 = 5年。

### 估值倍数

**EV/Revenue：**
```
EV/Revenue倍数 = 企业价值 ÷ 收入
隐含EV = 收入 × 倍数
```

**EV/EBITDA：**
```
EV/EBITDA倍数 = 企业价值 ÷ EBITDA
隐含EV = EBITDA × 倍数
```

**验证示例：**
```
来源声明：$4.36亿交易，基于$4500万收入的9.7倍收入倍数

验证：436 ÷ 45 = 9.69 ≈ 9.7x ✓
```

### 市场份额

**公式：**
```
市场份额 = (细分市场规模 ÷ 总市场规模) × 100
```

**验证示例：**
```
来源声明：在线细分市场($180亿)占总市场($650亿)的28%

验证：18 ÷ 65 = 0.277 = 27.7% ≈ 28% ✓
```

### 增长率

**同比增长率：**
```
YoY增长率 = (本年 - 上年) ÷ 上年 × 100
```

**基于端点计算CAGR：**
```
CAGR = (终值 ÷ 初值)^(1/n) - 1
```

## 共识方法论

当源数据包含多个估计值时，验证共识计算：

### 规模共识（范围）

**方法：** 所有来源的完整最小-最大范围

**示例：**
```
来源：$149亿, $183亿, $211亿, $212亿, $221亿
共识：$150-220亿（四舍五入到最接近的$10亿）
```

### CAGR共识（中心集群）

**方法：** 排除异常值（最高和最低），使用中心集群范围

**示例：**
```
来源：10.6%, 16.4%, 17.2%, 19.0%, 22.7%
排除异常值：10.6%（低）, 22.7%（高）
中心集群：16.4%, 17.2%, 19.0%
共识：16-19% 或 16-17%（保守）
```

### 预测共识

**方法：** 对规模范围的中点应用共识CAGR

**示例：**
```
规模范围：$150-220亿 → 中点：$185亿
CAGR共识：16-17%
16%时：18.5 × (1.16)^6 = $451亿
17%时：18.5 × (1.17)^6 = $475亿
共识预测：$450-480亿
```

## 四舍五入指南

这些是**典型惯例**——根据数值大小和模板风格进行调整：

| 值类型 | 典型四舍五入 | 示例 |
|--------|--------------|------|
| 大型市场规模($100亿+) | 最接近的$10亿 | 18.47 → $180亿 |
| 较小市场规模(<$100亿) | 最接近的$5亿或$1亿 | 2.3 → $25亿 |
| 规模范围 | 匹配来源精度 | 14.9-22.1 → $150-220亿 |
| CAGR | 整数%或0.5% | 16.4% → 16%或16.5% |
| 市场份额 | 最接近的5%或匹配来源 | 27.7% → 25%或30% |
| 收入($m) | 1位小数 | 18.47 → $18.5m |
| 倍数 | 1位小数 | 9.688 → 9.7x |

**四舍五入原则：**
- 四舍五入不应实质性改变数字——对于较小的值，使用更精细的精度
- 一致性比精度更重要——对相似数字使用相同的四舍五入方式
- 创建范围时，低端向下舍入，高端向上舍入
- 对于汇总统计（均值、中位数），匹配输入数据的精度

## 验证检查清单

在使用源数据中的任何计算值之前：

### 公式验证
- [ ] 预测使用正确的CAGR公式：`PV × (1 + r)^n`
- [ ] 倍数计算为EV ÷ 指标（不是反向）
- [ ] 增长率在分母中使用正确的基准年
- [ ] 百分比份额在适用时总和约为100%

### 输入验证
- [ ] 基准年数据与源文档匹配
- [ ] CAGR/增长率与声明的来源方法论匹配
- [ ] 时间段(n)计算正确
- [ ] 货币和单位一致（$bn vs $m）

### 输出验证
- [ ] 计算结果与来源声明的数字匹配
- [ ] 如果不匹配，调查方法论差异
- [ ] 四舍五入应用一致
- [ ] 结果合理（无数量级错误）

### 共识验证
- [ ] 所有来源都包含在范围计算中
- [ ] 异常值排除方法论已记录
- [ ] 中点计算使用正确的平均方法
- [ ] 范围边界代表实际最小/最大值或已记录的子集

## 需要调查的红色警报

**预测不匹配：**
- 计算的预测与来源差异超过5%
- 可能原因：不同的基准年、不同的CAGR或四舍五入

**倍数不匹配：**
- 计算的倍数与来源不同
- 可能原因：不同的指标定义（LTM vs. NTM，收入 vs. 净收入）

**共识不匹配：**
- 您的共识与来源的共识不同
- 可能原因：来源排除了某些数据点，不同的异常值处理

**有疑问时：** 在脚注中注明差异并显示您的计算方法。

## 金融术语解释

### 1. CAGR（复合年增长率）

**定义**：CAGR（Compound Annual Growth Rate）是衡量投资或指标在特定时期内平均年增长率的指标。

**计算公式**：
- CAGR = (终值 ÷ 初值)^(1/年数) - 1

**特点**：
- 假设增长率是恒定的
- 平滑了期间的波动
- 便于比较不同时期的增长率

**应用场景**：
- 评估投资回报
- 预测市场规模
- 比较公司增长表现

### 2. 企业价值（EV）

**定义**：企业价值是衡量公司总价值的指标，考虑了股权和债务。

**计算公式**：
- EV = 市值 + 债务 - 现金及等价物

**组成部分**：
- **市值**：普通股的市场价值
- **债务**：短期和长期债务
- **现金**：现金及现金等价物

**用途**：
- 并购估值
- 跨公司比较
- 财务建模

### 3. 估值倍数（Valuation Multiples）

**定义**：估值倍数是用于评估公司价值的比率，通常基于财务指标。

**常用倍数**：
- **EV/Revenue**：企业价值与收入比
- **EV/EBITDA**：企业价值与EBITDA比
- **P/E**：市盈率

**应用场景**：
- 相对估值
- 投资决策
- 并购定价

**注意事项**：
- 需要结合行业特点
- 注意指标定义（LTM vs NTM）
- 考虑公司特殊情况

### 4. 市场份额（Market Share）

**定义**：市场份额是指公司在特定市场中的销售额占整个市场销售额的百分比。

**计算公式**：
- 市场份额 = (公司收入 / 市场总收入) × 100%

**重要性**：
- 反映竞争地位
- 评估增长机会
- 影响估值

**计算注意事项**：
- 确保市场定义一致
- 考虑时间范围
- 注意数据来源

### 5. 同比增长率（YoY Growth）

**定义**：同比增长率是指与上一年相比的增长百分比。

**计算公式**：
- YoY增长率 = (本年值 - 上年值) / 上年值 × 100%

**优点**：
- 直观易懂
- 便于年度比较
- 反映季节性影响

**局限性**：
- 可能受基数影响
- 无法反映长期趋势

### 6. 共识估计（Consensus Estimates）

**定义**：共识估计是多位分析师对财务指标的平均预测值。

**数据来源**：
- 卖方分析师报告
- 金融数据提供商
- 专业财经媒体

**类型**：
- 当前季度估计
- 年度估计
- 长期增长估计

**影响**：
- 股价对实际业绩与共识的差异敏感
- 共识估计是投资分析的重要参考

### 7. 异常值（Outliers）

**定义**：异常值是与其他数据点显著不同的值。

**识别方法**：
- 可视化检查（箱线图）
- 统计方法（Z-score、IQR）
- 领域知识

**处理方法**：
- 排除异常值
- 单独分析
- 使用稳健统计方法

**注意事项**：
- 异常值可能包含重要信息
- 排除前需要验证原因

### 8. 四舍五入（Rounding）

**定义**：四舍五入是将数字近似到指定精度的过程。

**规则**：
- 四舍六入五取偶（银行家舍入）
- 四舍五入到指定小数位

**原则**：
- 保持一致性
- 避免误导
- 注意精度要求

**应用场景**：
- 财务报告
- 演示文稿
- 数据分析

### 9. LTM（过去12个月）

**定义**：LTM（Last Twelve Months）是指最近12个月的财务数据。

**计算方法**：
- 使用最近四个季度的数据
- 或滚动12个月数据

**优点**：
- 反映最新业务状况
- 消除季节性影响
- 便于比较

**应用场景**：
- 估值分析
- 业绩评估
- 财务建模

### 10. NTM（未来12个月）

**定义**：NTM（Next Twelve Months）是指未来12个月的预测财务数据。

**计算方法**：
- 基于分析师预测
- 或公司指引

**优点**：
- 反映预期表现
- 支持投资决策
- 便于估值

**应用场景**：
- 估值分析
- 投资决策
- 财务规划
