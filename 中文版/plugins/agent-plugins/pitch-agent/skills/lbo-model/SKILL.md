## 面向Claude的关键说明 — 首先阅读

### 环境：Office JS vs Python

**如果在Excel内运行（Office加载项/Office JS环境）：**
- 直接使用Office JS（`Excel.run(async (context) => {...})`）— 不要使用Python/openpyxl
- 通过`range.formulas = [["=B5*B6"]]`写入公式 — Office JS公式在实时工作簿中本地重新计算
- 相同的公式优先于硬编码规则适用：设置`range.formulas`，永远不要对应该是计算的单元格设置`range.values`
- 使用`range.format.font.color` / `range.format.fill.color`表示蓝色/黑色/紫色/绿色约定
- 无需单独的重新计算步骤 — Excel原生处理计算
- **合并单元格陷阱：** 不要在合并范围上调用`.merge()`然后设置`.values`（抛出`InvalidArgument` — 范围仍报告原始尺寸）。相反：仅向左上角单元格写入值（`ws.getRange("A7").values = [["SOURCES & USES"]]`），然后合并并格式化完整范围（`ws.getRange("A7:F7").merge(); ws.getRange("A7:F7").format.fill.color = "#1F4E79";`）

**如果生成独立.xlsx文件（无实时Excel会话）：**
- 使用Python/openpyxl，如下所述
- 写入公式字符串（`ws["D20"] = "=B5*B6"`），然后在交付前运行`recalc.py`

本技能的其余部分使用openpyxl示例，但相同原则适用于Office JS — 只需转换API调用。

### 核心原则
* **每个计算必须是Excel公式** — 永远不要在Python中计算值并硬编码结果到单元格。使用openpyxl时，写`cell.value = "=B5*B6"`（公式字符串），而不是`cell.value = 1250`（计算结果）。模型必须是动态的，当输入变化时更新。
* **使用模板结构** — 遵循`examples/LBO_Model.xlsx`或用户提供的模板中的组织。不要发明自己的布局。
* **使用正确的单元格引用** — 所有公式应引用适当的单元格。永远不要输入应该来自其他单元格的数字。
* **保持符号约定一致** — 遵循模板使用的任何符号约定（有些对流出使用负数，有些使用正数）。在整个过程中保持一致。
* **分部分工作，在每个步骤与用户验证** — 完全完成一个部分，向用户展示构建内容，运行部分的验证检查，并在进入下一部分之前获得确认。不要端到端构建整个模型然后呈现 — 后续部分依赖于较早部分，所以在回报已经构建后发现在资金来源与运用中的错误意味着需要到处返工。

### 公式颜色约定
* **蓝色（0000FF）**：硬编码输入 — 输入的不引用其他单元格的数字
* **黑色（000000）**：带计算的公式 — 使用运算符或函数的任何公式（`=B4*B5`、`=SUM()`、`=-MAX(0,B4)`）
* **紫色（800080）**：链接到**同一标签**上单元格的链接 — 无计算的直接引用（`=B9`、`=B45`）
* **绿色（008000）**：链接到**不同标签**上单元格的链接 — 跨表引用（`=Assumptions!B5`、`='Operating Model'!C10`）

### 填充颜色调色板 — 专业蓝灰色（除非用户/模板另有指定）
* **保持最少** — 仅对单元格填充使用蓝色和灰色。不要引入绿色、黄色、红色或多种强调色。专业的LBO模型使用克制。
* **默认填充调色板：**
  * **部分标题**（资金来源与运用、运营模型等）：深蓝色`#1F4E79`，白色粗体文本
  * **列标题**（第1年、第2年等）：浅蓝色`#D9E1F2`，黑色粗体文本
  * **输入单元格**：浅灰色`#F2F2F2`（或仅白色）— 蓝色*字体*是信号，填充是次要的
  * **公式/计算单元格**：白色，无填充
  * **关键输出**（IRR、MOIC、退出权益）：中蓝色`#BDD7EE`，黑色粗体文本
* **这是整个调色板。** 3种蓝色 + 1种灰色 + 白色。如果模板使用自己的颜色，则遵循模板。
* 注意：上述蓝色/黑色/紫色/绿色**字体**颜色用于区分输入与公式与链接。这些与这里的**填充**调色板分开 — 两者共同起作用。

### 数字格式标准
* **货币**：`$#,##0;($#,##0);"-"`或`$#,##0.0`，取决于模板
* **百分比**：`0.0%`（一位小数）
* **倍数**：`0.0"x"`（一位小数）
* **MOIC/详细比率**：`0.00"x"`（两位小数以确保精度）
* **所有数字单元格**：右对齐

## 常见问题区域

以下计算模式在LBO模型中经常出现问题。遇到这些时特别注意：

### 平衡部分
* 当两部分必须相等时（例如，资金来源 = 资金运用），通常有一项是"塞子"（平衡数）
* 识别哪一项是塞子，并将其计算为差额

### 税务计算
* 税务公式应仅引用相关收入行和税率
* 不应引用不相关的部分（例如，债务时间表）
* 考虑亏损是否产生税盾或被忽略

### 利息和循环引用
* 利息计算可能产生循环（如果它们引用受现金流影响的余额）
* 使用**期初余额**（不是平均或期末）来打破循环
* 模式：利息 → 现金流 → 偿还 → 期末余额（如果利息使用期末余额，这会循环回来）

### 债务偿还/现金清扫
* 当存在多个债务批次时，通常有优先级顺序
* 现金清扫应尊重优先级瀑布
* 余额不能为负 — 适当使用MAX或MIN函数

### 回报计算（IRR/MOIC）
* 现金流必须有正确的符号：投资 = 负数，收益 = 正数
* 如果使用XIRR，需要相应的日期
* 如果使用IRR，现金流应在连续期间
* MOIC = 总收益 / 总投资

### 敏感性表
* **使用奇数维度**（5×5或7×7）— 永远不要4×4或6×6。奇数维度保证一个真正的中心单元格。
* **中心单元格 = 基础情景。** 围绕模型的真实假设对称构建行和列轴值（例如，如果基础进入倍数 = 10.0x，轴 = `[8.0x, 9.0x, 10.0x, 11.0x, 12.0x]`）。然后中心单元格的IRR/MOIC必须等于模型的真实IRR/MOIC输出 — 这是验证表连接正确的证明。
* **突出显示中心单元格** — 中蓝色填充（`#BDD7EE`）+粗体字体，使基础情景在视觉上锚定。
* Excel的DATA TABLE功能可能不适用于openpyxl — 相反，写入明确引用行/列标题的公式
* 每个单元格应显示不同的值 — 如果全部相同，公式没有正确变化
* 使用混合引用（例如，`$A5`表示行输入，`B$4`表示列输入）

## 需要避免的常见错误

| 错误 | 出什么问题 | 如何修复 |
|-------|-----------------|------------|
| 硬编码计算值 | 当输入变化时模型不更新 | 始终使用引用源单元格的公式 |
| 复制后错误的单元格引用 | 公式指向错误的单元格 | 验证所有链接，使用适当的$锚定 |
| 循环引用错误 | 模型无法计算 | 对利息类型计算使用期初余额，打破循环 |
| 部分不平衡 | 应该匹配的总数不匹配 | 确保一项是塞子（计算为差额） |
| 不可能为负的余额 | 支付/使用超过可用 | 适当使用MAX(0, ...)或MIN函数 |
| IRR/回报错误 | 错误符号或不完整范围 | 检查现金流符号并确保公式覆盖所有期间 |
| 敏感性表显示相同值 | 公式不随输入变化 | 检查单元格引用 — 需要混合引用（$A5, B$4） |
| 结转不连接 | 期初 ≠ 上期期末 | 验证期间之间的链接 |
| 符号约定不一致 | 加法变成减法或反之亦然 | 在整个过程中遵循模板的约定 |

## 金融术语和知识详细解释

### 1. LBO（杠杆收购）

**定义**：杠杆收购是一种收购形式，其中收购方使用大量借入资金（债务）来支付收购成本，目标公司的资产通常作为债务的抵押品。

**核心结构**：
```
收购价格 = 50-80% 债务 + 20-50% 权益
```

**典型LBO结构**：
| 组成部分 | 典型比例 |
|---------|---------|
| 债务 | 50-80% |
| 权益 | 20-50% |

**为什么重要**：
- 私募股权最常用的收购策略
- 通过债务融资放大权益回报
- 目标公司现金流用于偿还债务
- 5-7年后通过出售或IPO退出

---

### 2. MOIC（投入资本倍数）

**定义**：MOIC（Multiple on Invested Capital）衡量投资者投入资本的回报倍数。

**计算公式**：
```
MOIC = 投资者最终收到的总收益 / 投资者的总投入资本
```

**示例**：
```
投入：$100万
5年后退出收到：$250万
MOIC = $250万 / $100万 = 2.5x
```

**为什么重要**：
- 简单直接的回报衡量
- 不考虑时间价值
- 投资者快速比较交易
- LBO模型关键输出之一

---

### 3. IRR（内部收益率）

**定义**：IRR是使投资净现值为零的折现率。

**核心原理**：
```
当 IRR = 折现率时：
投入的现值 = 未来收益的现值
```

**为什么重要**：
- 考虑资金时间价值
- 年化回报率
- 便于比较不同期限的投资
- PE基金绩效标准衡量

**MOIC vs IRR**：
| 指标 | 优点 | 缺点 |
|------|------|------|
| MOIC | 简单直观 | 不考虑时间 |
| IRR | 考虑时间价值 | 可被短期交易夸大 |

---

### 4. 资金来源与运用（Sources & Uses）

**资金来源（Sources）**：收购资金的来源
```
资金来源：
- 优先债：$50M
- 夹层债：$20M
- 权益：$30M
总计：$100M
```

**资金运用（Uses）**：资金的具体用途
```
资金运用：
- 收购股权：$90M
- 交易费用：$5M
- 债务融资费用：$3M
- 运营资金：$2M
总计：$100M
```

**关键原则**：
- 资金来源 = 资金运用（必须平衡）
- 权益投资者通常需要20-50%
- 债务融资依赖目标现金流

---

### 5. 债务分层（Debt Tranches）

**定义**：LBO中不同类型的债务按优先级和风险/回报特征分层。

**典型分层结构**：

| 债务类型 | 优先级 | 利率 | 期限 |
|---------|--------|------|------|
| 优先担保债 | 最高 | 低（~5-7%） | 5-7年 |
| 夹层债 | 中 | 中（~10-15%） | 5-7年 |
| 高收益债 | 最低 | 高（~15-20%） | 7-10年 |

**瀑布式偿付**：
```
现金流 → 利息 → 优先债偿还 → 夹层债偿还 → 权益
```

---

### 6. 现金清扫（Cash Sweep）

**定义**：现金清扫是指用超额现金流偿还债务的机制。

**运作方式**：
1. 计算可用于偿还债务的 excess cash
2. 按优先级顺序偿还债务
3. 高级债务优先获得清偿
4. 偿还后债务余额减少，利息降低

**为什么重要**：
- 加速债务偿还
- 减少利息负担
- 提高未来现金流
- 降低LBO风险

---

### 7. 运营杠杆（Operating Leverage）

**定义**：运营杠杆描述固定成本与可变成本的比例。

**高运营杠杆公司**：
- 固定成本高，可变成本低
- 收入增长时利润率快速扩张
- 收入下降时利润率快速收缩

**公式**：
```
经营杠杆系数 = (EBIT变化%) / (收入变化%)
```

**为什么重要**：
- 影响EBIT利润率对收入变化的敏感性
- LBO中收入增长带来利润率扩张
- 关键假设：EBIT利润率从X%提升到Y%

---

### 8. 进入倍数（Entry Multiple）

**定义**：进入倍数是收购价格相对于EBITDA的倍数。

**计算公式**：
```
进入倍数 = 收购价格 / EBITDA
```

**典型倍数范围**：
| 行业 | 典型倍数 |
|------|---------|
| 成熟行业 | 6-8x |
| 成长型 | 8-12x |
| 科技/高增长 | 12-20x |

**为什么重要**：
- 决定初始债务水平
- 影响交易回报
- 退出倍数参考基准
- 投资决策核心指标

---

### 9. 退出倍数（Exit Multiple）

**定义**：退出倍数是5-7年后出售公司时的企业价值相对于EBITDA的倍数。

**预期退出分析**：
```
预期退出企业价值 = 退出EBITDA × 退出倍数
```

**回报影响**：
| 退出倍数变化 | 对回报的影响 |
|-------------|-------------|
| 进入8x，退出8x | 回报取决于债务偿还 |
| 进入8x，退出10x | 显著提升回报 |
| 进入8x，退出6x | 回报压缩或负回报 |

---

### 10. 权益回报路径（Path to Equity Return）

**LBO中权益回报的来源**：

1. **债务偿还**：
   - 用运营现金流偿还债务
   - 降低债务/EBITDA比率
   - 减少利息费用

2. **EBITDA增长**：
   - 收入增长 + 利润率扩张
   - 提高EBITDA
   - 即使倍数不变，企业价值也增加

3. **倍数扩张**：
   - 市场改善
   - 公司质量提升
   - 更高倍数出售

**典型回报分解**：
```
总MOIC = 2.5x
其中：
- 债务偿还贡献：0.8x
- EBITDA增长贡献：1.0x
- 倍数扩张贡献：0.7x
```

## Appendix: 金融背景知识

这份文件是"杠杆收购模型（LBO Model）"技能的详细说明。LBO 是私募股权（Private Equity, PE）行业的核心工具——它计算"用借来的钱买下一家公司，然后还债、改善经营、再卖掉，能赚多少钱"。

---

### 1. 什么是 LBO？

**类比：**
想象你是房地产投资者。你看中了一套价值 500 万的房子，相信装修后能卖 800 万。但你只有 100 万存款，于是：
1. 你用 100 万首付 + 银行贷款 400 万（5x 杠杆）买下房子
2. 花了 50 万装修（改善经营）
3. 把房子出租，租金还银行贷款
4. 5 年后房子价值涨到 800 万，卖掉
5. 还掉剩余贷款 300 万 → 净赚 500 万 - 100 万本金 = 400 万回报 = 4x！

LBO 就是这个逻辑——私募股权基金"借钱买公司、改善经营、还债、卖掉赚差价"。

---

### 2. LBO 的"五步法"

```
步骤 1：资金来源与运用（Sources & Uses）
  → 买公司需要多少钱？钱从哪来？

步骤 2：运营模型（Operating Model）
  → 公司每年赚多少钱？

步骤 3：债务时间表（Debt Schedule）
  → 每年要还多少钱？

步骤 4：回报计算（Returns Analysis）
  → PE 基金能赚多少倍？

步骤 5：敏感性分析
  → 如果买价贵了 10%？如果增长率更低？
```

---

### 3. LBO 的"三大回报指标"

| 指标 | 英文 | 含义 | 好标准 |
|------|------|------|--------|
| MOIC | Multiple on Invested Capital | 投 1 元赚多少元 | >2.0x |
| IRR | Internal Rate of Return | 年化回报率 | >20% |
| 退出倍数 | Exit Multiple | 卖的时候 EV/EBITDA | 不高于买入倍数 |

**举例**：PE 基金用 $1 亿买了公司，5 年后卖了 $2.5 亿。
- MOIC = 2.5x（投 1 元变 2.5 元）
- IRR ≈ 20%（年化）
- 这是一笔不错的交易

---

### 4. LBO 的"杠杆逻辑"

| 场景 | 无杠杆 | 3x 杠杆 | 5x 杠杆 |
|------|--------|--------|--------|
| 自有资金 | $100M | $25M | $17M |
| 借款 | $0 | $75M | $83M |
| 总购买价 | $100M | $100M | $100M |
| 5 年后卖 | $200M | $200M | $200M |
| 还贷 | $0 | -$75M | -$83M |
| 净得 | $200M | $125M | $117M |
| 回报倍数 | 2.0x | 5.0x | 7.0x |

**结论**：杠杆越高，自有资金的回报倍数越高——但风险也越大（如果公司出了问题还不起债就破产）。

---

### 5. 真实案例：史上最成功的 LBO

**交易**：黑石（Blackstone）2007 年收购希尔顿酒店

| 项目 | 数据 |
|------|------|
| 收购价 | $26B |
| 自有资金 | $5.5B |
| 借贷 | $20.5B |
| 持有期 | 11 年（2007-2018） |
| 出售价 | 三阶段出售，总计约 $40B+ |
| MOIC | ~3x |
| IRR | ~18-20% |

哪怕经历了 2008 年金融危机，希尔顿的 LBO 依然是史上最成功的 PE 交易之一。

---

### 给小白的一句话

> LBO 就是 PE 行业的"杠杆买房术"——用 20% 自己的钱 + 80% 银行借款买下一家公司，用公司赚的钱还银行的债，5-7 年后把公司卖掉，净赚几倍回报。杠杆是一把双刃剑：如果公司经营好、卖得贵，杠杆放大回报（5x）；如果公司出问题、卖得便宜，杠杆放大亏损（血本无归）。LBO 模型就是帮你算清楚"在不同的买价、卖价、杠杆水平下，究竟能赚多少"。
