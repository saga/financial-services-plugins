# pptx-author

当运行在**无界面模式**（托管代理/CMA模式）时，且需要将PowerPoint演示文稿作为**文件产物**交付，而不是通过 `mcp__office__powerpoint_*` 编辑实时文档时，使用此技能。

## 输出约定

- 写入到 `./out/<name>.pptx`。如果 `./out/` 不存在则创建。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建演示文稿

编写一个简短的Python脚本并通过Bash运行。使用 `python-pptx`：

```python
from pptx import Presentation
from pptx.util import Inches, Pt

prs = Presentation("./templates/firm-template.pptx")  # 如果提供了模板
# 或者: prs = Presentation()

slide = prs.slides.add_slide(prs.slide_layouts[5])    # 仅标题布局
slide.shapes.title.text = "Valuation Summary"
# ... 添加表格 / 图表 / 文本框 ...

prs.save("./out/pitch-<target>.pptx")
```

## 约定（镜像实时Office `pitch-deck`技能）

- **每张幻灯片一个主题。** 标题陈述要点；正文提供支持。
- **每个数字都可追溯到模型。** 如果数字来自 `./out/model.xlsx`，在脚注中标明工作表和单元格。
- **使用公司模板**（当模板挂载在 `./templates/` 时）；否则使用默认布局。
- **图表**：当保真度很重要时，优先嵌入从模型渲染的PNG，而不是原生pptx图表。
- **不对外发送。** 此技能只写入文件；从不发送邮件或上传。

## 何时不使用

如果 `mcp__office__powerpoint_*` 工具可用（Cowork插件模式），请改用这些工具——它们通过审查检查点操作用户的实时文档。此技能是无界面运行时生成文件的备选方案。

## 金融术语解释

### 1. PowerPoint演示文稿（Pitch Deck）

**定义**：Pitch Deck是一种用于向投资者、客户或合作伙伴展示商业理念或投资机会的演示文稿。

**典型结构**：
- **封面页**：公司名称、logo、日期
- **执行摘要**：核心价值主张
- **市场机会**：市场规模、增长趋势
- **产品/服务**：解决方案描述
- **商业模式**：盈利方式
- **竞争分析**：竞争格局
- **财务预测**：收入、利润预测
- **融资需求**：资金用途
- **团队介绍**：核心团队背景

**设计原则**：
- 简洁明了，避免信息过载
- 视觉吸引力强
- 数据可视化（图表、图形）
- 清晰的逻辑流程

### 2. 估值摘要（Valuation Summary）

**定义**：估值摘要是对公司或资产价值的总结，通常包括多种估值方法的结果。

**常见估值方法**：
- **DCF（折现现金流）**：基于未来现金流的现值
- **可比公司分析**：与同类上市公司比较
- **交易可比性分析**：与同类并购交易比较
- **资产基础法**：基于资产价值

**关键输出**：
- 企业价值（Enterprise Value）
- 股权价值（Equity Value）
- 每股价值（Per Share Value）
- 估值区间（Valuation Range）

### 3. 演示文稿模板（Presentation Template）

**定义**：演示文稿模板是预定义的设计格式，确保演示文稿的一致性和专业性。

**模板组成**：
- **母版幻灯片**：统一的字体、颜色、布局
- **布局模板**：标题页、内容页、图表页等
- **主题颜色**：品牌色彩方案
- **字体样式**：标题字体、正文字体

**优点**：
- 提高制作效率
- 保持品牌一致性
- 确保专业外观

### 4. 数据可视化（Data Visualization）

**定义**：数据可视化是将数据转换为图形或图表的过程，便于理解和分析。

**常见图表类型**：
- **柱状图**：比较不同类别数据
- **折线图**：显示趋势变化
- **饼图**：展示比例关系
- **散点图**：显示变量关系
- **瀑布图**：展示累计变化

**设计原则**：
- 清晰标注坐标轴
- 使用适当的刻度
- 添加数据标签
- 保持简洁

### 5. 图表嵌入（Chart Embedding）

**定义**：图表嵌入是将外部生成的图表插入到演示文稿中的过程。

**方法**：
- **PNG嵌入**：将图表保存为图片后插入
- **原生图表**：使用PowerPoint内置图表功能
- **链接图表**：保持与源文件的动态链接

**选择因素**：
- 保真度要求
- 更新频率
- 文件大小限制
- 编辑需求

### 6. 企业价值（Enterprise Value, EV）

**定义**：企业价值是衡量公司总价值的指标，考虑了股权和债务。

**计算公式**：
```
EV = 市值 + 债务 - 现金及等价物
```

**组成部分**：
- **市值**：普通股的市场价值
- **债务**：短期和长期债务
- **现金**：现金及现金等价物

**用途**：
- 公司估值
- 并购交易定价
- 财务分析

### 7. 财务预测（Financial Projections）

**定义**：财务预测是对未来财务表现的估计，通常包括收入、利润和现金流。

**组成部分**：
- 收入预测
- 成本预测
- 利润预测
- 现金流预测

**方法**：
- 自上而下方法
- 自下而上方法
- 基于市场的方法

**重要假设**：
- 增长率
- 利润率
- 资本支出

### 8. 融资需求（Funding Requirements）

**定义**：融资需求是指公司为实现其目标所需的资金数额。

**资金用途**：
- 产品开发
- 市场推广
- 团队扩张
- 运营资金
- 收购

**融资类型**：
- 股权融资
- 债务融资
- 混合融资

**演示要点**：
- 所需金额
- 资金用途明细
- 预期回报

### 9. 可比公司分析（Comparable Company Analysis）

**定义**：可比公司分析是通过比较类似上市公司来评估目标公司价值的方法。

**步骤**：
1. 选择可比公司
2. 收集财务数据
3. 计算估值倍数
4. 应用倍数到目标公司

**常用倍数**：
- EV/Revenue
- EV/EBITDA
- P/E

**注意事项**：
- 确保可比性
- 考虑行业差异
- 调整非经常性项目

### 10. 交易可比性分析（Transaction Comparables）

**定义**：交易可比性分析是通过比较类似并购交易来评估目标公司价值的方法。

**步骤**：
1. 识别类似交易
2. 收集交易数据
3. 计算交易倍数
4. 应用倍数到目标公司

**优势**：
- 反映市场实际支付价格
- 考虑协同效应
- 适合并购场景

**局限性**：
- 交易数据可能有限
- 交易条件差异大
- 时间敏感性

## Appendix: 金融背景知识

这份文件是"PPT 创作（PPTX Author）"技能的详细说明。在投资银行中，Python 生成 PPT 是一个高阶技能——当你需要在没有 Office 环境的服务器上批量生成 PPT 时，Python 是唯一的方案。

---

### 1. 什么是 Python 生成 PPT？

**类比：**
想象你是一个印刷厂工人。你的任务是"印 100 份同样的菜单"。你不会自己手写 100 遍——你会设置好印刷机（Python 脚本），让机器自动印。

Python 生成 PPT 就是投行的"自动化排版机"——写一段代码，让它自动生成 50 页的 Pitch Deck。

---

### 2. 为什么投行需要用 Python 生成 PPT？

| 场景 | 说明 |
|------|------|
| 批量生产 | 同一模板生成 100 份不同的客户 PPT |
| 无界面环境 | 服务器上没有 Office 软件 |
| 自动化流程 | CI/CD 管道中自动生成输出 |
| 数据驱动 | 从 Excel 数据库自动填充数字到 PPT |

---

### 3. python-pptx 的核心操作

```python
from pptx import Presentation

# 1. 打开模板
prs = Presentation('template.pptx')

# 2. 选择幻灯片
slide = prs.slides[0]

# 3. 修改文本框
for shape in slide.shapes:
    if shape.has_text_frame:
        for paragraph in shape.text_frame.paragraphs:
            for run in paragraph.runs:
                if '{{REVENUE}}' in run.text:
                    run.text = run.text.replace('{{REVENUE}}', '$1.2B')

# 4. 保存
prs.save('output.pptx')
```

---

### 4. PPT 自动生成的"三大注意事项"

| 注意项 | 说明 |
|--------|------|
| 字体兼容 | 服务器上可能没有你需要的字体（如"微软雅黑"） |
| 图片嵌入 | 图表必须转成 PNG 再嵌入 |
| 模板一致性 | 模板中的占位符必须统一命名（如 `{{TARGET_PRICE}}`） |

---

### 5. 真实案例：自动生成 100 份客户 Pitch

**背景**：某投行需要为 100 个潜在客户各生成一份 20 页的 Pitch Deck

| 手动方式 | Python 方式 |
|---------|------------|
| 1 人 × 50 小时 = 50 小时 | 1 脚本 × 5 分钟 = 5 分钟 |
| 容易出错（复制粘贴） | 零出错率 |
| 改一个字要改 100 份 | 改一次代码全更新 |

### 给小白的一句话

> PPTX Author 就是投行 PPT 的"工业革命"——以前需要手工一页一页画图表、写数字，现在写一段 Python 代码就能自动生成 100 份 PPT。这对于需要批量生产的场景（比如给 100 个客户各自定制一份 Pitch Deck）简直是"作弊器"。但要记住：自动化的前提是"模板标准"——如果你每个客户的 PPT 长得都不一样，机器也帮不了你。标准化的模板 + 正确的数据填充 = 机器替代重复劳动。
