# DCF模型构建器

## 概述

本技能遵循投资银行标准创建机构质量级DCF模型用于股权估值。每次分析生成一个详细的Excel模型（敏感性分析包含在DCF工作表的底部）。

## 工具

- 默认使用用户提供和MCP服务器提供的所有信息进行数据源获取。

## 关键约束 — 首先阅读这些

这些约束适用于所有DCF模型构建。开始前请审查：

**环境：Office JS vs Python/openpyxl：**
- **如果在Excel内运行（Office加载项/Office JS环境）：** 直接使用Office JS — 不要使用Python/openpyxl。通过`range.formulas = [["=D19*(1+$B$8)"]]`写入公式。使用`range.format.*`进行样式设置。相同的公式优先于硬编码规则适用：设置`.formulas`，永远不要对派生单元格设置`.values`。
- **如果生成独立.xlsx文件（无实时Excel会话）：** 使用Python/openpyxl，如下所述，然后在交付前运行`recalc.py`。
- 本技能其余部分使用openpyxl示例 — 在该环境中转换为Office JS API调用，但所有原则（公式字符串、单元格注释、部分检查点、敏感性表循环）适用相同。

**⚠️ Office JS合并单元格陷阱：** 使用合并单元格构建部分标题时，不要在合并范围上调用`.merge()`然后设置`.values` — Office JS仍报告范围的原始尺寸，将抛出`InvalidArgument: The number of rows or columns in the input array doesn't match the size or dimensions of the range`。相反，先单独向左上角单元格写入值，然后合并并格式化完整范围：

```js
// 错误 — 抛出InvalidArgument:
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.values = [["MARKET DATA & KEY INPUTS"]];  // 1×1数组 vs 1×8范围 → 失败

// 正确 — 先在单个单元格写入值，然后合并并格式化范围:
ws.getRange("A7").values = [["MARKET DATA & KEY INPUTS"]];
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.format.fill.color = "#1F4E79";
hdr.format.font.bold = true;
hdr.format.font.color = "#FFFFFF";
```

这适用于DCF中每个合并部分标题（市场数据、情景块、现金流预测、终值、估值汇总、敏感性表）。

**公式优先于硬编码（不可协商）：**
- 每个预测、利润率、折现因子、现值和敏感性单元格必须是活动的Excel公式 — 永远不是在Python中计算的值然后写为数字
- 使用openpyxl时：`ws["D20"] = "=D19*(1+$B$8)"`是正确的；`ws["D20"] = calculated_revenue`是错误的
- 唯一允许的硬编码数字是：（1）原始历史输入，（2）假设驱动因素（增长率、WACC输入、终值g），（3）当前市场数据（股价、债务余额）
- 如果发现自己正在Python中计算某物并写入结果 — 停止。模型必须在用户更改假设时灵活变动。

**逐步与用户验证（不要端到端构建）：**
- 数据检索后 → 向用户展示原始输入块（收入、利润率、股份、净债务）并在预测前确认
- 收入预测后 → 展示预测顶线和增长率，在构建利润率前确认
- FCF构建后 → 展示完整FCF时间表，在计算WACC前确认逻辑
- WACC后 → 展示计算和输入，在折现前确认
- 终值+现值后 → 展示股权桥梁（EV → 股权价值 → 每股价格），在敏感性表前确认
- 在每个阶段捕获错误 — 在敏感性表构建后发现的错误利润率假设意味着重建所有下游内容

**敏感性表：**
- **使用奇数行和列**（标准：5×5，有时7×7）— 这保证一个真正的中心单元格
- **中心单元格 = 基础情景。** 构建轴值，使中间行标题和中间列标题完全等于模型的真实假设（例如，如果基础WACC = 9.0%，中间行是9.0%；如果终值g = 3.0%，中间列是3.0%）。因此中心单元格的输出必须等于模型的真实隐含股价 — 这是验证表构建正确的合理性检查。
- **用中蓝色填充（`#BDD7EE`）+粗体字体突出显示中心单元格**，以便立即可见哪个单元格是基础情景。
- 填充所有单元格（通常3个表×25个单元格 = 75个），使用完整DCF重新计算公式
- 使用openpyxl循环（或Office JS循环）以编程方式写入公式
- 无占位符文本，无线性近似，无需手动步骤
- 每个单元格必须重新计算该假设组合的完整DCF

**单元格注释：**
- 在创建每个硬编码值时添加单元格注释
- 格式："来源：[系统/文档]，[日期]，[参考]，[URL（如适用）]"
- 每个蓝色输入必须有注释才能进入下一部分
- 不要推迟到最后或写"TODO: 添加来源"

**模型布局规划：**
- 在写任何公式之前定义所有部分行位置
- 首先写所有标题和标签
- 第二步写所有部分分隔符和空白行
- 然后使用锁定的行位置写入公式
- 创建后立即测试公式

**公式重新计算：**
- 交付前运行`python recalc.py model.xlsx 30`
- 修复所有错误直到状态为"success"
- 需要零公式错误（#REF!、#DIV/0!、#VALUE!等）

**情景块：**
- 为熊市/基础/牛市案例创建单独块
- 在每个块内跨预测年份显示假设
- 使用IF公式：`=IF($B$6=1,[Bear单元格],[Base单元格])`或`=IF($B$6=3,[Bull单元格],[Base单元格])`
- 验证公式引用正确的情景块单元格

## DCF流程工作流

### 第1步：数据检索和验证

从MCP服务器、用户提供的数据和网络获取数据。

**数据源优先级：**
1. **MCP服务器**（如果配置）— 来自Daloopa等提供商的structured财务数据
2. **用户提供的数据** — 来自他们研究的 historical financials
3. **网络搜索/获取** — 当前价格、beta、债务和现金（如需要）

**验证清单：**
- 验证净债务 vs 净现金（估值关键）
- 确认稀释流通股（检查近期回购/发行）
- 验证历史利润率与业务模型一致
- 交叉检查增长率与行业基准
- 验证税率合理（通常21-28%）

### 第2步：历史分析（3-5年）

分析和记录：
- **收入增长趋势**：计算CAGR，识别驱动因素
- **利润率进展**：跟踪毛利率、EBIT利润率、FCF利润率
- **资本密集度**：D&A和CapEx占收入百分比
- **营运资本效率**：NWC变化占收入增长百分比
- **回报指标**：ROIC、ROE趋势

创建汇总表显示：
```
历史指标（LTM）：
收入：X百万美元
收入增长：X% CAGR
毛利率：X%
EBIT利润率：X%
D&A占收入百分比：X%
CapEx占收入百分比：X%
FCF利润率：X%
```

### 第3步：构建收入预测

**方法论：**
1. 从最新实际收入开始（LTM或最近财年）
2. 对每个预测年度应用增长率
3. 同时显示美元金额和计算的增长率%

**增长率框架：**
- 第1-2年：较高增长，反映近期可见性
- 第3-4年：逐渐缓和至行业平均水平
- 第5年+：接近永续增长率

**公式结构：**
- 收入（年N）= 收入（年N-1）×（1 + 增长率）
- 增长率%（年N）= 收入（年N）/收入（年N-1）- 1

**三种情景方法：**
```
熊市案例：保守增长（例如，8-12%）
基础案例：最可能的情景（例如，12-16%）
牛市案例：乐观增长（例如，16-20%）
```

### 第4步：运营费用建模

**固定/可变成本分析：**

运营费用应建模现实的运营杠杆：
- **销售和营销**：通常占收入的15-40%，取决于业务模型
- **研发**：科技公司通常10-30%
- **一般和管理费用**：通常占收入的8-15%，显示随着公司规模扩大而杠杆化

**关键原则：**
- 所有百分比基于收入，而非毛利润
- 建模运营杠杆：百分比应随收入规模下降
- 保持S&M、R&D、G&A的单独行项目
- 计算EBIT = 毛利润 - 总运营费用

**利润率扩张框架：**
```
当前状态 → 目标状态（第5年）
毛利率：X% → Y%（基于规模、效率论证）
EBIT利润率：X% → Y%（收入增长+运营费用杠杆的结果）
```

### 第5步：自由现金流计算

**按正确顺序构建FCF：**

```
EBIT
(-) 税费（EBIT × 税率）
= NOPAT（税后净营业利润）
(+) D&A（非现金费用，占收入百分比）
(-) CapEx（占收入百分比，通常4-8%）
(-) Δ NWC（营运资本变化）
= 无杠杆自由现金流
```

**营运资本建模：**
- 计算为收入变化的百分比（delta收入）
- 典型范围：收入变化的-2%到+2%
- 负数 = 现金来源（营运资本释放）
- 正数 = 现金使用（营运资本建设）

**维护 vs 增长CapEx：**
- 维护CapEx：维持当前运营（~2-3%收入）
- 增长CapEx：支持扩张（额外2-5%收入）
- 总CapEx应与公司增长战略一致

### 第6步：资本成本（WACC）研究

**权益成本CAPM方法：**

```
权益成本 = 无风险利率 + Beta × 股权风险溢价

其中：
- 无风险利率 = 当前10年期国债收益率
- Beta = 相对于市场指数的5年月度股票beta
- 股权风险溢价 = 5.0-6.0%（市场标准）
```

**债务成本计算：**

```
税后债务成本 = 税前债务成本 ×（1 - 税率）

从以下确定税前债务成本：
- 信用评级（如有）
- 公司债券当前收益率
- 财务中的利息费用/总债务
```

**资本结构权重：**

```
权益市值 = 当前股价 × 流通股
净债务 = 总债务 - 现金及等价物
企业价值 = 市值 + 净债务

权益权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值

WACC =（权益成本 × 权益权重）+（税后债务成本 × 债务权重）
```

**特殊情况：**
- **净现金状况**：如果现金 > 债务，净债务为负
  - 债务权重可能为负
  - WACC计算相应调整
- **无债务**：WACC = 权益成本

**典型WACC范围：**
- 大型股、稳定：7-9%
- 成长型公司：9-12%
- 高增长/高风险：12-15%

### 第7步：折现率应用（5-10年预测）

**年中惯例：**
- 假设现金流发生在年中
- 折现期：0.5、1.5、2.5、3.5、4.5等
- 折现因子 = 1 /（1 + WACC）^期

**现值计算：**
```
对于每个预测年份：
FCF现值 = 无杠杆FCF × 折现因子

示例（第1年）：
FCF = $1,000
WACC = 10%
期 = 0.5
折现因子 = 1 / (1.10)^0.5 = 0.9535
现值 = $1,000 × 0.9535 = $954
```

**预测期选择：**
- **5年**：大多数分析的标准
- **7-10年**：具有更长增长跑道的高增长公司
- **3年**：成熟、稳定的企业

### 第8步：终值计算

**永续增长法（首选）：**

```
终值FCF = 最后一年FCF ×（1 + 永续增长率）
终值 = 终值FCF /（WACC - 永续增长率）

关键约束：永续增长率 < WACC（否则价值无限）
```

**永续增长率选择：**
- 保守：2.0-2.5%（GDP增长率）
- 中等：2.5-3.5%
- 激进：3.5-5.0%（仅适用于市场领导者）

**不要超过**：无风险利率或长期GDP增长

**退出倍数法（替代）：**
```
终值 = 最后一年EBITDA × 退出倍数

其中退出倍数来自：
- 行业可比交易倍数
- 先例交易倍数
- 典型范围：8-15x EBITDA
```

**终值现值：**
```
终值现值 = 终值 /（1 + WACC）^最后期

其中最后期考虑时间：
5年模型与年中惯例：期 = 4.5
```

**终值合理性检查：**
- 应占企业价值的50-70%
- 如果>75%，模型可能过度依赖终值假设
- 如果<40%，检查终值假设是否过于保守

### 第9步：企业价值到股权价值桥梁

**估值汇总结构：**

```
(+) PV预测FCFs合计 = $X百万
(+) 终值现值 = $Y百万
= 企业价值 = $Z百万

(-) 净债务[如果是负数则为+净现金] = $A百万
= 股权价值 = $B百万

÷ 稀释流通股 = C百万股
= 隐含每股价格 = $XX.XX

当前股价 = $YY.YY
隐含回报 =（隐含价格/当前价格）- 1 = XX%
```

**关键调整：**
- **净债务 = 总债务 - 现金及等价物**
  - 如果为正：从EV中减去（减少股权价值）
  - 如果为负（净现金）：加到EV（增加股权价值）
- **使用稀释股**：包括期权、RSU、可转换证券
- **其他调整**（如适用）：
  - 少数股东权益
  - 养老金负债
  - 经营租赁义务

**估值输出格式：**
```csv
估值组成部分,金额（$M）
PV明确FCFs,X.X
PV终值,Y.Y
企业价值,Z.Z
(-)净债务,A.A
股权价值,B.B
,,
流通股（M）,C.C
隐含每股价格,$XX.XX
当前股价,$YY.YY
隐含上涨/下跌,+XX%
```

### 第10步：敏感性分析

在DCF工作表底部构建**三个敏感性表**，展示估值如何随不同假设变化：

1. **WACC vs 永续增长率** — 展示企业价值对折现率和永续增长的敏感性
2. **收入增长 vs EBIT利润率** — 展示顶线增长和运营杠杆的影响
3. **Beta vs 无风险利率** — 展示权益成本组成部分的敏感性

**实施**：这些是简单的2D网格（不是Excel的"数据表"功能），每个单元格有公式。每个单元格必须包含该特定假设组合的完整DCF重新计算。参见关键约束部分，了解使用openpyxl以编程方式填充所有75个单元格的详细要求。

### 情景块选择模式 — 遵循此方法

**假设按每个情景的单独块组织：**

**关键结构 — 每个部分标题三行：**

```csv
BEAR CASE ASSUMPTIONS（部分标题，跨单元格合并）
Assumption,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,12%,10%,9%,8%,7%
EBIT利润率（%）,45%,44%,43%,42%,41%

BASE CASE ASSUMPTIONS（部分标题，跨单元格合并）
Assumption,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,16%,14%,12%,10%,9%
EBIT利润率（%）,48%,49%,50%,51%,52%

BULL CASE ASSUMPTIONS（部分标题，跨单元格合并）
Assumption,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,20%,18%,15%,13%,11%
EBIT利润率（%）,50%,51%,52%,53%,54%
```

**每个情景块必须有一行列标题**，显示预测年份（FY2025E、FY2026E等），紧接在部分标题下方。没有这个，用户无法知道哪个假设值对应哪个年份。

**如何引用假设 — 创建合并列：**
1. 案例选择器单元格（例如，B6）包含1=Bear、2=Base或3=Bull
2. 创建具有INDEX或OFFSET公式的合并列，从正确的情景块拉取
3. 预测公式引用合并列（干净的单元格引用）
4. 每个情景块包含跨预测年份的完整DCF假设集

**推荐的合并列模式（使用INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不是这样 — 分散的IF语句：**
`=IF($B$6=1,[Bear块单元格],[Base块单元格])`或`=IF($B$6=3,[Bull块单元格],[Base块单元格])`

合并列方法集中逻辑，使模型更容易审计。

### 正确的收入预测模式

**创建具有INDEX公式的合并列，然后在预测中引用它：**

**第1步 — FY1增长的合并列：**
`=INDEX([Bear FY1增长]:[Bull FY1增长], 1, $B$6)`

**第2步 — 收入预测引用合并列：**
`收入第1年: =D29*(1+$E$10)`

其中：
- D29 = 上年收入
- $E$10 = FY1增长的合并列单元格（包含INDEX公式）
- $B$6 = 案例选择器（1=Bear, 2=Base, 3=Bull）

这种方法比在每个预测公式中嵌入IF语句更清晰，并且使审计使用哪种情景假设更容易。

### 正确的FCF公式模式

**使用具有INDEX公式的合并列，然后在FCF计算中引用它们：**

**合并列方法：**
```csv
项目,公式,引用
D&A,=E29*$E$21,$E$21 = D&A%的合并列
CapEx,=E29*$E$22,$E$22 = CapEx%的合并列
Δ NWC,=(E29-D29)*$E$23,$E$23 = NWC%的合并列
无杠杆FCF,=E57+E58-E60-E62,E57=NOPAT E58=D&A E60=CapEx E62=Δ NWC
```

每个合并列单元格包含INDEX公式，根据案例选择器从适当的情景块拉取。这保持预测公式干净和可审计。

在写公式之前，确认情景块行位置并设置合并列。

### 正确的单元格注释格式

**每个硬编码值需要此格式：**

"来源：[系统/文档]，[日期]，[参考]，[URL（如适用）]"

**示例：**
```csv
项目,来源注释
```

### 正确的假设表结构

**关键：每个情景块需要三个结构元素：**

1. **部分标题行**（合并单元格）：例如，"BEAR CASE ASSUMPTIONS"
2. **列标题行**显示年份 — 这是必需的，不要跳过
3. **数据行**与假设值

**结构：**
```csv
BEAR CASE ASSUMPTIONS（部分标题 - 跨A:G列合并）
Assumption,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,X%,X%,X%,X%,X%

WACC,X%,,,,

BASE CASE ASSUMPTIONS（部分标题 - 跨A:G列合并）
Assumption,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,X%,X%,X%,X%,X%
EBIT利润率（%）,X%,X%,X%,X%,X%
永续增长率,X%,,,,
WACC,X%,,,,

BULL CASE ASSUMPTIONS（部分标题 - 跨A:G列合并）
Assumption,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,X%,X%,X%,X%,X%
EBIT利润率（%）,X%,X%,X%,X%,X%
永续增长率,X%,,,,
WACC,X%,,,,
```

**没有显示预测年份（FY2025E、FY2026E等）的列标题行，用户无法知道哪个假设值对应哪个年份。这一行是强制的。**

**然后创建合并列**（通常是右侧下一列），使用INDEX公式根据案例选择器从选定的情景块拉取。这个合并列是您的预测公式引用的内容。

### 正确的行规划流程

**1. 首先写所有标题和标签：**
```csv
行,内容

4,案例选择器

```

**2. 写所有部分分隔符和空白行**

**3. 然后使用锁定的行位置写入公式**

**4. 创建后立即测试公式**

**想象它是建筑施工：**
- 好：先浇地基，然后建墙（稳定结构）
- 坏：先建墙，然后浇地基（墙会倒塌）

**Excel版本：**
- 好：先添加标题，然后写公式（公式稳定）
- 坏：先写公式，然后添加标题（公式会断裂）

### 正确的敏感性表实现

**重要**：这些不是Excel的"数据表"功能。这些是简单的网格，您使用openpyxl在其中编写常规公式。是的，这意味着总共约75个公式（3个表×25个单元格），但这是直接的且必需的。

**公式编程填充：**

每个敏感性表必须完全填充公式，重新计算每个假设组合的隐含股价。**不要使用Excel的数据表功能**（它需要手动干预，无法通过openpyxl自动化）。

**实施方法 — 具体示例：**

**表结构 — 5×5网格（奇数维度，基础情景居中）：**

如果模型的基础WACC = 9.0%，基础永续增长率 = 3.0%，围绕这些值对称构建轴：

```csv
WACC vs 永续增长率,  2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%,       [公式], [公式], [公式], [公式], [公式]
              8.5%,       [公式], [公式], [公式], [公式], [公式]
              9.0%,       [公式], [公式], [★  ], [公式], [公式]   ← 中间行 = 基础WACC
              9.5%,       [公式], [公式], [公式], [公式], [公式]
             10.0%,       [公式], [公式], [公式], [公式], [公式]
                                   ↑
                          中间列 = 基础永续g
```

**★ = 中心单元格。** 它的公式输出必须等于模型的真实隐含股价（来自估值汇总）。对此单元格应用中蓝色填充（`#BDD7EE`）和粗体字体，使基础情景在视觉上锚定。

**轴值规则：** `axis_values = [base - 2*step, base - step, base, base + step, base + 2*step]` — 围绕基础对称，奇数保证中心。

**公式模式 — 单元格B88（WACC=8.0%，永续增长率=2.0%）：**

B88中的公式应使用以下内容重新计算隐含价格：
- 来自行标题的WACC：`$A88`（8.0%）
- 来自列标题的永续增长率：`B$87`（2.0%）

**推荐方法：** 引用主DCF计算但替换这些值。

**示例公式结构：**
`=([使用$A88作为折现率的PV FCFs合计] + [使用B$87作为增长率和$A88作为WACC的终值] - [净债务]) / [股份]`

**关键 — 为5x5网格中的每个单元格写一个公式（25个单元格每个表，75个单元格总计）。** 使用openpyxl在循环中以编程方式写入这些公式。不要跳过此步骤或留占位符文本。

**Python实现模式：**
```python
# 填充敏感性表的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用wacc_value和term_growth_value的公式
        formula = f"=<使用{wacc_value}和{term_growth_value}的DCF重新计算>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性表在打开模型时必须立即工作，无需用户手动步骤。**

<常见错误>

本节包含构建DCF模型时避免的所有错误模式。

### 错误：简化敏感性表近似或占位符文本

**不要使用线性近似：**

```
// 错误 — 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设线性关系

// 错误 — 除法捷径
B105: =B88/(1+(E48-0.07))      // 不重新计算完整DCF
```

**不要留占位符文本：**
```
// 错误 — 占位符注释
"注意：使用Excel数据表功能（数据 → 假设分析 → 数据表）填充敏感性表。"

// 错误 — 空单元格
[留空单元格因为"这很复杂"]
```

**不要混淆术语：**
- ❌ "敏感性表需要Excel的数据表功能"（不 — 那是我们无法使用的特定Excel工具）
- ✅ "敏感性表是每个单元格有公式的简单网格"（是的 — 这是我们构建的）

**为什么这些捷径是错误的：**
- 线性近似公式实际上不会重新计算DCF — 只是应用简单的数学调整
- 关系不是线性的，所以结果将不准确
- 占位符文本需要手动用户干预
- 模型交付时不能立即使用
- 不专业或不适合客户
- 空单元格 = 不完整的交付物

**需要拒绝的常见合理化：**
"写75+个公式感觉很复杂，所以我将留个注释让用户手动完成。"

**现实：** 当您在Python中使用openpyxl循环时，写75个公式是直接的。每个公式遵循相同的模式 — 只是替换行/列值。这是交付物的必需部分。

**代替：** 用重新计算该特定假设组合完整DCF的公式填充每个敏感性单元格

### 错误：缺少单元格注释

**不要这样做：**
- 创建所有硬编码输入而不加注释
- 认为"我稍后会添加它们"
- 写"TODO: 添加来源"
- 留下没有文档的蓝色输入

**为什么错误：**
- 无法验证数据来源
- 违反xlsx技能要求
- 不是审计就绪
- 以后修复浪费时间和资源

**代替：** 在创建每个单元格时添加注释，而不是最后

### 错误：公式行引用偏移

**症状：**
FCF部分引用错误的假设行：
`D&A:  =E29*$E$34    // 应该是$E$21，但引用了错误的行`
`CapEx: =E29*$E$41   // 应该是$E$22，但行偏移了`

**为什么会发生：**
1. 先写公式
2. 然后插入标题
3. 所有行引用偏移
4. 现在公式指向错误的单元格 → #REF!错误

**代替：** 先锁定行布局，然后写公式

### 错误：每个假设跨情景的单行

**不要像这样构建假设：**
```csv
Assumption,Bear,Base,Bull
收入增长 FY1,10%,13%,16%
收入增长 FY2,9%,12%,15%
```
这种垂直布局使得很难看到每个情景内跨年份的进展。

**为什么错误：**
- 难以看到每个情景内的假设跨年份演变
- 难以比较跨整个预测期的情景假设
- 对于审查情景逻辑不够直观

**代替：**
- 为每个情景（熊市、基础、牛市）创建单独的块
- 在每个块内跨预测年份水平显示假设
- 这使得每个情景的假设更容易作为连贯集合来审查

### 错误：无边框

**不要交付没有边框的模型：**
- 无部分区分
- 所有单元格混在一起
- 难以阅读和不专业

**为什么错误：**
- 不适合客户
- 难以浏览
- 看起来业余

**代替：** 在所有主要部分周围添加边框

### 错误：错误的字体颜色或无字体颜色区分

**不要这样做：**
- 所有文本都是黑色
- 只使用填充颜色（无字体颜色变化）
- 混淆哪些单元格是蓝色 vs 黑色

**为什么错误：**
- 无法区分输入和公式
- 审计变得不可能
- 违反xlsx技能要求

**代替：** 所有硬编码输入用蓝色文本，所有公式用黑色文本，跨表链接用绿色

### 错误：运营费用基于毛利润

**不要这样做：**
`S&M: =E33*0.15    // E33 = 毛利润（错误）`

**为什么错误：**
- 运营费用随收入而非毛利润规模
- 产生不现实的利润率进展
- 不是企业实际运营方式

**代替：**
`S&M: =E29*0.15    // E29 = 收入（正确）`

### 前5大错误汇总

1. **公式行引用偏移** → 在写公式之前定义所有行位置
2. **缺少单元格注释** → 在创建单元格时添加注释，而不是最后
3. **简化敏感性表** → 用完整DCF重新计算公式填充所有单元格，而不是近似
4. **情景块引用错误** → 确保IF公式从正确的熊市/基础/牛市块拉取
5. **无边框** → 添加专业的部分边框以适合客户的外观

此外，注意这些错误：

### WACC计算错误
- 在资本结构中混淆账面价值和市值
- 错误使用股权beta而非资产/无杠杆beta
- 错误应用税率到债务成本
- 错误的无风险利率（必须使用当前10年期国债）
- 未能调整净债务 vs 净现金状况

### 增长假设缺陷
- 永续增长 > WACC（创造无限价值）
- 预测增长与历史表现不一致
- 忽略行业增长约束
- 收入增长与单位经济不一致
- 利润率扩张没有运营理由

### 终值错误
- 使用错误的增长方法（永续 vs 退出倍数）
- 终值 >80%的企业价值（表明过度依赖）
- 终值利润率与稳态假设不一致
- 终值错误的折现期

## 金融术语和知识详细解释

### 1. DCF（折现现金流）估值法

**定义**：DCF是一种内在价值估值方法，通过预测公司未来产生的自由现金流，并将其折现回现值来确定公司的公平价值。

**核心原理**：
```
今天的100元比明年今天的100元更值钱
因为今天的100元可以投资并获得回报
```

**折现公式**：
```
现值 = 未来现金流 / (1 + 折现率)^年数
```

**为什么重要**：
- 基于公司基本面（现金流）而非市场情绪
- 估值领域最广泛使用的方法之一
- 投资银行标准做法
- 适用于任何产生正现金流的公司

---

### 2. WACC（加权平均资本成本）

**定义**：WACC是公司为融资其运营和增长而支付的平均成本，反映了债权人和股东的综合回报要求。

**计算公式**：
```
WACC = (权益权重 × 权益成本) + (债务权重 × 税后债务成本)
```

**组成部分**：

| 组成部分 | 定义 | 如何估算 |
|---------|------|---------|
| 无风险利率 | 政府债券收益率 | 10年期国债收益率 |
| Beta | 股票相对市场的波动性 | 5年月度回归 |
| 股权风险溢价 | 投资者承担股票风险的额外回报 | 市场标准5-6% |
| 债务成本 | 借款利率 | 基于信用评级或利息费用 |

**为什么重要**：
- 决定了公司价值的时间成本
- 用于折现未来现金流
- 影响资本结构决策
- 典型的WACC范围：7-15%

---

### 3. 终值（Terminal Value）

**定义**：终值是预测期结束后公司继续运营所产生的价值，通常占DCF模型企业价值的50-70%。

**两种计算方法**：

**永续增长法**：
```
终值 = 最后一年FCF × (1 + 永续增长率) / (WACC - 永续增长率)
```

**退出倍数法**：
```
终值 = 最后一年EBITDA × 行业倍数
```

**为什么重要**：
- DCF模型中最大的价值驱动因素
- 对假设高度敏感
- 需要与行业可比公司一致
- 永续增长率通常 ≤ GDP增长

---

### 4. 无杠杆自由现金流（Unlevered Free Cash Flow）

**定义**：无杠杆FCF是公司在支付运营费用、资本支出和税款后可产生的现金流，不考虑融资活动。

**计算公式**：
```
无杠杆FCF = EBIT
          - 税款（EBIT × 税率）
          + D&A
          - CapEx
          +/- 营运资本变化
```

**为什么重要**：
- 反映企业核心运营产生的现金
- 不受资本结构影响
- 用于企业价值估值
- 比净利润更可靠的价值指标

---

### 5. 资本资产定价模型（CAPM）

**定义**：CAPM计算投资者投资股票所要求的回报，基于其系统性风险。

**公式**：
```
权益成本 = 无风险利率 + Beta × 股权风险溢价
```

**各组成部分解读**：

| 参数 | 含义 | 典型值 |
|------|------|-------|
| 无风险利率 | 投资无风险资产的回报 | 10年期国债 |
| Beta | 股票与市场的共同波动 | 0.8-1.5 |
| ERP | 股票相对于无风险资产的额外回报 | 5-6% |

**为什么重要**：
- 计算WACC的核心输入
- 衡量系统性风险
- 投资决策的关键参考
- 市场共识的标准化方法

---

### 6. 企业价值（Enterprise Value, EV）

**定义**：企业价值是公司总价值的度量，包括所有债权人和股东权益。

**计算公式**：
```
EV = 市值 + 净债务
   = 普通股市值 + 优先股市值 + 少数股东权益 + 总债务 - 现金
```

**为什么重要**：
- 用于收购和并购估值
- 计算EV/EBITDA倍数的基础
- 比市值更全面的价值指标
- 反映所有资本提供者的债权

---

### 7. 股权价值（Equity Value）

**定义**：股权价值是股东在公司中拥有的价值，即扣除所有债务后的企业价值。

**桥梁**：
```
股权价值 = 企业价值 - 净债务
        = 企业价值 - (总债务 - 现金)
```

**每股价值**：
```
每股价值 = 股权价值 / 稀释流通股
```

**为什么重要**：
- 直接比较公开股价
- 估值最终输出
- 投资决策依据
- 收购定价基础

---

### 8. 敏感性分析（Sensitivity Analysis）

**定义**：敏感性分析研究关键假设变化如何影响估值结果。

**常见敏感性表**：

| 表1 | WACC vs 永续增长率 |
|-----|-------------------|
| 表2 | 收入增长 vs EBIT利润率 |
| 表3 | Beta vs 无风险利率 |

**为什么重要**：
- 识别关键价值驱动因素
- 评估模型不确定性
- 支持投资决策
- 风险管理工具

---

### 9. 情景分析（Scenario Analysis）

**定义**：情景分析为同一公司构建多个估值，基于不同的宏观和公司特定假设。

**三种情景**：

| 情景 | 描述 | 增长率特征 |
|------|------|-----------|
| 熊市 | 悲观假设 | 低于平均 |
| 基础 | 最可能假设 | 符合指引 |
| 牛市 | 乐观假设 | 高于平均 |

**为什么重要**：
- 全面了解风险/回报
- 识别关键驱动因素
- 投资委员会报告标准
- 决策支持工具

---

### 10. NOPAT（税后净营业利润）

**定义**：NOPAT是假设公司没有债务时的税后营业利润。

**计算公式**：
```
NOPAT = EBIT × (1 - 税率)
```

**与净利润的区别**：
- NOPAT不扣除利息费用
- 反映假设全权益融资的利润
- 用于计算无杠杆FCF
- 更好的运营绩效指标

**为什么重要**：
- 无杠杆FCF计算的起点
- 消除资本结构影响
- 便于公司间比较
- 管理层业绩衡量指标

## Appendix: 金融背景知识

这份文件是"现金流折现模型（DCF Model）"技能的详细说明。DCF 是投资银行和股权研究中最核心的估值方法——它把一个公司的"未来所有赚钱能力"折算成今天的价值。

---

### 1. 什么是 DCF？

**类比：**
想象你的邻居老王说："我给你一个选择——今天给你 1000 元，或者一年后给你 1100 元。"你会选哪个？

如果你选了今天的 1000 元，说明你心目中"今天的 1000 元 > 一年后的 1100 元"。这是因为：
- 今天拿到钱可以投资（放银行赚利息）
- 一年后老王可能赖账（风险）
- 一年后的钱买不到今天这么多东西（通胀）

DCF 就是把这个逻辑扩展到公司估值上——把公司未来 5-10 年能赚的所有现金，全部打一个"折扣"折回今天的价值。

---

### 2. DCF 的"核心公式"

```
企业价值（EV）= FCF₁/(1+r)¹ + FCF₂/(1+r)² + ... + FCFₙ/(1+r)ⁿ + 终值/(1+r)ⁿ
```

其中：
- FCF = 自由现金流（公司真正可以自由支配的现金）
- r = 折现率（WACC，加权平均资本成本）
- n = 预测年数
- 终值 = 预测期之后的剩余价值

---

### 3. DCF 的"六大步骤"

| 步骤 | 内容 | 常见错误 |
|------|------|---------|
| 1 | 预测未来 5-10 年自由现金流 | 增长率太乐观 |
| 2 | 计算加权平均资本成本（WACC） | 用错了 Beta |
| 3 | 计算终值（Terminal Value） | 永续增长率太高 |
| 4 | 折现所有现金流到今天的价值 | 忘记折现终值 |
| 5 | 减去净负债 = 股权价值 | 忘了现金也算资产 |
| 6 | 除以流通股数 = 每股价值 | 忘了稀释性证券 |

---

### 4. WACC 怎么算？

```
WACC = E/(E+D) × Ke + D/(E+D) × Kd × (1-t)

其中：
E = 股权市值, D = 债务市值
Ke = 股权成本 = Rf + β × (Rm - Rf)
Kd = 债务成本 = 公司借款利率
t = 税率
```

**举例**：如果 Rf（无风险利率）=3%, β=1.2, 市场风险溢价=5%
→ Ke = 3% + 1.2×5% = 9%

如果公司 80% 股权 + 20% 债务，债务利率 5%
→ WACC = 80%×9% + 20%×5%×(1-21%) = 7.2% + 0.79% = 7.99%

---

### 5. DCF 的优缺点

| 优点 | 缺点 |
|------|------|
| 理论上最严谨 | 对假设极其敏感 |
| 不受市场情绪影响 | WACC 变 1% → 估值变 15% |
| 能分析"值不值" | 终值通常占总估值 60-80% |
| 适用于任何公司 | 现金流难预测的公司不适合 |

---

### 给小白的一句话

> DCF 就是金融估值的"核武器"——它把公司未来所有的赚钱能力（自由现金流）全部打折扣折回今天的价值。但这把"核武器"有个致命弱点：你输入的"假设"稍有偏差，算出来的"目标价"可能差 50%。比如你把"增长率"从 5% 改成 6%，估值可能从 $40 飙到 $55。所以聪明的分析师不只看 DCF 得出一个数字，而是看 DCF 的"敏感性范围"——"在最好情况下值 $55，最坏情况下值 $30，当前股价 $42，风险回报比还行"。
