# SEC备案数据提取参考

**使用时机：** 仅当模型模板特别要求从SEC备案（10-K、10-Q）提取数据时才参考此文件。对于直接提供数据或使用其他数据源的模板，不需要此参考。

---

## 从SEC备案提取数据（10-K/10-Q）

从SEC备案填充模型模板时，直接从SEC备案提取财务数据。

### 第1步：定位备案

1. 使用SEC EDGAR：`https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany&CIK=[股票代码]&type=10-K`
2. 对于季度数据，使用`type=10-Q`

### 第2步：识别报告货币

提取数据前，识别报告货币：
- 查看封面或标题的货币标注
- 查看报表标题（例如，"以千美元为单位"）
- 审查附注1（重要会计政策摘要）

**常见货币指标**

| 指标 | 货币 |
|-----------|----------|
| $、USD | 美元 |
| €、EUR | 欧元 |
| £、GBP | 英镑 |
| ¥、JPY | 日元 |
| ¥、CNY、RMB | 人民币 |
| CHF | 瑞士法郎 |
| CAD、C$ | 加元 |

将模型货币设置为与备案匹配；在假设选项卡中记录。

### 第3步：导航至财务报表

在10-K或10-Q中，找到：
- **第8项**（10-K）或**第1项**（10-Q）：财务报表
- 要提取的关键部分：
  - 合并经营报表（利润表）
  - 合并资产负债表
  - 合并现金流量表
  - 财务报表附注（用于明细表详情）

### 第4步：数据提取映射

**利润表（来自合并经营报表）**

| 备案行项目 | 模型行项目 |
|------------------|-----------------|
| 净收入/净销售额 | 收入 |
| 销售成本 | COGS |
| 销售、一般和行政费用 | SG&A |
| 折旧和摊销 | D&A |
| 利息费用净额 | 利息费用 |
| 所得税费用 | 税款 |
| 净利润 | 净利润 |

**资产负债表（来自合并资产负债表）**

| 备案行项目 | 模型行项目 |
|------------------|-----------------|
| 现金及现金等价物 | 现金 |
| 应收账款净额 | AR |
| 存货 | 存货 |
| 物业、厂房和设备净额 | PP&E（净额） |
| 总资产 | 总资产 |
| 应付账款 | AP |
| 短期债务/长期债务流动部分 | 流动债务 |
| 长期债务 | LT债务 |
| 留存收益 | 留存收益 |
| 股东权益合计 | 权益合计 |

**现金流量表（来自合并现金流量表）**

| 备案行项目 | 模型行项目 |
|------------------|-----------------|
| 净利润 | 净利润 |
| 折旧和摊销 | D&A |
| 应收账款变化 | ΔAR |
| 存货变化 | Δ存货 |
| 应付账款变化 | ΔAP |
| 资本支出 | CapEx |
| 发行普通股收益 | 权益发行 |
| 债务发行/偿还收益 | 债务活动 |
| 支付的股息 | 股息 |

### 第5步：从附注提取支持详情

对于明细表，从财务报表附注中提取：
- **附注：债务** → 到期安排、利率、契约
- **附注：物业、厂房和设备** → 总PP&E、累计折旧、使用寿命
- **附注：收入** → 分部细分、地区拆分
- **附注：租赁** → 运营与融资租赁义务

### 第6步：历史数据要求

最少提取3年历史数据：
- 10-K提供3年IS/CF、2年BS
- 对于第3年BS，从上年10-K提取
- 如需要，可使用10-Q填补季度粒度

### 数据提取清单

- 识别报告货币和比例（千、百万）
- 3年历史利润表
- 3年历史现金流量表
- 3年历史资产负债表
- 验证IS净利润 = CF期初净利润（每年）
- 验证BS现金 = CF期末现金（每年）
- 从附注提取债务到期安排
- 提取D&A详情或使用寿命假设
- 注意任何需要正常化的一次性/非经常性项目

### 处理常见备案差异

| 差异 | 如何处理 |
|-----------|------------|
| D&A嵌入COGS/SG&A | 从现金流量表提取D&A |
| "其他"行项目金额重大 | 查看附注细分 |
| 重述 | 使用重述数据，在假设中注明 |
| 财年 ≠ 日历年 | 用财年截止日标记（例如，FYE 2025年1月） |
| 非美元报告货币 | 调整模型货币以匹配备案 |

## 金融术语和知识详细解释

### 1. SEC备案概述

**SEC（证券交易委员会）**：
- 美国证券市场的监管机构
- 要求上市公司定期提交财务报告
- EDGAR系统提供公开访问

**主要备案类型**：

| 备案 | 频率 | 内容 |
|------|------|------|
| 10-K | 每年 | 年度报告，全面财务信息 |
| 10-Q | 每季度 | 季度报告，简要财务信息 |
| 8-K | 事件驱动 | 重大事件及时披露 |

---

### 2. 10-K结构解析

**10-K年度报告的主要部分**：

| 部分 | 内容 |
|------|------|
| 第1项 | 业务概述 |
| 第1A项 | 风险因素 |
| 第2项 | 属性列表 |
| 第3项 | 法律程序 |
| 第7项 | 管理层讨论与分析（MD&A） |
| 第8项 | 财务报表和补充数据 |
| 第15项 | 附表和报告确认 |

**最重要的部分**：
- **MD&A**：管理层对财务业绩的讨论
- **财务报表**：审计后的三表
- **附注**：重要会计政策和或有事项

---

### 3. EDGAR系统使用

**EDGAR搜索**：
- 网站：`https://www.sec.gov/cgi-bin/browse-edgar`
- 使用股票代码(CIK)搜索公司
- 按类型筛选（10-K、10-Q等）
- 可下载PDF和HTML格式

**快速查找技巧**：
- 在HTML版本中使用"Ctrl+F"搜索关键词
- PDF版本更适合打印和存档
- 交互式数据（XBRL）可提取特定数字

---

### 4. 收入确认原则

**ASC 606（IFRS 15同等）**：
- 五步法确认收入：
  1. 识别与客户的合同
  2. 识别履约义务
  3. 确定交易价格
  4. 分配交易价格
  5. 在履约时确认收入

**重要性**：
- 不同公司可能使用不同收入确认政策
- 影响收入的时间点
- 可能导致收入与现金流不匹配

---

### 5. 正常化调整（Normalized Adjustments）

**常见正常化项目**：

| 项目 | 调整原因 |
|------|---------|
| 重组费用 | 非经常性业务出售 |
| 诉讼费用 | 一次性法律和解 |
| 资产减值 | 非现金费用 |
| 衍生品收益/损失 | 市场价值波动 |
| 税率变化 | 会计政策变更 |

**为什么重要**：
- 使历史数据更具可比性
- 更好地反映核心业务表现
- 估值中使用的"调整后"指标

---

### 6. XBRL（可扩展商业报告语言）

**定义**：SEC要求上市公司以XBRL格式提交财务数据，这是一种标准化标记语言。

**用途**：
- 便于数据提取和分析
- 支持自动化财务比较
- 提高财务信息透明度

**在财务建模中的应用**：
- 可直接从XBRL标签提取数据
- 减少手动输入错误
- 加快模型构建速度

## Appendix: 金融背景知识

这份文件是"SEC 申报文件（SEC Filings）"参考文档的详细说明。在美国，所有上市公司必须定期向 SEC（美国证券交易委员会）提交财务报告——这些公开文件是分析师获取公司信息的"第一手资料"。

---

### 1. 什么是 SEC 申报文件？

**类比：**
想象你是一个体检中心。所有参加医保的市民必须每年体检，并把体检报告提交给医保局。这些报告公开供任何人查阅——你可以知道任何一个市民的身高、体重、血压、胆固醇。

SEC 申报文件就是美国上市公司的"体检报告"——每季度/每年必须向 SEC 提交，公开供所有人查阅。

---

### 2. 最重要的"五大 SEC 文件"

| 文件 | 英文 | 频率 | 内容 |
|------|------|------|------|
| 10-K | Annual Report | 每年 | 完整年报：业务描述、财务数据、风险因素 |
| 10-Q | Quarterly Report | 每季度 | 季报：简略版 10-K |
| 8-K | Current Report | 事件触发 | 重大事件：并购、破产、管理层变动 |
| S-1 | IPO Registration | 上市时 | 招股说明书 |
| Proxy | DEF 14A | 每年 | 股东大会资料、高管薪酬 |

---

### 3. 10-K 年报的"四大篇章"

| 部分 | 内容 | 分析师看什么 |
|------|------|------------|
| Part 1 | 业务描述、风险因素 | 竞争优势、关键风险 |
| Part 2 | 财务数据、MD&A | 收入趋势、管理层讨论 |
| Part 3 | 管理层、薪酬 | CEO/CFO 背景、薪酬结构 |
| Part 4 | 附件、签名 | 财务报表附注 |

**最核心的是 MD&A（管理层讨论与分析）** —— 管理层用自己的话解释"今年发生了什么、为什么"。

---

### 4. "读 10-K"的三个境界

| 境界 | 水平 | 表现 |
|------|------|------|
| 初级 | 看数字 | 只看收入、利润、EPS |
| 中级 | 看附注 | 关注表外负债、会计政策变化 |
| 高级 | 看措辞 | 对比今年的措辞 vs 去年，发现微妙变化 |

**高级技巧：** 比如去年的 MD&A 说"我们持续看到强劲增长"，今年变成"尽管环境充满挑战，我们依然实现增长"——词语微妙变化意味着管理层在"打预防针"。

---

### 5. 真实案例：Enron 的 10-K 中有答案

**背景**：2001 年安然（Enron）破产——历史上最大的公司丑闻

**如果在破产前仔细读 10-K**：
- "Related party transactions"（关联交易）附注长达 3 页——非常不寻常
- 负债表外有几十亿美元的"特殊目的实体"（SPV）——字很小但很重要
- MD&A 中管理层说"情况很好"——但数字上利润质量越来越差

**教训**：答案在 10-K 里——只是当时没有人仔细读。

---

### 给小白的一句话

> SEC 申报文件就是美国上市公司的"体检报告"——10-K（年报）、10-Q（季报）、8-K（突发事件）是每个分析师必须会读的三样东西。10-K 最详细但最难读（200-300 页），10-Q 是季报（50-80 页）你每季度看，8-K 是"突发新闻"（并购、高管离职）你第一时间追。巴菲特的秘诀是：读 10-K 不要从第 1 页开始——先读 MD&A（管理层自己的解释），再读财务报表，最后看风险因素。因为 MD&A 里的措辞变化，常常比财务数字更早透露公司的真实状况。
