# 电子表格审计

审计公式和数据的准确性，检查错误。审计深度取决于范围——从对选定区域的快速公式检查到完整的财务模型完整性审计。

## 步骤 1：确定审计范围

如果用户已经指定了范围，则使用该范围。否则**询问用户**：

> 您希望我审计什么范围？
> - **selection** — 仅当前选定的区域
> - **sheet** — 仅当前活动工作表
> - **model** — 整个工作簿，包括财务模型完整性检查（资产负债表平衡、现金对账、结转、逻辑合理性）

**model** 范围是最深层次的审计——在将 DCF、LBO、三表模型、合并模型、可比公司分析或任何整合财务模型发送给客户或内部使用之前使用此范围。

---

## 步骤 2：公式级检查（所有范围）

无论范围如何，都运行以下检查：

| 检查项 | 检查内容 |
|---|---|
| 公式错误 | `#REF!`、`#VALUE!`、`#N/A`、`#DIV/0!`、`#NAME?` |
| 公式中的硬编码 | `=A1*1.05` — `1.05` 应该是单元格引用 |
| 不一致的公式 | 破坏行/列中相邻单元格模式的公式 |
| 偏移一个单位的范围 | `SUM`/`AVERAGE` 遗漏首行或末行 |
| 被粘贴覆盖的公式 | 看起来像公式但实际上是硬编码值的单元格 |
| 循环引用 | 有意或无意的循环引用 |
| 损坏的跨表链接 | 引用已移动或删除的单元格 |
| 单位/规模不匹配 | 千位与百万位混合，百分比存储为整数 |
| 隐藏行/工作表 | 可能包含覆盖值或过时计算 |

---

## 步骤 3：模型完整性检查（仅 MODEL 范围）

如果范围是 **model**，识别模型类型（DCF / LBO / 三表模型 / 合并模型 / 可比分析 / 自定义）并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项 | 检查内容 |
|---|---|
| 输入/公式分离 | 输入是否与计算清晰分离？ |
| 颜色约定 | 蓝色=输入，黑色=公式，绿色=链接——或模型使用的任何约定，是否一致应用？ |
| 工作表流程 | 逻辑顺序（假设 → 损益表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期标题 | 所有工作表一致？ |
| 单位 | 一致（千位 vs 百万位 vs 实际值）？ |

### 3b. 资产负债表

| 检查项 | 测试 |
|---|---|
| 资产负债表平衡 | 总资产 = 总负债 + 权益（每个期间） |
| 留存收益结转 | 上期留存收益 + 净利润 − 股息 = 当前留存收益 |
| 商誉/无形资产 | 来自收购假设（如涉及并购） |

如果资产负债表不平衡，**量化每个期间的差距并追踪问题所在**——在此修复之前，其他一切都不重要。

### 3c. 现金流量表

| 检查项 | 测试 |
|---|---|
| 现金对账 | 现金流量表期末现金 = 资产负债表现金（每个期间） |
| 现金流量求和 | 经营现金流 + 投资现金流 + 融资现金流 = 现金变动 |
| 折旧摊销匹配 | 现金流量表上的折旧摊销 = 损益表上的折旧摊销 |
| 资本支出匹配 | 现金流量表上的资本支出与资产负债表上的PP&E结转匹配 |
| 营运资金变动 | 符号与资产负债表变动匹配（应收账款变动、应付账款变动、存货变动） |

### 3d. 损益表

| 检查项 | 测试 |
|---|---|
| 收入构建 | 与部门/产品明细一致 |
| 税务 | 税费支出 = 税前收入 × 税率（允许递延税项调整） |
| 股份数量 | 与稀释计划表一致（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是 LBO/三表模型中常见的有意循环
- 如果是有意的：验证迭代切换存在且正常工作
- 如果是无意的：追踪循环并指出如何打破它

### 3f. 逻辑与合理性

| 检查项 | 标记条件 |
|---|---|
| 增长率 | 收入增长 > 100% 且无解释 |
| 利润率 | 超出行业标准 |
| 终端价值主导 | 终端价值 > DCF企业价值的约75%（黄色警告） |
| 曲棍球棒效应 | 预测在远期年份不切实际地增长 |
| 复合增长 | EBITDA到第10年复合增长到荒谬的金额 |
| 边界情况 | 模型在0%或负增长、负EBITDA、杠杆为负时崩溃 |

### 3g. 特定模型类型的错误

**DCF模型：**
- 贴现率应用于错误期间（年中 vs 年末）
- 终端价值未贴现回现
- WACC使用账面价值而非市场价值
- 自由现金流包含利息费用（应该是无杠杆的）
- 税盾重复计算

**LBO模型：**
- 债务偿还与现金扫尾机制不匹配
- PIK利息未累加到本金
- 管理层滚动投资未反映在回报中
- 退出倍数应用于错误的EBITDA（LTM vs NTM）
- 费用未从首日权益中扣除

**合并模型：**
- 摊薄/增厚分析使用错误的股份数（交易前 vs 交易后）
- 协同效应未分阶段实现
- 购买价格分配不平衡
- 未包含现金的机会成本
- 交易费用未列入资金来源与用途

**三表模型：**
- 营运资金变动符号错误
- 折旧与PP&E计划表不匹配
- 债务到期时间表与本金偿还不匹配
- 股息超过净利润且无解释

## 步骤 4：报告

输出发现结果表格：

| # | 工作表 | 单元格/范围 | 严重性 | 类别 | 问题 | 建议修复 |
|---|---|---|---|---|---|---|

**严重性等级：**
- **Critical（严重）** — 输出错误（资产负债表不平衡、公式损坏、现金不对账）
- **Warning（警告）** — 风险（硬编码、不一致的公式、边界情况失败）
- **Info（信息）** — 风格/最佳实践（颜色编码、布局、命名）

对于 **model** 范围，在报告前添加摘要行：

> 模型类型: [DCF/LBO/三表/...] — 总体: [Clean / Minor Issues / Major Issues] — [N] critical, [N] warnings, [N] info

**未经询问不要更改任何内容** — 先报告，根据请求修复。

## 注意事项

- **先确保资产负债表平衡** — 如果不平衡，下游所有内容都值得怀疑
- **硬编码覆盖是沉默错误的头号来源** — 积极搜索
- **符号约定错误**（现金流出的正负号）极其常见
- 如果模型使用VBA宏，注意任何无法仅从公式审计的宏驱动计算

## 金融术语解释

### 1. DCF（折现现金流）

**定义**：DCF（Discounted Cash Flow）是一种估值方法，通过预测公司未来产生的现金流并将其折现回当前价值来评估公司价值。

**核心原理**：
- 预测未来自由现金流
- 选择合适的贴现率（通常是WACC）
- 将未来现金流折现到现值
- 加上终端价值得到企业价值

**关键组成部分**：
- **自由现金流**：经营活动产生的现金减去资本支出
- **贴现率**：反映投资风险的回报率
- **终端价值**：预测期结束后公司的价值

**应用场景**：
- 企业估值
- 投资决策
- 并购交易定价

### 2. LBO（杠杆收购）

**定义**：LBO（Leveraged Buyout）是一种收购方式，收购方使用大量债务融资来购买目标公司的股权。

**特点**：
- 高杠杆：债务占融资比例通常超过50%
- 目标公司资产作为贷款抵押
- 目标公司现金流用于偿还债务

**流程**：
1. 收购方成立特殊目的实体
2. 借入资金（通常占60-80%）
3. 用借款收购目标公司
4. 用目标公司现金流偿还债务
5. 几年后通过IPO或出售退出

**典型参与者**：
- 私募股权公司
- 投资银行（提供融资）
- 目标公司管理层

### 3. 三表模型（3-Statement Model）

**定义**：三表模型是将损益表、资产负债表和现金流量表整合在一起的财务模型，展示三者之间的内在联系。

**三张表的关系**：
- **损益表**：展示公司的盈利能力（收入、费用、利润）
- **资产负债表**：展示公司的财务状况（资产、负债、权益）
- **现金流量表**：展示公司的现金流动情况（经营、投资、融资活动）

**关键勾稽关系**：
- 净利润影响留存收益
- 留存收益影响股东权益
- 现金流量表期末现金等于资产负债表现金
- 折旧摊销连接损益表和现金流量表

### 4. WACC（加权平均资本成本）

**定义**：WACC（Weighted Average Cost of Capital）是公司融资的平均成本，考虑了股权和债务的成本。

**计算公式**：
- WACC = (E/V × Re) + (D/V × Rd × (1 - Tc))
- E = 股权价值
- D = 债务价值
- V = E + D（企业价值）
- Re = 股权成本
- Rd = 债务成本
- Tc = 税率

**用途**：
- 作为DCF模型的贴现率
- 评估投资项目的门槛回报率
- 公司资本预算决策

### 5. 自由现金流（FCF）

**定义**：自由现金流是公司在支付所有运营费用和资本支出后剩余的现金。

**计算公式**：
- FCF = 经营现金流 - 资本支出
- 或 FCF = EBIT × (1 - 税率) + 折旧摊销 - 资本支出 - 营运资金变动

**类型**：
- **无杠杆自由现金流（UFCF）**：不考虑利息费用，用于企业估值
- **杠杆自由现金流（LFCF）**：考虑利息费用，用于股权估值

**重要性**：
- 衡量公司真正创造现金的能力
- 可用于股息、股票回购或再投资
- DCF估值的核心输入

### 6. 终端价值（Terminal Value）

**定义**：终端价值是指预测期结束后公司的价值，通常占DCF估值的很大比例。

**计算方法**：
1. **永续增长法**：TV = FCF × (1 + g) / (WACC - g)
   - g = 长期增长率
2. **退出倍数法**：TV = EBITDA × 退出倍数

**注意事项**：
- 终端价值通常占DCF总价值的50-80%
- 如果终端价值超过75%，可能表示预测期过短或假设过于激进

### 7. EBITDA

**定义**：EBITDA代表息税折旧摊销前利润，是衡量公司核心业务盈利能力的指标。

**计算公式**：
- EBITDA = 营业收入 + 折旧 + 摊销

**用途**：
- 比较不同资本结构的公司
- 作为估值倍数的基础（EV/EBITDA）
- 评估经营现金流的近似值

**局限性**：
- 忽略资本支出需求
- 不考虑利息和税费
- 可能被管理层操纵

### 8. 营运资金（Working Capital）

**定义**：营运资金是公司用于日常运营的资金，等于流动资产减去流动负债。

**计算公式**：
- 营运资金 = 流动资产 - 流动负债
- 营运资金变动 = 当期营运资金 - 上期营运资金

**组成部分**：
- 流动资产：现金、应收账款、存货等
- 流动负债：应付账款、短期债务等

**重要性**：
- 衡量公司的短期偿债能力
- 影响现金流量表
- 营运资金增加会减少现金流

### 9. 循环引用（Circular Reference）

**定义**：循环引用是指公式直接或间接地引用自身所在的单元格。

**类型**：
- **有意循环引用**：在LBO和三表模型中常见，用于计算利息、债务和现金之间的关系
- **无意循环引用**：公式错误导致的意外循环

**处理方法**：
- 在Excel中启用迭代计算
- 设置最大迭代次数
- 检查并修复无意循环

### 10. 摊薄/增厚分析（Accretion/Dilution Analysis）

**定义**：在并购交易中，分析交易对每股收益（EPS）的影响。

**计算方法**：
- 预计EPS变化 = (合并后EPS - 收购方当前EPS) / 收购方当前EPS
- **摊薄**：EPS下降
- **增厚**：EPS上升

**考虑因素**：
- 融资结构（现金、股票、债务）
- 协同效应
- 交易费用
- 融资成本

### 11. 可比公司分析（Comps Analysis）

**定义**：通过比较目标公司与类似上市公司的估值倍数来确定公司价值。

**步骤**：
1. 选择可比公司
2. 收集财务数据
3. 计算估值倍数（P/E、EV/Revenue、EV/EBITDA等）
4. 应用倍数到目标公司

**优点**：
- 基于市场数据，客观透明
- 易于理解和解释

**局限性**：
- 难以找到完全可比的公司
- 市场情绪可能影响倍数

### 12. 敏感性分析（Sensitivity Analysis）

**定义**：分析关键假设变化对模型输出的影响。

**常见应用**：
- 收入增长率变化对估值的影响
- 贴现率变化对NPV的影响
- 退出倍数变化对IRR的影响

**展示方式**：
- 单因素敏感性表
- 双因素敏感性矩阵
- 龙卷风图

**用途**：
- 识别关键价值驱动因素
- 评估投资风险
- 支持决策制定

### 13. 资本支出（CapEx）

**定义**：资本支出是公司用于购买或升级长期资产的支出。

**类型**：
- 维持性资本支出：维持现有运营所需
- 增长性资本支出：扩大业务所需

**在财务模型中的处理**：
- 在现金流量表中作为投资活动现金流流出
- 影响资产负债表中的固定资产
- 通过折旧影响损益表

### 14. PIK利息（Payment-in-Kind Interest）

**定义**：PIK利息是一种利息支付方式，利息不以现金支付，而是累加到本金中。

**特点**：
- 常见于高杠杆交易（如LBO）
- 延迟现金流出
- 利息会增加债务余额

**会计处理**：
- 利息费用计入损益表
- 债务余额增加
- 不影响当期现金流

### 15. 协同效应（Synergies）

**定义**：协同效应是指并购后合并公司的价值超过单独两家公司价值之和的部分。

**类型**：
- **收入协同效应**：通过交叉销售、定价能力等增加收入
- **成本协同效应**：通过整合运营、裁员等降低成本

**实现挑战**：
- 协同效应的实现需要时间
- 整合过程可能产生额外成本
- 协同效应预测往往过于乐观

## Appendix: 金融背景知识

这份文件是"Excel 审计（Audit XLS）"技能的详细说明。在投资银行中，一个财务模型中的公式错误可能导致估值偏差数亿美元——所以在模型提交给客户之前，必须经过严格的审计。

---

### 1. 什么是 Excel 审计？

**类比：**
想象你是建筑工程师。一栋 30 层的大楼建好了，在入住之前必须经过"结构安全审查"：地基够不够稳、承重墙有没有裂缝、电线有没有安全隐患。

Excel 审计就是财务模型的"结构安全审查"——检查每一个公式、每一个假设、每一个链接是否正确。

---

### 2. Excel 审计的"四个层级"

| 层级 | 范围 | 检查内容 |
|------|------|---------|
| Selection | 选定区域 | 公式逻辑、绝对/相对引用 |
| Sheet | 单个工作表 | 公式一致性、数据类型 |
| Model | 整个工作簿 | 资产负债表平衡、勾稽关系、逻辑检查 |
| Integrity | 财务模型完整性 | 现金流、业绩合理性、审计痕迹 |

---

### 3. 财务模型完整性检查（Integrity Check）

| 检查项 | 检查方法 | 通过标准 |
|--------|---------|---------|
| 资产负债表配平 | 总资产 - 总负债 - 权益 | = 0 |
| 现金对账 | 现金流量表期末现金 = 资产负债表现金 | 一致 |
| 结转一致性 | 前一期期末 = 下一期期初 | 一致 |
| 逻辑检查 | Depreciation/PPE 比率合理 | 在正常范围 |
| 增长率检查 | 收入增长不突然 50% 又 -20% | 平滑 |

---

### 4. 投行模型中最常犯的"五大错误"

| 错误 | 后果 | 审计方法 |
|------|------|---------|
| 硬编码 | 改假设后公式不自动更新 | 遍历所有单元格找常量 |
| 循环引用 | Excel 卡死，结果不可信 | 检查循环引用开关 |
| 引用断裂 | 跨 Sheet 引用指向错误位置 | 追踪引用链 |
| 数据类型错 | 文本格式的数字不参与计算 | 检查数字格式 |
| 隐藏行列 | 遗漏了被隐藏的关键数据 | 展开所有行列 |

### 5. 真实案例：JP Morgan "伦敦鲸"事件

**背景**：2012 年，JP Morgan 的伦敦交易团队使用了一个有公式错误的 VaR（风险值）模型。

**错误**：在 Excel 中复制公式时，错误地将"除以总和"改成了"除以两个数字的平均值"，导致风险被低估了一半。

**结果**：这个错误导致 JP Morgan 损失了 $6.2B，CEO 在国会听证会上被质询。

**教训**：一个 Excel 公式错误可以毁掉一个银行。

### 给小白的一句话

> Excel 审计就是财务模型的"质量检验"——一栋大楼建成后必须有工程师检查结构安全，一个财务模型用完后也必须有审计师检查公式有没有 Bug。在投资银行，一个模型可能被送给客户、监管机构、法院——任何一个公式错了，轻则丢客户，重则被告上法庭、被罚几个亿。JP Morgan 的"伦敦鲸"事件就是血淋淋的教训：一个 Excel 复制粘贴错误导致了 $62 亿美元的损失。所以审计模型不是浪费时间，而是在替你"避免坐牢"。
