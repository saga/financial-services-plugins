# 演示文稿检查报告格式

## 报告模板

```markdown
# 演示文稿检查报告：[演示文稿名称]

## 摘要
- 问题总数：X
- 严重：X（数字不匹配、事实错误）
- 重要：X（叙述与数据一致性、语言）
- 次要：X（格式）

## 严重问题

### 数字一致性
1. **[问题名称]**（幻灯片X、Y）
   - 幻灯片X：[数值]
   - 幻灯片Y：[数值]
   - 操作：[建议]

### 数据与叙述一致性
1. **[问题名称]**（幻灯片X、Y）
   - 声明："[引用文本]"
   - 数据显示：[矛盾]
   - 操作：[建议]

## 重要问题

### 语言润色
1. **[问题类型]**（幻灯片X）
   - 当前："[引用文本]"
   - 建议："[替换文本]"

## 次要问题

### 格式
1. **[问题类型]**（幻灯片X）
   - [描述和修复建议]

## 最终检查清单
- [ ] 数字已核对
- [ ] 叙述与数据匹配
- [ ] 语言符合投资银行标准
- [ ] 图表有来源
- [ ] 格式一致
```

## 问题严重程度分类

**严重**（交付给客户前必须修复）：
- 幻灯片之间的数字不匹配
- 计算错误
- 事实不准确（名称、标题、日期）
- 数据与叙述矛盾

**重要**（应该修复）：
- 非正式语言
- 模糊且缺乏具体性的声明
- 术语不一致
- 缺少图表来源

**次要**（润色项目）：
- 字体/颜色不一致
- 日期格式变化
- 间距/对齐问题
- 孤立文本

## 金融术语解释

### 1. 演示文稿检查（Deck Check）

**定义**：演示文稿检查是对财务演示文稿进行全面审查，确保内容准确、格式规范、语言专业。

**目的**：
- 确保数字准确性
- 验证叙述与数据一致性
- 确保语言符合行业标准
- 检查格式一致性

**检查维度**：
- **内容准确性**：数字、事实、引用
- **逻辑一致性**：叙述与数据的匹配
- **语言专业性**：术语使用、表达规范
- **格式规范性**：字体、颜色、布局

### 2. 投资银行标准（IB Standards）

**定义**：投资银行标准是指投资银行行业普遍遵循的专业规范和最佳实践。

**内容标准**：
- 数据准确性和一致性
- 叙述的严谨性和逻辑性
- 术语使用的专业性

**格式标准**：
- 字体和字号规范
- 颜色方案
- 布局和间距
- 图表样式

### 3. 数据叙述一致性（Data-Narrative Alignment）

**定义**：数据叙述一致性是指演示文稿中的文字叙述与展示的数据保持一致，不出现矛盾。

**重要性**：
- 增强可信度
- 避免误导观众
- 体现专业性

**检查要点**：
- 文字描述与图表数据一致
- 增长率等派生指标正确
- 预测数据与假设一致

### 4. 数字一致性（Number Consistency）

**定义**：数字一致性是指在演示文稿的不同部分，相同的数据保持一致。

**常见问题**：
- 同一数字在不同幻灯片上显示不同值
- 格式不一致（如 $100M vs $100,000,000）
- 单位不一致（如千位 vs 百万位）

**检查方法**：
- 全局搜索相同数字
- 验证派生指标计算
- 核对数据源

### 5. 图表来源（Chart Sources）

**定义**：图表来源是指图表数据的出处，通常标注在图表下方或页脚。

**重要性**：
- 增强数据可信度
- 满足合规要求
- 便于追溯数据

**规范**：
- 清晰标注数据来源
- 包括时间范围
- 注明数据提供商

### 6. 专业语言（Professional Language）

**定义**：专业语言是指在商业和金融领域使用的正式、精确的表达方式。

**特点**：
- 使用行业术语
- 避免口语化表达
- 保持客观中立
- 数据支持论点

**常见问题**：
- 使用缩写形式
- 使用第一人称
- 使用无依据的最高级
- 使用模糊量词

### 7. 格式一致性（Formatting Consistency）

**定义**：格式一致性是指演示文稿在字体、颜色、布局等方面保持统一的风格。

**要素**：
- **字体**：标题、正文、注释的字体规范
- **颜色**：品牌颜色的使用
- **布局**：对齐方式、间距
- **图表**：图表样式和元素

**重要性**：
- 提升视觉效果
- 体现专业性
- 便于阅读

### 8. 事实准确性（Factual Accuracy）

**定义**：事实准确性是指演示文稿中的信息（名称、日期、数据等）真实无误。

**检查要点**：
- 公司名称和职位正确
- 日期和时间范围准确
- 数据来源可靠
- 统计数据正确

**验证方法**：
- 交叉核对多个来源
- 确认最新信息
- 检查引用准确性

### 9. 术语一致性（Terminology Consistency）

**定义**：术语一致性是指在整个演示文稿中，相同概念使用统一的术语。

**重要性**：
- 避免混淆
- 提升专业性
- 便于理解

**常见问题**：
- 同一概念使用不同术语
- 缩写不一致
- 中英文混用

**规范**：
- 建立术语表
- 统一缩写使用
- 保持术语风格一致

### 10. 交付前检查（Pre-Delivery Checklist）

**定义**：交付前检查是在将演示文稿发送给客户或用于重要场合之前进行的最后审查。

**检查内容**：
- 数字准确性
- 叙述与数据一致性
- 语言专业性
- 格式规范性
- 图表来源

**重要性**：
- 确保演示文稿质量
- 避免低级错误
- 维护专业形象

## Appendix: 金融背景知识

这份文件是"PPT 检查报告格式（Report Format）"参考文档的详细说明。在投行中，当 IB Check Deck 技能检查完一份 Pitch Deck 后，需要生成一份结构化的"审计报告"——告诉制作团队 PPT 哪里有问题、该怎么改。

---

### 1. 什么是 Check Deck 报告？

**类比**：你的论文交给导师。导师不会直接给你改——而是在旁边用红色批注标记："这里数据不对劲"、"这个引用没有来源"、"这段格式跟前面不统一"。

Check Deck 报告就是投行 PPT 的"导师批注"——指出问题，但不直接动手改。

---

### 2. 报告的"三层分类法"

| 严重级别 | 含义 | 举例 |
|---------|------|------|
| 致命 | 不改就不能发 | 公司名拼错、财报数字算错 |
| 重要 | 应该改，但不致命 | 字体不统一、日期没更新 |
| 建议 | 改了更好，不管也没事 | 可以加个图表让这一页更好看 |

---

### 给小白的一句话

> Report Format 就是投行 PPT 的"体检报告的格式"——当 IB Check Deck 扫描完你的 Pitch Deck 后，他需要用一个标准格式告诉你"哪里病了、病得多严重、怎么治"。问题按严重程度分三级：致命（不改就丢客户）、重要（影响专业性）、建议（锦上添花）。记住：报告不是"批评你的作品"——而是"让它能活着见到客户"的必要保障。
