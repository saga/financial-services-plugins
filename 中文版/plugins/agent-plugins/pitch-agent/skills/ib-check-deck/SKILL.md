# IB演示文稿检查器

对演示文稿执行全面QC，涵盖四个维度。读取每张幻灯片，然后报告发现。

## 环境检查

本技能在PowerPoint加载项和聊天中均可工作。开始前识别你所在的环境：

- **加载项** — 从实时打开的演示文稿读取。
- **聊天** — 从上传的`.pptx`文件读取。

这是只读报告 — 无编辑 — 所以两种环境中的工作流相同。

## 工作流

### 读取演示文稿

从每张幻灯片提取文本，跟踪每行来自哪张幻灯片。你需要幻灯片级归属来定位每个发现（"$500M出现在幻灯片3和8，但幻灯片15显示$485M"）。30张幻灯片的演示文稿内容太多，无法可靠保存在工作内存中 — 将提取的文本写入文件，以便数字检查脚本可以处理它。

脚本期望带幻灯片标记的markdown输入。格式为：

```
## 幻灯片1
[幻灯片1文本内容]

## 幻灯片2
[幻灯片2文本内容]
```

### 1. 数字一致性

对你收集的内容运行提取脚本：

```bash
python scripts/extract_numbers.py /tmp/deck_content.md --check
```

它标准化单位（$500M vs $500MM vs $500,000,000 → 相同数字），将值分类（收入、EBITDA、倍数、利润率），并标记当同一指标类别在不同幻灯片上显示冲突值时。这是人类在第五次通读时最可能遗漏的部分。

除了脚本标记的内容，验证：
- 计算正确（总计求和、百分比相加、增长率匹配端点）
- 单位风格一致 — 演示文稿应选择$M或$MM并坚持使用
- 时间段对齐 — FY vs LTM vs 季度，明确标记

### 2. 数据与叙述对齐

将主张映射到 supposed 支持它们的数据。这是演示文稿悄悄出错的地方 — 有人编辑了幻灯片7上的图表但忘记了幻灯片4上的叙述。

- 趋势陈述（"利润率下降"） → 图表确实朝那个方向吗？
- 市场地位主张（"#1玩家"） → 收入和份额数据支持它吗？
- 合理性 — "100亿美元市场中的#1" 收入2亿美元是0.2%份额；那不是#1

### 3. 语言润色

IB演示文稿有语域。扫描任何打破它的内容：随意措辞（"相当好"、"很多"）、缩写、感叹号、没有数字模糊量词、同一概念的不一致术语。

参见 `references/ib-terminology.md` 了解替换模式。

### 4. 视觉和格式QC

对每张幻灯片运行标准视觉验证检查。你在寻找：缺失的图表来源引用、缺失的轴标签、排版不一致、数字格式漂移（同一演示文稿内1,000 vs 1K）、日期格式漂移、脚注和免责声明缺口。

视觉验证捕捉文本提取中不会出现的重叠、溢出和对比度问题。不要跳过它 — 没有来源引用的图表在文本转储中看起来与正确引用的图表相同。

## 输出

使用 `references/report-format.md` 作为结构。按严重性分类：

- **关键** — 数字不匹配、事实错误、数据与叙述矛盾。这些阻碍客户交付。
- **重要** — 语言、缺失来源、术语漂移。应该修复。
- **轻微** — 字体大小、间距、日期格式。润色。

以关键项开头。如果没有任何关键项，明确说明 — "未发现数字不一致"是一个发现，不是没有发现。

## Appendix: 金融背景知识

### 什么是IB演示文稿检查（IB Deck Check）？

投行演示文稿检查是对推介材料的全面质量控制流程，确保：
- 所有数字一致（同一数字在整份演示文稿中相同）
- 叙述与数据匹配（主张有数据支持）
- 语言专业（符合投行标准）
- 格式完美（视觉一致性和专业性）

### 关键术语解释

**QC（Quality Control，质量控制）**：系统检查工作产品（如演示文稿、模型）以确保准确性、一致性和专业性。

**数字一致性（Number Consistency）**：确保同一数字在整份演示文稿中保持不变。例如，收入在幻灯片3、7、12中应相同。

**数据与叙述对齐（Data-narrative Alignment）**：确保幻灯片上的主张（如"利润率扩张"）有图表数据支持。

**IB术语（IB Terminology）**：投行专用语言，要求精确、专业、无模糊措辞。例如：
- 正确："利润率扩张150个基点"
- 错误："利润率好多了"

**视觉验证（Visual Verification）**：检查每张幻灯片的视觉元素：
- 图表有来源引用
- 轴标签完整
- 字体大小一致
- 无文本溢出或重叠
- 颜色对比度充足

**严重性分类（Severity Classification）**：
- **关键（Critical）**：阻止客户交付的问题
- **重要（Important）**：应该修复的问题
- **轻微（Minor）**：润色类问题

**脚注和免责声明（Footnotes and Disclaimers）**：法律要求的披露，必须出现在每张图表/表格上，包括数据来源。

**脚本提取（Script Extraction）**：使用Python脚本从演示文稿中提取数字，自动检测不一致。

**markdown输入**：带幻灯片标记的文本格式，便于脚本处理。

**同一指标类别（Same Metric Category）**：如所有收入数字、所有EBITDA数字、所有倍数等，应跨幻灯片一致。

**模糊量词（Vague Quantifiers）**：没有具体数字的词，如"很多"、"大量"、"相当好" — 在IB演示文稿中应避免。

**术语漂移（Terminology Drift）**：同一概念在演示文稿中使用不同术语，如幻灯片3用"收入"，幻灯片7用"销售额" — 应保持一致。
