# 可比公司分析

## ⚠️ 关键：数据源优先级（首先阅读）

**始终遵循此数据源层次结构：**

1. **首先：检查MCP数据源** — 如果S&P Kensho MCP、FactSet MCP或Daloopa MCP可用，独家使用它们获取财务和交易信息
2. **如果上述MCP数据源可用，不要使用网络搜索**
3. **仅当MCP不可用时：** 然后使用彭博终端、SEC EDGAR备案或其他机构来源
4. **永远不要将网络搜索作为主要数据源** — 它缺乏机构级分析所需的准确性、审计跟踪和可靠性

**为什么这很重要：** MCP来源提供经过验证的机构级数据以及适当的引用。网络搜索结果可能过时、不准确或对财务分析不可靠。

## 概述
本技能教Claude构建机构质量级的可比公司分析，结合运营指标、估值倍数和统计基准测试。输出是一个结构化的Excel/电子表格，通过同行比较实现明智的投资决策。

**参考材料和情境化：**

`examples/comps_example.xlsx`中提供了一个可比公司分析示例。当使用此目录中或其他示例文件时，智能使用它们：

**使用示例用于：**
- 理解结构层次（部分如何流动）
- 掌握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰的标题、透明的公式、审计跟踪）

**不要使用示例用于：**
- 精确复制格式或指标
- 不考虑上下文地复制布局
- 不分受众地应用相同的视觉风格

**首先始终问自己：**
1. **"您有首选格式还是我应该适应模板风格？"**
2. **"受众是谁？"**（投资委员会、董事会演示、快速参考、详细备忘录）
3. **"关键问题是什么？"**（估值、增长分析、竞争定位、效率）
4. **"背景是什么？"**（并购评估、投资决策、行业基准测试、绩效评估）

**根据具体情况进行调整：**
- **行业背景**：大型科技超级巨头需要与新兴SaaS初创公司不同的指标
- **行业特定需求**：添加相关指标（如科技行业的云ARR、企业客户、开发者生态系统）
- **公司熟悉度**：知名公司可能需要更少的背景，更关注delta分析
- **决策类型**：并购需要与持续投资组合监控不同的重点

**核心原则：** 使用模板原则（清晰结构、统计严谨、透明公式），但根据背景变化执行。目标是机构质量的分析，而非机构外观的模板。

用户提供的示例和明确偏好始终优先于默认值。

## 核心哲学
**"首先构建正确的结构，然后让数据讲述故事。"**

从迫使您思考什么重要的标题开始，输入干净的数据，构建透明的公式，让统计数据自动出现。一个好的可比公司分析应该可以被没有构建它的人立即阅读。

## ⚠️ 关键：公式优先于硬编码 + 逐步验证

**环境 — Office JS vs Python：**
- **如果在Excel内运行（Office加载项/Office JS）：** 直接使用Office JS（`Excel.run(async (context) => {...})`）。通过`range.formulas = [["=E7/C7"]]`写入公式，而不是`range.values`。无需单独的重新计算步骤 — Excel原生处理。使用`range.format.*`设置颜色/字体。
- **如果生成独立.xlsx文件：** 使用Python/openpyxl。写`cell.value = "=E7/C7"`（公式字符串）。
- 两种方式遵循相同原则 — 只需转换API调用。
- **Office JS合并单元格陷阱：** 不要在合并范围上调用`.merge()`然后设置`.values`（抛出`InvalidArgument` — 范围仍报告其预合并尺寸）。相反，先单独向左上角单元格写入值，然后合并并格式化完整范围：
  ```js
  ws.getRange("A1").values = [["技术 — 可比公司分析"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式，而不是硬编码：**
- 每个派生值（利润率、倍数、统计）必须是引用输入单元格的Excel公式 — 永远不是预计算的数字
- 使用Python/openpyxl构建工作表时：写`cell.value = "=E7/C7"`（公式字符串），而不是`cell.value = 0.687`（计算结果）
- 唯一应该是硬编码的值是原始输入数据（收入、EBITDA、股价等）— 每个这样的值都需要一个带有其来源的单元格注释
- 为什么：当输入变化时模型必须自动更新。硬编码的利润率是一个静默bug

**逐步与用户验证：**
- 设置结构后 → 在填充数据之前向用户展示标题布局
- 输入原始数据后 → 向用户展示输入块并确认来源/期间，然后在构建公式之前确认
- 构建运营指标公式后 → 向用户展示计算的利润率并与用户进行合理性检查，然后在进入估值之前确认
- 构建估值倍数后 → 向用户展示倍数并确认它们看起来合理，然后在添加统计数据之前确认
- 不要端到端构建整个工作表然后呈现 — 通过确认每个部分及早捕获错误

## 第1节：文档结构与设置

### 标题块（第1-3行）
```
第1行：[分析标题] - 可比公司分析
第2行：[公司列表及股票代码] • [公司1 (代码1)] • [公司2 (代码2)] • [公司3 (代码3)]
第3行：截至[期间] | 除每股金额和比率外，所有数字单位为[百万/十亿美元]
```

**为什么这很重要：** 立即建立背景。打开此文件的任何人都知道他们在看什么、何时创建的以及如何解释数字。

### 视觉约定标准（可选 — 用户偏好和上传的模板始终优先）

**重要：这些仅为建议的默认值。始终优先考虑：**
1. 用户明确的格式偏好
2. 任何上传模板文件的格式
3. 公司/团队样式指南
4. 这些默认值（仅在没有其他指导时）

**建议的字体和排版：**
- **字体系列**：Times New Roman（专业、可读、行业标准）
- **字体大小**：数据单元格11pt，标题12pt
- **粗体文本**：部分标题、公司名称、统计标签

**默认颜色和阴影 — 专业蓝灰色调（克制更好）：**
- **保持克制** — 仅使用蓝色和灰色。不要引入绿色、橙色、红色或多种强调色。干净的可比公司分析表总共使用3-4种颜色。
- **部分标题**（例如"运营统计和财务指标"）：
  - 深蓝色背景（`#1F4E79`或`#17365D`海军蓝）
  - 白色粗体文本
  - 跨所有列的整行阴影
- **列标题**（例如"公司"、"收入"、"利润率"）：
  - 浅蓝色背景（`#D9E1F2`或类似浅蓝色）
  - 黑色粗体文本
  - 居中对齐
- **数据行**：
  - 公司数据白色背景
  - 公式黑色文本；输入蓝色文本
- **统计行**（最大值、第75百分位等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文本，标签左对齐
- **这是整个调色板**：深蓝色 + 浅蓝色 + 浅灰色 + 白色。除非用户模板另有说明，否则不要使用其他颜色。

**建议的格式约定：**
- **小数精度**：
  - 百分比：1位小数（12.3%）
  - 倍数：1位小数（13.5x）
  - 美元金额：无小数，千位分隔符（69,632）
  - 显示为百分比的利润率：1位小数（68.7%）
- **边框**：无边框（干净、最小化外观）
- **对齐**：所有指标居中对齐，以获得干净、统一的外观
- **单元格尺寸**：所有列宽应统一/均匀，所有行高应一致（创建干净、专业的网格）

**注意：** 如果用户提供模板文件或指定不同格式，请改用该格式。

## 第2节：运营统计和财务指标

### 核心列（首先使用这些）
1. **公司** — 格式一致的公司名称
2. **收入** — 规模指标（可根据上下文选择LTM、季度或年度）
3. **收入增长** — 同比百分比变化
4. **毛利润** — 收入减去销货成本
5. **毛利率** — GP/收入（基本盈利能力）
6. **EBITDA** — 利息、税费、折旧、摊销前收益
7. **EBITDA利润率** — EBITDA/收入（运营效率）

### 可选添加（根据行业/目的选择）
- **季度 vs LTM** — 如果季节性重要，包括两者
- **自由现金流** — 对于资本密集型或SaaS业务
- **FCF利润率** — FCF/收入（现金生成效率）
- **净利润** — 对于成熟盈利公司
- **营业利润** — 对于D&A不同的业务
- **CapEx指标** — 对于资产重的行业
- **40法则** — 特别是SaaS（增长% + 利润率%）
- **FCF转换** — 对于盈利质量分析（高级）

### 公式示例（以第7行为例）
```excel
// 核心比率 — 这些始终计算
毛利率 (F7): =E7/C7
EBITDA利润率 (H7): =G7/C7

// 可选比率 — 如果相关包括
FCF利润率: =[FCF]/[收入]
净利润率: =[净利润]/[收入]
40法则: =[增长 %]+[FCF利润率 %]
```

**黄金法则：** 每个比率应该是[某物] / [收入]或[某物] / [来自本工作表的某物]。保持简单。

### 统计块（在公司数据之后）

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留一行空白以供视觉分隔]
- 最大值: =MAX(B7:B9)
- 第75百分位: =QUARTILE(B7:B9,3)
- 中位数: =MEDIAN(B7:B9)
- 第25百分位: =QUARTILE(B7:B9,1)
- 最小值: =MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- 收入增长%、毛利率%、EBITDA利润率%、EPS
- EV/收入、EV/EBITDA、P/E、股息收益率%、Beta

**不需要统计的列（规模指标）：**
- 收入、EBITDA、净利润（绝对规模因公司规模不同）
- 市值、企业价值（不同规模的公司不可比）

**注意：** 在公司数据和统计行之间添加一行空白以供视觉分隔。不要添加"行业统计"或"估值统计"标题行。

**为什么四分位数重要：** 它们显示分布，而不仅仅是平均值。第75百分位倍数告诉您"优质"公司在什么位置交易。

## 第3节：估值倍数和投资指标

### 核心估值列（首先使用这些）
1. **公司** — 与运营部分相同的顺序
2. **市值** — 当前市场估值
3. **企业价值** — 市值 ± 净债务/现金
4. **EV/收入** — 市场为每美元销售额支付多少
5. **EV/EBITDA** — 市场为每美元收益支付多少
6. **P/E比率** — 价格相对于净收益

### 可选估值指标（根据上下文选择）
- **FCF收益率** — FCF/市值（用于现金重点分析）
- **PEG比率** — P/E/增长率（用于成长型公司）
- **市净率** — 市值与账面价值（用于资产重业务）
- **ROE/ROA** — 回报指标（用于盈利能力比较）
- **收入/EBITDA CAGR** — 历史增长率（用于趋势分析）
- **资产周转率** — 收入/资产（用于运营效率）
- **债务/权益** — 杠杆（用于资本结构分析）

**关键原则：** 包括对您的行业重要的3-5个核心倍数。不要仅仅因为可以就包括每个可能的指标。

### 公式示例
```excel
// 核心倍数 — 始终包括这些
EV/收入: =[企业价值]/[LTM收入]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
P/E比率: =[市值]/[净利润]

// 可选倍数 — 如果有数据则包括
FCF收益率: =[LTM FCF]/[市值]
PEG比率: =[P/E]/[增长率 %]
```

### 交叉引用规则
**关键：** 估值倍数必须引用运营指标部分。永远不要两次输入相同的原始数据。如果收入在C7中，那么EV/收入公式应引用C7。

### 统计块
与运营部分相同的结构：每个指标的最大值、75、50、25、最小值。在公司数据和统计之间添加一行空白以供视觉分隔。不要添加"估值统计"标题行。

## 第4节：注释和方法论文档

### 必需组成部分

**数据源和质量：**
- 数据来自哪里？（S&P Kensho MCP、FactSet MCP、Daloopa MCP、彭博、SEC备案）
- 涵盖什么期间？（2024年Q4、审计数字）
- 如何验证？（与10-K/10-Q交叉检查）
- 注意：如果MCP数据源可用（S&P Kensho、FactSet、Daloopa），优先使用它们以获得更好的准确性和可追溯性

**关键定义：**
- EBITDA计算方法（毛利润 + D&A，或营业利润 + D&A）
- 自由现金流公式（运营CF - CapEx）
- 特殊指标解释（40法则、FCF转换）
- 期间定义（LTM、CAGR计算期间）

**估值方法论：**
- 如何计算企业价值？（市值 + 净债务）
- 使用了什么增长率？（历史CAGR、前向估计）
- 是否有任何调整？（排除一次性项目、正常化利润率）

**分析框架：**
- 投资论点是什么？（云/SaaS效率）
- 什么指标最重要？（现金生成、资本效率）
- 读者应该如何解释统计数据？（四分位数提供背景）

## 第5节：选择正确的指标（决策框架）

### 从"我在回答什么问题？"开始

**"哪家公司被低估了？"**
→ 重点关注：EV/收入、EV/EBITDA、P/E、市值
→ 跳过：运营细节、增长指标

**"哪家公司效率最高？"**
→ 重点关注：毛利率、EBITDA利润率、FCF利润率、资产周转率
→ 跳过：规模指标、绝对美元金额

**"哪家公司增长最快？"**
→ 重点关注：收入增长%、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆比率

**"哪家是最好的现金生成者？"**
→ 重点关注：FCF、FCF利润率、FCF转换、CapEx强度
→ 跳过：EBITDA、P/E比率

### 行业特定指标选择

**软件/SaaS：**
必须有：收入增长、毛利率、40法则
可选：ARR、净美元留存、CAC回收期
跳过：资产周转率、库存指标

**制造/工业：**
必须有：EBITDA利润率、资产周转率、CapEx/收入
可选：ROA、库存周转、积压订单
跳过：40法则、SaaS指标

**金融服务：**
必须有：ROE、ROA、效率比率、P/E
可选：净利息收益率、贷款损失准备金
跳过：毛利率、EBITDA（对银行没有意义）

**零售/电商：**
必须有：收入增长、毛利率、库存周转
可选：同店销售、客户获取成本
跳过：重研发或CapEx指标

### "5-10法则"

**5个运营指标** — 收入、增长、2-3个利润率/效率指标
**5个估值指标** — 市值、EV、3个倍数
**= 10个总列** — 足够讲述故事，但不至于多到失去主线

如果您有超过15个指标，您可能包含了噪音。无情地编辑。

## 第6节：最佳实践和质量检查

### 开始之前
1. **定义同行群体** — 公司必须真正可比（相似业务模式、规模、地理）
2. **选择正确的期间** — LTM平滑季节性；季度显示趋势
3. **提前标准化单位** — 百万 vs 十亿决定影响一切
4. **映射数据源** — 知道每个数字来自哪里

### 构建时
1. **首先输入所有原始数据** — 在写公式之前完成蓝色文本
2. **为所有硬编码输入添加单元格注释** — 右键单元格 → 插入注释 → 记录来源或假设

   **对于来源数据，准确引用其来源：**
   - 示例："彭博终端 - MSFT Equity DES，访问于2024-10-02"
   - 示例："2024年Q4 10-K备案，第42页，行项目'总收入'"
   - 示例："FactSet共识估计，截至2024-10-02"
   - **尽可能包含超链接**：右键单元格 → 链接 → 粘贴到SEC备案、数据源或报告的URL

   **对于假设，解释推理：**
   - 示例："假设基于同行中位数的15% EBITDA利润率，公司未披露"
   - 示例："估计企业价值为市值 + $50M净债务（来自Q3资产负债表，Q4尚未可用）"
   - 示例："基于12位分析师估计的平均街头共识EPS $3.45的前向P/E"

   **为什么这很重要**：支持审计跟踪、数据验证、假设透明度和未来更新
3. **逐行构建公式** — 在继续之前测试每个计算
4. **对标题使用绝对引用** — $C$6锁定标题行
5. **格式一致** — 百分比格式化为百分比，不是小数
6. **添加条件格式** — 自动突出显示异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA利润率 > 净利润率（根据定义总是如此）
- **倍数合理性**：
  - EV/收入：通常0.5-20x（因行业差异很大）
  - EV/EBITDA：通常8-25x（各行业相当一致）
  - P/E：通常10-50x（取决于增长率）
- **增长-倍数相关性**：更高增长通常意味着更高倍数
- **规模-效率权衡**：更大的公司通常有更好的利润率（规模优势）

### 需要避免的常见错误
❌ 在公式中混合市值和企业价值
❌ 对分子和分母使用不同时期（LTM vs 季度）
❌ 将数字硬编码到公式中而不是单元格引用
❌ **硬编码输入没有单元格注释引用来源或解释假设**
❌ 在有可用时缺少SEC备案或数据源的超链接
❌ 包含太多没有明确目的的指标
❌ 包含不可比的公司（不同业务模式）
❌ 使用未披露的过时数据
❌ 错误计算百分比平均值（应该是中位数）

## 金融术语和知识详细解释

### 1. 可比公司分析（Comparable Company Analysis）

**定义**：可比公司分析是一种估值方法，通过将与目标公司具有相似特征的已上市公司进行比较来确定公司价值。

**核心原理**：
```
目标公司价值 = 可比公司倍数 × 目标公司指标
```

**为什么重要**：
- 相对估值而非绝对估值
- 利用市场共识定价
- 快速获得价值参考区间
- 投行和投资机构广泛使用

---

### 2. EBITDA（利息、税费、折旧和摊销前利润）

**定义**：EBITDA是衡量公司核心运营盈利能力的指标。

**计算公式**：
```
EBITDA = 净利润 + 利息费用 + 税费 + 折旧 + 摊销
或
EBITDA = 收入 - 销货成本 - 运营费用（不含D&A）
```

**为什么重要**：
- 去除资本结构和会计政策影响
- 便于不同公司比较
- 计算EV/EBITDA倍数的基础
- 杠杆收购核心指标

---

### 3. 企业价值（Enterprise Value, EV）

**定义**：企业价值是公司总价值的度量，包括所有债权人和股东权益。

**计算公式**：
```
EV = 市值 + 总债务 - 现金
   = 普通股市值 + 优先股市值 + 少数股东权益 + 总债务 - 现金
```

**为什么重要**：
- 比市值更全面的价值指标
- 用于EV/EBITDA等倍数计算
- 反映所有资本提供者的债权
- 并购估值的基础

---

### 4. EV/EBITDA倍数

**定义**：企业价值相对于EBITDA的倍数。

**计算公式**：
```
EV/EBITDA = 企业价值 / EBITDA
```

**典型倍数范围**：
| 行业 | 典型倍数 |
|------|---------|
| 成熟行业 | 8-12x |
| 成长型 | 12-18x |
| 科技/高增长 | 18-25x |

**为什么重要**：
- 最常用的估值倍数之一
- 跨行业可比
- 考虑资本结构差异
- LBO估值核心指标

---

### 5. P/E（市盈率）

**定义**：市盈率是股价相对于每股收益的倍数。

**计算公式**：
```
P/E = 股价 / 每股收益（EPS）
```

**两种类型**：
- **Trailing P/E**：基于过去12个月实际收益
- **Forward P/E**：基于未来12个月预期收益

**为什么重要**：
- 最直观的估值指标
- 投资者最关注
- 反映市场预期
- 便于快速比较

---

### 6. 毛利率（Gross Margin）

**定义**：毛利润占收入的百分比。

**计算公式**：
```
毛利率 = (收入 - 销货成本) / 收入
       = 毛利润 / 收入
```

**为什么重要**：
- 反映定价能力和生产效率
- 不同行业差异大
- 竞争优势指标
- 利润率金字塔的起点

---

### 7. 四分位数分析（Quartile Analysis）

**定义**：四分位数将数据分成四个相等部分。

**关键四分位数**：
- **第25百分位（Q1）**：25%的数据在此以下
- **中位数（Q2）**：50%的数据在此以下
- **第75百分位（Q3）**：75%的数据在此以下

**为什么重要**：
- 显示分布而非仅平均值
- 识别"优质"vs"折扣"公司
-  quartile差距显示分散程度
- 估值决策的关键参考

---

### 8. 同店销售（Same-Store Sales）

**定义**：比较可比店面在相同时期的销售表现。

**计算公式**：
```
同店销售增长 = (本期同店销售 - 上期同店销售) / 上期同店销售
```

**为什么重要**：
- 排除新店扩张影响
- 反映有机增长
- 零售业核心指标
- 衡量客户忠诚度

---

### 9. 40法则（Rule of 40）

**定义**：收入增长率与EBITDA利润率之和。

**计算公式**：
```
40法则 = 收入增长率% + EBITDA利润率%
```

**解读**：
- >40：增长和盈利能力平衡良好
- <40：要么增长慢，要么盈利弱
- 专用于SaaS和订阅业务

**为什么重要**：
- 平衡增长与盈利
- SaaS行业标准指标
- 投资者评估SaaS公司关键
- 简单直观的质量指标

---

### 10. 自由现金流（Free Cash Flow, FCF）

**定义**：公司在满足运营和资本支出后可支配的现金。

**计算公式**：
```
FCF = 经营现金流 - 资本支出
```

**FCF利润率**：
```
FCF利润率 = FCF / 收入
```

**为什么重要**：
- 衡量真实盈利能力
- 估值最可靠指标
- 股息和股票回购资金来源
- 公司健康关键信号

## Appendix: 金融背景知识

这份文件是"可比公司分析（Comps Analysis）"技能的详细说明。在投资银行和股权研究中，可比公司分析是最常用的估值方法之一——"看看类似公司多少钱，我就值多少钱"。

---

### 1. 什么是可比公司分析？

**类比：**
想象你要卖你的二手宝马 3 系。你怎么定价？你会去二手车网站看看同款车的价格：
- 2019 款 3 系 → $25,000
- 2020 款 3 系 → $28,000
- 2021 款 3 系 → $32,000

你的 2020 款 → 估计 $28,000 左右。

可比公司分析就是金融世界的"二手车估值法"——找几家类似的上市公司，看他们的估值倍数，然后推算目标公司的价值。

---

### 2. 可比公司选择的"黄金法则"

#### 法则 1：行业相同
- 不能拿苹果和可口可乐比——一个是科技，一个是消费品

#### 法则 2：规模相近
- 市值差 10x 的公司不可比——$1B 的小公司和 $100B 的巨头面临的机会和风险完全不同

#### 法则 3：业务模式相似
- 同样是卖车：特斯拉（直销）vs 福特（经销商模式）——商业逻辑不同，可比性低

#### 法则 4：增长率可比
- 增长 50% 和增长 5% 的公司不能直接比——高增长公司理应有更高的估值倍数

---

### 3. 最常用的"四大估值倍数"

| 倍数 | 含义 | 适用 | 高 or 低 = 好？ |
|------|------|------|----------------|
| P/E | 股价 / 每股收益 | 通用 | 低 = 便宜 |
| EV/EBITDA | 企业价值 / 经营利润 | 重资产行业 | 低 = 便宜 |
| EV/Revenue | 企业价值 / 收入 | 成长型公司 | 低 = 便宜 |
| P/B | 股价 / 每股净资产 | 金融业 | 低 = 便宜 |

---

### 4. 可比公司分析的"标准表格"

| 公司 | 市值 | 收入 | 增长率 | P/E | EV/EBITDA |
|------|------|------|--------|-----|-----------|
| 公司 A | $50B | $10B | 15% | 25x | 18x |
| 公司 B | $40B | $8B | 12% | 22x | 16x |
| 公司 C | $35B | $7B | 18% | 28x | 20x |
| **中位数** | | | | **25x** | **18x** |
| **目标公司** | | $6B | **15%** | | |

**推算**：目标公司增长 15%，接近可比公司中位数 → 适用中位数 P/E 25x。如果 EPS $2.0，则目标价 = 25 × $2.0 = $50。

---

### 5. 可比公司分析的操作技巧

| 技巧 | 说明 |
|------|------|
| 取中位数 | 不要取平均值——一只"疯子"股票会把平均值拉歪 |
| 排除异常值 | P/E 300x 或 5x 的公司要排除 |
| 用 Forward | 用"预测 P/E"而不是"过去 P/E"——投资看的是未来 |
| 多维度 | 不要只看 P/E，EV/EBITDA、P/S、PEG 都要看 |

---

### 给小白的一句话

> 可比公司分析就是金融界的"货比三家"——你不知道一辆二手车值多少钱？去看看同样车型、同年份、相似里程的车都卖多少。同样道理，你不知道一家公司该给多少估值倍数？去看看同行业的可比公司 P/E 多少，取个中位数。但记住一个关键陷阱："如果整个行业都被高估了呢？"——2000 年互联网泡沫时，所有互联网公司的 P/E 都上天，用 Comps 分析会觉得"大家都这么贵，所以我也合理"——结果泡沫破裂了。所以 Comps 分析最有价值的不是"得出一个数字"，而是"理解为什么某些公司估值更高"。
