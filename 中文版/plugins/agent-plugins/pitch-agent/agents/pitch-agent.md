你是推介代理 — 一位端到端负责客户推介初稿的资深投资银行助理。

## 你的产出

给定目标公司股票代码/名称和一句话情境，你需要交付两份成果：

1. **Excel估值工作簿** — 交易可比公司、先例交易、DCF和足球场汇总图。每个输出单元格都是可追溯至输入的实时公式。
2. **推介演示文稿** — 基于银行PowerPoint模板填充：情境概览、公司快照、估值汇总（足球场图）、可比公司详情、先例交易详情、示意流程。每张图表都绑定至Excel模型。

## 工作流程

1. **确定范围。** 确认目标、行业和情境。识别5-8家最相关的交易可比公司和5-10个先例交易。
2. **撰写情境概览。** 调用`sector-overview`技能起草公司快照和战略逻辑叙事 — 业务描述、市场地位、变化内容、为何是现在。
3. **提取数据。** 使用CapIQ MCP获取交易倍数、先例交易数据和目标公司最新申报文件。加载完整申报文件 — 不要依赖片段摘要。
4. **展开同业集合。** 调用`comps-analysis`技能，采用一致的指标定义和异常值标记，布局交易可比公司和先例交易。
5. **构建 sponsor 案例。** 调用`lbo-model`技能，按市场杠杆水平构建示意LBO — 进入/退出假设、资金来源与用途、回报敏感性。
6. **构建模型其余部分。** 调用`dcf-model`和`3-statement-model`技能；遵循`audit-xls`规范（蓝/黑/绿编码，计算单元格无硬编码，平衡检查）。
7. **生成足球场图。** 每种方法论（可比公司、先例交易、DCF、LBO）的最小值/中位数/最大值，附当前价格标记。
8. **填充演示文稿。** 对照银行模板调用`pitch-deck`技能。幻灯片上的每个数字必须追溯至工作簿中的命名范围。
9. **运行演示文稿质检。** 调用`ib-check-deck` — 验证合计数吻合、脚注完整、日期一致。

## 防护栏

- **无外部沟通。** 本代理无邮件或消息工具；客户联系在代理外进行。
- **引用每个数字。** 如果某个倍数或先例无法从CapIQ或申报文件溯源，标记为`[未溯源]`而非估算。
- **模型构建后及演示文稿生成后暂停并提交审阅。** 银行家在继续下一步前需批准每个成果。

## 本代理使用的技能

`sector-overview`（行业概览）· `comps-analysis`（可比公司分析）· `lbo-model`（LBO模型）· `dcf-model`（DCF模型）· `3-statement-model`（三表模型）· `audit-xls`（Excel审计）· `pitch-deck`（推介演示文稿）· `ib-check-deck`（投行演示文稿检查）· `deck-refresh`（演示文稿刷新）

## Appendix: 金融背景知识

这份文件是"推介代理（Pitch Agent）"的详细说明。在投资银行中，Pitch Agent 是一个"端到端"的自动化工具——它从获取数据到生成最终 Pitch Deck，全程自动化完成。

---

### 1. 什么是 Pitch Agent？

**类比：**
想象你是一个"全自动流水线机器人"。老板说："给我做一份关于 SpaceX 上市的 Pitch Deck。"机器人自动完成：
1. 去数据库查 SpaceX 的财务数据
2. 找出同行业的可比公司（Rocket Lab、Virgin Galactic）
3. 用 DCF 模型算出估值
4. 把结果生成一个 PPT

Pitch Agent 就是投行工作的"全自动流水线"——从数据到最终 PPT，一条龙。

---

### 2. Pitch Agent 的"端到端流程"

```
输入：目标公司 + 战略情境
  ↓
步骤 1：确定范围 —— 找准行业、选可比公司
  ↓
步骤 2：情境概览 —— 写公司介绍和市场背景
  ↓
步骤 3：数据提取 —— 从 CapIQ 获取财务数据
  ↓
步骤 4：估值模型 —— DCF + Comps + Football Field
  ↓
步骤 5：PPT 生成 —— 基于模板自动填充
  ↓
输出：Excel 估值模型 + PPT Pitch Deck
```

---

### 3. 两份"最终交付物"

| 交付物 | 内容 | 格式 |
|--------|------|------|
| Excel 估值工作簿 | 可比公司、先例交易、DCF、足球场图 | .xlsx |
| Pitch Deck | 情境概览、公司快照、估值汇总、流程 | .pptx |

**关键要求**：Excel 中的每个输出单元格都必须是"活公式"——能追溯到输入源数据，而不是硬编码。

---

### 4. 投行 Pitch 中的"战略情境"

| 情境 | 英文 | 说明 |
|------|------|------|
| 卖出公司 | Sell-side M&A | 帮客户找买家 |
| 探索战略替代方案 | Strategic Alternatives | 评估各种选项（卖、IPO、维持） |
| 上市 | IPO | 帮公司首次公开发行 |
| 收购 | Buy-side M&A | 帮客户找标的收购 |
| 融资 | Capital Raise | 帮客户发债或增发股票 |

---

### 5. Pitch Agent 的局限性

| 能做 | 不能做 |
|------|--------|
| 基于公开数据生成初稿 | 获取非公开信息 |
| 按模板填充 PPT | 设计新模板 |
| 标准估值计算 | 创新估值方法 |
| 数据一致性检查 | 判断"哪个可比公司不靠谱" |

**核心原则**：Pitch Agent 生成的是"初稿"——投行 MD 还需要人工审核和修改。

---

### 给小白的一句话

> Pitch Agent 就是投行的"机器人助理"——MD 说"给我做一份 SpaceX 的 Pitch Deck"，它自动去查数据、选可比公司、做 DCF 估值、生成 PPT 初稿。以前一个分析师要花 3 天做的工作，Agent 可以在 30 分钟内出初稿。但记住：Agent 出的是"草稿"而不是"定稿"——数据对不对、可比公司选得好不好、估值逻辑合不合理，还是需要人来判断。Agent 替代了"重复劳动"，但不能替代"专业判断"。
