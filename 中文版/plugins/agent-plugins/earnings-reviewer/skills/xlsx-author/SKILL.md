# xlsx-author

当运行**无头模式**（托管代理/CMA模式）且需要将Excel工作簿作为**文件制品**交付，而不是通过`mcp__office__excel_*`编辑实时工作簿时，使用此技能。

## 输出约定

- 写入 `./out/<name>.xlsx`。如果 `./out/` 不存在则创建。
- 在最终消息中返回相对路径，以便编排层可以收集它。

## 如何构建工作簿

编写一个简短的Python脚本并使用Bash运行。使用`openpyxl`库：

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active; ws.title = "Inputs"
ws["B2"] = "Revenue"; ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")           # 蓝色 = 硬编码输入
calc = wb.create_sheet("DCF")
calc["C5"] = "=Inputs!C2*(1+Inputs!C3)"        # 黑色 = 公式
wb.save("./out/model.xlsx")
```

## 约定（镜像`audit-xls`）

- **蓝色/黑色/绿色。** 蓝色 = 硬编码输入，黑色 = 公式，绿色 = 链接到另一个工作表/文件。
- **计算单元格中无硬编码。** 每个计算单元格都是一个公式；每个输入都在Inputs标签上。
- **命名范围**用于从演示文稿或备忘录引用的任何值。
- **平衡检查。** 包含一个检查标签，用于核对（资产负债表平衡、现金流与现金匹配等）并显示TRUE/FALSE。
- **每个文件一个模型。** 除非明确要求，否则不要附加到现有工作簿。

## 何时不使用

如果`mcp__office__excel_*`工具可用（Cowork插件模式），请改用这些工具 — 它们驱动用户的实时工作簿并带有审核检查点。此技能是无头运行时生成文件的备选方案。

## 金融术语和知识解释

### 1. 无头模式详解

**定义**：无头模式是指在没有图形用户界面(GUI)的情况下运行软件程序。

**在金融领域的应用**：
| 场景 | 应用说明 |
|------|----------|
| **自动化报表生成** | 定时生成财务报告、业绩报表 |
| **批量数据处理** | 处理大量财务数据文件 |
| **服务器端计算** | 云端执行财务模型计算 |
| **CI/CD流水线** | 自动化测试和部署财务工具 |

**优点**：
- 资源占用低
- 可远程执行
- 易于自动化和调度
- 适合服务器环境

**局限性**：
- 无法直接交互
- 需要编程能力
- 复杂格式处理有限

### 2. Excel工作簿结构

**典型财务模型工作簿结构**：

| 标签名称 | 用途 | 内容特征 |
|----------|------|----------|
| **Inputs** | 假设输入 | 蓝色字体，硬编码值 |
| **Schedule** | 明细计算 | 折旧、债务、股份等时间表 |
| **IS** | 损益表 | 收入、费用、净利润 |
| **BS** | 资产负债表 | 资产、负债、权益 |
| **CF** | 现金流量表 | 经营、投资、筹资现金流 |
| **Valuation** | 估值计算 | DCF、Comps、交易倍数 |
| **Output** | 输出汇总 | 关键指标、图表 |
| **Check** | 完整性检查 | 平衡验证、TRUE/FALSE |

**最佳实践**：
- 输入集中管理
- 计算逻辑清晰
- 检查机制完善
- 输出易于阅读

### 3. 硬编码 vs 公式

**定义**：
- **硬编码(Hardcode)**：直接输入的固定值（如增长率假设）
- **公式(Formula)**：基于其他单元格计算得出的值

**对比分析**：

| 方面 | 硬编码 | 公式 |
|------|--------|------|
| **灵活性** | 低，需手动修改 | 高，自动更新 |
| **可审计性** | 差，难以追踪 | 好，可追溯来源 |
| **错误风险** | 高，容易遗漏更新 | 低，联动更新 |
| **适用场景** | 假设输入 | 计算逻辑 |

**模型设计原则**：
```
所有输入集中在Inputs标签（蓝色）
所有计算使用公式引用（黑色）
跨表引用使用命名范围（绿色）
```

---

### 4. 命名范围详解

**定义**：给单元格或单元格区域分配描述性名称。

**创建方法**：
```python
from openpyxl import Workbook
wb = Workbook()
ws = wb.active
ws['B2'] = 100
wb.create_named_range('Revenue_2024', ws, 'B2')
```

**优点**：
- 公式更易读（`=Revenue_2024*1.1` vs `=Inputs!B2*1.1`）
- 便于跨表引用
- 提高模型可维护性
- 减少错误

**命名约定**：
- 使用下划线分隔单词
- 包含时间维度（如`Revenue_2024`）
- 保持一致性

---

### 5. 模型完整性检查

**定义**：在财务模型中加入验证机制确保计算正确。

**检查类型**：

| 检查项 | 验证方法 | 错误处理 |
|--------|----------|----------|
| 资产负债表平衡 | `=IF(总资产=总负债+权益,TRUE,差额)` | 显示差额 |
| 现金匹配 | `=IF(CF!期末现金=BS!现金,TRUE,差额)` | 显示差额 |
| 公式错误 | 检查`#REF!`、`#VALUE!`等 | 标记错误单元格 |
| 单位一致性 | 检查千/百万/实际值混用 | 警告提示 |

**检查标签结构**：
```
A列：检查项描述
B列：检查结果（TRUE/FALSE/差额）
C列：状态（通过/失败/警告）
```

### 6. openpyxl详解

**定义**：openpyxl是Python库，用于读写Excel 2010+ .xlsx文件。

**主要功能**：
| 功能 | 说明 |
|------|------|
| 创建工作簿 | `Workbook()` |
| 选择工作表 | `wb.active`或`wb['Sheet']` |
| 写入数据 | `ws['A1'] = value` |
| 写入公式 | `ws['B2'] = '=A1*2'` |
| 设置样式 | 字体、颜色、对齐 |
| 创建图表 | 柱状图、折线图等 |

**示例代码**：
```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill

wb = Workbook()
ws = wb.active
ws.title = "Inputs"

# 输入单元格（蓝色）
ws["B2"] = "Revenue"
ws["C2"] = 1_250_000_000
ws["C2"].font = Font(color="0000FF")

# 计算单元格（黑色）
calc = wb.create_sheet("DCF")
calc["C5"] = "=Inputs!C2*(1+Inputs!C3)"

wb.save("./out/model.xlsx")
```

## Appendix: 金融背景知识

这份文件是"Excel 创作（XLSX Author）"技能的详细说明。在 Earnings Reviewer 的上下文中，XLSX Author 负责将财报分析结果输出成一个格式规范的 Excel 工作簿——包含差异对比表、预测修订和估值更新。

---

### 1. Earnings Reviewer 的 Excel 输出

在财报分析中，Excel 工作簿是"证据包"——所有你在报告中写的数字，都必须在 Excel 里有"可追溯的来源"。

**典型 Sheet 结构：**

| Sheet | 内容 |
|-------|------|
| Actuals | 本季度实际数字（从 10-Q 提取） |
| Consensus | 华尔街一致性预期 |
| Variance | 实际 vs 共识 vs 上次预测的三方对比 |
| Revised Forecast | 修订后的未来预测 |
| Valuation | 更新后的 DCF/Comps 估值表 |

---

### 2. 核心要求

每一个关键数字必须标注来源：哪个文件（10-Q）、哪一行、哪个项目。"拍脑袋"的数字在投行不可接受。

---

### 给小白的一句话

> Earnings Reviewer 的 XLSX Author 就是财报分析的"证据制造机"——你的报告说"公司 Beat 了共识 3%"，这个 3% 是怎么算出来的？实际收入来自 10-Q 第 12 页，共识来自 FactSet，方差 = (实际-共识)/共识——每一步计算都必须在 Excel 里留下可追溯的痕迹，以备高级分析师逐行核查。
