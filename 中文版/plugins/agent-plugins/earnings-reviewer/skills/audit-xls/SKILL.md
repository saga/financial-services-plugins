# 审计电子表格

审计公式和数据的准确性和错误。范围决定深度——从对选定区域的快速公式检查到完整的财务模型完整性审计。

## 步骤1：确定范围

如果用户已指定范围，使用该范围。否则**询问他们**：

> 你想让我审计什么范围？
> - **selection（选定区域）** — 仅当前选定的范围
> - **sheet（工作表）** — 仅当前活动工作表
> - **model（模型）** — 整个工作簿，包括财务模型完整性检查（资产负债表平衡、现金匹配、滚动计算、逻辑合理性）

**model（模型）** 范围是最深的——在发送给客户或内部客户之前，用于DCF、LBO、三表联动、合并、可比公司或任何集成财务模型。

---

## 步骤2：公式级检查（所有范围）

无论范围如何，都运行以下检查：

| 检查项 | 检查内容 |
|---|---|
| 公式错误 | `#REF!`、`#VALUE!`、`#N/A`、`#DIV/0!`、`#NAME?` |
| 公式内硬编码 | `=A1*1.05` — `1.05` 应该是单元格引用 |
| 不一致的公式 | 打破行/列中相邻单元格模式的公式 |
| 偏移范围 | `SUM`/`AVERAGE` 遗漏第一行或最后一行 |
| 被覆盖的公式 | 看起来像公式但实际上是硬编码值的单元格 |
| 循环引用 | 有意或无意的 |
| 损坏的跨表链接 | 引用已移动或删除的单元格 |
| 单位/规模不匹配 | 千与百万混合，百分比存储为整数 |
| 隐藏行/标签 | 可能包含覆盖或过期计算 |

---

## 步骤3：模型完整性检查（仅限MODEL范围）

如果范围是**model**，识别模型类型（DCF / LBO / 三表联动 / 合并 / 可比公司 / 自定义）并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项 | 检查内容 |
|---|---|
| 输入/公式分离 | 输入是否与计算明确分离？ |
| 颜色约定 | 蓝色=输入，黑色=公式，绿色=链接 — 或模型使用的任何约定，是否一致应用？ |
| 标签流程 | 逻辑顺序（假设 → 损益表 → 资产负债表 → 现金流 → 估值）？ |
| 日期标题 | 所有标签一致？ |
| 单位 | 一致（千 vs 百万 vs 实际值）？ |

### 3b. 资产负债表

| 检查项 | 测试方法 |
|---|---|
| 资产负债表平衡 | 总资产 = 总负债 + 权益（每个期间） |
| 留存收益滚动 | 期初留存收益 + 净利润 − 股息 = 期末留存收益 |
| 商誉/无形资产 | 从收购假设流入（如果是并购） |

如果资产负债表不平衡，**量化每个期间的差距并追踪断点**——在此修复之前，其他一切都不重要。

### 3c. 现金流量表

| 检查项 | 测试方法 |
|---|---|
| 现金匹配 | 现金流量表期末现金 = 资产负债表现金（每个期间） |
| 现金流总和 | 经营现金流 + 投资现金流 + 筹资现金流 = 现金变动 |
| 折旧摊销匹配 | 现金流量表上的折旧摊销 = 损益表上的折旧摊销 |
| 资本支出匹配 | 现金流量表上的资本支出与资产负债表上的PP&E滚动一致 |
| 营运资金变化 | 符号与资产负债表变动匹配（应收账款变动、应付账款变动、存货变动） |

### 3d. 损益表

| 检查项 | 测试方法 |
|---|---|
| 收入构建 | 与部门/产品明细一致 |
| 税费 | 税费支出 = 税前收入 × 税率（允许递延税调整） |
| 股份数量 | 与稀释时间表一致（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是LBO/三表联动模型中常见的有意循环
- 如果是有意的：验证迭代切换存在并正常工作
- 如果是无意的：追踪循环并标记如何打破它

### 3f. 逻辑与合理性

| 检查项 | 标记条件 |
|---|---|
| 增长率 | 收入增长>100%而无解释 |
| 利润率 | 超出行业标准 |
| 终值主导 | 终值 > DCF企业价值的约75%（黄色警告） |
| 曲棍球棒效应 | 预测在后期不切实际地上升 |
| 复合增长 | EBITDA到第10年复合到荒谬的金额 |
| 边缘情况 | 模型在0%或负增长、负EBITDA、杠杆变为负数时崩溃 |

### 3g. 特定模型类型的错误

**DCF模型：**
- 折现率应用到错误期间（年中 vs 年末）
- 终值未折现
- WACC使用账面价值而非市场价值
- 自由现金流包含利息支出（应为无杠杆）
- 税盾重复计算

**LBO模型：**
- 债务偿还与现金扫尾机制不匹配
- PIK利息未计入本金
- 管理层滚动投资未反映在回报中
- 退出倍数应用到错误的EBITDA（LTM vs NTM）
- 费用未从第1天权益中扣除

**合并模型：**
- 摊薄/增厚使用错误的股份数量（交易前 vs 交易后）
- 协同效应未分阶段实施
- 购买价格分配不平衡
- 未包含现金的机会成本利息
- 交易费用未列入资金来源与用途

**三表联动模型：**
- 营运资金变化符号错误
- 折旧与PP&E时间表不匹配
- 债务到期时间表与本金偿还不匹配
- 股息超过净利润而无解释

输出发现表格：

| # | 工作表 | 单元格/范围 | 严重性 | 类别 | 问题 | 建议修复 |
|---|---|---|---|---|---|---|

**严重性：**
- **Critical（严重）** — 输出错误（资产负债表不平衡、公式损坏、现金不匹配）
- **Warning（警告）** — 风险（硬编码、不一致公式、边缘情况失败）
- **Info（信息）** — 风格/最佳实践（颜色编码、布局、命名）

对于**model**范围，添加摘要行：

> 模型类型：[DCF/LBO/三表联动/...] — 总体：[清洁 / 次要问题 / 主要问题] — [N]个严重，[N]个警告，[N]个信息

**未经询问不要更改任何内容** — 先报告，按需修复。

- **资产负债表平衡优先** — 如果不平衡，下游一切都值得怀疑

## 金融术语和知识解释

### 1. DCF模型深度解析

**定义**：DCF（Discounted Cash Flow）即折现现金流模型，通过将未来现金流折现到当前价值来计算公司的内在价值。

**核心公式**：
```
企业价值 = Σ(FCF_t / (1+WACC)^t) + TV / (1+WACC)^n
```

**关键要素**：
- **自由现金流(FCF)**：= 经营现金流 - 资本支出，是公司可自由支配的现金
- **折现率(WACC)**：加权平均资本成本，反映投资风险
- **终值(TV)**：预测期结束后的公司价值
- **预测期(n)**：通常5-10年

**常见错误**：
- 折现率应用到错误期间（年中vs年末）
- 终值未折现或折现错误
- 自由现金流包含利息支出（应为无杠杆）
- WACC使用账面价值而非市场价值

**敏感性分析**：
| 变量变化 | 对估值影响 |
|----------|------------|
| WACC +1% | 估值下降约10% |
| 长期增长率 +0.5% | 估值上升约5-10% |

---

### 2. LBO模型详解

**定义**：LBO（Leveraged Buyout）即杠杆收购，收购方使用大量债务融资来收购目标公司。

**融资结构**：
| 资金来源 | 占比 | 特点 |
|----------|------|------|
| 股权 | 20-30% | PE基金出资 |
| 高级债务 | 40-50% | 优先级最高，利率较低 |
| 夹层债务 | 20-30% | 次级债务，利率较高 |

**关键机制**：
- **现金扫尾(Cash Sweep)**：超额现金自动用于偿还债务
- **PIK利息(Payment-in-Kind)**：利息以增加本金形式支付，不消耗现金
- **债务摊销**：定期偿还本金
- **退出策略**：通常3-7年后通过IPO或出售退出

**常见错误**：
- 债务偿还与现金扫尾机制不匹配
- PIK利息未计入本金
- 退出倍数应用到错误的EBITDA（LTM vs NTM）

---

### 3. 三表联动模型

**定义**：将损益表、资产负债表和现金流量表相互连接的动态财务模型。

**联动关系**：
```
损益表 → 净利润 → 资产负债表（留存收益）
资产负债表 → 营运资金变动 → 现金流量表
现金流量表 → 期末现金 → 资产负债表（现金科目）
```

**三表循环**：
1. **损益表**：计算收入、费用、净利润
2. **资产负债表**：记录资产、负债、权益，需满足恒等式
3. **现金流量表**：反映现金流入流出，连接前后两表

**验证要点**：
- 资产负债表必须平衡（资产=负债+权益）
- 现金流量表现金变动必须等于资产负债表现金变化
- 折旧摊销必须在三表间一致

---

### 4. 资产负债表平衡

**会计恒等式**：`资产 = 负债 + 所有者权益`

**平衡检查方法**：
```
检查单元格 = IF(总资产 = 总负债+权益, "TRUE", 总资产 - (总负债+权益))
```

**不平衡原因**：
- 公式错误或遗漏
- 循环引用未正确设置迭代
- 时间标签不一致
- 合并报表抵消错误

**修复步骤**：
1. 量化每个期间的差距
2. 从最近期间向前追溯
3. 检查循环引用设置
4. 验证所有链接公式

---

### 5. 循环引用解析

**定义**：公式引用其自身单元格或依赖链最终回到自身。

**类型**：
- **有意循环**：LBO/三表联动模型中的利息→债务→现金→利息循环
- **无意循环**：公式错误导致的意外引用

**处理方法**：
```
Excel设置：文件 → 选项 → 公式 → 启用迭代计算
最大迭代次数：100-1000
最大误差：0.0001
```

**常见有意循环**：
| 循环类型 | 循环路径 | 应用场景 |
|----------|----------|----------|
| 利息循环 | 利息→债务→现金→利息 | LBO、三表联动 |
| 税率循环 | 税费→净利润→权益→WACC→折现→估值 | DCF |

---

### 6. WACC详解

**定义**：WACC（Weighted Average Cost of Capital）即加权平均资本成本。

**计算公式**：
```
WACC = (E/(D+E)) × Re + (D/(D+E)) × Rd × (1-T)

其中：
- E = 权益市场价值
- D = 债务市场价值
- Re = 权益成本（CAPM模型）
- Rd = 债务成本
- T = 税率
```

**CAPM模型**：
```
Re = Rf + β × (Rm - Rf)

其中：
- Rf = 无风险利率（如国债收益率）
- β = 贝塔系数（系统性风险）
- Rm - Rf = 市场风险溢价
```

**常见错误**：
- 使用账面价值而非市场价值计算权重
- 债务成本未考虑税盾
- Beta值未去杠杆化/再杠杆化

---

### 7. 终值计算

**定义**：终值是DCF模型中预测期结束后的公司价值，通常占总价值的70%以上。

**方法一：永续增长法**
```
TV = FCF_n × (1 + g) / (WACC - g)

其中：
- FCF_n = 最后一年自由现金流
- g = 长期增长率（通常2-3%，接近GDP增长）
- WACC = 加权平均资本成本
```

**方法二：退出倍数法**
```
TV = EBITDA_n × Exit Multiple

其中：
- EBITDA_n = 最后一年EBITDA
- Exit Multiple = 行业平均EV/EBITDA倍数
```

**注意事项**：
- 终值>企业价值75%时需警惕（黄色警告）
- 增长率g必须小于WACC，否则公式发散
- 退出倍数应与可比公司一致

---

### 8. 营运资金管理

**定义**：营运资金 = 流动资产 - 流动负债

**组成部分**：
| 流动资产 | 流动负债 |
|----------|----------|
| 现金 | 短期借款 |
| 应收账款 | 应付账款 |
| 存货 | 应付职工薪酬 |
| 预付费用 | 应交税费 |

**变动计算**：
```
营运资金变动 = 当期营运资金 - 上期营运资金
```

**现金流量表处理**：
- 营运资金增加 → 现金流出（负号）
- 营运资金减少 → 现金流入（正号）

**常见错误**：符号约定错误（现金流出的正负号）

---

### 9. 协同效应分析

**定义**：协同效应是并购后由于整合带来的额外价值。

**类型**：
| 类型 | 描述 | 示例 |
|------|------|------|
| **成本协同** | 合并后成本节约 | 裁员、整合采购 |
| **收入协同** | 合并后收入增加 | 交叉销售、产品整合 |
| **财务协同** | 资本成本降低 | 税收优惠、债务评级提升 |

**量化方法**：
```
协同效应现值 = Σ(协同效应_t / (1+WACC)^t)
```

**注意事项**：
- 协同效应需分阶段实施
- 实现协同效应需要整合成本
- 协同效应风险应纳入敏感性分析

---

### 10. 摊薄/增厚分析

**定义**：衡量并购交易对每股收益(EPS)的即时影响。

**计算公式**：
```
交易后EPS = (收购方净利润 + 目标方净利润 + 协同效应 - 利息费用) / 交易后股份数

增厚/摊薄 = (交易后EPS / 收购方当前EPS) - 1
```

**判断标准**：
- **Accretion（增厚）**：> 0，交易后EPS增加
- **Dilution（摊薄）**：< 0，交易后EPS减少

**关键因素**：
- 支付方式（现金vs股票）
- 目标公司估值倍数
- 协同效应规模
- 融资成本

**常见错误**：
- 使用错误的股份数量（交易前vs交易后）
- 未考虑协同效应分阶段实施
- 忽略融资利息成本
