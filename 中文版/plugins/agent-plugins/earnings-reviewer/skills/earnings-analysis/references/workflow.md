# 财报更新详细工作流程

本文档提供财报更新流程每个阶段的详细分步说明。

## ⚠️⚠️⚠️ 重要警告：始终使用最新财报数据 ⚠️⚠️⚠️

**请先停止并阅读：**

训练数据已过时。请主动搜索并获取最新的财报材料。使用过时的财报数据是财报分析中最常见的错误。

**开始前：**
1. **检查今天的日期** - 写下当前日期
2. **搜索最新数据** - 使用网络搜索查找最新财报
3. **验证日期** - 确认财报发布在过去3个月内
4. **如果超过3个月** - 获取的是错误季度，请重新搜索

## 第一阶段：财报数据收集（30-60分钟）

### 步骤1：确定最新财报期

**重要**：始终搜索最新财报 - 不要依赖知识截止日期。
**重要**：切勿使用训练数据中的财报数据 - 它已经过时。

**步骤1a：搜索最新财报发布**

**🚨 主动搜索 - 训练数据已过时。🚨**

**强制步骤1：检查今天的日期**
- **明确写下今天的日期**：[月份] [日期], [年份]
- **用它来验证**找到的任何财报是否在3个月内
- **示例**："今天是2024年10月29日"

**强制步骤2：搜索"最新财报"**
- **使用网络搜索**，查询词如：
  - `[公司名称] latest earnings results`
  - `[公司名称] most recent quarterly earnings`
  - `[股票代码] earnings latest quarter`
- **或搜索公司投资者关系网站**：
  - 访问 `investor.[company].com` 或 `[company].com/investors`
  - 导航到"新闻稿"、"新闻"或"财报"部分
  - **按日期排序找到最新发布**
  - 查找关键字："earnings"、"results"、"financial results"、"quarterly results"

**强制步骤3：验证发布日期**
- **查看找到的财报发布日期**
- **计算**：此日期是否在今天之前的3个月内？
- **如果是** → 继续下一步
- **如果否（超过3个月）** → 🚨 错误季度 - 重新搜索更新的财报

**❌ 常见错误避免：**
- ❌ 不搜索就使用训练中的财报数据
- ❌ 根据预期假设"2024年Q3"是最新的
- ❌ 不检查日期就获取找到的第一个财报发布
- ❌ 不将发布日期与今天的日期进行比较
- ❌ 当发布日期超过4个月时仍继续

**✅ 正确方法：**
- ✅ 首先检查今天的日期
- ✅ 明确搜索"latest"或"most recent"
- ✅ 阅读材料上的实际发布日期
- ✅ 确认发布日期在今天的3个月内
- ✅ 如果不确定，使用不同术语重新搜索

**强制步骤4：确定季度**
- **阅读标题/大标题**确定季度（Q1、Q2、Q3、Q4或财季）
- **阅读文档本身的发布日期**
- **验证季度名称和日期都是最新的**

3. **如果IR网站不明确，可使用替代搜索方法：**
   - 网络搜索：`[公司名称] latest earnings results`
   - 网络搜索：`[公司名称] most recent quarterly earnings`
   - 网络搜索：`[股票代码] earnings latest quarter`
   - SEC EDGAR：搜索公司并查看最新的10-Q或10-K提交日期

**找到最新数据的示例搜索：**
- "Nike latest earnings results" → 返回最近报告的季度
- "AAPL most recent quarterly earnings" → 显示Apple最新财报
- "Tesla Q3 2024 earnings" → 结果确认2024年Q3存在

**步骤1b：了解公司财年日历**

通过搜索确定最新季度后，了解公司的财年以正确解读：

**常见财年模式：**
- **日历年(CY)**：Q1=1月-3月, Q2=4月-6月, Q3=7月-9月, Q4=10月-12月
- **Nike财年**：Q1=6月-8月, Q2=9月-11月, Q3=12月-2月, Q4=3月-5月（5月财年末）
- **Apple财年**：Q1=10月-12月, Q2=1月-3月, Q3=4月-6月, Q4=7月-9月（9月财年末）
- **Walmart财年**：Q1=2月-4月, Q2=5月-7月, Q3=8月-10月, Q4=11月-1月（1月财年末）

许多公司在财报发布标题中注明其财年。如有需要，搜索 `[company] fiscal year calendar`。

**步骤1c：强制验证 - 验证获取的最新数据**

🛑 **停止 - 在验证所有这些之前不要继续：**

- [ ] ✅ **今天的日期已写下**：[月份] [日期], [年份]
- [ ] ✅ **已主动搜索**使用"latest earnings"或"most recent earnings"
- [ ] ✅ **找到的财报发布日期**：[月份] [日期], [年份]
- [ ] ✅ **已验证发布在今天的3个月内**（计算一下！）
- [ ] ✅ **没有仅根据今天的日期假设季度**
- [ ] ✅ **可以看到确认季度/期间的实际新闻稿**
- [ ] ✅ **已打开并阅读**实际财报材料（不仅仅是假设它们存在）

**🚨 危险信号 - 如果以下任何一项为真，获取了错误季度：**
- 🚨 发布日期超过90天
- 🚨 依赖预期而非通过搜索发现的内容
- 🚨 实际上没有看到确认该季度存在的新闻稿或文件
- 🚨 未搜索就使用训练数据
- 🚨 无法说明确切的发布日期
- 🚨 找到的发布日期来自2023年或更早（当今天是2024+时）

**如果出现任何危险信号**：停止并重新搜索。不要使用过时数据继续。

**步骤1c：处理命名变体**

公司使用不同的术语 - 识别这些模式：

**季度术语：**
- "Q1 2024"、"Q1 FY24"、"First Quarter 2024"、"1Q24"
- "Third Quarter Fiscal 2024"、"Q3 FY2024"、"3Q FY24"

**财报发布标题：**
- "[公司] Reports Q3 2024 Results"
- "[公司] Announces Third Quarter Fiscal 2024 Financial Results"
- "[公司] Q3 Revenue Grew 15% Year-over-Year"

**SEC文件搜索：**
- 公司名称可能与通用名称不同（例如"Meta Platforms, Inc." vs "Facebook"）
- 按股票代码搜索可可靠找到文件
- 查找最新的10-Q（季度）或10-K（如果是Q4则为年度）

### 步骤2：收集财报材料

搜索并确认最新季度后，收集以下内容：

**⚠️ 重要：搜索并访问实际文件 - 不要依赖训练数据。**

**主要材料（必需）：**
- **财报新闻稿** - 通常在公司投资者关系网站的"新闻稿"或"新闻"下
  - 导航到IR网站并找到实际新闻稿
  - 搜索模式："[公司名称] latest earnings"、"[公司名称] Q[X] [年份] earnings results"
  - 查找PDF或HTML版本
  - **验证日期与步骤1中找到的日期匹配**（应在过去1-3个月内）
  - **阅读实际文件**确认季度并获取报告数字

- **10-Q或10-K文件** - 在SEC EDGAR上（sec.gov/edgar/searchedgar/companysearch.html）
  - 按股票代码搜索
  - 对于第1-3季度：查找最新的10-Q
  - 对于Q4：查找10-K（年度报告）
  - 注意：可能在财报发布后1-5天提交
  - 直接链接格式：`https://www.sec.gov/cgi-bin/viewer?accession=[accession-number]`

- **财报电话会议记录** - 🚨 **验证记录上的日期** 🚨
  - **搜索**："[Company] latest earnings call transcript"或"[Company] Q[X] [Year] earnings call transcript"
  - **来源**：
    - 公司IR网站（有些直接发布记录）
    - Seeking Alpha：搜索"[Company] [latest quarter] earnings call transcript"
    - AlphaStreet、Motley Fool（替代来源）
  - **重要日期检查**：
    - ✅ **使用任何记录前，验证记录本身上的日期**
    - ✅ **记录日期必须与步骤1中的财报发布日期匹配**
    - ✅ **如果记录显示"2023年Q2"但发布是"2024年Q3"，获取了错误记录**
    - 🚨 **常见错误**：不检查日期就获取旧记录
  - 如果记录尚未可用，请收听网络直播重播或注明等待记录

**补充材料（如可用）：**
- **投资者演示/幻灯片** - 通常与新闻稿一起发布在IR网站上
  - 通常标题为"Q[X] [Year] Earnings Presentation"或"Investor Presentation"
  - PDF格式，包含管理层在财报电话会议期间展示的幻灯片

- **补充数据文件** - 一些公司提供包含详细指标的Excel文件
  - 查找"Supplemental Financial Information"或"Investor Data Sheet"

**参考材料（用于比较）：**
- **上一季度业绩** - 用于QoQ比较
  - 来自上一季度的财报发布（90天前）

- **去年同期** - 用于YoY比较
  - 来自去年同一季度（4个季度前）

- **先前预测** - 如果此前覆盖过这家公司
  - 来自上次财报更新或初始报告
  - 检查本季度指标的先前预测

- **一致预期** - 来自Bloomberg、FactSet、Refinitiv或Yahoo Finance
  - 重要：使用财报发布前的预期
  - 查找"as of [date before earnings]"以确保是公告前的一致预期
  - 需要用于超预期/不及预期分析

**🛑 继续步骤3前的强制验证：**

**日期 - 验证所有日期匹配：**
- [ ] ✅ **今天的日期已写下**：_______________
- [ ] ✅ **财报发布日期**：_______________（必须在今天的3个月内）
- [ ] ✅ **财报电话会议记录日期**：_______________（必须与发布日期±1天匹配）
- [ ] ✅ **10-Q/10-K提交日期**：_______________（必须与发布季度相同）
- [ ] ✅ **所有材料显示同一季度**（例如，都显示"2024年Q3"，不是混合季度）

**搜索与访问 - 验证主动搜索已完成：**
- [ ] ✅ **已搜索**"latest earnings"（不是根据当前日期假设）
- [ ] ✅ **已访问**实际财报新闻稿并阅读
- [ ] ✅ **已打开**实际财报电话会议记录并验证日期
- [ ] ✅ **已确认**通过检查日期这是最新季度
- [ ] ✅ 从实际发布中获得完整财务业绩（收入、EPS、利润率等）
- [ ] ✅ 获得带来源日期的财报前一致预期

**🚨 危险信号 - 如果以下任何一项为真，停止：**
- 🚨 未实际搜索或访问财报材料
- 🚨 从记忆或训练数据而不是当前文件工作
- 🚨 财报发布日期超过90天
- 🚨 无法说明财报发布的确切日期
- 🚨 记录日期与发布日期不匹配
- 🚨 材料显示不同季度（例如，发布说Q3但记录说Q2）
- 🚨 不验证日期就获取第一个结果

### 步骤4：确定电话会议的关键主题

收听或阅读财报电话会议记录并记录：
- 管理层语气（自信、谨慎、防御性？）
- 强调的关键主题（产品发布、地域趋势、竞争）
- 分析师问题（投资者关心什么？）
- 提供的指引（上调、下调、维持、引入？）
- 任何惊喜或意外评论

## 第二阶段：分析（2-3小时）

### 步骤5：超预期/不及预期分析

对于每个超预期或不及预期的关键指标，解释：

**如果超预期：**
- 什么驱动了超预期表现？
- 是一次性还是可持续的？
- 管理层是否指引未来更高？
- 这如何影响我们的投资主题？

**如果不及预期：**
- 哪里出了问题？
- 是公司特定问题还是行业普遍问题？
- 管理层是否正在采取纠正措施？
- 这如何影响我们的投资主题？

**示例格式：**
```
■ **收入超预期3%，受DTC强劲表现驱动**

收入135亿美元超出我们131亿美元的预期4亿美元(3%)，超出一致预期132亿美元3亿美元(2%)。超预期主要受直接面向消费者渠道驱动，该渠道同比增长18%（vs.我们预期的12%），抵消了弱于预期的批发业务(-5% vs.预期持平)。管理层指出强劲的数字需求和成功的产品发布（Pegasus 40跑鞋、新Jordan配色）是关键驱动因素。DTC现在占总收入的42%，而一年前为38%，证明渠道转型战略成功。
```

### 步骤6：部门/地域/产品分析

按以下方面分析业绩：
- 业务部门（如果是多部门公司）
- 地域（北美、欧洲、中国等）
- 产品类别
- 渠道（零售、批发、电子商务）

确定：
- 什么超出预期？
- 什么低于预期？
- 与前几个季度相比的趋势
- 管理层对每个领域前景的评论

### 步骤7：利润率分析

分析盈利能力：
- 毛利率：上升还是下降？为什么？
- 营业利润率：上升还是下降？为什么？
- 关键驱动因素（定价、产品组合、成本、杠杆效应）
- 未来前景

### 步骤8：指引分析

如果公司提供指引：
- 将新指引与先前指引比较
- 与内部预测和市场预期比较
- 评估可信度（公司是否有保守指引的记录？还是超预期的记录？）
- 确定指引背后的关键假设

如果公司未提供指引：
- 明确注明
- 根据业绩和评论提供独立展望

### 步骤9：更新财务模型

更新以下预测：
- 当前年度（剩余季度）
- 下一年度
- 可能再下一年度

**清晰显示：**
```
更新预测：
─────────────────────────────────────────────────
                        旧预测      新预测      变化        原因
2024财年E收入          $XX.XB      $XX.XB      +X.X%      [简要原因]
2024财年E EBITDA       $X.XB       $X.XB       +X.X%      [简要原因]
2024财年E EPS          $X.XX       $X.XX       +X.X%      [简要原因]

2025财年E收入          $XX.XB      $XX.XB      +X.X%      [简要原因]
2025财年E EBITDA       $X.XB       $X.XB       +X.X%      [简要原因]
2025财年E EPS          $X.XX       $X.XX       +X.X%      [简要原因]
```

### 步骤10：更新估值与目标价格

基于更新的预测：
- 重新计算DCF（使用更新的现金流）
- 更新可比公司倍数（如果同行已发布财报）
- 确定新的公允价值
- 决定目标价格是否变化

**目标价格决策：**
- 如果预测变化显著(>5%) → 通常改变目标价格
- 如果预测变化轻微(<5%) → 可能维持目标价格
- 如果投资主题强化/弱化 → 即使预测没有变化也可能改变

### 步骤11：评估评级影响

决定是否改变评级：
- 如果业绩显著好于预期 + 指引上调 → 考虑上调评级
- 如果业绩显著差 + 指引下调 → 考虑下调评级
- 如果符合预期或喜忧参半 → 通常维持评级

**考虑因素：**
- 股票反应（上涨/下跌/持平？）
- 估值（相对于新预测昂贵/便宜？）
- 风险/回报（不对称性是否转移？）

## 第三阶段：图表生成（1-2小时）

### 步骤12：生成8-12个图表

创建专注于季度趋势和新内容的图表。

**必需图表（共8-12个）：**

1. **季度收入进展**（柱状图）
   - 最近8-12个季度
   - 显示每个季度与预期的超预期/不及预期
   - 突出当前季度

2. **季度EPS进展**（柱状图）
   - 最近8-12个季度
   - 显示与预期的超预期/不及预期
   - 调整后和GAAP

3. **季度利润率趋势**（折线图）
   - 毛利率、EBIT利润率、净利率
   - 最近8-12个季度
   - 显示趋势

4. **按部门/地域划分的收入**（堆叠柱状图或表格）
   - 当前季度vs.同比
   - 部门增长率

5. **关键运营指标**（多线图）
   - 客户数量、ARPU、销售单位等（相关指标）
   - 最近8-12个季度

6. **超预期/不及预期摘要**（瀑布图或表格）
   - 显示超预期/不及预期的构成部分
   - 什么驱动了与预期的差异

7. **预测修正图表**（前后比较）
   - 旧财年预测vs.新财年预测
   - 显示变化的柱状图

8. **估值图表**（市盈率或EV/EBITDA倍数）
   - 历史倍数范围
   - 当前倍数
   - 公允价值倍数

**可选图表（如有空间）：**
- 同行比较（如同行已发布财报）
- 指引vs.市场比较
- 现金流指标
- 资产负债表亮点（如值得注意）

**图表样式指南：**
- 专注于趋势（季度进展）
- 突出变化（超预期/不及预期、预测修正）
- 保持简单清晰（这是快速周转报告）

## 第四阶段：报告创建（2-3小时）

### 步骤13：创建DOCX报告

使用DOCX技能创建8-12页报告。

完整的逐页模板和格式要求请参见[report-structure.md](report-structure.md)。

**关键步骤：**
1. 创建第1页，包含财报摘要和快速要点
2. 添加详细业绩分析（第2-3页）
3. 包含关键指标和指引（第4-5页）
4. 更新投资主题（第6-7页）
5. 提供估值和预测（第8-10页）
6. 如有需要添加附录（第11-12页）

## 金融术语和知识解释

### 1. 财报数据收集流程 (Earnings Data Collection Process)

财报分析的第一步是收集准确的最新数据，这包括：
- **财报新闻稿**：公司官方发布的财务业绩摘要
- **10-Q/10-K文件**：提交给SEC的正式财务报告
- **电话会议记录**：管理层讨论业绩的会议记录
- **投资者演示**：管理层在财报电话会议中使用的演示文稿

### 2. 一致预期 (Consensus Estimates)

一致预期是华尔街分析师对公司财务指标的平均预测，通常包括：
- 收入预测
- 每股收益(EPS)预测
- 利润率预测

一致预期是衡量公司业绩"超预期"或"不及预期"的基准。

### 3. 超预期/不及预期分析 (Beat/Miss Analysis)

当公司实际业绩与市场预期进行比较时：
- **超预期(Beat)**：实际业绩好于一致预期
- **不及预期(Miss)**：实际业绩差于一致预期
- **符合预期(Inline)**：实际业绩与预期持平

### 4. 同比与环比 (YoY vs QoQ)

- **同比(Year-over-Year, YoY)**：与去年同一时期比较
- **环比(Quarter-over-Quarter, QoQ)**：与上一个季度比较

这两种比较方法帮助分析师评估业务趋势。

### 5. 财年日历 (Fiscal Year Calendar)

不同公司采用不同的财年结束日期：
- **日历年**：12月31日结束
- **零售公司**：1月31日结束（避开圣诞季）
- **科技公司**：9月30日结束

了解公司的财年日历对于正确解读季度业绩至关重要。

### 6. 指引 (Guidance)

指引是管理层对未来财务业绩的预测，通常包括：
- 收入范围
- 每股收益范围
- 利润率预期

指引是投资者评估公司未来前景的重要参考。

### 7. 估值方法 (Valuation Methodologies)

常用的估值方法包括：
- **DCF(折现现金流)**：通过折现未来现金流计算公司价值
- **可比公司分析**：与同行业公司比较估值倍数
- **先例交易分析**：参考类似公司的并购交易价格

### 8. 目标价格 (Price Target)

目标价格是分析师对股票未来12个月合理价值的估计，通常基于：
- DCF估值
- 可比公司倍数
- 盈利预测

### 9. 评级 (Rating)

分析师对股票的投资建议，常见评级包括：
- **买入(Buy)**：建议买入
- **跑赢大盘(Outperform)**：预期表现优于市场
- **持有(Hold)**：建议持有
- **跑输大盘(Underperform)**：预期表现逊于市场
- **卖出(Sell)**：建议卖出

### 10. 投资主题 (Investment Thesis)

投资主题是分析师对公司的核心投资观点，解释为什么认为公司股票值得投资。财报分析的重要部分是评估业绩对投资主题的影响。

## Appendix: 金融背景知识

这份文件是"财报分析工作流（Workflow）"参考文档的详细说明。一份 Earnings Update 报告的生产不是"突发灵感"——而是严格按照标准化工作流程执行的。

---

### 1. 财报分析的"标准时间线"

| 时间点 | 事件 | 工作 |
|--------|------|------|
| T-0（财报发布时刻） | 新闻稿 + 10-Q 公开 | 立即读取关键数字 |
| T+15 分钟 | 第一轮市场反应 | 记录盘后股价变动 |
| T+1 小时 | 电话会议开始 | 边听边标注关键问题 |
| T+2 小时 | 数据收集完成 | 对比实际 vs 共识 vs 上次预测 |
| T+4 小时 | 写完第一稿 | 发给高级分析师审核 |
| T+24 小时 | 最终稿发出 | 包含所有修订后观点 |

---

### 2. 工作流的"三不中断原则"

| 原则 | 说明 |
|------|------|
| 不中断数据流 | 从 FactSet 拉完数据直接导入模型，不要手工录入 |
| 不中断分析流 | 边看数据边写笔记，不要做完了再回忆 |
| 不中断审核流 | 写完一段立即发给同事看，不要全部写完了才发现方向错了 |

---

### 给小白的一句话

> Workflow 就是财报分析的"流水线操作手册"——不是"灵感来了写一段"，而是像工厂一样按秒表操作：财报出来 15 分钟内读完关键数字、1 小时后听完电话会议、4 小时内出第一稿、24 小时内发布最终版。这不是急性子——而是你要抢在你的竞争对手（隔壁高盛的分析师）之前把报告发到基金经理的邮箱里。谁先到，谁的股票推荐先被下单。
