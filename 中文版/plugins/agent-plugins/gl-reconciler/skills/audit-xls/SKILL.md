# 审计电子表格

审计公式和数据的准确性和错误。范围决定深度 — 从对选定区域的快速公式检查到完整的财务模型完整性审计。

## 步骤1：确定范围

如果用户已经给出范围，则使用它。否则**询问用户**：

> 你希望我审计什么范围？
> - **selection** — 仅当前选定的范围
> - **sheet** — 仅当前活动工作表
> - **model** — 整个工作簿，包括财务模型完整性检查（资产负债表平衡、现金对账、滚动计算、逻辑合理性）

**model**范围是最深的 — 在发送给客户或投资委员会之前，对DCF、LBO、三表模型、合并、可比分析或任何集成财务模型使用此范围。

---

## 步骤2：公式级检查（所有范围）

无论范围如何，都运行以下检查：

| 检查项 | 检查内容 |
|---|---|
| 公式错误 | `#REF!`, `#VALUE!`, `#N/A`, `#DIV/0!`, `#NAME?` |
| 公式中的硬编码 | `=A1*1.05` — `1.05` 应该是单元格引用 |
| 不一致的公式 | 破坏行列中相邻单元格模式的公式 |
| 差一错误范围 | `SUM`/`AVERAGE` 遗漏第一行或最后一行 |
| 覆盖的公式 | 看起来像公式但实际上是硬编码值的单元格 |
| 循环引用 | 有意或无意的 |
| 损坏的跨表链接 | 引用已移动或删除的单元格 |
| 单位/比例不匹配 | 千与百万混合，百分比存储为整数 |
| 隐藏行/标签页 | 可能包含覆盖值或过期计算 |

## 步骤3：模型完整性检查（仅限MODEL范围）

如果范围是**model**，识别模型类型（DCF / LBO / 三表 / 合并 / 可比分析 / 自定义）并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项 | 检查内容 |
|---|---|
| 输入/公式分离 | 输入是否与计算清晰分离？ |
| 颜色约定 | 蓝色=输入，黑色=公式，绿色=链接 — 或模型使用的任何约定是否一致应用？ |
| 标签页流程 | 逻辑顺序（假设 → 利润表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期标题 | 所有标签页一致？ |
| 单位 | 一致（千 vs 百万 vs 实际值）？ |

### 3b. 资产负债表

| 检查项 | 测试方法 |
|---|---|
| 资产负债表平衡 | 总资产 = 总负债 + 所有者权益（每个期间） |
| 留存收益滚动 | 期初留存收益 + 净利润 − 股利 = 期末留存收益 |
| 商誉/无形资产 | 从收购假设流入（如有并购） |

如果资产负债表不平衡，**量化每个期间的差距并追溯断点** — 在修复之前其他都不重要。

### 3c. 现金流量表

| 检查项 | 测试方法 |
|---|---|
| 现金对账 | 现金流量表期末现金 = 资产负债表现金（每个期间） |
| 现金流量总和 | 经营现金流 + 投资现金流 + 筹资现金流 = 现金变动 |
| 折旧摊销匹配 | 现金流量表上的折旧摊销 = 利润表上的折旧摊销 |
| 资本支出匹配 | 现金流量表上的资本支出与资产负债表上的PP&E滚动一致 |
| 营运资金变动 | 符号与资产负债表变动匹配（应收账款变动、应付账款变动、存货变动） |

### 3d. 利润表

| 检查项 | 测试方法 |
|---|---|
| 收入构建 | 与部门/产品明细一致 |
| 税金 | 税费 = 税前收入 × 税率（考虑递延税调整） |
| 股份数 | 与稀释时间表一致（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是LBO/三表模型中常见的有意循环
- 如果有意：验证迭代切换存在并正常工作
- 如果无意：追溯循环并标记如何打破它

### 3f. 逻辑与合理性

| 检查项 | 标记条件 |
|---|---|
| 增长率 | 收入增长 >100% 且无解释 |
| 利润率 | 超出行业标准 |
| 终端价值主导 | 终端价值 > DCF企业价值的约75%（黄色警告） |
| 曲棍球杆效应 | 预测在远期年份不切实际地上升 |
| 复利效应 | EBITDA到第10年复合增长到荒谬金额 |
| 边缘情况 | 模型在0%或负增长、负EBITDA、杠杆为负时崩溃 |

### 3g. 特定模型类型的错误

**DCF：**
- 折现率应用于错误期间（年中 vs 年末）
- 终端价值未折现回现值
- WACC使用账面价值而非市场价值
- 自由现金流包含利息费用（应为无杠杆）
- 税盾重复计算

**LBO：**
- 债务偿还与现金扫尾机制不匹配
- PIK利息未计入本金
- 管理层滚动未反映在回报中
- 退出倍数应用于错误的EBITDA（LTM vs NTM）
- 费用未从首日股权中扣除

**合并：**
- 摊薄/增厚使用错误的股份数（交易前 vs 交易后）
- 协同效应未分阶段实施
- 购买价格分配不平衡
- 未包含现金的机会成本利息
- 交易费用未在资金来源与用途中

**三表模型：**
- 营运资金变动符号错误
- 折旧与PP&E时间表不匹配
- 债务到期时间表与本金偿还不匹配
- 股利超过净利润且无解释

## 步骤4：报告

输出发现表：

| # | 工作表 | 单元格/范围 | 严重程度 | 类别 | 问题 | 建议修复 |
|---|---|---|---|---|---|---|

**严重程度：**
- **Critical** — 输出错误（资产负债表不平衡、公式损坏、现金不对账）
- **Warning** — 风险（硬编码、不一致的公式、边缘情况失败）
- **Info** — 样式/最佳实践（颜色编码、布局、命名）

对于**model**范围，添加摘要行：

> 模型类型: [DCF/LBO/三表/...] — 总体: [清洁 / 次要问题 / 主要问题] — [N] 严重, [N] 警告, [N] 信息

**未经询问不要更改任何内容** — 先报告，按请求修复。

---

## 注意事项

- **资产负债表平衡优先** — 如果不平衡，下游所有内容都值得怀疑
- **硬编码覆盖是无声错误的头号来源** — 积极搜索
- **符号约定错误**（现金流出的正负）极为常见
- 如果模型使用VBA宏，注意任何仅从公式无法审计的宏驱动计算

## 金融术语和知识解释

### 1. 财务模型审计 (Financial Model Audit)

财务模型审计是验证模型准确性、完整性和合理性的过程，确保模型输出可靠。

### 2. DCF模型 (Discounted Cash Flow)

DCF模型是一种估值方法，通过将未来现金流折现到现值来评估企业价值。

### 3. LBO模型 (Leveraged Buyout)

LBO模型是用于分析杠杆收购交易的财务模型，考虑债务融资和股权回报。

### 4. 三表模型 (3-Statement Model)

三表模型整合利润表、资产负债表和现金流量表，展示公司的财务状况。

### 5. 资产负债表平衡 (BS Balance)

资产负债表平衡要求总资产等于总负债加所有者权益，是财务模型的基本要求。

### 6. 现金对账 (Cash Tie-out)

现金对账确保现金流量表的期末现金与资产负债表的现金余额一致。

### 7. 循环引用 (Circular Reference)

循环引用是指公式依赖于自身的计算结果，在LBO模型中有时是有意设计的。

### 8. 终端价值 (Terminal Value)

终端价值是企业在预测期结束后的价值，通常占DCF估值的大部分。

### 9. WACC (Weighted Average Cost of Capital)

WACC是加权平均资本成本，用于折现未来现金流。

### 10. 协同效应 (Synergies)

协同效应是合并后企业预期获得的额外价值，如成本节约或收入增长。
