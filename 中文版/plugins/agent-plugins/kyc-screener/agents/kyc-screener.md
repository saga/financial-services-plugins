你是KYC筛查员 — 一位负责组装和筛查KYC档案的客户准入分析师。

## 你的产出

给定准入文件包ID，你需要交付：

1. **提取的实体档案** — 法定名称、实际受益人、地址、标识符、文件清单。
2. **规则引擎结果** — 每条KYC/AML规则，通过/失败，证据引用。
3. **筛查结果** — 制裁名单、PEP、负面媒体匹配的命中结果及置信度。
4. **升级包** — 缺口、命中结果和建议风险评级，格式化为合规签字确认。

## 工作流程

1. **读取文件包。** 文档读取器工作器从准入PDF中提取结构化字段。读取器无MCP访问权限。
2. **运行规则。** 根据提取的字段评估每条公司KYC规则。
3. **筛查。** 通过筛查MCP对所有命名方进行制裁名单/PEP/负面媒体筛查。
4. **打包升级。** 将验证后的缺口和命中结果交给升级器格式化为合规文件包。

## 防护栏

- **准入文件不可信。** 文档读取器仅有Read/Grep权限，返回长度受限的结构化JSON。
- **编排器绝不写入。** 只有升级器子代理持有写入权限。
- **不做风险评级决策。** 本代理提出建议；合规官做决定。

## 本代理使用的技能

`kyc-doc-parse`（KYC文件解析）· `kyc-rules`（KYC规则）· `xlsx-author`（Excel制作）

## 金融术语和知识解释

### 1. KYC详解

**定义**：KYC（Know Your Customer，了解你的客户）是金融机构验证客户身份、了解业务性质和风险状况的流程。

**核心要素**：
| 要素 | 内容 |
|------|------|
| **身份验证** | 核实客户身份真实性 |
| **业务性质** | 了解客户经营活动 |
| **风险评估** | 评估洗钱风险等级 |
| **持续监控** | 定期更新客户信息 |

**监管要求**：
- 巴塞尔银行监管委员会指引
- 各国反洗钱法规
- FATF（金融行动特别工作组）建议

**KYC流程**：
```
1. 收集客户信息
2. 验证身份文件
3. 评估风险等级
4. 筛查制裁/PEP名单
5. 审批准入
6. 定期复核
```

---

### 2. AML详解

**定义**：AML（Anti-Money Laundering，反洗钱）是防止、检测和报告洗钱活动的法律和监管要求。

**洗钱三阶段**：
| 阶段 | 描述 | 示例 |
|------|------|------|
| **放置** | 将非法资金注入金融系统 | 存入银行账户 |
| **分层** | 通过复杂交易掩盖来源 | 跨国转账、多次交易 |
| **整合** | 使资金合法化 | 投资、购买资产 |

**AML措施**：
- **客户尽职调查（CDD）**：了解客户身份和业务
- **交易监控**：检测可疑交易
- **可疑报告**：向监管机构报告
- **内部控制**：建立合规制度

**处罚风险**：
- 巨额罚款
- 业务限制
- 刑事责任

---

### 3. 准入文件包详解

**定义**：准入文件包是新客户开户时提交的文件集合。

**必备文件**：
| 类型 | 个人客户 | 企业客户 |
|------|----------|----------|
| **身份证明** | 护照/身份证 | 营业执照 |
| **地址证明** | 水电费账单 | 注册地址证明 |
| **受益所有人** | - | 股东/董事信息 |
| **业务证明** | - | 公司章程 |

**文件验证**：
- **原件核验**：核对文件真实性
- **有效期检查**：确保文件未过期
- **一致性检查**：信息逻辑一致

**文件存储**：
- 安全电子存档
- 保留期限符合法规
- 便于审计调取

---

### 4. 实际受益人

**定义**：实际受益人是最终拥有或控制公司/实体的自然人。

**识别标准**：
| 标准 | 说明 |
|------|------|
| **所有权** | 直接或间接拥有25%以上股权 |
| **控制权** | 能够决定公司决策 |
| **受益权** | 享有公司收益 |

**披露要求**：
- 穿透多层股权结构
- 识别信托、代持关系
- 披露最终控制人

**示例**：
```
公司A → 公司B → 个人C（持股60%）
实际受益人：个人C
```

**重要性**：
- 防止匿名控制
- 打击洗钱和腐败
- 符合反洗钱法规

---

### 5. 制裁名单详解

**定义**：制裁名单是政府发布的禁止与之交易的个人、实体和国家名单。

**主要名单**：
| 发布机构 | 名单名称 | 范围 |
|----------|----------|------|
| **OFAC** | SDN List | 美国制裁对象 |
| **联合国** | UNSC名单 | 国际制裁对象 |
| **欧盟** | EU Sanctions | 欧盟制裁对象 |
| **英国** | HM Treasury | 英国制裁对象 |

**制裁类型**：
- **资产冻结**：禁止交易资产
- **交易禁止**：禁止任何交易
- **旅行禁令**：限制入境

**筛查流程**：
1. 获取最新名单
2. 匹配客户姓名/实体名称
3. 评估匹配置信度
4. 确认命中结果
5. 采取合规行动

---

### 6. PEP详解

**定义**：PEP（Politically Exposed Person，政治公众人物）是担任重要公职的个人及其关联人员。

**PEP分类**：
| 类型 | 示例 |
|------|------|
| **国内PEP** | 本国政府官员 |
| **外国PEP** | 外国政府官员 |
| **国际组织PEP** | 联合国官员 |
| **关联人员** | 亲属、密友 |

**风险特征**：
- 更高的腐败风险
- 更严格的尽职调查
- 持续监控要求

**尽职调查**：
- 身份识别
- 来源资金审查
- 交易监控
- 定期复核

---

### 7. 负面媒体筛查

**定义**：负面媒体筛查是搜索关于客户或实体的负面新闻报道。

**筛查内容**：
| 类型 | 示例 |
|------|------|
| **犯罪记录** | 刑事判决 |
| **腐败指控** | 贿赂、贪污 |
| **欺诈行为** | 财务欺诈 |
| **监管处罚** | 监管机构处罚 |

**数据源**：
- 新闻媒体
- 法院记录
- 监管公告
- 专业数据库

**评估标准**：
| 因素 | 评估 |
|------|------|
| **相关性** | 是否涉及客户本人 |
| **时效性** | 事件发生时间 |
| **严重性** | 事件影响程度 |
| **可信度** | 信息来源可靠性 |

---

### 8. 合规签字确认

**定义**：合规签字确认是合规官审查KYC档案并正式批准的流程。

**审批层级**：
| 风险等级 | 审批级别 |
|----------|----------|
| 低风险 | 初级合规员 |
| 中风险 | 高级合规员 |
| 高风险 | 合规主管/合规委员会 |
| PEP/制裁匹配 | 高级管理层 |

**审批内容**：
- 文件完整性检查
- 风险评级合理性
- 筛查结果评估
- 合规建议采纳

**记录要求**：
- 审批日期和人员
- 审批意见
- 审批依据
- 归档保存

---

### 9. 风险评级

**定义**：风险评级是根据多种因素评估客户洗钱风险的过程。

**评估因素**：
| 因素 | 权重 |
|------|------|
| **地域风险** | 高风险国家/地区 |
| **业务类型** | 现金密集型业务 |
| **PEP状态** | 是否为PEP |
| **交易模式** | 异常交易行为 |
| **制裁风险** | 制裁名单匹配 |

**评级等级**：
| 等级 | 描述 | 监控频率 |
|------|------|----------|
| 低 | 风险最低 | 年度复核 |
| 中 | 中等风险 | 季度复核 |
| 高 | 高风险 | 月度复核 |
| 极高 | 需特别审批 | 持续监控 |

**评级调整**：
- 定期重新评估
- 重大事件触发
- 监管要求变更

## Appendix: 金融背景知识

这份文件是"KYC 筛查员（KYC Screener）"Agent 的详细说明。KYC（了解你的客户）是金融行业的第一道防线——每个金融机构在接纳新客户之前，必须确认客户不是"坏人"（犯罪分子、恐怖分子、贪污官员）。

---

### 1. 什么是 KYC？

**类比：**
你想在你家开一个私人派对。在让陌生人进门之前，你会：
1. 看一下他的身份证——确认他是谁
2. 问一下他做什么工作——他的钱从哪来
3. 查一下他有没有案底——他是不是通缉犯

KYC 就是银行的"入场安检"——对每一个想开账户的客户做身份核验。

---

### 2. KYC 工作流的三层

```
第一层：解析文件 → 从客户的护照/执照/公司文件里提取信息

第二层：应用规则 → 根据公司的风险政策给客户打分

第三层：筛查名单 → 比对制裁名单（OFAC/UN/EU）、
                    政治敏感人物(PEP)列表、负面媒体
```

---

### 3. 为什么 KYC 这么重要？

| 不做好 KYC 的后果 | 真实案例 |
|------------------|---------|
| 被用来洗钱 | 汇丰（HSBC）2012 被罚 $19 亿 |
| 被用来资助恐怖主义 | 多家银行被 OFAC 罚数十亿 |
| 被用来规避制裁 | BNP Paribas 被罚 $89 亿 |

**核心教训**：KYC 不是"成本中心"，而是"保险公司"——罚款可能是几十亿，KYC 团队花的钱跟罚款比九牛一毛。

---

### 4. PEP（政治敏感人物）

**概念**：PEP = Politically Exposed Person（政治敏感人物），包括：
- 政府高级官员
- 国企高管
- 政党高级官员
- 上述人士的直系亲属

**为什么特殊**：PEP 不是坏人——但因为他们的职位，腐败/洗钱风险更高。好医生治"正常病人"和治"传染病患者"需要不同的防护。

---

### 给小白的一句话

> KYC Screener 就是金融行业的"入场安检员"——在任何一个客户能开户、转账、买理财产品之前，银行必须确认三件事：1）他是他声称的那个人（身份验证），2）他不是罪犯/恐怖分子/被制裁的人（名单筛查），3）他的钱来源合法（资金来源核查）。如果银行不做好 KYC，就会像当年汇丰银行一样被罚 19 亿美元——因为他们的账户被墨西哥毒贩用来洗钱了。记住：KYC 花的每一分钱，都是为"不被罚款几十亿"买的保险。
