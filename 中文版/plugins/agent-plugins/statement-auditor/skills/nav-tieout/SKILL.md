# NAV结平

给定生成的LP对账单和期间的NAV包（通过nav MCP），独立重新计算LP资本账户并逐行比较。

> **生成的对账单是测试对象。** NAV包是事实来源。

## 重新计算LP资本账户

```
期初资本（上期对账单期末）
  + 供款（本期计提的资本调用）
  − 分配（现金 + 实物分配）
  + 分配的净收入/（亏损）
       = LP% × （已实现 + 未实现P&L − 管理费 − 基金费用）
  − 附带权益分配（如果本期已结晶）
期末资本
```

通过NAV包提取每个输入：LP承诺%、基金级P&L组件、费用和费用总计、水柱输出。

## 比较

对对账单上的每一行，与你的重新计算值比较。容差：`0.01`。对每个不匹配，注意驱动它的输入（例如，"分配的净收入不同 — 对账单使用12.40%所有权，NAV包显示在Q1转移后12.38%"）。

## 附加检查

- 此对账单上的期末资本 = 下期草稿的期初资本（如果可用）。
- 所有LP期末资本总和 = 基金NAV（在舍入内）。
- 承诺、未拨付和可支配数字与承诺登记册一致。

## 输出

每行通过/失败，重新计算值与对账单值并列，以及标志列表。编辑对账单 — 发布器在审查后根据标志行动。

**本文件专属**：

- **实物分配（In-kind Distribution）**：以证券而非现金形式向 LP 分配。
- **可支配（Recallable）**：GP 可以召回的已分配金额，通常用于覆盖未来费用。

- **容差（Tolerance）**：可接受的小差异范围，通常 0.01 货币单位，用于舍入差异。

- **发布器（Publisher）**：负责根据结平检查结果更新和分发 LP 对账单的代理或人员。

## Appendix: 金融背景知识

这份文件是"NAV 核对（NAV Tieout）"技能的详细说明。在私募基金行业，NAV（净资产值）是衡量基金价值的核心数字——所有费用、分配、业绩报酬都基于 NAV 计算。NAV Tieout 确保"基金账上的 NAV"和"报给 LP 的 NAV"一致。

---

### 1. 什么是 NAV？

**通俗解释**：NAV = Net Asset Value（净资产值）= 基金持有的所有资产的总市值 - 所有负债。

**类比**：你的净资产 = 你的房子市值 + 银行存款 + 股票账户 - 房贷 - 信用卡欠款。基金的 NAV 就是基金的"净资产"——等于基金所有投资的总市值减去欠债。

---

### 2. NAV Tieout 的"核对三步骤"

```
步骤 1：拿到 GP 的估值包 → 列出每笔投资的估值

步骤 2：独立快速验证 → 用 Comps 或 DCF 快速验证关键数字是否在合理区间

步骤 3：逐笔核对 → GP 的估值数字 vs 基金账本上的数字，每笔必须一致
```

---

### 给小白的一句话

> NAV Tieout 就是给基金的"身价"做体检——基金对外说"我们值 100 亿"，你要逐笔核查这 100 亿是怎么算出来的：每笔投资估价靠谱吗？有没有遗漏的负债？管理费和业绩报酬扣对了吗？因为 NAV 是一切费用和分配的基础——NAV 多算 1%，GP 可能多收几百万的管理费，LP 就少拿几百万的回报。
