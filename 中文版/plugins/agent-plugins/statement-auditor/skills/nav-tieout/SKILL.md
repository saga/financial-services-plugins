# NAV结平

给定生成的LP对账单和期间的NAV包（通过nav MCP），独立重新计算LP资本账户并逐行比较。

> **生成的对账单是测试对象。** NAV包是事实来源。

## 重新计算LP资本账户

```
期初资本（上期对账单期末）
  + 供款（本期计提的资本调用）
  − 分配（现金 + 实物分配）
  + 分配的净收入/（亏损）
       = LP% × （已实现 + 未实现P&L − 管理费 − 基金费用）
  − 附带权益分配（如果本期已结晶）
期末资本
```

通过NAV包提取每个输入：LP承诺%、基金级P&L组件、费用和费用总计、水柱输出。

## 比较

对对账单上的每一行，与你的重新计算值比较。容差：`0.01`。对每个不匹配，注意驱动它的输入（例如，"分配的净收入不同 — 对账单使用12.40%所有权，NAV包显示在Q1转移后12.38%"）。

## 附加检查

- 此对账单上的期末资本 = 下期草稿的期初资本（如果可用）。
- 所有LP期末资本总和 = 基金NAV（在舍入内）。
- 承诺、未拨付和可支配数字与承诺登记册一致。

## 输出

每行通过/失败，重新计算值与对账单值并列，以及标志列表。编辑对账单 — 发布器在审查后根据标志行动。

## Appendix: 金融背景知识

### 什么是NAV结平（NAV Tie-out）？

NAV结平是确保LP对账单上的资本账户余额与基金NAV包中的计算完全一致的过程。这是基金会计中的关键控制，防止错误分发对账单。

### 关键术语解释

**LP（Limited Partner，有限合伙人）**：基金的投资者，承担有限责任，通常不参与基金日常管理。

**资本账户（Capital Account）**：记录每个LP在基金中权益的账户，包括：
- 期初余额
- 资本供款（Capital Calls）
- 分配（Distributions）
- 净收入/亏损分配
- 期末余额

**NAV（Net Asset Value，资产净值）**：基金的总资产减去总负债后的净值，除以份额数得到每份NAV。

**NAV包（NAV Pack）**：基金会计准备的完整NAV计算包，包括：
- 资产估值备忘录
- 基金级损益表
- 水柱计算（Waterfall）
- 所有LP资本账户计算

**附带权益（Carried Interest, "Carry"）**：GP从基金利润中获得的分成，通常20%，仅在LP收回本金和优先回报后支付。

**结晶（Crystallized）**：附带权益在基金退出投资时实际计算和分配的过程。

**已实现P&L（Realized P&L）**：已退出投资的实际损益。

**未实现P&L（Unrealized P&L）**：尚未退出投资的账面损益。

**水柱（Waterfall）**：描述现金流如何在不同利益相关者之间分配的层级结构，通常：
1. 返还LP本金
2. 优先回报（如8%年化）
3. GP追补（Catch-up）
4. 80/20或类似比例的剩余收益分成

**资本调用（Capital Call）**：基金要求LP供款的通知，通常基于即将进行的投资或费用。

**实物分配（In-kind Distribution）**：以证券而非现金形式向LP分配。

**承诺（Commitment）**：LP承诺向基金供款的总额。

**未拨付（Unfunded）**：LP尚未供款的承诺余额。

**可支配（Recallable）**：GP可以召回的已分配金额，通常用于覆盖未来费用或重组。

**nav MCP**：通过Model Context Protocol访问基金NAV系统的接口，用于提取准确的NAV组件。

**容差（Tolerance）**：可接受的小差异范围，通常为0.01货币单位，用于舍入差异。

**事实来源（Source of Truth）**：在结平过程中，NAV包是权威数据源，对账单应与之匹配。

**发布器（Publisher）**：负责根据结平检查结果更新和分发LP对账单的代理或人员。
