# 审计电子表格

审计公式和数据的准确性和错误。范围决定深度——从对选定区域的快速公式检查到完整的财务模型完整性审计。

## 步骤1：确定范围

如果用户已经指定了范围，请使用该范围。否则**询问他们**：

> 您希望我审计什么范围？
> - **selection（选定区域）** — 仅当前选定的范围
> - **sheet（工作表）** — 仅当前活动工作表
> - **model（模型）** — 整个工作簿，包括财务模型完整性检查（资产负债表平衡、现金匹配、滚动计算、逻辑合理性）

**model（模型）**范围是最深的——在发送给客户或内部客户之前，用于DCF、LBO、三表模型、合并、可比公司分析或任何集成财务模型。

---

## 步骤2：公式级检查（所有范围）

无论范围如何，都运行这些检查：

| 检查项 | 查找内容 |
|---|---|
| 公式错误 | `#REF!`、`#VALUE!`、`#N/A`、`#DIV/0!`、`#NAME?` |
| 公式中的硬编码 | `=A1*1.05` — `1.05` 应该是单元格引用 |
| 不一致的公式 | 破坏行/列中相邻公式模式的公式 |
| 偏移范围 | `SUM`/`AVERAGE`遗漏第一行或最后一行 |
| 粘贴覆盖的公式 | 看起来像公式但实际上是硬编码值的单元格 |
| 循环引用 | 有意或无意的 |
| 损坏的跨表链接 | 引用已移动或删除的单元格 |
| 单位/比例不匹配 | 千与百万混合，%存储为整数 |
| 隐藏行/标签页 | 可能包含覆盖或过时的计算 |

## 步骤3：模型完整性检查（仅MODEL范围）

如果范围是**model**，识别模型类型（DCF / LBO / 三表 / 合并 / 可比公司 / 自定义）并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项 | 查找内容 |
|---|---|
| 输入/公式分离 | 输入是否与计算清晰分离？ |
| 颜色约定 | 蓝色=输入，黑色=公式，绿色=链接——或模型使用的任何约定，应用是否一致？ |
| 标签页流程 | 逻辑顺序（假设→损益表→资产负债表→现金流量表→估值）？ |
| 日期标题 | 所有标签页一致？ |
| 单位 | 一致（千 vs 百万 vs 实际值）？ |

### 3b. 资产负债表

| 检查项 | 测试 |
|---|---|
| 资产负债表平衡 | 总资产 = 总负债 + 权益（每期） |
| 留存收益滚动 | 前期留存收益 + 净利润 − 股息 = 当前留存收益 |
| 商誉/无形资产 | 来自收购假设（如果有并购） |

如果资产负债表不平衡，**量化每期的缺口并追踪断裂点**——在修复之前其他都不重要。

### 3c. 现金流量表

| 检查项 | 测试 |
|---|---|
| 现金匹配 | 现金流量表期末现金 = 资产负债表现金（每期） |
| 现金流量总和 | 经营活动现金流 + 投资活动现金流 + 筹资活动现金流 = 现金变动 |
| 折旧摊销匹配 | 现金流量表上的折旧摊销 = 损益表上的折旧摊销 |
| 资本支出匹配 | 现金流量表上的资本支出与资产负债表上的PP&E滚动匹配 |
| 营运资金变化 | 符号与资产负债表变动匹配（应收账款变动、应付账款变动、存货变动） |

### 3d. 损益表

| 检查项 | 测试 |
|---|---|
| 收入构建 | 与细分/产品明细匹配 |
| 税费 | 税费支出 = 税前收入 × 税率（允许递延税调整） |
| 股份数 | 与稀释时间表匹配（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是LBO/三表模型中常见的有意循环
- 如果是有意的：验证迭代切换存在并正常工作
- 如果是无意的：追踪循环并指出如何打破它

### 3f. 逻辑与合理性

| 检查项 | 标记条件 |
|---|---|
| 增长率 | 收入增长>100%无解释 |
| 利润率 | 超出行业标准 |
| 终端价值主导 | 终端价值 > ~75%的DCF企业价值（黄色标记） |
| 曲棍球棒效应 | 预测在后期不切实际地增长 |
| 复合增长 | EBITDA到第10年复合增长到荒谬的金额 |
| 边缘情况 | 模型在0%或负增长、负EBITDA、杠杆为负时崩溃 |

### 3g. 特定模型类型的错误

**DCF：**
- 折现率应用于错误期间（年中 vs 年末）
- 终端价值未折现
- WACC使用账面价值而非市场价值
- FCF包含利息费用（应为无杠杆）
- 税盾重复计算

**LBO：**
- 债务偿还与现金扫尾机制不匹配
- PIK利息未计入本金
- 管理层滚动未反映在回报中
- 退出倍数应用于错误的EBITDA（LTM vs NTM）
- 费用未从第1天股权中扣除

**合并：**
- 摊薄/增厚使用错误的股份数（交易前 vs 交易后）
- 协同效应未分阶段实施
- 购买价格分配不平衡
- 未包含现金的机会成本利息
- 交易费用不在来源与用途中

**三表：**
- 营运资金变动符号错误
- 折旧与PP&E时间表不匹配
- 债务到期时间表与本金偿还不匹配
- 股息超过净利润无解释

## 步骤4：报告

输出发现表：

| # | 工作表 | 单元格/范围 | 严重性 | 类别 | 问题 | 建议修复 |
|---|---|---|---|---|---|---|

**严重性：**
- **Critical（严重）** — 输出错误（资产负债表不平衡、公式损坏、现金不匹配）
- **Warning（警告）** — 风险（硬编码、不一致公式、边缘情况失败）
- **Info（信息）** — 样式/最佳实践（颜色编码、布局、命名）

对于**model**范围，添加摘要行：

> Model type: [DCF/LBO/三表/...] — Overall: [Clean / Minor Issues / Major Issues] — [N] critical, [N] warnings, [N] info

**未经询问不要更改任何内容** — 先报告，根据请求修复。

## 注意事项

- **先检查资产负债表平衡** — 如果不平衡，下游所有内容都值得怀疑
- **硬编码覆盖是沉默错误的首要来源** — 积极搜索
- **符号约定错误**（现金流出的正负号）极为常见
- 如果模型使用VBA宏，注意任何无法仅从公式审计的宏驱动计算

## 金融术语解释

### 1. 财务模型审计（Financial Model Audit）

**定义**：财务模型审计是对财务模型进行系统性检查，以确保其准确性、完整性和逻辑一致性的过程。

**目的**：
- 发现公式错误和计算错误
- 验证财务报表之间的一致性
- 确保模型符合最佳实践
- 提高模型可信度

**范围**：
- 公式级检查：语法错误、引用错误、逻辑错误
- 结构检查：输入输出分离、命名规范、颜色编码
- 完整性检查：资产负债表平衡、现金流量匹配
- 合理性检查：增长率、利润率、估值假设

### 2. 循环引用（Circular Reference）

**定义**：循环引用是指公式直接或间接地引用自身单元格的情况。

**类型**：
- **有意循环**：在LBO和三表模型中，利息计算→债务余额→现金余额→利息的循环是常见的有意设计
- **无意循环**：由于公式错误导致的意外循环

**处理方法**：
- 在Excel中启用迭代计算
- 设置最大迭代次数和收敛阈值
- 定期检查循环是否按预期工作

### 3. LBO模型（杠杆收购模型）

**定义**：LBO（Leveraged Buyout）模型是用于评估杠杆收购交易的财务模型。

**关键组成部分**：
- **资本结构**：股权和债务的组合
- **现金流量预测**：预测的自由现金流
- **债务偿还**：根据现金流量偿还债务
- **退出分析**：计算投资回报率

**关键指标**：
- **内部收益率（IRR）**：衡量投资回报
- **倍数回报（MOIC）**：投资倍数
- **债务覆盖率**：衡量偿债能力

### 4. DCF模型（折现现金流模型）

**定义**：DCF（Discounted Cash Flow）模型是一种估值方法，通过将未来现金流折现到当前价值来评估公司或投资的价值。

**步骤**：
1. 预测未来自由现金流
2. 确定折现率（WACC）
3. 计算终端价值
4. 折现所有现金流
5. 计算净现值

**关键假设**：
- 现金流增长率
- 资本成本
- 终端价值倍数

### 5. 三表模型（3-Statement Model）

**定义**：三表模型是将损益表、资产负债表和现金流量表集成在一起的财务模型。

**集成关系**：
- 损益表的净利润流入资产负债表的留存收益
- 资产负债表的变动驱动现金流量表
- 现金流量表的期末现金回到资产负债表

**用途**：
- 预测公司未来财务表现
- 评估投资决策
- 支持融资活动

### 6. 终端价值（Terminal Value）

**定义**：终端价值是指预测期结束后公司的剩余价值。

**计算方法**：
- **永续增长法**：TV = 最后一年FCF × (1 + g) / (WACC - g)
- **退出倍数法**：TV = 最后一年EBITDA × 行业倍数

**注意事项**：
- 终端价值通常占DCF总价值的很大比例
- 增长率假设应合理（通常接近GDP增长率）

### 7. WACC（加权平均资本成本）

**定义**：WACC是公司融资的平均成本，考虑了股权和债务的成本。

**计算公式**：
```
WACC = (E/V) × Re + (D/V) × Rd × (1 - T)
```
其中：
- E = 股权价值
- D = 债务价值
- V = E + D
- Re = 股权成本
- Rd = 债务成本
- T = 税率

### 8. 自由现金流（Free Cash Flow）

**定义**：自由现金流是公司在支付运营费用和资本支出后可用于分配给投资者的现金。

**计算公式**：
```
FCF = EBIT × (1 - 税率) + 折旧与摊销 - 资本支出 - 营运资金变动
```

**用途**：
- 评估公司价值
- 决定股息和回购
- 评估投资机会

### 9. 协同效应（Synergies）

**定义**：协同效应是指合并后公司的价值超过合并前两家公司价值之和的部分。

**类型**：
- **收入协同效应**：通过交叉销售、定价能力提升增加收入
- **成本协同效应**：通过整合运营、消除重复降低成本

**实现挑战**：
- 协同效应可能需要时间实现
- 整合成本可能很高
- 并非所有预期协同效应都能实现

### 10. 摊薄/增厚分析（Accretion/Dilution Analysis）

**定义**：摊薄/增厚分析用于评估合并交易对每股收益的影响。

**计算方法**：
```
每股收益变化 = (合并后EPS - 收购方原EPS) / 收购方原EPS × 100%
```

**结果解读**：
- **Accretive（增厚）**：合并后EPS > 原EPS
- **Dilutive（摊薄）**：合并后EPS < 原EPS

## Appendix: 金融背景知识

这份文件是"Excel 审计（Audit XLS）"技能的详细说明。在 Statement Auditor 的上下文中，Audit XLS 负责在 LP 对账单核对完成后验证 Excel 输出——确保没有"本该标记为异常但被忽略了"的差异。

---

### 1. LP 对账单审计的"零容忍"原则

| 错误 | 容忍度 |
|------|--------|
| LP 的资本账户余额不对 | 零容忍——必须修正后才能发出 |
| 管理费计算错误 | 零容忍——直接影响 LP 的成本 |
| 分配金额有偏差 | 零容忍——多分/少分都是"果实错误" |
| 格式问题（字体/对齐） | 可容忍——不影响数字准确性 |

---

### 给小白的一句话

> Audit XLS 在 Statement Auditor 中就是"最后的防火墙"——在 LP 对账单发出之前，Audit XLS 必须再次确认：所有被标记为"差异"的条目真的被列在了 Breaks Sheet 上吗？有没有 $500,000 的差异被错误地标记成了"忽略"？因为一旦对账单发给了 LP，任何错误都可能演变成 LP 的法律诉讼——"你让我以为我赚了 8%，实际上是 6%"。
