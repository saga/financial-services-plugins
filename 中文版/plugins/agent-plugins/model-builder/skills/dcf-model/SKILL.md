# DCF模型构建器

## 概述

本技能遵循投资银行标准，创建机构级的股权估值DCF模型。每次分析都会生成一个详细的Excel模型（并在DCF工作表的底部包含敏感性分析）。

## 工具

- 默认使用用户提供的信息以及所有可用的MCP服务器进行数据溯源。

## 核心约束 - 请先阅读

这些约束适用于所有DCF模型的构建过程。在开始前请务必确认：

**环境：Office JS与Python/openpyxl：**
- **如果在Excel内运行（Office加载项 / Office JS环境）：** 直接使用Office JS — 不要使用Python/openpyxl。通过`range.formulas = [["=D19*(1+$B$8)"]]`写入公式。无需单独的重算步骤，Excel会原生执行计算。使用`range.format.*`进行样式设置。同样的"公式优于硬编码"原则依然适用：设置`.formulas`，绝不要为衍生单元格设置`.values`。
- **如果生成独立的.xlsx文件（无活动Excel会话）：** 按下文所述使用Python/openpyxl，然后在交付前运行`recalc.py`。
- 本技能的其余部分使用openpyxl作为示例 — 在对应环境下请转换为Office JS API调用，但所有原则（公式字符串、单元格批注、各部分检查点、敏感性分析表循环）完全一致。

**⚠️ Office JS合并单元格陷阱：** 在使用合并单元格构建各节标题时，不要在调用`.merge()`后对合并范围设置`.values` — Office JS仍会报告范围的原始维度，并抛出`InvalidArgument: The number of rows or columns in the input array doesn't match the size or dimensions of the range`。正确做法是仅将值写入左上角的单元格，然后再合并并格式化整个范围：

```js
// 错误做法 — 会抛出InvalidArgument异常：
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.values = [["市场数据与关键输入"]];  // 1×1数组对比1×8范围 → 失败

// 正确做法 — 先在单个单元格赋值，再合并并格式化范围：
ws.getRange("A7").values = [["市场数据与关键输入"]];
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.format.fill.color = "#1F4E79";
hdr.format.font.bold = true;
hdr.format.font.color = "#FFFFFF";
```

这适用于DCF中的每一个合并标题（市场数据、情景块、现金流预测、终值、估值汇总、敏感性分析表）。

**公式优于硬编码（不可逾越的底线）：**
- 每一个预测值、利润率、折现因子、现值和敏感性单元格都**必须**是活的Excel公式 — 绝不能在Python中计算后以数字形式写入。
- 使用openpyxl时：`ws["D20"] = "=D19*(1+$B$8)"`是正确的；`ws["D20"] = calculated_revenue`是**错误**的。
- 仅允许以下内容为硬编码数值：(1) 原始历史数据输入，(2) 假设驱动因素（增长率、WACC输入项、永续增长率g），(3) 当前市场数据（股价、负债余额）。
- 如果你发现自己在Python中进行计算并写入结果 — 请立即**停止**。当用户修改假设时，模型必须能够联动。

**与用户进行分布确认（不要一次性完成全部构建）：**

- 数据检索后 → 向用户展示原始输入块（收入、利润率、股份数、净债务）并在预测前确认
- 收入预测后 → 向用户展示预测顶线和增长率，在构建利润率之前确认
- FCF构建后 → 向用户展示完整的FCF时间表，在计算WACC前确认
- WACC后 → 向用户展示计算和输入，在折现前确认
- 终值 + 现值后 → 向用户展示股权桥接（EV → 股权价值 → 每股价值），在敏感性分析表前确认
- 在每个阶段捕捉错误 — 在构建敏感性分析表后发现利润率假设错误意味着要重建下游所有内容

**敏感性分析表：**

- **使用奇数行和列**（标准：5×5，有时7×7）— 这保证一个真正的中心单元格
- **中心单元格 = 基准情形。** 构建轴值，使中间行标题和中间列标题恰好等于模型的实际假设（例如，如果基准WACC = 9.0%，中间行是9.0%；如果永续增长率g = 3.0%，中间列是3.0%）。因此，中心单元格的输出必须等于模型的实际隐含股价 — 这是表格构建正确的合理性检查。
- **高亮中心单元格** — 使用中蓝填充（`#BDD7EE`）+ 粗体字体，使其立即可见哪个单元格是基准情形。
- 用完整DCF重新计算公式计算**所有单元格**（通常3个表 × 25个单元格 = 75个）— 使用openpyxl循环（或Office JS循环）以编程方式写入公式
- **无占位符文本、无线性近似、无手动步骤**
- 每个单元格必须重新计算该假设组合的完整DCF

**单元格批注：**

- 在创建每个硬编码值时添加单元格批注
- 格式："来源：[系统/文档]，[日期]，[引用]，[如适用，附URL]"
- 在转到下一部分前，每个蓝色输入都必须有批注
- 不要推迟到结尾或写"TODO：添加来源"

**模型布局规划：**

- 在写入任何公式**之前**定义所有部分行位置
- 首先写入所有标题和标签
- 其次写入所有部分分隔符和空白行
- 然后使用锁定的行位置写入公式
- 创建后立即测试公式

**公式重新计算：**

- 在交付前运行`python recalc.py model.xlsx 30`
- 修复所有错误直到状态为"success"
- 需要零公式错误（#REF!、#DIV/0!、#VALUE!等）

**情景块：**

- 为熊市/基准/牛市情形创建单独的块
- 在每个块内水平显示预测年份的假设
- 使用IF公式：`=IF($B$6=1,[熊市单元格],IF($B$6=2,[基准单元格],[牛市单元格]))`
- 验证公式引用正确的情景块单元格

## DCF流程工作流

### 步骤1：数据检索与验证

从MCP服务器、用户提供的数据和网页中获取数据。

**数据源优先级：**
1. **MCP服务器**（如已配置）— 来自Daloopa等提供商的结构化财务数据
2. **用户提供的数据** — 来自其研究的财务历史
3. **网页搜索/获取** — 当前价格、贝塔、债务和现金（需要时）

**验证清单：**
- 验证净债务 vs. 净现金（对估值至关重要）
- 确认稀释股份数（检查近期的回购/发行）
- 验证历史利润率与商业模式一致
- 交叉检查收入增长率与行业基准
- 验证税率是否合理（通常21-28%）

### 步骤2：历史分析（3-5年）

分析并记录：
- **收入增长趋势**：计算CAGR，识别驱动因素
- **利润率进展**：跟踪毛利率、EBIT利润率、FCF利润率
- **资本密集度**：D&A和资本支出占收入的百分比
- **营运资本效率**：NWC变化占收入增长的百分比
- **回报指标**：ROIC、ROE趋势

创建汇总表显示：
```
历史指标（LTM）：
收入：$X百万
收入增长：X% CAGR
毛利率：X%
EBIT利润率：X%
D&A占收入%：X%
资本支出占收入%：X%
FCF利润率：X%
```

### 步骤3：构建收入预测

**方法论：**
1. 从最新实际收入（LTM或最近财年）开始
2. 为每个预测年份应用增长率
3. 显示美元金额**和**计算的增长%

**增长率框架：**
- 第1-2年：反映近期可见性的较高增长
- 第3-4年：逐渐向行业平均缓和
- 第5年+：接近永续增长率

**公式结构：**
- 收入（第N年）= 收入（第N-1年）×（1 + 增长率）
- 增长率%（第N年）= 收入（第N年）/收入（第N-1年）- 1

**三情景方法：**
```
熊市情形：保守增长（例如8-12%）
基准情形：最可能情境（例如12-16%）
牛市情形：乐观增长（例如16-20%）
```
