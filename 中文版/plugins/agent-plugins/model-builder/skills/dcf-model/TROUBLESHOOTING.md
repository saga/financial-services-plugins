# DCF模型故障排除指南

**何时阅读此文件：** 如果recalc.py显示错误，或估值结果似乎不合理，或案例选择器无法正常工作。

## 模型返回错误值

### #REF! 错误
- 通常由在插入标题后引用错误行的公式引起
- 解决方案：使用正确的行引用重建，或按照布局规划重新开始
- 预防：在编写公式之前定义所有行位置

### #DIV/0! 错误
- 除以零或空单元格
- 解决方案：添加IF语句处理零值：`=IF([除数]=0,0,[被除数]/[除数])`

### #VALUE! 错误
- 计算中使用了错误的数据类型（文本而非数字）
- 解决方案：验证所有输入都格式化为数字

## 估值似乎不合理

### 隐含价格过高
- 检查终值是否不超过企业价值的80%
- 验证终值增长率<WACC
- 审查增长假设是否现实
- 考虑利润率是否过于乐观

### 隐含价格过低
- 验证净债务vs净现金是否正确
- 检查WACC是否过高
- 审查预测是否过于保守
- 考虑终值增长率是否过低

## 案例选择器不工作

### 切换场景时合并列不更新
- 验证案例选择器单元格包含1、2或3
- 检查INDEX/OFFSET公式引用了正确的行范围和选择器单元格
- 确保使用绝对引用（$B$6）作为选择器
- 通过手动更改选择器单元格并验证预测值是否更新来进行测试

## 金融术语和知识详细解释

### 1. DCF模型中的常见错误

**#REF! 错误**：
- 删除或移动了被引用的单元格
- 剪切粘贴而不是复制粘贴公式
- 删除工作表或行时未更新引用

**#DIV/0! 错误**：
- 试图除以零（例如，计算利润率时收入为零）
- 引用了空单元格作为除数

**#VALUE! 错误**：
- 文本值用于数学计算
- 日期被Excel误识别为文本
- 货币符号或逗号未从单元格中移除

---

### 2. WACC与终值增长率的关系

**关键约束**：终值增长率必须小于WACC

**为什么重要**：
- 如果终值增长率≥WACC，公式会产生无穷大或负值
- 这在数学上是无效的

**实务中的典型范围**：
| 场景 | WACC | 终值增长率 |
|------|------|-----------|
| 保守 | 10% | 2.0-2.5% |
| 中性 | 9% | 2.5-3.0% |
| 乐观 | 8% | 3.0-3.5% |

---

### 3. 估值合理性的快速检查

**隐含价格过高的常见原因**：
1. 终值占比>75%企业价值
2. 终值增长率接近或超过WACC
3. 利润率假设过于乐观
4. 折现率设置过低

**隐含价格过低的常见原因**：
1. 净债务计算错误（现金vs债务）
2. WACC过高
3. 预测过于保守
4. 终值增长率过低
5. 股份数使用错误（未稀释）

---

### 4. 案例选择器的工作原理

**INDEX函数方法**：
```
=INDEX(假设范围, 选择器单元格)
```
- 选择器=1：返回熊市假设
- 选择器=2：返回基准假设
- 选择器=3：返回牛市假设

**为什么使用绝对引用**：
- 确保选择器单元格地址在复制公式时不变
- 例如：$B$6永远指向B6单元格
