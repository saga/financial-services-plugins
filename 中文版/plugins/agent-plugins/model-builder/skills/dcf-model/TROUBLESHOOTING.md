# DCF模型故障排除指南

**何时阅读此文件：** 如果recalc.py显示错误，或估值结果似乎不合理，或案例选择器无法正常工作。

## 模型返回错误值

### #REF! 错误
- 通常由在插入标题后引用错误行的公式引起
- 解决方案：使用正确的行引用重建，或按照布局规划重新开始
- 预防：在编写公式之前定义所有行位置

### #DIV/0! 错误
- 除以零或空单元格
- 解决方案：添加IF语句处理零值：`=IF([除数]=0,0,[被除数]/[除数])`

### #VALUE! 错误
- 计算中使用了错误的数据类型（文本而非数字）
- 解决方案：验证所有输入都格式化为数字

## 估值似乎不合理

### 隐含价格过高
- 检查终值是否不超过企业价值的80%
- 验证终值增长率<WACC
- 审查增长假设是否现实
- 考虑利润率是否过于乐观

### 隐含价格过低
- 验证净债务vs净现金是否正确
- 检查WACC是否过高
- 审查预测是否过于保守
- 考虑终值增长率是否过低

## 案例选择器不工作

### 切换场景时合并列不更新
- 验证案例选择器单元格包含1、2或3
- 检查INDEX/OFFSET公式引用了正确的行范围和选择器单元格
- 确保使用绝对引用（$B$6）作为选择器
- 通过手动更改选择器单元格并验证预测值是否更新来进行测试

## 金融术语和知识详细解释

### 1. DCF模型中的常见错误

**#REF! 错误**：
- 删除或移动了被引用的单元格
- 剪切粘贴而不是复制粘贴公式
- 删除工作表或行时未更新引用

**#DIV/0! 错误**：
- 试图除以零（例如，计算利润率时收入为零）
- 引用了空单元格作为除数

**#VALUE! 错误**：
- 文本值用于数学计算
- 日期被Excel误识别为文本
- 货币符号或逗号未从单元格中移除

---

### 2. WACC与终值增长率的关系

**关键约束**：终值增长率必须小于WACC

**为什么重要**：
- 如果终值增长率≥WACC，公式会产生无穷大或负值
- 这在数学上是无效的

**实务中的典型范围**：
| 场景 | WACC | 终值增长率 |
|------|------|-----------|
| 保守 | 10% | 2.0-2.5% |
| 中性 | 9% | 2.5-3.0% |
| 乐观 | 8% | 3.0-3.5% |

---

### 3. 估值合理性的快速检查

**隐含价格过高的常见原因**：
1. 终值占比>75%企业价值
2. 终值增长率接近或超过WACC
3. 利润率假设过于乐观
4. 折现率设置过低

**隐含价格过低的常见原因**：
1. 净债务计算错误（现金vs债务）
2. WACC过高
3. 预测过于保守
4. 终值增长率过低
5. 股份数使用错误（未稀释）

---

### 4. 案例选择器的工作原理

**INDEX函数方法**：
```
=INDEX(假设范围, 选择器单元格)
```
- 选择器=1：返回熊市假设
- 选择器=2：返回基准假设
- 选择器=3：返回牛市假设

**为什么使用绝对引用**：
- 确保选择器单元格地址在复制公式时不变
- 例如：$B$6永远指向B6单元格

## Appendix: 金融背景知识

这份文件是"DCF 模型排障指南（Troubleshooting Guide）"的详细说明。DCF 是最容易出错的估值模型——因为涉及大量公式链接：一个 #REF! 会让整个模型崩塌。

---

### 1. DCF 排障的"急救顺序"

| 优先级 | 错误 | 解决办法 |
|--------|------|---------|
| 1 | #REF! | 修复被引用的单元格/行号 |
| 2 | #DIV/0! | 加 IF 保护分母为 0 的情况 |
| 3 | WACC 不合理 | 检查债务成本、股权成本、税率 |
| 4 | 终值占估值 90%+ | 增长假设太乐观或预测期太短 |
| 5 | 隐含倍数不合理 | P/E 只有 1x 或 200x——模型有问题 |

---

### 2. 终极检查：Sanity Check

算完 DCF 后，不管数字跑没跑出来，都要做 sanity check：

- 隐含 P/E 倍数在 8-30x 之间吗？
- 隐含 EV/EBITDA 在 5-20x 之间吗？
- 如果一家公司 DCF 算出值 $5，同行值 $100——你的模型大概率有 bug

---

### 给小白的一句话

> DCF 排障指南就是投行的"急救手册"——当你发现 DCF 跑出来的数字离谱得可笑（公司值 -$500 万？P/E 只有 0.3x？），不要立刻怀疑自己——按顺序排查：先看有没有 #REF!（公式丢了引用），再看 WACC 算没算错（这是最常见的坑），再检查终值是否占了 90%+（意味着你假设的增长太离谱），最后做 sanity check（隐含倍数跟同行比合理吗？）。
