# 三表财务模型模板完成

完成并填充集成财务模型模板，损益表、资产负债表和现金流量表之间有正确的链接。

## ⚠️ 关键原则 — 填充任何模板前阅读

**环境 — Office JS vs Python：**

- **如果在Excel内运行（Office加载项 / Office JS）：** 直接使用Office JS。通过`range.formulas = [["=D14*(1+Assumptions!$B$5)"]]`写入公式 — 计算单元格绝不使用`range.values`。不需要单独重算；Excel原生计算。使用`context.workbook.worksheets.getItem(...)`导航选项卡。
- **如果生成独立的.xlsx文件：** 使用Python/openpyxl。写入`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`，然后在交付前运行`recalc.py`。
- **Office JS合并单元格陷阱：** 不要在调用`.merge()`后对合并范围设置`.values` — 会抛出`InvalidArgument`，因为范围仍报告其合并前的维度。应仅将值写入左上角单元格，然后合并+格式化整个范围：`ws.getRange("A1").values = [["损益表"]]; const h = ws.getRange("A1:G1"); h.merge(); h.format.fill.color = "#1F4E79";`
- 以下所有原则在两种环境中完全相同。

**公式优于硬编码（不可协商）：**
- 每个预测单元格、滚调表、链接和小计都必须是Excel公式 — 绝不能是预先计算的值
- 使用Python/openpyxl时：写入公式字符串（`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`），**不是**计算结果（`ws["D15"] = 12500`）
- **唯一应包含硬编码数字的单元格是：** (1) 历史实际数，(2) "假设"选项卡中的假设驱动因素
- 如果你发现自己用Python计算值并写入结果 — **停止**。写入公式。
- 原因：当场景切换或假设更改时，模型必须能够灵活调整。硬编码会无声地破坏每个下游完整性检查。

**与用户逐步验证：**
1. **映射模板后** → 向用户展示你识别的选项卡/部分，并在接触任何单元格前确认
2. **填充历史数据后** → 向用户展示历史数据块，并确认值/期间与源数据匹配
3. **构建IS预测后** → 运行小计检查，向用户展示预测的IS，在转到BS前确认
4. **构建BS后** → 向用户展示平衡检查（资产 = 负债+权益），在转到CF前确认
5. **构建CF后** → 向用户展示现金结平（CF期末现金 = BS现金），在最终确定前确认
6. **不要**从头到尾填充整个模型然后完整呈现 — 在每个报表处中断，展示工作，及早发现错误

## 格式 — 专业蓝/灰调色板（除非模板/用户另行指定）

**保持颜色最小化。** 单元格填充仅使用蓝色和灰色。不要引入绿色、黄色、橙色或多个强调色 — 干净的模型使用克制。

| 元素 | 填充 | 字体 |
|------|------|------|
| 章节标题（IS / BS / CF标题） | 深蓝 `#1F4E79` | 白色粗体 |
| 列标题（FY2024A、FY2025E等） | 浅蓝 `#D9E1F2` | 黑色粗体 |
| 输入单元格（历史数、假设驱动因素） | 浅灰 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨选项卡链接 | 白色 | 绿色 `#008000` |
| 检查行 / 关键总计 | 中蓝 `#BDD7EE` | 黑色粗体 |

**总共只有3种蓝色 + 1种灰色 + 白色。** 如果模板有自己的配色方案，请改用模板。

字体颜色表示单元格*是什么*（输入/公式/链接）。填充颜色表示你在*哪里*（标题/数据/检查）。

## 模型结构

### 识别模板选项卡组织

模板的选项卡命名约定和组织各不相同。填充前，查看所有选项卡以了解模板的结构。以下是常见的选项卡名称及其典型内容：

| 常见选项卡名称 | 要查找的内容 |
|--------------|--------------|
| IS、P&L、损益表 | 损益表 |
| BS、资产负债表 | 资产负债表 |
| CF、CFS、现金流量表 | 现金流量表 |
| WC、营运资本 | 营运资本计划表 |
| DA、D&A、折旧、PP&E | 折旧与摊销计划表 |
| 债务、债务计划表 | 债务计划表 |
| NOL、税务、DTA | 净经营亏损计划表 |
| 假设、输入、驱动因素 | 驱动假设和输入 |
| 检查、审计、验证 | 错误检查仪表板 |

**模板审查清单**
- 识别模板中存在哪些选项卡（并非所有模板都包含所有计划表）
- 注意上面未列出的任何模板特定选项卡
- 了解选项卡依赖关系（例如，哪些计划表流入主报表）
- 定位每个选项卡上的输入单元格 vs. 公式单元格

### 理解模板结构

在填充模板之前，熟悉其现有布局，确保数据输入到正确的位置，公式保持完整。

**识别行结构**
- 定位每个选项卡顶部的模型标题
- 识别章节标题及其视觉分隔
- 找到指示$百万、%、x等的单位行
- 注意区分实际数 vs. 预测数期间的列标题
- 确认期间标签（例如，FY2024A、FY2025E）
- 识别输入单元格 vs. 公式单元格（通常通过字体颜色区分）

**识别列结构**
- 确认最左侧列中的行项目标签
- 验证历史年份在预测年份之前
- 注意分隔历史和预测期间的视觉边框
- 检查所有选项卡列顺序一致

**使用命名范围**

模板通常使用命名范围来标识关键输入和输出。输入数据前：
- 查看模板中的现有命名范围（Excel中的公式 → 名称管理器）
- 常见命名范围包括：收入增长率、成本百分比、关键输出（净收入、EBITDA、总债务、现金）、场景选择器单元格
- 确保输入到馈入这些命名范围的单元格中

### 预测期间

- 模板通常从最后一个历史年份向前预测5年
- 验证历史（A）vs. 预测（E）列清楚分隔
- 确认列使用财年符号（例如，FY2024A、FY2025E）

## 利润率分析

**注意：仅当用户提示或模板明确要求时才执行以下利润率分析。如果没有提示，跳过此部分。**

在损益表（IS）选项卡上计算并显示盈利能力利润率，以跟踪运营效率并启用同行比较。

### 要包含的核心利润率

| 利润率 | 公式 | 衡量内容 |
|--------|---------|------------------|
| 毛利率 | 毛利润 / 收入 | 定价能力、生产效率 |
| EBITDA利润率 | EBITDA / 收入 | 核心运营盈利能力 |
| EBIT利润率 | EBIT / 收入 | D&A后的运营盈利能力 |
| 净收入利润率 | 净收入 / 收入 | 底线盈利能力 |

### 带利润率的损益表布局

在每个利润项目下方直接显示利润率百分比：
- 毛利润率% 在毛利润下方
- EBIT利润率% 在EBIT下方
- EBITDA利润率% 在EBITDA下方
- 净收入利润率% 在净收入下方

## 信贷指标

**注意：仅当用户提示或模板明确要求时才执行以下信贷分析。如果没有提示，跳过此部分。**

在资产负债表（BS）选项卡上计算并显示信贷/杠杆指标，以评估财务健康、债务能力和契约合规性。

### 要包含的核心信贷指标

| 指标 | 公式 | 衡量内容 |
|------|---------|------------------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | （总债务 - 现金）/ LTM EBITDA | 净现金的杠杆 |
| 利息覆盖率 | EBITDA / 利息支出 | 偿债能力 |
| 债务 / 总资本 | 总债务 / （总债务 + 权益） | 资本结构 |
| 债务 / 权益 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | （流动资产 - 存货）/ 流动负债 | 即时流动性 |

### 信贷指标层级检查

验证上行显示最强的信贷状况：
- 杠杆：上行 < 基准 < 下行（越低越好）
- 覆盖率：上行 > 基准 > 下行（越高越好）
- 流动性：上行 > 基准 > 下行（越高越好）

**本文件专属**：
- **实际数（Actuals）**：财报中报告的历史实际数字，标记为"A"（如 FY2024A）。
- **预测（Estimates/E）**：对未来期间的预测数字，标记为"E"（如 FY2025E）。
- **场景选择器（Scenario Selector）**：允许用户在不同假设集（熊市/基准/牛市）之间切换的单元格。
- **重算（Recalc）**：强制 Excel 重新计算所有公式，确保模型正确更新。
- **签核（Sign-off）**：财务主管审查并正式批准模型的流程。
