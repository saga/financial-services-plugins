# SEC filings数据提取参考

**何时使用：** 仅当模型模板特别要求从SEC filings（10-K、10-Q）提取数据时才参考此文件。对于直接提供数据或使用其他数据源的模板，不需要此参考。

---

## 从SEC filings提取数据（10-K / 10-Q）

当用上市公司数据填充模型模板时，直接从SEC filings提取财务数据。

### 第1步：找到Filing

1. 使用SEC EDGAR：`https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany&CIK=[TICKER]&type=10-K`
2. 对于季度数据，使用`type=10-Q`

### 第2步：识别报告货币

在提取数据之前，识别报告货币：
- 查看封面或标题中的报告货币
- 查看报表标题（例如，"以千美元为单位"）
- 查看附注1（重要会计政策摘要）

**常见货币指标**

| 指标 | 货币 |
|-----------|----------|
| $，USD | 美元 |
| €，EUR | 欧元 |
| £，GBP | 英镑 |
| ¥，JPY | 日元 |
| ¥，CNY，RMB | 人民币 |
| CHF | 瑞士法郎 |
| CAD，C$ | 加拿大元 |

将模型货币设置为与filing匹配；在"假设"标签中记录。

### 第3步：导航到财务报表

在10-K或10-Q中，找到：
- **第8项**（10-K）或**第1项**（10-Q）：财务报表
- 要提取的关键部分：
  - 合并经营报表（利润表）
  - 合并资产负债表
  - 合并现金流量表
  - 财务报表附注（用于明细表详情）

### 第4步：数据提取映射

**利润表（来自合并经营报表）**

| Filing行项目 | 模型行项目 |
|------------------|-----------------|
| 净收入/净销售额 | 收入 |
| 销货成本 | COGS |
| 销售、一般和管理费用 | SG&A |
| 折旧和摊销 | D&A |
| 利息费用，净额 | 利息费用 |
| 所得税费用 | 税款 |
| 净利润 | 净利润 |

**资产负债表（来自合并资产负债表）**

| Filing行项目 | 模型行项目 |
|------------------|-----------------|
| 现金及现金等价物 | 现金 |
| 应收账款，净额 | AR |
| 存货 | 存货 |
| 物业、厂房和设备，净额 | PP&E（净额）|
| 总资产 | 总资产 |
| 应付账款 | AP |
| 短期债务/流动部分长期债务 | 流动债务 |
| 长期债务 | LT债务 |
| 留存收益 | 留存收益 |
| 股东权益合计 | 权益合计 |

**现金流量表（来自合并现金流量表）**

| Filing行项目 | 模型行项目 |
|------------------|-----------------|
| 净利润 | 净利润 |
| 折旧和摊销 | D&A |
| 应收账款变化 | ΔAR |
| 存货变化 | Δ存货 |
| 应付账款变化 | ΔAP |
| 资本支出 | CapEx |
| 发行普通股所得款项 | 权益发行 |
| 债务发行/偿还所得款项 | 债务活动 |
| 支付的股息 | 股息 |

### 第5步：从附注提取支持性明细

对于明细表，从财务报表附注中提取：
- **附注：债务** → 到期时间表、利率、契约
- **附注：物业、厂房和设备** → 总PP&E、累计折旧、可用年限
- **附注：收入** → 分部明细、地区分布
- **附注：租赁** → 经营与融资租赁义务

### 第6步：历史数据要求

最少提取3年历史数据：
- 当前财政年度（FY0）实际数据
- 上一财政年度（FY-1）实际数据
- 上上财政年度（FY-2）实际数据

对于某些模型，还需要FY-3和FY-4，特别是：
- 计算历史CAGR
- 趋势分析
- 季节性调整

### 第7步：数据验证检查

提取后进行验证：
- **规模检查**：数字合理吗？（例如，总资产在$1B-$100B范围内）
- **增长率检查**：历史增长率是否符合业务性质？
- **利润率检查**：利润率是否与行业一致？
- **平衡检查**：资产负债表是否平衡？

### 第8步：文档和来源注释

对于每个提取的数据点，添加：
- 来源：SEC EDGAR，文件名，日期
- URL：具体filing的链接
- 期间：数据所代表的财政期间

## 金融术语和知识详细解释

### 1. SEC与SEC EDGAR

**SEC（证券交易委员会）**：
- 美国联邦政府机构
- 负责监管证券市场
- 要求上市公司定期披露财务信息

**SEC EDGAR**：
- 电子数据收集、分析和检索系统
- 公众可免费访问
- 包含所有上市公司的 filings

**为什么重要**：
- 上市公司必须向SEC提交财务报告
- 投资者可免费获取公司财务数据
- 财务建模的主要数据来源

---

### 2. 10-K与10-Q的区别

**10-K（年度报告）**：
- 提供公司全年业务的全面概述
- 包括经审计的财务报表
- 包含详细的业务描述、风险因素、管理层讨论
- 每年提交一次

**10-Q（季度报告）**：
- 提供季度财务信息
- 通常未经审计
- 包含简化的财务报表
- 每个财政年度提交三次

**为什么重要**：
- 10-K提供更全面的信息
- 10-Q提供更及时的数据更新
- 两者结合使用可以获得完整的财务图景

---

### 3. 财务报表附注（Notes to Financial Statements）

**定义**：财务报表附注提供超出主要报表的详细信息。

**常见附注类型**：

| 附注 | 内容 |
|------|------|
| 附注1 | 重要会计政策 |
| 附注2 | 应收账款和存货明细 |
| 附注3 | PP&E和折旧明细 |
| 附注4 | 债务明细（利率、到期日、契约）|
| 附注5 | 租赁义务 |
| 附注6 | 承诺和或有事项 |
| 附注7 | 股东权益 |
| 附注8 | 收入细分 |

**为什么重要**：
- 提供报表数字的详细分解
- 揭示重要的会计判断
- 包含管理层讨论的关键信息
- 对估值和信用分析至关重要

---

### 4. XBRL（可扩展商业报告语言）

**定义**：XBRL是一种基于XML的标准化语言，用于商业财务报告。

**为什么重要**：
- 使财务数据可机器读取
- 便于数据分析
- 提高财务信息透明度
- SEC要求使用XBRL提交

**使用技巧**：
- 使用XBRL标签提取特定数据
- 注意不同公司可能使用不同的标签名称
- 验证XBRL数据的准确性

---

### 5. 报告货币与功能货币

**报告货币**：
- 公司在财务报表中报告所使用的货币
- 通常是公司注册地或主要运营地的货币

**功能货币**：
- 公司主要经济环境中使用的货币
- 影响外币交易折算

**为什么重要**：
- 确保模型货币与报告货币一致
- 对于跨国公司的建模至关重要
- 影响外汇风险分析

---

### 6. 历史数据提取的最佳实践

**最少需要的数据**：
- 3-5年历史数据
- 当前季度数据（如果可用）
- 同行公司数据用于比较

**数据验证清单**：
- [ ] 数据与源文件匹配
- [ ] 数字格式正确（千vs百万）
- [ ] 时期对齐
- [ ] 比率和百分比计算正确
- [ ] 资产负债表平衡

---

### 7. 财务数据的分类

**实际数据（Actual/A）**：
- 历史期间的实际财务结果
- 最可靠的基准

**估计数据（Estimate/E）**：
- 对未来期间的预测
- 基于假设和判断

**LTM（过去十二个月）**：
- 最近连续十二个月的汇总
- 用于趋势分析

---

### 8. 附注中的关键信息

**债务附注揭示**：
- 债务到期时间表
- 利率安排（固定vs浮动）
- 财务契约
- 或有债务

**收入附注揭示**：
- 收入来源细分（产品/地区/客户）
- 收入确认政策
- 季节性模式

**租赁附注揭示**：
- 经营租赁义务
- 融资租赁义务
- 折现率假设

## Appendix: 金融背景知识

这份文件是"SEC 申报文件参考（SEC Filings Reference）"文档的详细说明。在构建财务模型时，所有历史数据必须来自官方的 SEC 申报文件——不能凭想象、不能用二手网站的数据。

---

### 1. 构建模型必须拉取的"五大 SEC 文件"

| 文件 | 频率 | 关键内容 |
|------|------|---------|
| 10-K | 年报 | 完整财务数据 + 管理层讨论（MD&A） |
| 10-Q | 季报 | 简化版 10-K |
| 8-K | 事件触发 | 并购、破产、管理层变动 |
| S-1 | IPO 时 | 招股说明书（最详细的公司介绍） |
| Proxy | 年报 | 高管薪酬、董事会信息 |

---

### 2. 为什么必须用 SEC 文件？

**原因**：SEC 文件是法律文件——公司高管对其真实性负法律责任。如果你用错了数据（比如从 Yahoo Finance 摘的数字跟 10-K 不一致），你的模型就建立在错误基础上。

---

### 给小白的一句话

> SEC Filings Reference 就是模型数据的"唯一正版来源"——你构建模型的所有历史数据必须来自官方 SEC 申报文件（10-K/10-Q/8-K），不能用二手网站的数据。因为 SEC 文件是法律文件——CEO 和 CFO 在上面签字承担刑事责任。Yahoo Finance 或 Google Finance 可能把"调整后 EPS"当成了"GAAP EPS"——你用错了数据，模型再精妙也是一堆废纸。
