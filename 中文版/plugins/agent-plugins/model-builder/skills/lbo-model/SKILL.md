## 模板要求

**此技能使用LBO模型模板。始终首先检查是否附加了模板文件。**

在开始任何LBO模型之前：
1. **如果附加了模板文件**：完全使用其结构 - 复制并用用户数据填充
2. **如果未附加模板**：询问用户：*"你有想让我使用的特定LBO模板吗？如果没有，我可以使用标准模板，包含资金来源与用途、运营模型、债务计划表和回报分析。"*
3. **如果使用标准模板**：复制`examples/LBO_Model.xlsx`作为起点，用用户的假设填充

**重要**：当附加了`LBO_Model.xlsx`之类的文件时，你必须将其用作模板 - 不要从头构建。即使模板看起来复杂或功能超出需要，也要复制并根据用户要求调整它。绝不在提供模板时"从头构建"。

## 给CLAUDE的关键指令 - 首先阅读

### 环境：Office JS vs Python

**如果在Excel内运行（Office加载项 / Office JS环境）：**
- 直接使用Office JS（`Excel.run(async (context) => {...})`）— 不要使用Python/openpyxl
- 通过`range.formulas = [["=B5*B6"]]`写入公式 — Office JS公式在实时工作簿中原生重新计算
- 同样的"公式优于硬编码"原则适用：为任何应为计算的内容设置`range.formulas`，绝不为其设置`range.values`
- 使用`range.format.font.color` / `range.format.fill.color`应用蓝/黑/紫/绿约定
- 不需要单独的重算步骤 — Excel原生处理计算
- **合并单元格陷阱：** 不要在调用`.merge()`后对合并范围设置`.values`（会抛出`InvalidArgument` — 范围仍报告原始维度）。正确做法：`ws.getRange("A7").values = [["资金来源与用途"]]`，然后合并+格式化范围（`ws.getRange("A7:F7").merge(); ws.getRange("A7:F7").format.fill.color = "#1F4E79";`）

**如果生成独立的.xlsx文件（无实时Excel会话）：**
- 按下文使用Python/openpyxl
- 写入公式字符串（`ws["D20"] = "=B5*B6"`），然后在交付前运行`recalc.py`

本技能其余部分使用openpyxl示例编写，但同样的原则适用于Office JS — 只需转换API调用。

### 核心原则

* **每个计算都必须是Excel公式** — 绝不要在Python中计算值并将结果硬编码到单元格中。使用openpyxl时，写入`cell.value = "=B5*B6"`（公式字符串），**不是**`cell.value = 1250`（计算结果）。模型必须是动态的，当输入更改时更新。

* **使用模板结构** — 遵循`examples/LBO_Model.xlsx`或用户提供的模板中的组织。不要发明自己的布局。

* **使用正确的单元格引用** — 所有公式都应引用适当的单元格。绝不要输入本应来自其他单元格的数字。

* **保持符号约定一致性** — 遵循模板使用的任何符号约定（有些对流出使用负数，有些使用正数）。全程保持一致。

* **逐节工作，每一步都与用户验证** — 完整完成一个部分，向用户展示构建的内容，运行该部分的验证检查，并在继续下一部分之前获得确认。绝不要从头到尾构建整个模型然后呈现它 — 后面的部分依赖于前面的部分，因此在回报已经构建后发现资金来源与用途中的错误意味着要重做所有地方。

### 公式颜色约定

* **蓝色（0000FF）**：硬编码输入 — 不引用其他单元格的键入数字
* **黑色（000000）**：带计算的公式 — 任何使用运算符或函数的公式（`=B4*B5`、`=SUM()`、`=-MAX(0,B4)`）
* **紫色（800080）**：链接到**同一选项卡**上的单元格 — 无计算的直接引用（`=B9`、`=B45`）
* **绿色（008000）**：链接到**不同选项卡**上的单元格 — 跨表引用（`=Assumptions!B5`、`='Operating Model'!C10`）

### 填充调色板 — 专业蓝/灰色（除非用户/模板另行指定，否则为默认）

* **保持最小化** — 单元格填充仅使用蓝色和灰色。不要引入绿色、黄色、红色或多个强调色。专业的LBO模型使用克制。

* **默认填充调色板：**
  * **章节标题**（资金来源与用途、运营模型等）：深蓝`#1F4E79`配白色粗体文本
  * **列标题**（第1年、第2年等）：浅蓝`#D9E1F2`配黑色粗体文本
  * **输入单元格**：浅灰`#F2F2F2`（或白色）— 蓝色*字体*是信号，填充是次要的
  * **公式/计算单元格**：白色：无填充
  * **关键输出**（IRR、MOIC、退出股权）：中蓝`#BDD7EE`配黑色粗体文本

* **这就是整个调色板。** 3种蓝色 + 1种灰色 + 白色。如果模板使用自己的颜色，请改用模板。

* 注意：上面的蓝色/黑色/紫色/绿色**字体**颜色用于区分输入vs公式vs链接。这些与此处的**填充**调色板是分开的 — 两者协同工作。

### 数字格式标准

* **货币**：`$#,##0;($#,##0)`，或根据模板使用`$#,##0.0`
* **百分比**：`0.0%`（1位小数）
* **倍数**：`0.0"x"`（1位小数）
* **MOIC/详细比率**：`0.00"x"`（2位小数以保持精度）
* **所有数字单元格**：右对齐

## 模板分析阶段 - 首先执行此操作

在填充任何公式之前，彻底检查模板：

1. **映射结构** — 识别每个部分的位置以及它们如何相互关联。注意哪些部分流入其他部分。
2. **了解时间线** — 哪些列代表哪些期间？是否有"交割"或"备考"列？预测期从哪里开始？
3. **识别输入vs公式单元格** — 模板通常使用颜色编码、边框或阴影来指示哪些单元格需要输入vs公式。尊重这些约定。
4. **仔细阅读现有标签** — 行标签准确告诉你预期的计算。不要假设 - 阅读模板要求的内容。
5. **检查现有公式** — 有些模板部分填充。除非明确要求，否则不要覆盖工作公式。
6. **注意模板特定约定** — 符号约定、小计结构、部分组织方式、是否有用于不同组件的单独选项卡等。

## 填充公式 - 通用方法

对于需要公式的每个单元格，遵循此层次结构：

### 步骤1：检查模板
* 单元格是否已有公式？如果是，验证其正确性并继续。
* 是否有指示预期计算的注释或备注？
* 行/列标签是否使计算显而易见？
* 相邻单元格是否显示你应该遵循的模式？

### 步骤2：检查用户指令
* 用户是否指定了特定的计算方法？
* 是否有影响此公式的所述假设？
* 是否提到了特殊要求？

* 如果模板和用户都没有指定，请使用标准LBO建模约定
* 记录你做出的任何假设
* 如果真的不确定，询问用户

## 常见问题区域

以下计算模式经常在整个LBO模型中引发问题。遇到这些时请特别注意：

### 平衡部分
* 当两个部分必须相等时（例如，资金来源 = 用途），通常有一个项目是"插入项"（平衡数字）
* 识别哪个项目是插入项并将其计算为差额

### 税务计算
* 税务公式应仅引用相关的收入行和税率
* 不应引用不相关的部分（例如，债务计划表）
* 考虑亏损是否产生税盾或只是被忽略

### 利息和循环引用
* 如果利息引用受现金流影响的余额，利息计算可能产生循环
* 使用**期初余额**（不是平均值或期末）来打破循环引用
* 模式：利息 → 现金流量 → 偿还 → 期末余额（如果利息使用期末余额，这会循环回去）

### 债务偿还 / 现金清扫
* 当存在多个债务部分时，通常有优先顺序
* 现金清扫应尊重优先水柱
* 余额不能为负 - 适当地使用MAX或MIN函数

### 回报计算（IRR/MOIC）
* 现金流必须具有正确的符号：投资 = 负，收益 = 正
* 如果使用XIRR，需要相应的日期
* 如果使用IRR，现金流应在连续的期间内
* MOIC = 总收益 / 总投资

## Appendix: 金融背景知识

这份文件是"LBO 模型（LBO Model）"技能的详细说明。在 PE 基金中，LBO 是 PE 最核心的收购方式——用借来的钱买公司，然后用公司的现金流还债，最后卖掉赚差价。

---

### 1. LBO 的"五步成诗"

```
步骤 1：资金来源与用途 → 总共需要多少钱？其中多少是借的？多少是 PE 自己的？

步骤 2：债务时间表 → 借来的钱怎么还？（强制性还款 vs 可选还款）

步骤 3：预测三表 → 收购后公司的盈利会怎样变化？

步骤 4：回报计算 → 5 年后卖掉，PE 能赚多少？（IRR/MOIC）

步骤 5：敏感性分析 → 如果买贵了/卖便宜了/增长慢了，回报怎么变？
```

---

### 2. LBO 的"杠杆魔法"

| 杠杆（债务/EBITDA） | 投入 100 | 退出 | MOIC |
|---------------------|---------|------|------|
| 0x（全现金） | 100 | 150 | 1.5x |
| 4x（60%借的） | 40 | 150（还 60 债 = 90 到手） | 2.25x |
| 6x（75%借的） | 25 | 150（还 75 债 = 75 到手） | 3.0x |

**核心洞察**：借钱越多，PE 自己投入越少，回报倍数越高——但风险也在上升。

---

### 给小白的一句话

> LBO 模型就是 PE 用"借鸡生蛋"的数学原理——借钱买公司、公司赚钱还债、5 年后卖掉找"接盘侠"。杠杆是双刃剑：借 60%，投入 40 收回 90，MOIC 从 1.5x 暴增到 2.25x——但如果在这 5 年里公司现金流出问题，债还不上了，就破产了。PE 不是靠"买的公司好"赚钱，而是靠"用借来的钱放大好公司的回报"赚钱。
