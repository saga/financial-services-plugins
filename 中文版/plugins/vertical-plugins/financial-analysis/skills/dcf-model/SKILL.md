# DCF 模型构建器

## 概览

本技能遵循投资银行标准，创建机构级的权益估值 DCF 模型。每次分析都会生成一个详细的 Excel 模型（并在 DCF 工作表的底部包含敏感性分析）。

## 工具

- 默认使用用户提供的信息以及所有可用的 MCP 服务器进行数据溯源。

## 核心约束 - 请先阅读

这些约束适用于所有 DCF 模型的构建过程。在开始前请务必确认：

**环境：Office JS 与 Python/openpyxl：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS 环境）：** 直接使用 Office JS —— 不要使用 Python/openpyxl。通过 `range.formulas = [["=D19*(1+$B$8)"]]` 写入公式。无需单独的重算步骤，Excel 会原生执行计算。使用 `range.format.*` 进行样式设置。同样的“公式优于硬编码”原则依然适用：设置 `.formulas`，绝不要为衍生单元格设置 `.values`。
- **如果生成独立的 .xlsx 文件（无活动 Excel 会话）：** 按下文所述使用 Python/openpyxl，然后在交付前运行 `recalc.py`。
- 本技能的其余部分使用 openpyxl 作为示例 —— 在对应环境下请转换为 Office JS API 调用，但所有原则（公式字符串、单元格批注、各部分检查点、敏感性分析表循环）完全一致。

**⚠️ Office JS 合并单元格陷阱：** 在使用合并单元格构建各节标题时，不要在调用 `.merge()` 后对合并范围设置 `.values` —— Office JS 仍会报告范围的原始维度，并抛出 `InvalidArgument: The number of rows or columns in the input array doesn't match the size or dimensions of the range`。正确做法是仅将值写入左上角的单元格，然后再合并并格式化整个范围：

```js
// 错误做法 —— 会抛出 InvalidArgument 异常：
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.values = [["市场数据与关键输入"]];  // 1×1 数组对比 1×8 范围 → 失败

// 正确做法 —— 先在单个单元格赋值，再合并并格式化范围：
ws.getRange("A7").values = [["市场数据与关键输入"]];
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.format.fill.color = "#1F4E79";
hdr.format.font.bold = true;
hdr.format.font.color = "#FFFFFF";
```

这适用于 DCF 中的每一个合并标题（市场数据、情景块、现金流预测、终值、估值汇总、敏感性分析表）。

**公式优于硬编码（不可逾越的底线）：**
- 每一个预测值、利润率、折现因子、现值和敏感性单元格都**必须**是活的 Excel 公式 —— 绝不能在 Python 中计算后以数字形式写入。
- 使用 openpyxl 时：`ws["D20"] = "=D19*(1+$B$8)"` 是正确的；`ws["D20"] = calculated_revenue` 是**错误**的。
- 仅允许以下内容为硬编码数值：(1) 原始历史数据输入，(2) 假设驱动因素（增长率、WACC 输入项、永续增长率 g），(3) 当前市场数据（股价、负债余额）。
- 如果你发现自己在 Python 中进行计算并写入结果 —— 请立即**停止**。当用户修改假设时，模型必须能够联动。

**与用户进行分布确认（不要一次性完成全部构建）：**
- 数据检索后 → 向用户展示原始输入块（营收、利润率、股份数、净负债），确认后再进行预测。
- 营收预测后 → 展示预测的顶线数据和增长率，确认后再构建利润率结构。
- FCF (自由现金流) 构建后 → 展示完整的 FCF 计划表，确认逻辑后再计算 WACC。
- WACC 之后 → 展示计算过程和输入项，确认后再进行折现。
- 终值 + 现值之后 → 展示权益价值桥接（企业价值 EV → 权益价值 → 每股价值），确认后再构建敏感性分析表。
- 在每个阶段核查错误 —— 如果在敏感性表构建后才发现利润率假设错误，意味着之后的所有内容都要重做。

**敏感性分析表：**
- **使用奇数的行数和列数**（标准为 5×5，有时为 7×7）—— 这能确保存在唯一的中心单元格。
- **中心单元格 = 基准情景。** 构建坐标轴数值，使中间行标题和中间列标题精确等于模型实际的假设值（例如，如果基准 WACC = 9.0%，则中间行为 9.0%；如果永续增长率 g = 3.0%，则中间列为 3.0%）。中心单元格的输出必须等于模型实际隐含的每股股价 —— 这是核对表格构建是否正确的一致性检查。
- **高亮中心单元格**，使用中蓝色填充 (`#BDD7EE`) + 粗体字体，以便一眼看出基准情景。
- 为所有单元格（通常为 3 张表 × 25 个单元格 = 75 个）填充完整的 DCF 重算公式。
- 使用 openpyxl 循环（或 Office JS 循环）以编程方式写入公式。
- 严禁占位文字，严禁线性近似，严禁需要手动操作的步骤。
- 每个单元格必须针对该假设组合重新计算完整的 DCF。

**单元格批注：**
- 在创建每个硬编码数值时同步添加单元格批注。
- 格式：“来源：[系统/文档], [日期], [参考章节], [URL（如有）]”。
- 在进入下一节之前，每个蓝色输入项都必须有批注。
- 不要推迟到最后，也不要写“TODO: 添加来源”。

**模型布局规划：**
- 在写入任何公式之前，先确定所有章节的行位置。
- 先写入所有章节标题和标签。
- 其次写入所有章节分隔符和空行。
- 然后使用已锁定位置的行写入公式。
- 公式创建后立即进行测试。

**公式重算：**
- 交付前运行 `python recalc.py model.xlsx 30`。
- 修正所有错误，直到状态显示为“success”。
- 要求公式错误为零（如 #REF!、#DIV/0!、#VALUE! 等）。

**情景模块：**
- 为悲观 (Bear)、基准 (Base)、乐观 (Bull) 情景创建独立的模块。
- 在每个模块内水平展示各预测年份的假设。
- 使用 IF 公式：`=IF($B$6=1,[悲观单元格],IF($B$6=2,[基准单元格],[乐观单元格]))`。
- 验证公式引用的情景模块单元格是否正确。

## DCF 操作流程

### 第 1 步：数据检索与验证

从 MCP 服务器、用户提供的数据和网页中获取数据。

**数据源优先级：**
1. **MCP 服务器**（如果已配置）—— 来自 Daloopa 等提供商的结构化财务数据。
2. **用户提供的数据** —— 来自其研究报告的历史财务数据。
3. **网页搜索/抓取** —— 当前股价、Beta 系数、以及需要的负债和现金。

**验证清单：**
- 核实净负债 vs 净现金（对估值至关重要）。
- 确认摊薄后发行在外股数（检查近期的回购/增发）。
- 验证历史利润率是否与商业模式一致。
- 将营收增长率与行业基准进行交叉对比。
- 验证税率是否合理（通常在 21-28% 之间）。

### 第 2 步：历史数据分析（3-5 年）

分析并记录：
- **营收增长趋势**：计算复合年均增长率 (CAGR)，识别驱动因素。
- **利润率演变**：跟踪毛利率、EBIT 利润率、FCF 利润率。
- **资本密集度**：折旧摊销 (D&A) 和资本支出 (CapEx) 占营收的百分比。
- **营运资金效率**：净营运资金 (NWC) 变动占营收增长的百分比。
- **回报指标**：ROIC、ROE 趋势。

创建汇总表展示：
```
历史指标 (LTM):
营业收入: $X million
营收增长: X% CAGR
毛利率: X%
EBIT 利润率: X%
D&A 占营收比: X%
CapEx 占营收比: X%
FCF 利润率: X%
```

### 第 3 步：构建营收预测

**方法论：**
1. 从最新的实际营收开始（LTM 或最近一个财年）。
2. 应用各预测年份的增长率。
3. 同时展示金额数值和计算出的增长率 %。

**增长率框架：**
- 第 1-2 年：较高的增长率，反映近期的能见度。
- 第 3-4 年：逐渐向行业平均水平放缓。
- 第 5 年以上：接近永续增长率。

**公式结构：**
- 营收(第 N 年) = 营收(第 N-1 年) × (1 + 增长率)
- 增长率 %(第 N 年) = 营收(第 N 年) / 营收(第 N-1 年) - 1

**三极情景法：**
```
悲观情景: 保守增长 (例如 8-12%)
基准情景: 最可能发生的情景 (例如 12-16%)
乐观情景: 乐观增长 (例如 16-20%)
```

### 第 4 步：经营费用建模

**固定/变动成本分析：**

经营费用建模应反映真实的经营杠杆：
- **销售与市场费用 (S&M)**：通常占营收的 15-40%，取决于商业模式。
- **研发费用 (R&D)**：科技公司通常占 10-30%。
- **一般与行政费用 (G&A)**：通常占营收的 8-15%，随着公司规模扩大呈现杠杆效应。

**核心原则：**
- 所有百分比均基于**营业收入**，而非毛利。
- 建模经营杠杆：占比应随营收规模扩大而下降。
- 保持 S&M、R&D、G&A 独立分项。
- 计算 EBIT = 毛利 - 总经营费用 (OpEx)。

**利润率扩张框架：**
```
现状 → 目标状态 (第 5 年)
毛利率: X% → Y% (基于规模、效率给出理由)
EBIT 利润率: X% → Y% (由营收增长 + 费用杠杆共同实现)
```

### 第 5 步：自由现金流计算

**按照正确顺序构建 FCF：**

```
EBIT
(-) 税收 (EBIT × 税率)
= NOPAT (税后净经营利润)
(+) 折旧额与摊销额 (D&A, 非现金费用, 占营收 %)
(-) 资本支出 (CapEx, 占营收 %, 通常为 4-8%)
(-) 净营运资金变动 (Δ NWC)
= 无杠杆自由现金流 (Unlevered Free Cash Flow)
```

**营运资金建模：**
- 计算为营收变动（Δ Revenue）的百分比。
- 典型范围：营收变动的 -2% 到 +2%。
- 负数 = 现金来源（营运资金释放）。
- 正数 = 现金占用（营运资金增加）。

**维持性 vs 增长性资本支出：**
- 维持性 CapEx：维持当前运营（约占营收的 2-3%）。
- 增长性 CapEx：支持扩张（额外占用营收的 2-5%）。
- 总 CapEx 应与公司的增长战略保持一致。

### 第 6 步：资本成本 (WACC) 研究

**使用 CAPM 模型计算权益成本：**

```
权益成本 = 无风险利率 + Beta × 股权风险溢价

其中:
- 无风险利率 = 当前 10 年期国债收益率
- Beta = 5 年期月度股票 Beta (对比市场指数)
- 股权风险溢价 = 5.0-6.0% (市场标准)
```

**债务成本计算：**

```
税后债务成本 = 税前债务成本 × (1 - 税率)

通过以下方式确定税前债务成本:
- 信用评级 (如有)
- 公司债券的当前收益率
- 财务报表中的 利息支出 / 总债务
```

**资本结构权重：**

```
权益市值 = 当前股价 × 发行在外股数
净负债 = 总债务 - 现金及等价物
企业价值 (EV) = 市值 + 净负债

权益权重 = 市值 / 企业价值
债务权重 = 净负债 / 企业价值

WACC = (权益成本 × 权益权重) + (税后债务成本 × 债务权重)
```

**特殊情况：**
- **净现金头寸**：如果 现金 > 债务，则净负债为负。
  - 债务权重可能为负。
  - WACC 计算据此调整。
- **无债务**：WACC = 权益成本。

**典型的 WACC 范围：**
- 大盘蓝筹、稳定型：7-9%
- 成长型公司：9-12%
- 高增长/高风险：12-15%

### 第 7 步：折现率应用（5-10 年预测）

**期中惯例 (Mid-Year Convention)：**
- 假设现金流发生在年中。
- 折现期：0.5, 1.5, 2.5, 3.5, 4.5 等。
- 折现因子 = 1 / (1 + WACC)^折现期

**现值 (PV) 计算：**
```
对于每个预测年份:
FCF 现值 = 无杠杆自由现金流 × 折现因子

示例 (第 1 年):
FCF = $1,000
WACC = 10%
折现期 = 0.5
折现因子 = 1 / (1.10)^0.5 = 0.9535
PV = $1,000 × 0.9535 = $954
```

**预测期选择：**
- **5 年**：大多数分析的标准。
- **7-10 年**：具有较长增长跑道的高增长公司。
- **3 年**：成熟、极其稳定的业务。

### 第 8 步：终值 (Terminal Value) 计算

**永续增长法（首选）：**

```
终年 FCF = 最后一预测年 FCF × (1 + 永续增长率)
终值 = 终年 FCF / (WACC - 永续增长率)

核心约束: 永续增长率 < WACC (否则价值无穷大)
```

**永续增长率选择：**
- 保守：2.0-2.5% (约等于 GDP 增长率)
- 中等：2.5-3.5%
- 进取：3.5-5.0% (仅适用于市场领导者)

**严禁超过**：无风险利率或长期 GDP 增长率。

**退出倍数法（备选）：**
```
终值 = 最后一预测年 EBITDA × 退出倍数

退出倍数来源:
- 行业可比交易倍数
- 先例交易倍数
- 典型范围: 8-15倍 EBITDA
```

**终值的现值：**
```
终值的现值 = 终值 / (1 + WACC)^最终折现期

最终折现期考虑了时间点:
使用期中惯例的 5 年模型: 折现期 = 4.5
```

**终值合理性检查：**
- 应代表企业价值 (EV) 的 50-70%。
- 如果 >75%，模型可能过度依赖终值假设。
- 如果 <40%，检查终值假设是否过于保守。

### 第 9 步：企业价值到权益价值的跨桥

**估值汇总结构：**

```
(+) 预测期 FCF 现值之和 = $X million
(+) 终值的现值 = $Y million
= 企业价值 (EV) = $Z million

(-) 净负债 [或 + 净现金，如果为负] = $A million
= 权益价值 = $B million

÷ 摊薄后发行在外股数 = C million shares
= 隐含每股股价 = $XX.XX

当前股价 = $YY.YY
隐含回报 = (隐含股价 / 当前股价) - 1 = XX%
```

**关键调整项：**
- **净负债 = 总债务 - 现金及等价物**
  - 如果为正：从 EV 中减去（降低权益价值）。
  - 如果为负（净现金）：加回到 EV（增加权益价值）。
- **使用摊薄后股数**：包括期权、受限股 (RSU)、可转债。
- **其他调整**（如适用）：
  - 少数股东权益
  - 养老金负债
  - 经营租赁义务

**估值输出格式：**
```csv
估值组成部分,金额 ($M)
预测期明确 FCF 现值,X.X
终值的现值,Y.Y
企业价值 (EV),Z.Z
(-) 净负债,A.A
权益价值,B.B
,,
发行在外股数 (M),C.C
隐含每股股价,$XX.XX
当前股价,$YY.YY
隐含上涨/(下跌)空间,+XX%
```

### 第 10 步：敏感性分析

在 DCF 工作表底部构建**三张敏感性分析表**，展示估值随不同假设的变化情况：

1. **WACC vs 永续增长率** —— 展示企业价值对折现率和永续增长的敏感性。
2. **营收增长率 vs EBIT 利润率** —— 展示顶线增长和经营杠杆的影响。
3. **Beta vs 无风险利率** —— 展示对权益成本组成部分的敏感性。

**实现方式**：这些是简单的二维网格（**不是** Excel 的“模拟运算表”功能），每个单元格都有公式。每个单元格必须包含该特定假设组合下完整的 DCF 重新计算。详见“核心约束”部分中关于使用 openpyxl 以编程方式填充所有 75 个单元格的详细要求。

### 情景模块选择模式 —— 遵循此方法

**假设按情景分类组织在独立的模块中：**

**核心结构 —— 每个小节标题占用三行：**

```csv
悲观情景假设 (小节标题, 跨行合并单元格)
假设,FY1,FY2,FY3,FY4,FY5
营收增长 (%),12%,10%,9%,8%,7%
EBIT 利润率 (%),45%,44%,43%,42%,41%

基准情景假设 (小节标题, 跨行合并单元格)
假设,FY1,FY2,FY3,FY4,FY5
营收增长 (%),16%,14%,12%,10%,9%
EBIT 利润率 (%),48%,49%,50%,51%,52%

乐观情景假设 (小节标题, 跨行合并单元格)
假设,FY1,FY2,FY3,FY4,FY5
营收增长 (%),20%,18%,15%,13%,11%
EBIT 利润率 (%),50%,51%,52%,53%,54%
```

**每个情景模块必须在小节标题下方紧接着一行列标题**，展示预测年份（FY2025E, FY2026E 等）。没有这一行，用户无法区分哪个数值对应哪个年份。

**如何引用假设 —— 创建合并列：**
1. 情景选择器单元格（如 B6）包含 1=悲观, 2=基准, 或 3=乐观。
2. 创建一个合并列，使用 INDEX 或 OFFSET 公式从正确的情景模块中提取数据。
3. 预测公式引用该合并列（清晰的单元格引用）。
4. 每个情景模块包含全套预测年份的 DCF 假设。

**推荐的合并列模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不要使用散落在各处的 IF 语句：**
`=IF($B$6=1,[悲观模块单元格],IF($B$6=2,[基准模块单元格],[乐观模块单元格]))`

合并列方法集中了逻辑，使模型更易于审计。

### 正确的营收预测模式

**使用 INDEX 公式创建合并列，然后在预测中引用它：**

**第 1 步 —— FY1 增长的合并列：**
`=INDEX([悲观 FY1 增长]:[乐观 FY1 增长], 1, $B$6)`

**第 2 步 —— 营收预测引用合并列：**
`第 1 年营收: =D29*(1+$E$10)`

其中：
- D29 = 前一年营收
- $E$10 = FY1 增长的合并列单元格（包含 INDEX 公式）
- $B$6 = 情景选择器 (1=悲观, 2=基准, 3=乐观)

**这种方法比在每个预测公式中嵌入 IF 语句更整洁**，且更容易审计具体使用了哪些情景假设。

### 正确的 FCF 公式模式

**使用带有 INDEX 公式的合并列，然后在 FCF 计算中引用它们：**

**合并列方法：**
```csv
项目,公式,参考说明
D&A,=E29*$E$21,$E$21 = D&A % 合并列
资本支出,=E29*$E$22,$E$22 = CapEx % 合并列
Δ NWC,=(E29-D29)*$E$23,$E$23 = NWC % 合并列
无杠杆 FCF,=E57+E58-E60-E62,E57=NOPAT E58=D&A E60=CapEx E62=Δ NWC
```

**合并列中的每个单元格都包含一个 INDEX 公式**，根据情景选择器从相应的情景模块中提取数值。这能保持预测公式的简洁与可审计性。

在写入公式前，请先确认情景模块的行位置并设置好合并列。

### 正确的单元格批注格式

**每个硬编码数值都需要此格式：**

“来源：[系统/文档], [日期], [参考章节], [URL（如有）]”

**示例：**
```csv
项目,来源批注
股价,Source: Market data script 2025-10-12 Close price
发行在外股数,Source: 10-K FY2024 Page 45 Note 12
历史营收,Source: 10-K FY2024 Page 32 Consolidated Statements
Beta,Source: Market data script 2025-10-12 5-year monthly beta
一致预期,Source: Management guidance Q3 2024 earnings call
```

1. **节标题行**（合并单元格）：例如“悲观情景假设”
2. **年份列标题行** —— 这是必选的，不要跳过
3. **数据行**，带有具体的假设数值

**结构：**
```csv

EBIT 利润率 (%),X%,X%,X%,X%,X%
```

### 正确的敏感性分析表实现

**重要提示**：这些**不使用** Excel 的“模拟运算表”功能。它们是简单的网格，你将使用 openpyxl 写入常规公式。是的，这意味着总共约 75 个公式（3 张表 × 每张 25 个单元格），但这很直接且是必须的。

**以编程方式填充公式：**

每张敏感性分析表必须完整填充公式，这些公式会针对每种假设组合重新计算隐含股价。**不要使用 Excel 的模拟运算表 (Data Table) 功能**（它需要手动干预，且无法通过 openpyxl 自动化）。

**实现方法 —— 具体示例：**

**表格结构 —— 5×5 网格（奇数维度，基准情景位于中心）：**

如果模型的基准 WACC = 9.0%，基准永续增长率 = 3.0%，则围绕这些数值对称构建轴线：

```csv
WACC vs 永续增长率,   2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%,       [公式], [公式], [公式], [公式], [公式]
              8.5%,       [公式], [公式], [公式], [公式], [公式]
              9.0%,       [公式], [公式], [★  ], [公式], [公式]   ← 中间行 = 基准 WACC
              9.5%,       [公式], [公式], [公式], [公式], [公式]
             10.0%,       [公式], [公式], [公式], [公式], [公式]
                                    ↑
                                 中间列 = 基准永续 g
```

**★ = 中心单元格。** 它的公式输出**必须**等于模型实际隐含的股价（来自估值汇总）。为此单元格应用中蓝色填充 (`#BDD7EE`) 和粗体字体，使基准情景在视觉上锚定。

**坐标轴数值规则：** `轴数值 = [基准 - 2*步长, 基准 - 1*步长, 基准, 基准 + 1*步长, 基准 + 2*步长]` —— 围绕基准对称，奇数确保中心存在。

**公式模式 —— 单元格 B88 (WACC=8.0%, 永续增长=2.0%):**

B88 中的公式应使用以下内容重算隐含价格：
- 来自行标题的 WACC：`$A88` (8.0%)
- 来自列标题的永续增长：`B$87` (2.0%)

**推荐做法：** 引用主 DCF 计算过程，但替换掉这些特定变量。

**示例公式结构：**
`=([使用 $A88 作为折现率的 FCF 现值总和] + [使用 B$87 作为增长率且 $A88 作为 WACC 的终值] - [净负债]) / [总股数]`

**核心要求 —— 为 5x5 网格中的每一个单元格都写入公式（每张表 25 个循环，共计 75 个）。** 使用 openpyxl 在循环中以编程方式写入这些公式。不要跳过此步骤，也不要保留占位文字。

**Python 实现模式：**
```python
# 填充敏感性表的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用 wacc_value 和 term_growth_value 的公式
        formula = f"=<使用 {wacc_value} 和 {term_growth_value} 重新计算的 DCF>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性分析表必须在打开模型时立即生效，无需用户进行任何手动步骤。**

本节包含构建 DCF 模型时所有应避免的错误模式。

### 错误：简化敏感性分析表的近似法或占位文字

**不要使用线性近似：**

```
// 错误 - 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设是线性关系

// 错误 - 除法捷径
B105: =B88/(1+(E48-0.07))      // 没有真正重新计算完整 DCF
```

**不要保留占位文字：**
```
// 错误 - 占位说明
"注：请使用 Excel 模拟运算表功能 (数据 → 模拟分析 → 模拟运算表) 来填充敏感性分析表。"

// 错误 - 空单元格
[因为“这太复杂了”而留空]
```

**不要混淆术语：**
- ❌ “敏感性表需要 Excel 的模拟运算表功能” (不 —— 那是一个特定的 Excel 工具，我们不宜使用)
- ✅ “敏感性表是每个单元格都有公式的简单网格” (是 —— 这正是我们要构建的)

**为什么这些捷径是错误的：**
- 线性近似公式并未真正在折现现金流 —— 它们只是应用了简单的数学调整。
- 金融关系是非线性的，因此结果会不准确。
- 占位文字需要用户手动干预。
- 交付的模型无法立即使用。
- 不够专业，不具备客户交付水平。
- 空单元格 = 交付物不完整。

**常见的推脱理由：**
“手动写 75 个以上的公式感觉很复杂，我留个备注让用户手动完成吧。”

**事实是：** 当你在 Python 中使用 openpyxl 配合循环时，写入 75 个公式是非常简单的。每个公式都遵循相同的模式 —— 只是替换行/列的数值。这是交付物中必须包含的部分。

**替代方案：** 为每一个敏感性单元格填充公式，针对具体的假设组合重新计算完整的 DCF。

### 错误：缺失单元格批注

**不要这样做：**
- 创建了所有硬编码输入项但没有批注。
- 心想“以后再补”。
- 写入“TODO: 添加来源”。
- 保留蓝色输入项而无文档说明。

**为什么这是错误的：**
- 无法验证数据来源。
- 不符合 xlsx 技能要求。
- 无法通过审计审核。
- 以后再修补是浪费时间。

**替代方案：** 在**创建**每个硬编码单元格时同步添加批注。

### 错误：公式行引用偏移

**症状：**
FCF 章节引用了错误的假设行：
`D&A:  =E29*$E$34    // 应为 $E$21，但引用了错误的行`
`CapEx: =E29*$E$41   // 应为 $E$22，但行号发生了偏移`

**为什么会发生这种事：**
1. 先写了公式。
2. 然后插入了标题行。
3. 所有的行引用都发生了偏移。
4. 现在公式指向了错误的单元格 → #REF! 错误。

**替代方案：** 先锁定行布局，然后再写公式。

### 错误：跨情景的每个假设仅占一行

**不要这样构造假设：**
```csv
假设,悲观,基准,乐观
营收增长 FY1,10%,13%,16%
营收增长 FY2,9%,12%,15%
```
这种纵向布局使得很难在一张情景图中看出跨年份的进展。

**为什么这是错误的：**
- 很难看到每个情景内部假设随年份演变的情况。
- 很难在完整预测期内横向对比情景假设。
- 在审查情景逻辑时不够直观。

**替代方案：**
- 为每个情景（悲观、基准、乐观）创建独立的模块。
- 在每个模块内，将假设按预测年份横向排列。
- 这使得每个情景的假设作为一组连贯的数据更易于审查。

### 错误：没有边框

**不要交付没有边框的模型：**
- 没有任何章节划分。
- 所有单元格混杂在一起。
- 难以阅读且显得不专业。

**为什么这是错误的：**
- 不具备客户交付水平。
- 难以导航。
- 看起来很不专业。

**替代方案：** 在所有主要章节周围添加边框。

### 错误：错误的字体颜色或没有区分颜色

**不要这样做：**
- 所有文字都是黑色。
- 只使用填充色（没有字体颜色变化）。
- 搞混了哪些该用蓝色文字，哪些该用黑色。

**为什么这是错误的：**
- 无法区分输入项和公式。
- 审计变得不可能。
- 违反了 xlsx 技能要求。

**替代方案：** 所有硬编码输入项使用**蓝色**文字，所有公式使用**黑色**文字，跨表链接使用**绿色**文字。

### 错误：经营费用基于毛利计算

**不要这样做：**
`S&M: =E33*0.15    // E33 = 毛利 (错误)`

**为什么这是错误的：**
- 经营费用随营收规模变动，而非毛利。
- 会产生不真实的利润率演变情况。
- 这不是实际业务运作的方式。

**替代方案：**
`S&M: =E29*0.15    // E29 = 营业收入 (正确)`

### 前五大错误摘要

1. **公式行引用偏移** → 在写公式前确定所有行位置。
2. **缺失单元格批注** → 在单元格创建时添加批注，不要最后再加。
3. **敏感性分析表简化处理** → 用公式填充所有单元格以实现完全重算，不要用近似值。
4. **情景模块引用错误** → 确保 IF 公式引自正确的悲观/基准/乐观模块。
5. **没有边框** → 添加专业的章节边框，提升交付质量。

此外，还要注意以下错误：

### WACC 计算错误
- 在资本结构中混淆了账面价值和市场价值。
- 错误地混用了权益 Beta 和资产/无杠杆 Beta。
- 债务成本中税率应用错误。
- 无风险利率错误（必须使用当前 10 年期国债收益率）。
- 未根据净负债 vs 净现金头寸进行调整。

### 增长假设缺陷
- 永续增长率 > WACC（会产生无穷大价值）。
- 预测增长率与历史表现严重脱节。
- 忽略了行业增长瓶颈。
- 营收增长与单位经济效益 (unit economics) 逻辑不符。
- 没有任何经营理由的利润率大幅扩张。

### 终值错误
- 使用了错误的增长方法（永续法 vs 倍数法）。
- 终值超过企业价值 (EV) 的 80%（表明过度依赖终值假设）。
- 终值利润率与稳态假设不一致。
- 终值折现期计算错误。

### 现金流预测错误
- 经营费用基于毛利而非营收计算。
- D&A/CapEx 占比与其商业模式不匹配。
- 营运资金变动计算不正确。
- 各年份间税率不一致且无合理解释。
- NOPAT 计算错误。

**这些是最常见的错误。在开始任何 DCF 构建之前，请重新阅读本节内容。**

## Excel 文件创建

**本技能使用 `xlsx` 技能完成所有电子表格操作。** xlsx 技能提供以下支持：
- 标准化的公式构建规则。

## 质量标准评分表

每一个 DCF 模型都必须最大化以下指标：
1. **真实的营收与利润率假设**：基于历史表现。
2. **恰当的资本成本计算**：使用正确的 CAPM 方法论。
3. **全面的敏感性分析**：展示估值范围。
4. **清晰的终值计算**：带有支持理由。
5. **专业的模型结构**：支持情景分析。
6. **透明的文档说明**：所有关键假设均有标注。

## 输入项要求

### 最低要求输入项
1. **公司标识**：股票代码或公司名称。
2. **增长假设**：预测期的营收增长率（或设为“使用一致预期”）。
3. **可选参数**：
   - 预测年限（默认：5 年）。
   - 情景案例（悲观/基准/乐观的增长与利润率假设）。
   - 永续增长率（默认：2.5-3.0%）。
   - 如果不使用 CAPM，则提供特定的 WACC 输入项。

## Excel 模型结构

### 工作表架构

创建**两张工作表**：

1. **DCF** —— 包含敏感性分析的主估值模型。
2. **WACC** —— 资本成本计算。

**核心要求**：敏感性分析表放在 DCF 工作表的底部（不要单独放一张表）。这能让所有估值输出项集中在一起。

### 公式重算（强制性）

在创建或修改 Excel 模型后，**必须**使用 xlsx 技能中的 recalc.py 脚本重算所有公式：

```bash
python recalc.py [Excel文件路径] [超时秒数]
```

示例：
```bash
python recalc.py AAPL_DCF_Model_2026-05-06.xlsx 30
```

该脚本将执行：
- 使用 LibreOffice 重算所有工作表中的公式。
- 扫描**所有**单元格是否存在 Excel 错误（#REF!、#DIV/0!、#VALUE!、#NAME?、#NULL!、#NUM!、#N/A）。
- 返回包含错误位置和数量的详细 JSON。

**预期的输出格式：**
```json
{
  "status": "success",           // 或 "errors_found"
  "total_errors": 0,              // 总错误数
  "total_formulas": 42,           // 文件中的公式总数
  "error_summary": {}             // 仅在发现错误时出现
}
```

**如果发现错误**，输出将包含详情：
```json
{
  "status": "errors_found",
  "total_errors": 2,
  "total_formulas": 42,
  "error_summary": {
    "#REF!": {
      "count": 2,
      "locations": ["DCF!B25", "DCF!C25"]
    }
  }
}
```

在交付模型前，**必须修复所有错误**并重新运行 recalc.py 直到状态变为 "success"。

### 格式标准

**重要提示**：遵循 xlsx 技能中关于公式构建规则和数字格式的规范。DCF 技能增加了特定的视觉表现标准。

**配色方案 —— 两层逻辑**：

**第一层：字体颜色 (xlsx 技能强制要求)**
- **蓝色文字 (RGB: 0,0,255)**：所有硬编码输入项（股价、股数、历史数据、假设值）。
- **黑色文字 (RGB: 0,0,0)**：所有公式和计算项。
- **绿色文字 (RGB: 0,128,0)**：跨表链接（引用自 WACC 工作表的项）。

**第二层：填充色 —— 专业蓝/灰调（除非用户另有说明，否则作为默认值）**
- **保持简约** —— 填充色仅限蓝色和灰色。不要引入绿色、黄色、橙色或多种辅助色。颜色过多的模型显得很不专业。
- **默认填充调色板：**
  - **大节标题**：深蓝色背景 (RGB: 31,78,121 / `#1F4E79`)，白色粗体文字。
  - **小节/列标题**：浅蓝色背景 (RGB: 217,225,242 / `#D9E1F2`)，黑色粗体文字。
  - **输入单元格**：浅灰色背景 (RGB: 242,242,242 / `#F2F2F2`)，蓝色字体 —— 或者为了极致简约，直接保留白底加蓝字。
  - **计算单元格**：白色背景，黑色字体。
  - **输出/汇总行**（每股价值、EV 等）：中蓝色背景 (RGB: 189,215,238 / `#BDD7EE`)，黑色粗体文字。
- **就这些 —— 3 种蓝 + 1 种灰 + 白色。** 克制增加更多颜色的冲动。
- 用户提供的模板或明确的颜色偏好始终优先于这些默认设置。

**两层颜色如何协作：**
- 输入单元格：蓝色字体 + 浅灰填充 = “此处为硬编码输入”
- 公式单元格：黑色字体 + 白色背景 = “此处为计算值”
- 跨表链接：绿色字体 + 白色背景 = “此处引用自另一张表”
- 关键输出：黑色粗体 + 中蓝填充 = “这是最终答案”

**字体颜色告诉你它“是什么”（输入/公式/链接）。填充色告诉你你“在哪里”（标题/数据/输出）。**

### 边框标准（专业外观的必选要求）

**厚边框** (1.5pt) 包围主要章节：
- 关键输入项 (KEY INPUTS)
- 预测假设 (PROJECTION ASSUMPTIONS)
- 5 年现金流预测 (5-YEAR CASH FLOW PROJECTION)
- 终值 (TERMINAL VALUE)
- 估值汇总 (VALUATION SUMMARY)
- 每一张敏感性分析表

**中等边框** (1pt) 分隔子章节：
- 公司详情 vs 历史表现
- 增长假设 vs EBIT 利润率 vs FCF 参数

**细边框** (0.5pt) 用于数据表内部：
- 情景假设表（悲观 | 基准 | 乐观 | 已选）
- 历史 vs 预测财务矩阵

**无边框：** 表格内单个单元格之间（保持整洁、可扫视）。

**边框是强制性的** —— 没有专业边框的模型不符合客户交付标准。

**数字格式**（遵循 xlsx 技能标准）：
- **年份**：格式化为文本字符串（例如 "2024" 而非 "2,024"）。
- **百分比**：`0.0%`（保留一位小数）。
- **货币**：百万级金额用 `$#,##0`；每股金额用 `$#,##0.00` —— 务必在标题中标注单位（如“营业收入 ($mm)”）。
- **零值**：使用数字格式显示为短横线 "-"（例如 `$#,##0;($#,##0);-`）。
- **大额数字**：带有千分位分隔符的 `#,##0`。
- **负数**：带括号的 `(#,##0)`（不用负号）。

**单元格批注 (所有硬编码输入项必选)**：

根据 xlsx 技能，所有硬编码数值必须带有记录来源的单元格批注。格式：“来源：[系统/文档], [日期], [参考章节], [URL（如有）]”

**核心要求**：在**单元格创建时**同步添加批注。不要留到最后。

**WACC 关键公式：**
```
市值 = 股价 × 股数
净负债 = 总债务 - 现金
企业价值 = 市值 + 净负债
权益权重 = 市值 / 企业价值
债务权重 = 净负债 / 企业价值
WACC = (权益成本 × 权益权重) + (税后债务成本 × 债务权重)
```

**术语提醒**：“敏感性表”是指具有行标题、列标题且每个数据格都有公式的简单二维网格。**不要使用** Excel 的“模拟运算表”功能（数据 → 模拟分析 → 模拟运算表）。你将使用 openpyxl 将常规 Excel 公式写入每一个单元格。

1. **WACC vs 永续增长率** (第 87-100 行) —— 5x5 网格 = 25 个公式。
2. **营收增长率 vs EBIT 利润率** (第 102-115 行) —— 5x5 网格 = 25 个公式。
3. **Beta vs 无风险利率** (第 117-130 行) —— 5x5 网格 = 25 个公式。

**总计需要写入的公式数：75 个**（这是必选要求）。

**核心要求**：所有敏感性分析表的单元格必须使用 openpyxl 以编程方式注入公式。严禁使用线性近似捷径。严禁保留关于手动步骤的占位文字或说明。严禁以“过程太复杂”为由留空 —— 请使用 Python 循环来生成公式。

**表格设置：**

   - 使用行标题值（例如 WACC = 9.0%）。

## 情景选择器实现

**三级情景框架：**

### 悲观情景 (Bear Case)
- 保守的营收增长（处于历史区间的低端）。
- 利润率受压或不扩张。
- 较高的 WACC（风险溢价增加）。
- 较低的永续增长率。
- 较高的资本支出假设。

### 基准情景 (Base Case)
- 营收增长参考一致预期或管理层指引。
- 基于经营杠杆的适度利润率扩张。
- 反映当前市场的 WACC。
- 符合 GDP 增长的永续增长率 (2.5-3.0%)。
- 标准的资本支出假设。

### 乐观情景 (Bull Case)
- 乐观的营收增长（处于预测的高端）。
- 显著的利润率扩张。
- 较低的 WACC（风险溢价降低）。
- 较高的永续增长率 (3.5-5.0%)。
- 资本支出强度下降。

**公式实现：**

**不要在各处使用嵌套的 IF 公式。** 取而代之的是创建一个合并列，使用 INDEX 或 OFFSET 公式从对应的情景模块中提取数值。

**推荐模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`，其中 `B10:D10` = 悲观/基准/乐观情景值，`1` = 行偏移，`$B$6` = 情景选择器单元格 (1, 2, 或 3)。

**然后在所有预测中引用该合并列**：
`第 1 年营收: =D29*(1+$E$10)`，其中 $E$10 是合并列中第 1 年增长率的值。

这种方法集中了情景逻辑，使模型更易于审计和维护。

## 常见变体

### 高增长科技公司
- 较长的预测期 (7-10 年)。
- 较高的初期增长率 (20-30%)。
- 随时间推移利润率显著扩张。
- 较高的 WACC (12-15%)。
- 建立单位经济模型（用户数、ARPU 等）。

### 成熟/稳定型公司
- 较短的预测期 (3-5 年)。
- 温和的增长率 (GDP +1-3%)。
- 利润率稳定。
- 较低的 WACC (7-9%)。
- 聚焦现金生成和资本分配。

### 周期性公司
- 跨经济周期建模。
- 使用周期中值对利润率进行正常化。
- 考虑波谷和波峰情景。
- 针对周期性调整 Beta。

### 多业务板块公司
- 为每个业务板块建立独立的折现模型。
- 各板块采用不同的增长率和利润率。
- 分部加总 (SOTP) 估值。
- 考虑协同效应。

> **💡 Appendix: 领域知识小贴士**
> 
> 欢迎来到专业的金融估值世界！如果你是第一次接触这些概念，别担心，这里有一些简单的类比：
>
> 1. **DCF (现金流量折现模型)**：你可以把它想象成一台“金钱时光机”。它的核心逻辑是：公司未来的每一分钱，在今天看来由于通胀和风险，都是要打折扣的。DCF 能帮我们算出公司未来所有能挣到的钱，折合成今天的价值是多少。
> 2. **WACC (加权平均资本成本)**：这可以理解为公司筹集资金的“租金”。不管是用股东的钱还是银行的贷款，都是有成本的。这个百分比代表了公司每年必须跑赢的“及格线”。
> 3. **终值 (Terminal Value)**：我们很难预测 100 年后的事，所以财务模型通常只精确预测前 5-10 年。10 年之后公司还会永远存在下去，那部分无限远景的价值就叫“终值”。
> 4. **敏感性分析 (Sensitivity Analysis)**：这就像是在玩“如果……会怎样？”的游戏。如果 WACC 提高 1%，股价会跌多少？如果营收增长不如预期，估值会变多少？这能帮投资者看清最坏和最好的情况。
> 5. **净负债 (Net Debt)**：这就是“公司欠的钱”减去“公司兜里的现金”。如果一个公司现金比债多（净现金），那它的底气就更足，价值也就更高！
