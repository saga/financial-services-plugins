# 工作流模式

## 顺序式工作流

针对复杂任务，应将操作拆分为清晰的顺序步骤。通常建议在 `SKILL.md` 的开头部分为 Claude 提供流程概览：

```markdown
填充 PDF 表单涉及以下步骤：

1. 分析表单（运行 analyze_form.py）
2. 创建字段映射（编辑 fields.json）
3. 验证映射（运行 validate_fields.py）
4. 填充表单（运行 fill_form.py）
5. 核查输出结果（运行 verify_output.py）
```

## 条件式工作流

对于具有分支逻辑的任务，应引导 Claude 经过决策点：

```markdown
1. 确定修改类型：
   **创建新内容？** → 遵循下文的“创建工作流”
   **编辑现有内容？** → 遵循下文的“编辑工作流”

2. 创建工作流：[步骤]
3. 编辑工作流：[步骤]
```

> **💡 Appendix: 领域知识小贴士**
>
> **1. 什么是“工作流” (Workflow)？**
> 在金融行业，每一项任务（如合规审查或财报分析）都必须遵循严密的标准。工作流就像是一本“金融操作手册”，确保每一个步骤都可审计、可追踪，避免因遗漏环节而导致合规风险或资金损失。
>
> **2. 字段映射 (Field Mapping) 的妙处：**
> 想象你正在处理成百上千份不同格式的资产负债表。映射就是建立一种“对应关系”，告诉电脑：这份表里的“现金”对应系统里的“A账户”。它是金融数据自动化的基础。
>
> **3. 验证 (Validation) 与核查 (Verification) 的双重防线：**
> 金融界有一句名言：“信任，但要核实”。验证是检查“格式”是否合规（比如金额是否为数字），而核查是确认“内容”是否真实准确。这种双重确认能有效防止由于数据错误导致的致命决策偏差。

## Appendix: 金融背景知识

这份文件是"工作流模式（Workflows）"参考文档的详细说明。在金融服务业中，很多任务都有固定的"标准流程"——工作流模式就是把这些流程规范化、模板化，确保每次执行都能达到一致的高质量。

---

### 1. 什么是工作流（Workflow）？

**类比：**
想象你是一家连锁餐厅的厨师。无论顾客去哪家分店，点的"宫保鸡丁"味道都应该一样。这是因为餐厅有"标准食谱"：鸡肉切多大、用多少辣椒、炒几分钟。

工作流就是金融服务的"标准食谱"——确保每个任务都按最佳实践执行。

---

### 2. 金融工作流的"两大类型"

#### 类型 1：顺序式工作流（Sequential Workflow）
**特点**：步骤按固定顺序执行，前一完成后才能开始下一步

**例子**：IPO 流程
```
1. 尽职调查 → 2. 撰写招股书 → 3. 向 SEC 提交 → 4. 路演 → 5. 定价 → 6. 上市
```

#### 类型 2：条件式工作流（Conditional Workflow）
**特点**：根据中间结果决定下一步走向

**例子**：信贷审批
```
1. 评估信用评分
   → 评分 > 800：自动通过
   → 评分 600-800：人工复核
   → 评分 < 600：拒绝
```

---

### 3. 工作流设计的"五大原则"

| 原则 | 说明 |
|------|------|
| 清晰性 | 每个步骤都有明确的目标和产出 |
| 可度量 | 每个步骤都有 KPI 或检查点 |
| 可追溯 | 每个决策都有记录，便于审计 |
| 可优化 | 定期回顾，持续改进 |
| 容错性 | 设计备选方案，应对异常情况 |

---

### 4. 真实案例：投行的并购工作流

某投行并购交易的标准工作流：

| 阶段 | 时间 | 关键产出 | 负责角色 |
|------|------|---------|---------|
| 项目启动 | Week 1 | 项目计划书 | 项目经理 |
| 初步分析 | Week 2-3 | 可行性分析报告 | 分析师 |
| 尽职调查 | Week 4-8 | DD 报告 | 多团队协作 |
| 估值建模 | Week 6-9 | 财务模型 | 分析师 |
| 交易谈判 | Week 10-14 | 交易条款清单 | 副总裁 |
| 交割准备 | Week 15-20 | 法律文件 | 律师+投行 |

### 给小白的一句话

> 工作流就是金融服务的"标准化作业流程"——想象你去麦当劳，无论在北京还是纽约，巨无霸汉堡的味道都一样。这不是因为厨师厉害，而是因为麦当劳有"标准食谱"。金融工作流也是一样的道理：把复杂的金融服务拆解成"一步一步的标准动作"，确保每次都能产出高质量、可追溯、可复现的结果。没有工作流，金融服务就靠"个人经验"，质量不稳定；有了工作流，即使是新人也能按图索骥，产出专业水准的成果。
