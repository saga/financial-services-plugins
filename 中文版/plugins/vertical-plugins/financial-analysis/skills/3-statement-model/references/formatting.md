# 财务建模格式规范指南

| 元素 | 格式规范 |
|---------|--------|
| 硬编码输入值 (Hard-coded inputs) | 蓝色字体 |
| 公式 (Formulas) | 黑色字体 |
| 跨表链接 (Links to other sheets) | 绿色字体 |
| 检查项单元格 (Check cells) | 报错显示红色，平稳显示绿色 |
| 负值 (Negative values) | 使用括号表示，不使用负号 |
| 货币 (Currency) | 大额数值不保留小数，每股收益保留 2 位小数 |
| 百分比 (Percentages) | 保留 1 位小数 |
| 表头 (Headers) | 加粗，带下框线 |
| 单位行 (Units row) | 在表头下方标注单位（百万元、$ millions、% 等） |

## 视觉分隔准则

- 历史年度与预测年度列之间使用细垂直边框分隔
- 各部分总计后使用粗下框线（例如：总资产）
- 小计使用单下框线
- 总计使用双下框线

## 总计与小计行格式

所有总计和小计行对应的数值必须使用**加粗字体**，以便清晰区分汇总数据与具体明细项。

### 利润表 (P&L) 选项卡
| 行项目 | 格式规范 |
|-----|------------|
| 营业总收入 (Gross Revenue) | 加粗 |
| 营收总成本 (Total Cost of Revenue) | 加粗 |
| 毛利 (Gross Profit) | 加粗 |
| 销售及行管费用总计 (Total SG&A) | 加粗 |
| EBITDA (息税折旧摊销前利润) | 加粗 |
| EBIT (息税前利润) | 加粗 |
| EBT (税前利润) | 加粗 |
| 税后净利润 (Net Profit After Tax) | 加粗 |

### 资产负债表 选项卡
| 行项目 | 格式规范 |
|-----|------------|
| 流动资产总计 (Total Current Assets) | 加粗 |
| 非流动资产总计 (Total Non-Current Assets) | 加粗 |
| 其他资产总计 (Total Other Assets) | 加粗 |
| 资产总计 (Total Assets) | 加粗 |
| 流动负债总计 (Total Current Liabilities) | 加粗 |
| 非流动负债总计 (Total Non-Current Liabilities) | 加粗 |
| 权益总额 (Total Equity) | 加粗 |
| 负债及权益总计 (Total Liabilities and Equity) | 加粗 |

### 现金流量表 选项卡
| 行项目 | 格式规范 |
|-----|------------|
| 营运资本变动前经营活动现金流 | 加粗 |
| 营运资本变动总计 | 加粗 |
| 经营活动产生的现金流量净额 | 加粗 |
| 投资活动产生的现金流量净额 | 加粗 |
| 筹资活动产生的现金流量净额 | 加粗 |
| 期末现金余额 | 加粗 |

**注：** 此列表未尽所有项目。凡是代表总计、小计或模型中汇总计算的行，均应应用加粗格式。

## 资产负债表平衡检查行格式

资产负债表检查行（位于负债及权益总计下方）需应用条件格式，非零值显示为红色。当资产负债表正确平稳（检查值 = 0）时，数值显示为黑色或标准格式。

| 检查值状态 | 字体颜色 |
|-------------|------------|
| = 0 (平衡/配平) | 黑色 (标准) |
| ≠ 0 (报错) | 红色 |

**实现方式：** 应用自定义数字格式 `[Red][<>0]0.00;[Red][<>0](0.00);0.00` 或使用 Excel 条件格式规则：“单元格数值 ≠ 0” → 红色字体。

## 利润率行格式

| 元素 | 格式规范 |
|---------|--------|
| 利润率 % 行 | 缩进，斜体，保留 1 位小数 |
| 正向趋势 | 无需特殊格式（或浅绿色） |
| 负向趋势 | 标记需审核（浅黄色） |
| 低于行业平均水平 | 考虑高亮以备讨论 |

## 信用指标格式

| 元素 | 格式规范 |
|---------|--------|
| 杠杆倍数 | 保留 1 位小数并带“x”后缀（如 2.5x） |
| 百分比 | 保留 1 位小数并带“%”后缀 |
| 净债务为负值 | 用括号表示，代表处于净现金状态 |
| 部分表头 | 加粗，“CREDIT METRICS” |
| 分隔线 | 信用指标部分上方留细边框 |

## 信用指标阈值颜色

| 指标 | 绿色 (安全) | 黄色 (预警) | 红色 (风险) |
|--------|-------|--------|-----|
| 总债务 / EBITDA | < 2.5x | 2.5x-4.0x | > 4.0x |
| 净债务 / EBITDA | < 2.0x | 2.0x-3.5x | > 3.5x |
| 利息保障倍数 | > 4.0x | 2.5x-4.0x | < 2.5x |
| 负债 / 总资本 | < 40% | 40%-60% | > 60% |
| 流动比率 | > 1.5x | 1.0x-1.5x | < 1.0x |
| 速动比率 | > 1.0x | 0.75x-1.0x | < 0.75x |

## 检查 (Checks) 选项卡的条件格式

- 单元格包含“通过 (pass)”指示 → 绿色填充
- 单元格包含“失败 (fail)”指示 → 红色填充
- 单元格包含“警告 (warning)”指示 → 黄色填充
- 差额单元格 = 0 → 浅绿色填充
- 差额单元格 ≠ 0 → 浅红色填充

## 利润率合理性标记

- 毛利率 < 0% → 错误：检查营收成本 (COGS)
- 毛利率 > 80% → 警告：核实收入/营收成本
- EBITDA 利润率 < 0% → 标记：存在经营亏损
- EBITDA 利润率 > 50% → 警告：数值异常偏高
- 净利率 < 0% → 标记：净亏损（在增长阶段可能视为可接收）
- 净利率 > 毛利率 → 错误：公式计算逻辑错误

## Appendix: 金融背景知识

这份文件是"财务建模格式规范（Formatting Standards）"的详细说明。在投行和 PE 行业，财务模型的格式不是"锦上添花"，而是"硬性要求"——一个格式混乱的模型会被直接打回重做。

---

### 1. 为什么格式如此重要？

**类比：**
想象你是一名医生，每天要开处方。如果处方上的字迹潦草、剂量单位混乱，药剂师可能配错药，病人可能吃错剂量。在金融行业，财务模型就是"处方"——格式错误可能导致投资决策失误。

**核心原因：**
1. **可读性**：合伙人要在 5 分钟内看懂你的模型
2. **可审计性**：审计师要追溯每个数字的来源
3. **可维护性**：团队多人协作时，格式统一才能避免混乱

---

### 2. 投行建模的"六大铁律"

#### 铁律 1：颜色编码
| 颜色 | 含义 | 惩罚 |
|------|------|------|
| 蓝色 | 手动输入 | 误用黑色 = 无法区分假设和公式 |
| 黑色 | 公式 | 误用蓝色 = 模型无法自动更新 |
| 绿色 | 跨表链接 | 误用黑色 = 不知道数据从哪来 |

#### 铁律 2：负数用括号
```
正确：(123)
错误：-123
```
**原因：** 负号在打印时容易看不清，括号更醒目。

#### 铁律 3：单位统一
```
所有金额列必须在表头标注单位：
- 百万元（RMB mn）
- 千万元
- 美元百万（$ millions）
```

#### 铁律 4：历史 vs 预测分隔
```
历史年度和预测年度之间必须有垂直边框分隔
```

#### 铁律 5：总计行加粗
```
所有小计、总计行必须加粗
```

#### 铁律 6：禁止硬编码
```
错误：直接在单元格输入 1000
正确：= 假设值单元格 x 增长率
```

---

### 3. 模型审阅的"恐怖故事"

**故事 1：**
某分析师把"百万"当成"十亿"输入模型，导致估值差了 1000 倍。投决会差点通过一笔"价值 5000 亿"的交易，实际只值 5 亿。

**故事 2：**
某模型中负数使用了"-"号而不是括号，打印出来时被误读为正数。客户据此买入，亏了 2000 万。

---

### 给小白的一句话

> 财务建模的格式规范就像"交通法规"——表面上看是"束缚"，实际上是"保护"。蓝色=假设、黑色=公式、绿色=链接，这套颜色编码让所有人都能在一秒钟内看懂"这个数是怎么来的"。如果格式混乱，就像道路上没有红绿灯——看起来大家都自由了，实际上事故频发。在投行，一个格式错误的模型可能直接导致"几亿的投资决策出错"，所以格式绝不是"小事"。
