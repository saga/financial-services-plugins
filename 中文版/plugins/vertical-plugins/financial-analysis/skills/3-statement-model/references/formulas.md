# 公式参考手册

**重要提示：** 除非用户另有说明，请务必使用本参考文档中列出的公式。

---

## 核心勾稽关系

```
资产负债表：        资产 = 负债 + 股东权益
净利润：           利润表净利润 → 经营活动现金流 (起点)
现金流量：         现金变动 = 经营现金流 + 投资现金流 + 筹资现金流
现金平账：         期末现金 (现金流量表) = 现金 (资产负债表资产项)
现金月度/年度：    期末现金 (月度) = 期末现金 (年度)
留存收益：         期初留存收益 + 净利润 - 股利 = 期末留存收益
股权融资：         普通股/资本公积变动 (资产负债表) = 股票发行 (筹资现金流)
第0年股权：        第0年募集的股权 = 第1年期初股权
```

## 毛利计算

**重要提示：** 毛利必须基于净收入（Net Revenue）计算，而非总收入（Gross Revenue）。

```
净收入 - 收入成本 = 毛利
```

| 术语 | 定义 |
|------|------------|
| 总收入 (Gross Revenue) | 扣除任何项目前的收入总额 |
| 净收入 (Net Revenue) | 总收入 - 退货 - 折让 - 折扣 |
| 收入成本 (Cost of Revenue) | 可直接归属于所销售产品/服务生产的直接成本 |
| 毛利 (Gross Profit) | 净收入 - 收入成本 |

**注意：** 在进行盈利能力计算时，始终以净收入（在大多数报表中也称为“净销售额”或简称为“收入”）作为起点。总收入会夸大真实的顶线表现。

## 利润率公式

```
毛利率 %            = 毛利 / 净收入
EBITDA              = EBIT + 折旧与摊销 (或 = 毛利 - 运营支出)
EBITDA 利润率 %     = EBITDA / 净收入
EBIT 利润率 %       = EBIT / 净收入
净利润率 %          = 净利润 / 净收入
```

## 信用指标公式

```
总债务              = 短期借款 + 长期借款
净债务              = 总债务 - 现金
总债务 / EBITDA      = 总债务 / EBITDA (取自利润表)
净债务 / EBITDA      = 净债务 / EBITDA (取自利润表)
利息覆盖率          = EBITDA / 利息支出 (取自利润表)
净利息支出 % 债务   = 净利息支出 / 长期借款
总杠杆率            = 总债务 / (总债务 + 总权益)
产权比率            = 总债务 / 总权益
流动比率            = 流动资产合计 / 流动负债合计
速动比率            = (流动资产合计 - 存货) / 流动负债合计
```

## 预测公式（净收入百分比法）

```
收入成本 (预测)      = 净收入 × 收入成本 % 假设
销售项 (预测)        = 净收入 × 销售项 % 假设
管理费 (预测)        = 净收入 × 管理费 % 假设
研发费 (预测)        = 净收入 × 研发费 % 假设
股权激励 (预测)      = 净收入 × 股权激励 % 假设
```

## 营运资金公式

```
应收账款 (AR)
  期初应收
  + 收入 (来自利润表)
  - 现金回收 (调和项)
  = 期末应收
  DSO (应收账款周转天数) = (应收 / 收入) × 365

存货 (Inventory)
  期初存货
  + 采购额 (调和项)
  - 销货成本 (来自利润表)
  = 期末存货
  DIO (存货周转天数) = (存货 / 销货成本) × 365

应付账款 (AP)
  期初应付
  + 采购额 (来自存货计算)
  - 现金支付 (调和项)
  = 期末应付
  DPO (应付账款周转天数) = (应付 / 销货成本) × 365

营运资金净额 = 应收账款 + 存货 - 应付账款
营运资金变动 (ΔWC) = 当前营运资金净额 - 上期营运资金净额
```

## 折旧与摊销表公式

```
期初 PP&E (原值)
+ 资本开支 (CapEx)
= 期末 PP&E (原值)

期初累计折旧
+ 折旧费用
= 期末累计折旧

PP&E (净值) = PP&E 原值 - 累计折旧
```

## 债务表公式

```
期初债务余额
+ 新增借款
- 偿还额
= 期末债务余额

利息支出 = 平均债务余额 × 利率
  (使用期初余额以避免循环引用，或在启用循环引用时进行迭代计算)
```

## 留存收益公式

```
期初留存收益
+ 净利润 (来自利润表)
+ 股权激励支付 (SBC) (来自利润表)
- 股利
= 期末留存收益
```

## 净经营亏损 (NOL) 表公式

```
NOL 结转表

期初 NOL 余额 (第1年 / 成立时 = 0)
+ 新产生的 NOL (如果 EBT < 0，则取 ABS(EBT)，否则为 0)
- 已抵用的 NOL (受限于应纳税所得额及抵用上限)
= 期末 NOL 余额

初始余额规则

对于新业务或第一个建模期间：
  期初 NOL 余额 = 0
  NOL 只能通过实现的亏损 (EBT < 0) 增加
  NOL 不能凭空产生或假设

NOL 抵用计算

税前利润 (EBT)
  如果 EBT > 0：
    可用 NOL = 期初 NOL 余额
    抵用上限 = EBT × 80% (2017年后联邦限制)
    已抵用 NOL = MIN(可用 NOL, 抵用上限)
    应纳税所得额 = EBT - 已抵用 NOL
  如果 EBT ≤ 0：
    已抵用 NOL = 0
    应纳税所得额 = 0
    新产生的 NOL = ABS(EBT)

带 NOL 的税费计算

应交所得税 = MAX(0, 应纳税所得额 × 税率)
  (税额不能为负；亏损会产生 NOL 资产)

NOL 对应的递延所得税资产 (DTA)

DTA - NOL 结转 = 期末 NOL 余额 × 税率
ΔDTA = 本期 DTA - 上期 DTA
  (DTA 增加 = 现金流量表中的非现金收益)
  (DTA 减少 = 现金流量表中的非现金支出)
```

## 资产负债表结构

```
资产
  现金 (来自现金流量表期末现金)
  应收账款 (来自营运资金)
  存货 (来自营运资金)
  流动资产合计
  
  PP&E 净值 (来自折旧摊销表)
  递延所得税资产 - NOL (来自 NOL 表)
  非流动资产合计
  资产总计

负债
  应付账款 (来自营运资金)
  一年内到期的长期借款 (来自债务表)
  流动负债合计
  
  长期借款 (来自债务表)
  负债合计

股东权益
  普通股
  留存收益 (来自留存收益表)
  股东权益合计

平衡校验：资产 - 负债 - 股东权益 = 0
```

## 现金流量表结构

```
经营活动现金流 (CFO)
  净利润 (关联：利润表)
  + 折旧与摊销 (关联：折旧摊销表)
  + 股权激励支付 (SBC) (关联：利润表或假设)
  - ΔDTA (递延所得税资产) (关联：NOL 表；DTA 增加代表现金流出)
  - ΔAR (关联：营运资金)
  - Δ存货 (关联：营运资金)
  + ΔAP (关联：营运资金)
  = CFO

投资活动现金流 (CFI)
  - 资本开支 (CapEx) (关联：折旧摊销表)
  = CFI

筹资活动现金流 (CFF)
  + 债务发行 (关联：债务表)
  - 债务偿还 (关联：债务表)
  + 股票发行 (关联：资产负债表普通股/资本公积)

## Appendix: 金融背景知识

这份文件是"公式参考手册（Formulas Reference）"的详细说明。在财务建模中，公式是"骨架"——它们确保三张表（资产负债表、利润表、现金流量表）之间的数据关系正确，不会自相矛盾。

---

### 1. 三张表的"勾稽关系"

财务建模的核心是"三张表必须配平"——就像解一道数学题，左边必须等于右边。

#### 核心等式 1：资产负债表
```
资产 = 负债 + 股东权益
```
**类比：** 你的全部财产（资产）= 欠别人的钱（负债）+ 真正属于你的钱（股东权益）

#### 核心等式 2：现金流量表
```
现金变动 = 经营现金流 + 投资现金流 + 筹资现金流
```
**类比：** 你这个月钱包里多了多少钱 = 工资收入 + 炒股盈亏 + 借钱/还钱

#### 核心等式 3：利润表与现金流量表
```
净利润 → 经营现金流的起点
```
**关键理解：** 利润 ≠ 现金。公司可以"账面赚钱"但实际"没钱"（因为客户还没付款）。

---

### 2. 关键财务指标公式详解

#### 2.1 毛利率（Gross Margin）
```
毛利率 = (营业收入 - 营业成本) / 营业收入
```
**意义：** 每卖 100 元产品，毛利是多少？
**例子：** 毛利率 40% = 卖 100 元，成本 60 元，毛利 40 元

#### 2.2 净利率（Net Margin）
```
净利率 = 净利润 / 营业收入
```
**意义：** 扣除所有费用后，实际赚多少？

#### 2.3 ROE（净资产收益率）
```
ROE = 净利润 / 股东权益
```
**意义：** 股东投入的每 1 元钱，能赚多少钱？
**巴菲特标准：** ROE > 15% 是好公司

#### 2.4 EBITDA（息税折旧前利润）
```
EBITDA = 营业利润 + 折旧 + 摊销
```
**意义：** 公司在"还利息、交税、提折旧"之前赚了多少钱？
**用途：** 比较不同公司的盈利能力（不受资本结构和税收政策影响）

### 3. 建模中的"颜色编码"

| 颜色 | 含义 | 例子 |
|------|------|------|
| 蓝色 | 手动输入的假设值 | 预测收入增长率 |
| 黑色 | 公式计算的结果 | 毛利润 = 收入 x 毛利率 |
| 绿色 | 跨表引用的链接 | 引用资产负债表的现金 |

**为什么需要颜色编码？**
- 让审阅者一眼看出"哪些是假设，哪些是计算"
- 避免"硬编码"（把公式结果手动输入，导致模型无法自动更新）

---

### 4. 真实案例：一个"配不平"的模型

某实习生做的财务模型：

| 检查项 | 结果 | 问题 |
|--------|------|------|
| 资产 = 负债 + 权益 | 不等 | 差了 500 万 |
| 现金流量表净额 | +300 万 | |
| 资产负债表现金变动 | +200 万 | |
| 差异 | 100 万 | 某笔投资支出重复计算 |

**教训：** 模型配不平 = 有错误。找到错误之前，所有估值结论都不可信。

---

### 给小白的一句话

> 财务模型的公式就像乐高积木的"接口"——每一块积木（数据）必须通过正确的接口（公式）连接在一起，否则整个城堡（模型）就会倒塌。三张表的勾稽关系就是"接口标准"：资产负债表必须平衡，现金流量表必须和资产负债表的现金变动一致，利润表的净利润必须是现金流量表的起点。如果模型"配不平"，就像你算账时发现"钱包里的钱"和"记账本上的钱"对不上——一定有哪里算错了。
