# 三表财务模型模板填充指南

完成并填充整合财务模型模板，确保利润表（IS）、资产负债表（BS）和现金流量表（CF）之间的勾稽关系正确链接。

## ⚠️ 核心原则 — 填充模板前必读

**运行环境 — Office JS 与 Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS。通过 `range.formulas = [["=D14*(1+Assumptions!$B$5)"]]` 编写公式，绝不要对派生单元格使用 `range.values`。无需单独重算，Excel 会原生计算。使用 `context.workbook.worksheets.getItem(...)` 在标签页间跳转。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。编写 `ws["D15"] = "=D14*(1+Assumptions!$B$5)"`，并在交付前运行 `recalc.py`。
- **Office JS 合并单元格陷阱：** 不要先调用 `.merge()` 再在合并区域上设置 `.values`，这会抛出 `InvalidArgument` 错误，因为该区域仍会报告其合并前的维度。应先在左上角单元格写入值，然后再合并并格式化整个区域：`ws.getRange("A1").values = [["INCOME STATEMENT"]]; const h = ws.getRange("A1:G1"); h.merge(); h.format.fill.color = "#1F4E79";`
- 以下所有原则在两种环境下均同样适用。

**公式优于硬编码（不可逾越的底线）：**
- 每个预测单元格、余额滚动（Roll-forward）、报表链接和小计必须使用 Excel 公式，绝不能是预先计算好的数值。
- 使用 Python/openpyxl 时：编写公式字符串（`ws["D15"] = "=D14*(1 ...)"`），而非计算结果（`ws["D15"] = 12500`）。
- **唯二**允许包含硬编码数字的单元格是：(1) 历史实绩值（Actuals），(2) “假设”标签页中的驱动因素。
- 如果你发现自己在 Python 中计算了一个值并将其写入单元格 —— 请立即停止，改为编写公式。
- 原因：模型必须能够在情景切换或假设更改时灵活变动。硬编码会使所有后续的完整性检查静默失效。

**与用户进行分布确认：**
1. **完成模板映射后** → 向用户展示你识别出的标签页/科目，并在改动单元格前进行确认。
2. **填充历史数据后** → 向用户展示历史数据块，确认数值和期间与数据源匹配。
3. **完成利润表预测后** → 运行小计核对，向用户展示利润表预测，确认后再进行资产负债表。
4. **完成资产负债表后** → 向用户展示每一期的平衡检查（资产 = 负债 + 权益），确认后再进行现金流量表。
5. **完成现金流量表后** → 向用户展示现金钩稽结果（现金流期末现金 = 资产负债表现金），确认后再完成终稿。
6. **不要一次性填充整个模型并提交** —— 在每张报表完成时暂停，展示工作成果，及早发现错误。

## 格式规范 — 专业蓝/灰配色（除非模板/用户另有要求）

**保持颜色极简。** 单元格填充仅使用蓝色和灰色。严禁加入绿色、黄色、橙色或多种点缀色 —— 一个干净的模型需要克制。

| 元素 | 填充颜色 | 字体颜色 |
|---|---|---|
| 报表标题（IS / BS / CF 标题） | 深蓝 `#1F4E79` | 白色加粗 |
| 列标题（FY2024A, FY2025E 等） | 浅蓝 `#D9E1F2` | 黑色加粗 |
| 输入单元格（历史值、假设驱动因素） | 浅灰 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨表链接 | 白色 | 绿色 `#008000` |
| 检查行 / 核心合计 | 中蓝 `#BDD7EE` | 黑色加粗 |

**即：3 种蓝色 + 1 种灰色 + 白色。** 如果模板已有自带色系，请遵循模板。

字体颜色代表单元格的**性质**（输入/公式/链接）。填充颜色代表你所处的**位置**（表头/数据/检查项）。

## 模型结构

### 识别模板标签页组织方式

不同模板的标签页命名和组织方式各异。在填充前，请查看所有标签页以理解结构。以下是常见的标签页名称及其典型内容：

| 常见标签页名称 | 寻找的内容 |
|------------------|----------------------|
| IS, P&L, Income Statement | 利润表（损益表） |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资本明细表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销明细表 |
| Debt, Debt Schedule | 债务明细表 |
| NOL, Tax, DTA | 净经营亏损与税务明细表 |
| Assumptions, Inputs, Drivers | 驱动因素假设与输入参数 |
| Checks, Audit, Validation | 错误检查仪表盘 |

**模板审查清单**
- 识别模板中存在的标签页（并非所有模板都包含每个明细表）。
- 记录未在上方列出的模板特定标签页。
- 理解标签页间的依赖关系（例如，哪些明细表的数据会流入主表）。
- 定位每个标签页上的输入单元格与公式单元格。

### 理解模板结构

在填充前，先熟悉现有布局，确保数据填入正确位置且公式保持完整。

**识别行结构**
- 找到每个标签页顶部的模型标题。
- 识别科目表头及其视觉区分。
- 找到显示单位的行（单位：百万、$、%、倍数等）。
- 注意区分历史值（Actuals）与预测值（Estimates）期间的列标题。
- 确认期间标签（如 FY2024A, FY2025E）。
- 识别输入单元格与公式单元格（通常通过字体颜色区分）。

**识别列结构**
- 确认最左列的会计科目标签。
- 核实历史年度位于预测年度之前。
- 注意历史期间与预测期间之间的视觉边界。
- 检查所有标签页的列顺序是否一致。

**使用命名区域**
模板通常为关键输入和输出使用“命名区域”。在输入数据前：
- 检查模板中现有的命名区域（Excel 中：公式 → 名称管理器）。
- 常见的命名区域包括：营收增长率、成本占比、关键输出指标（净利润、EBITDA、总债务、现金）、情景切换开关。
- 确保输入的数据单元格确实流向了这些命名区域。

### 预测期间
- 模板通常从最后一个历史年度向后预测 5 年。
- 验证历史 (A) 与预测 (E) 列是否清晰隔离。
- 确认列标题使用财年表示法（如 FY2024A, FY2025E）。

## 利润率分析

**注意：仅当用户要求或模板明确需要时才进行以下分析。若未提及，请跳过此部分。**

在利润表（IS）页签计算并展示盈利能力指标，以追踪运营效率并进行同业对比。

### 核心利润率指标

| 指标 | 公式 | 衡量维度 |
|--------|---------|------------------|
| 毛利率 | 毛利 / 营业收入 | 定价能力、生产效率 |
| EBITDA 利润率 | EBITDA / 营业收入 | 核心业务盈利能力 |
| EBIT 利润率 | EBIT / 营业收入 | 扣除折旧摊销后的盈利能力 |
| 净利率 | 净利润 / 营业收入 | 最终盈利水平 |

### 包含利润率的利润表布局

在每个利润项目下方直接显示利润率百分比：
- 毛利下方显示毛利率 (%)
- EBIT 下方显示 EBIT 利润率 (%)
- EBITDA 下方显示 EBITDA 利润率 (%)
- 净利润下方显示 净利率 (%)

## 信用指标

**注意：仅当用户要求或模板明确需要时才进行以下分析。若未提及，请跳过此部分。**

在资产负债表（BS）页签计算并展示信用/杠杆指标，以评估财务健康状况、债务承受能力及融资契约合规性。

### 核心信用指标

| 指标 | 公式 | 衡量维度 |
|--------|---------|------------------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | (总债务 - 现金) / LTM EBITDA | 扣除现金后的杠杆水平 |
| 利息保障倍数 | EBITDA / 利息支出 | 偿债能力 |
| 债务 / 总资本 | 总债务 / (总债务 + 权益) | 资本结构 |
| 产权比率 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | (流动资产 - 存货) / 流动负债 | 变现能力 |

### 信用指标层级检查

验证“乐观情景”显示出最强的信用状况：
- 杠杆率：乐观 < 基准 < 悲观（越低越好）
- 利息覆盖率：乐观 > 基准 > 悲观（越高越好）
- 流动性：乐观 > 基准 > 悲观（越高越好）

### 融资契约监控

如果已知债务契约条款，需添加明确的合规性检查，将实际指标与契约阈值进行对比。

## 情景分析 (基准 / 乐观 / 悲观)

在假设标签页使用情景切换开关（下拉菜单），配合 CHOOSE 或 INDEX/MATCH 函数。

| 情景 | 描述 |
|----------|-------------|
| 基准情景 (Base) | 管理层指引或市场共识预期值 |
| 乐观情景 (Upside) | 高于指引的增长、利润率扩张 |
| 悲观情景 (Downside) | 低于趋势的增长、利润率收缩 |

**关键敏感性驱动因素**：营收增长、毛利率、管理及销售费用 %、应收/存货/应付周转天数、资本开支 %、利率、税率。

**情景审计检查**：切换开关时所有报表均会随之变动、所有情景下资产负债表均平衡、现金钩稽正确、层级逻辑成立（在净利润、EBITDA、自由现金流、利润率方面，乐观 > 基准 > 悲观）。

## 完成模型模板的步骤

本部分提供了在保持现有公式并确保数据完整性的前提下，完成任何三表模型模板的通用指南。

### 第 1 步：分析模板结构

在输入任何数据之前，彻底审查布局以理解其架构：

**识别输入单元格与公式单元格**
- 寻找区分两者的视觉线索（字体颜色、单元格阴影）。
- 通用惯例：蓝色字体 = 输入值，黑色字体 = 公式值，绿色字体 = 跨表链接。
- 使用 Excel 的“追踪引用单元格/从属单元格”功能来理解逻辑。
- 检查可能控制关键输入的命名区域。

**映射模板流程**
- 识别哪些标签页流向其他标签页（例如：假设 → 利润表 → 资产负债表 → 现金流量表）。
- 记录所有辅助明细表及其与主表的链接逻辑。

### 第 2 步：在不破坏公式的情况下填充数据

**数据输入金律**

| 原则 | 描述 |
|------|-------------|
| 仅编辑预测输入格 | 绝不要覆盖包含公式的单元格，除非有意替换该公式 |
| 保持单元格格式 | 复制数据时，使用“选择性粘贴 - 数值”（Ctrl+Shift+V） |
| 匹配模板单位 | 输入前核实模板使用的是千、百万还是原数值 |
| 遵循正负号惯例 | 遵循模板现有的正负号逻辑（例如费用是正数记账还是负数记账） |
| 检查循环引用 | 如果模板使用迭代计算，请确保已开启“启用迭代计算” |

**安全填充流程**
1. 识别专门指定的输入区域（通常有高亮或标签）。
2. 先填充历史数据，验证其在这些期间的公式计算是否正确。
3. 填充驱动预测计算的假设参数。
4. 审查输出结果，确认公式运行符合预期。
5. 若必须修改公式单元格，请先备份原始公式。

**处理预置公式**
- 如果公式引用的单元格尚未填充，可能会出现临时错误（如 #REF!, #DIV/0!），待输入完整后会消失。
- 当公式产生意外结果时，追踪其引用项以定位缺失或错误的输入。
- 严禁在未检查跨表公式依赖的情况下删除行/列。

### 第 3 步：验证公式

**公式完整性检查**

在依赖模板输出前，验证公式是否正常运行：

| 检查类型 | 方法 |
|------------|--------|
| 追踪引用项 | 选中公式格 → 公式 → 追踪引用单元格，确认其引用了正确的输入 |
| 追踪从属项 | 验证关键输入是否流向了预期的输出项 |
| 分步计算 | 使用“公式求值”功能逐步拆解复杂计算 |
| 检查硬编码 | 预测公式应引用假设参数，而非包含硬编码数值 |
| 测试已知值 | 输入简单的测试值验证输出是否符合逻辑 |
| 跨期一致性 | 确保同一预测逻辑应用于所有预测期间 |

**需警惕的常见公式问题**
- 混合绝对/相对引用导致在跨期复制时产生错误结果。
- 指向已删除区域或外部文件的断路链接（#REF! 错误）。
- 营收尚未产生的前期出现除以零错误（#DIV/0! 错误）。
- 循环引用警告（在利息计算中可能是刻意为之）。
- 预测列之间的公式不一致（使用 Ctrl+\ 查找差异）。

**验证跨表链接**
- 确认出现在多个标签页上的值是动态链接的，而非手动重复输入的。
- 验证明细表合计与主表相应科目完全一致。
- 确保所有标签页的时间跨度完全对齐。

### 第 4 步：按工作表质量检查

填充模板后，对每张表进行如下验证：

**利润表 (IS) 质检**
- 历史期间的营收数据与源数据匹配。
- 所有费用科目之和等于报告的总额。
- 小计（毛利、EBIT、EBT、净利润）计算正确。
- 所得税计算逻辑合理（能正确处理亏损结转）。
- 预测驱动因素引用的是假设页（无硬编码）。

**资产负债表 (BS) 质检**
- **资产 = 负债 + 权益**（每一期必须平衡，这是核心检查）。
- 现金余额与现金流量表期末现金一致。
- 营运资本科目与辅助明细表勾稽（如有）。
- 留存收益滚动正确：期初 + 净利润 - 股利 +/- 调整 = 期末。
- 债务余额与债务明细表一致。

**现金流量表 (CF) 质检**
- 经营性现金流顶部的净利润与利润表一致。
- 非现金加回项（折旧摊销、股份支付等）与源报表一致。
- 营运资本变动的正负号正确（资产增加 = 现金流出 = 负数）。
- 资本开支（CapEx）与固定资产滚动明细一致。
- 筹资活动与资产负债表中的债务及权益变动一致。
- 期末现金等于资产负债表中的现金。

### 第 5 步：跨表完整性核对

验证单表后，确认三张报表已完美整合：

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表平衡表 | 资产 - 负债 - 权益 | = 0 |
| 现金钩稽 | CF 期末现金 - BS 现金 | = 0 |
| 净利润链接 | IS 净利润 - CF 起始净利润 | = 0 |
| 留存收益钩稽 | 期初 RE + 净利润 - 股利 - BS 期末 RE | = 0 |

### 第 6 步：最终复核

在认为模型完成前：
- 切换所有情景，确保每种情况下均通过检查。
- 解决或说明所有 #REF!, #DIV/0!, #VALUE!, #NAME? 错误。
- 确认所有输入格已填充（搜索占位符）。
- 验证所有标签页的单位（Units）保持一致。

### 1. 三张报表分别是什么——"公司的三张体检报告"

如果你去医院体检，医生会给你三张最重要的报告：**血常规**（看各项指标是否正常）、**CT 扫描**（看你身体的"结构"是否健康）、**心电图**（看你的血液循环是否通畅）。一家公司的财务报告也是三张表：

| 报表 | 中文 | 像什么 | 回答的问题 |
|------|------|--------|------------|
| **Income Statement (IS)** | 利润表/损益表 | 血常规 | "公司赚了多少钱？" |
| **Balance Sheet (BS)** | 资产负债表 | CT 扫描 | "公司有多少家底？" |
| **Cash Flow Statement (CF)** | 现金流量表 | 心电图 | "公司的钱进进出出健康吗？" |

**打一个最直观的生活比方——你的个人财务状况**：

- **利润表（IS）** = 你的"月薪单"
  - 收入：工资 $10,000
  - 支出：房租 $3,000、吃饭 $2,000、交通 $500
  - 净利润：$4,500 → "这个月攒了多少钱？"

- **资产负债表（BS）** = 你的"全部身家清单"
  - 资产：银行存款 $50,000、房子 $500,000
  - 负债：房贷 $300,000
  - 净资产（权益）：$250,000 → "你值多少钱？"

- **现金流量表（CF）** = 你的"银行流水"
  - 工资入账：$10,000（经营性现金流）
  - 买家具：-$2,000（投资性现金流）
  - 还房贷：-$3,000（融资性现金流）
  - 净变化：+$5,000 → **"银行账户这个月多了还是少了？"**

**关键洞察：净利润 ≠ 银行存款变化**
- 你月薪 $10,000（利润表），但可能你花了 $15,000 买了台新电脑（投资性流出），银行账户反而少了 $5,000。
- 这就是为什么需要三张表一起看——只看利润表会"被骗"。

---

### 2. 三表如何勾稽——"一根链条上的三个齿轮"

三张报表不是各说各的，而是通过公式**互相链接**。改一个数字，三张表都会变。

**核心链接1：净利润（IS）→ 留存收益（BS）→ 现金流量表（CF）**
```
利润表底线 → 净利润 $100M
    ↓ 进入
资产负债表 → 留存收益增加 $100M（前提是不分红）
    ↓ 同时也是
现金流量表的起点（CF 的第一行就是净利润）
```

**核心链接2：折旧（IS）→ 固定资产（BS）→ 现金流加回（CF）**
```
利润表 → 折旧费用 $10M（减少利润）
    ↓
资产负债表 → 固定资产减少 $10M（累计折旧增加）
    ↓
现金流量表 → 加回 $10M（因为折旧不是真金白银的支出，所以要从现金流角度"加回来"）
```

**核心链接3：存货变动（BS）→ 营运资本变动（CF）**
```
资产负债表 → 期末存货比期初多了 $5M（存货增加）
    ↓
现金流量表 → 营运资本变动 -$5M（钱"卡"在存货里了，现金流减少）
```

**为什么必须配平（Balance）**：
- 资产负债表必须满足：**资产 = 负债 + 权益**
- 如果不等，说明某个地方有错误——净利润没有正确进入留存收益、或者折旧没有正确减少固定资产、或者现金没有正确反映在资产负债表上。
- **配平是财务模型的"及格线"**——不平的模型不应该被任何人使用。

**真实案例**：2012 年，摩根大通（JPMorgan）的"伦敦鲸"事件——一个交易员的模型里，VaR（风险价值）计算错了，但没有人检查模型的勾稽关系。当问题暴露时，摩根大通已经损失了 $6.2B（62 亿美元）。

### 3. 利润表深度——"不是所有收入都是真收入"

利润表看起来很简单（收入 - 成本 = 利润），实际上每一行都有"坑"。

**收入（Revenue）≠ 现金到账**
- 很多公司采用"权责发生制（Accrual Accounting）"——货发出去了就确认收入，不管客户有没有付钱。
- 这就是为什么利润表上的营收漂亮，但现金流量表上可能一分钱没收到。
- **典型陷阱**：应收账款暴增但营收增长 → 公司可能在"送"货，客户不付钱。

**毛利率 vs 净利率——"从毛利润到净利润，中间有多少'剥皮'？"**
```
收入：$1,000M
  - 营业成本（COGS）：$600M
  = 毛利润：$400M → 毛利率 40%（衡量"生产/采购效率"）
  
  - 销售管理费用（SG&A）：$200M
  - 折旧摊销（D&A）：$50M
  = 营业利润（EBIT）：$150M → 营业利润率 15%（衡量"经营效率"）
  
  - 利息：$20M
  = 税前利润：$130M
  
  - 税（20%）：$26M
  = 净利润：$104M → 净利率 10.4%（"落到口袋里"的比例）
```

**看不同利润数字的含义**：
- **毛利率** 告诉你"这个生意的本质有多赚钱"。高端奢侈品毛利率 60-80%，超市只有 2-5%。
- **营业利润率** 告诉你"管理层效率如何"。花多少在销售和管理上。
- **净利率** 告诉你"最后落到股东口袋里的有多少"——扣了利息和税。

### 4. 资产负债表深度——"左边和右边为什么永远相等？"

资产负债表的核心等式：**资产 = 负债 + 权益**

**打比方——借钱开奶茶店**：
- 你投入自己的积蓄 $30,000（权益）
- 向银行借了 $70,000（负债）
- 总共有 $100,000 可以支配

用这 $100,000 买了设备、装修、原材料（所有这些加起来 = 资产 $100,000）

**左边（资产）= 右边（负债 + 权益）**，永远相等。

**资产的分类**：
| 类型 | 例子 | 特点 |
|------|------|------|
| 流动资产 | 现金、应收账款、存货 | 1 年内能变现 |
| 长期资产 | 厂房、设备、专利 | 1 年以上才能变现 |

**负债的分类**：
| 类型 | 例子 | 特点 |
|------|------|------|
| 流动负债 | 应付账款、短期借款 | 1 年内要还 |
| 长期负债 | 长期借款、债券 | 1 年以上才到期 |

**流动比率（Current Ratio）**：流动资产 / 流动负债
- > 2.0 → 公司短期偿债能力很强（好）
- < 1.0 → 公司可能还不上短期债务（危险！）
- 但 > 5.0 也可能意味着资产利用效率太低（现金躺在账上不投资）

**资产负债表的三个关键验证**：
1. **留存收益钩稽**：期末留存 = 期初留存 + 净利润 - 分红
2. **现金钩稽**：BS 现金 = CF 期末现金
3. **固定资产钩稽**：期末 PP&E = 期初 PP&E + CapEx - 折旧

### 5. 现金流量表深度——"Cash is King（现金为王）"

利润表和资产负债表都可能被会计手法"美化"（比如调整折旧年限、改变收入确认方式），但**现金流量表很难造假**——银行账户里的钱多没多、少没少，是客观事实。

**现金流的三个来源**：

| 现金流类型 | 是什么 | 健康的公司应该什么样 |
|-----------|--------|---------------------|
| **经营性现金流（Operating CF）** | 卖产品/服务赚来的钱 | 正数，且大于净利润 |
| **投资性现金流（Investing CF）** | 买设备/收购/卖资产 | 通常是负数（在投资未来） |
| **融资性现金流（Financing CF）** | 借钱/还钱/发股票 | 可正可负 |

**判断公司质量的黄金法则**：
- **好公司**：经营性现金流 > 净利润（说明利润质量高，真金白银进来了）
- **危险公司**：经营性现金流 < 净利润（利润是"纸面富贵"，钱没进来）
- **快要死的公司**：经营性现金流为负，只能靠借钱/发股票活着

**经典案例——安然（Enron）**：
2001 年，安然的利润表漂亮得不得了——营收 $100B+，华尔街追捧。但如果你看现金流量表，经营性现金流是负的——公司赚的钱根本不存在，全是会计造假。安然最终在 2001 年 12 月破产，成为美国历史上最大的破产案之一。

**折旧为什么在现金流量表被"加回来"**：
- 利润表上，折旧是一项费用（减少了 $10M 利润）
- 但折旧不是真正的现金支出——你买设备的时候已经花过钱了，折旧只是把那次支出分摊到每年
- 所以计算现金流时，要把折旧"加回来"：经营性现金流 = 净利润 + 折旧 + 其他非现金调整

### 6. 情景分析——"如果……会怎样？"

金融模型从来不是一个"答案"，而是一个"范围"。没有人能准确预测未来 5 年的营收增长率、利润率和资本支出。所以，专业分析师会构建多个情景：

| 情景 | 营收增长率 | 利润率假设 | 思想实验 |
|------|-----------|-----------|---------|
| **乐观（Bull）** | 15%/年 | 持续扩张 | "如果打败所有竞争对手" |
| **基准（Base）** | 8%/年 | 保持稳定 | "最可能发生的情况" |
| **悲观（Bear）** | 2%/年 | 持续萎缩 | "如果经济衰退" |

**情景分析的价值**：
- 不是预测"精确的未来"，而是界定"可能性范围"
- 如果悲观情景下公司也值 $50/股，而股价现在是 $40/股 → 很安全的投资
- 如果乐观情景下公司才值 $15/股，而股价现在是 $30/股 → 非常危险

---

### 给小白的一句话

> **三表模型是金融分析的"地基"——利润表告诉你"公司赚了多少"，资产负债表告诉你"公司有多少家底"，现金流量表告诉你"真金白银流向哪里"。三张表通过净利润、折旧、营运资本等链接点互相勾稽，资产负债表配平（资产 = 负债 + 权益）是模型的"及格线"。净利润 ≠ 银行现金变化——这就是为什么需要三张表一起看。现金流量表最诚实（Cash is King），经营性现金流是否大于净利润是判断利润质量的"黄金标准"。安然（Enron）的利润表完美但现金流为负，最终破产——这告诉我们永远要"三表一起看，不看单表做判断"。**
