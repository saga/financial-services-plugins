# 竞态格局图谱

构建一套完整的竞争分析演示文稿（PPT）。这分为两个阶段：先收集需求并获得大纲确认，随后开始制作。

## 环境检查

此技能可同时在 PowerPoint 插件和聊天界面中使用。开始前请确认当前环境——虽然操作机制不同，但工作流程一致：

- **插件环境** —— 演示文稿已实时打开，直接在其中构建幻灯片。
- **聊天环境** —— 生成一个 `.pptx` 文件（或在用户上传的文件基础上构建）。

以下内容适用于上述两种情况。

## 第一阶段 — 确定分析范围

“竞争分析”对不同的人意味着不同的含义。在进行任何研究或制作幻灯片之前，请使用 `ask_user_question` 来锁定用户的真实需求。切勿凭空猜测——一份包含 20 页同业基准比对的报告和一份 5 页的市场地图都属于“竞争分析”，但其形式完全不同。

尽量在一个回合内收集完毕（该工具最多可提 4 个问题）：

- **范围** —— 是以单一目标公司为核心并环绕其竞对？还是多公司并列，不设特定主角？
- **竞对集合** —— 包含哪些公司？如果用户指名了某些公司，请务必使用这些公司。如果用户说“老样子的那几家”，请提出一组名单供其确认。
- **受众与深度** —— 是给业内人士的快速简报，还是给门外汉的全面入门指南？这将决定你是否需要包含市场规模、行业经济学和历史背景，还是可以直接跳到对比环节。
- **投资背景** —— 是否需要包含乐观/中性/悲观（Bull/Base/Bear）的情景分析和信号指示？这属于下文的第 9 步；如果只是战略回顾而非投资逻辑分析，则可跳过。

如果用户上传了包含竞对数据的 Excel/CSV，请在提取数据前确认各列对应的指标。**数据源保真度**至关重要：请直接使用给定值，不要重新计算或重新取整。

## 第二阶段 — 提纲、确认、再构建

**在大纲获得批准前，不要创建幻灯片。** 提出幻灯片标题和单行内容要点，呈交给用户并获得许可。一份竞争分析报告通常包含 10-20 页环环相扣的内容——因为第 4 页出错而大规模重制成本极高。大纲是低成本迭代的最佳环节。

在提交大纲时，使用 `ask_user_question` 咨询结构性决策非常有效：例如选用哪种定位可视化图表（2×2 矩阵 / 雷达图 / 层级图 —— 见下文第 5 步），以及如何对竞对进分类（按商业模式 / 细分市场 / 市场姿态 —— 见第 4 步）。这些审美和逻辑判断通常取决于用户的个人偏好。

## 准则 — 贯穿全程

### 指令忠实度

当用户明确具体要求时，那是必须遵守的规定，而非建议：
- **幻灯片标题与章节名称** —— 字词必须完全一致。如果用户说“概览与竞争范围”，不要自行改写为“FY2024 竞争格局”。
- **图表 vs. 表格** —— 二者不可互换。“嵌入式图表”意味着带数据标签的真实图表对象，而非美化后的表格。
- **完整的数据序列** —— 如果列出了 7 个竞对，请包含全部 7 个。如果时间跨度是 2015-2025 年，请包含每一年。
- **精确数值与比例** —— “比 DoorDash 高 4:1，比 Lyft 高 8:1”意味着必须使用这些比例，而非简写为“7.6 倍于 Lyft”。

### 资料质量（当来源冲突时）

1. 10-K / 年度报告（经审计）
2. 业绩电话会 / 投资者简报（管理层评论）
3. 卖方研究报告（分析师预测，对分析私有公司规模非常有用）
4. 行业报告（麦肯锡、高德纳 —— 市场规模、趋势）
5. 新闻（仅限近期动态；需根据核心资料核实）

### 数据可比性

- 所有竞对指标应来自同一财年；若有例外需明确标记（如“FY24” vs “H1 2024”）。
- 各竞对之间的指标定义应保持一致。
- 国际数据统一折算为美元；注明汇率和日期。
- 缺失数据用“-”或“N/A”表示，预估值加注“[E]”标记 —— 严禁留白。
- 每个数据都必须注明出处："[公司] [文件名称] ([日期])"。

### 设计规范

- **幻灯片标题应是洞察，而非标签。** 例如“规模领先者正与垂直玩家拉开差距”，而非“竞争分析”。
- **信号指示语需量化。** 例如“利润率低于 40%”，而非“利润率下滑”。
- **评级需显示实际数值。** 例如“●●● $160B”，而非简单的“●●●”。
- **图表必须是真实的图表对象** —— 而非伪装成图表的文字表格。

**排版** —— 必须明确设置，不要依赖默认值：
- 幻灯片标题：28-32pt 粗体
- 章节标题：18-20pt 粗体
- 正文：14-16pt（严禁低于 14pt）
- 表格文字：14pt
- 来源/脚注：14pt，灰色
- 全文相同类型的元素大小应保持一致。

**图表：**
- 图例置于图表边界内，不要悬浮在绘图区上方。
- 饼图（≤6 个切片）使用右侧图例，折线/柱状图（≤4 个序列）使用底部图例。
- 超过 6 个序列时 -> 拆分为多个图表或改用表格。
- 饼图需在切片上直接显示百分比，而不仅仅在图例中显示。

**表格：**
- 浅灰色表头行，粗体。
- 数字右侧对齐，文字左侧对齐。
- 设置足够的单元格内边距，确保文字不触碰边框。

**颜色：** 最多使用 2-3 种颜色。色调应柔和 —— 藏青色、灰色，外加一种对比色。跨页面的色彩含义应保持一致。

### 强制要求 vs. 灵活空间

| 必须严格遵守 (Always) | 可根据情况调整 (Case-by-case) |
|---|---|
| 用户指定的标题/章节文字 | 用户未指定时的创意标题 |
| 指定用图表则用图表，指定用表则用表 | 未指定时的可视化类型选择 |
| 包含用户列出的每一个竞对/数据点 | 未指定时的竞对选取数量 |
| 用户指定的精确数值 | 未指定精度时的取整处理 |
| 标题文字不溢出、不重叠 | 竞对分类的数量 |
| 页面元素不留残缺 | 比较维度的选取 |

## 分析流程

### 第 0 步 — 定义行业关键指标

在开始之前：该行业的核心运行指标是哪 3-5 个？在各竞对比对中需始终如一地使用。

| 行业 | 关键指标 |
|---|---|
| SaaS | ARR（年度经常性收入）, NRR（净留存率）, CAC 回本周期, LTV/CAC, 40原则 |
| 支付 | GPV（交易总额）, 变现率, 附加率, 交易利润率 |
| 电商平台 | GMV（商品交易总额）, 变现率, 买卖家比例, 复购率 |
| 零售 | 同店销售增长, 存货周转率, 每平方英尺销售额 |
| 物流 | 货运量, 单位成本, 准时交付率, 产能利用率 |

若行业未列出 —— 请挑选投资者和运营商公认的基准指标。

### 第 1 步 — 市场背景

规模、增长速度、驱动因素、负面阻力。必须注明来源。

正确做法：“嵌入式支付在 2024 年规模达 800-1000 亿美元，复合年均增长率为 20-25% (麦肯锡 2024)”
错误做法：“该市场规模庞大且增长迅速”

### 第 2 步 — 行业经济学

描绘价值流向。方法视行业结构而定：
- **垂直结构** —— 产业链各环节及其典型利润率。
- **平台/网络结构** —— 生态系统参与者及其间的价值交换。
- **碎片化结构** —— 行业整合动态、不同规模下的利润率差异。

### 第 3 步 — 目标公司概况

```
| 指标 | 数值 |
|---|---|
| 营收 | $4.96B |
| 增长 | 同比 +26% |
| 毛利率 | 45% |
| 盈利能力 | $373M 调整后 EBITDA |
| 客户数量 | 134K |
| 留存率 | 92% |
| 市场份额 | 约 15% |
```

跨行业/多业务公司需增加业务拆分：

```
| 业务部门 | 营收 | 营收同比 | 营收占比 | EBITDA | EBITDA同比 | 利润率 |
|---|---|---|---|---|---|---|
| A 部门 | $25.1B | +26% | 57% | $6.5B | +31% | 26% |
| B 部门 | $13.8B | +31% | 31% | $2.5B | +64% | 18% |
| C 部门 | $5.1B | -2% | 12% | -$74M | -16% | -1% |
| 总计 | $44.0B | +18% | 100% | $6.5B* | - | 15% |
```
*必要时注明公司总部成本

### 第 4 步 — 竞对分布图

按最契合的视角进行分组（若用户未指定，这是使用 `ask_user_question` 的好时机）：
- 按商业模式 —— 平台型 / 垂直型 / 水平型
- 按细分市场 —— 企业级 / 中小企业 / 消费者
- 按竞争姿态 —— 直接竞争 / 邻近领域 / 新兴威胁
- 按出身背景 —— 现任巨头 / 颠覆者 / 新入局者

### 第 5 步 — 定位可视化

| 类型 | 使用场景 |
|---|---|
| 2×2 矩阵 | 存在两个主导竞争因素时 |
| 雷达图/蜘蛛图 | 多维度综合比较时 |
| 层级图 | 存在明显的战略梯队时 |
| 价值链地图 | 垂直整合型行业 |
| 生态系统图 | 平台化市场 |

具体行业的 2×2 坐标轴对，请参考 `references/frameworks.md`。

### 第 6 步 — 竞对深度分析

每个竞对对应两张表。

**量化指标：**
```
| 指标 | 数值 |
|---|---|
| 营收 | $X.XB |
| 增长 | 同比 +XX% |
| 毛利率 | XX% |
| 市值 | $X.XB |
| 盈利能力 | $XXXM EBITDA |
| 客户数量 | XXK |
| 留存率 | XX% |
| 市场份额 | 约 XX% |
```

**定性评估：**
```
| 类别 | 评估内容 |
|---|---|
| 业务概况 | 对方是做什么的（1句话简述） |
| 竞争优势 | 2-3 个要点 |
| 竞争劣势 | 2-3 个要点 |
| 发展战略 | 当前的优先级重点 |
```

### 第 7 步 — 横向比较分析

```
| 维度 | 公司 A | 公司 B | 公司 C |
|---|---|---|---|
| 规模规模 | ●●● $160B | ●●○ $45B | ●○○ $8B |
| 增长潜力 | ●●○ +26% | ●●● +35% | ●●○ +22% |
| 利润水平 | ●●○ 7.5% | ●○○ 3.2% | ●●● 15% |
```

### 第 8 步 — 战略背景

并购交易（估值倍数、收购动机）、合作趋势、募资模式、监管变动。并购交易表格式参阅 `references/schemas.md`。

### 第 9 步 — 综合总结

**护城河评估** —— 针对以下维度将各竞对评定为 强 / 中 / 弱：

| 护城河 | 评估重点 |
|---|---|
| 网络效应 | 用户/供应商飞轮效应的强度；双边 vs 同边效应 |
| 转换成本 | 技术集成的深度、合同绑定、用户行为习惯 |
| 规模经济 | 随着规模扩大而产生的单位成本优势；最低效率规模 |
| 无形资产 | 品牌影响力、专有数据、监管许可/牌照、专利 |

**必须包含的综合要素：**
- 持久优势（难以复制）—— 对应护城河类别。
- 结构性脆弱点（难以修复）。
- 现状 vs. 未来轨迹。

**针对投资决策场景**（若第一阶段确认不需要则跳过）：

```
| 情景 | 概率 | 核心驱动因素 |
|---|---|---|
| 牛市 (Bull) | 30% | 市场份额超预期增长, 利润率扩张 |
| 中性 (Base) | 50% | 维持当前发展轨迹 |
| 熊市 (Bear) | 20% | 竞争压力加剧, 利润率被压缩 |
```

## 质量核查清单

结束前的最后检查：

**指令忠实度**
- 幻灯片标题与用户指定的文字逐字匹配。
- 图表和表格的使用完全遵循用户要求。
- 用户列出的每个竞对、年份和数据点均无遗漏。
- 精确数值和格式完全一致。

**数据一致性**
- 数据直接从源文件提取，未经重新计算。
- 同一指标在不同页面出现时数值一致。
- 小数位数与原始资料保持一致。

**排版设计**
- 标题不溢出。
- 元素之间无重叠。
- 所有文字都在框内，无截断。

**内容严谨性**
- 每个数据点都有来源标注。
- 指标所在的会计周期一致（或有明确标记）。
- 标题体现的是“洞察”而非单纯的“陈述”。
- 图表是真正的对象，数据可编辑。

对每一页幻灯片进行标准的视觉校验 —— 这能发现一些在读取 XML 代码时被忽略的元素重叠、溢出或对比度过低的文字问题。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **EBITDA（息税折旧摊销前利润）**：把它想象成一家公司通过核心业务真正“赚到手”的闲钱。我们在对比不同公司时喜欢用它，因为它剔除了税收优惠或不同的会计摊销策略对账面数字的“干扰”。
> 2. **护城河 (Moat)**：这是股神巴菲特最爱的词。就像城堡外宽阔的水沟，好的商业“护城河”能让竞争对手很难抢走你的客户。常见的有：品牌（让你愿意为logo多付钱）、转换成本（让你觉得换个App太麻烦）。
> 3. **40原则 (Rule of 40)**：这是判断一家软件（SaaS）公司好坏的高端直觉。把增长率和利润率加起来，如果超过40%，那就是妥妥的优等生。它意味着公司在疯狂增长的同时，还能保持不错的赚钱效率。
> 4. **CAGR（复合年均增长率）**：如果你想知道一个市场或公司在几年里平均每年长高了多少，就看这个。它不看每年的颠簸，只看起始点和终点的“平均速度”。
> 5. **牛市/中性/熊市情景 (Bull/Base/Bear Case)**：分析师对自己预测的三种“剧本”。牛市是“一切顺利”的最美幻象，熊市是“诸事不顺”的最坏打算，中性则是最可能发生的现实。
