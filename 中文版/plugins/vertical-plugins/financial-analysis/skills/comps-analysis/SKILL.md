# 可比公司分析 (Comparable Company Analysis)

## ⚠️ 关键：数据源优先级（请先阅读）

**请始终遵循以下数据源层级：**

1.  **首选：检查 MCP 数据源** - 如果可以使用 S&P Kensho MCP、FactSet MCP 或 Daloopa MCP，请直接使用这些工具获取财务和交易信息。
2.  **若有上述 MCP 数据源，切勿使用网络搜索。**
3.  **仅在 MCP 不可用时：** 才使用彭博终端（Bloomberg Terminal）、SEC EDGAR 申报文件或其他机构级来源。
4.  **严禁将网络搜索作为主要数据源** - 网络数据缺乏机构级分析所需的准确性、审计追踪和可靠性。

**原因：** MCP 来源提供经过验证的机构级数据并附带正确的引用。网络搜索结果对于财务分析而言可能已经过时、不准确或不可靠。

---

## 概述
本技能旨在指导 Claude 构建机构级可比公司分析，将运营指标、估值倍数和统计基准相结合。输出结果是一个结构化的 Excel/电子表格，旨在通过同行对比辅助投资决策。

**参考材料与上下文应用：**

`examples/comps_example.xlsx` 中提供了一个可比公司分析示例。在应用本技能目录下的示例文件时，请灵活处理：

**示例的正确用途：**
- 了解结构层级（各部分的衔接流程）
- 掌握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰的表头、透明的公式、审计追踪）

**示例的错误用途：**
- 机械照搬格式或指标
- 不考虑具体情况直接复制布局
- 不分受众群体套用相同的视觉风格

**在开始前请始终自问：**
1. **“您是否有偏好的格式，还是由我来调整模板风格？”**
2. **“受众是谁？”**（投资委员会、董事会演示、快速参考、详细备忘录）
3. **“核心问题是什么？”**（估值、增长分析、竞争地位、效率）
4. **“上下文背景是什么？”**（并购评估、投资决策、行业基准分析、绩效回顾）

**根据具体情况进行调整：**
- **行业背景**：大型科技股需要的指标与新兴 SaaS 初创公司完全不同。
- **特定行业需求**：及早添加相关指标（例如科技行业的云 ARR、企业客户数、开发者生态系统）。
- **公司熟悉度**：知名公司可能不需要太多背景介绍，应更多关注增量分析（Delta Analysis）。
- **决策类型**：并购分析与日常投资组合监控的侧重点不同。

**核心原则：** 遵循模板原则（结构清晰、统计严谨、公式透明），但在执行时根据背景灵活变化。目标是产出机构级质量的分析，而非仅仅是外表像机构级的模板。

用户提供的示例和明确偏好始终优先于默认设置。

## 核心哲学
**“先构建正确的结构，再让数据说话。”**

从迫使你进行战略思考的表头开始，输入干净的数据，构建透明的公式，让统计结果自然呈现。一份优秀的 Comps 表格应当让没有参与构建的人也能一目了然。

## ⚠️ 关键：公式重于硬编码 + 逐步验证

**环境分析 —— Office JS vs Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS (`Excel.run(async (context) => {...})`)。通过 `range.formulas = [["=E7/C7"]]` 写入公式，而非 `range.values`。无需单独的重算步骤 —— Excel 会原生处理。使用 `range.format.*` 设置颜色/字体。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。写入 `cell.value = "=E7/C7"`（公式字符串）。
- 无论哪种方式，原则一致 —— 只是 API 调用不同。
- **Office JS 合并单元格陷阱：** 切勿先调用 `.merge()` 再对合并后的区域设置 `.values`（这会抛出 `InvalidArgument` 错误 —— 区域仍报告合并前的维度）。正确的做法是仅向左上角单元格写入值，然后再进行合并和格式化：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式重于硬编码：**
- 每一个派生数值（利润率、倍数、统计数据）必须引用输入单元格的 Excel 公式 —— 严禁粘贴预先计算好的数字。
- 使用 Python/openpyxl 构建表格时：写入 `cell.value = "=E7/C7"`（公式字符串），而非 `cell.value = 0.687`（计算结果）。
- 唯一的硬编码值应为原始输入数据（收入、EBITDA、股价等），且每一个输入值都应附带注明来源的单元格批注。
- **原因**：当输入值改变时，模型必须能自动更新。硬编码的利润率是一个随时可能爆发的隐藏错误。

**与用户逐步验证：**
- 设置好结构后 → 在填充数据前向用户展示表头布局。
- 输入原始数据后 → 在构建公式前展示输入块，并确认数据来源/期间。
- 构建运营指标公式后 → 在进入估值部分前展示计算出的利润率并进行合理性检查。
- 构建估值倍数后 → 在添加统计数据前展示倍数并确认其合理性。
- **不要**等构建完整个表格后再演示 —— 通过确认每个部分及早发现错误。

## 第一部分：文档结构与设置

### 表头块（第 1-3 行）
```
第 1 行：[分析标题] - 可比公司分析
第 2 行：[带有股票代码的公司列表] • [公司 1 (TICK1)] • [公司 2 (TICK2)] • [公司 3 (TICK3)]
第 3 行：截至 [日期] | 除每股金额和比率外，所有金额单位均为 [百万/十亿美元 USD]
```

**重要性：** 立即建立上下文。任何打开此文件的人都能立刻明白分析对象、创建时间以及如何解读数字。

### 视觉规范标准（可选 - 用户偏好和上传的模板始终优先）

**关键：以下仅为建议默认设置。请始终优先考虑：**
1. 用户的明确格式偏好
2. 任何上传模板文件中的格式
3. 公司/团队风格指南
4. 这些默认值（仅在无其他指导时使用）

**建议字体与排版：**
- **字体**：Times New Roman（专业、易读，行业标准）
- **字号**：数据单元格为 11pt，表头为 12pt
- **加粗**：部分标题（Section headers）、公司名称、统计标签

**默认颜色与阴影 —— 专业蓝/灰调（极简为佳）：**
- **保持克制** —— 仅使用蓝色和灰色。切勿引入绿色、橙色、红色或多种强调色。一份整洁的 Comps 表格总共使用 3-4 种颜色。
- **部分标题**（例如 “运营统计与财务指标”）：
  - 深蓝色背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色加粗文字
  - 整行跨列着色
- **列标题**（例如 “公司”、“收入”、“利润率”）：
  - 浅蓝色背景（`#D9E1F2` 或类似的淡蓝色）
  - 黑色加粗文字
  - 居中对齐
- **数据行**：
  - 公司数据背景为白色
  - 公式使用黑色文字；硬编码输入使用蓝色文字
- **统计行**（最大值、75 分位数等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文字，标签左对齐
- **这就是全部配色**：深蓝 + 浅蓝 + 浅灰 + 白。除非用户模板另有规定，否则不要使用其他颜色。

**建议格式惯例：**
- **小数精度**：
  - 百分比：1 位小数 (12.3%)
  - 倍数：1 位小数 (13.5x)
  - 金额：无小数，使用千位分隔符 (69,632)
  - 以百分比显示的利润率：1 位小数 (68.7%)
- **边框**：不设边框（界面整洁极简）
- **对齐**：所有指标居中对齐，确保整齐划一
- **单元格尺寸**：所有列宽应整齐一致，所有行高应保持一致（营造专业的网格感）

**注**：如果用户提供了模板文件或指定了不同的格式，请改用用户的格式。

## 第二部分：运营统计与财务指标

### 核心列（以此开始）
1.  **公司 (Company)** - 命名格式保持一致
2.  **收入 (Revenue)** - 规模指标（根据情况使用 LTM、季度或年度数据）
3.  **收入增长率 (Revenue Growth)** - 同比百分比变化
4.  **毛利润 (Gross Profit)** - 收入减去销售成本
5.  **毛利率 (Gross Margin)** - 毛利润/收入（基本盈利能力）
6.  **EBITDA** - 息税折旧摊销前利润
7.  **EBITDA 利润率 (EBITDA Margin)** - EBITDA/收入（运营效率）

### 可选补充（根据行业/目的选择）
- **季度 vs LTM** - 如果存在季节性影响，则两者均包含
- **自由现金流 (Free Cash Flow)** - 适用于资本密集型或 SaaS 业务
- **FCF 利润率** - FCF/收入（现金生成效率）
- **净利润 (Net Income)** - 适用于成熟且盈利的公司
- **营业利润 (Operating Income)** - 适用于折旧摊销（D&A）差异较大的业务
- **资本支出指标 (CapEx metrics)** - 适用于重资产行业
- **40原则 (Rule of 40)** - 专门用于 SaaS（增长率% + 利润率%）
- **FCF 转化率 (FCF Conversion)** - 用于收益质量分析（高级）

### 公式示例（以第 7 行为例）
```excel
// 核心比率 - 这些始终通过计算得出
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率 - 若相关则包含
FCF 利润率: =[FCF]/[Revenue]
净利率: =[Net Income]/[Revenue]
40原则: =[增长率%]+[FCF 利润率%]
```

**黄金法则：** 每个比率都应该是 [某指标] / [收入] 或 [表内指标] / [表内指标]。保持简单。

### 统计块（位于公司数据下方）

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留出一个空行以实现视觉隔离]
- 最大值 (Maximum): =MAX(B7:B9)
- 75 分位数 (75th Percentile): =QUARTILE(B7:B9,3)
- 中位数 (Median): =MEDIAN(B7:B9)
- 25 分位数 (25th Percentile): =QUARTILE(B7:B9,1)
- 最小值 (Minimum): =MIN(B7:B9)
```

**需要统计数据的列（可比指标）：**
- 收入增长率 %、毛利率 %、EBITDA 利润率 %、EPS
- EV/Revenue、EV/EBITDA、P/E、股息率 %、Beta

**不需要统计数据的列（规模指标）：**
- 收入、EBITDA、净利润（绝对规模随公司规模变化）
- 市值、企业价值（不同规模公司之间不可比）

**注**：在公司数据和统计行之间添加一个空行。不要添加 “行业统计” 或 “估值统计” 的标题行。

**分位数的意义：** 它们展示了分布情况，而不仅仅是平均值。75 分位倍数能告诉你 “优质” 公司的交易价格。

## 第三部分：估值倍数与投资指标

### 核心估值列（以此开始）
1.  **公司** - 顺序与运营指标部分一致
2.  **市值 (Market Cap)** - 当前市场估值
3.  **企业价值 (Enterprise Value)** - 市值 ± 净债务/现金
4.  **EV/Revenue** - 每一美元销售额的市场定价
5.  **EV/EBITDA** - 每一美元收益的市场定价
6.  **市盈率 (P/E Ratio)** - 价格相对于净收益的比率

### 可选估值指标（根据背景选择）
- **FCF 收益率 (FCF Yield)** - FCF/市值（用于关注现金的分析）
- **PEG 比率** - P/E / 增长率（用于成长型公司）
- **市净率 (Price/Book)** - 市值 vs 账面价值（用于重资产业务）
- **ROE/ROA** - 回报指标（用于盈利能力对比）
- **收入/EBITDA 复合增长率 (CAGR)** - 历史增长率（用于趋势分析）
- **资产周转率 (Asset Turnover)** - 收入/资产（用于运营效率分析）
- **资产负债率 (Debt/Equity)** - 杠杆率（用于资本结构分析）

**关键原则：** 包含 3-5 个对你所在行业至关核心的倍数。平衡信息的全面性与简洁性。

### 公式示例
```excel
// 核心倍数 - 务必包含这些
EV/Revenue: =[企业价值]/[LTM 收入]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
市盈率 (P/E): =[市值]/[净利润]

// 可选倍数 - 若数据可用则包含
FCF 收益率: =[LTM FCF]/[市值]
PEG 比率: =[P/E]/[增长率 %]
```

### 交叉引用规则
**关键：** 估值倍数必须引用运营指标部分。严禁重复输入相同的原始数据。如果收入在 C7，那么 EV/Revenue 公式必须引用 C7。

### 统计块
结构与运营指标部分相同：为每个指标计算最大值、75 分位、中位数、25 分位和最小值。在公司数据和统计数据之间留出一个空行。不要添加 “估值统计” 标题行。

## 第四部分：附注与方法论文档

### 必要组成部分

**数据来源与质量：**
- 数据来源？（S&P Kensho MCP, FactSet MCP, Daloopa MCP, Bloomberg, SEC 申报文件）
- 覆盖期间？（2024 年第四季度，经审计数据）
- 如何验证？（与 10-K/10-Q 进行核对）
- 注：为了获得更高的准确性和追溯性，请优先使用 MCP 数据源。

**关键定义：**
- EBITDA 计算方法（毛利润 + 折旧摊销，或 营业利润 + 折旧摊销）
- 自由现金流公式（经营性现金流 - 资本支出）
- 特殊指标说明（40原则、FCF 转化率）
- 时间周期定义（LTM、CAGR 计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净债务）
- 使用了什么增长率？（历史复合增长率、预测数据）
- 是否进行了任何调整？（剔除一次性项目、利润率规范化）

**分析框架：**
- 投资逻辑是什么？（云服务/SaaS 效率）
- 哪些指标最重要？（获现能力、资本效率）
- 读者应如何解读统计数据？（分位数提供背景参考）

## 第五部分：选择正确的指标（决策框架）

### 从“我要回答什么问题？”开始

**“哪家公司被低估了？”**
→ 重点关注：EV/Revenue、EV/EBITDA、P/E、市值
→ 跳过：运营细节、增长指标

**“哪家公司的效率最高？”**
→ 重点关注：毛利率、EBITDA 利润率、FCF 利润率、资产周转率
→ 跳过：规模指标、绝对金额

**“哪家公司增长最快？”**
→ 重点关注：收入增长率 %、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆率

**“哪家公司的获现能力最强？”**
→ 重点关注：FCF、FCF 利润率、FCF 转化率、资本支出强度
→ 跳过：EBITDA、P/E 倍数

### 行业特定指标选择

**软件/SaaS：**
必选：收入增长率、毛利率、40原则
可选：ARR、净金额留存率 (NDR)、CAC 回本周期
跳过：资产周转率、库存指标

**制造/工业：**
必选：EBITDA 利润率、资产周转率、资本支出/收入
可选：ROA、库存周转率、积压订单
跳过：40原则、SaaS 指标

**金融服务：**
必选：ROE、ROA、成本收入比 (Efficiency Ratio)、P/E
可选：净息差 (NIM)、拨备
跳过：毛利率、EBITDA（对于银行没有意义）

**零售/电子商务：**
必选：收入增长率、毛利率、库存周转率
可选：同店销售增长、获客成本 (CAC)
跳过：高额研发或资本支出指标

### “5-10 原则”

**5 个运营指标** - 收入、增长、2-3 个利润率/效率指标
**5 个估值指标** - 市值、企业价值、3 个倍数
**= 10 列总计** - 足以说明问题，又不至于多到让人抓不住重点。

如果你的指标超过 15 个，可能包含了不少噪音。请无情剔除。

## 第六部分：最佳实践与质量检查

### 开始之前
1.  **定义同行组** - 公司必须具备真正的可比性（商业模式、规模、地理位置相似）
2.  **选择正确的期间** - LTM 能平滑季节性波动；季度数据则展示趋势
3.  **预先统一单位** - 万元、亿元或百万、十亿的选择会影响全局
4.  **映射数据源** - 明确每个数字的出处

### 构建过程中
1.  **先输入所有原始数据** - 在编写公式前完成蓝色文字的填充
2.  **为所有硬编码输入添加单元格批注** - 右键单元格 → 插入批注 → 记录来源或假设

    **对于引用的数据，请准确注明出处：**
    - 示例：“Bloomberg Terminal - MSFT Equity DES, accessed 2024-10-02”
    - 示例：“Q4 2024 10-K 申报文件, 第 42 页, 项目 ‘总收入’”
    - 示例：“FactSet 共识预期，截至 2024-10-02”
    **尽可能包含超链接**：右键单元格 → 链接 → 粘贴 SEC 申报文件、数据源或报告的 URL。

    **对于假设，说明理由：**
    - 示例：“基于同行中位数假设 EBITDA 利润率为 15%，公司未披露该数据”
    - 示例：“估算企业价值 = 市值 + 5000 万美元净债务（源自 Q3 资产负债表，Q4 尚未发布）”
    - 示例：“前瞻 P/E 基于市场共识 EPS $3.45（12 位分析师预期的平均值）”

    **重要性**：实现审计追踪、数据验证、假设透明度并方便未来更新。
3.  **逐行构建公式** - 在继续下一步之前测试每个计算结果
4.  **在表头使用绝对引用** - $C$6 可锁定标题行
5.  **保持格式一致** - 百分比显示为百分比，而非小数
6.  **添加条件格式** - 自动突出显示异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA 利润率 > 净利率（根据定义这是必然的）
- **倍数合理性**：
    - EV/Revenue：通常为 0.5-20x（随行业变化极大）
    - EV/EBITDA：通常为 8-25x（在各行业间相对一致）
    - P/E：通常为 10-50x（取决于增长率）
- **增长与倍数的相关性**：高增长通常意味着高倍数
- **规模与效率的权衡**：大公司通常拥有更好的利润率（规模效益）

### 应避免的常见错误
❌ 在公式中混淆市值和企业价值
❌ 分子和分母使用不同的时间段（LTM vs 季度）
❌ 在公式中硬编码数字而非引用单元格
❌ **硬编码输入未附带注明来源或说明假设的批注**
❌ 在有条件时不提供指向 SEC 申报文件或数据源的超链接
❌ 包含太多没有明确目的的指标
❌ 包含不具可比性的公司（商业模式不同）
❌ 使用过时数据且未予披露
❌ 错误地计算百分比的平均值（应使用中位数）

### 1. Comps 的本质——"看看隔壁老王卖多少钱"

**可比公司分析（Comps）** 是三种主流估值方法中最直观的一种。它的核心逻辑非常朴素：

> 如果市场上和"你的公司"类似的其他公司，都以 EV/EBITDA = 12 倍的价格在交易，那你的公司大概率也值 12 倍。

**打一个生活中的比方**：

你想卖掉你的二手房（100 平米、朝南、中等装修）。你不知道该标多少钱。于是你打开房产 App，搜一下同一个小区、同样面积、同样户型的房子挂多少钱：
- 隔壁 301 挂了 500 万 → 500 万/100 平米 = 5 万/平米
- 楼上 502 挂了 520 万 → 5.2 万/平米
- 楼下 101 挂了 480 万 → 4.8 万/平米
- **中位数**：5 万/平米

那你的房子大概值 5万×100平米 = **500 万**。

**这就是 Comps 的本质**。唯一不同的是，给公司估值时用的"每平米价格"变成了"估值倍数"（如 EV/EBITDA、P/E），而房子的"面积"变成了公司的"财务指标"（如 EBITDA、净利润）。

**为什么 Comps 是最"接地气"的估值方法**：
- DCF（现金流折现）：需要预测未来 5-10 年的现金流，任何一个假设变了一点，结果就差几倍。像"算命"。
- Precedent Transactions（先例交易）：需要找过去发生过的类似收购案例，数据少、难找。像"考古"。
- Comps（可比公司）：直接看市场上活生生的交易价格，数据多、容易算。像"看房价"。

**但 Comps 的致命弱点是**：如果整个市场都在疯狂（比如 2021 年的科技股泡沫），那 Comps 会告诉你"所有公司都很贵，所以我们的目标公司也很值钱"——但泡沫破裂后，这些估值都会崩塌。这就是为什么 Comps 不能单独使用，必须和 DCF 等方法交叉验证。

---

### 2. 企业价值 vs 市值——"买公司到底要花多少钱？"

在做 Comps 时，最基础也最容易搞错的概念就是**企业价值（Enterprise Value / EV）**和**市值（Market Cap / Equity Value）**的区别。

**市值（Market Cap）** = 股价 × 总股数。这是"买下公司所有股票需要花的钱"。

**企业价值（EV）** = 市值 + 净负债。这是"真正买下这整家公司需要花的钱"。

**为什么 EV 比市值更准确？**

**打比方——买房**：
你想买一套 500 万的房子。
- **方案 A**：全款买 → 你花 500 万，房子归你。你的成本（EV）= 500 万。
- **方案 B**：房子还有 300 万房贷没还 → 你付 200 万给房主（市值），但还要承接 300 万的贷款。你真正的成本（EV）= 200 万 + 300 万 = 500 万。

**对于公司来说**：
- 市值（付给股东的钱）= 200 万（房主的权益）
- 净负债（你还要承接的债务）= 300 万（房贷）
- 企业价值（买下公司的真实成本）= 200 万 + 300 万 = 500 万

**EV 的计算公式**：
```
企业价值 = 市值 + 总负债 - 现金及现金等价物
         = 市值 + 净负债
```

为什么要减去现金？继续用买房打比方：如果房子本身附带了 50 万现金（存在一个你接手后可以用的账户里），那你实际的成本 = 500 万 - 50 万 = 450 万。公司的现金也是一样的道理——你买下公司后，公司账上的现金就是你的了，相当于"回血"。

**为什么 Comps 必须用 EV 倍数而不是市值倍数**：

假设两家公司：
- A 公司：市值 100 亿，零负债，账上 10 亿现金 → EV = 90 亿
- B 公司：市值 100 亿，负债 50 亿，账上 5 亿现金 → EV = 145 亿

如果只看市值，两家公司"一样贵"。但实际上 B 公司比 A 公司贵了 61%（145 亿 vs 90 亿），因为它背了更多的债。

**所以 Comps 中的 EV/EBITDA、EV/Revenue 用 EV，而 P/E（市盈率）用市值**——因为 P/E 只看股东的收益（净利润属于股东），而 EV/EBITDA 看整个公司的经营成果（EBITDA 属于所有出资人——股东+债权人）。

### 3. 同行公司的选择——"到底是苹果比苹果，还是苹果比橙子？"

可比公司分析最核心也最难的一步是：**选谁当"可比公司"？**

**这不是科学，是艺术**——因为世界上没有两家完全一样的公司。即使是"最接近的可比公司"，也总有差异：规模不同、增长率不同、利润率不同、地域不同、商业模式细微差别。

**选同行公司的五个维度**：

**① 行业（Industry）**——最基本的要求
- 同行必须是同一个行业的。你不能拿可口可乐（消费品）去比英伟达（芯片）。
- 但"同一个行业"的定义可能很宽——"科技行业"太宽了（包括硬件、软件、互联网、芯片），需要进一步缩小到子行业。

**② 商业模式（Business Model）**——比行业更重要
- 同样是"卖书"：亚马逊（线上平台 + 第三方卖家 + 云计算）和 Barnes & Noble（实体书店）是完全不同的商业模式，不能直接比。
- 同样是"出行"：Uber（平台模式，没有自己的车）和 Hertz（资产模式，自有车队）不能直接比。

**③ 规模（Size）**——收入和市值要在一个量级
- 一个年收入 1 亿美元的公司，不能和年收入 1000 亿美元的公司直接比——大公司有规模效应、品牌溢价、融资成本优势。
- 通常要求可比公司的收入或市值在目标公司的 0.5x-2x 范围内。

**④ 增长率（Growth Rate）**——成长阶段不同，倍数差几倍
- 高增长公司（30%+ 年增长）的估值倍数可能比低增长公司（5% 增长）高 3-5 倍。
- 如果你选了 5 家增长率 5% 的公司来比一家增长率 30% 的公司，Comps 会严重低估目标公司。

**⑤ 地理位置（Geography）**——不同市场的估值体系不同
- A 股、港股、美股的估值体系完全不同。A 股科技公司可能 EV/EBITDA 30 倍，美股可能 15 倍。你不能拿 A 股的倍数去估一家美股公司。

**真实的"同行选择博弈"——投行的"小九九"**：

在投行做 Comps 时，选择哪些公司当"可比公司"有一个"潜规则"：

- 如果你是**卖方顾问**（帮客户卖公司）→ 你会倾向于选择高倍数的可比公司，让目标公司看起来更值钱。
- 如果你是**买方顾问**（帮客户买公司）→ 你会倾向于选择低倍数的可比公司，让目标公司看起来"买得划算"。

**经典案例**：一家科技物流公司要 IPO。它可以被归为两类：
- 如果归为"物流公司" → 同行 EV/EBITDA 倍数 = 8.1x
- 如果归为"商业流程外包（BPO）" → 同行 EV/EBITDA 倍数 = 9.6x

投行会选择哪一种？当然是 BPO——估值凭空高了 18.5%。这 1.5 倍的差距，可能意味着 IPO 融资多出几亿美元。这就是为什么"可比公司的选择"在投行是一个高度战略性的决策。

### 4. Garbage In, Garbage Out——"Comps 的命门"

在金融圈有一个广为人知的缩写：**GIGO（Garbage In, Garbage Out，垃圾进，垃圾出）**。

Comps 是 GIGO 的重灾区——因为 Comps 不产生"内在价值"，它只是反映"市场情绪"。如果输入的数据是错的、过时的、不具可比性的，算出来的估值就是垃圾。

**五个最常见的 GIGO 陷阱**：

**陷阱一：混用时间段**
- 公司 A 用了 2024 年 Q4 的 LTM（过去 12 个月）数据
- 公司 B 用了 2024 年 Q2 的数据（已经过了两个季度）
- 公司 C 用了 2025 年分析师的预测数据
- → 三个不同时间点的数据混在一起算中位数 = 垃圾

**陷阱二：没有调整一次性项目**
- 某公司今年 EBITDA 特别高，因为卖了一栋楼（一次性收益）
- 如果你不剔除这个一次性项目，它的 EV/EBITDA 倍数会异常低
- → 你会以为这家公司"很便宜"，实际上它的主营业务根本不赚钱

**陷阱三：用错了倍数**
- 对一家 EBITDA 为负的公司（比如亏损的成长型公司），你用 EV/EBITDA 来估值
- EBITDA 是负数 → 倍数也是负数 → 毫无意义
- → 应该改用 EV/Revenue 或 EV/毛利

**陷阱四：可比公司太少**
- 你只选了 3 家"可比公司"来算中位数
- 如果其中一家是异常值（比如被收购传闻推高了股价），整个估值就歪了
- → 至少需要 5-8 家可比公司，理想情况是 8-12 家

**陷阱五：忽视资本结构差异**
- 公司 A 零负债，公司 B 负债率 80%
- 你用 P/E（市盈率）来比这两家公司
- P/E 只看股东收益，负债率高的公司 P/E 可能反而更低（因为利息支出减少了净利润）
- → 应该用 EV/EBITDA（不受资本结构影响）来比

### 5. Football Field——"一页看懂公司值多少钱"

在投行的 Pitchbook 里，你一定见过这样一页：横轴上画着各种估值方法的结果，像一个个"橄榄球场"上的码线。

这就是 **Football Field Chart（橄榄球场图）**，它的意义是：

> 把 DCF、Comps、Precedent Transactions、52 周股价区间等不同估值方法的结果画在同一张图上，让读者一眼看出"公司大概值多少钱"。

**Football Field 的典型结构**：

```
估值方法                        隐含股价范围
──────────────────────────────────────────────
52 周股价区间              |====●====|
                          $45          $68

DCF 估值                  |======●======|
                          $55           $90

可比公司分析 (Comps)       |=====●=====|
                          $60          $85

先例交易分析               |=======●=======|
                          $70             $105
                          ├──────┼──────┤
                          $55    $75    $95
                          (25分位)(中位)(75分位)
```

**为什么叫 Football Field**：因为图上的每个估值范围就像橄榄球场上的码线，而股票当前的价格就像"球的位置"。

**看 Football Field 的三个层次**：

- **第一层**：当前股价在不在估值范围内？如果在外面，可能被高估或低估。
- **第二层**：不同估值方法给出的范围差异大不大？如果差异很大（比如 DCF 说 $55，Precedent 说 $105），说明这家公司的估值高度不确定。
- **第三层**：哪几种方法最可靠？对于成熟稳定的公司，Comps 和 DCF 可能最可靠；对于并购目标，Precedent Transactions 最相关。

### 6. EV/EBITDA——"估值的'世界语'"

在所有估值倍数中，**EV/EBITDA** 是最常用的一个——它被称为估值的"世界语"，因为：

**① 不受资本结构影响**：
- EV 包含了负债和权益 → 不管公司借了多少钱，EV/EBITDA 都公平比较
- P/E 只看净利润 → 负债率高的公司利息多、税少，P/E 会失真

**② 不受折旧政策影响**：
- EBITDA 在折旧之前 → 不管公司用直线折旧还是加速折旧，EBITDA 都一样
- 净利润受折旧方法的巨大影响 → 两个完全相同的工厂，折旧政策不同，净利润可以差 30%

**③ 不受税率影响**：
- EBITDA 在利息和税收之前 → 不管公司在哪个国家、享受什么税收优惠，EBITDA 都是"纯经营"的衡量
- 净利润受税率影响 → 在爱尔兰（税率 12.5%）的公司净利润天然比在法国（税率 25%）高

**EV/EBITDA 的典型范围**：
| 行业 | 典型 EV/EBITDA 范围 |
|------|---------------------|
| 稳定消费品（可口可乐、宝洁） | 12-18x |
| 科技/软件（微软、Salesforce） | 15-30x |
| 工业制造（通用电气、波音） | 8-14x |
| 零售（沃尔玛、Costco） | 8-15x |
| 高增长 SaaS（Snowflake、Datadog） | 20-60x+ |

**注意**：上表只是"大概范围"，具体倍数随市场环境大幅波动。2021 年科技泡沫时，很多 SaaS 公司的 EV/EBITDA 超过 100 倍——这在正常市场是不可想象的。

### 给小白的一句话

> **可比公司分析（Comps）是三种估值方法中最"接地气"的一种——它的逻辑就是"看看隔壁老王卖多少钱"。但你必须在 EV（企业价值）和市值之间做区分——EV 才是"买下整家公司真正需要的钱"。选同行公司是艺术不是科学：如果你选高倍数的同行，估值就偏高；选低倍数的同行，估值就偏低——这就是投行的"小九九"。Comps 最怕 "Garbage In, Garbage Out"——时间混用、一次性项目、用错倍数、可比公司太少、忽视资本结构差异，都会让你的估值变成垃圾。记住一句话：Comps 告诉你"市场认为它值多少"，DCF 告诉你"它本质上值多少"，两相结合才是完整的估值视角。**
- [ ] 视觉格式是否遵循惯例（蓝色 = 输入，黑色 = 公式）
- [ ] 合理性检查是否通过（利润率逻辑、倍数合理）
- [ ] 时间戳是否为当前日期（“截至 [日期]”）
- [ ] 公式审计是否显示无误（无 #DIV/0!, #REF!, #N/A）

## 持续改进

完成一次 Comps 分析后，请思考：
1. 统计数据是否揭示了意料之外的洞察？
2. 是否存在限制分析的数据缺口？
3. 利害关系人是否询问了你未包含的指标？
4. 实际耗时与预期耗时的对比？
5. 下次如何让分析更有价值？

最优秀的 Comps 分析会随着每次迭代而进化。保存模板，从反馈中学习，并根据决策者的实际需求精炼结构。

> **💡 Appendix: 领域知识小贴士**
>
> 即使你从来没学过金融，用生活中的例子也能秒懂这些“高大上”的概念：
>
> 1. **什么是可比公司分析 (Comps)？**
>    想象你要买一套房子，但不知道房东开的价格合不合理。你会去看隔壁栋、后面那条街类似户型的房子最近卖了多少钱。这就是 Comps。我们将性质相似的公司放在一起比价，就能知道某家公司是“买贵了”还是“捡漏了”。
>
> 2. **营运指标 (Operating Metrics)——“身体素质”：**
>    如果公司是一个运动员，收入就是他的身高，增长率是他的成长速度，而利润率（Margin）就是他的体脂率。体脂率低（利润率高）说明公司很强壮，每一块肌肉（每一块钱收入）都能转化成实实在在的力气（利润）。
>
> 3. **估值倍数 (Valuation Multiples)——“价格放大镜”：**
>    市盈率 (P/E) 最常用。如果一家公司每年赚 1 块钱，你愿意花 20 块钱买它，那市盈率就是 20 倍。这代表你愿意为了它未来的盈利等待 20 年。通常，大家觉得更有潜力的公司，这个倍数就越高。
>
> 4. **企业价值 (EV) vs 市值 (Market Cap)——“买房的总价”：**
>    *市值* 就像是房子的“首付”；而 *企业价值 (EV)* 才是“成交总价”。因为买下一家公司时，你不仅买下了它的股票（首付），还要负责它的债务（房贷），当然它账上的现金（你兜里的钱）可以抵消一部分。EV = 股票总价 + 欠别人的钱 - 剩下的现金。
>
> 5. **EBITDA——“赚钱的真本事”：**
>    这个词听起来很酷，其实它剔除了很多干扰（比如交多少税、利息支出等）。它衡量的是一家公司核心业务到底能产生多少现金，是衡量公司经营效率的“金标准”。
