### 1. Pitch Deck——"投行的门面也是武器"

**Pitch Deck（推介材料 / 投行 PPT）** 是投行家用来向客户展示专业能力、说服客户接受某笔交易的核心工具。它在投行中的地位，就像"武士的刀"——既代表身份，也是决胜的关键。

**Pitch Deck 的四个核心功能**：
1. **展示专业性**：向客户证明"你懂我的行业、你懂我的公司"
2. **说服行动**：让客户接受你的建议——IPO、并购、融资
3. **建立信任**：客户把几亿美金的交易交给你，他必须先信任你
4. **差异化竞争**：多家投行在抢同一笔交易 → Pitch Deck 是最好的"竞标书"

**Pitch Deck 通常包含什么**：
- 封面 + 目录
- 市场概览（行业规模、趋势、竞争格局）
- 公司定位（竞争优势、财务表现、增长故事）
- 估值分析（DCF、Comps、先例交易）
- 交易建议（为什么现在？什么结构？预期价格范围？）
- 团队介绍 + 附录

**打比方——Pitch Deck 就像"相亲时的自我介绍"**：
- 你不能说"我很好，你来吧"（太敷衍）
- 你要展示：学历（行业知识）、收入（估值能力）、人品（过往交易经验）、未来规划（交易建议）
- 而且不能有错别字（排版错误）、不能说大话（数字错误）→ 任何一个细节失误都会被直接 Pass

**所以这意味着什么**：本技能的四个检查维度——数字钩稽、逻辑支撑、语言精炼、视觉排版——就是确保 Pitch Deck 在"相亲"时不给投行丢脸。

### 2. 数字钩稽一致性——"要么全对，要么全错"

**数字钩稽一致性（Cross-Page Number Consistency）** 是投行 PPT 中最重要的检查维度——同一个数字在不同页面出现时，必须一模一样。

**为什么这如此重要**：
- 客户会在不同的页面找同一个数字来验证你
- 如果第 3 页说"营收 $500M"，第 15 页说"营收 $520M" → 客户认为你不靠谱
- 数字不一致 = 你的分析过程是不严谨的 = 接下来的交易可能也不靠谱

**最常见的"不一致陷阱"**：

**① 千位分隔符/小数位数不一致**：
- 第 3 页：$500M（500 百万）
- 第 15 页：$0.5B（0.5 十亿）
- 虽然数值相同，但格式不一致 → 客户可能以为你写错了
- 规则：全文统一使用一种格式

**② 来源版本不一致**：
- 第 3 页的营收来自 Q3 财报
- 第 15 页的营收来自 Q4 财报（更新了）
- 两个数字都是对的，但引用的时间点不同 → 客户会困惑

**③ 计算口径不一致**：
- 第 3 页的 EBITDA = 调整后 EBITDA（剔除非经常性项目）
- 第 15 页的 EBITDA = 法定 EBITDA（未调整）
- 两个都是"EBITDA"，但口径完全不同

**打比方——数字钩稽就像"选举计票"**：
- 如果一个候选人在 A 屏幕上显示 500 票，B 屏幕显示 520 票 → 选举舞弊！
- 投行 PPT 也是一样——同一个数字在整份文档中必须完全一致

**所以这意味着什么**：本技能要求"不同页面间的数字钩稽一致性"——这看似是一个"小细节"，但在投行的世界里，一个不一致的数字等于"不够专业"的标签。

### 3. "So What" 原则——"每句话都要问'所以呢？'"

在投行 PPT 中，每一页的标题和内容都必须回答同一个问题：**所以呢？（So What?）**

**什么是 "So What" 原则**：
- 不是告诉客户"事实是什么"（客户自己知道）
- 而是告诉客户"这个事实意味着什么"（客户需要你的判断）
- 每一页 PPT 的核心信息都应该能通过"So What"测试

**例子对比**：

| 错误的标题（纯陈述） | 正确的标题（有 So What） |
|-------------------|---------------------|
| "行业规模 $500B" | "$500B 的市场规模为中大型并购提供了足够多的标的" |
| "公司营收增长 15%" | "15% 的营收增速远高于行业平均的 5%，表明公司在持续抢占市场份额" |
| "可比公司 EV/EBITDA 中位数 12x" | "可比公司 12x EV/EBITDA 的中位数意味着公司当前估值仍有 30% 的上行空间" |

**如何应用 "So What" 原则检查 PPT**：
读每一页的标题 → 问自己"然后呢？"→ 如果回答不了，这页就是"数据堆砌"而不是"分析"

**打比方——"So What" 就像医生给病人解读体检报告**：
- 错误方式："你的血糖是 7.8"（纯事实 → 病人听不懂）
- 正确方式："你的血糖偏高，已经进入了糖尿病前期——你需要控制饮食、增加运动"（So What → 病人知道该怎么做）
- 投行对待客户的方式也是一样——不是报数据，而是给建议

**所以这意味着什么**：本技能要求检查"数据对叙事逻辑的支撑性"——就是看每一页是否通过了"So What"测试。如果通不过，这页就需要重写。

### 4. 投行语言的精炼度——"50 页 PPT，客户只看 10 分钟"

投行 PPT 通常 50-80 页，但客户（CEO/CFO）只有 10-15 分钟来翻阅。这就是为什么投行 PPT 的语言必须极度精炼——每一句话都要"值钱"。

**投行语言精炼的三个标准**：

**① 没有废话**：
- "我们认为" → 多余（PPT 里写的就是你的观点）
- "非常"、"极其"、"相当" → 都是水词，去掉
- "值得注意的是" → 如果值得注意，为什么不直接写出来？

**② 主谓宾清晰**：
- 病句："通过优化成本结构，使公司的盈利能力得到提升"
- 正确："成本优化使 EBITDA 利润率从 18% 提升到 22%"
- 投行 PPT 用主动语态，不用被动语态

**③ 数字说话**：
- 抽象："公司盈利能力显著提升"
- 具体："EBITDA 利润率从 18% 提升到 22%，同比 +400bp"
- 投行 PPT 能用量化就不说形容词

**投行 PPT 中常见的"废话"和替代**：
| 废话 | 替换 |
|------|------|
| "我们坚信" | 直接说结论 |
| "值得关注的是" | 删掉 |
| "显著增长" | "增长 25%" |
| "良好的市场前景" | "TAM $200B，CAGR 12%" |
| "较强的竞争优势" | "市占率 35%，第二大竞争对手 12%" |

**打比方——投行 PPT 的语言就像"电报"**：
- 电报按字数收费 → 每个字都要"值钱"
- 投行 PPT 按"客户时间"收费 → 每个词都要"有用"
- 一句话能说清的不说两句，一个数字能说清的不说形容词

**所以这意味着什么**：本技能要求检查"符合投行专业语境的语言精炼度"——删掉所有废话、量化所有结论、让每一句话都通过"So What"审查。

### 5. 投行 PPT 视觉排版——"一个像素的偏移都可能是致命伤"

**视觉排版与格式质量控制** 是投行 PPT 检查中的"最后一关"——在内容和语言都 ok 之后，确保视觉上没有任何瑕疵。

**投行 PPT 的排版标准**：

**① 标题不溢出**：标题永远在一行内完成，不会因为太长而换行
**② 元素不重叠**：文本框、图表、Logo 之间没有"碰撞"
**③ 文字不截断**：所有文字内容都在可见区域内，没有被文本框裁剪
**④ 颜色统一**：全文档使用同一套颜色（公司品牌色），没有"花里胡哨"的个人创意
**⑤ 图表是真正的对象**：不是截图粘贴（无法编辑、分辨率低），而是可编辑的 Office 图表对象

**为什么排版如此重要**：
- 排版错误 = 不注重细节 = 不专业
- 一个 LP（有限合伙人）看到 PPT 上的文字被截断了 → "这家 PE 连 PPT 都做不好，投后管理能好到哪里？" → 不投
- 错过数千万美元的承诺资金——因为一个文本框没对齐

**打比方——投行 PPT 的排版就像"瑞士手表"**：
- 瑞士手表的齿轮之间精确到微米
- 投行 PPT 的元素之间精确到像素
- 任何一点的偏移都是"瑕疵"——在投行的世界，瑕疵是不可接受的

**所以这意味着什么**：本技能要求"逐页视觉校验"——这不是"差不多就行"，而是"差一点都不行"。在投行，一个排版错误可能让你丢掉一笔交易。
