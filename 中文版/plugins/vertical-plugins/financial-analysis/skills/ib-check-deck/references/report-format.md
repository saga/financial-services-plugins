# 演示文稿核查报告格式

## 报告模板

```markdown
# 演示文稿核查报告：[演示文稿名称]

## 概要
- 问题总计：X
- 关键级别：X（数据不符、事实错误）
- 重要级别：X（叙事与数据协调、语言表达）
- 次要级别：X（格式问题）

## 关键问题

### 数据一致性
1. **[问题名称]**（幻灯片 X, Y）
   - 幻灯片 X：[数值]
   - 幻灯片 Y：[数值]
   - 处理建议：[建议方案]

### 数据与叙事协调性
1. **[问题名称]**（幻灯片 X, Y）
   - 陈述内容：“[引用文本]”
   - 数据显示：[矛盾之处]
   - 处理建议：[建议方案]

## 重要问题

### 语言润色
1. **[问题类型]**（幻灯片 X）
   - 当前内容：“[引用文本]”
   - 建议修改：“[替换建议]”

## 次要问题

### 格式规范
1. **[问题类型]**（幻灯片 X）
   - [描述与修正方案]

## 最终检查清单
- [ ] 数据勾稽完成
- [ ] 叙事逻辑与数据相符
- [ ] 语言表达符合投行（IB）标准
- [ ] 图表来源已标注
- [ ] 格式保持统一
```

## 问题严重程度分类

**关键（Critical）**（交付客户前必须修正）：
- 跨幻灯片的数据不匹配（数值矛盾）
- 计算错误
- 事实性表述不准（姓名、职务、日期）
- 数据与叙事逻辑相违背

**重要（Important）**（应当修正）：
- 措辞过于口语化或非正式
- 缺乏具体依据的模糊陈述
- 专业术语使用不一致
- 遗漏图表数据来源

**次要（Minor）**（打磨细节）：
- 字体或颜色不统一
- 日期格式不一致
- 间距、对齐或缩进问题
- 孤行短语（排版导致的单行残余文字）

> **💡 Appendix: 领域知识小贴士**
>
> 1. **什么是 Deck Check？**
> 在投资银行（IB）或咨询行业，“Deck”指的就是 PPT 演示文稿。Deck Check 是项目初稿完成后进行的“终极体检”，旨在确保发给客户的每一页 PPT 都像艺术品一样严谨、精确且美观。
>
> 2. **数据勾稽（Number Reconciliation）**
> 这是投行的基本功。比如你的第 5 页说公司去年收入 1 亿，第 50 页的财务报表里也必须是 1 亿。如果两处对不上，就会显得非常不专业。“勾稽”就是要把这些分散在各处的数据全部对齐。
>
> 3. **投行标准（IB Standards）**
> 投行标准代表了一种极度的职业主义（Professionalism）。它不仅要求数字 100% 正确，还追求语言的极简、客观和专业。避开“大概”、“也许”这种词，用词要表现出对行业的深刻理解。
>
> 4. **叙事与数据的关系（Narrative vs Data）**
> 想象你在讲故事。如果你的文字（叙事）在夸奖公司“业绩突飞猛进”，但旁边的折线图（数据）却是平的一条线，这就叫“叙事与数据不协调”。好的报告应该是数据为文字提供证据，文字为数据赋予灵魂。
