# 化用提炼自用排版幻灯片指令

解构剖析投行或机构自家祖传的老旧 PPT 美化模块，强行吸收之后化生成此名将模型可随地任意调用来排版的内嵌独立装配器。

## 流程拆解

1. **索要吸取养分的原文件底盘** (若未告知):
   - "请交出包含你们专属风格的演示原始板 (.pptx) "
   - 它必须含有效且严格嵌套的标准母版占位节点（Placeholder）。
2. **祭出 `ppt-template-creator` 深研探针**:
   - 阅读 xml 结构解析剥皮，了解这套幻灯片底部的留白界限尺寸与字体层级结构约束图。
3. **搜集信息确认**:
   - 敲定并固化该项新出生的自动排版技能包它的官方响亮大名 (比如 `jpm-client-deck-framer`)。
4. **生成封藏组装包**:
   - 将这套繁文缛节全部封存打包，生成出一个具有无尽复制自动填字制图功能的 skill 自给自足工具箱发还给用户。

### 1. 为什么投行的 PPT 不能随便做——"每一页都是几亿美金的门面"

在投行和 PE 行业，PPT 不是"演示文稿"，而是"**交易的门面**"——你给客户、投资委员会、LP 看的 PPT，代表了你的专业度和对细节的关注。

**投行 PPT 和普通 PPT 的本质区别**：

| 维度 | 普通 PPT | 投行 PPT |
|------|---------|---------|
| 目的 | 演讲辅助 | 独立说服文档 |
| 信息密度 | 低（几个要点） | 极高（一页几百字+图表） |
| 排版 | 随意 | 像素级对齐 |
| 颜色 | 花哨 | 严格的品牌色 |
| 母版 | 通常不用 | 强制使用 |
| 修改频率 | 偶尔 | 每天数次更新 |
| 版本管理 | 不重要 | 极其重要（V1→V27） |

**为什么投行 PPT 的信息密度这么高**：
- 客户（CEO/CFO）可能没有时间听你讲 PPT → PPT 必须自己"能说话"
- 投资委员会可能只花 5 分钟翻看 → 每一页必须包含足够的信息
- 有些 PPT 会在不同时区传阅 → 不能依赖"口头解释"

**打比方**：投行的 PPT 就像"建筑图纸"——
- 普通人看房子只看外观（普通 PPT：好看就行）
- 建筑师看房子的结构图纸（投行 PPT：精准、可复制、每个细节都有意义）

**真实案例**：某 PE 基金在向 LP 募资时，一家 LP 的尽职调查团队发现该基金的 PPT 中，三个图表的数据格式不统一（有的是"$M"，有的是"百万美元"）。LP 认为"连 PPT 都做不专业的基金，投后管理也不会专业"——最终没有投。一个小细节，导致损失了数千万美元的承诺资金。

**所以这意味着什么**：本命令就是帮你把投行的"排版基因"固化下来——以后生成任何 PPT，都不需要手动调整格式，自动符合机构的品牌标准。

### 2. 母版（Master Slide）——"PPT 的'宪法'"

**母版（Slide Master）** 是 PPT 中最强大但最容易被普通人忽略的功能——它定义了所有幻灯片的"底层规则"。

**母版控制了哪些东西**：
- **字体**：标题用什么字体、正文用什么字体、大小多少
- **颜色**：标题色、正文色、强调色
- **占位符（Placeholder）**：哪里放标题、哪里放正文、哪里放图表
- **背景**：公司 Logo、页码位置、页脚信息
- **间距**：左留白、右留白、上下边距

**为什么投行必须用母版**：
1. **一致性**：100 页 PPT，每一页的标题位置、字体大小完全一样 → 不能靠人工对齐
2. **效率**：改一次母版，所有页面的格式自动更新 → 不需要一页一页改
3. **品牌标准**：公司 Logo 位置、颜色不能随便变 → 母版确保合规
4. **防错**：即使新手做 PPT，也不会把标题放到奇怪的位置

**母版 vs 手动修改**：
- **手动修改**：选择文字 → 改字体 → 改大小 → 改颜色 → 重复 100 页 = 几个小时
- **母版修改**：打开母版 → 改一次 → 所有 100 页自动更新 = 1 分钟

**打比方**：母版就像"楼房的框架结构"——
- 框架决定了每层楼的高度、柱子的位置、墙壁的分布
- 你可以在框架内自由装修（加内容），但不能改变框架本身
- 如果框架不牢（母版不好），整栋楼都会有问题

**所以这意味着什么**：本命令的第一步是"解构母版"——理解现有 PPT 的底层规则，才能生成符合规则的新 PPT。

### 3. Placeholder（占位符）——"PPT 的'预留停车位'"

**Placeholder（占位符）** 是母版中预留的"内容区域"——告诉 PPT："这里可以放文字"、"这里可以放图表"、"这里可以放图片"。

**Placeholder 的类型**：
- **标题占位符**：预留标题位置 → 会自动带格式
- **正文占位符**：预留正文位置 → 支持多级缩进
- **图表占位符**：预留图表位置 → 插入图表时自动对齐
- **图片占位符**：预留图片位置 → 拖入图片时自动裁剪
- **页码占位符**：页码位置 → 自动更新

**为什么 Placeholder 如此重要**：
- 没有 Placeholder，你每次都要手动调整位置 → 耗时且不精确
- 有 Placeholder → 不管什么内容，自动放入正确的位置、使用正确的格式
- Placeholder 是"自动化排版"的基础 → 本技能的核心就是把内容填入 Placeholder

**Placeholder 的层级结构**：
- 标题 → 一级正文 → 二级正文 → 三级正文...
- 每一级有不同的字体大小、颜色、缩进
- 这确保了 100 页 PPT 的"视觉一致性"

**打比方**：Placeholder 就像"停车场里的车位"——
- 车位（Placeholder）规定了车停哪里、怎么停
- 无论是什么车（内容），只要停在车位里，看起来就整齐
- 没有车位 → 车到处乱停 → 一片混乱

**所以这意味着什么**：本命令要求模板"含有效且严格嵌套的标准母版占位节点"——因为只有有了 Placeholder，才能自动化生成内容。

### 4. 投行 PPT 的"颜色纪律"——"不是好不好看，而是对不对"

在投行 PPT 中，颜色不是"美学选择"，而是"纪律要求"。每个颜色都有特定的含义。

**投行 PPT 的四种颜色许可**：

**① 公司主色（Brand Color）**：
- 每个投行/PE 有自己的一套颜色（通常是蓝、绿、灰）
- 高盛 = 蓝色系、摩根大通 = 蓝色+黑色、贝莱德 = 黑色+红色
- **用途**：标题、图表的主要线条、封面

**② 图表色板（Chart Palette）**：
- 通常有 5-8 种颜色，用于区分不同系列
- 顺序固定：第一个系列永远用颜色 A，第二个永远用颜色 B
- **忌讳**：不能随意换色 → 客户看到红色的柱状图会紧张

**③ 对比色（Contrast Color）**：
- 用于强调关键数据（如"我们的估值比市场高 15%"）
- 通常用公司色的互补色 → 但必须极其克制，一页只能用 1-2 次

**④ 灰色系（Gray Scale）**：
- 用于次级信息（来源、注释、脚注）
- 用于图表背景（淡灰）和网格线（更淡的灰）

**投行 PPT 的颜色禁忌**：
- 不能用渐变色（gif、彩虹色）→ 看起来不专业
- 不能超过 3 个非灰色系 → 太花哨
- 图表不能用"默认配色" → 默认配色太丑

**打比方**：投行 PPT 的颜色纪律就像"军装"——
- 你不能因为"喜欢红色"就穿红色军装 → 军装有严格的颜色规定
- 将军（标题）、军官（正文）、士兵（脚注）各有各的颜色
- 颜色是用来传递信息的，不是用来"好看"的

**所以这意味着什么**：本命令从模板中"萃取版式精魄"——包括颜色体系。生成的 PPT 必须严格遵循原模板的颜色纪律。

### 5. 一个好的投行 PPT 长什么样——"不只是好看，更是信息锤子"

在投行，"好看的 PPT"和"好的 PPT"是完全不同的概念。一个好的投行 PPT，必须像一把"信息锤子"——每一页都在敲打一个核心信息。

**好 PPT 的五个标准**：

**① 标题即结论（Actionable Title）**：
- 错误示范："季度业绩分析" → 太泛
- 正确示范："Q3 营收同比增长 15%，主要受亚太市场 30% 增长的驱动" → 看完标题就知道结论

**② 数据支撑（Data-Backed）**：
- 标题说"增长"，图表必须显示增长
- 数据来源必须标注（"来源：公司年报"）
- 关键数字必须格式化（$1,234M，而不是 $1234567）

**③ 逻辑流畅（Logical Flow）**：
- 先讲"发生了什么" → 再讲"为什么会这样" → 最后讲"这意味着什么"
- 每一页都是上一页的延伸，不能跳跃

**④ 视觉干净（Visual Clean）**：
- 没有多余的元素 → 装饰就是噪音
- 图表不"过度设计" → 3D 效果、阴影、动画都是禁忌
- 留白合理 → 信息密度高不等于"挤满"

**⑤ 防错（Error-Proof）**：
- 所有数字与 Excel 模型链接 → 不能有数字不一致
- 字体一致、颜色一致、对齐一致 → 不能有"视觉误差"
- 每页的数据可以被独立验证 → 不能"我说是什么就是什么"

**打比方**：好的投行 PPT 就像"好的法律文件"——
- 标题就是判决书的主文（一看就懂）
- 数据就像证据（支持判决）
- 逻辑就像推理（证据 → 结论）
- 格式就像法律格式（严谨、规范、不容质疑）

**所以这意味着什么**：本命令虽然主要处理"形式"（排版），但"形式"服务于"内容"——一个好的模板不仅是"好看"，更是"能用"和"好用"。
