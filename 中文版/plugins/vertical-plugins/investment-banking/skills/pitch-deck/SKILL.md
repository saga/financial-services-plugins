# 投行演示文稿填充 (Pitch Deck Population)

## 核心任务
本技能的主要目标是确保当用户提供了 PPT 模板和源数据（如 Excel 财务报表、行业研究数据）时，能够精准、美观地完成数据到幻灯片的迁移。

### ⚠️ 关键局限性说明
由于验证过程主要基于服务器端的渲染引擎（如 LibreOffice），其对字体、渐变、形状位置的渲染可能与 Microsoft PowerPoint 存在细微偏差。
**交付必读：** 在交付最终文件时，必须提醒用户：“本文件已经过渲染验证，但由于软件兼容性差异，请务必在 Microsoft PowerPoint 中进行最后的人工复核。”

## 填充流程

### 1. 数据提取与校验 (Data Extraction)
- **备份原始模板**：修改前先创建 `[文件名]_backup.pptx`。
- **单位统一**：确保所有源数据（百万 vs 十亿）与 PPT 模板中的预设标签一致。

### 2. 内容映射 (Content Mapping)
- **审视布局**：在修改前先完整预览一遍模板，理解其设计逻辑。
- **处理占位符**：识别模板中的占位符（如 `[Insert Data Here]` 或带有说明的黄色色块）。

### 3. 模板填充规范 (Populating Standards)
- **删除说明框**：模板中的彩色说明框通常代表“操作指引”，在填充真实数据后应将其彻底删除，不要保留其颜色和框线。
- **对象化处理**：
  - **表格**：必须创建真实的 PPT 表格对象，**严禁**使用管道符（|）或空格制表。
  - **图形**：箭头、形状等需使用 PowerPoint 的原生形状对象。
- **Logo 放置**：若提供了公司 Logo，应放置在右上角并保持全片位置一致；未提供时请使用 `[LOGO NOT PROVIDED]` 标记。

### 4. 视觉对齐与格式
- **字体一致性**：同二级标题、同位置的数据框必须使用完全相同的字体大小、颜色。
- **四舍五入原则**：市场规模（>10B）通常保留至整数；CAGR 及利润率通常保留至小数点后一位；估值倍数（Multiples）通常保留一位小数（如 `9.7x`）。

## 关键检查清单 (Checklist)
- [ ] 文本是否在背景下清晰可辨（对比度检查）？
- [ ] 表格是否为原生对象而非文本模拟？
- [ ] 幻灯片各处的关键数据是否一致（跨片校验）？
- [ ] 是否清理了所有的 `[括号]` 占位符？

## Appendix: 金融背景知识

这份文件是"投行 Pitch Deck（推介演示文稿）"技能的详细说明。在投行世界里，Pitch Deck 是每一个投行家（Banker）最重要的"武器"——它决定了你能不能拿到项目、能不能帮客户卖掉公司、能不能帮公司融到资。高盛、摩根士丹利、瑞银这些顶级投行每年要制作数千份 Pitch Deck，每一份都是几十页甚至上百页的精美 PPT。这份文件就是教你怎么用 AI 来高效生成这些专业级的投行演示文稿。

### 1. 什么是 Pitch Deck？它和普通 PPT 有什么区别？

**用一个生活化的类比：**

想象你是一个房地产中介。一个房主来找你，说"我想把我这套别墅卖了"。你不可能直接说"好，我帮你挂牌"。你首先要做一份"销售方案"——告诉房主：
- "我了解你的房子值多少钱"（市场分析）
- "我知道谁最有可能买"（潜在买家分析）
- "我有能力帮你卖出最高价"（我们的团队和经验）
- "这是我的具体执行计划"（时间表和流程）

Pitch Deck 就是投行的"销售方案 PPT"——只不过它卖的不是房子，而是"投行服务"。

**投行 Pitch Deck 和普通公司 PPT 的核心区别：**

| 维度 | 普通公司 PPT | 投行 Pitch Deck |
|------|-------------|----------------|
| 目的 | 展示产品/服务 | 争取项目/赢得客户 |
| 受众 | 普通消费者/员工 | CEO/CFO/董事会（最挑剔的决策者） |
| 数据密度 | 低，主要靠图片和口号 | 极高，每页都有精确的数据和来源 |
| 格式规范 | 随意，创意优先 | 极其严格，有固定的页码和排版规则 |
| 制作周期 | 几天到一周 | 几周到几个月，反复修改几十版 |
| 保密级别 | 公开 | 绝密，泄露可能导致严重法律后果 |

### 2. 投行 Pitch Deck 的五大类型

#### 2.1 卖方推介（Sell-side Pitch）

**场景**：一家公司的老板想卖掉自己的公司，正在面试多家投行，看哪家最适合帮自己卖。

**Pitch Deck 的核心内容：**
- "我们了解你的行业"（行业分析和市场趋势）
- "我们知道你的公司值多少钱"（估值分析，通常用三种方法：可比公司、可比交易、DCF）
- "我们知道谁最有可能买"（潜在买家名单，通常列出30-50个）
- "我们有成功卖掉类似公司的经验"（案例研究，展示过去帮类似公司卖出的交易）
- "这是我们的执行计划"（时间表、团队配置、费用结构）

**关键页**：估值页（Valuation Page）——这是整个 Pitch 中最重要的一页。它通常是一张"足球场图"（Football Field Chart），把三种估值方法的结果并排展示，形成一个价格区间。

#### 2.2 买方推介（Buy-side Pitch）

**场景**：一家公司想收购另一家公司，正在面试投行帮自己做"买方顾问"。

**Pitch Deck 的核心内容：**
- "我们理解你的战略目标"（为什么你想买？是为了市场份额？技术？还是人才？）
- "这是我们认为你应该考虑的目标公司名单"（Target List）
- "我们对每个目标做了初步估值"（Preliminary Valuation）
- "我们有成功帮客户收购的经验"（案例研究）
- "这是我们的尽调能力和执行计划"

#### 2.3 融资推介（Financing Pitch）

**场景**：一家公司需要借钱（发债）或发新股（IPO/增发），找投行帮忙。

**Pitch Deck 的核心内容：**
- "这是当前资本市场的环境"（利率水平、投资者情绪、可比交易）
- "这是你的信用状况/估值水平"（Credit Rating / Valuation）
- "我们建议的融资结构和条款"（Terms and Structure）
- "我们的分销能力"（我们能帮你找到哪些投资者？）

#### 2.4 防御策略推介（Defense Pitch）

**场景**：一家公司收到了恶意收购要约，需要投行帮忙制定防御策略。

**Pitch Deck 的核心内容：**
- "收购方的报价严重低估了你的价值"（Valuation Gap Analysis）
- "我们有多种防御策略"（Poison Pill、White Knight、Recapitalization 等）
- "我们的团队有成功防御恶意收购的经验"

#### 2.5 行业更新推介（Update Pitch）

**场景**：投行定期（每季度/每年）给现有客户发送行业动态更新，维持关系。

**Pitch Deck 的核心内容：**
- "这个行业最近发生了什么大事"
- "哪些公司被收购了、融到资了"
- "估值水平有什么变化"
- "我们认为你应该关注什么"

### 3. Pitch Deck 的核心页面详解

#### 3.1 足球场图（Football Field Chart）

这是投行估值页中最经典的图表。想象一个美式足球场，中间有一条中线，两边有得分区。

```
估值范围（百万美元）
    │
$500 ┤ ████████████████████████████████████████  ← DCF 估值区间
$450 ┤ ████████████████████████████████
$400 ┤ ██████████████████████████  ← 可比公司估值
$350 ┤ ████████████████████
$300 ┤ ████████████████  ← 可比交易估值
$250 ┤ ████████████
$200 ┤ ████████  ← 当前股价
    └──────────────────────────────────────────
         DCF    Comps   Trans   当前股价
```

这张图告诉客户："根据三种不同的估值方法，你的公司价值在 $250M-$500M 之间。而当前股价只有 $200M，说明市场严重低估了你。"

#### 3.2 可比公司表（Trading Comps）

这张表列出5-10家和目标公司"最像"的上市公司，对比它们的估值倍数：

| 公司 | 市值 | EV/EBITDA | P/E | 收入增速 | EBITDA率 |
|------|------|-----------|-----|---------|---------|
| 公司A | $5B | 15x | 22x | 12% | 25% |
| 公司B | $3B | 12x | 18x | 8% | 22% |
| 公司C | $8B | 18x | 25x | 15% | 28% |
| **中位数** | | **15x** | **22x** | **12%** | **25%** |
| **目标公司** | $2B | **10x** | **16x** | **20%** | **30%** |

**解读**：目标公司的 EV/EBITDA 只有 10x，远低于行业中位数 15x，但增长更快（20% vs 12%）、利润率更高（30% vs 25%）。这意味着目标公司被严重低估了。

#### 3.3 时间表页（Timetable / Process Timeline）

展示从签约到交割的完整时间线：

```
Week 1-2:  签署委任协议 → 启动项目
Week 3-4:  准备 CIM + 买家名单
Week 5-6:  发送 Teaser → 收集 NDA
Week 7-10: 发送 CIM → 收集 IOI
Week 11-14: 管理层路演 → 开放 VDR
Week 15-18: 收集 Final Bid → 谈判
Week 19-22: 签署 SPA → 监管审批
Week 23-26: 交割完成
```

### 4. Pitch Deck 的制作流程（投行内部真实流程）

**第一阶段：内容准备（1-2周）**
- MD（董事总经理）和 VP（副总裁）决定"讲什么故事"
- Associate 负责收集数据、做初步分析
- Analyst 负责搭建 Excel 模型

**第二阶段：PPT 制作（2-4周）**
- Analyst 在 PPT 里把 Excel 的数据和图表搬进去
- 反复修改：MD 看完第一版 → 提出20条修改意见 → Analyst 改完 → MD 又提出15条 → 循环往复
- 一个 Pitch Deck 通常要改 10-20 版才能最终定稿

**第三阶段：彩排（1-2天）**
- 负责演讲的 MD 和 VP 会反复彩排
- 每一页要讲几分钟、怎么过渡、怎么回答客户的刁钻问题，全部提前演练
- 有时候甚至会请"演讲教练"来指导

**第四阶段：正式演讲（1-2小时）**
- 通常由 MD 主讲，VP 和 Associate 在旁边辅助回答技术问题
- 客户（通常是 CEO/CFO/董事会）会提出各种刁钻问题
- 演讲结束后，投行团队要在一周内发送"跟进邮件"，重申核心观点

### 给小白的一句话

> Pitch Deck 就是投行的"求婚 PPT"——你不是在展示你的产品，而是在说服一个极其挑剔的 CEO 把价值几十亿的公司交给你来卖/买/融资。每一页都必须有数据支撑、每一个图表都必须精确到小数点后两位、每一个故事都必须让客户觉得"只有你能帮我做到"。高盛和摩根士丹利每年花几百万美元培训员工做 Pitch Deck，因为一份好的 Pitch 能带来几千万甚至上亿美元的佣金收入。
