# 投行演示文稿填充 (Pitch Deck Population)

## 核心任务
本技能的主要目标是确保当用户提供了 PPT 模板和源数据（如 Excel 财务报表、行业研究数据）时，能够精准、美观地完成数据到幻灯片的迁移。

### ⚠️ 关键局限性说明
由于验证过程主要基于服务器端的渲染引擎（如 LibreOffice），其对字体、渐变、形状位置的渲染可能与 Microsoft PowerPoint 存在细微偏差。
**交付必读：** 在交付最终文件时，必须提醒用户：“本文件已经过渲染验证，但由于软件兼容性差异，请务必在 Microsoft PowerPoint 中进行最后的人工复核。”

## 填充流程

### 1. 数据提取与校验 (Data Extraction)
- **备份原始模板**：修改前先创建 `[文件名]_backup.pptx`。
- **单位统一**：确保所有源数据（百万 vs 十亿）与 PPT 模板中的预设标签一致。

### 2. 内容映射 (Content Mapping)
- **审视布局**：在修改前先完整预览一遍模板，理解其设计逻辑。
- **处理占位符**：识别模板中的占位符（如 `[Insert Data Here]` 或带有说明的黄色色块）。

### 3. 模板填充规范 (Populating Standards)
- **删除说明框**：模板中的彩色说明框通常代表“操作指引”，在填充真实数据后应将其彻底删除，不要保留其颜色和框线。
- **对象化处理**：
  - **表格**：必须创建真实的 PPT 表格对象，**严禁**使用管道符（|）或空格制表。
  - **图形**：箭头、形状等需使用 PowerPoint 的原生形状对象。
- **Logo 放置**：若提供了公司 Logo，应放置在右上角并保持全片位置一致；未提供时请使用 `[LOGO NOT PROVIDED]` 标记。

### 4. 视觉对齐与格式
- **字体一致性**：同二级标题、同位置的数据框必须使用完全相同的字体大小、颜色。
- **四舍五入原则**：市场规模（>10B）通常保留至整数；CAGR 及利润率通常保留至小数点后一位；估值倍数（Multiples）通常保留一位小数（如 `9.7x`）。

## 关键检查清单 (Checklist)
- [ ] 文本是否在背景下清晰可辨（对比度检查）？
- [ ] 表格是否为原生对象而非文本模拟？
- [ ] 幻灯片各处的关键数据是否一致（跨片校验）？
- [ ] 是否清理了所有的 `[括号]` 占位符？

> **💡 Appendix: 领域知识小贴士**
>
> 1. **Pitch Book / Pitch Deck (标书/演示文稿)**：这是投行“卖稿子”的核心。不管是去争取一个 IPO 客户，还是向潜在买家推销一家公司，这份文档就是投行的“门面”。它不仅要有深度的数据支撑，视觉上的极其严谨也代表了投行的专业水准。
> 2. **占位符 (Placeholder)**：在大型项目中，分析师通常先做模板。比如“此处放未来三年的收入图表”。这些占位符通常用醒目的颜色标注，但在最后发给客户前，如果留着一个黄色框框，那属于极其严重的职业事故。
> 3. **视觉一致性 (Visual Consistency)**：在高级投行人眼中，如果两页 PPT 的标题位置偏移了 1 毫米，或者两列数字的对齐方式不一致，这就是不专业的表现。投行这种对细节的“强迫症”是为了向客户展示：我们对你的交易数据也同样严谨。
