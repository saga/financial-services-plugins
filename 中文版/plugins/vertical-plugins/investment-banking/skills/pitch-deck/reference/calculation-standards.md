## 关键验证公式

### CAGR 预测

**公式：**
```
未来值 = 现值 × (1 + CAGR)^n
```

**变量：**
- 现值：当前/基准年市场规模
- CAGR：复合年增长率（十进制，如 16.4% = 0.164）
- n：基准年与目标年之间的年数

**验证示例：**
```
源数据声称：$22.1bn（2024年），以16.4% CAGR增长至$55.0bn（2030年）

验证：22.1 × (1.164)^6 = 22.1 × 2.488 = 55.0 ✓
```

**计算 n（年数）：** 计算基准年与目标年之间的年数。例如：2024→2030 = 6年，2025→2030 = 5年。

### 估值倍数

**EV/收入：**
```
EV/收入倍数 = 企业价值 ÷ 收入
隐含 EV = 收入 × 倍数
```

**EV/EBITDA：**
```
EV/EBITDA倍数 = 企业价值 ÷ EBITDA
隐含 EV = EBITDA × 倍数
```

**验证示例：**
```
源数据声称：$436m 交易，9.7x 收入倍数，$45m 收入

验证：436 ÷ 45 = 9.69 ≈ 9.7x ✓
```

### 市场份额

**公式：**
```
市场份额 =（分部规模 ÷ 总市场规模）× 100
```

**验证示例：**
```
源数据声称：线上分部（$18bn）是总市场（$65bn）的 28%

验证：18 ÷ 65 = 0.277 = 27.7% ≈ 28% ✓
```

### 增长率

**同比：**
```
同比增速 =（本年 - 上年）÷ 上年 × 100
```

**从端点计算 CAGR：**
```
CAGR =（终值 ÷ 初始值）^(1/n) - 1
```

## 一致预期方法

当源数据包含多个预测时，验证一致预期计算：

### 规模一致预期（范围）

**方法：** 所有来源的完整最小-最大范围

**示例：**
```
来源：$14.9bn, $18.3bn, $21.1bn, $21.2bn, $22.1bn
一致预期：$15-22bn（取整到最近的 $1bn）
```

### CAGR 一致预期（中心聚类）

**方法：** 排除异常值（最高和最低），使用中心聚类范围

**示例：**
```
来源：10.6%, 16.4%, 17.2%, 19.0%, 22.7%
排除异常值：10.6%（低）, 22.7%（高）
中心聚类：16.4%, 17.2%, 19.0%
一致预期：16-19% 或 16-17%（保守）
```

### 预测一致预期

**方法：** 将一致预期 CAGR 应用于规模范围的中点

**示例：**
```
规模范围：$15-22bn → 中点：$18.5bn
CAGR 一致预期：16-17%
按 16%：18.5 × (1.16)^6 = $45.1bn
按 17%：18.5 × (1.17)^6 = $47.5bn
一致预期预测：$45-48bn
```

## 取整指南

以下为**典型惯例**——根据数值量级和模板风格调整：

| 值类型 | 典型取整 | 示例 |
|----------|------------------|---------|
| 大市场规模（$10bn+） | 取整到最近的 $1bn | 18.47 → $18bn |
| 较小市场规模（<$10bn） | 取整到最近的 $0.5bn 或 $0.1bn | 2.3 → $2.5bn |
| 规模范围 | 匹配源数据精度 | 14.9-22.1 → $15-22bn |
| CAGR | 整数%或0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 取整到最近的5%或匹配源数据 | 27.7% → 25% 或 30% |
| 收入（百万美元） | 1位小数 | 18.47 → $18.5m |
| 倍数 | 1位小数 | 9.688 → 9.7x |

**取整原则：**
- 取整不应实质性改变数值——对较小值使用更精细的精度
- 一致性比精度更重要——对相似数值使用相同取整
- 创建范围时，低端向下取整，高端向上取整
- 汇总统计（均值、中位数）匹配输入数据精度

## 验证清单

在源数据中使用任何计算值前：

### 公式验证
- [ ] 预测使用正确的 CAGR 公式：`PV × (1 + r)^n`
- [ ] 倍数计算为 EV ÷ 指标（非反向）
- [ ] 增长率使用正确的基准年在分母中
- [ ] 百分比份额合计约 100%（如适用）

### 输入验证
- [ ] 基准年数据与源文档匹配
- [ ] CAGR/增长率与所述源数据方法匹配
- [ ] 时间段（n）计算正确
- [ ] 货币和单位一致（$bn vs $m）

### 输出验证
- [ ] 计算结果与源数据的声称值匹配
- [ ] 如不匹配，调查方法差异
- [ ] 取整应用一致
- [ ] 结果合理（无量级误差）

### 一致预期验证
- [ ] 所有来源包含在范围计算中
- [ ] 异常值排除方法已记录
- [ ] 中点计算使用正确的平均方法
- [ ] 范围边界代表实际最小/最大值或已记录的子集

## 需调查的警示信号

**预测不匹配：**
- 计算结果与源数据差异 >5%
- 可能原因：不同的基准年、不同的 CAGR 或取整差异

**倍数不匹配：**
- 计算倍数与源数据不同
- 可能原因：不同的指标定义（LTM vs NTM，收入 vs 净收入）

**一致预期不匹配：**
- 您的一致预期与源数据的一致预期不同
- 可能原因：源数据排除了某些数据点、不同的异常值处理方式

**如有疑问：** 在脚注中注明差异并展示您的计算方法。
