## 模板分析流程

在填充任何模板之前，分析其结构：

### 步骤1：识别所有内容区域

扫描每张幻灯片，查找：
- **标题/页眉占位符** — 幻灯片标题位置
- **副标题/定义区域** — 次要标题或定义
- **内容框** — 主要内容区域（可能有标签侧边栏）
- **表格占位符** — 指定用于表格数据的区域
- **图表/视觉区域** — 图表、图表或图片的空间
- **指标标注框** — 突出显示的关键数字
- **脚注/来源栏** — 底部用于引用和注释的区域
- **Logo占位符** — 通常在右上角

### 步骤2：记录模板约定

每个模板都有自己的风格。观察：
- **配色方案** — 标题、背景、强调使用什么颜色？
- **字体选择** — 已设置哪些字体和大小？
- **框样式** — 内容框是否有侧边栏、边框或阴影？
- **项目符号样式** — 模板使用什么项目符号符号？
- **对齐模式** — 平行区域如何对齐？

### 步骤3：识别指令与输出区域

模板通常包含指导：
- **指令框** — 带指导文本的彩色框（通常黄色背景、白色文本）
- **占位符文本** — [方括号]中的文本表示要替换的内容
- **示例内容** — 显示预期格式的示例内容

**关键区别**：指令框告诉你该做什么；在最终输出中应重新格式化或删除。输出区域是内容放置的地方。

## 内容映射工作流程

### 步骤1：盘点源数据

创建所有可用数据的列表：
- 市场规模数据和范围
- 增长率（CAGR、同比）
- 公司名称和描述
- 细分市场定义
- 财务指标
- 来源引用和日期
- 脚注内容

### 步骤2：将数据与模板区域匹配

对于每个模板区域，识别：

| 模板区域 | 所需数据 | 来源位置 |
|------------------|---------------|-----------------|
| [区域名称] | [所需数据] | [查找位置] |

### 步骤3：识别差距

映射后，记录：
- **缺失数据** — 模板需要源数据中没有的数据
- **额外数据** — 源数据包含没有模板位置的数据
- **格式不匹配** — 数据存在但格式错误

### 步骤4：填充前解决差距

- 缺失数据：标记给用户或搜索其他来源
- 额外数据：确认是否应排除或需要调整模板
- 格式不匹配：将数据转换为所需格式

## 常见幻灯片类型和数据要求

这些是常见幻灯片类型的典型数据要求。您的特定模板可能有所不同——始终遵循模板的实际结构。

### 市场定义幻灯片

**典型内容区域：**
- 范围内的细分市场（含示例/主要参与者）
- 范围外的细分市场（含示例）
- 市场定义文本
- 范围理由/正当性

**数据映射注意事项：**
- 源数据应明确区分包含和排除的细分市场
- 主要参与者应映射到各自的细分市场
- 定义文本应与来源如何定义市场保持一致

**通常需要的数据：**
- 要包含的市场细分市场列表（含主要参与者示例）
- 要排除的市场细分市场列表（含示例）
- 市场定义文本
- 范围理由或正当性

**格式化原则**：平行区域（包含与排除）应使用匹配的格式。

**验证问题：**
- 每个细分市场是否有适当的符号（✓表示包含，×表示排除）？
- 主要参与者是否正确分配到细分市场？
- 定义是否与来源方法一致？

### 市场规模 / TAM幻灯片

**典型内容区域：**
- 当前市场规模（含年份）
- 增长率（含周期的CAGR）
- 未来预测（含目标年份）
- 来源细分表
- 共识/汇总数据
- 关键要点或见解

**通常需要的数据：**
- 带基准年份的市场规模数据
- 增长率（含时间段的CAGR）
- 带目标年份的预测数据
- 每个数据点的来源引用

**示例列标题：** Source | [基准年份] Size | CAGR | [目标年份] Projection

**格式化原则**：如果显示多个来源，包括共识/汇总行。

**数据映射注意事项：**
- 多个来源可能有不同的估计值——将每个映射到表格行
- 共识数据需要从各个来源计算
- 预测应使用CAGR公式验证

**验证问题：**
- 所有来源数据是否与原始文档匹配？
- 共识是否计算正确（不仅仅从一个来源复制）？
- 所有数据的预测年份是否一致？
- 基于CAGR的预测手动验证时是否匹配？

### 竞争格局幻灯片

**典型内容区域：**
- 以竞争对手为列的比较表
- 功能/能力行
- 财务指标行（收入、增长、市场份额）
- 关键观察或定位注释

**通常需要的数据：**
- 要比较的竞争对手列表
- 各自的功能或能力
- 财务指标（收入、增长、市场份额）（如果可用）
- 财务数据的时间段

**格式化原则**：标的公司应与竞争对手在视觉上区分（例如，粗体文本、不同背景颜色、边框或位于最右列）。

**数据映射注意事项：**
- 确保包含源数据中的所有竞争对手
- 功能比较应使用一致的标准
- 财务数据应来自可比时期

**验证问题：**
- 源数据中的所有竞争对手是否都有代表？
- 标的公司是否在视觉上突出？
- 财务数据是否来自同一时期？
- ✓/×的使用是否一致准确？

### 财务摘要幻灯片

**典型内容区域：**
- 关键指标标注（标题数据）
- 历史财务表（实际数据）
- 预测财务表（估计数据）
- 增长率和利润率
- 可选趋势图

**通常需要的数据：**
- 近年历史财务数据（实际数据）
- 未来年份预测财务数据（估计数据）
- 关键指标：收入、增长率、利润率、EBITDA

**示例列标题：** Metric | FY[Year-2] | FY[Year-1] | FY[Year]A | FY[Year+1]E | FY[Year+2]E

**格式化原则**：清楚地区分历史（A）和预测（E）数据。

**数据映射注意事项：**
- 清楚地区分历史（A）和预测（E）数据
- 确保指标定义与来源匹配（收入与净收入、EBITDA与调整后EBITDA）
- 增长率应一致计算

**验证问题：**
- 历史与预测期间是否清楚标注？
- 计算的增长率是否与来源或手动计算匹配？
- 指标定义是否与源文件一致？

### 交易可比幻灯片

**典型内容区域：**
- 交易表（日期、目标、收购方、交易价值）
- 估值倍数（EV/Revenue、EV/EBITDA）
- 汇总统计（均值、中位数、高、低）
- 标的公司的隐含估值

**通常需要的数据：**
- 交易详情：日期、目标、收购方、交易价值
- 估值倍数：EV/Revenue、EV/EBITDA
- 标的公司指标用于隐含估值

**格式化原则**：包含倍数的汇总统计（均值、中位数、高、低）。

**数据映射注意事项：**
- 倍数应从交易数据计算，而不仅仅复制
- 汇总统计需要对所有交易计算
- 隐含估值将倍数应用于标的公司指标

**验证问题：**
- 是否包含来源中的所有相关交易？
- 倍数计算是否正确（EV ÷ 指标）？
- 汇总统计是否涵盖表中的所有交易？
- 隐含估值是否清楚标注为说明性？

## 映射验证清单

在进行格式化之前，验证映射完整性：

### 数据完整性
- [ ] 每个模板占位符都有映射的源数据
- [ ] 所有来源引用都记录用于脚注
- [ ] 没有未映射的占位符[方括号]

### 数据准确性
- [ ] 数据与原始源文档完全匹配
- [ ] 年份和时间段正确记录
- [ ] 公司名称拼写正确
- [ ] 计算值（共识、预测、倍数）已验证

### 逻辑一致性
- [ ] 包含和排除的细分市场在逻辑上连贯
- [ ] 历史数据按时间顺序先于预测数据
- [ ] 比较数据使用一致的时间段
- [ ] 总计和小计正确求和

### 来源归因
- [ ] 每个数据点都可以追溯到一个来源
- [ ] 记录来源名称和出版年份
- [ ] 为特殊注释分配脚注编号

## 处理数据-模板不匹配

### 模板需要的数据多于可用数据

**选项：**
1. 明确标记差距供用户审查
2. 将区域标记为"数据不可用"并说明原因
3. 如有适当，搜索其他来源
4. 如果数据不存在，建议调整模板

**不要**：编造数据或做出无根据的估计。

### 来源数据多于模板容纳的数据

**选项：**
1. 包含最相关/最新的数据点
2. 在适当情况下汇总或聚合
3. 添加引用额外可用数据的脚注
4. 如果数据至关重要，建议扩展模板

### 数据格式与模板格式不匹配

**常见转换：**
- 单个数据 → 范围（使用来源的最小值-最大值）
- 详细细分 → 汇总类别
- 年度数据 → CAGR（从端点计算）
- 绝对值 → 百分比（计算份额）
- 多个来源 → 共识（应用方法）

### 模板使用不同的术语

**解决流程：**
1. 识别模板术语和来源术语
2. 确认它们指的是同一个概念
3. 在输出中使用模板术语
4. 如果需要澄清，添加脚注

## 模板特定适配

记住：本指南描述的是常见模式，而非要求。始终：

1. **遵循模板** — 如果模板使用不同的区域名称，使用这些名称
2. **匹配模板风格** — 使用模板现有的字体、颜色、项目符号样式
3. **保留模板结构** — 除非必要，否则不要重新排列区域
4. **尊重模板间距** — 内容应适合指定区域，无溢出

目标是按设计填充模板，而不是重新设计它。

## 📚 金融术语和知识解释

### 1. 数据映射（Data Mapping）

**定义**：将源数据字段与目标模板或系统中的对应字段进行匹配的过程。

**重要性**：
- 确保数据准确转移
- 保持数据完整性
- 提高工作效率
- 减少人为错误

**最佳实践**：
- 创建映射文档记录每个字段的对应关系
- 验证映射准确性
- 处理数据格式转换

### 2. TAM（总可寻址市场）

**定义**：Total Addressable Market，指产品或服务理论上可以服务的最大潜在客户群体和市场规模。

**计算方法**：
```
TAM = 目标市场客户总数 × 平均客户价值
```

**示例**：
- 中国在线教育市场
- 目标客户：3亿学生
- 年均消费：5000元
- TAM = 3亿 × 5000元 = 1.5万亿元

**用途**：帮助投资者理解市场的潜在规模和增长机会。

### 3. 市场细分（Market Segmentation）

**定义**：将市场划分为具有相似需求和特征的子群体的过程。

**常见细分维度**：
- **地理细分**：按地区、国家划分
- **人口统计细分**：按年龄、性别、收入划分
- **行为细分**：按购买行为、使用频率划分
- **心理细分**：按生活方式、价值观划分

**重要性**：
- 精准定位目标客户
- 优化营销策略
- 提高产品适配性

### 4. 同比增长率（YoY Growth）

**定义**：Year-over-Year Growth，指与上一年同期相比的增长率。

**计算公式**：
```
YoY增长率 = (本年值 - 上年值) / 上年值 × 100%
```

**示例**：
- 2023年收入：100万元
- 2022年收入：80万元
- YoY增长率 = (100 - 80) / 80 × 100% = 25%

**用途**：衡量业务的年度增长表现。

### 5. 财务预测（Financial Forecasting）

**定义**：基于历史数据和市场分析，对未来财务表现进行估计的过程。

**预测内容**：
- 收入预测
- 成本预测
- 利润预测
- 现金流预测

**方法**：
- **趋势分析**：基于历史趋势外推
- **比率分析**：使用关键比率预测
- **场景分析**：考虑不同情景下的结果

**注意事项**：
- 预测具有不确定性
- 需定期更新和调整
- 应包含敏感性分析

### 6. 估值（Valuation）

**定义**：确定资产、公司或投资价值的过程。

**常见方法**：
| 方法 | 描述 | 适用场景 |
|------|------|----------|
| **DCF** | 贴现现金流法 | 成熟稳定的企业 |
| **可比公司分析** | 与同类公司比较 | 有可比上市公司 |
| **交易可比分析** | 参考并购交易 | 有相关并购案例 |
| **资产基础法** | 基于资产价值 | 重资产公司 |

**关键因素**：
- 现金流预测
- 折现率选择
- 终端价值估算

### 7. 隐含估值（Implied Valuation）

**定义**：通过将市场倍数应用于目标公司的财务指标而得出的估值。

**计算方法**：
```
隐含估值 = 财务指标 × 市场倍数
```

**示例**：
- 目标公司EBITDA：5000万元
- 行业平均EV/EBITDA倍数：12x
- 隐含EV = 5000万元 × 12 = 6亿元

**用途**：快速估算公司价值，作为DCF估值的交叉验证。

### 8. 财务建模（Financial Modeling）

**定义**：创建数学模型来代表公司的财务表现的过程。

**类型**：
- **三表模型**：整合利润表、资产负债表、现金流量表
- **DCF模型**：贴现现金流估值模型
- **LBO模型**：杠杆收购模型
- **并购模型**：合并财务模型

**步骤**：
1. 收集历史财务数据
2. 建立假设
3. 构建预测
4. 进行敏感性分析
5. 生成输出和报告

### 9. 敏感性分析（Sensitivity Analysis）

**定义**：分析关键假设变化对模型输出影响的过程。

**目的**：
- 识别关键驱动因素
- 评估风险
- 理解估值范围
- 支持决策制定

**方法**：
- **单因素分析**：一次改变一个变量
- **多因素分析**：同时改变多个变量
- **情景分析**：设定不同情景（乐观、基准、悲观）

**示例**：
- 如果收入增长率下降1%，估值下降多少？
- 如果折现率上升1%，净现值如何变化？

### 10. 投资银行推介流程

**典型流程**：
1. **初步接触**：了解客户需求和目标
2. **准备材料**：创建Pitch Deck、信息备忘录
3. **投资者路演**：向潜在投资者展示
4. **尽职调查**：投资者审查公司资料
5. **估值谈判**：协商交易价格和条款
6. **交易执行**：完成交易文件和交割

**关键角色**：
- **投资银行家**：负责交易执行
- **分析师**：负责财务分析和建模
- **客户关系经理**：维护客户关系
- **合规官**：确保符合法规要求
