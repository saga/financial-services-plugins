## 视觉层次和布局

### 框和区域布局

幻灯片布局根据内容要求和模板设计而变化。常见元素包括：
- 标题区域，包含标题和副标题
- 带标签侧边栏的内容框
- 结构化数据表格
- 可视化数据图表
- 幻灯片底部的脚注栏

具体布局应遵循提供的模板。常见内容类型及其典型结构：
- **市场定义幻灯片**：带项目符号内容的标签框 + 评论区域
- **TAM/规模幻灯片**：指标标注 + 数据表 + 关键要点
- **竞争分析**：比较表或矩阵
- **财务摘要**：带支持数据表的图表

### 对齐原则

**平行区域的垂直对齐：**

垂直堆叠的框应具有一致的：
- 左边距位置
- 项目符号缩进
- 文本起始位置
- 框宽度

水平相邻的框应具有一致的：
- 顶部位置
- 高度（在内容允许的范围内）
- 内部填充

## 文本格式化

### 项目符号结构

避免无结构的文本堆砌。将内容分解为可扫描的项目符号。

**正确结构示例：**
```
✓ 消费者移动和网页语言学习应用
  (多邻国、Babbel、Memrise、Busuu)
✓ B2B企业语言培训平台
  (goFLUENT、Speexx、Learnship)
✓ 在线辅导市场
  (italki、Preply、Cambly)
```

**错误结构示例（文本堆砌）：**
```
消费者移动/网页应用（多邻国、Babbel、Memrise、Busuu）
B2B企业平台（Speexx、Rosetta Stone企业版）
在线辅导市场（Preply、italki、Cambly）
```

### 项目符号符号指南

| 上下文 | 符号 | 用途 |
|---------|--------|-------|
| 包含/正面 | ✓（勾选） | 范围内的项目、存在的功能 |
| 排除/负面 | ×（叉号） | 范围外的项目、缺失的功能 |
| 中性列表 | •（圆点） | 一般枚举、评论 |
| 编号序列 | 1. 2. 3. | 流程步骤、排名 |
| 子项目符号 | ‣ 或 – | 主项目符号下的次要要点 |

调整符号使用以匹配模板的现有约定。

### 项目符号一致性

一个框/区域内的所有项目符号应具有相同的格式：
- 框内使用相同的项目符号符号（除非有意区分）
- 所有主项目符号具有相同的缩进级别
- 相同的项目符号大小
- 项目符号和文本之间的间距相同
- 同一级别的所有项目符号文本使用相同的字体大小

### 字体大小指南

这些是典型范围 - 根据模板规范调整：

| 元素 | 典型大小（pt） | 样式 |
|---------|-------------------|-------|
| 幻灯片标题 | 40-48 | 粗体 |
| 副标题/定义 | 18-22 | 粗体 |
| 区域标题 | 14-16 | 常规 |
| 正文/项目符号 | 12-14 | 常规 |
| 表格标题 | 10-12 | 粗体 |
| 表格正文 | 9-11 | 常规 |
| 脚注 | 8-9 | 斜体 |

### 文本密度指南

- **每个内容框最多6-7个项目符号**（根据空间调整）
- **每个项目符号最多2行**
- **括号示例**在同一行或缩进在下方
- **避免孤立单词** - 调整换行以避免单个单词出现在新行

## 表格创建

### 关键：使用实际表格对象

**表格必须是实际的表格对象，而不是带制表符间距的文本。**

带制表符的文本永远无法正确对齐，看起来不专业。始终创建合适的表格对象。

### 表格结构指南

1. **列对齐**：
   - 文本列：左对齐（标题和内容）
   - 数字列：居中对齐或右对齐
   - 标题应与其列内容对齐

2. **标题行**：
   - 粗体文本
   - 阴影背景（使用模板的品牌颜色）
   - 对比文本颜色以提高可读性
   - 对齐方式与列内容对齐

3. **交替行**（可选）：
   - 交替行上的浅色阴影提高可读性

4. **摘要/总计行**：
   - 粗体文本
   - 较粗的顶部边框（分隔线）
   - 明显的背景阴影

5. **表格宽度**：
   - 填充指定区域宽度
   - 避免表格在空白区域浮动

### XML实现模式，请参见[`xml-reference.md`](xml-reference.md#表格实现)

## 图表和图片处理

### 从Excel粘贴图表

从Excel粘贴图表时：

1. **只粘贴图表** - 不包含源数据表
2. **调整大小以填充指定区域** - 图表不应显示为小缩略图
3. **保持纵横比** - 不扭曲图表
4. **验证可读性** - 轴标签、图例、数据标签必须清晰可读

### 从Excel粘贴表格

从Excel粘贴表格时：

1. **只粘贴格式化表格** - 排除任何源数据或计算
2. **调整大小以填充指定区域** - 表格应占据其完整区域
3. **验证列宽** - 调整以使文本不被截断
4. **检查格式保留** - 颜色、边框、字体可能需要调整

### 大小指南

**最小尺寸原则：**
- 图表：应占据其指定区域的相当大部分
- 表格：完全填充指定区域宽度
- 图片：根据上下文适当调整大小，绝不要缩略图大小

**尺寸过小的视觉指标（避免这些）：**
- 图表仅占据可用空间的一小部分
- 文本标签无法阅读
- 视觉元素周围有大片空白区域
- 视觉元素显示为"缩略图"

### 正确尺寸调整流程

1. 确定目标区域尺寸
2. 粘贴图表/表格
3. 立即调整大小以填充目标区域
4. 验证所有文本保持可读
5. 必要时调整内部元素（图例位置、轴标签）

## 数据可视化

### 关键指标展示

显示关键指标（如TAM、CAGR、预测）时，考虑显示值之间的关系而非静态列出：

- **视觉流程指示器**：显示进展的形状（箭头、V形、连接器）
- **大小层次**：主要指标使用较大字体，标签使用较小字体
- **空间排列**：定位元素以显示逻辑流程

### 箭头和流程指示器

如果使用箭头或流程指示器：
- 使用PowerPoint形状对象，而不是文本字符
- 最终演示文稿中不使用基于文本的箭头（→, ⟹）
- 使用PowerPoint的形状工具或XML形状元素创建箭头

**XML实现，请参见[`xml-reference.md`](xml-reference.md#箭头形状)**

## 字体一致性

### 跨框字体一致性

同一层级的所有文本框应使用相同的字体大小。

**应匹配的同级框：**

| 框类型 | 应与以下匹配 |
|----------|-------------------|
| "包含细分市场"内容 | "排除细分市场"内容 |
| "定义"内容 | "范围理由"内容 |
| 左列项目符号 | 右列项目符号 |
| 所有标签框 | 彼此 |
| 所有区域标题 | 彼此 |

### 验证流程

1. 识别同一层级的所有文本框
2. 检查每个框的字体大小
3. 如果任何框不同，调整所有框以匹配
4. 如果内容适合，默认使用较大尺寸；否则一致使用较小尺寸

**例外**：子项目符号或次要文本可以使用比主项目符号更小的字体，但这必须在所有框中保持一致。

## 模板适配

这些标准应根据所使用的具体模板进行调整：

1. **颜色**：使用模板的品牌颜色而非指定特定颜色
2. **字体**：使用模板的字体家族
3. **间距**：匹配模板的现有间距约定
4. **布局**：遵循模板的区域结构

无论模板如何，保持不变的关键原则：
- 文本在其背景上必须可读
- 表格必须是实际的表格对象
- 内容应适当填充可用空间
- 平行元素的格式应一致
- 图表/图片应正确调整大小

## 📚 金融术语和知识解释

### 1. Pitch Deck（推介材料）

**定义**：Pitch Deck是投资银行或初创企业向潜在投资者展示的演示文稿，用于介绍公司业务、市场机会、财务状况和投资亮点。

**关键组成部分**：
- **执行摘要**：公司业务的简要概述
- **市场机会**：TAM/SAM/SOM分析
- **竞争分析**：竞争对手对比
- **商业模式**：盈利方式说明
- **财务预测**：收入、利润预测
- **团队介绍**：核心团队背景

**重要性**：Pitch Deck是融资过程中的关键文件，直接影响投资者的决策。

### 2. TAM/SAM/SOM（市场规模术语）

**定义**：
- **TAM（Total Addressable Market）**：总可寻址市场，指产品或服务可以服务的最大潜在市场规模
- **SAM（Serviceable Addressable Market）**：可服务的可寻址市场，指公司实际可以服务的市场部分
- **SOM（Serviceable Obtainable Market）**：可获得的服务市场，指公司在特定时间内可以实际获得的市场份额

**计算公式**：
```
TAM = 目标客户数量 × 平均客户价值
SAM = TAM × 服务区域渗透率
SOM = SAM × 预期市场份额
```

**应用场景**：在Pitch Deck中展示市场机会时使用，帮助投资者理解增长潜力。

### 3. CAGR（复合年增长率）

**定义**：Compound Annual Growth Rate，指一项投资在特定时期内的平均年增长率。

**计算公式**：
```
CAGR = (最终值 / 初始值)^(1/年数) - 1
```

**示例**：
- 如果某市场从100亿元增长到200亿元，用时5年
- CAGR = (200/100)^(1/5) - 1 ≈ 14.87%

**重要性**：CAGR是衡量业务增长的关键指标，帮助投资者评估增长速度和可持续性。

### 4. EBITDA（息税折旧摊销前利润）

**定义**：Earnings Before Interest, Taxes, Depreciation, and Amortization，是衡量公司经营业绩的指标。

**计算公式**：
```
EBITDA = 营业收入 + 利息费用 + 所得税费用 + 折旧 + 摊销
```

**用途**：
- 评估公司核心业务盈利能力
- 比较不同资本结构的公司
- 用于估值倍数计算（如EV/EBITDA）

**注意**：EBITDA不考虑资本支出和营运资金需求，需结合其他指标综合分析。

### 5. EV（企业价值）

**定义**：Enterprise Value，是衡量公司总价值的指标，包括股权价值和债务价值。

**计算公式**：
```
EV = 股权价值 + 债务 - 现金及等价物
```

**组成部分**：
- **股权价值**：公司股票市值
- **债务**：包括短期和长期债务
- **现金及等价物**：公司持有的现金和可随时变现的资产

**用途**：用于计算估值倍数，如EV/Revenue、EV/EBITDA。

### 6. 估值倍数（Valuation Multiples）

**定义**：用于比较不同公司价值的比率，是投资银行和私募股权领域常用的估值方法。

**常见倍数**：
| 倍数类型 | 计算公式 | 适用场景 |
|----------|----------|----------|
| EV/Revenue | 企业价值 / 收入 | 成长型公司、科技公司 |
| EV/EBITDA | 企业价值 / EBITDA | 成熟企业、制造业 |
| P/E | 股价 / 每股收益 | 公开上市公司 |
| P/B | 股价 / 每股账面价值 | 金融机构 |

**使用原则**：
- 选择与目标公司业务模式匹配的倍数
- 使用可比公司或交易数据作为参考
- 注意不同会计处理对倍数的影响

### 7. 可比公司分析（Comparable Company Analysis）

**定义**：通过比较目标公司与同行业其他上市公司的财务指标和估值倍数，来评估目标公司价值的方法。

**步骤**：
1. **选择可比公司**：业务相似、规模相近的上市公司
2. **收集财务数据**：收入、EBITDA、净利润等
3. **计算估值倍数**：EV/Revenue、EV/EBITDA等
4. **应用倍数**：将平均倍数应用于目标公司指标
5. **得出估值**：计算目标公司的合理价值范围

**注意事项**：
- 确保可比公司具有可比性
- 考虑市场周期和行业趋势
- 调整非经常性项目的影响

### 8. 交易可比分析（Transaction Comparables）

**定义**：通过分析同行业近期并购交易的估值倍数，来评估目标公司价值的方法。

**优势**：
- 反映实际市场交易价格
- 包含协同效应预期
- 适用于并购估值场景

**数据来源**：
- 公开并购公告
- 专业数据库（如S&P Capital IQ、PitchBook）
- 新闻报道和行业报告

### 9. 幻灯片设计原则

**视觉层次**：
- 使用字体大小和粗细区分信息重要性
- 标题 > 副标题 > 正文 > 脚注
- 关键数据使用更大字体突出显示

**留白原则**：
- 不要过度填充内容
- 适当留白提高可读性
- 保持视觉平衡

**一致性**：
- 整个演示文稿使用统一的字体、颜色和布局
- 项目符号样式保持一致
- 图表风格统一

### 10. 投资者演示技巧

**时间管理**：
- 标准Pitch Deck通常10-15分钟完成
- 每页停留时间约1-2分钟
- 重点突出，避免信息过载

**讲故事技巧**：
- 从问题出发，引出解决方案
- 展示市场机会和增长潜力
- 突出竞争优势和差异化

**Q&A准备**：
- 预测常见问题
- 准备数据支持答案
- 保持回答简洁准确
