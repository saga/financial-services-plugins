# 财务数据包分析 (Data Pack Builder)

为私募股权（PE）、投资银行和资产管理机构构建专业、标准化的财务数据分析包。将零散的原始数据转化为经过精细打磨、可供投资委员会直接审阅的 Excel 模型。

**核心原则：** 整个工作流必须调用 `xlsx` 技能进行所有的 Excel 文件创建与操作。

## 成功交付的三大支柱

### 1. 数据准确性（零误差容忍）
- **数据回溯**：每一个数字必须能追踪到源文档的具体页码（Page Reference）。
- **公式驱动**：除历史初值外，所有计算必须使用公式，禁止硬编码（Hardcoding）。
- **钩稽检查**：
  - 资产 = 负债 + 权益（资产负债表必须配平）。
  - 现金流量表净额必须与资产负债表现金变动一致。

### 2. 核心格式规范

- **规则 1：金额类数据 (Money)** -> 使用货币格式（带 $ 或 ¥）。负数使用括号 `(123)` 而非 `-123`。
- **规则 2：运营类数据 (Volumes)** -> 使用数值格式（带千分位，无货币符号）。用于店数、员工数、产量等。
- **规则 3：百分比 (Ratios)** -> 使用百分比格式，保留一位小数（如 `15.0%`）。
- **规则 4：字体颜色（关键规范）**：
  - **蓝色 (RGB: 0,0,255)**：所有手动输入的历史数据或假设值（Hardcoded inputs）。
  - **黑色 (RGB: 0,0,0)**：所有公式和计算结果。
  - **绿色 (RGB: 0,128,0)**：跨工作表引用（Links to other sheets）。

### 3. 标准化结构 (8-Tab Structure)
除非另有说明，数据包应包含以下 8 个标准工作表：
1. **Executive Summary**：执行摘要（一页纸精华）。
2. **Historical P&L**：历史利润表（收入、毛利、EBITDA 拆解）。
3. **Balance Sheet**：资产负债表（流动性及负债分析）。
4. **Cash Flow**：现金流量表（间接法，核对现金生成能力）。
5. **Op Metrics**：运营指标（KPI、单位经济效益分析）。
6. **Segment Analysis**：分业务线/区域表现。
7. **Market Analysis**：市场规模及竞争格局。
8. **Investment Highlights**：投资亮点及风险。

## 操作流程 (Workflow)

1. ** Phase 1: 提取与清洗** —— 处理源文件，建立日期和科目的对应关系。
2.  **Phase 2: 标准化调整 (Normalization)** —— 识别并调整非经常性损益（如一次性重组成本、资产处置损益）。
3.  **Phase 3: 模型构建** —— 在 Excel 中建立完整的钩稽关系，应用蓝/黑字体规范。
4.  **Phase 4: 场景模拟** —— 在管理层预测（Management Case）基础上，建立风险调减后的基本案（Base Case）。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **Data Pack (数据包/分析包)**：在 PE 投资中，这是投资经理最基础也最繁重的工作。如果把投资比作买房，Data Pack 就是那份包含周边房价、房子结构、漏水记录、水电费在内的所有底层数据的“超级体检报告”。
> 2. **蓝黑字体规范 (Blue and Black Convention)**：这是华尔街和顶级投资机构雷打不动的标准。看到蓝色字体，你就知道这个数字是“死”的，是从报表上抄来的；看到黑色字体，说明它是“活”的，是根据模型公式算出来的。这样在改动一个假设时，分析师能瞬间知道哪些数字会联动。
> 3. **标准化调整 (Normalization / Adjustment)**：公司提供的利润往往是有“水分”或“杂质”的。比如，去年公司写字楼着火赔了一笔钱，这叫非经常性损失，今年不会再发生。我们需要把这些杂项加回去或减掉，还原出业务真实的、可持续的盈利能力。
> 4. **钩稽关系 (Articulation)**：财务报表不是孤立的。利润表最后一行的利润，会流进资产负债表的留存收益；资产负债表上的现金变动，必须等于现金流量表的结果。这叫“三表联动”，错一个数字，整个模型就崩了。

## Appendix: 金融背景知识

这份文件是"财务数据包（Data Pack Builder）"技能的详细说明。在 PE 和投行中，Data Pack 是分析一家公司的"基础资料包"——它把零散、混乱的原始财务数据整理成标准化、可直接用于分析和汇报的 Excel 文件。

---

### 1. 什么是 Data Pack？

**类比：**
想象你是一个厨师，准备做一道复杂的菜。你的食材（原始数据）散落在菜市场各处：有的在肉铺，有的在鱼摊，有的在蔬菜店。Data Pack 就是你把这些食材全部买回来、洗净、切好、分类装盘的"备菜过程"。

在金融业中，Data Pack 就是"备好的菜"——原始数据已经清洗、标准化、格式化，分析师可以直接用来"炒菜"（建模和估值）。

---

### 2. Data Pack 的"三大支柱"

#### 支柱 1：数据准确性
- 每一个数字必须能追溯到源文档的具体页码
- 公式驱动：除历史初值外，所有计算必须使用公式
- 钩稽检查：三张表必须配平

#### 支柱 2：格式规范
- 金额类数据使用货币格式
- 运营类数据使用数值格式
- 百分比保留一位小数
- 颜色编码：蓝色=输入，黑色=公式，绿色=链接

#### 支柱 3：可读性
- 清晰的标签和注释
- 标准化的工作表命名
- 关键假设单独列示

---

### 3. Data Pack 的"标准结构"

| 工作表 | 内容 |
|--------|------|
| 假设（Assumptions） | 所有关键假设集中展示 |
| 利润表（P&L） | 收入、成本、利润 |
| 资产负债表（BS） | 资产、负债、权益 |
| 现金流量表（CF） | 经营、投资、筹资现金流 |
| 财务比率 | 盈利能力、运营效率、偿债能力 |
| 可比公司 | 同行业公司的关键指标对比 |

---

### 4. 真实案例：某 PE 的 Data Pack 失误

某 PE 基金在尽调中收到的 Data Pack 存在以下问题：

| 问题 | 后果 |
|------|------|
| 历史数据硬编码 | 无法追溯数据来源 |
| 三张表不配平 | 模型不可信 |
| 缺少数据来源标注 | 无法验证准确性 |
| 格式混乱 | 投决会看不懂 |

结果：投决会要求重新做 Data Pack，延误 2 周，最终错失交易窗口。

### 给小白的一句话

> Data Pack 就是金融分析的"食材准备"——你不可能把生肉、生菜、生米直接端上桌，必须先洗、切、配好。Data Pack 就是把公司乱七八糟的财务报表、银行对账单、税务文件整理成"一盘精致的菜"（标准化的 Excel 模型），让投资委员会能直接"品尝"（看懂并做出决策）。如果 Data Pack 做得烂（数字错误、格式混乱、来源不明），就像端上一盘没洗干净的菜——客户不仅不会吃，还会质疑你的专业性。
