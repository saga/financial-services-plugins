## Appendix: 金融背景知识

这份文件是"交易追踪器（Deal Tracker）"的命令说明。在投行里，Deal Tracker是所有初级员工（Analyst和Associate）最常用但又最不被人看见的"隐形工具"。它不像Pitch Deck（精美PPT）那样能展示你的设计才华，也不像Financial Model（Excel财务模型）那样能体现你的数学功底——但它如果管理不好，会导致你丢掉客户、漏掉重要截止日、甚至让一宗原本能赚几千万佣金的交易因为沟通失误而流产。

---

### 1. 投行中的一个典型交易涉及多少方人马？

让我们用一个真实规模的并购交易来感受一下"为什么需要追踪器"：

假设某全球500强制药公司（买方）要花$50亿收购一家生物技术公司（目标）。在整个交易过程中，以下各方会参与其中，每天进行高频沟通：

| 参与方 | 角色 | 大概人数 |
|-------|------|---------|
| 买方公司内部团队 | 拍板、提供资金 | 5-10人（CEO、CFO、战略部、法务部） |
| 卖方公司内部团队 | 谈判、提供资料 | 5-10人（CEO、CFO、董事会） |
| 买方投行 | 顾问、算账、谈判 | 10-20人（MD, VP, Associate, Analyst） |
| 卖方投行 | 顾问、营销、推进流程 | 10-20人 |
| 买方律所 | 法律尽调、合同起草 | 5-10人 |
| 卖方律所 | 法律尽调、合同审查 | 5-10人 |
| 审计师（四大） | 财务尽调 | 5-10人 |
| 税务顾问 | 交易结构税务优化 | 2-5人 |
| 行业顾问 | 市场/竞争尽调 | 2-5人 |
| **总计** | | **50-90+人** |

这50-90个人分散在不同的公司、不同的时区（如果买方在纽约、目标公司在伦敦，你就需要协调EST和GMT两个时区），不同的邮件系统——如果管理不当，就是一场灾难。

### 2. Deal Tracker 追踪什么？

#### 2.1 关键里程碑（Milestones）

一个经典并购交易的关键时间节点：

| 里程碑 | 说明 | 谁负责 |
|--------|------|-------| 
| NDA签署 ✅ | 保密协议签署，信息可以正式交换 | 法务团队 |
| 管理层介绍会议 | 买方高层第一次见卖方管理层 | 投行安排 |
| VDR（虚拟资料室）开放 | 机密数据在线阅览开放 | 卖方投行 |
| 初步尽调完成 | 买方完成初步业务和财务尽调 | 买方团队+顾问 |
| IOI提交截止 | 买方提交非约束性初步报价 | 买方投行 |
| 详细尽调启动 | 开放更多数据+管理层访谈 | 各方 |
| Final Bid 截止 | 提交最终约束性报价 | 买方投行 |
| 签署SPA（股权购买协议） | 正式签合同 | 法务+投行+双方 |
| 监管审批 | 反垄断、FDI等审批 | 法务 |
| 交割（Closing） | 钱到账，股权过户 → 完成 | 所有人 |

#### 2.2 待办清单（Action Items）

每一次会议（不管是电话会、视频会还是当面会）结束后的30分钟内，投行初级员工必须完成一份"会议纪要+待办清单"，并发给所有参会人员。这听起来简单，但在一个50-90人的项目中，遗漏任何一条都可能造成重大损失。

**典型待办清单格式：**

| 编号 | 待办事项 | 负责人 | 截止时间 | 优先级 | 状态 |
|-----|---------|-------|---------|-------|------|
| 1 | 提供2023年按地区分拆的收入数据 | 卖方CFO团队 | 3月15日 | 🔴 高 | 逾期 |
| 2 | 回答买方关于客户集中度的第二轮问卷 | 卖方CEO | 3月18日 | 🟡 中 | 进行中 |
| 3 | 发送修订后的SPA草案（含价格调整机制） | 卖方律所 | 3月22日 | 🔴 高 | 待完成 |
| 4 | 安排下一轮管理层访谈（技术团队） | 卖方投行 | 3月20日 | 🟡 中 | 完成 |

#### 2.3 买方沟通追踪

对每一个潜在买家，追踪以下信息：
- 收到材料了吗？（Teaser已发？CIM已发？）
- NDA签了没有？
- VDR看了没有？看了多长时间？（如果一家买家的分析团队在VDR里泡了200小时，说明他们是认真的）
- 提交报价了没有？
- 有什么特殊问题或顾虑？
- 他们的内部决策进度到哪了？

### 3. 投行里 Deal Tracker 的各种形态

虽然市面上有很多"专业项目管理软件"（如Asana、Monday.com等），但在实际投行工作中，Deal Tracker 的表现形式比你想象的要土得多：

**初级员工的噩梦：Excel版 Deal Tracker**
大多投行的 Deal Tracker 是Excel文件，A列：事项，B列：负责人，C列：截止日期，D列：状态。这个Excel会被邮件反复"飞来飞去"——Analyst更新了一个截止日期、发给所有人；Associate加了两条待办事项、再发给所有人；VP改了优先级，又发一遍。一天下来，同一个Excel可能被发了8-10次。

**更高级的做法：共享文档**
用Google Sheet/腾讯文档/Notion来追踪，所有人可以实时看到更新，不用担心版本冲突。

**最专业的做法：Deal Management Platform**
大投行有自己的内部系统来管理所有的Live Deals（在线项目），这些系统集成了CRM、文档管理、邮件追踪、时间线等功能，但一般只对内部团队开放。

### 4. Deal Tracker 管理的常见血泪教训

| 教训 | 真实场景 | 后果 |
|------|---------|------|
| 漏掉一个截止日 | A公司必须在下周五前提交补充尽调材料，结果投行忘记提醒了 | 买方以为卖方不配合，退出交易 |
| 行动项分配模糊 | "由卖方团队提供"——到底是谁？CEO？CFO？法务？ | 所有人都以为别人在做，结果没人做 |
| 版本混乱 | 同一时间有3个不同版本的"最新Deal Tracker"在流通 | 有人按照过时的截止日期排了工作 |
| 保密信息泄露 | Deal Tracker被不小心发给了错误的买家名单 | 严重违反保密协议，客户起诉投行 |

### 给小白的一句话

> Deal Tracker就是投行的"项目施工总指挥中心"——一个几十亿美金的并购交易，涉及律师、会计师、银行家、企业高管等50-100号人、几十条关键时间节点、每天新增几十条待办事项。如果没有人把这些信息集中在一个地方管理，整个项目就像一个没有指挥的乐队——各吹各的号、各拉各的调，最终变成一团噪音而不是一首优美的交响乐。
