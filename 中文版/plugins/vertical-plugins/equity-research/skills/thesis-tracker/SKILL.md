# 投资逻辑跟踪器 (Thesis Tracker)

description: 维护并更新投资组合持仓及观察名单的投资逻辑。跟踪关键数据点、催化剂以及随时间推移的投资逻辑里程碑。适用于根据新信息更新投资逻辑、评估持仓理由或检查投资逻辑是否依然成立的情况。触发词包括“更新 [公司] 的投资逻辑”、“我的投资逻辑是否依然成立”、“逻辑核查”、“为 [公司] 添加数据点”或“审查我的持仓”。

## 工作流程

### 第 1 步：定义或加载投资逻辑

若创建新逻辑：
- **公司**：名称与股票代码
- **头寸**：多头 (Long) 或 空头 (Short)
- **投资逻辑陈述**：1-2 句核心逻辑（例如：“做多 ACME —— 随着产品组合转向软件业务，凭借定价能力和运营杠杆实现利润率扩张”）
- **核心支柱**：3-5 个支持性论据
- **核心风险**：3-5 个可能导致逻辑失效的风险因素
- **催化剂**：可能证实或推翻逻辑的近期事件（财报发布、产品上线、监管决定）
- **目标价 / 估值**：如果逻辑兑现，该资产价值几何
- **止损触发点**：什么情况下你会选择平仓退出

若更新现有逻辑，请询问用户新的数据点或事态进展。

### 第 2 步：更新日志

针对每个新数据点或进展：

- **日期**：事件发生时间
- **数据点**：发生了什么变化（财报超预期、管理层变动、竞争对手动作等）
- **逻辑影响**：该点是加强、削弱还是中和了某个特定支柱？
- **操作**：维持不变 / 加仓 / 减仓 / 清仓
- **最新确定性**：高 / 中 / 低

### 第 3 步：投资逻辑计分卡

维护一份动态计分卡：

| 核心逻辑支柱 | 初始预期 | 当前状态 | 趋势 |
|--------|---------------------|----------------|-------|
| 营收增长 >20% | 符合预期 | 三季度增长 22% | 稳定 |
| 利润率扩张 | 滞后 | 利润率同比持平 | 令人担忧 |
| 新产品发布 | 待定 | 推迟至二季度 | 观察 |

### 第 4 步：催化剂日历

跟踪即将发生的催化剂：

| 日期 | 事件 | 预期影响 | 备注 |
|------|-------|-----------------|-------|
| | | | |

### 第 5 步：输出

适用于以下场景的投资逻辑摘要：
- 晨会讨论
- 投资组合审查
- 风险委员会陈述

格式：包含计分卡、最新更新及当前确定性水平的精炼 Markdown 或 Word 文档。

## 重要说明

- 投资逻辑必须是**可证伪**的 —— 如果没有任何证据能推翻它，那它就不是投资逻辑，而是偏见。
- 跟踪“证实证据”的同时，必须同样严格地记录“证伪证据”。
- 即使没有发生重大事件，也应至少每季度审查一次投资逻辑。
- 如果用户管理多个头寸，建议进行完整的投资组合逻辑审查。
- 以结构化格式存储投资逻辑数据，以便跨会话参考。

> **💡 Appendix: 领域知识小贴士**
>
> *   **投资逻辑 (Thesis)**：简单来说，就是你买入一只股票的“剧本”。没剧本的买入叫“盲赌”，有剧本的才叫“投资”。
> *   **多头/空头 (Long/Short)**：“多头”就是看好它，觉得它会涨；“空头”则是看衰它，赌它会跌。
> *   **催化剂 (Catalyst)**：能给股价“点火”的事件。比如，一款新游戏上线，或者是公司换了个厉害的 CEO。
> *   **利润率扩张 (Margin Expansion)**：公司赚钱更有本事了。比如以前做 100 块生意赚 10 块，现在能赚 15 块，说明它的成本控制更强或东西卖得更贵了。
> *   **运营杠杆 (Operating Leverage)**：一种“规模效应”。比如写个代码，卖给 1 个人和卖给 1 万个人成本差不多，那卖得越多，剩下的几乎全都是纯利润。
> *   **可证伪性 (Falsifiability)**：这是投资高手的防翻车指南。你得提前写好：“如果发生 A 这种倒霉事，就证明我想错了，我就认亏走人”，而不是死缠烂打。
