# NAV 核对 (NAV Tie-out)

根据已生成的 LP 报告和当期的 NAV 估值包（通过 nav MCP 获取），独立重新计算 LP 的资本账户，并进行逐行比对。

> **生成的报告是待测对象。** NAV 估值包是唯一事实来源 (Source of truth)。

## 重新计算 LP 资本账户

```
期初资本 (上一期报告期末值)
  + 出资 (本期已支付的资本缴付)
  − 分配 (现金 + 实物分配)
  + 分配的净损益 (收益/亏损)
      = LP 持股比例 × (已实现 + 未实现盈亏 − 管理费 − 基金费用)
  − 业绩报酬 (Carried Interest) 分配 (若在本期触发/提取)
期末资本
```

从 NAV 估值包中提取各项输入值：LP 认缴比例、基金级损益组件、费用及开支总额、瀑布式分配结果。

## 核对 (Compare)

将报告中的每行内容与你重新计算的值进行比较。容差：`0.01`。对于每个不匹配项，注明驱动该差异的输入因素（例如，“分配的损益不同——报告使用了 12.40% 的持股比例，而 NAV 估值包显示 Q1 转让后持股比例为 12.38%”）。

## 额外检查

- 本期报告的期末资本 = 下期草案的期初资本（如有）。
- 所有 LP 期末资本之和 = 基金 NAV（在舍入误差范围内）。
- 认缴额 (Commitment)、未缴额 (Unfunded) 和可回收额 (Recallable) 与认缴登记簿一致。

## 输出

每行提供通过/失败状态，将重新计算的值与报告值并列显示，并列出标记项。请勿修改原始报告——发布者将在审核标记项后采取后续行动。

> **💡 Appendix: 领域知识小贴士**
>
> - **NAV (资产净值)**：简单来说，这就是基金的“家底”。把基金手里所有的投资项目估值、现金加起来，扣掉还没付的欠款，剩下的就是 NAV。
> - **LP (有限合伙人)**：就是基金的投资人。他们负责出钱，但不参与日常管理。
> - **Tie-out (核对/勾稽)**：这是金融界的“终极对账”。我们要确保发给投资人的报告和基金后台的最原始账本能完全对上，连一分钱的误差都不能放过。
> - **Capital Call (资本缴付)**：当基金发现好项目要投钱时，会向 LP “呼叫”拿钱。LP 签约时承诺投 1000 万，但通常是分批给，这次要 100 万，就叫一次出资。
> - **Waterfall (瀑布式分配)**：一种分钱规则。赚了钱之后，水（现金流）会按顺序流过每一层台阶：先还 LP 的本金，再给 LP 约定的保底收益，剩下的最后才轮到基金管理人分红。
