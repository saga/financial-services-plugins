# 差异说明 (Variance commentary)

给定本期实际数、上期实际数以及相同口径的预算数，生成一份差异说明表。

## 阈值 (Threshold)

若符合以下**任一**条件，则该项需进行详细说明：

- 绝对差异 ≥ 公司设定的重要性水平（使用提供的数值；默认为该行金额的 5% 或固定的最低限额，取其大者）。
- 属于“必须说明”清单中的项目（如：收入、人力成本、现金）。

## 针对每个标记的行项目

| 列名 | 内容说明 |
|---|---|
| **项目 (Line)** | 会计科目或报表项名称 |
| **本期/上期/预算** | 对应的三个数值 |
| **较上期变动 (Δ vs prior)** 和 **较预算变动 (Δ vs budget)** | 差额及其百分比 |
| **驱动因素 (Driver)** | 一句话解释变动的底层原因，而非对数字的重复陈述 |

驱动因素解释的是“为什么 (Why)”，而不是“是什么 (What)”：“因 5 月产品上线增加了 GPU 预留实例，导致云服务支出增加 120 万美元” —— 正确；“云服务支出增加了 120 万美元 (18%)” —— 错误（这只是在念数字）。

## 追溯驱动因素

通过内部总账 (internal-gl) MCP 查看该项目背后的具体活动（日记账来源明细、供应商构成、人员变动、量价分析）。如果从数据中无法明确判断原因，应写明“驱动因素不明——标记待财务主管复核”，严禁凭空编造。

## 输出要求

差异说明表以及一段简练的叙述性总结（3-5 句），概括本期间变动最显著的项目。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **差异分析 (Flux analysis / Variance Analysis)**：这是财务分析的基础。如果你的花费比上个月多了 20%，老板肯定想知道钱花哪了。这种分析帮助管理层判断公司是运营效率降低了，还是业务规模扩大了。
> 2. **重要性水平 (Materiality)**：在财务审计中，我们不需要关注每一个“一分钱”的差异。公司会设一个“及格线”（阈值），超过这个金额的变动才值得花时间去调查。
> 3. **驱动因素 (Driver)**：这是金融分析的灵魂。老板不喜欢听“因为数字变大了所以变大了”，他想听的是“因为我们新招了 50 个员工，所以工资支出涨了”。找到那个“开关”或“拉动点”，就是找到了驱动因素。
> 4. **量价分析 (Volume × Rate)**：解释驱动因素的常用模型。比如收入增加，是因为我们卖的东西变多了（量），还是因为我们涨价了（价）。

## Appendix: 金融背景知识

这份文件是"差异说明（Variance Commentary）"技能的详细说明。在基金管理中，每月的财务报告不仅要展示数字，还要解释"为什么这个月比上个月多/少了"。Variance Commentary 就是给海量财务数据"加注释"——让阅读报告的人（管理层、LP）快速理解变化的原因。

---

### 1. 什么是 Variance Commentary？

**类比：**
想象你每个月的体重记录：
- 上个月：80 公斤
- 这个月：78 公斤
- 差异：-2 公斤

"-2 公斤"只是数字。Variance Commentary 会解释："减了 2 公斤，因为本月每天跑步 5 公里 + 戒掉了宵夜。"

在基金中，差异说明就是解释"为什么这个月的收入少了 5%、费用多了 3%"——而不只是展示数字。

---

### 2. Variance 分析的"三个维度"

#### 维度 1：本期 vs 上期（MoM / QoQ）
- 这个月比上个月差在哪里？
- 是季节性波动还是趋势性变化？

#### 维度 2：本期实际 vs 预算（Actual vs Budget）
- 收入是否达标？费用是否超支？
- 预算编制是否合理？

#### 维度 3：本期 vs 去年同期（YoY）
- 业务是否在增长？
- 结构是否有变化？

---

### 3. Variance 说明的"黄金法则"

#### 法则 1：不要重复数字
```
错误："收入下降了 100 万，因为收入下降了100万。"
正确："收入下降了 100 万，因为最大客户 ABC 公司本季度未续约，导致 80 万收入损失。"
```

#### 法则 2：区分"可控"和"不可控"
```
可控：营销费用超支 → 因为增加了市场活动
不可控：收入下降 → 因为行业整体萎缩
```

#### 法则 3：量化驱动因素
```
错误："成本增加了。"
正确："成本增加了 15%（300万），其中人工成本增加 200万（新招 10人），原材料增加 100万（钢材涨价20%）。"
```

---

### 4. 阈值（Threshold）的设定

不是所有差异都需要说明——只有"显著"的差异才值得解释。

| 阈值类型 | 说明 | 例子 |
|---------|------|------|
| 百分比阈值 | 差异超过 ±5% | 收入下降 6%，需要说明 |
| 绝对金额阈值 | 差异超过 ±$10 万 | 费用增加 2%，但只有 $2000，不需要说明 |
| 必须说明清单 | 特定项目必须说明 | 收入、人工成本、现金流 |

---

### 5. 真实案例：某基金的月度差异说明

**背景**：某对冲基金 3 月 vs 2 月的差异分析

| 项目 | 2 月 | 3 月 | 差异 | 说明 |
|------|------|------|------|------|
| 管理费 | $100 万 | $150 万 | +$50 万 | 新 LP 加入，承诺资本增加 $5000 万 |
| 交易费用 | $20 万 | $40 万 | +$20 万 | 季度末集中调仓，交易量翻倍 |
| 法律服务费 | $10 万 | $30 万 | +$20 万 | 一起诉讼的法律费用一次性支出 |
| 收入分配 | $200 万 | $800 万 | +$600 万 | 两只被投企业 IPO 退出，实现退出收益 |

---

### 给小白的一句话

> Variance Commentary 就是财务报告的"弹幕解说"——纯数字的报表像一本没有注释的教科书，看起来干巴巴的。差异说明就像弹幕一样在关键数字旁边加解释："这个月多赚了500万，是因为卖掉了两家公司（退出收益）"。好的差异说明要做到"一针见血"：不看数字的人，通过读说明也能理解发生了什么。记住三个字——"Why"（为什么）、"What"（影响什么）、"Now What"（接下来怎么办）。
