# 差异说明 (Variance commentary)

给定本期实际数、上期实际数以及相同口径的预算数，生成一份差异说明表。

## 阈值 (Threshold)

若符合以下**任一**条件，则该项需进行详细说明：

- 绝对差异 ≥ 公司设定的重要性水平（使用提供的数值；默认为该行金额的 5% 或固定的最低限额，取其大者）。
- 属于“必须说明”清单中的项目（如：收入、人力成本、现金）。

## 针对每个标记的行项目

| 列名 | 内容说明 |
|---|---|
| **项目 (Line)** | 会计科目或报表项名称 |
| **本期/上期/预算** | 对应的三个数值 |
| **较上期变动 (Δ vs prior)** 和 **较预算变动 (Δ vs budget)** | 差额及其百分比 |
| **驱动因素 (Driver)** | 一句话解释变动的底层原因，而非对数字的重复陈述 |

驱动因素解释的是“为什么 (Why)”，而不是“是什么 (What)”：“因 5 月产品上线增加了 GPU 预留实例，导致云服务支出增加 120 万美元” —— 正确；“云服务支出增加了 120 万美元 (18%)” —— 错误（这只是在念数字）。

## 追溯驱动因素

通过内部总账 (internal-gl) MCP 查看该项目背后的具体活动（日记账来源明细、供应商构成、人员变动、量价分析）。如果从数据中无法明确判断原因，应写明“驱动因素不明——标记待财务主管复核”，严禁凭空编造。

## 输出要求

差异说明表以及一段简练的叙述性总结（3-5 句），概括本期间变动最显著的项目。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **差异分析 (Flux analysis / Variance Analysis)**：这是财务分析的基础。如果你的花费比上个月多了 20%，老板肯定想知道钱花哪了。这种分析帮助管理层判断公司是运营效率降低了，还是业务规模扩大了。
> 2. **重要性水平 (Materiality)**：在财务审计中，我们不需要关注每一个“一分钱”的差异。公司会设一个“及格线”（阈值），超过这个金额的变动才值得花时间去调查。
> 3. **驱动因素 (Driver)**：这是金融分析的灵魂。老板不喜欢听“因为数字变大了所以变大了”，他想听的是“因为我们新招了 50 个员工，所以工资支出涨了”。找到那个“开关”或“拉动点”，就是找到了驱动因素。
> 4. **量价分析 (Volume × Rate)**：解释驱动因素的常用模型。比如收入增加，是因为我们卖的东西变多了（量），还是因为我们涨价了（价）。
