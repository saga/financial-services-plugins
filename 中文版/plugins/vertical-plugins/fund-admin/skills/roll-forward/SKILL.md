# 余额变动调节 (Roll-forward)

给定账户（或账户组）、实体及期间，生成一份将期初余额与期末余额对齐的变动表。

## 结构

```
期初余额 (根据上期结账数据)             X
  + 增加项 / 新发生的活动              A
  + 本期计提 (Accruals)               B
  - 上期计提冲回 (Reversals)          (C)
  - 支付 / 结算 (Payments)            (D)
  ± 重分类 / 调整 (Reclasses)          E
  ± 汇兑转换 (FX)                      F
期末余额 (根据本期期末总账 GL)          Y
```

## 逐行核对 (Tie-out)

- **期初余额** — 引用上期结账包或上期期末的总账余额。
- **每一项活动行** — 通过内部总账 (internal-gl) MCP 进行 GL 查询（账户 + 日期范围 + 日记账来源过滤）。需引用查询结果。
- **期末余额** — 引用本期期末日期的总账余额。

调节表**必须算通 (Footing)**：`X + A + B - C - D + E + F = Y`。如果不相等，差额即为“未解释项”——请如实反映，严禁为了凑平而随意填补数值。

## 输出要求

包含“核对至 (Ties to)”列的变动表，该列需注明每一行所引用的 GL 查询或文档。此外还需包含计算准确性检查（通过/失败，以及如有未解释的垂直差额）。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **Roll-forward (余额变动调节)**：在财务中，这就像追踪一个水池里的水量。你不仅要知道月底池里有多少水（期末余额），还要清楚月初有多少（期初），这个月流进来多少（增加/计提），又排出去多少（支付/冲回）。由于金融机构对准确性要求极高，每一滴水的流向都必须有据可查。
> 2. **总账 (General Ledger, GL)**：它是公司所有财务数据的“大账本”。无论是买办公用品还是支付上亿的投资款，最后都会在 GL 里体现。
> 3. **算通 (Footing)**：这是一个会计行话，指垂直核对表格中的数字相加是否等于总数。在电子表格时代，这看起来很简单，但在涉及复杂的汇率调节和手工调整时，经常会出现差错。
> 4. **计提 (Accruals)**：一种“先记账后给钱”的原则。比如，你这个月用了电，但电费单还没来，你必须先“预估”一笔电费记在账上。

## Appendix: 金融背景知识

这份文件是"余额变动调节（Roll-forward）"技能的详细说明。在基金会计中，Roll-forward 是连接"上期余额"和"本期余额"的桥梁——它详细展示了一笔资产或负债在期间内是如何变化的。

---

### 1. 什么是 Roll-forward？

**类比：**
想象你有一个零花钱账本：
- 月初：500 元
- 本月收入：工资 5000 元 + 中彩票 2000 元
- 本月支出：房租 2000 元 + 吃饭 1000 元 + 购物 800 元
- 月末余额 = 500 + 7000 - 3800 = 3700 元

Roll-forward 就是把"月初 500 元变成月末 3700 元"的每一步都列出来——不是只告诉你"现在有 3700"，而是解释"钱从哪来、花到哪去"。

---

### 2. Roll-forward 的标准结构

```
期初余额（上期期末）
  + 增加项 / 新发生的业务
  + 本期预提（Accruals）
  - 上期预提冲回（Reversals）
  - 支付 / 结算（Payments）
  ± 重分类 / 调整（Reclasses）
  ± 汇兑转换（FX）
= 期末余额（本期期末）
```

---

### 3. Roll-forward 的"六大组件"详解

| 组件 | 英文 | 含义 | 例子 |
|------|------|------|------|
| 期初余额 | Opening Balance | 上期结束时这个账户的值 | 上季度末现金 $1000 万 |
| 增加项 | Additions | 本期新增的金额 | 新 LP 出资 $500 万 |
| 预提 | Accruals | 本期应计但未付的费用 | 管理费应计 $50 万 |
| 冲回 | Reversals | 上期预提的冲销 | 上期预提的审计费 $20 万冲回 |
| 支付 | Payments | 实际支付出去的钱 | 支付律师费 $10 万 |
| 调整 | Adjustments | 修正错误或重分类 | 科目分类错误修正 |

---

### 4. Roll-forward 的"对账锚点"

**核心规则：**
1. 期初余额 = 上期期末余额
2. 期末余额 = 本期 GL 余额
3. 期间变动 = 增加 - 减少 ± 调整

**如果这三个不匹配 → 有错误。**

---

### 5. 为什么要做 Roll-forward？

**三大价值：**
1. **审计支持**——审计师最看重 Roll-forward，因为它能证明"资产负债表上每一个数字都是怎么来的"
2. **内部控制**——防止有人"悄悄"修改期末余额
3. **透明度**——LP 可以清楚看到"钱去哪了"

---

### 6. 真实案例：某基金的 Roll-forward 审计发现

**背景**：审计师检查某基金的"管理费预提"Roll-forward

| 组件 | 金额 |
|------|------|
| 期初余额 | $100 万 |
| + 本期预提 | $200 万 |
| - 实际支付 | -$280 万 |
| = 期末余额 | $20 万 |

**发现矛盾**：本期预提只有 $200 万，但实际支付了 $280 万——说明上期期末余额 $100 万可能被低估（上期少预提了）。

**审计结论**：基金必须修正上季度的 NAV，并重新出具 LP 报告。

---

### 给小白的一句话

> Roll-forward 就是会计的"流水账日记"——不是只告诉你"月初有500元，月底剩300元"，而是把每一天、每一笔钱怎么进来、怎么花出去的全部记录下来。在基金会计中，Roll-forward 是审计师最重视的一页——因为它能证明"每一个数字都是有来龙去脉的"，如果有人偷偷篡改了一个数字，Roll-forward 立刻就会"对不上"。
