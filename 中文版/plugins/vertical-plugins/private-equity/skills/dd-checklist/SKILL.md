# 尽职调查清单 (Due Diligence Checklist)

description: 生成并跟踪全面的尽职调查清单，根据目标公司的行业、交易类型和复杂性进行定制。涵盖所有主要工作流，包含请求列表、状态跟踪和红旗升级机制。用于启动尽职调查、组织数据室审查或跟踪未完成项目。触发词："尽职调查清单"、"DD跟踪器"、"尽职调查请求列表"、"我们还需要什么"或"数据室审查"。

## 工作流程

### 步骤1：确定尽职调查范围

向用户询问：
- **目标公司**：名称、行业、商业模式
- **交易类型**：平台收购、附加收购、成长型股权、资本重组、分拆
- **交易规模/复杂性**：决定尽职调查深度
- **主要关注点**：任何已知的优先处理问题（客户集中度、监管、环境等）
- **时间表**：意向书/交割目标日期？

### 步骤2：生成工作流清单

针对所有主要工作流生成清单，根据行业定制：

**财务尽职调查**
- 收益质量(QoE) — 收入和EBITDA调整
- 营运资本分析 — 标准化vs实际
- 债务和类债务项目
- 资本支出（维护vs增长）
- 税务结构和风险敞口
- 审计历史和会计政策
- 备考调整（经常性、协同效应）

**商业尽职调查**
- 市场规模和增长(TAM/SAM/SOM)
- 竞争定位和市场份额
- 客户分析 — 集中度、留存率、NPS
- 定价能力和合同结构
- 销售渠道和未完成订单
- 市场拓展有效性

**法律尽职调查**
- 公司结构和组织架构图
- 重大合同（客户、供应商、合作伙伴）
- 诉讼历史和待决索赔
- IP组合和保护
- 监管合规
- 雇佣协议和竞业禁止条款

**运营尽职调查**
- 管理团队评估
- 组织结构和关键人员风险
- IT系统和基础设施
- 供应链和供应商依赖
- 设施和房地产
- 保险覆盖

**人力资源尽职调查**
- 组织架构图和人员趋势
- 薪酬基准
- 福利和养老金义务
- 关键员工留存风险
- 文化评估
- 工会/劳动协议

**IT/技术尽职调查**（适用于技术驱动型企业）
- 技术栈和架构
- 技术债务评估
- 网络安全态势
- 数据隐私合规(GDPR、CCPA、SOC2)
- 产品路线图和研发支出
- 可扩展性评估

**环境/ESG**（如适用）
- 环境负债
- 监管合规历史
- ESG风险和机遇

### 步骤3：状态跟踪

跟踪每个项目：

| 项目 | 工作流 | 优先级 | 状态 | 负责人 | 备注 |
|------|--------|--------|------|--------|------|
| QoE报告 | 财务 | P0 | 待处理 | | |
| 客户访谈 | 商业 | P0 | 进行中 | | 完成10个中的3个 |

状态选项：未开始 → 已请求 → 已收到 → 审查中 → 完成 → 红旗

### 步骤4：红旗摘要

维护尽职调查期间发现的红旗清单：
- 发现内容
- 所属工作流
- 严重性（交易终止/重大/可管理）
- 缓解措施或解决方案
- 对估值或交易条款的影响

### 步骤5：输出

- Excel工作簿，每个工作流一个标签（默认）
- 汇总仪表板：各工作流完成百分比、未完成项目、红旗
- 交易团队的每周状态更新格式

## 行业特定补充

根据行业自动添加相关项目：
- **软件/SaaS**：ARR质量、群组分析、托管成本、SOC2认证
- **医疗保健**：监管批准、报销风险、付款方组合
- **工业**：设备状况、环境修复、安全记录
- **金融服务**：监管资本、合规历史、信用质量
- **消费品**：品牌健康、渠道组合、季节性、库存管理

## 重要提示

- 优先处理P0项目，这些是意向书或交割的关键
- 标记卖方响应缓慢的项目——可能表明存在问题
- 将数据室内容与清单交叉引用以识别差距
- 随着尽职调查的进展更新清单——这是一份活文件

## 金融术语和知识解释

### 1. 尽职调查 (Due Diligence, DD)

**定义**：在投资或收购前对目标公司进行的全面调查和分析过程。

**目的**：
- 验证信息真实性
- 识别潜在风险
- 评估估值合理性
- 支持投资决策

**主要工作流**：

| 工作流 | 关注点 | 关键问题 |
|--------|--------|----------|
| 财务DD | 财务状况和业绩 | 收益质量、现金流、债务 |
| 商业DD | 市场和竞争地位 | 市场规模、竞争优势 |
| 法律DD | 法律合规风险 | 合同、诉讼、IP |
| 运营DD | 运营能力 | 管理团队、IT系统 |
| HR DD | 人力资源风险 | 人才、文化、薪酬 |

### 2. TAM/SAM/SOM

**定义**：市场规模的三层分析框架。

**TAM (Total Addressable Market)**：
- 定义：整个市场的潜在收入机会
- 示例：全球SaaS市场规模

**SAM (Serviceable Addressable Market)**：
- 定义：公司可以实际服务的市场部分
- 考虑因素：地理、行业、客户类型

**SOM (Serviceable Obtainable Market)**：
- 定义：公司可以实际获得的市场份额
- 考虑因素：竞争、资源、执行能力

**关系**：TAM > SAM > SOM

### 3. 收益质量 (Quality of Earnings, QoE)

**定义**：对公司报告收益真实性和可持续性的深入分析。

**分析要点**：
- 收入确认政策
- 非经常性项目调整
- 客户集中度风险
- 应收账款质量
- 现金流与净利润匹配度

**调整类型**：
| 调整项 | 说明 |
|--------|------|
| 一次性费用 | 重组、资产处置 |
| 非现金项目 | 折旧、摊销 |
| 异常收入 | 偶然销售、政府补贴 |
| 会计变更 | 政策调整影响 |

### 4. 营运资本 (Working Capital)

**定义**：公司日常运营所需的资金。

**计算公式**：
```
营运资本 = 流动资产 - 流动负债
```

**关键组成部分**：
- 应收账款
- 存货
- 应付账款
- 现金

**分析指标**：
```
营运资本周转天数 = 365 / 营运资本周转率
```

### 5. 资本支出 (Capital Expenditure, Capex)

**定义**：用于购买或升级长期资产的支出。

**分类**：
| 类型 | 目的 | 示例 |
|------|------|------|
| 维护性Capex | 维持现有运营 | 设备维修、替换 |
| 增长性Capex | 扩大产能/市场 | 新工厂、研发 |

**重要性**：
- 影响现金流
- 决定未来增长能力
- 需要与折旧匹配

### 6. EBITDA (Earnings Before Interest, Taxes, Depreciation, and Amortization)

**定义**：税息折旧摊销前利润，衡量核心运营盈利能力。

**计算公式**：
```
EBITDA = 净利润 + 利息 + 税费 + 折旧 + 摊销
```

**用途**：
- 比较不同资本结构的公司
- 评估运营效率
- 计算估值倍数

**局限性**：
- 未考虑资本支出
- 未考虑营运资本变化
- 可被会计政策影响

### 7. 红旗指标 (Red Flags)

**定义**：尽职调查中发现的潜在风险信号。

**常见红旗类型**：
| 类型 | 示例 |
|------|------|
| 财务 | 收入下降、现金流紧张 |
| 运营 | 关键人才流失、系统老化 |
| 法律 | 未决诉讼、合规问题 |
| 市场 | 客户集中度过高 |

**严重性分级**：
- **交易终止型**：无法解决的致命问题
- **重大型**：需要调整估值或条款
- **可管理型**：可通过缓解措施解决

### 8. 备考调整 (Pro Forma Adjustments)

**定义**：为反映预期协同效应或经常性运营而对财务报表进行的调整。

**常见调整**：
- 协同效应（成本节约、收入提升）
- 非经常性项目剔除
- 并购会计调整
- 营运资本调整

**目的**：提供更准确的合并后业绩预测。

### 9. 数据室 (Data Room)

**定义**：存放目标公司机密信息的安全在线平台。

**典型内容**：
- 财务报表和审计报告
- 重大合同
- 法律文件
- 员工信息
- 运营数据
- 市场研究

**管理要点**：
- 控制访问权限
- 跟踪文件查看记录
- 确保数据安全

### 10. 意向书 (Letter of Intent, LOI)

**定义**：表达投资或收购意向的非约束性文件。

**主要内容**：
- 交易结构和估值
- 尽职调查条款
- 排他性条款
- 关键条件
- 时间表

**重要性**：标志着正式尽职调查的开始。
