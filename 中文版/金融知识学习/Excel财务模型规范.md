# Excel 财务模型规范（Formatting & Conventions）

> 适用文件：xlsx-author/SKILL.md, audit-xls/SKILL.md, formatting.md, pitch-deck/SKILL.md, 任何提到"格式/颜色/检查/构建"的文件

---

## 1. 为什么财务模型要"格式化"

投行的财务模型动辄 200+ MB、100+ 个 Sheet、几十万公式。如果每个人按自己的习惯写——那别人根本看不懂。

**类比**：就像每个国家有交通规则（英国靠左、美国靠右）。没规则 = 撞车；有规则 = 顺畅。Excel 模型的格式化规范就是"投行的交通规则"。

更现实的原因：**一个模型从实习生做到 MD 审查要经过 4-5 个人的手**。没有统一规范 = 每一层审查都在浪费时间"找数字"。

## 2. 蓝黑字体公约（"死" vs "活"）

### 2.1 核心规则

| 颜色 | 含义 | 例子 | 记忆 |
|------|------|------|------|
| **蓝色** | 硬编码数字（Hard-coded） | 历史收入 $100M、税率 21% | **"死"的**：这些数字是人手工输入的，不随公式变化 |
| **黑色** | 公式（Formula） | =D5*E5, =SUM(F5:F20) | **"活"的**：这些值是公式算出来的，改一个前提全部联动 |
| **绿色** | 跨 Sheet 引用 | ='损益表'!B20 | **"外来"的**：引用了另一个 Sheet，不是在当前 Sheet 内 |
| **红色** | 跨文件引用 | ='[model.xlsx]Sheet1'!A1 | **"外面来的"**：引用了另一个 Excel 文件 |

### 2.2 为什么不全部用黑色？

**改动时一眼就知道哪些会变。**

举例：你要调 2026 年收入增长率从 5% 到 8%。蓝色单元格告诉你——"就改这个"。所有黑色单元格会自动重新计算。如果全是黑色，你根本分不清该改哪个。

### 2.3 Level 2：进阶参考

某些团队还在颜色上加"数据类型"分层：

| 标记 | 含义 |
|------|------|
| 蓝字黄底 | 用户现在要填的输入项 |
| 蓝字灰底 | 历史数据（不用改） |
| 蓝字白底 | 一般性假设 |
| 黑色加框（Trade-flag）| 特殊调整（如已签合同的影响） |

## 3. 单元格格式规范

### 3.1 数字格式（Number Format）

| 数值范围 | 格式 | 例子 | 原因 |
|----------|------|------|------|
| < $1M | `0.0` | 123.4 | 精确到 $100K |
| $1M - $1B | `#,###.0` | 1,234.5 | 精确到 $100K |
| > $1B | `#,###.0` | 1,234,567.8 | 精确到 $100K |
| 金额 | `$#,###.0` 或 `#,###.0` | $1,234.5 | 统一 |
| 百分比 | `0.0%` | 12.3% | 小数点后一位 |
| 倍数 | `0.0x` | 2.5x | 标准 |
| 股份数 | `#,###.0` | 100.0（百万股） | 单位一致 |

**黄金法则**：**一个 Sheet 内，同一列的数字格式必须统一**。不能第一行显示 $1,200.5，第五行显示 1.2B。

### 3.2 检查

- 勾稽检查（Balance Check）：`资产 = 负债 + 权益` 的差值应等于 0
- 合计公式（Sum of the Parts）：各部分之和应等于显示的总数
- 没有"DIV/0!" 或 "#REF!" 错误

### 3.3 打印（Print）

- 按页面设置打印范围（Print Range）
- 页眉有公司名、页脚有页码
- 比例适合一页显示

## 4. 时间序列规范

### 4.1 从左到右，最老到最新

```
| 2021 | 2022 | 2023 | 2024E | 2025E | 2026E | 2027E |
```

- "A"（Actual）标记在历史年上方
- "E"（Estimate）标记在预测年上方
- "LTM"（Last Twelve Months）表示最近 12 个月

### 4.2 季度 vs 年度

- **年度**：更常用于投行 PPT
- **季度**：更常用于内部预测和管理
- **混合**：先说年度数据，然后按季度拆分

## 5. Sheet 命名与组织

### 5.1 命名原则

| 坏名字 | 好名字 | 原因 |
|--------|--------|------|
| Sheet1 | IS_Model | 找得到 |
| 计算 | 估值_DCF | 一目了然 |
| 假设_v2 | 假设_基准 | 版本管理 |

### 5.2 典型 Sheet 顺序

```
1. 封面页     (Cover)         — 模型名称、日期、作者、版本
2. 摘要       (Summary)       — 一页结论（给 MD 看）
3. 假设       (Assumptions)   — 所有输入假设的集合
4. 损益表      (IS)           — Income Statement
5. 资产负债表   (BS)           — Balance Sheet
6. 现金流量表   (CF)           — Cash Flow Statement
7. 债务明细     (Debt)         — 各笔贷款的具体条款
8. DCF 估值    (DCF)          — 折现现金流
9. Comps       (Comps)        — 可比公司
10. 敏感性分析  (Sensitivities) — "如果...会怎样"
```

## 6. Excel 审计（Model Audit）

### 6.1 审计是什么

逐行检查模型有没有数学错误、逻辑错误、格式错误。

### 6.2 审计三步法

| 步骤 | 做什么 | 找什么 |
|------|--------|--------|
| **1. 结构检查** | 走一遍 Sheet 组织 | Sheet 引用是否断裂、命名是否合理 |
| **2. 公式检查** | 逐个公式审查 | 除以零、硬编码藏在公式里、循环引用 |
| **3. 逻辑检查** | 结果合理性 | 增长率 200%、负的折旧 |

### 6.3 光电（Sanity Check / Sense Check）

`sanity check` 直译是"神智检查"——数据太离谱就是"疯"了。

| 检查 | 举例 |
|------|------|
| 收入增速 | 一家成熟公司突然增长 50%——可能错了 |
| 利润率 | 毛利率从 30% 跳到 80%——可能错了 |
| 资本支出 | CapEx 突然是收入的两倍——可能错了 |
| 现金 | 现金为负——可能错了（借不到钱） |

## 7. 真实案例：格式错误导致的问题

### TransAlta（加拿大电力公司，2003）

一个分析师在 Excel 中拷贝粘贴时，不小心把不同 Sheet 的数据混在一起。TransAlta 因此报出亏损 $2400 万，股价暴跌。事后证明：是一个格式化错误——不是公司真的亏了这么多，而是数字贴错了位置。

### Fannie Mae（美国房贷巨头，2003）

一个 $13 亿的会计差错，因为 Excel 中"勾稽检查"公式写错。一个本应等于 0 的差值显示了 $13 亿——但没有人检查这个 check。

## 给小白的一句话

> Excel 模型格式就像一本工具书的目录、页码、字体——你不需要发明自己的排版系统。蓝色 = 原始材料（不可动），黑色 = 机器的加工结果（自动算），绿色 = 从别处借来的（引用）。这就让你在 200 个 Sheet 的巨型模型中，三秒内找到"这个数字是我要改的，还是系统在帮我们算的"。不按这个规矩来——你的模型就是"天书"，别人看不懂，老板用不了。
