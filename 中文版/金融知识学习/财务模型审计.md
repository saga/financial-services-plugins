> 适用文件：audit-xls/SKILL.md（所有 Agent 的）, check-model/SKILL.md, debug-model.md, TROUBLESHOOTING.md, 任何提到"审计/检查/校验/模型质量"的文件

---

## 1. 什么是财务模型审计

**财务模型审计** 是逐行检查 Excel 模型有没有数学错误、逻辑错误、格式错误、引用断裂的过程。

**类比**：盖好了一栋 50 层的大楼（100 个 Sheet、几万公式的模型），不能直接让人住——必须派建筑验收员逐层检查："这个承重墙够不够？电路有没有接错？水压行不行？"财务模型审计就是"模型验收员"。

### 1.1 不做审计的代价

- 一个公式引用了错误 Sheet 的单元格 → **整个估值错误**
- 一次循环引用 → **模型崩溃**（Excel 转圈圈）
- 隐藏在公式中的硬编码数字 → **改一个参数但另一个没动** → 结论前后矛盾

**真实案例**：摩根大通在 2012 年"伦敦鲸"事件中损失 $62 亿——事后发现内部的 VaR 模型在 Excel 复制粘贴时点错了列，导致风险报告低估了实际风险。

## 2. 模型审计的五大检查

### 2.1 结构检查（Structural Audit）

| 检查什么 | 怎么查 | 为什么重要 |
|----------|--------|----------|
| Sheet 命名是否规范 | 扫描命名 | 不好找 = 效率低 50% |
| 数据流向是否合理 | 追踪 Input → Calc → Output | 不该有"回头引用" |
| 是否有循环引用 | Excel 公式审核工具 | 循环引用让模型不可靠 |
| 是否有外部文件链接 | Excel 编辑链接 | 换电脑后这个链接断了 |

### 2.2 公式检查（Formula Audit）

| 检查什么 | 标志 |
|----------|------|
| **除以零** | `#DIV/0!` 错误 |
| **引用错误** | `#REF!` 错误 |
| **名称错误** | `#NAME?` 错误 |
| **隐藏的硬编码** | 在公式里面夹数字（如 `=B5*0.21`，应该是 `=B5*税率_单元格`） |
| **不一致的公式** | 同一行的公式突然不一样 |

**典型错误**：
```
=B5*0.21*E5            ← 硬编码 0.21（21% 税率藏在公式里）
应该写成：
=B5*$H$1*E5            ← 税率在 H1，别人能改
```

### 2.3 逻辑检查（Logic Audit / Sanity Check）

| 逻辑检查 | 怎么查 |
|----------|--------|
| 收入增长率 50% | 行业平均增长率是多少？ |
| 毛利率从 30% 到 70% | 什么导致这个飞跃？合理吗？ |
| CapEx 是收入的 200% | 加班加疯了？ |
| 现金为负 | 公司要破产了——说明模型里缺一个"债务融资"的开关 |
| 市场份额超过 100% | 不可能，有错 |

### 2.4 钩稽检查（Cross-reference / Articulation Check）

| 检查 | 怎么算 |
|------|--------|
| 资产 = 负债 + 权益 | BS 检查（Balance Sheet Check）：每一年的 `总资产 = 总负债 + 总权益` |
| 现金变动 = CF 结果 | BS 上的现金变动 应该 = 现金流量表的最终结果 |
| 净利润流入留存收益 | IS 的净利润 → 应该影响 BS 的留存收益 |

### 2.5 格式检查（Formatting Audit）

参见 [Excel财务模型规范.md](Excel财务模型规范.md)。

## 3. 从零模型 vs 翻新模型的审计区别

| | 从零模型（New Build） | 翻新模型（Renovation） |
|---|---------------------|---------------------|
| **类比** | 建新房子 = 全流程验收 | 装修老房子 = 查水电路承重 |
| **重点** | 结构完整性、逻辑连贯 | 遗留下来的"死角" |
| **常见问题** | 公式很干净 | 同一个 Sheet 里有 5 年前的公式还留着 |
| **风险** | 全局系统性错误 | 局部灾难 |

**翻新模型常见陷阱**：
- 某人把 2023 年的"临时数据"硬编码在公式里，2026 年还活着
- 一个"隐藏行"里有 500 条已废弃的计算
- 某人引入时改了某列公式，但没改旁边列的

## 4. 模型审计工具

### 4.1 Excel 内置

| 工具 | 作用 |
|------|------|
| **公式审核 → 追踪引用** | 看这个单元格引用了谁 |
| **公式审核 → 追踪依赖** | 看谁用了这个单元格 |
| **公式审核 → 显示公式** | 一键看到所有公式而不是结果 |
| **错误检查** | 找到 #DIV/0!, #REF! 等 |
| **循环引用** | 在状态栏点"循环引用"跳转 |
| **名称管理器** | 管理所有命名的单元格 |

### 4.2 第三方工具

| 工具 | 适合 |
|------|------|
| **Operis** | 英国最大的模型审计公司，有自己的软件 |
| **BPM** (Best Practice Modelling) | Excel 最佳实践培训 |
| **Modano** | 模块化的财务模型开发平台 |

## 5. 典型审计流程

```
第 1 步：模型结构检查
          ↓ 没问题
第 2 步：公式审计（逐个 Sheet）
          ↓ 没问题  
第 3 步：钩稽检查（三表是否连接？）
          ↓ 没问题
第 4 步：敏感性分析（改假设是否合理？）
          ↓ 没问题
第 5 步：SANITY CHECK（结果合理吗？）
          ↓ 没问题
第 6 步：出具审计报告
```

---

## 6. 审计报告的格式

| 章节 | 内容 |
|------|------|
| **参与者与日期** | 什么版本、谁审计、日期 |
| **发现汇总** | 高/中/低风险分类 |
| **详细发现** | 每项问题：位置、描述、建议修复 |
| **通过检查** | 列出所有通过的项目 |
| **未解决问题** | 需要模型所有者回答的问题 |
| **总体评估** | 通过 / 待修复后通过 / 不通过 |

---

## 给小白的一句话

> 财务模型审计就是"收货时的验货单"——建了一个巨复杂的 Excel 模型，不能直接交差，必须先过一遍检查：公式有没有除以零？三表勾稽对得上吗？改一个数字后结果合理吗？一个好模型能经得起"任何人都可以随意改假设"的灵活性；一个没审计的模型 = "炸弹"，你不知道哪一个公式里藏着 Bug，下一个人改了会爆出一堆错误。
