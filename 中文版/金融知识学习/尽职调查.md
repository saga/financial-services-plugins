# 尽职调查（Due Diligence, DD）

> 适用文件：dd-checklist/SKILL.md, dd-prep/SKILL.md, dd-meeting-prep/SKILL.md, kyc-doc-parse/SKILL.md, process-letter/SKILL.md, deal-screening/SKILL.md, 任何提到"尽调/DD/核查"的文件

---

## 1. 什么是尽职调查

**尽职调查（Due Diligence）** 是一套系统性的调查流程——在决定投钱之前，把目标的"里里外外"查个底朝天。

**类比**：买房之前你会请验房师（查房子有没有漏水、地基有没有裂缝），买车之前你会查 Carfax（有没有事故记录）。尽职调查就是企业的"全身体检"——只是比验房复杂一千倍。

---

## 2. 为什么需要 DD

### 2.1 信息不对称

卖方知道的永远比买方多。你不查，那卖方的"小毛病"就永远不会主动告诉你。

### 2.2 真实案例

- **惠普收购 Autonomy（2011）**：$110 亿收购英国软件公司，一年后减记 $88 亿。惠普指责 Autonomy 造假夸大收入——事后看，财务 DD 没做到位
- **三菱日联投资 Grab（2020）**：因新冠疫情，尽调只能远程完成。好在这笔投资本身是 OK 的
- **Theranos 丑闻**：如果投资人做了真正的技术 DD（拿机器去独立实验室验证），就不会被骗 $7 亿

### 2.3 "不做 DD 的代价"

| 事件 | 损失 | 原因 |
|------|------|------|
| 惠普-Autonomy | $88 亿 | 财务造假未被查出 |
| 拜耳-孟山都 | $100 亿+ 诉讼 | 法律 DD 低估草甘膦诉讼风险 |
| Wirecard | 全部化为乌有 | 审计 DD 没能发现 $19 亿欧元假存款 |

## 3. DD 的三大类型

### 3.1 商业尽调（Commercial / Business DD）

**核心问题**：这公司到底赚不赚钱？为什么？

| 查什么 | 具体内容 |
|--------|---------|
| 市场位置 | 行业大小、增长率、市场份额 |
| 竞争格局 | 主要对手、护城河、替代品 |
| 商业模式 | 怎么赚钱？（订阅？交易？广告？） |
| 客户分析 | 客户集中度（一个客户占 30% 收入 = 红色警报） |
| 增长动力 | 未来的增长从哪来？ |
| 定价能力 | 涨价了客户会不会跑？ |

**典型产出**：市场需求分析报告、竞对对比矩阵、客户留存率分析

### 3.2 财务尽调（Financial DD）

**核心问题**：账上的数字是真的吗？

| 查什么 | 具体内容 |
|--------|---------|
| 收入质量 | 是否提前确认？是否有"渠道填塞"？ |
| 利润质量 | EBITDA 是否加回了"正常化调整"？ |
| 现金流 | FCF 和净利润是否一致？ |
| 债务 | 有没有隐藏的担保、表外负债？ |
| 营运资本 | 季节性波动、正常营运资本水平 |
| 税务 | 有没有历史欠税、转让定价风险？ |
| 关联交易 | 和老板其他公司有什么往来？ |

**典型产出**：质量 of 收益报告（QoE）、净债务调整表、正常化 EBITDA 对照

**QoE（Quality of Earnings）** 是财务 DD 的灵魂——区分"真实的利润"和"会计做出来的利润"。**这是投资决策最重要的依据之一。**

### 3.3 法律尽调（Legal DD）

**核心问题**：有没有潜在的诉讼风险、合规风险？

| 查什么 | 具体内容 |
|--------|---------|
| 公司架构 | 股权结构是否清晰？有没有代持？ |
| 合同 | 关键客户/供应商合同的保护条款 |
| 知识产权 | 专利、商标、著作权是否归属于公司？ |
| 诉讼 | 正在进行的诉讼、潜在索赔 |
| 劳动问题 | 有没有劳动纠纷、遣散费义务 |
| 合规 | 反贿赂、出口管制、数据隐私 |

**典型产出**：法律 DD 报告、诉讼风险矩阵、合同审查摘要

## 4. DD 的其他专业类型

### 4.1 IT/技术 DD
- 代码质量、技术架构
- 技术债务评估
- 安全漏洞扫描
- 数据隐私合规（GDPR）

### 4.2 运营 DD
- 供应链韧性
- 产能利用率
- 制造成本对标

### 4.3 环境与社会 DD（ESG DD）
- 碳排放
- 环保违规
- 劳工标准

### 4.4 管理团队 DD
- 创始人/CEO 背景调查
- 管理层履历核查
- 关键人员留任计划

## 5. DD 流程

### 5.1 阶段一：初步 DD（2-4 周）

**在签 NDA 之后、出价之前**

- 分析公开信息（年报、新闻、研报）
- 管理层面谈（做 PPT 的演示）
- 行业专家访谈
- **产出**：初步判断——值不值得继续往下走

### 5.2 阶段二：详细 DD（6-12 周）

**在签 LOI/条款清单之后、签最终协议之前**

- 进入数据室（Data Room）翻阅所有材料
- 第三方顾问进场（四大会计师事务所、律所）
- 多位职能专家（税务、环保、IT）同时推进
- **产出**：确定性问题清单、价格调整依据

### 5.3 阶段三：签约后/交割前 DD

**交易宣布后到交割前（1-3 个月）**

- 确认没有"重大不利变化"（Material Adverse Change, MAC）
- 最终营运资本确认
- **产出**：交割条件清单

## 6. DD 的组织方式

### 6.1 数据室（Data Room）

一个安全的线上文件夹，卖方把所有文件放进去，买方团队在里面查阅。

**典型数据室结构**：
```
/01_公司架构       — 股权结构、子公司、公司章程
/02_财务          — 审计报告、管理账目、税务申报
/03_客户与收入     — 客户合同、收入明细
/04_员工         — 劳动合同、股权激励计划
/05_法律          — 诉讼、合同、知识产权
/06_监管          — 许可证、合规文件
/07_IT           — 系统架构、软件许可
/08_保险         — 保单
```

### 6.2 专家网络（Expert Network）

GLG、AlphaSights、Third Bridge 等公司连接买方和行业专家。买方可以匿名面试前员工、竞争对手、客户——获取非公开的行业见解。

### 6.3 DD 请求清单（DD Request List）

买方发给卖方的"你要给我看这些东西"清单。动辄 100-300 项。

## 7. DD 发现的处理

### 7.1 发现分级

| 级别 | 含义 | 行动 |
|------|------|------|
| **致命问题（Deal Breaker）** | 欺诈、重大诉讼、商业模式有结构性缺陷 | 放弃交易 |
| **重大（Material）** | 影响估值或条款 | 压价 5-20% 或加保护条款 |
| **中等（Moderate）** | 可控 | 交割后整改、保留部分资金 |
| **轻微（Minor）** | 无实质影响 | 记录但不采取行动 |

### 7.2 保护条款

通过 DD 发现问题后，可以在协议中加入：

| 条款 | 作用 |
|------|------|
| **陈述与保证（Reps & Warranties）** | "卖方保证：X 是真实的，如果出事你来赔" |
| **赔偿条款（Indemnification）** | 如果 DD 后还出了某类问题，卖方赔多少 |
| **保证金(Holdback/Escrow)** | 扣下 10-20% 收购款，放着 12-18 个月，出事了退不回来 |
| **声明与保证保险（R&W Insurance）** | 给陈述保证买保险，出问题保险公司赔 |

## 8. 买方 DD vs 卖方 DD

| | 买方 DD | 卖方 DD |
|---|---------|---------|
| **目的** | 了解目标、保护自己 | 包装公司、吸引买家 |
| **谁做** | 买方自己 | 卖方请的投行/顾问 |
| **报告给** | 投委会 | 多个潜在买家 |
| **深度** | 极深 | 有选择性（不好的藏起来） |
| **类比** | 验房师 | 售楼广告 |

**卖方 DD 辩证看**：一个公司请四大做了卖方 DD 不等于"干净"——卖方 DD 是卖方付钱做的，报告偏向美化目标公司。

---

## 给小白的一句话

> 尽职调查就像给对象做"婚前体检"——恋爱时（签 NDA 前）只看到表面，求婚前（签 LOI 后）做全面体检（财务+法律+商业），婚礼前（交割前）再确认没变卦。跳过这步结婚的（不做 DD 就买），轻则婚后发现对方欠债（隐藏负债），重则直接倾家荡产（惠普-Autonomy 减记 $88 亿）。
