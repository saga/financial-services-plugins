# 电子表格审计

审计公式和数据的准确性及错误。审计深度由范围决定——从针对选定区域的快速公式检查，到针对完整财务模型的合规性审计。

## 第 1 步：确定范围

如果用户已指定范围，直接执行。否则，**请询问用户**：

> 您希望审计哪个范围？
> - **选定区域 (selection)** —— 仅针对当前选中的单元格范围
> - **工作表 (sheet)** —— 仅针对当前活动工作表
> - **模型 (model)** —— 针对整个工作簿，包括财务模型完整性检查（资产负债表配平、现金勾稽、滚动平衡、逻辑合理性）

**模型**范围审计最为深入——在将 DCF、LBO、三表联动、并购、可比公司分析或任何综合财务模型发送给客户或投资委员会 (IC) 之前，请务必使用此范围。

---

## 第 2 步：公式级检查（所有范围）

无论范围如何，均需运行以下检查：

| 检查项目 | 关注点 |
|---|---|
| 公式错误 | `#REF!`, `#VALUE!`, `#N/A`, `#DIV/0!`, `#NAME?` |
| 公式中的硬编码 | 如 `=A1*1.05` —— 这里的 `1.05` 应该定义在单元格引用中 |
| 不一致公式 | 在某行或某列中，某个公式破坏了相邻单元格的模式 |
| 范围选取偏差 | `SUM`/`AVERAGE` 漏掉了首行或末行 |
| 被覆盖的公式 | 看起来像公式但实际被粘贴为硬编码数值的单元格 |
| 循环引用 | 无论是有意为之还是无意产生的 |
| 断裂的跨表链接 | 引用了已移动或已删除的工作表单元格 |
| 单位/量级不匹配 | 千位与百万位混用，百分比被存为整数 |
| 隐藏行/标签页 | 可能包含覆盖值或陈旧的计算规则 |

---

## 第 3 步：模型完整性检查（仅限“模型”范围）

如果范围是 **model**，请识别模型类型（DCF / LBO / 三表联动 / 并购 / 可比分析 / 自定义），并运行以下相应的完整性检查。

### 3a. 结构审查

| 检查项目 | 关注点 |
|---|---|
| 输入项与公式分离 | 输入项是否与计算过程清晰分开？ |
| 颜色规范 | 是否遵循“蓝色=输入、黑色=公式、绿色=链接”或模型自定的统一规范？ |
| 标签页流向 | 逻辑顺序是否合理（假设 → 利润表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期表头 | 所有标签页的日期表头是否一致？ |
| 单位 | 全文单位是否统一（千位 vs 百万位 vs 绝对值）？ |

### 3b. 资产负债表 (BS)

| 检查项目 | 测试方法 |
|---|---|
| BS 配平 | 每一期：总资产 = 总负债 + 所有者权益 |
| 留存收益 (RE) 滚动 | 期初 RE + 净利润 - 股利 = 期末 RE |
| 商誉/无形资产 | 是否来源于收购假设（针对并购模型） |

如果资产负债表无法配平，**请量化每一期的差额并追踪断裂处**——在修复此问题之前，其他任何审计都无意义。

### 3c. 现金流量表 (CF)

| 检查项目 | 测试方法 |
|---|---|
| 现金勾稽 | 每一期：现金流量表期末现金 = 资产负债表现金项 |
| 现金流加总 | CFO + CFI + CFF = 现金净变动额 |
| 折旧摊销 (D&A) 匹配 | 现金流量表中的 D&A = 利润表中的 D&A |
| 资本开支 (CapEx) 匹配 | 现金流量表中的 CapEx 与资产负债表中的固定资产 (PP&E) 滚动表匹配 |
| 营运资本 (WC) 变动 | 符号是否与资产负债表的科目变动一致（如应收增加则现金流减少） |

### 3d. 利润表 (IS)

| 检查项目 | 测试方法 |
|---|---|
| 收入构建 | 是否与分业务板块/产品的细节勾稽 |
| 税项 | 所得税费用 = 税前利润 × 税率（需考虑递延所得税调整） |
| 股份数 | 是否与稀释表（期权、可转债、回购）一致 |

### 3e. 循环引用

- “利息 → 债务余额 → 现金 → 利息”是 LBO/三表模型中常见的有意循环
- 若为有意：验证是否存在并激活了“迭代计算切换” (Iteration Toggle)
- 若为无意：追踪循环路径并标记如何打破闭环

### 3f. 逻辑与合理性

| 检查项目 | 预警情况 |
|---|---|
| 增长率 | 收入增长超过 100% 却无合理解释 |
| 利润率 | 明显超出行业通行准则 |
| 终值占比过高 | 终值 (TV) 超过 DCF 企业价值 (EV) 的 ~75%（黄色预警） |
| 曲棍球棒效应 | 预测期末年的数据呈现不切实际的爆发式增长 |
| 复合增长 | EBITDA 在第 10 年复合增长到了荒谬的数值 |
| 边界用例 | 模型在增长率为 0% 或负数、EBITDA 为负、杠杆率为负时发生报错 |

### 3g. 特定模型类型的常见漏洞

**DCF (现金流折现模型):**
- 折现率应用时期错误（期中 vs 期末）
- 终值未折现回现值
- WACC 使用账面价值而非市场价值
- 自由现金流 (FCF) 错误包含了利息支出（应为无杠杆现金流）
- 税盾 (Tax Shield) 被双重计入

**LBO (杠杆收购模型):**
- 债务偿还与现金强制清扫 (Cash Sweep) 机制不匹配
- 实物支付利息 (PIK Interest) 未累积至本金
- 管理层股权平移 (Rollover) 未在回报率中体现
- 退出倍数应用的 EBITDA 统计口径错误（LTM vs NTM）
- 交易费用/开支未从初始股本投入中扣除

**并购模型 (Merger):**
- 增厚/稀释分析使用了错误的股份数（交易前 vs 交易后）
- 协同效应未体现分阶段实现
- 购买价格分配 (PPA) 无法配平
- 未计入因支付现金而损失的利息收入 (Foregone Interest)
- 资金来源与用途表 (Sources & Uses) 中漏掉交易费用

**三表联动模型:**
- 营运资本变动符号正负反转
- 折旧额与固定资产 (PP&E) 排期表不符
- 债务到期计划与本金偿付不匹配
- 股利支付在没有解释的情况下超过了净利润

## 第 4 步：报告

输出审计发现表：

| # | 工作表 | 单元格/范围 | 严重程度 | 分类 | 问题描述 | 建议修复 |
|---|---|---|---|---|---|---|

**严重程度定级：**
- **严峻 (Critical)** —— 结果错误（资产负债表不平、公式断裂、现金无法勾稽）
- **警告 (Warning)** —— 潜在风险（硬编码、公式不一致、极端数值下报错）
- **提示 (Info)** —— 格式/最佳实践（颜色规范、布局、命名习惯）

对于 **model** 范围，需在表前增加一行总结：

> 模型类型：[DCF/LBO/三表/...] — 总体评价：[清洁 / 存在次要问题 / 存在重大问题] — [N]项严峻问题, [N]项警告, [N]项提示

**未经询问请勿私自修改** —— 先报告，根据要求再进行修复。

## 备忘录

- **资产负债表配平优先** —— 如果表不平，后续所有的计算都是可疑的。
- **硬编码的覆盖值是“沉默杀手”** —— 这是隐藏 Bug 的头号来源，需强势搜索。
- **正负号习惯错误**（尤其是对于现金流出的处理）极其常见。
- 如果模型使用了 VBA 宏，请注明哪些由宏驱动的计算无法通过常规公式审计完成。

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到华尔街级别的 Excel 诊疗室！如果你是第一次接触这些“金融黑话”，这里有几个锦囊帮你快速入戏：
>
> 1. **资产负债表 (BS) 为什么要“平”？**
>    资产负债表就像是一张财富快照。左边是你拥有的东西（资产），右边是这些东西是怎么来的（欠别人的债，或者自己的钱）。由于每一分钱都有去向，如果“总资产”不等于“负债+权益”，那一定是你在做账时把钱弄丢了或者从天上掉钱了。配平（Balance）是所有财务模型的及格线！
>
> 2. **现金勾稽 (Cash Tie-out) 是什么鬼？**
>    想象你在记账，你手里的现金余额必须得跟你的收支清单（现金流量表）最后算出来的数对得上。在复杂的模型里，现金就像血液一样在三张表之间流动，如果“现金流量表”的期末数跟“资产负债表”上的现金项对不上，那就说明模型的血液循环系统“漏血”了。
>
> 3. **硬编码 (Hardcodes) 为什么是禁忌？**
>    如果你的公式是 `=A1 * 0.25`，这个 `0.25` 就是硬编码。万一以后税率变了，你得翻遍几千个单元格去改。真正的模型高手会把它写在一个专门的“假设项”单元格里，让公式变成 `=A1 * $B$2`。这样改一下，全表动，这才是“会呼吸”的模型。
>
> 4. **DCF 和 LBO 是什么？**
>    - **DCF (现金流折现)**：一种“算命”法。把公司未来几十年赚的钱，按照现在的购买力“打个折”加起来，算出这公司现在值多少钱。
>    - **LBO (杠杆收购)**：一种“借钱买房”式投资。大头钱是借的，自己出小头，主要看能不能靠公司赚的钱把债还了，最后还能把公司卖个好价钱。
>
> 5. **曲棍球棒与杠杆平衡**：
>    如果你的模型预测收入在第五年突然起飞（像曲棍球棒一样折上去），或者算出了负的税收，审计器就会提醒你：“嘿，现实中可没这么多美差，是不是公式写错了？”
