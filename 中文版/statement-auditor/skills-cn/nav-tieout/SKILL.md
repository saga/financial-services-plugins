# NAV 核对 (NAV Tie-out)

基于已生成的 LP 对账单和当期的 NAV 数据包（通过 nav MCP 提供），独立重新计算 LP 的资本账户，并进行逐行对比。

> **已生成的对账单是待核对对象。** NAV 数据包是唯一事实来源（Source of Truth）。

## 重新计算 LP 资本账户

```
期初资本（上期对账单期末额）
  + 出资（本期已支付的实缴资本/出资通知）
  − 分配（现金 + 实物）
  + 分配的净收益 / (亏损)
      = LP 持股比例 % × (已实现 + 未实现损益 − 管理费 − 基金费用)
  − 业绩报酬分配 (Carried interest，若在本期结转)
期末资本
```

从 NAV 数据包中提取各项输入：LP 认缴比例 %、基金层面的损益（P&L）组成部分、费用和总支出、分配瀑布流（Waterfall）输出结果。

## 核对

对于对账单上的每一行，将其与你重新计算的数值进行对比。容差：`0.01`。对于每一个不匹配项，注明差异来源（例如，“分配损益不一致——对账单使用的是 12.40% 的所有权比例，而 NAV 数据包显示在第一季度转让后所有权比例为 12.38%”）。

## 额外检查项

- 本期对账单的期末资本 = 下期草案的期初资本（若可用）。
- 所有 LP 期末资本的总和 = 基金资产净值（在舍入误差范围内）。
- 认缴额（Commitment）、待缴额（Unfunded）和可召回额（Recallable）数据与认缴登记册保持一致。

## 输出

每行核对结果（通过/失败）、重新计算的数值与对账单数值对比，以及偏差标记列表。不要直接修改对账单——发布者将在审查标记后采取行动。

> **💡 Appendix: 领域知识小贴士**
>
> 1. **什么是 NAV (资产净值)?**
>    想象基金是一个大篮子，里面装满了股票或房产。NAV 就是把所有东西按今天市价卖掉，减去欠的债，剩下的钱。它是衡量基金“值多少钱”的标尺。
>
> 2. **什么是资本账户 (Capital Account)?**
>    如果把基金看作一家公司，LP（投资者）就是股东。资本账户就像是你在基金里的“个人存折”，记录了你投了多少钱、分到多少利润，以及你现在占有的那块“基金蛋糕”有多大。
>
> 3. **什么是核对 (Tie-out)?**
>    在金融界，核对就像是重新做一遍算术题。为了确保发给投资者的账单没出错，我们会用另一套独立的数据源（NAV 数据包）来反向计算。如果两个结果能“对上”，说明账目是准的。
>
> 4. **什么是 Carried Interest (业绩报酬)?**
>    这是给基金经理人的奖金。当基金赚了钱，经理人会按约定的比例（通常是 20%）提成。这部分钱会从投资者的收益中扣除。
>
> 5. **认缴 (Commitment) vs 待缴 (Unfunded)?**
>    在私募股权基金中，你答应投 100 万（认缴），但不用一次性给齐。这次给了 20 万，你还欠基金 80 万（待缴），等经理人发现好项目时再管你要，这叫“向你 call 钱”。
