# 代码同步与架构更新分析报告 (2026-05-06)

自上次中文版更新以来，仓库经历了一次重大的架构重构（Commit #81）。以下是核心变动的详细分析：

## 1. 目录结构大重组 (Restructuring)

原来的顶级业务目录（如 `investment-banking`, `equity-research` 等）已被移动到统一的 `plugins/` 架构下：

*   **`plugins/vertical-plugins/`**: 垂直行业的核心。这里是技能（Skills）、命令（Commands）和 MCP 服务器的原始出处。
*   **`plugins/agent-plugins/`**: 命名代理（Named Agents）的封装。每个目录（如 `equity-analyst`）是一个独立的插件包，包含了从垂直行业同步过来的技能拷贝。
*   **`managed-agent-cookbooks/`**: 新增目录。为 Claude Managed Agents 提供配置模板（`agent.yaml`），实现“一套代码，两种部署方式”。
*   **`claude-for-msft-365-install/`**: 新增目录。专门用于 Microsoft 365 插件的管理员安装工具。

## 2. 核心开发工作流的变化

现在的开发流程要求更高的规范性：
*   **禁止直接在 `agent-plugins` 中修改技能**：所有的核心逻辑必须在 `vertical-plugins/` 下修改。
*   **强制同步机制**：修改完垂直行业的技能后，必须运行 `python3 scripts/sync-agent-skills.py`。该脚本会将修改分发到所有相关的命名代理包中。
*   **提交前检查**：在 Commit 前必须运行 `python3 scripts/check.py`，用于校验 manifest 文件的完整性以及主从技能的一致性。

## 3. 功能与配置更新

*   **身份验证增强**：`claude-in-office` 插件现在支持 `entra_scope` 和 `graph_client_id`，允许更好的集成自定义 Microsoft Entra 应用。
*   **推理配置**：Manifest 增加了 `inference_headers` 配置项，允许在调用模型时注入自定义的 HTTP Header。
*   **可观测性**：新增了 `otlp_resource_attributes` 支持，增强了插件运行时的远程遥测能力。

## 4. 后续操作建议

1.  **更新本地映射**：如果你之前有针对特定目录的自动化脚本，请根据新的 `plugins/` 路径进行调整。
2.  **初始化环境**：建议运行 `pip install -r scripts/requirements.txt`（如果存在）以确保新的检查脚本可以正常运行。
3.  **中文版迁移**：目前的“中文版”目录结构仍保留了旧版的层级，建议在稳定后逐步将中文内容迁移至 `plugins/vertical-plugins/` 下对应的中文路径，或保持现有的镜像结构但同步其核心逻辑变化。
