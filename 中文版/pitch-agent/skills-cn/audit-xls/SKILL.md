# 电子表格勾稽与审计

审计公式与数据，确保其准确性并排查错误。审计深度由范围决定——从针对选定区域的快速公式检查，到完整的财务模型完整性审计。

## 第一步：确定范围

如果用户已指定范围，请直接使用。否则，**请询问用户**：

> 您希望审计的范围是？
> - **选定区域 (selection)** —— 仅针对当前选中的单元格区域
> - **工作表 (sheet)** —— 仅针对当前活动工作表
> - **模型 (model)** —— 针对整个工作簿，包括财务模型完整性检查（资产负债表平衡、现金流勾稽、滚动计算及逻辑合理性）

**模型 (model)** 范围审计最为深入——在向客户或投资委员会 (IC) 提交 DCF、LBO、三张表集成模型、并购模型、可比公司分析或任何综合财务模型之前，建议使用此范围。

---

## 第二步：公式级检查（适用于所有范围）

无论范围如何，均须执行以下检查：

| 检查项 | 关注内容 |
|---|---|
| 公式错误 | `#REF!`, `#VALUE!`, `#N/A`, `#DIV/0!`, `#NAME?` |
| 公式内嵌硬编码 | 例如 `=A1*1.05` —— 这里的 `1.05` 应当采用单元格引用 |
| 公式不一致 | 行或列中的某个公式破坏了相邻单元格的计算模式 |
| 区域引用错位 | `SUM` 或 `AVERAGE` 函数漏掉了首行或末行 |
| 数值覆盖公式 | 单元格看起来像公式，但实际被手动输入了硬编码数值 |
| 循环引用 | 确认是有意为之还是误操作 |
| 跨表链接失效 | 引用了已被移动或删除的单元格 |
| 单位/量级失调 | 千位与百万位混用，或百分比被存为整数 |
| 隐藏行/工作表 | 可能包含覆盖设置或过时的计算逻辑 |

---

## 第三步：模型完整性检查（仅限模型范围）

如果范围是 **模型 (model)**，请识别模型类型（DCF / LBO / 三张表 / 并购 / 可比公司 / 自定义），并执行以下相应的勾稽检查。

### 3a. 结构审查

| 检查项 | 关注内容 |
|---|---|
| 输入/公式分离 | 假设输入（Inputs）是否与计算逻辑明确分开？ |
| 颜色惯例 | 是否遵循“蓝色=输入，黑色=公式，绿色=链接”或模型既定规范，并保持一致？ |
| 模块流向 | 逻辑顺序是否正确（假设 → 利润表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期表头 | 在所有标签页中是否保持一致？ |
| 单位 | 全表单位是否统一（千位 vs 百万位 vs 绝对值）？ |

### 3b. 资产负债表 (BS)

| 检查项 | 测试逻辑 |
|---|---|
| 配平检查 | 资产总计 = 负债合计 + 权益合计（每一期均需成立） |
| 留存收益滚动 | 期初留存收益 + 净利润 - 股利 = 期末留存收益 |
| 商誉/无形资产 | 是否来源于并购假设（如果是 M&A 模型） |

如果资产负债表不平，请**量化每一期的差异并追踪断点所在** —— 在修复此问题之前，其他任何检查都暂无意义。

### 3c. 现金流量表 (CF)

| 检查项 | 测试逻辑 |
|---|---|
| 现金对帐 | 现金流量表期末现金 = 资产负债表货币资金（每一期均需成立） |
| 流量汇总 | 经营活动 + 投资活动 + 筹
