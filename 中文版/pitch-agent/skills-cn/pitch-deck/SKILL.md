## ⚠️ 关键渲染限制

**虽然使用 LibreOffice 进行验证，但它无法准确渲染 PowerPoint 文件。** 它可能会导致字体乱码、渐变失效、形状错位、文本换行错误以及部分表格格式异常。

**这意味着：** 在 LibreOffice 中通过视觉验证的幻灯片，在 Microsoft PowerPoint 中可能仍存在问题。验证环节可以捕获结构性问题（内容缺失、表格破损、遗留占位符格式），但**无法**捕获字体替换、细微的对齐偏移或渐变问题。

**必要动作：** 在交付输出时必须包含以下声明：
> “此文件已使用 LibreOffice 完成验证。由于渲染差异可能存在，请在分发前使用 Microsoft PowerPoint 进行最终检查。”

## 模板填充工作流

复用并跟踪进度：

```
演示文稿制作进度：
- [ ] 阶段 1：提取并验证源数据
- [ ] 阶段 2：将内容映射至模板板块
- [ ] 阶段 3：以正确格式填充幻灯片
- [ ] 阶段 4：验证 → 修复 → 重复，直至完善
- [ ] 阶段 5：最终校验
```

### 阶段 1：数据提取
1. 在进行任何修改前**创建原始模板备份** —— 复制为 `[文件名]_backup.pptx`。直接编辑 XML 或意外错误可能导致文件损坏。
2. 识别所有源材料（Excel、CSV、PDF 报告、Word 文档、数据库、网络资源）。
3. 从每个渠道提取相关数据点。
4. 对照原始出处验证所有数字。
5. 标准化单位和货币（将所有数值转化为模板使用的主单位/货币）。
6. 记录任何需要校验的计算过程 → 参见 [`calculation-standards.md`](reference/calculation-standards.md) 获取公式。

### 阶段 2：内容映射
1. **打开并视觉审查模板** —— 在修改前充分理解其结构、风格及现有内容。
2. 分析模板结构 —— 识别所有占位符区域和内容框。
3. 将源数据映射至对应的模板板块 → 参见 [`slide-templates.md`](reference/slide-templates.md) 获取映射指南。
4. 识别占位符指示框（由任务创建者提供的彩色说明框）。
5. 记录任何数据缺失或不匹配的情况 → 参见 [`slide-templates.md`](reference/slide-templates.md#handling-data-template-mismatches) 获取解决方案。

### 阶段 3：模板填充
1. **移除或重新格式化占位符框** —— 彩色说明框仅展示“要创建什么”，而非“如何格式化”。删除它们，并在原地创建格式规范的内容。参见 [核心反模式（严禁行为）](#核心反模式严禁行为)。
2. 使用映射好的内容填充各板块（首先关注内容）。
3. **随后应用格式** 以适配模板风格 → 参见 [`formatting-standards.md`](reference/formatting-standards.md)。
4. 将表格创建为真实的表格对象（严禁使用管道符/制表符分隔的文本） → 参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。
5. 将箭头/形状创建为 PowerPoint 对象 → 参见 [`xml-reference.md`](reference/xml-reference.md#arrow-shapes)。
6. 若任务文件中提供了公司 Logo 则插入；若无，请向用户标注："[LOGO NOT PROVIDED - please supply company logo]"。

### 阶段 4：验证 → 修复 → 重复

**这是一个反馈循环。重复直至所有检查通过或触发升级。**

```bash
# 转换为图片以便进行视觉验证
soffice --headless --convert-to pdf presentation.pptx
pdftoppm -jpeg -r 150 presentation.pdf slide
```

**验证清单（检查每一张幻灯片图片）：**
- [ ] 文本在背景下是否清晰易读？
- [ ] 表格是否为真实对象（列对齐，非分隔文本）？
- [ ] 图表/表格是否填满了指定区域？
- [ ] 板块内的项目符号格式是否统一？
- [ ] 同级别文本框的字号是否一致？
- [ ] 内容是否超出幻灯片边界？
- [ ] **是否清除了占位符格式**（严禁保留填入数据的彩色大框）？
- [ ] **是否存在文本模拟的“表格”**（严禁使用 `|` 或制表符生成假列）？
- [ ] **多页一致性**：在所有出现的页面中，同一指标/数值是否完全一致？

**修复周期协议：**

| 周期 | 行动 |
|-------|--------|
| 1 | 修复所有识别出的问题，重新验证 |
| 2 | 修复剩余问题，重新验证 |
| 3 | 若问题持续存在，记录问题并向用户升级反馈 |

**3 个周期后若问题仍未解决：**
1. 列出每个未解决的问题，注明页码和描述。
2. 说明已尝试的操作。
3. 交付文件时附带明确免责声明：“以下问题无法自动解决：[列表]。需要人工干预。”

**严禁**无限循环。某些问题（字体渲染、复杂形状对齐）可能需要在 PowerPoint 中手动干预。

### 阶段 5：最终校验

在交付前请完整检查 [最终质量标准清单](#最终质量标准清单)。

| 层级 | 示例 | 典型字号 | 样式 |
|-------|----------|--------------|-------|

### 字体一致性匹配

处于**同一层级**的文本框必须使用相同的字号：

| 同级元素 A | 必须匹配元素 B |
|------------|-----------------|
| “包含细分” | “排除细分” |
| “定义” | “范畴依据” |
| 左栏项目符号 | 右栏项目符号 |
| 所有模块标签 | 彼此匹配 |
| 所有板块标题 | 彼此匹配 |

### 演示文稿取整规范

这些是**典型惯例** —— 请根据数值量级和模板风格灵活调整：

| 数值类型 | 典型取整方式 | 示例 |
|------------|------------------|---------|
| 大型市场规模 ($10bn+) | 取整至 1bn | 18.5 → $19bn |
| 小型市场规模 (<$10bn) | 取整至 0.5bn 或 0.1bn | 2.3 → $2.5bn |
| 规模区间 | 与源数据精度保持一致 | 14.9-22.1 → $15-22bn |
| CAGR (复合年均增长率) | 整数 % 或 0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 取整至 5% 或匹配源数据 | 21.4% → 20% |
| 倍数 (Multiples) | 小数点后 1 位 | 9.69 → 9.7x |

**原则：** 取整不应实质性改变数值意义。数值越小，精度要求越高。

### 文本密度规则

- 每个内容框最多 6-7 条项目符号。
- 每条项目符号最多 2 行。
- 括号内的示例：放在同一行或缩进至下方。
- 严禁“孤行字”（即新行只有一个单词）。

### 对齐原则

**垂直堆叠的框**必须保持以下一致：
- 左边距、项目符号缩进、文本起始位置、边框宽度。

**水平相邻的框**必须保持以下一致：
- 顶部位置、高度（尽可能一致）、内部边距 (Padding)。

### 多页一致性

当同一数据出现在多张幻灯片上时：
- 使用完全相同的数值、格式和术语。
- 若在某一页更新了指标，必须同步更新所有出现的页面。
- 验证时需进行交叉比对，以纠正不匹配项。

## 强制性要求

以下要求无论何种模板均不可妥协：

| 要求 | 详情 |
|-------------|---------|
| **文本易读性** | 所有文本必须与背景有足够对比度。示例：深蓝/深绿/黑色背景配白色/浅色字；白色/浅灰/淡黄背景配黑色/深色字。 |
| **真实表格对象** | 表格数据必须是真正的表格对象，而非通过空格/制表符生成的文本。见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。 |
| **合理的图表/表格尺寸** | 粘贴的视觉元素必须填满指定区域。见 [`formatting-standards.md`](reference/formatting-standards.md#chart-and-image-handling)。 |
| **格式一致性** | 板块内的符号、字号、缩进必须统一。同级框必须使用相同字号。 |
| **内容边界** | 所有内容严禁超出幻灯片边缘。脚注框宽度：16:9 约为 32.5cm，4:3 约为 24cm。 |
| **严禁残留占位符格式** | 必须移除彩色指示框。主体内容应根据模板采用浅底深色字。 |

## 核心反模式：严禁行为

当将“占位符格式”误认为“输出格式”时，就会出现这些错误。识别这些模式至关重要。

### 反模式 1：将数据填入占位符框中

**表现：** 模板中有带有引导文字的彩色说明框（黄、橙等）。模型替换了引导文字，但**保留了彩色底框**。

**错误原因：** 彩色框本身即是占位符。它是在告诉你这里该放什么。最终产出应具有不同的格式 —— 通常是白色/浅色背景上的深色文本。

**识别测试：** 如果填充后的幻灯片上有巨大的彩色矩形且填满了文本，说明你复制了占位符格式而非替换它。

**关键区分 —— 两种“占位符”：**

| 类型 | 如何识别 | 处理方式 |
|------|-----------------|------------|
| **说明框 (Instruction boxes)** | 鲜艳颜色（黄、橙），包含“此处插入 X”等引导语 | **删除**整个形状，随后创建符合生产规范的内容 |
| **布局占位符 (Layout placeholders)** | 幻灯片母版的一部分，颜色中性且适配主题，“点击以添加文本” | **保留**形状，**仅替换**其中的文本内容 |

若不确定：检查该模板的新建空白页中是否存在该形状。布局占位符会持续存在，而说明框只是普通形状。

### 反模式 2：文本模拟的“表格”

**表现：** 模型使用分隔符（如 `|`、制表符、空格）来构建模拟表格的内容，而非创建真实的表格对象。

**错误原因：** 这**不是**表格。列永远无法精准对齐，无法统一格式化，且极其不专业。

**识别测试：** 如果你在输入 `|` 字符或依赖空格对齐列，你创建的是文本，不是表格。

**必须校验：** 创建任何表格后，请验证其为真实的表格对象。参见 [`xml-reference.md`](reference/xml-reference.md#critical-verify-tables-are-actual-table-objects) 获取验证方法。

### 反模式 3：沿用占位符的高对比度配色

**表现：** 占位符在彩色背景上使用浅色字（如黄底白字）。模型填入数据但保留了这种配色方案，导致产出难以阅读。

**错误原因：** 占位颜色故意设为显眼，以发出“替换我”的信号。正式幻灯片的主体内容通常采用浅底深色字。

**识别测试：** 如果正文区域（非页眉）出现鲜艳底色配白色/浅色字，说明你沿用了占位符格式。

**正确做法：** 应用生产格式 —— 正文内容通常采用白色或浅色背景配深色文本（#000000 或 #333333）。页眉和点缀区域可使用品牌色。

### 总结：占位符 vs. 生产交付

| 元素 | 占位符 (输入) | 生产交付 (输出) |
|---------|---------------------|---------------------|
| 说明框 | 彩色背景，引导文本 | 移除或重新格式化 |
| 数据区域 | “[此处插入数据]”文本 | 格式整洁的真实数据 |
| 表格 | 描述表格应包含的内容 | 具有行列结构的真实表格对象 |
| 正文 | 彩色背景上的浅色字 | 浅色背景上的深色字 |

**占位符告诉你“要创建什么”，而不是“要呈现为什么格式”。**

## 错误处理

**若 PDF/图片转换失败：**
1. 检查是否安装 LibreOffice：`which soffice`。
2. 尝试备用命令：`libreoffice --headless --convert-to pdf presentation.pptx`。
3. 若仍失败，手动打开 PowerPoint/LibreOffice 进行导出。

**若源数据存在冲突或不一致：**
1. **优先顺序**：首先使用任务文件中明确提供的数据。
2. 若使用了其他来源（网搜、外部文档），请向用户标注。
3. 明确记录所有数据差异。
4. 添加脚注说明选用的数据源依据。

**若计算结果与源文件预测不符：**
1. 展示你的计算方法论。
2. 记录差异及可能原因（基准年不同、方法论差异）。
3. 若差异重大，同时呈现两个数值。
4. 交由用户裁定。

## 表格结构指南

创建表格时（必须是真实的表格对象）：

**列对齐方式：**
- 文本列：左对齐（表头与内容一致）。
- 数值列：右对齐或居中对齐（表头与内容一致）。

**表头行：**
- 文本加粗。
- 背景底纹（使用模板品牌色）。
- 使用白色或高对比度文本。

**市场共识 / 合计行：**
- 文本加粗。
- 上方添加分隔线。
- 使用独特的背景底纹。

**宽度：** 必须完全填满指定的板块宽度。

XML 实现细节请参考 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。

## 脚注格式

**格式：**
```
Sources: [来源 1] (年份), [来源 2] (年份).
Notes: (1) [第一条备注]; (2) [第二条备注].
```

**示例：**
```
Sources: Grand View Research (2024), Mordor Intelligence (2024), Markets and Markets (2023).
Notes: (1) 不包含硬件收入; (2) 涵盖 B2B 与 B2C 细分市场。
```

幻灯片主体中的所有上标数字（¹、²、³）必须在脚注中有对应的 Notes 条目。

## 最终质量标准清单

在交付填充后的模板前，请核实：

> **💡 Appendix: 领域知识小贴士**
>
> 1. **什么是 Pitch Deck？**
>    它是投行带去见客户的“敲门砖”。想象一下，如果你要帮一个大公司融资或者把它卖掉，你需要一份精美且有说服力的 PPT 来告诉潜在投资者：这个生意有多赚钱、市场有多大、我们要多少钱。这就是 Pitch Deck。
>
> 2. **CAGR（复合年均增长率）：**
>    别被名字吓到，它就是投资界的“魔法拉平公式”。如果一个市场第一年涨 10%，第二年跌 5%，第三年涨 20%，CAGR 就是算出一个“平均数”，描述这三年里每年的平均增长水平。
>
> 3. **TAM（总潜在市场规模）：**
>    就是这个生意“天花板”的高度。比如你卖冰激凌，TAM 就是全世界所有人都在买冰激凌时，这个市场能有多少钱。
>
> 4. **倍数 (Multiples)：**
>    估值的简易计算器。比如类似的咖啡店通常卖到年利润的 10 倍，这个“10x”就是倍数。如果你的店利润是 100 万，按照 10 倍来算，你的店大概就值 1000 万。
>
> 5. **可比交易法 (Transaction Comparables)：**
>    俗称“看看邻居家卖了多少钱”。你要估算一个公司的身价，最直接的方法就是看最近半年里，跟它规模、行业相似的公司在二手交易中卖了多少钱。
