# 财务模型格式化标准指南

| 元素 | 格式要求 |
|---------|--------|
| 硬编码输入 (静态录入项) | 蓝色字体 |
| 公式 | 黑色字体 |
| 跨表链接 (引用其他工作表) | 绿色字体 |
| 勾稽核查项 | 报错显示红色，平衡显示绿色 |
| 负值 | 使用圆括号，不使用负号 |
| 货币符号 | 高额数值不保留小数，每股收益保留 2 位小数 |
| 百分比 | 保留 1 位小数 |
| 标题行 | 加粗，带下边框 |
| 单位行 | 在标题下方添加单位行（如：百万美元、%、等） |

## 视觉分隔规范

- 在历史数据列与预测数据列之间使用细垂直边框分隔
- 在各板块总计（如：资产总计）下方使用粗底边框
- 小计项 (Subtotals) 使用单底边框
- 总计项 (Grand Totals) 使用双底边框

## 总计与小计行格式规范

所有总计和小计行的数值必须使用**加粗格式**，以便将汇总数据与明细项目清晰地区分开。

### 损益表 (P&L) 标签页
| 行项目 | 格式要求 |
|-----|------------|
| 营业总收入 | 加粗 |
| 营业成本总计 | 加粗 |
| 毛利润 | 加粗 |
| 销售及管理费用 (SG&A) 总计 | 加粗 |
| EBITDA (息税折旧摊销前利润) | 加粗 |
| EBIT (息税前利润) | 加粗 |
| EBT (税前利润) | 加粗 |
| 税后净利润 | 加粗 |

### 资产负债表 (BS) 标签页
| 行项目 | 格式要求 |
|-----|------------|
| 流动资产总计 | 加粗 |
| 非流动资产总计 | 加粗 |
| 其他资产总计 | 加粗 |
| 资产总计 | 加粗 |
| 流动负债总计 | 加粗 |
| 非流动负债总计 | 加粗 |
| 所有者权益总计 | 加粗 |
| 负债及所有者权益总计 | 加粗 |

### 现金流量表 (CFS) 标签页
| 行项目 | 格式要求 |
|-----|------------|
| 营运资本变动前经营现金流 | 加粗 |
| 营运资本变动总额 | 加粗 |
| 经营活动产生的现金流量净额 | 加粗 |
| 投资活动产生的现金流量净额 | 加粗 |
| 筹资活动产生的现金流量净额 | 加粗 |
| 期末现金余额 | 加粗 |

**注：** 以上列表并未穷尽。模型中任何代表总额、小计或汇总计算的行项目均应应用加粗格式。

## 资产负债表勾稽核查行格式

资产负债表核查行（位于“负债及所有者权益总计”下方）应使用条件数字格式，将非零值显示为红色。当资产负债表正确平衡（差额 = 0）时，数值显示为黑色或标准格式。

| 核查值 | 字体颜色 |
|-------------|------------|
| = 0 (已平衡) | 黑色 (标准) |
| ≠ 0 (存在错误) | 红色 |

**实现方式：** 应用自定义数字格式 `[Red][<>0]0.00;[Red][<>0](0.00);0.00` 或使用 Excel 条件格式规则：“单元格数值 ≠ 0” → 红色字体。

## 利润率行格式规范

| 元素 | 格式要求 |
|---------|--------|
| 利润率 % 行 | 缩进，斜体，保留 1 位小数 |
| 正向趋势 | 无特殊格式（或使用浅绿色） |
| 负向趋势 | 标记以供复核（使用浅黄色） |
| 低于同业平均水平 | 考虑高亮显示以备讨论 |

## 信用指标格式规范

| 元素 | 格式要求 |
|---------|--------|
| 杠杆倍数 | 1 位小数并带“x”后缀（如：2.5x） |
| 百分比 | 1 位小数并带“%”后缀 |
| 净债务为负 | 使用圆括号，表示处于净现金头寸 |
| 板块标题 | 加粗，“信用指标 (CREDIT METRICS)” |
| 分隔线 | 在信用指标板块上方添加细边框 |

## 信用指标阈值颜色预警

| 指标项目 | 绿色 (安全) | 黄色 (预警) | 红色 (风险) |
|--------|-------|--------|-----|
| 总债务 / EBITDA | < 2.5x | 2.5x-4.0x | > 4.0x |
| 净债务 / EBITDA | < 2.0x | 2.0x-3.5x | > 3.5x |
| 利息保障倍数 | > 4.0x | 2.5x-4.0x | < 2.5x |
| 债务 / 总资本比率 | < 40% | 40%-60% | > 60% |
| 流动比率 | > 1.5x | 1.0x-1.5x | < 1.0x |
| 速动比率 | > 1.0x | 0.75x-1.0x | < 0.75x |

## 核查页 (Checks Tab) 条件格式

- 单元格包含“通过 (Pass)”标识 → 绿色填充
- 单元格包含“未通过 (Fail)”标识 → 红色填充
- 单元格包含“警告 (Warning)”标识 → 黄色填充
- 差额单元格 = 0 → 浅绿色填充
- 差额单元格 ≠ 0 → 浅红色填充

## 利润率合理性预警

- 毛利率 < 0% → **错误**：需复核营业成本 (COGS)
- 毛利率 > 80% → **警告**：需验证收入或营业成本的准确性
- EBITDA 利润率 < 0% → **重点标记**：存在经营性亏损
- EBITDA 利润率 > 50% → **警告**：数值异常偏高
- 净利率 < 0% → **重点标记**：净亏损（在扩张期或可接受）
- 净利率 > 毛利率 → **错误**：公式逻辑存在问题

> **💡 Appendix: 领域知识小贴士**
>
> **1. 为什么财务报表里的负数都在“口袋”（圆括号）里？**
> 在专业的金融模型里，我们很少用减号“-”来表示负数。因为在密集的表格中，减号很容易被看成是边框线或者连字符。用圆括号 `(100)` 把数字包起来，不仅更加美观，还能让人一眼就看出这是“支出的现金”或“亏损”。
>
> **2. 什么是 EBITDA？（读作：e-bit-dah）**
> 想象你经营一家咖啡店。EBITDA 指的是你卖咖啡赚到的钱，但还没支付银行利息、还没给国家交税、还没算咖啡机每年的折旧费用。它是衡量生意“造血能力”最直观的指标，因为它剔除了不同公司之间税务或贷款方式的不同，让大家在同一个起跑线上比拼赚钱能力。
>
> **3. 勾稽核查 (Checks) 到底在核查什么？**
> 财务有个金科玉律：资产必须等于负债加股东权益。我们在模型里专门做一行“核查行”，用资产减去负债和权益。如果结果不是 0，那就说明模型“炸了”，或者是漏算了某项数据。显示红色就像是警报灯，提醒建模师：你的表格没对齐！
>
> **4. 杠杆倍数 (2.5x) 是什么意思？**
> “x”代表倍数。如果一个公司的“净债务/EBITDA”是 3.5x，通俗点说就是：如果这家公司把每年赚到的现金全部拿去还债，需要 3.5 年才能还清。倍数越高，说明借的钱相对于赚钱能力越多，风险也就越大。
