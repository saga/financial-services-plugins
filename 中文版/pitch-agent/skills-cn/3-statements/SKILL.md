# 三表财务模型模板填充指南

完成并填充集成财务模型模板，确保利润表（IS）、资产负债表（BS）和现金流量表（CF）之间建立正确的勾稽关系。

## ⚠️ 核心原则 —— 在填充任何模板前必读

**开发环境 —— Office JS 对比 Python：**
- **如果在 Excel 内部运行（Office 加载项 / Office JS）：** 直接使用 Office JS。通过 `range.formulas = [["=D14*(1+Assumptions!$B$5)"]]` 写入公式 —— 严禁对派生单元格使用 `range.values`。无需单独重算，Excel 会原生计算。使用 `context.workbook.worksheets.getItem(...)` 进行工作表切换。
- **如果生成独立的 .xlsx 文件：** 使用 Python/openpyxl。写入 `ws["D15"] = "=D14*(1+Assumptions!$B$5)"`，随后在交付前运行 `recalc.py`。
- **Office JS 合并单元格陷阱：** 严禁先调用 `.merge()` 再对合并区域设置 `.values` —— 这会因区域仍报告合并前的维度而抛出 `InvalidArgument` 错误。应先将值写入左上角单元格，再进行合并及格式化：`ws.getRange("A1").values = [["利润表"]]; const h = ws.getRange("A1:G1"); h.merge(); h.format.fill.color = "#1F4E79";`
- 以下所有原则在上述两种环境下均同样适用。

**公式优于硬编码（不可逾越的底线）：**
- 每一个预测单元格、滚动推演、报表勾稽和分类小计都必须是 Excel 公式 —— 严禁使用预计算的固定数值。
- 使用 Python/openpyxl 时：写入公式字符串（`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`），而非计算结果（`ws["D15"] = 12500`）。
- **唯二**允许包含硬编码数字的单元格是：(1) 历史实际数据，(2) “假设”工作表中的驱动因素。
- 如果你发现自己在 Python 中计算了一个值并将其写入单元格 —— **请立即停止**。改为写入公式。
- 原理：模型必须能够在情景切换或假设变更时灵活变动。硬编码会静默破坏后续的所有完整性检查。

**与用户进行分步确认：**
1. **模板映射完成后** → 向用户展示你识别出的工作表/板块，确认无误后再操作单元格。
2. **填充历史数据后** → 展示历史数据区块，确认数值和会计期间与原始数据匹配。
3. **构建利润表（IS）预测后** → 运行小计检查，向用户展示预测的利润表，确认后方可进入资产负债表。
4. **构建资产负债表（BS）后** → 展示每一期的配平检查（资产 = 负债 + 所有者权益），确认后方可进入现金流量表。
5. **构建现金流量表（CF）后** → 展示现金平账检查（CF 期末现金 = BS 货币资金），确认后最终定案。
6. **严禁一次性填充整个模型后直接交付** —— 必须在每张报表完成时中断并展示工作成果，及早发现错误。

## 格式规范 —— 专业蓝灰配色（除非模板或用户另有要求，否则作为默认方案）

**保持颜色简洁。** 仅使用蓝色和灰色作为单元格填充色。严禁引入绿色、黄色、橙色或多种强调色 —— 专业的模型应保持克制。

| 元素 | 填充色 (Fill) | 字体颜色 (Font) |
|---|---|---|
| 板块标题 (IS / BS / CF 标题) | 深蓝色 `#1F4E79` | 白色 加粗 |
| 列标题 (FY2024A, FY2025E 等) | 浅蓝色 `#D9E1F2` | 黑色 加粗 |
| 输入单元格 (历史数据、假设驱动) | 浅灰色 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨表链接 | 白色 | 绿色 `#008000` |
| 检查行 / 核心合计数 | 中蓝色 `#BDD7EE` | 黑色 加粗 |

**配色原则为：3 种蓝色 + 1 种灰色 + 白色。** 若模板已有其自身的配色方案，请遵循模板。

字体颜色代表单元格的**性质**（输入/公式/链接）。填充颜色代表你所处的**位置**（标题/数据/检查）。

## 模型结构

### 识别模板工作表组织

不同模板的命名习惯和组织方式各异。在填充前，请预览所有工作表以理解结构。以下是常见的工作表名称及其典型内容：

| 常见工作表名称 | 寻找的内容 |
|------------------|----------------------|
| IS, P&L, Income Statement | 利润表 / 损益表 |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资本表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销表 |
| Debt, Debt Schedule | 债务偿还表 |
| NOL, Tax, DTA | 净经营亏损及税务表 |
| Assumptions, Inputs, Drivers | 驱动假设与输入项 |
| Checks, Audit, Validation | 错误检查仪表盘 |

**模板审查清单**
- 识模板中存在哪些工作表（并非所有模板都包含完整明细表）。
- 记录上述未列出的模板特有工作表。
- 理解工作表间的依赖关系（例如，哪些明细表流入主表）。
- 定位每个工作表上的输入单元格与公式单元格。

### 理解模板架构

在填充模板前，熟悉现有布局以确保数据填入正确位置且公式完整。

**行结构识别**
- 找到每个工作表顶部的模型标题。
- 识别板块标题及其视觉分隔。
- 找到单位行，确认单元为：百万元、$、%、x 倍等。
- 留意区分“实际数”（Actuals）与“预测数”（Estimates）的列标题。
- 确认期间标签（如 FY2024A, FY2025E）。
- 识别输入单元格与公式单元格（通常通过字体颜色区分）。

**列结构识别**
- 确认最左侧列的科目名称。
- 验证历史年度在预测年度之前。
- 注意分隔历史期间与预测期间的视觉边界。
- 检查所有工作表的列顺序是否保持一致。

**使用命名区域 (Named Ranges)**
专业模板常使用命名区域。输入数据前：
- 审查模板中现有的命名区域（Excel 中：公式 → 名称管理器）。
- 常见的命名区域包括：营收增长率、成本百分比、核心输出（净利润、EBITDA、总债务、现金）、情景选择单元格。
- 确保输入的数据填入了馈入这些命名区域的单元格。

### 预测期间
- 模板通常从最后一个历史年度向后预测 5 年。
- 验证历史 (A) 与预测 (E) 列是否清晰分隔。
- 确认列标题使用财年符号（如 FY2024A, FY2025E）。

## 利润率分析

**注意：仅在用户要求或模板明确要求时执行以下利润率分析。若无要求，请跳过此部分。**

在利润表（IS）工作表计算并显示盈利能力指标，以追踪经营效率并进行同行对比。

### 核心盈利指标

| 指标 | 公式 | 衡量维度 |
|--------|---------|------------------|
| 毛利率 (Gross Margin) | 毛利润 / 营业收入 | 定价权、生产效率 |
| EBITDA 利润率 | EBITDA / 营业收入 | 核心业务盈利能力 |
| EBIT 利润率 | EBIT / 营业收入 | 扣除折旧摊销后的经营获利能力 |
| 净利率 (Net Income Margin) | 净利润 / 营业收入 | 最终盈利水平 |

### 带利润率分析的利润表布局

在每个利润科目下方直接显示百分比：
- 毛利润下方显示毛利率 %
- EBIT 下方显示 EBIT 利润率 %
- EBITDA 下方显示 EBITDA 利润率 %
- 净利润下方显示净利率 %

## 信用指标分析

**注意：仅在用户要求或模板明确要求时执行以下信用分析。若无要求，请跳过此部分。**

在资产负债表（BS）工作表计算并显示信用/杠杆指标，以评估财务健康状况、举债能力及契约遵守情况。

### 核心信用指标

| 指标 | 公式 | 衡量维度 |
|--------|---------|------------------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | (总债务 - 现金) / LTM EBITDA | 扣除现金后的净杠杆 |
| 利息覆盖率 | EBITDA / 利息支出 | 债务偿付能力 |
| 债务 / 总资本 | 总债务 / (总债务 + 权益) | 资本结构 |
| 产权比率 | 总债务 / 所有者权益合计 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期偿债能力 |
| 速动比率 | (流动资产 - 存货) / 流动负债 | 即时变现能力 |

### 信用指标逻辑检查

验证在“乐观情景”（Upside）下是否显示最强的信用状况：
- 杠杆率：乐观 < 基准 < 悲观（越低越好）
- 覆盖率：乐观 > 基准 > 悲观（越高越好）
- 流动性：乐观 > 基准 > 悲观（越高越好）

### 债务契约追踪

如果已知债务契约（Covenants），请添加明确的合规性检查，将实际指标与契约阈值进行对比。

## 情景分析 (基准 / 乐观 / 悲观)

在假设工作表中使用情景切换开关（下拉菜单），配合 `CHOOSE` 或 `INDEX/MATCH` 公式。

| 情景 | 描述 |
|----------|-------------|
| 基准情景 (Base) | 管理层预期或市场共识预测 |
| 乐观情景 (Upside) | 高于预期的增长、利润率扩张 |
| 悲观情景 (Downside) | 低于趋势的增长、利润率收缩 |

**核心敏感驱动因素**：营收增长、毛利率、销售及管理费用 %、DSO/DIO/DPO（周转天数）、资本支出 %、利率、税率。

**情景审计检查**：切换开关是否联动所有报表；所有情景下资产负债表是否配平；现金是否勾稽；逻辑层级是否成立（净利润、EBITDA、自由现金流和利润率：乐观 > 基准 > 悲观）。

## 完成模型模板的通用步骤

本节提供在保留现有公式并确保数据完整性的前提下，完成任何三表模型模板的通用指导。

### 第一步：分析模板结构

在输入任何数据前，彻底审查模板架构：

**识别输入单元格与公式单元格**
- 寻找区分输入与公式的视觉线索（颜色、阴影）。
- 通用惯例：蓝色字体 = 输入，黑色字体 = 公式，绿色字体 = 跨表链接。
- 使用 Excel 的“追踪引用单元格”（公式 → 追踪引用单元格）理解关联。
- 检查是否存在控制核心输入的命名区域。

**映射模板流向**
- 识别工作表的勾稽流向（如：假设 → IS → BS → CF）。
- 记录支撑明细表及其与主表的链接。
- 在填充前记录模板特有的科目项和结构。

### 第二步：在不破坏公式的情况下填入数据

**数据输入金律**

| 原则 | 详细说明 |
|------|-------------|
| 仅编辑输入单元格 | 严禁覆盖包含公式的单元格，除非有意替换该公式 |
| 保留单元格引用 | 复制数据时，使用“选择性粘贴数值”（Ctrl+Shift+V）以防破坏格式或覆盖公式 |
| 匹配模板单位 | 输入前验证模板是使用“千”、“百万”还是“原值” |
| 尊重符号惯例 | 遵循模板现有的正负号习惯（如费用记为正值还是负值） |
| 检查循环引用 | 若模板包含迭代计算，确保开启了“启用迭代计算” |

**安全填充流程**
1. 识别专门用于输入的单元格（通常有高亮或标签）。
2. 先填入历史数据，验证这些期间的公式计算是否正确。
3. 输入驱动预测计算的假设参数。
4. 审查计算输出，确认公式按预期运行。
5. 若必须修改公式单元格，请先备份原始公式。

**处理预置公式**
- 如果公式引用了尚未填充的单元格，出现临时错误（#REF!, #DIV/0!）是正常的，直到输入完整。
- 当公式产生意外结果时，追踪其引用项以定位缺失或错误的输入。
- 严禁在未检查跨表公式依赖的情况下删除行/列。

### 第三步：验证公式

**公式完整性检查**

在依赖模板输出前，验证公式运行是否正确：

| 检查类型 | 方法 |
|------------|--------|
| 追踪引用项 | 选中公式单元格 → 公式 → 追踪引用单元格，验证其引用了正确的输入 |
| 追踪从属项 | 验证核心输入项是否流向了预期的输出单元格 |
| 分步求值 | 使用“公式”→“分步求值”分析复杂的计算逻辑 |
| 检查硬编码 | 预测公式应引用假设，严禁在公式中包含硬编码数值 |
| 测试已知值 | 输入简单的测试值，验证公式是否产生预期的计算结果 |
| 跨期一致性 | 确保同一公式逻辑应用于整个预测期间 |

**需警惕的常见公式问题**
- 绝对/相对引用混淆，导致横向复制时结果错误。
- 外部链接断开或引用了已删除的区域（出现 #REF! 错误）。
- 在营收尚未产生的早期期间出现除以零（出现 #DIV/0! 错误）。
- 循环引用警告（在计算利息支出时可能是故意的）。
- 预测列之间的公式不一致（使用 Ctrl+\ 寻找差异）。

**验证跨表勾稽链接**
- 确认出现在多个工作表上的数值是通过链接引用（而非重复录入）。
- 验证明细表的合计项与主表中的相应行项完全对等。
- 检查所有工作表的会计期间标签是否对齐。

### 第四步：各报表专项质检

填充模板后，对每张报表进行如下验证：

**利润表 (IS) 质检项**
- 历史期间的营收数据与原始来源一致。
- 所有费用明细项通过求和等于报告的总额。
- 小计科目（毛利润、EBIT、EBT、净利润）计算正确。
- 所得税计算逻辑合理（能正确处理亏损）。
- 预测驱动因素全部引用自“假设”页（无硬编码）。
- 期间变动趋势符合常理。

**资产负债表 (BS) 质检项**
- 每一期均满足 资产 = 负债 + 所有者权益（首要检查）。
- 货币资金余额与现金流量表期末现金一致。
- 营运资本科目与支撑明细表对应（如有）。
- 留存收益滚动推演正确：期初留存 + 净利润 - 股利 +/- 调整 = 期末留存。
- 债务余额与债务偿还表一致（如有）。
- 所有科目符号正确（资产通常为正，负债多数为正）。

**现金流量表 (CF) 质检项**
- 经营活动（CFO）顶部的净利润与利润表一致。
- 非现金加回项（折旧摊销 D&A、股份支付 SBC 等）与来源一致。
- 营运资本变动正负号正确（资产增加 = 现金流出 = 负数）。
- 资本支出（CapEx）与 PP&E 明细表或固定资产滚动推演一致。
- 筹资活动与资产负债表中的债务和权益科目变动勾稽。
- 期末现金等于资产负债表中的货币资金。
- 期初现金等于上一期的期末现金。

**明细表质检项**
- 期初余额等于上一期的期末余额。
- 滚动逻辑完整（期初 + 增加 - 减少 = 期末）。
- 明细表合计数勾稽至主表行项。
- 所用假设与“假设”页一致。

### 第五步：跨表勾稽完整性检查

在验证完单张报表后，确认三表勾稽关系正常：

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表配平 | 资产 - 负债 - 所有者权益 | = 0 |
| 现金勾稽 | CF 期末现金 - BS 货币资金 | = 0 |
| 净利润勾稽 | IS 净利润 - CF 起始净利润 | = 0 |
| 留存收益勾稽 | 期初留存 + 净利润 - 股利 - BS 期末留存 | = 0（视情况调整 SBC 等） |

### 第六步：最终审查

在宣布模型完成前：
- 切换所有情景（如有），验证每种情景下检查都能通过。
- 检查并解决所有 #REF!, #DIV/0!, #VALUE!, 和 #NAME? 错误。
- 确认所有输入单元格已填充（搜索占位符）。
- 验证所有工作表的单位保持一致。
- 在进行任何额外微调前保存一个“干净版本”。

## 模型验证与审计

本节总结了针对已完成模板的所有验证检查和审计程序。

### 核心勾稽关系（必须始终成立）

公式详情请参阅 [references/formulas.md](references/formulas.md)。

| 检查项 | 公式 | 预期结果 |
|-------|---------|-----------------|
| 资产负债表配平 | 资产 - 负债 - 所有者权益 | = 0 |
| 现金勾稽 | CF 期末现金 - BS 货币资金 | = 0 |
| 现金月/年对齐 | 期末现金（月度） - 期末现金（年度） | = 0 |
| 净利润勾稽 | IS 净利润 - CF 起始净利润 | = 0 |
| 留存收益滚动 | 期初留存 + 净利润 + SBC - 股利 - BS 期末留存 | = 0 |
| 权益融资检查 | Δ普通股/资本公积 (BS) - 权益募资 (CFF) | = 0 |
| 第 0 年权益检查 | 第 0 年筹资额 - 第 1 年期初权益资本 | = 0 |

### 循环引用处理

利息支出会产生循环：利息支出 → 净利润 → 现金 → 债务余额 → 利息支出

在 Excel 中启用迭代计算：文件 → 选项 → 公式 → 启用迭代计算。设置最大迭代次数为 100，最大误差为 0.001。同时在“假设”页添加一个循环引用断路开关。

### 检查类别

**第 1 部分：货币单位一致性**
- 在“假设”页识别并记录币种。
- 所有工作表使用一致的货币符号和量级（缩进）。
- 单位行与模型币种匹配。

**第 2 部分：资产负债表完整性**
- 每一期均满足 资产 = 负债 + 所有者权益。
- 公式：资产 - 负债 - 权益 (必须 = 0)。

**第 3 部分：现金流完整性**
- 现金勾稽至 BS（CF 期末现金 = BS 资产负债表现金）。
- 净利润勾稽至 IS（CF 净利润 = IS 净利润）。
- 加回项（D&A, SBC 等）勾稽至明细表/报表。
- 营运资本变动勾稽至明细表。
- 资本支出（CapEx）勾稽至折旧摊销明细表。

**第 4 部分：留存收益**
- 滚动检查：期初留存 + 净利润 + SBC - 股利 = 期末留存。
- 显示各组件拆解以便调试。

**第 5 部分：营运资本**
- 应收、存货、应付勾稽至资产负债表。
- DSO, DIO, DPO 合理性检查（若超出正常范围则标记）。

**第 6 部分：债务明细表**
- 总债务勾稽至 BS（短期 + 长期债务）。
- 利息计算勾稽至 IS。

**第 6b 部分：权益融资**
- 权益募资所得勾稽至 BS 普通股/资本公积增加额。
- 权益募资现金流勾稽检查 (必须配平)。

**第 6c 部分：净经营亏损 (NOL) 表**
- 第 1 年/成立初期 NOL = 0。
- 仅当 EBT < 0 时 NOL 才会增加（实现亏损产生 NOL）。
- DTA 勾稽至 BS（NOL 表中的 DTA = 资产负债表递延所得税资产）。
- NOL 抵扣 ≤ EBT 的 80%（遵循 2017 年后的税务限制）。
- NOL 余额非负；税前利润小于等于 0 时所得税费用为 0。

**第 7 部分：情景层级**
- 绝对指标：乐观 > 基准 > 悲观（净利润、EBITDA、自由现金流）。
- 利润率：乐观 > 基准 > 悲观（毛利率%、EBITDA%、净利率%）。
- 信用指标：乐观 < 基准 < 悲观（针对杠杆率，逻辑相反）。

**第 8 部分：公式完整性**
- 营业成本、销售/管理/研发费用、股份支付均受“营收 %”驱动（无硬编码）。
- 整个预测期公式保持一致。
- 无 #REF!, #DIV/0!, #VALUE! 错误。

**第 9 部分：信用指标阈值**
- 根据契约阈值将指标标记为 绿/黄/红 灯。
- 汇总所有异常预警。

### 总检查公式

将所有板块状态聚合为一个总检查：
- 若所有部分均通过 → "✓ 所有检查通过"
- 若任一部分失败 → "✗ 检测到错误 - 请查看下方明细"

### 快速调试流程

当总状态显示错误时：
1. 滚动查找红色高亮的板块。
2. 识别哪个类别的检查未通过。
3. 跳转至源工作表进行调查。
4. 修复底层问题。
5. 返回“检查”页验证修复结果。

> **💡 Appendix: 领域知识小贴士**
>
> 欢迎来到财务建模的世界！如果你是一张白纸也别担心，这里有几个锦囊让你秒懂“三表模型”：
>
> 1.  **“三表”到底是什么？**
>     *   **利润表 (IS)**：就是公司的“战报”，记录这一整年赚了多少、花了多少，最后剩多少利润。
>     *   **资产负债表 (BS)**：这是一个“抓拍”，显示在某一个特定的瞬间，公司手里有多少家当（资产），欠多少债（负债），还有多少是老板自己的钱（所有者权益）。记住这个铁律：**资产 = 负债 + 权益**，不平就说明账错了！
>     *   **现金流量表 (CF)**：这是公司的“呼吸机”，记录钱到底是怎么进进出出的。利润不等于现金（比如卖了货对方还没给钱），所以这张表能告诉你公司手里的真金白银够不够花。
>
> 2.  **什么是“勾稽关系”？**
>     想象这三张表是连在一起的齿轮：利润表的终点（净利润）会通过管道流进资产负债表的“蓄水池”（留存收益），同时也会成为现金流量表的起点。现金流量表算出的期末现金，最后必须精准地填回资产负债表顶部的“现金”那一栏。如果这几个数据对不上，齿轮就卡住了。
>
> 3.  **蓝黑颜色的潜规则**
>     在专业财务圈，蓝色代表“假设”（你拍脑袋定的或者历史上的真实数据），黑色代表“公式”（它是自动算出来的）。如果一个预测出来的利润是蓝色的，那是大忌，因为你手动改了它，它就失去了灵活性。
>
> 4.  **为什么要分“情景”？**
>     做生意有顺风顺水的时候（乐观情景），也有倒霉透顶的时候（悲观情景）。一个好的模型能让你动动手指，就能看到在不同的市场环境下，公司会不会破产，或者能赚多少钱。这就是金融圈常说的“压力测试”！
