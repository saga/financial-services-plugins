# 演示文稿审核报告格式

## 报告模板

```markdown
# 演示文稿审核报告：[演示文稿名称]

## 概要
- 问题总计：X
- 关键 (Critical)：X（数据不匹配、事实错误）
- 重要 (Important)：X（叙述与数据一致性、语言表达）
- 次要 (Minor)：X（格式规范）

## 关键性问题

### 数据一致性
1. **[问题名称]** (第 X, Y 页)
   - 第 X 页：[取值]
   - 第 Y 页：[取值]
   - 改进建议：[处理意见]

### 数据与叙述一致性
1. **[问题名称]** (第 X, Y 页)
   - 文中陈述："[引用文本]"
   - 数据显示：[冲突点]
   - 改进建议：[处理意见]

## 重要性问题

### 语言润色
1. **[问题类型]** (第 X 页)
   - 当前表述："[引用文本]"
   - 建议修改："[替换文本]"

## 次要性问题

### 格式规范
1. **[问题类型]** (第 X 页)
   - [描述与修正方案]

## 最终检查清单
- [ ] 数据已完成平核 (Reconciled)
- [ ] 叙述与数据内容吻合
- [ ] 语言表达符合投行 (IB) 标准
- [ ] 图表均已注明来源
- [ ] 格式外观保持一致
```

## 问题严重程度分类

**关键 (Critical)**（交付客户前必须修正）：
- 跨页数据不匹配
- 计算逻辑错误
- 事实性错误（名称、职衔、日期）
- 数据与文字结论相悖

**重要 (Important)**（建议修正）：
- 语言表述过于口语化或非正式
- 缺乏具体依据的模糊陈述
- 专业术语使用不统一
- 图表缺失引注来源

**次要 (Minor)**（完善项）：
- 字体或配色不统一
- 日期格式不规范（如 2024.1.1 与 24/01/01 混用）
- 间距或对齐问题
- 孤行文字（Orphaned text）

> **💡 Appendix: 领域知识小贴士**
> 
> 在高端金融圈（比如投行和私募），一份 PPT（我们俗称 **Deck**）就是项目的脸面。这里有几个你必须知道的“行话”概念：
> 
> 1.  **数据平核 (Reconciliation)：** 简单说就是“对账”。如果你的第 5 页说公司利润是 1 个亿，第 50 页变成了 1.1 个亿，哪怕只差一块钱，投资人也会怀疑你的专业度。在金融报告里，数据必须在全文任何角落都维持“绝对一致”。
> 2.  **投行标准 (IB Standards)：** 投行对文档有一种近乎强迫症的追求。这包括：不能有错别字、标点符号要全角半角统一、语言要“高端大气”（比如用“提升利用率”而不是“多用用”）、甚至每一张图表的配色都要符合品牌的色号。
> 3.  **数据与叙述一致性 (Data-Narrative Alignment)：** 这是一个高级坑。有时候数据是对的，文字也没错，但放在一起就“打架”。比如图表显示增长率正在放缓，但文字却写着“业绩爆发式增长”，这就属于典型的叙数不符，会严重影响结论的公信力。
> 4.  **孤行文字 (Orphaned text)：** 想象一下，一段话的最后一个字孤零零地掉到了下一行，或者一个段落的最后一行跑到了下一页。在金融文档里，这被视为不够精致的表现，必须要通过微调间距来消除。
