# 余额变动调节 (Roll-forward)

根据给定的科目（或科目组）、实体和期间，生成一份期初至期末的余额变动调节表。

## 结构

```
期初余额 (依据上期结账)                    X
  + 增加项 / 新增业务                      A
  + 本期计提金额                            B
  − 前期计提回转                           (C)
  − 付款 / 结算                            (D)
  ± 重分类 / 调整                          E
  ± 汇率折算                               F
期末余额 (依据期末总账)                    Y
```

## 逐行勾稽 (Tie each line)

- **期初** — 上期结账底稿，或上期末的总账 (GL) 余额。
- **各变动行** — 通过 `internal-gl` MCP 进行的总账查询（科目 + 日期范围 + 凭证来源过滤）。请务必引用查询。
- **期末** — 期间末日的总账 (GL) 余额。

调节表**必须验算平衡**：`X + A + B − C − D + E + F = Y`。如果不平衡，产生的差距即为未解释项 —— 应当直接反映，严禁强行“倒挤”拉平。

## 输出

输出一份包含“勾稽至”列的余额变动调节表，注明每一行所引用的总账查询或文档，并提供平账检查反馈（通过/失败，以及如有未解释差异的金额）。

> **💡 Appendix: 领域知识小贴士**
> 1. **余额变动调节 (Roll-forward)**：想象一下你的存钱罐，要把“月初剩的钱”变成“月底剩的钱”，你就得把中间每一笔存进去和花掉的钱都对上。在会计上，这就是为了证明账目确实“有迹可循”，没有凭空消失或多出来的钱。
> 2. **总账 (General Ledger / GL)**：它是公司的“终极大账本”，记录了公司所有的财务往来。我们可以把它看作是一个巨大的数据库，每一个细小的数字最终都会汇集到这里。
> 3. **计提与回转 (Accruals & Reversals)**：这是会计为了反映真实情况的小技巧。比如你本月用了电，但下月才交电费。本月为了准确，需要先把这笔没交的费“计提”出来。等到下月交了钱，再把这笔预留的记录“回转”掉，像拼图一样严丝合缝。
> 4. **勾稽 (Tying/Checking)**：这是一个专业的会计词汇，简单说就是“对账”。如果调节表上的数字和总账上的数字完全一致，我们就说它们“勾稽上了”。
> 5. **倒挤 (Plugging)**：如果账不平，新手可能会偷偷改一个数字让它强行变平，这就是“倒挤”。在专业财务中这是绝对不允许的，不平就是不平，必须找出原因。
