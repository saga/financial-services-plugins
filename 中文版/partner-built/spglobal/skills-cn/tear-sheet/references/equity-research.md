# 股权研究情况报告

## 目的
面向买方或卖方分析师评估投资的密集快照。强调估值、预测估计、财务轨迹和分析师共识。一切都支持或挑战投资论点。

**默认页面长度：** 1页。股权研究情况报告通常为单页——密度是关键。如果用户要求更多空间，可扩展至2页。

## 查询计划

从这些查询开始。如果结果不完整，分解为更窄的后续查询。

**查询1 — 公司概况+市场数据：**
"[公司] 公司概况 行业 板块 市值 企业价值 股价 52周高低价 流通股数 Beta"
→ 公司标题、业务描述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询2 — 历史财务数据：**
"[公司] 年度损益表 收入 毛利润 EBITDA 净利润 EPS 自由现金流 总债务 现金及等价物 过去4个财年"
→ 财务摘要（提取4年；显示3年；仅使用最早年份计算同比变化）
→ **立即写入** `/tmp/tear-sheet/financials.csv`

**查询2b — 细分（如果有）：**
"[公司] 按细分/业务单元分类的收入 过去2个财年"
→ 收入和细分分解（需要2年计算同比变化）。如果细分数据不可用，跳过此部分——不要留空表格。
→ **立即写入** `/tmp/tear-sheet/segments.csv`（如果没有细分数据则跳过）

**查询3 — 估值+共识：**
"[公司] 市盈率 EV/EBITDA EV/收入 估值倍数 共识 收入 EPS估计 分析师推荐 目标价格"
→ 估值快照、共识估计
→ **立即写入** 倍数到 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 估计到 `/tmp/tear-sheet/consensus.csv`

**查询4 — 财报：**
"[公司] 最新财报电话会议 要点 指引"
→ 财报要点
→ **立即写入** `/tmp/tear-sheet/earnings.txt`

**查询5 — 股票表现：**
"[公司] 股价 回报 1个月 3个月 6个月 1年 YTD表现"
→ 股票表现（不写入中间文件——数据直接写入文档）

**查询6（如果用户提供可比公司）：**
"[可比公司] 市值 EV/收入 EV/EBITDA 收入增长" — 每个可比公司重复
→ 可比公司背景，用于估值快照
→ **立即写入** `/tmp/tear-sheet/peer-comps.csv`

## 部分

按优先级列出。如果限制为一页，从底部裁剪。

### 1. 公司标题
紧凑的键值块，按照全局样式配置渲染为两列无边框表格。

左列：股票代码（交易所）、行业/板块、总部
右列：股价、52周区间、市值、企业价值、流通股数、Beta

### 2. 业务描述
2-3句紧凑的文字。**为分析师读者用自己的话重写**——不要逐字粘贴CIQ公司摘要。CIQ摘要是输入；输出应该是简洁且面向论点的。如果读者已经了解公司背景，就不需要重复行业基础知识。专注于正在发生的变化——战略转型、投资组合重塑、新的增长向量——而不是维基百科式的概述。例如，对于《财富》500强公司，不要在"向机构提供财务数据"上花费一句话。相反："正在将投资组合重塑为更高利润率的数据和分析业务，正在进行 Mobility 分拆，并通过其 Kensho 子公司积极投资 AI。"

对于不太知名的公司，在论点相关的动态之前可以简要说明"他们做什么"。

### 3. 估值快照
股权研究情况报告的核心。

| 指标 | 滞后 | 前瞻（NTM） |
|---|---|---|
| 市盈率 | | |
| EV/EBITDA | | |
| EV/收入 | | |
| P/自由现金流 | | |
| 股息收益率 | | |

**当前倍数可用时，前瞻倍数是必需的。** 当工具返回前瞻数据时，仅显示滞后倍数是一个重大差距——分析师基于前瞻盈利对公司估值。如果前瞻倍数不可用，仅显示滞后倍数并注明"前瞻估计不可用"。

如果用户提供了可比公司，添加每列可比公司（或如果有3+可比公司，则添加"可比公司中位数"列）。如果没有用户可比公司，且工具返回可比公司数据，添加可比公司中位数列。如果都没有，仅显示公司倍数。

### 4. 共识估计
华尔街的预期——这是此类情况报告的关键差异化因素。

**数据可用性说明：** 共识数据的深度因公司和覆盖范围而异。包括工具返回的内容——分析师数量、目标价格、买入/持有/卖出分布——但不要编造或估计任何共识数字。对于覆盖较少的公司，此部分可能仅包含收入和EPS估计，这仍然有价值。

| 指标 | FY[当年]估计 | FY[次年]估计 |
|---|---|---|
| 收入 | | |
| EPS（正常化） | | |
| EBITDA | | |

尽可能使用正常化/调整后EPS——GAAP EPS可能因一次性项目而扭曲，对前瞻估计不太有用。

在主估计表下方，如果工具返回了数据，则包含紧凑的分析师共识块：

| 分析师共识 | |
|---|---|
| 平均目标价格 | $XXX |
| 估计数量 | XX |
| 买入/持有/卖出 | XX / XX / XX |

如果目标价格或推荐数据不可用，省略分析师共识块，而不是显示空行。不要编造这些数字。

如果完全没有共识数据，跳过整个部分并注明"共识估计不可用。"不要估计。

### 5. 财务摘要（3年）
使用实际财年标签的密集单一表格。

| 指标（$M） | FY20XX | FY20XX | FY20XX |
|---|---|---|---|
| 收入 | | | |
| 收入增长% | | | |
| 毛利率% | | | |
| EBITDA | | | |
| EBITDA利润率% | | | |
| 净利润 | | | |
| EPS（摊薄） | | | |
| 自由现金流 | | | |
| 净债务 | | | |

从原始数据计算派生指标（增长%、利润率），而不是单独查询。

**资本结构（财务摘要下方的独立紧凑子表格）：**
不要将这些行附加到3年财务摘要表——先前年份列中的空白单元格看起来像缺失数据。相反，在下方渲染为独立的2列表格（指标 | 值），仅显示最近财年：

| 指标 | FY[最近] |
|---|---|
| 总债务 | $XXM |
| 现金及等价物 | $XXM |
| 净债务 | $XXM |

这与Corp Dev和IB/M&A情况报告中使用的资本结构格式一致。保持紧凑——财务摘要和此子表格之间不要有额外间距。

### 6. 收入和细分分解
如果工具返回了细分数据，包含紧凑的细分收入表。这对股权研究必不可少——分析师需要查看哪些业务线在推动增长。

| 细分 | 收入（$M） | 占总额% | 同比增长 |
|---|---|---|---|
| [细分A] | | | |
| [细分B] | | | |

一页纸保持紧凑：仅表格，无定性段落。如果细分数据不可用，跳过此部分——不要包含空白或占位符表格。

### 7. 最新财报要点
包括季度和日期（例如"2025财年Q3 — 2025年10月"）。3-4个要点，**从投资角度**——这个读者群需要细分层面的细节，而不是战略主题：

**如果提供了指引，优先列出。** 如果管理层发布了前瞻指引，将其作为第一个要点，并加粗**指引：**前缀（例如"**指引：** 2026财年EPS为$19.40-$19.65，意味着9-10%增长；有机收入增长6-8%。"）。指引是分析师最可操作的前瞻数据点——不要埋在运营要点中。

**超预期/低于预期背景（数据允许的情况下）：** 如果共识估计数据可用于报告期，将主要结果要点框定为超预期/低于预期："收入$X超预期/低于预期共识$Y Z%。"如果报告期的共识数据不可用，仅用绝对数字和增长率描述结果——不要编造共识数字。

然后：
- 细分层面表现：哪些业务线加速或减速
- 利润率轨迹：成本结构或盈利能力趋势的任何评论

归因于管理层："管理层强调..."或"CFO指出..."，而不是声明性声明。本部分应该感觉像财报摘要，而不是新闻稿总结。

### 8. 关键运营指标
2-3个行业相关KPI（如果有）。如果没有行业特定指标，从已检索的财务数据计算ROE和ROIC。可选——如果空间紧张则删除。

## 格式说明
- **这是最密集的情况报告类型。** 它应该比任何其他模板都明显更压缩。
- 财务摘要表：8pt文字（大小：16），行高240 DXA。紧凑间距最小。
- 估值快照和共识估计：8.5pt文字，占据视觉中心——分析师首先扫描这些。
- 正文：8.5pt（大小：17）——小于全局默认9pt。在一页纸上，每半点都很重要。
- 部分间距：最小化。章节标题前6pt（非全局12pt），标题后2pt。
- 突出显示任何显示明显拐点的指标（增长转正、利润率扩大/压缩）。
- 始终优先考虑信息而非空白。
