# 投资银行 / 并购情况报告

## 目的
交易背景下的公司概况——潜在目标、收购方或推销书入选。强调战略定位、细分细节、业务关系和并购活动。估值以交易倍数框架，而非投资论点。

**默认页面长度：** 1-2页。IB公司概况通常会延伸到第二页，尤其是当细分和并购数据丰富时。

## 查询计划

从这些查询开始。如果结果不完整，分解并跟进。

**查询1 — 概况+识别：**
"[公司] 业务描述 总部 成立年份 员工 行业 板块 所有权结构 主要股东"
→ 标题、业务概述
→ **立即写入** `/tmp/tear-sheet/company-profile.txt`

**查询2 — 细分+财务数据：**
"[公司] 按细分/业务单元分类的收入 过去2个财年 以及 年度损益表 收入 毛利润 EBITDA 营业收入 净利润 资本支出 自由现金流 总债务 现金及等价物 过去4个财年"
→ 细分分解（需要2年计算同比变化）、财务摘要（提取4年；显示3年；仅使用最早年份计算同比变化）
→ **立即写入** 原始财务数据到 `/tmp/tear-sheet/financials.csv`
→ **立即写入** 细分数据到 `/tmp/tear-sheet/segments.csv`（如果没有返回细分数据则跳过）

**查询3 — 估值+可比公司：**
如果用户提供了特定可比公司，查询每个：
"[可比公司] 企业价值 EV/收入 EV/EBITDA 收入增长 EBITDA利润率"
否则：
使用竞争对手工具识别3-5个公开同行，然后提取每个的EV/收入（NTM）、EV/EBITDA（NTM）、收入增长%、EBITDA利润率%。
还要查询："[公司] 企业价值 市值 EV/收入 EV/EBITDA 估值倍数"
→ 标题（EV/市值）、交易可比公司
→ **立即写入** 公司倍数到 `/tmp/tear-sheet/valuation.csv`
→ **立即写入** 同伴数据到 `/tmp/tear-sheet/peer-comps.csv`

**查询4 — 并购活动：**
"[公司] 收购 剥离 已完成交易 过去5年 交易价值"
→ 公司自己的交易
→ **立即写入** `/tmp/tear-sheet/ma-activity.csv`

**查询5 — 可比交易（数据允许）：**
使用查询1结果中的特定行业（例如"云可观测性软件并购交易"而非"科技并购"）。如果用户指定了可比公司，也尝试："[可比公司] 收购 交易倍数。"
→ 可比交易
→ **附加** 可比交易到 `/tmp/tear-sheet/ma-activity.csv`（添加`type=precedent`以区分公司自己的交易）

**查询6 — 关系+所有权：**
"[公司] 主要客户 供应商 合作伙伴 业务关系 机构所有权 内部人士所有权"
→ 业务关系、所有权（数据允许）
→ **立即写入** `/tmp/tear-sheet/relationships.txt`

## 部分

按优先级列出。见下面的"页面预算和裁剪顺序"获取明确的裁剪指导。

### 1. 公司标题
带交易相关标识符的键值块。

| 字段 | 备注 |
|---|---|
| 公司名称 | 大号、粗体 |
| 股票代码和交易所 | 如果是上市公司；私营公司则省略 |
| 总部位置 | 城市、州/国家 |
| 成立年份 | |
| 员工数 | 近似人数 |
| 市值/估值 | 如果是上市公司；如果是私营公司则为最新已知估值 |
| 企业价值 | |
| 所有权 | 公开/PE支持（注明赞助商）/家族/其他 |

对于私营公司：显著注明"私营公司"；跳过股票代码、交易所。

### 2. 业务概述
**这是推销书段落，不是CIQ摘要。** 不要粘贴数据工具中的公司描述。用推销书风格重写4-6句：
- 表征收入模型（经常性 vs 交易性、订阅 vs 许可）
- 说明规模化客户群（"为80%的《财富》500强银行提供服务"）
- 按名称识别竞争护城河（"专有的X数据集"、"唯一拥有Y的提供商"）
- 框定对买方或卖方重要的增长向量

语气应该自信和具体。阅读此段落的投资银行家应该立即了解公司的战略定位。

### 3. 收入和细分分解
按业务单元或产品线分类的收入。

| 细分 | 收入（$M） | 占总额% | 同比增长 |
|---|---|---|---|
| [细分A] | | | |
| [细分B] | | | |
| **合计** | | **100%** | |

**如果细分数据不可用：** 用2-3句话定性描述收入组合来替换表格（例如"收入主要来自订阅许可证（约70%）和专业服务（约30%）。"）。空白表格看起来像损坏——定性描述仍然传达有用信息。

### 4. 财务摘要（3年+最近十二个月）
列标题为实际财年。**如果有LTM（最近十二个月）数据可用，在最近财年右侧添加一列LTM列。** 标注为"LTM [季度结束日期]"（例如"LTM 2025年Q3"）。银行家使用LTM进行估值交叉检查——仅FY视图可能已过时6个月以上。

| 指标（$M） | FY20XX | FY20XX | FY20XX | LTM [日期] |
|---|---|---|---|---|
| 收入 | | | | |
| 收入增长% | | | | |
| 毛利润 | | | | |
| 毛利率% | | | | |
| EBITDA | | | | |
| EBITDA利润率% | | | | |
| 营业收入 | | | | |
| 净利润 | | | | |
| 资本支出 | | | | |
| 自由现金流 | | | | |
| FCF利润率% | | | | |

在并购背景下，利润率和FCF转换与绝对规模一样重要——买方评估的是效率 和现金生成潜力。

**资本结构**（财务摘要下方的紧凑子表格）：

| 指标 | 值 |
|---|---|
| 总债务 | $XXM |
| 现金及等价物 | $XXM |
| 净债务 | $XXM |
| 净债务/EBITDA | X.Xx |
| 标普信用评级 | XX（如有） |

从已检索的资产负债表数据中提取。这是任何银行家公司概况的标准——总债务、杠杆率和信用评级告诉买方融资情况。如果信用评级不可用，省略该行。

### 5. 交易可比公司
公司倍数**和运营指标**——银行家一起使用这些，而不是单独使用倍数。

**前瞻倍数是必需的。** 表格必须包含NTM倍数（如果工具返回）。当存在前瞻数据时，仅显示滞后倍数是重大差距——银行家基于前瞻盈利定价交易。

**期望有可比公司列。** 如果用户提供了可比公司，每个都有自己的列。如果没有，使用竞争对手工具识别3-5个公开同行，提取其倍数和运营指标，并包含。仅包含公司自身倍数的交易可比公司表，如果没有可比公司背景，对任何银行用途来说都是不完整的。

| 指标 | [公司] | [可比公司1] | [可比公司2] | [可比公司3] | 可比公司中位数 |
|---|---|---|---|---|---|
| EV/收入（NTM） | | | | | |
| EV/EBITDA（NTM） | | | | | |
| 收入增长% | | | | | |
| EBITDA利润率% | | | | | |

如果在对等查询后仅获得公司自身倍数，则显示这些并在表格下方列出选定的同行名称。

对于私营公司，跳过此部分。

### 6. 并购活动
两部分。**期望在此部分包含交易价值。** 银行家注意到它们何时缺失时会注意到。

**硬规则：如果并购工具返回交易价值，它必须出现在输出中。** 永远不要将已知交易价值降级为"未披露"——这是数据完整性违规。仅在工具真正对交易返回无值时使用"未披露"。

**a）公司自己的交易：**

**表格必须包含专用交易价值列。** 不要将交易价值合并到注释或理由列——银行家扫描数字，专用列可以实现这一点。

| 日期 | 目标/剥离 | 交易价值（$M） | 类型 | 理由 |
|---|---|---|---|---|
| | | | 收购/剥离 | |

如果未找到："过去5年无披露交易。"

**b）可比行业交易（数据允许）：**
使用早期识别的特定行业（例如"云基础设施软件并购"而非"科技并购"）。如果用户指定了可比公司，也查找涉及这些公司的交易。

| 日期 | 目标 | 收购方 | EV（$M） | EV/收入 | EV/EBITDA |
|---|---|---|---|---| |

如果工具不可用："可比交易数据不可用。"不要编造可比倍数。

### 7. 关键业务关系
利用S&P Capital IQ关系数据。按类型分组：

**客户：** 前3-5名指定客户
**供应商：** 关键供应商和技术提供商
**合作伙伴：** 战略联盟、分销、合资企业
**竞争对手：** 关键竞争名称

每个条目包括关系类型和带括号描述符解释关系性质（例如"AWS — 云基础设施（主要托管提供商）"、"摩根大通 — 客户（企业分析平台部署）"）。仅名称而没有上下文是没有用的。给本节喘息空间——它具有战略价值，大多数竞争情况报告都缺少它。

### 8. 所有权快照（数据允许——如果工具未返回任何内容则省略）
如果工具返回了机构所有权、内部人士所有权百分比或主要持有人，包含紧凑块：

主要机构持有人（如有）：按所有权百分比列出前3-5名。
内部人士所有权：总百分比。

对于PE支持的公司，注明赞助商、收购年份和所有权份额。所有权结构直接影响交易复杂性——广泛持有的上市公司、创始人控制的公司和PE支持的公司每种都需要根本不同的交易方式。

如果所有权数据未返回，省略整个部分而不是显示占位符。

**不要包含管理团队表。** 没有S&P Global工具返回高管数据——培训数据中的管理层名称会过时。

## 页面预算和裁剪顺序

**目标：最多2页。** IB概况可以使用两页，但要避免延伸到第三页。如果内容超过2页，按此顺序裁剪（移动到下一个之前完全裁剪每个）：

1. 所有权快照（第8节）——完全省略
2. 可比行业交易（第6b节）——保留公司自己的并购，裁剪可比交易
3. 业务关系（第7节）——压缩至最多3个竞争对手+每个供应商/合作伙伴/客户各2个，将描述符降级为仅括号
4. 资本结构（第4节）——将净债务/EBITDA合并为财务摘要中的单行
5. 交易可比公司（第5节）——减少至3个同行

永远不要裁剪：业务概述（2）、收入和细分分解（3）、财务摘要（4核心）、并购活动公司交易（6a）。

## 格式说明
- 业务概述和细分分解获得最大视觉权重。
- **并购活动是标志性部分：** 为并购表格标题使用强调色（#2E75B6）填充白色粗体文字——这是唯一使用与标准#D6E4F0不同标题颜色的表格。它应该在扫描时引人注目。
- **业务关系：空间允许时给予喘息空间。** 每个关系组之间6pt，每个条目全描述符。但是如果命中第3页，按上面的裁剪顺序开始压缩——本节价值高但也最可压缩。
- 对于私营公司，标题、业务概述、财务和关系形成核心；其他部分可能稀疏。
- **两页纪律：** 渲染前，估算页数。如果业务关系+并购活动合计将超过一页，按裁剪顺序开始裁剪——本节价值高但也最可压缩。
