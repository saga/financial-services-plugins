# 财报预览

description: 在财报发布前准备买方/卖方分析师的预览——包括共识估计、历史超预期/低于预期、关键风险和机会、预期细分数据、估值影响和交易建议。在季度财报发布前15-30天使用，或者在财报发布后但在完整纪要和模型更新之前使用。对于经历重大变化的公司的重要财报（首次盈利、新产品发布后、重大指引变化后），价值最高。当用户提到"即将发布的财报"、"预览[公司]财报"、"[公司]下周发布"、"预期"、"财报预览"或" Earnings preview"时使用。

## 工作流程

### 第1步：验证上下文

确认是否已有公司上下文。如果用户说"苹果"而不是"AAPL"，询问或使用工具查找正确的股票代码。

检查用户是否指定了：
- **公司**（必需）
- **财报日期或季度**（例如"下周的财报"或"Q3"）
- **用户是否有特殊关注点**（例如"关注指引"或"iPhone销量"）

如果缺少日期/季度，使用工具查找下一个预定财报日期。

### 第2步：收集基本信息

运行并行查询收集：

**查询1 — 基本信息：**
"[公司] 股票代码 板块 市值 股价 52周区间"
→ 公司标识符

**查询2 — 共识估计：**
"[公司] 共识估计 收入 EPS EBITDA 下季度 华尔街估计"
→ 共识估计快照

**查询3 — 近期股价表现：**
"[公司] 股价 1个月 3个月 YTD 表现"
→ 近期股价走势

**查询4 — 历史超预期/低于预期（最近4-6个季度）：**
"[公司] 财报 超预期 低于预期 意外 过去4个季度"
→ 历史超预期/低于预期模式

**查询5 — 近期发展：**
"[公司] 最新新闻 近期发展 重大公告 过去60-90天"
→ 可能影响财报的近期事件

### 第3步：收集财务数据

**查询6 — 历史财务数据（最近4个季度）：**
"[公司] 季度损益表 收入 EBITDA 净利润 EPS 过去4个季度"
→ 历史财务数据，用于与共识比较

**查询7 — 上一季度指引（如果适用）：**
"[公司] 指引 上一季度指引 管理层指引 收入 EPS 下一季度"
→ 管理层上一季度发布的指引

**查询8 — 细分数据（如果有）：**
"[公司] 细分收入 业务单元 产品线 过去4个季度"
→ 预期的细分数据

### 第4步：构建预览

使用以下框架——跳过任何数据不可用的部分。

#### 标题

```
[公司名称]（[股票代码]）| [季度]FY[年份] 财报预览
财报日期： [日期]
共识预期： 收入 [$X.X亿] | EPS [$X.XX] | EBITDA [$XXXM]
```

#### 预期 vs 共识

| 指标 | 共识估计 | 上一季度实际 | 同比变化 |
|---|---|---|---|
| 收入 | | | |
| EBITDA | | | |
| 净利润 | | | |
| EPS | | | |

如果无法获得共识或上一季度实际数据，用2-3句话定性描述预期。

#### 超预期/低于预期历史

**仅在数据可用时包含。** 过去的超预期/低于预期模式可以预测未来结果。

列出最近4个季度，每行格式：
- **FQ X [年份]：[结果]。** [超预期/低于预期的简短描述。]

如果没有历史超预期/低于预期数据，跳过此部分——不要编造。

#### 关键风险和机会

这是预览的价值所在——前瞻性地框定风险和机会。

**可能超预期的领域：**
列出2-3个可能超出共识的因素：
- 产品发布后的需求信号
- 成本削减举措
- 可能有利的季节性
- 细分或地区表现

**可能低于预期的领域：**
列出2-3个可能低于共识的因素：
- 宏观逆风
- 竞争压力
- 供应链问题
- 指引下调的早期信号

**仅在有具体数据支持时才框定为风险/机会。** 如果没有具体信号，用"观察"而不是"预期"。

#### 预期细分数据（如果有）

如果工具返回了细分数据：

| 细分 | 共识预期 | 同比 | 备注 |
|---|---|---|---|
| [细分A] | | | |
| [细分B] | | | |

一页纸保持紧凑——仅表格。如果细分数据不可用，跳过此部分。

#### 估值影响

**基于当前估值水平和近期股价走势的简要评估。**

如果有分析师目标价和当前股价：
```
目标价： $[XX] | 当前价：$[XX] | 上涨/下跌空间：XX%
```

如果有历史估值倍数：
```
当前NTM EV/EBITDA：[Xx] | 历史范围：[Xx - Xx]
```

如果这些数据不可用，用1-2句话定性描述估值状况。

#### 交易建议

**这是预览的结尾和最重要的部分。** 给出可操作的交易建议。

使用以下框架之一——选择最合适的：

**财报前交易：**
- "财报后股价可能波动。建议在财报前[做多/做空/观望]。关键催化剂是[具体触发因素]。"

**方向性赌注：**
- "预计超预期。[建议做多/做空]，[目标价位]，止损[价位]。"

**事件驱动：**
- "如果指引上调/下调超过X%，预计股价将移动X%。建议[具体策略]。"

**如果数据不足以给出建议：**
- "数据不足以给出有信心的交易建议。在财报前观望。"

永远不要说"一切取决于财报"——这是显而易见的。给出具体的、可执行的建议。

#### 近期发展（仅相关）

仅包含可能影响财报结果的近期事件。如果90天内没有任何相关新闻，跳过此部分。

格式：
- **催化剂/事件：** [描述] — [预期影响]

#### 分析师覆盖和目标价

如果有数据：

| 分析师 | 评级 | 目标价 | 日期 |
|---|---|---|---|
| | | | |

如果超过5-6个分析师，用前5-6个或省略此表。

如果数据不可用，跳过此部分——不要显示空行。

---

## 部分优先级和裁剪

**目标：1-2页。**

按此顺序裁剪（首先裁剪底部的）：

1. 交易建议（永远不裁剪）
2. 预期 vs 共识（永远不裁剪）
3. 关键风险和机会（永远不裁剪）
4. 超预期/低于预期历史（如果数据稀疏则裁剪）
5. 估值影响（如果数据稀疏则裁剪）
6. 分析师覆盖（如果超过5-6个分析师，裁剪到5-6个）
7. 近期发展（如果超过3-4个事件，裁剪到3-4个）
8. 预期细分数据（如果超过5-6个细分，裁剪到前5-6个）

**永远不要完全跳过交易建议、预期 vs 共识或关键风险和机会。**

## 格式说明
- 这是分析师预览——密度很重要，但不如股权研究情况报告那么压缩。
- 一页半是理想的；如果内容超过两页，按上述顺序裁剪。
- 使用加粗和颜色来突出关键数字和预期。
- 在标题中突出显示财报日期——读者需要立即知道何时发布。
- 交易建议应该是最后一部分，并以粗体或框架突出显示，使其可快速扫描。
